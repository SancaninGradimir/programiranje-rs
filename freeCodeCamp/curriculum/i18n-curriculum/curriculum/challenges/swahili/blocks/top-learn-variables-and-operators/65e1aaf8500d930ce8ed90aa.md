# --description--

Sumator `+` postoji u dve vrste: binarna vrsta koju ste koristili u prethodnom izazovu i unarni tip.

Unarni operator, ili rečima, operator `+` koji se koristi samo za jednu vrednost, ništa ne radi sa brojem. Ali ako operand nije broj, unarni operator ga konvertuje u broj.

Na primer:

```js
// No effect on numbers
let x = 1;
console.log( +x ); // 1

let y = -2;
console.log( +y ); // -2

// Converts non-numbers
console.log( +true ); // 1
console.log( +"" );   //
```

Potreba za konverzijom sekvence slova u brojeve veoma je česta. Na primer, ako dobijate vrednost iz polja forme za HTML, to je obično sekvenca slova. Da li želite da ih dodate?

Binarni inkrementator bi dodao kao sekvencu karaktera:

```js
let apples = "2";
let oranges = "3";

alert( apples + oranges ); // "23", the binary plus concatenates strings
```

Ako želiš da ih koristiš kao brojeve, moraš da ih promeniš i zatim dodaš:

```js
let apples = "2";
let oranges = "3";

// both values converted to numbers before the binary plus
alert( +apples + +oranges ); // 5

// the longer variant
// alert( Number(apples) + Number(oranges) ); // 5
```

Iz perspektive matematičara, obilje operatora može izgledati čudno. Ali iz perspektive programera, nema ništa posebno: unary operateri se koriste prvo, konvertuju niz karaktera u brojeve, a zatim binarni operater dodaje te vrednosti.

Zašto se unarni operateri koriste za vrednosti pre binarnih? Kao što vidite, to je zbog njihovog višeg prioriteta.

# --questions--

## --text--
Prema opisu unarnih i binarnih inkrementera `+` u JavaScript, možete tačno dodati sledeće sekvence znakova `'2'` i `'3'` kao brojeve koristeći unarni inkrementer `+`?

## --answers--

`console.log('2' + '3'); // Outputs: '23'`

---

`console.log(+2 + +3); // Outputs: 5`

---

`console.log(+'2' + +'3'); // Outputs: 5`

---

`console.log(Number('2') + Number('3')); // Outputs: '23'`
