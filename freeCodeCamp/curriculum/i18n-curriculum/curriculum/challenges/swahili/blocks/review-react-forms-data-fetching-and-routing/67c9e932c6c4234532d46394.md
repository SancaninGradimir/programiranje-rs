## Rad sa formularima u React

- **Kontrolisani polja za unos**: Ovo je mesto gde čuvate vrednost polja za unos u *state* i ažurirate ga putem događaja `onChange`. Ovo vam daje potpunu kontrolu nad podacima forme i omogućava trenutnu validaciju i prikaz uslova.

```jsx
import { useState } from "react";

function App() {
  const [name, setName] = useState("");

  const handleChange = (e) => {
    setName(e.target.value);
  };

  const handleSubmit = (e) => {
    e.preventDefault();
    console.log(name);
  };

  return (
    <>
      <form onSubmit={handleSubmit}>
        <label htmlFor="name">Your name</label> <br />
        <input value={name} id="name" onChange={handleChange} type="text" />
        <button type="submit">Submit</button>
      </form>
    </>
  );
}

export default App;
```

- **Nekontrolisane ulazne komponente**: Umesto rukovanja ulaznim delovima putem veze `useState`, nekontrolisane komponente u HTML čuvaju svoja interna stanja uz pomoć DOM. Pošto DOM kontroliše vrednosti unosa, potrebno je dohvatiti vrednosti ulaznih delova za `ref`.

```jsx
import { useRef } from "react";

function App() {
 const nameRef = useRef();

 const handleSubmit = (e) => {
   e.preventDefault();
   console.log(nameRef.current.value);
 };

 return (
   <form onSubmit={handleSubmit}>
     <label htmlFor="name">Your</label>{" "}
     <input type="text" ref={nameRef} id="name" />
     <button type="submit">Submit</button>
   </form>
 );
}

export default App;
```

## Rad sa vezom za `useActionState`

- **Akcije na serveru**: Ovo su funkcije koje se izvršavaju na serveru kako bi omogućile obradu formulara direktno na serveru bez potrebe za krajnjom tačkom (endpoint) API. Evo primera iz Next.js aplikacije:

```js
"use server";

async function submitForm(formData) {
 const name = formData.get("name");
 return { message: `Hello, ${name}!` };
}
```

Komanda za `"user server"` prikazuje funkciju kao akciju servera.

- **Kiungo cha `useActionState`**: Ova veza ažurira stanje na osnovu rezultata slanja forme. Ovde je osnovna sintaksa za komponentu `useActionState`:

```js
const [state, action, isPending] = useActionState(actionFunction, initialState, permalink);
```

- `state` je trenutno stanje koje akcija vraća.
- `action` je funkcija koja pokreće server akciju.
- `isPending` je vrednost tačno ili netačno koja pokazuje da li akcija teče.
- Parametar `actionFunction` je sama server akcija.
- `initialState` je parametar koji pokazuje početno stanje pre nego što akcija počne.
- `permalink` je opciona niz znakova koja pokazuje URL jedinstveni identifikator stranice koju obrazac menja.

## Dostupnost podataka u React

- **Opcije za dohvaćanje podataka**: Postoji mnogo različitih načina za dohvaćanje podataka u React. Možete koristiti naivni (native) Fetch API, ili alate trećih strana kao što su Axios ili SWR.
- **Često korišćene promene stanja prilikom dohvaćanja podataka**: Bez obzira na metod koji koristite za dohvaćanje podataka u React, postoje delovi stanja koje ćete morati pratiti. Prvo su sami podaci. Drugo je praćenje da li su podaci i dalje dostupni. Treće je stanje koje će držati sve greške koje mogu nastupiti tokom procesa dohvaćanja podataka.

```js
const [data, setData] = useState(null);
const [loading, setLoading] = useState(true);
const [error, setError] = useState(null);
```

Pošto je dostupnost podataka sporedni efekat, bolje je koristiti `Fetch API` unutar veze `useEffect`.

```js
useEffect(() => {
  const fetchData = async () => {
    try {
      const res = await fetch("https://jsonplaceholder.typicode.com/posts");
      
      if (!res.ok) {
        throw new Error("Network response was not ok");
      }

      const data = await res.json();
      setData(data);
    } catch (err) {
      setError(err);
    } finally {
      setLoading(false);
    }
  };

  fetchData();
}, []);
```

Zatim možete prikazati poruku učitavanja ako nije potpuna dostupnost podataka, poruku greške ako je došlo do greške prilikom dohvaćanja podataka, ili rezultate.

```jsx
if (loading) {
  return <p>Loading...</p>;
}

if (error) {
  return <p>{error.message}</p>;
}

return (
  <ul>
    {data.map((post) => (
      <li key={post.id}>{post.title}</li>
    ))}
  </ul>
);
```

Ako želite koristiti Axios, morate ga instalirati i uvesti:

```bash
npm i axios
```

```js
import axios from "axios";
```

Zatim možete dobiti podatke koristeći `axios.get`:

```jsx
const [data, setData] = useState(null);
const [loading, setLoading] = useState(true);
const [error, setError] = useState(null);

useEffect(() => {
  const fetchData = async () => {
    try {
      const res = await axios.get(
        "https://jsonplaceholder.typicode.com/users"
      );
      setData(res.data);
    } catch (err) {
      setError(err);
    } finally {
      setLoading(false);
    }
  };

  fetchData();
}, []);
```

Da biste dobili podatke koristeći link `useSWR`, prvo morate da ga instalirate i dovedete.

```bash
npm i swr
```

```js
import useSWR from "swr";
```

Evo kako možete koristiti link da biste dobili podatke:

```jsx
import useSWR from "swr";

const fetcher = (url) => fetch(url).then((res) => res.json());

const FetchTodos = () => {
 const { data, error } = useSWR(
   "https://jsonplaceholder.typicode.com/todos",
   fetcher
 );

 if (!data) {
   return <h2>Loading...</h2>;
 }
 if (error) {
   return <h2>Error: {error.message}</h2>;
 }

 return (
   <>
     <h2>Todos</h2>
     <div>
       {data.map((todo) => (
         <h3 key={todo.id}>{todo.title}</h3>
       ))}
     </div>
   </>
 );
};

export default FetchTodos;
```

## Rad sa vezom `useOptimistic`

- **Povez za `useOptimistic`**: Ovaj mehanizam se koristi za održavanje interfejsa korisnika (UI) aktivnim dok čeka da akcija async bude završena u pozadini. Pomaže u upravljanju "optimističnim ažuriranjima" unutar UI-a, tehnika koja pruža trenutne ažuracije interfejsu na osnovu očekivanog rezultata akcije, kao što je čekanje odgovora sa servera.

Evo osnovne sintakse:

```js
const [optimisticState, addOptimistic] = useOptimistic(actualState, updateFunction);
```

- `optimisticState` je privremeno stanje koje poboljšava trenutno za bolji korisnički doživljaj.
- `addOptimistic` je funkcija koja izvršava optimističko ažuriranje pre stvarnog menjanja stanja.
- `actualState` je stvarna vrednost stanja koja proizlazi iz rezultata akcije, kao što je dobijanje podataka sa servera.
- `updateFunction` je funkcija koja određuje kako bi optimistično stanje trebalo da se ažurira kada se pozove.

Ovo je primer korišćenja veze za `useOptimistic` u sekciji `TaskList`:

```jsx
"use client";

import { useOptimistic } from "react";

export default function TaskList({ tasks, addTask }) {
  const [optimisticTasks, addOptimisticTask] = useOptimistic(
    tasks,
    (state, newTask) => [...state, { text: newTask, pending: true }]
  );

  async function handleSubmit(e) {
    e.preventDefault();
    const formData = new FormData(e.target);

    addOptimisticTask(formData.get("task"));

    addTask(formData);
    e.target.reset();
  }

  return <>{/* UI */}</>;
}
```

- **`startTransition`**: Ovo se koristi za prikaz dela interfejsa (UI) i prikaz ažuriranja stanja kao nekritične promene. To omogućava da se UI promeni tokom kompleksnih ažuriranja. Evo osnovne sintakse:

```js
startTransition(action);
```

`action` izvršava ažuriranje stanja ili inicijalizuje logiku vezanu za nehitne promene. Ovo osigurava da hitna ažuriranja korisničkog interfejsa (kao što je kucanje ili klikanje) ne budu blokirana.

## Rad sa vezom za `useMemo`

- **Memorizacija**: Ovo je tehnika optimizacije gde su rezultati poziva skupih funkcija sačuvani (zapamćeni) na osnovu specifičnih argumenata. Kada se isti argumenti ponovo pozovu, vraćaju se sačuvani rezultati umesto da se funkcija ponovo izračunava.
- **Poveznik `useMemo`**: Ovaj veznik se koristi za skladištenje izračunatih vrednosti. Evo primera čuvanja rezultata sortiranja velikog niza podataka. `expensiveSortFunction` će se izvršiti samo kada `largeArray` promeni:

```js
const memoizedSortedArray = useMemo(
  () => expensiveSortFunction(largeArray),
  [largeArray]
);
```

## Alat za upravljanje zavisnostima

- **Definicija zavisnosti**: U programu, zavisnost je kada jedan deo ili modul u programu zavisi od drugog kako bi pravilno funkcionisao. Zavisnost je uobičajena u programima jer omogućava programerima da koriste funkcije ili alate koje su prethodno napravili drugi. Dve glavne zavisnosti potrebne za projekat React biće paketi `react` i `react-dom`:

```json
"dependencies": {
  "react": "^18.3.1",
  "react-dom": "^18.3.1"
}
```

- **Definicija menadžera paketa**: Da biste upravljali zavisnostima softvera u projektu, potrebno je da koristite menadžer paketa. Menadžer paketa je alat koji se koristi za instalaciju, ažuriranje i uklanjanje zavisnosti. Mnogi popularni programski jezici kao JavaScript, Python, Ruby i Java, koriste menadžere paketa. Popularni menadžeri paketa za JavaScript su npm, Yarn i pnpm.
- **Datoteka `package.json`**: Ovo je ključna konfiguraciona datoteka u projektima koja sadrži metadata o vašem projektu, uključujući ime, verziju i zavisnosti. Takođe opisuje skripte, informacije o licenci i druge postavke koje pomažu u upravljanju projektom i njegovim zavisnostima.
- **Datoteka `package-lock.json`**: Ova datoteka će zaključati tačnu verziju svih paketa korišćenih u vašem projektu. Kada ažurirate paket, nova verzija će biti ažurirana i u ovoj fajl za zaključavanje.
- **Direktorijum `node_modules`**: Ovaj direktorijum sadrži tačan kod zavisnosti naveden u vašoj datoteci `package.json`, uključujući direktne zavisnosti vašeg projekta i zavisnosti tih zavisnosti.
- **Zavisnosti za razvoj**: Ovo su paketi koji se koriste samo za razvoj, a ne za produkciju. Primer je biblioteka za testiranje kao što je Jest. Instaliraćete Jest kao zavisnost za razvoj jer vam je potreban za interno testiranje vaše aplikacije, ali nije potreban za pokretanje aplikacije u produkciji.

## Planer puta za React

- **Uvod**: React Router je biblioteka treće strane koja omogućava dodavanje rutiranja u React aplikacije. Da bi počeo, potrebno je da instaliraš React Router u postojeći React projekat na sledeći način:

```bash
npm i react-router
```

Zatim unutar fajla `main.jsx` ili `index.jsx`, biće vam potrebno da postavite strukturu putanje kao što sledi:

```jsx
import { StrictMode } from "react";
import { createRoot } from "react-dom/client";
import { BrowserRouter, Routes, Route } from "react-router";
import App from "./App.jsx";

import "./index.css";

createRoot(document.getElementById("root")).render(
  <StrictMode>
    <BrowserRouter>
      <Routes>
        <Route path="/" element={<App />} />
      </Routes>
    </BrowserRouter>
  </StrictMode>
);
```

`path` i `element` služe za povezivanje URL sa UI sekcijom. U ovom slučaju, postavljamo putanju za početnu stranicu koja se odnosi na sekciju `App`.

- **Mnogi komentari i konfiguracija putanja**: Uobičajeno je u velikim aplikacijama da imaju mnogo komentara i putanja organizovane na sledeći način:

```jsx
<Routes>
  <Route index element={<Home />} />
  <Route path="about" element={<About />} />

  <Route path="products">
    <Route index element={<ProductsHome />} />
    <Route path=":category" element={<Category />} />
    <Route path=":category/:productId" element={<ProductDetail />} />
    <Route path="trending" element={<Trending />} />
  </Route>
</Routes>
```

Parametri `index` u ovim primerima namijenjeni su da predstavljaju osnovni put za određeni dio puta. Stoga, deo `Home` će biti prikazan u korijenskom putu `/` kada je deo `ProductsHome` prikazan u putu `/products`.

- **Grupisanje unutar putanje**: Možete grupisati putanju unutar druge putanje, što uzrokuje dodavanje putanje deteta na putanju roditelja.

```jsx
<Route path="products">
  <Route path="trending" element={<Trending />} />
</Route>
```

U gornjem primeru, putanja za popularne proizvode će biti products/trending.

- **Promenljivi delovi**: Promenljivi deo je tamo gde bilo koji deo putanje URL se menja.

```jsx
<Route path=":category" element={<Category />} />
```

U ovom primeru imamo promenljivi deo nazvan `category`. Kada korisnik klikne URL kao products/brass-instruments, onda se sadržaj menja na sekciju `Category` i možete dobiti relevante podatke putem tog promenljivog dela.

- **Poveznica za `useParams`**: Ova poveznica se koristi za dobijanje promenljivih parametara iz putanje URL.

```jsx
import { useParams } from "react-router";

export default function Category() {
  let params = useParams();
  {/* Accessing the category param: params.category */}
  {/* rest of code goes here */}
}
```

## Sistemi za React

- **Uvod**: Sistemi React pružaju funkcije kao što su rutiranje, optimizacija slika, pristup podacima, verifikacija identiteta i još mnogo toga. To znači da možda nećete morati da postavljate zasebne frontend i backend aplikacije za neke slučajeve upotrebe. Primeri sistema React su Next.js i Remix.
- **Rutiranje u Next.js**: Ovaj okvir za rutiranje uključuje podršku za dinamičke rute, paralelne rute, rukovače ruta (route handlers), preusmeravanja, internacionalizaciju i više.

Ovo je primer kako kreirati prilagođeni rukovatelja zahteva:

```js
export async function GET() {
  const res = await fetch("https://example-api.com");
  const data = await res.json();

  return Response.json({ data });
}
```

- **Optimizacija slika u Next.js-u**: Sekcija `Image` dodaje izvornu funkcionalnost HTML od `img`, omogućavajući brzo učitavanje stranica i optimizaciju veličine. To znači da će slike biti učitane tek kada uđu u vidokrug (viewport), a sekcija `Image` direktno serviratiće slike odgovarajuće veličine za svaki uređaj.

```jsx
import Image from "next/image";

export default function Page() {
  return (
    <Image src="link-to-image-goes-here" alt="descriptive-title-goes-here" />
  );
}
``` 

## Prosleđivanje svojstava detijskim komponentama (Prop drilling)

- **Definicija**: Prenos parametara je proces prenošenja parametara iz roditeljske regije ka duboko lokalisovanim regionima potomstva, čak i kada određeni regioni potomstva ne zahtevaju te parametre.

## Upravljanje stanjem

- **API konteksta**: Kontekst se odnosi na situaciju kada rodenski deo generiše informacije za dete delove bez potrebe za direktnim prosleđivanjem kroz parametre. `createContext` koristi se za kreiranje objekta konteksta koji predstavlja kontekst koji će drugi delovi čitati. `Provider` koristi se za prosleđivanje vrednosti konteksta detetovim delovima.

```jsx
import { useState, createContext } from "react";

const CounterContext = createContext();

const CounterProvider = ({ children }) => {
  const [count, setCount] = useState(0);

  return (
    <CounterContext.Provider value={{ count, setCount }}>
      {children}
    </CounterContext.Provider>
  );
};

export { CounterContext, CounterProvider };
```

- **Redux**: Redux upravlja stanjem (state) pružajući centralno skladište i strogu kontrolu nad ažuriranjima stanja. Koristi predvidljiv obrazac akcija (actions), reducersa i middleware-a. Akcije su opterećenje podataka koje šalju podatke iz vaše aplikacije u Redux skladište, što se često dešava usled interakcije korisnika. Reducersi su funkcije koje definišu kako bi stanje trebalo da se promeni u skladu sa tim akcijama, osiguravajući da se stanje ažurira nepromenljivo (immutably). Middleware, s druge strane, služi kao most između slanja akcija i reducer-a, omogućavajući vam da dodate funkcionalnost Redux-u (npr., logovanje, rukovanje akcijama async) bez menjanja osnovnog toka.

- **Zustand**: Ovo rešenje za upravljanje stanjem je odlično za male do srednje aplikacije. Radi korišćenjem hook-a ``useStore`` za direktno dobijanje stanja (state) u komponentama i stranicama. Ovo vam omogućava da menjate i dohvatate podatke bez potrebe za akcijama, reducerima ili provider-om.
