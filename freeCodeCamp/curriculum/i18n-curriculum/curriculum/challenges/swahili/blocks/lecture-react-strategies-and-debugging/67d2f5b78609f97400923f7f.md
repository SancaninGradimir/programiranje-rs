# --description--

Kako raste vaš program, upravljanje načinom na koji podaci teku između komponenti može biti izazovno.

Kada počnete, hook ya `useState` ya React to može biti dovoljno, ali kada dodate komponente, možete naići na probleme kao što su:

- Prosleđivanje parametara kroz komponente koje ih ne zahtevaju, poznato i kao prop drilling
- Održavanje konzistentnosti podataka u različitim delovima vaše aplikacije
- Rešavanje složenih ažuriranja koja utiču na više delova istovremeno

Iste tako, mogu nastati i drugi izazovi, što dovodi do koda koji je težak za održavanje, debagovanje i testiranje. Ovde dolaze biblioteke za upravljanje stanjem – one pružaju centralno mesto gde delovi mogu dobiti ili ažurirati podatke kojima treba.

Istražite nekoliko različitih opcija upravljanja stanjem koje imate i liniju kako ih koristiti.

Context API je menadžer stanja napravljen unutar React koji omogućava deljenje stanja između komponenti bez korišćenja biblioteke treće strane. To je poboljšanje bazirano na hook-u `useState`, pa je savršeno za slučajeve kao što su promena tema ili stanje autentifikacije identiteta korisnika.

Međutim, Context API nije pogodan za česta ažuriranja i može izazvati nepotrebna ponovna renderovanja, zbog čega nije idealan za složeno upravljanje stanjem u aplikacijama kao što su e-trgovina i društvene mreže.

Evo jednostavnog brojača koji pokazuje osnovnu upotrebu Context API-ja:

```jsx
import { useState, createContext } from 'react';

const CounterContext = createContext();

const CounterProvider = ({ children }) => {
  const [count, setCount] = useState(0);

  return (
    <CounterContext.Provider value={{ count, setCount }}>
      {children}
    </CounterContext.Provider>
  );
};

export { CounterContext, CounterProvider };
```

Ovaj kod kreira context i provider za deljenje stanja `count` kroz celu aplikaciju.

`CounterProvider` koristi hook `useState` za inicijalizaciju i upravljanje stanjem `count` i njegovom funkcijom za ažuriranje. Obe vrednosti se prosleđuju komponentama potomcima putem `Provider`-a.

Kada celu aplikaciju obaviješ komponentom `CounterProvider`, stanje `count` postaje dostupno svuda u aplikaciji.

Ovo je kako možete navigirati `CounterProvider` u vašoj aplikaciji:

```jsx
import { CounterProvider } from './context/CounterContext';

function App() {
  return (
    <CounterProvider>
        {/* App components */}
    </CounterProvider>
  );
}

export default App;
```

Ovo je kako možete koristiti stanje `count`:

```jsx
import React, { useContext } from 'react';
import { CounterContext } from '../context/CounterContext';

const Counter = () => {
  const { count, setCount } = useContext(CounterContext);

  return (
    <>
      <div style={{ textAlign: 'center' }}>
        <h1>Context API Counter</h1>
        <button style={{ marginRight: '5px' }} onClick={() => setCount(count - 1)}>
          Decrease
        </button>
        <span>{count}</span>
        <button style={{ marginLeft: '5px' }} onClick={() => setCount(count + 1)}>
          Increase
        </button>
      </div>
    </>
  );
};

export default Counter;
```

Kao što možete videti, stanje `count` na njegovom uređaju za podešavanje, `setCount`, je inicijalizovano putem funkcije `useContext`.

Trenutni status za `count` se prikazuje, a `setCount` se koristi za povećanje i smanjenje stanja `count` kada korisnik klikne na odgovarajuće dugmad za smanjivanje i povećavanje.

Još jedna popularna biblioteka za upravljanje stanjem je Redux, koja je jedna od najpoznatijih biblioteka za upravljanje stanjima korišćenjem React. Koristi se već dugo i odlična je za velike aplikacije kao što su e-trgovina (eCommerce), platforme društvenih mreža, forumi za diskusiju, itd.

Redux upravlja upravljanjem stanjem pružajući centralno skladište i strogu kontrolu nad ažuriranjima stanja. Koristi predvidljiv obrazac akcija, reducersa i middleware-a.

Akcije su paket informacija koji šalje podatke iz vaše aplikacije u Redux skladište, što se često dešava usled interakcije korisnika.

Reducers su funkcije koje opisuju kako bi se stanje trebalo promeniti kao odgovor na te akcije, osiguravajući da je stanje ažurirano nepromenljivo.

Middleware, s druge strane, funkcioniše kao most između dispatch-ovanja akcija i reducera, omogućavajući vam da dodate funkcionalnost Reduxu (na primer, logovanje, rukovanje asinhronim akcijama) bez menjanja osnovnog toka.

Stalni prigovori oko Reduxa predstavljaju veliki obim osnovnog koda (*boilerplate*) neophodnog za početak. Kao odgovor, tim Reduxa lansirao je "Redux Toolkit" i "RTK Query", koji značajno pojednostavljuju proces konfiguracije.

Obično definiš sve akcije i redusere u jednom fajlu koristeći funkciju `createSlice()`. Uobičajeno je pozvati taj fajl nakon završetka sa rečju `Slice`, na primer, `productSlice`, `userSlice`, `counterSlice`, itd.

Evo fajla `counterSlice` da vam pokaže osnove:

```jsx
import { createSlice } from '@reduxjs/toolkit';

const counterSlice = createSlice({
  name: 'counter',

  initialState: { count: 0 },

  reducers: {
    increment: (state) => {
      state.count += 1;
    },
    decrement: (state) => {
      state.count -= 1;
    },
  },
});

export const { increment, decrement } = counterSlice.actions;

export default counterSlice.reducer;
```

Odavde, moraš da prođeš kroz ceo program sa `Provider`, izabere deo stanja (state) iz slice-a za `useSelector()`, zatim koristi `useDispatch()` da učiniš stanje aktivnim.

Još jedna opcija koju treba uzeti u obzir je Zustand.

Zustand je lagana biblioteka za upravljanje stanjem sa API lakoćom. Bazira se na hookovima, pa postoji mali osnovni strukturalni kod u poređenju sa Redux-om, što ga čini lakim i brzim za početak.

Zustand je odličan za male do srednje aplikacije. Radi korišćenjem hook-a `useStore` da bi dobio stanje direktno u komponentama i stranicama. Ovo vam omogućava da menjate i dobijate podatke bez potrebe za akcijama, reducerima ili providerima.

Ovde je `useCounterStore` koja izvršava drugu funkciju za računanje:

```jsx
import { create } from 'zustand';

const useCounterStore = create((set) => ({
  count: 0,
  increment: () => set((state) => ({ count: state.count + 1 })),
  decrement: () => set((state) => ({ count: state.count - 1 })),
}));

export default useCounterStore;
```

I ovo je kako možete inicijalizovati i koristiti stanja u vašoj aplikaciji:

```jsx
// Import the useCounterStore (it's just a hook)
import useCounterStore from '../useCounterStore';

const Counter = () => {
  // Initialize the states with the useCounterStore hook
  const { count, increment, decrement } = useCounterStore();

  return (
    <>
      <div style={{ textAlign: 'center' }}>
        <h1>Zustand Counter</h1>
        <button style={{ marginRight: '5px' }} onClick={() => decrement()}>
          Decrease
        </button>
        <span>{count}</span>
        <button style={{ marginLeft: '5px' }} onClick={() => increment()}>
          Increase
        </button>
      </div>
    </>
  );
};

export default Counter;
```
