# --interactive--

Petlja `for...in` je najkorisnija kada treba da prolaziš kroz svojstva objekta. Ona prolazi kroz sva nabrojiva svojstva objekta, uključujući nasleđena svojstva iz prototype lanca i svojstva koja nisu numerička.

Dobijeno svojstvo je svojstvo koje se može dobiti iz lanca prototipa objekta. Nenumeričko svojstvo je svojstvo koje nije broj ili niz slova koji se može konvertovati u broj.

Evo osnovne sintakse `for...in` petlje:

```js
for (variable in object) {
  // code block to be executed
}
```

Parametar u primeru predstavlja trenutni atribut ponovnog objekta.

Istražite nekoliko primera ili razumevanje kako radi ciklus `for...in`.

U ovom prvom primeru imamo nešto sa `fruit` i želimo da iteriramo kroz svaki atribut i izpišemo vrednost na konzolu.

:::interactive_editor

```js
const fruit = {
  name: 'apple',
  color: 'red',
  price: 0.99
};

for (const prop in fruit) {
  console.log(fruit[prop]);
}
```

:::

Promenljiva `prop` predstavlja trenutno svojstvo objekta. `fruit[prop]` koristi se za pristup vrednosti svakog svojstva.

U prvoj iteraciji `prop` će biti `name`. U drugoj iteraciji `prop` će biti `color`, i tako dalje.

Rezultati ispisani na konzolu će biti `apple`, `red`, i `0.99`.

U drugom primeru imamo ugnježdeni objekat i želimo da prođemo kroz svako svojstvo i ispišemo njegovu vrednost u konzolu.

:::interactive_editor

```js
const person = {
  name: 'John',
  age: 30,
  address: {
    street: '123 Main St',
    city: 'Anytown',
    state: 'CA'
  }
};

for (const prop in person) {
  console.log(person[prop]);
}
```

:::

Svojstvo `address` je objekat. Petlja `for...in` će takođe proći kroz svojstva objekta `person` i ispisati ceo objekat `address` u konzolu.

Ovo je kako će rezultati izgledati na konzoli:

```md
John
30
{ street: '123 Main St', city: 'Anytown', state: 'CA' }
```

Ako želite da iterirate preko svojstava objekta `address`, možete ga postaviti unutar druge petlje od `for...in` i unutar prve.

:::interactive_editor

```js
const person = {
  name: 'John',
  age: 30,
  address: {
    street: '123 Main St',
    city: 'Anytown',
    state: 'CA'
  }
};

function isObject(obj) {
  return typeof obj === 'object' && !Array.isArray(obj) && obj !== null;
}

for (const prop in person) {
  if (isObject(person[prop])) {
    for (const nestedProp in person[prop]) {
      console.log(person[prop][nestedProp]);
    }
  } else {
    console.log(person[prop]);
  }
}
```

:::

U ovom primeru, imamo specijalnu funkciju za `isObject` koja proverava da li je vrednost nešto.

Funkcija `Array.isArray` se koristi za proveru da li vrednost predstavlja niz podataka. Dodavanjem logičkog operatora NOT (`!`) pre ove funkcije, proveravamo da li vrednost NIJE niz podataka.

Razlog zašto ne možemo koristiti `typeof` jednako kao `'object'` je zato što se redovi podataka takođe smatraju objektima u JavaScript. Želimo da uklonimo redove podataka iz validacije.

Takođe, zbog istorijske greške u JavaScript, `typeof null` vraća `'object'`. Stoga želimo da takođe uklonimo vrednosti `null` iz pregleda.

Ako je uslov istinit, pokreće se još jedan ciklus za `for...in` koji će iterirati kroz svojstva objekta definisanog unutra i napisati vrednost na konzolu.

Svojstvo `nestedProp` predstavlja trenutno svojstvo objekta definisanog unutra.

Ovo je kako će izgledati modifikovani rezultati na konzoli:

```md
"John"
30
"123 Main St"
"Anytown"
"CA"
```

Petlja za `for...in` je važna kada vam treba iterirati preko svojstava objekta.

Nije preporučljivo koristiti petlju `for...in` za iteraciju po elementima niza podataka. Umesto toga, koristite petlju `for...of` ili druge metode za nizove kao što su `forEach`, `map`, `filter` i `reduce`, koje ćete naučiti u budućim lekcijama.
