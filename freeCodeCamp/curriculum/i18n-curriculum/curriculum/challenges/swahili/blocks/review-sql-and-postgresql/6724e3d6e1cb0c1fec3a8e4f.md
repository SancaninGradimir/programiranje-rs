## Uvod u Relacione Baze Podataka

- **Relacione baze podataka**: One skladište podatke u povezanim tabelama koje imaju redove i kolone. Svaki red predstavlja zapis, a svaka kolona predstavlja atribut elementa podataka.
- **Prednosti relacionih baza podataka**: Lako se skaliraju, koriste se u različitim oblastima (npr., zdravstvo, biznis, igre), i dizajnirane su za održavanje pouzdanih podataka.
- **Uobičajene primene**: Razvoj mreže, sistemi računovodstva imovine, e-trgovina, zdravstvene usluge i korporativne aplikacije.

## Važna napomena

- **Šema baze podataka (schema)**: Relaciona baza podataka zahteva šemu koja opisuje njenu strukturu — tabele, kolone, tipove podataka, ograničenja i odnose.
- **Primarni ključ**: Jedinstveni identifikatori za svaki red u tabeli. Ključan je za integritet podataka i koristi se za povezivanje zapisa između tabela putem stranih ključeva.
- **Strani ključ**: Reference na primarni ključ u drugoj tabeli, koristi se za povezivanje povezanih podataka u tabelama.
- **Odnos (Relationship)**: Povezujući tabele putem primarnih i stranih ključeva, možete organizovati strukturirane podatke i postavljati smislena pitanja.
- **Dijagrami entitet-veza (ERDs)**: Prikazuje kako su entiteti (tabele) povezani u šemi baze podataka.
- **Integritet podataka**: Implementira se korišćenjem ključeva i tipova podataka. Osigurava konzistentnost i tačnost pohranjenih podataka.

## Osnovni principi za SQL

- **Maswali (queries)**: Zahtev za dobijanje specifičnih podataka iz baze podataka.

```sql
SELECT * FROM dogs WHERE age < 3;
```

- **WHERE klauzula**: Filtrira rezultate prema uslovima. Koristite operatore poređenja kao što su `<`, `=`, `>`, itd.
- **ORDER BY**: Dohvata i sortira rezultate prema koloni.

```sql
SELECT columns FROM table_name ORDER BY column_name;
```

## Operacije za tabele

- **Dio `CREATE TABLE`**: Ovaj deo se koristi za kreiranje nove tabele u bazi podataka.

```sql
CREATE TABLE first_table();
```

- **Funkcija za `ALTER TABLE ADD COLUMN`**: Ova funkcija se koristi za dodavanje kolone u postojeću tabelu.

```sql
ALTER TABLE table_name ADD COLUMN column_name DATATYPE;
```

- **Sekcija `ALTER TABLE DROP COLUMN`**: Ova sekcija se koristi za brisanje kolone iz postojećeg tabela.

```sql
ALTER TABLE table_name DROP COLUMN column_name;
```

- **Kifungu cha `ALTER TABLE RENAME COLUMN`**: Ovaj član se koristi za promenu imena kolone u tabeli.

```sql
ALTER TABLE table_name RENAME COLUMN column_name TO new_name;
```

- **Kifungu cha `DROP TABLE`**: Ova funkcija se koristi za brisanje cele tabele iz baze podataka.

```sql
DROP TABLE table_name;
```

- **Sekcija za `ALTER DATABASE RENAME`**: Ovaj odeljak se koristi za promenu imena baze podataka.

```sql
ALTER DATABASE database_name RENAME TO new_database_name;
```

- **Sekcija `DROP DATABASE`**: Ova sekcija se koristi za brisanje cele baze podataka.

```sql
DROP DATABASE database_name;
```

## Ograničenja i tačnost podataka

- **Klauzula za `ALTER TABLE ADD COLUMN` sa ograničenjem**: Ova klauzula se koristi za dodavanje kolone sa ograničenjem u postojeću tabelu.

```sql
ALTER TABLE table_name ADD COLUMN column_name DATATYPE CONSTRAINT;
```

- **Ograničenje od `NOT NULL`**: Ovo ograničenje osigurava da kolona ne može imati NULL vrednosti.

```sql
column_name VARCHAR(50) NOT NULL
```

- **Kifungu cha `ALTER TABLE ADD PRIMARY KEY`**: Ovaj deo se koristi za dodavanje ograničenja primarnog ključa u tabelu.

```sql
ALTER TABLE table_name ADD PRIMARY KEY(column_name);
```

- **Kifungu cha `ALTER TABLE DROP CONSTRAINT`**: Ova klauzula se koristi za brisanje ograničenja iz tabele.

```sql
ALTER TABLE table_name DROP CONSTRAINT constraint_name;
```

- **Izjava `ALTER TABLE ADD COLUMN` sa stranim ključem**: Ova izjava se koristi za dodavanje kolone stranog ključa koja referencira drugu tabelu.

```sql
ALTER TABLE table_name ADD COLUMN column_name DATATYPE REFERENCES referenced_table_name(referenced_column_name);
```

- **Kifungu cha `ALTER TABLE ADD UNIQUE`**: Ovaj deo se koristi za dodavanje UNIQUE ograničenja na kolonu.

```sql
ALTER TABLE table_name ADD UNIQUE(column_name);
```

- **Kifungu cha `ALTER TABLE ALTER COLUMN SET NOT NULL`**: Ovaj deo se koristi za postavljanje ograničenja NOT NULL na postojeću kolonu.

```sql
ALTER TABLE table_name ALTER COLUMN column_name SET NOT NULL;
```

- **Sekcija `INSERT` sa NULL vrednostima**: Ova sekcija pokazuje kako umetnuti NULL vrednosti u tabelu.

```sql
INSERT INTO table_name(column_a) VALUES(NULL);
-- or
INSERT INTO table_name(column_b) VALUES('value'); -- if column_a allows nulls
```

- **Kompozitni primarni ključ**: Ovo ograničenje opisuje primarni ključ koji uključuje više kolona.

```sql
CREATE TABLE course_enrollments (
    student_id INT,
    course_id INT,
    PRIMARY KEY (student_id, course_id)
);
```

## Operacije sa podacima (CRUD)

- **Kifungu cha `INSERT`**: Ova klauzula se koristi za unos jednog reda u tabelu.

```sql
INSERT INTO table_name(column_1, column_2) VALUES(value1, value2);
```

- **Izjava za `INSERT` sa preskočenim kolonama**: Ova izjava pokazuje kako uneti vrednost bez navođenja imena kolona, oslanjajući se na podrazumevani redosled kolona u tabeli.

```sql
INSERT INTO dogs VALUES ('Gino', 3);
```

- **Изjava sa više redova `INSERT`**: Ova izjava se koristi za unos više redova u tabelu u jednoj operaciji.

```sql
INSERT INTO dogs (name, age) VALUES 
('Gino', 3),
('Nora', 2);
```

- **Odeljak za `UPDATE`**: Ovaj odeljak se koristi za korekciju podataka koji postoje u tabeli, prema uslovu.

```sql
UPDATE table_name SET column_name=new_value WHERE condition;
```

- **Izraz za `DELETE`**: Ovaj izraz se koristi za brisanje redova iz tabele na osnovu uslova.

```sql
DELETE FROM table_name WHERE condition;
```

## Vrste podataka

- **Tip podataka `NUMERIC`**: Ovaj tip podataka se koristi za skladištenje decimalnih brojeva sa specifičnom preciznošću i skalom.

```sql
price NUMERIC(10, 2)
```

- **Tip podataka `TEXT`**: Ovaj tip podataka se koristi za skladištenje sekvence karaktera sa varijabilnom dužinom bez specifičnog ograničenja.

```sql
column_name TEXT
```

- **Tip podataka `INTEGER`**: Ovaj tip podataka se koristi za skladištenje celih brojeva bez decimalnih delova.

```sql
units_sold INTEGER
```

- **Tip podataka `SMALLINT` na `BIGINT`**: Ovo su tipovi INTEGER sa malim i velikim rasponima, respektivno.

- **Tip podataka `SERIAL`**: Ovaj tip podataka se koristi za kreiranje sekvenci punih brojeva koji se automatski inkrementiraju u PostgreSQL-u.

```sql
id SERIAL
```

- **Svojstvo `AUTO_INCREMENT`**: Ovo svojstvo se koristi u MySQL-u za kreiranje sekvenci automatski inkrementiranih celih brojeva.

```sql
id INT AUTO_INCREMENT
```

- **Aina ya data `VARCHAR`**: Ovaj tip podataka se koristi za skladištenje sekvence karaktera sa promenljivom dužinom i definisanim gornjim ograničenjem.

```sql
name VARCHAR(50)
```

- **Vrsta podataka `DATE`**: Ova vrsta podataka se koristi za skladištenje datuma (godina, mesec, dan).

```sql
event_date DATE
```

- **Tip podataka `TIME`**: Ovaj tip podataka se koristi za skladištenje vremena (sati, minute, sekunde).

```sql
start_time TIME
```

- **Tip podataka `TIMESTAMP`**: Ovaj tip podataka se koristi za skladištenje datuma i vremena, opciono sa informacijama o vremenskoj zoni.

```sql
event_timestamp TIMESTAMP
event_timestamp TIMESTAMP WITH TIME ZONE
```

- **Tip podataka `BOOLEAN`**: Ovaj tip podataka se koristi za skladištenje booleanskih vrednosti.

```sql
is_active BOOLEAN
```

## Odnosi u bazi podataka

- **Vrste odnosa**: Ovo su različiti načini na koje tabele mogu biti povezane u relacionoj bazi podataka.
  - Jedan-na-jedan
  - Jedan-na-više
  - Više-na-jedan
  - Više-na-više
  - Samoreferencijalni (rekurzivni)

- **Odnos jedan-na-jedan**: Ovaj tip odnosa znači da svaki zapis u jednoj tabeli odgovara samo jednom zapisu u drugoj tabeli.

```md
One employee is assigned exactly one vehicle.
Tables: employees, vehicles
```

- **Odnos jedan-mnogo**: Ovaj tip odnosa znači da se jedan zapis u tabeli može povezati sa više zapisa u drugoj tabeli.

```md
One customer can have many orders.
Tables: customers → orders
```

- **Više-na-više odnos putem tabele za spajanje**: Ovaj tip odnosa implementira se korišćenjem tabele za spajanje koja sadrži strane ključeve iz svih povezanih tabela.

```sql
CREATE TABLE books_authors (
    author_id INT REFERENCES authors(id),
    book_id INT REFERENCES books(id)
);
```

- **Samoreferencijalni odnos**: Ovaj tip odnosa nastaje kada tabela referiše na sebe, stvarajući hijerarhijsku strukturu.

```md
An employee table where each employee may report to another employee.
```

## SQL gornji (Joins)

- **Klauzulica za `INNER JOIN`**: Ovaj JOIN vraća samo redove sa podacima koji se poklapaju u obe tabele.

```sql
SELECT *
FROM products
INNER JOIN sales ON products.product_id = sales.product_id;
```

- **Kifungu cha `FULL OUTER JOIN`**: Ovaj JOIN vraća sve redove iz obe tabele, uključujući neusklađene redove iz bilo koje tabele.

```sql
SELECT *
FROM products
FULL OUTER JOIN sales ON products.product_id = sales.product_id;
```

- **Kifungu cha `LEFT OUTER JOIN`**: Ovo spajanje vraća sve redove iz leve tabele i odgovarajuće redove iz desne tabele.

```sql
SELECT *
FROM products
LEFT JOIN sales ON products.product_id = sales.product_id;
```

- **Klauzulă za `RIGHT OUTER JOIN`**: Ovaj join vraća sve redove iz desne tabele i odgovarajuće redove iz leve tabele.

```sql
SELECT *
FROM products
RIGHT JOIN sales ON products.product_id = sales.product_id;
```

- **Kifungu cha `SELF JOIN`**: Ovaj join se koristi za spajanje tabele sa njom samom, poređenjem redova unutar te tabele.

```sql
SELECT A.column_name, B.column_name
FROM table_name A
JOIN table_name B ON A.related_column = B.related_column;
```

- **Kifungu cha `CROSS JOIN`**: Ovaj join vraća kartezijanski proizvod dve tabele, kombinujući svaki red iz prve tabele sa svakim redom iz druge tabele.

```sql
SELECT *
FROM table1
CROSS JOIN table2;
```

## Specifične komande za PostgreSQL

- **Komanda za prijavu `psql`**: Ova komanda se koristi za pristup PostgreSQL-u sa specifičnim korisničkim imenom i bazom podataka.

```bash
psql --username=freecodecamp --dbname=postgres
```

- **Amri ya `\l`**: Ova komanda navodi sve baze podataka u formatu PostgreSQL-a.

```sql
\l
```

- **Komande za `CREATE DATABASE` na `\c`**: Ove komande se koriste za kreiranje nove baze podataka i povezivanje sa njom.

```sql
CREATE DATABASE database_name;
\c database_name
```

- **Amri ya `\d`**: Ova komanda navodi sve tabele u trenutnoj bazi podataka.

```sql
\d
```

- **Amri ya `\d table_name`**: Ova komanda prikazuje strukturu/prikaz specifične tabele.

```sql
\d table_name
```

- **Komanda za `\q`**: Ova komanda dolazi iz klijenta PostgreSQL-a.

```bash
\q
```

## Relacione baze podataka naspram nerelacionih baza podataka

- **Ne-relacione baze podataka (NoSQL)**: Čuvaju nestrukturirane ili semi-strukturirane podatke. Ne zahtevaju strogi shema i idealne su za dinamične modele podataka.
- **Izbor između relacionih i ne-relacionih baza podataka**: Zavisi od prirode vaših podataka i potreba aplikacije.
- **Relacioni naspram ne-relacionih**: Izaberite relacione za strukturirane i stabilne podatke; NoSQL za fleksibilnost i brzo menjajuće podatke.

## Popularni sistemi RDBMS

- **MySQL**: Open-source, robusan, široko korišćen u razvoju mreža, podržan od strane velike zajednice.
- **PostgreSQL**: Open-source, napredno, proširivo. Podržava specifične tipove podataka i aplikacije na strani servera.
- **SQLite**: Lagan, zasnovan na fajlovima, bez servera. Pogodan za male aplikacije.
- **Microsoft SQL Server**: Komercijalna baza podataka sa licencama.
- **Oracle Database**: Komercijalni RDBMS poznat po visokim performansama i skalabilnosti.
