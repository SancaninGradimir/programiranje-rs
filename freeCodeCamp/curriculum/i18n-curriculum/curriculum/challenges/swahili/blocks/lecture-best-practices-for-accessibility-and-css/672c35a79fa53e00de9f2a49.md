# --interactive--

Skrivanje sadržaja na web stranici je uobičajena tehnika u razvoju web aplikacija, ali je važno to učiniti na način koji ne otežava pristupačnost. Različite metode skrivanja mogu različito uticati na način na koji asistivne tehnologije interpretiraju i prikazuju sadržaj korisnicima. Ispitajmo najbolje prakse za skrivanje sadržaja uz osiguravanje da ostane pristupačan. Jedan uobičajen način skrivanja sadržaja je korišćenje `display: none`. Evo primera:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<p class="hidden">Hidden text</p>
<p>Visible text</p>
```

```css
.hidden {
  display: none;
}
```

:::

Iako ovo vizuelno skriva sadržaj, uklanja ga i iz stabla pristupačnosti. Stabak pristupačnosti je struktura koju koriste tehnologije asistivne, kao što su čitači ekrana, da bi interpretirale i interagirale sa sadržajem na veb stranici. On predstavlja sadržaj i njegovo deskriptivno značenje na način koji tehnologije asistivne mogu razumeti i prikazati korisniku.

Korišćenje `display: none` znači da čitači ekrana i druge asistivne tehnologije ne mogu pristupiti ovom sadržaju jer nije uključen u stabalicu pristupačnosti. Stoga je važno koristiti ovu metodu samo kada želite potpuno ukloniti sadržaj iz vizuelnog i pristupačnog predstavljanja.

Drugi način za skrivanje sadržaja je korišćenje `visibility: hidden`:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<p class="hidden">Hidden text</p>
<p>Visible text</p>
```

```css
.hidden {
  visibility: hidden;
}
```

:::

`visibility: hidden` skriva sadržaj vizuelno, ali ga ostavlja u toku dokumenta, što znači da i dalje zauzima prostor na stranici. Kao `display: none`, `visibility: hidden` takođe uklanja sadržaj iz stabla pristupačnosti. To znači da asistivne tehnologije, kao što su čitači ekrana, neće moći pristupiti skrivenom sadržaju. Koristite `visibility: hidden` samo kada želite da sakrijete sadržaj od svih, uključujući osobe koje koriste asistivne tehnologije.

Za sadržaj koji treba da bude skriven vizuelno, ali ostaje dostupan čitačima ekrana, možete koristiti tehniku poznatu kao "visually hidden" ili "screen reader only". Evo primera korišćenja klase CSS od `.sr-only`, što je uobičajena tehnika za vizuelno skrivanje sadržaja uz zadržavanje dostupnosti čitačima ekrana:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<p class="sr-only">Hidden text</p>
<p>Visible text</p>
```

```css
.sr-only {
  position: absolute;
  width: 1px;
  height: 1px;
  padding: 0;
  margin: -1px;
  overflow: hidden;
  clip: rect(0, 0, 0, 0);
  white-space: nowrap;
  border: 0;
}
```

:::

U ovom primeru, koristimo atribute kao `position`, `clip`, i `white-space`, o kojima ćete naučiti u budućim lekcijama. Trenutno, samo znajte da ova pravila za CSS skrivaju sadržaj vizuelno, ali ostaju dostupni čitačima ekrana. To je važno za pružanje dodatnog konteksta korisnicima čitača ekrana bez uticaja na vizuelni raspored.

Da biste promenili izgled sadržaja, razmislite o korišćenju atributa `hidden`:

:::interactive_editor

```html
<p hidden>This content is hidden</p>
<p>This content is visible</p>
```

:::

Funkcija `hidden` je podržana od strane mnogih savremenih pregledača i skriva sadržaj vizuelno, kao i iz stabla pristupačnosti. Može se lako promeniti korišćenjem JavaScript. Na kraju, budite pažljivi kada je radi o skrivanju važnog sadržaja. Ako je informacija važna za razumevanje ili korišćenje sajta, ona mora biti vidljiva i dostupna svim korisnicima. Skrivajte sadržaj samo kada to zaista poboljšava korisničko iskustvo. Prateći ove najbolje prakse, možete osigurati da vaš sadržaj ostane dostupan svim korisnicima, bez obzira na način na koji saobraćaju sa vašim sajtom.
