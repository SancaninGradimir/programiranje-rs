# --description--

npm zahteva način za skladištenje važnih informacija o tvom projektu – stvari poput kojih paketa se koriste, verziju tih paketa i koje komande mogu biti izvršene. To se dešava u fajlu `package.json`.

Zamislite fajl `package.json` kao npm konfiguracioni fajl za projekat. To je jednostavan fajl za JSON koji opisuje vaš projekat koristeći parove ključ-vrednost ili da prati stvari kao što su:

* Naziv projekta i njegova verzija
    
* Autor
    
* Glavna datoteka koja treba da se inicijalizuje prvo
    
* Licenca
    
* Repozitorijum koda, ključnih reči i opisa
    
* Bilo koja zavisnost potrebna projektu

Ovo je primer fajla za `package.json`:

```json
{
  "name": "my-app",
  "version": "1.0.0",
  "description": "A simple NPM project",
  "main": "index.js",
  "scripts": {
    "start": "node index.js",
    "test": "echo \"Running tests...\""
  },
  "repository": {
    "type": "git",
    "url": "git+https://github.com/username/project.git"
  },
  "keywords": ["fCC", "js"],
  "author": "Kolade Chris",
  "license": "ISC",
  "dependencies": {
    "react": "^19.2.1"
  },
  "devDependencies": {
    "cypress": "^15.7.1"
  }
}
```

Polje `main` se najčešće postavlja na `index.js` i govori Node.js-u koju datoteku prvo treba pokrenuti.

Sekcija `license` koristi se za određivanje pravnih uslova koje drugi ljudi mogu koristiti, modifikovati i distribuirati kod sadržan u vašem paketu. Postoji mnogo različitih licenci kao što su `MIT`, `ISC` ili `GPL`, a sve imaju različite uslove i dozvole. Podrazumevana je `ISC`, Licenca Internet Systems Consortium, koja je besplatna licenca sa mnogo dozvola.

Kako tvoj projekat raste i dodaješ pakete, npm ih smešta u jednu od dve kategorije:

* `dependencies`, koji su paketi potrebni da vaša aplikacija radi u produkciji
    
* `devDependencies`, koji su paketi potrebni samo tokom razvoja

Na primer, ako tvoj projekat koristi React za korisnički interfejs, on će biti u `dependencies` jer je potreban da bi aplikacija radila nakon objavljivanja. Alati poput Cypress-a, koji se koriste samo za testiranje pre objavljivanja, pripadaju `devDependencies`.

Postoji nekoliko jednostavnih načina za kreiranje datoteke `package.json`. Najčešći je pokretanje sledeće komande:

```bash
npm init
```

Ovu komandu treba da pokreneš u glavnom direktorijumu projekta. Ona pokreće interaktivni proces podešavanja u kojem npm postavlja nekoliko pitanja. Tvoji odgovori popunjavaju polja kao što su naziv projekta, verzija, opis, ulazna datoteka, ključne reči, autor i licenca.

Ako želiš da preskočiš pitanja i odmah kreiraš `package.json` sa podrazumevanim vrednostima, možeš pokrenuti:

```bash
npm init -y
```

au

```bash
npm init --yes
```

Ovo odmah kreira datoteku `package.json` sa podrazumevanim vrednostima za većinu polja.

Obično je bolje koristiti `npm init` bez zastavice `-y` ili `--yes` kako bi prošlo kroz svaki deo i definisalo podatke vašeg projekta. Ako ne odgovorite na pitanje, npm koristi podrazumevanu vrednost. Na primer, ako ostavite pitanje o licenci prazno, npm će ga postaviti na `ISC` po defaultu.

Možda ćete otkriti da `dependencies` i `devDependencies` još uvek nisu navedeni. To je zato što se ove sekcije dodaju tek nakon instaliranja vašeg prvog paketa.

Takođe ćete videti da deo za `repository` nedostaje. Iako je npm pitao o tome tokom `npm init`, nije ga popunio automatski kasnije. Na primer, kada pokrenete komandu kao `git remote add origin` ili `git push`, npm neće ažurirati deo repozitorijuma koda umesto vas — trebalo bi da ga dodate sami ako želite da bude uključen.

U narednim lekcijama, naučićete više o zavisnostima u `package.json` i kako da instalirate, ažurirate i obrišete.
