# --interactive--

Semantički elementi za kontrolu obrazaca kao što su `input`, `select`, `textarea`, `button` i `fieldset` imaju ugrađena stanja koja su dostupna pomoćnim tehnologijama.

Na primer, možete koristiti atribut `disabled` za onemogućavanje dugmeta ili atribut `checked` da pokaže da je čekboks odabran.

Međutim, ako praviš prilagođenu kontrolu, moraš koristiti ARIA atribute kako bi njeno stanje bilo dostupno pomoćnim tehnologijama.

U ovom tutorijalu, razmotrićemo nekoliko ARIA stanja koja se često koriste na specifičnim kontrolnim komponentama.

Prvo ARIA stanje koje ćemo razmotriti je `aria-selected`. Ono označava da je element izabran. Možeš ga koristiti na prilagođenim kontrolama poput tabova, lista ili mreža.

Evo primera kako možete koristiti `aria-selected` u specifičnom kontroleru kartica:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div role="tablist">
  <button role="tab" aria-selected="true">Tab 1</button>
  <button role="tab" aria-selected="false">Tab 2</button>
  <button role="tab" aria-selected="false">Tab 3</button>
</div>
<script src="index.js"></script>
```

```css
[role="tablist"] {
  display: flex;
  border-bottom: 2px solid #ddd;
  gap: 0.25rem;
  font-family: system-ui, sans-serif;
}

[role="tab"] {
  appearance: none;
  border: none;
  background: none;
  padding: 0.5rem 1rem;
  cursor: pointer;
  font-size: 1rem;
  color: #444;
  border-radius: 4px 4px 0 0;
  transition: background-color 0.2s, color 0.2s;
}

[role="tab"]:hover {
  background-color: #f3f3f3;
}

[role="tab"][aria-selected="true"] {
  background-color: #fff;
  color: #0078d4;
  border: 2px solid #0078d4;
  border-bottom: 2px solid #fff; 
  font-weight: 600;
  position: relative;
  z-index: 1;
}

[role="tab"]:focus {
  outline: 2px solid #0078d4;
  outline-offset: 2px;
}

```

```js
document.addEventListener("click", (event) => {
  const clickedTab = event.target.closest('[role="tab"]');
  if (!clickedTab) return;

  const tablist = clickedTab.closest('[role="tablist"]');
  const tabs = tablist.querySelectorAll('[role="tab"]');

  tabs.forEach((tab) => {
    const isSelected = tab === clickedTab;
    tab.setAttribute("aria-selected", isSelected);
    tab.tabIndex = isSelected ? 0 : -1;
  });
});

document.addEventListener("keydown", (event) => {
  const activeTab = document.activeElement;
  if (activeTab.getAttribute("role") !== "tab") return;

  const tablist = activeTab.closest('[role="tablist"]');
  const tabs = Array.from(tablist.querySelectorAll('[role="tab"]'));
  const index = tabs.indexOf(activeTab);

  let newIndex = index;
  if (event.key === "ArrowRight") newIndex = (index + 1) % tabs.length;
  if (event.key === "ArrowLeft") newIndex = (index - 1 + tabs.length) % tabs.length;

  if (newIndex !== index) {
    tabs[newIndex].focus();
    tabs[newIndex].click();
  }
});

```

:::

Tabovi služe za prikaz više panela sadržaja na malom prostoru. `aria-selected` označava koji je tab trenutno izabran.

Kada korisnik izabere tab, atribut `aria-selected` tog taba postavlja se na `true`, dok se kod ostalih tabova postavlja na `false`.

Još jedno često ARIA stanje je `aria-disabled`. Ono označava da je element onemogućen za korisnike pomoćnih tehnologija, poput čitača ekrana. Važno je razumeti da `aria-disabled` zapravo ne onemogućava element. Na tebi kao programeru je da pomoću stilova i JavaScript-a učiniš da izgleda i ponaša se kao onemogućen. Ovaj atribut se ponekad koristi i na izvornim HTML elementima umesto atributa `disabled`, u zavisnosti od konteksta.

Evo primera kako možete koristiti `aria-disabled` na specifičnom gumbu za uređivanje:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div role="button" tabindex="-1" aria-disabled="true">Edit</div>
```

```css
[role="button"] {
  display: inline-block;
  background-color: #0078d4;
  color: #fff;
  padding: 0.5rem 1rem;
  border-radius: 6px;
  font-family: system-ui, sans-serif;
  font-size: 1rem;
  text-align: center;
  cursor: pointer;
  user-select: none;
  transition: background-color 0.2s, transform 0.1s, opacity 0.2s;
}

[role="button"]:not([aria-disabled="true"]):hover {
  background-color: #005fa3;
}

[role="button"]:not([aria-disabled="true"]):focus {
  outline: 2px solid #005fa3;
  outline-offset: 2px;
}

[role="button"]:not([aria-disabled="true"]):active {
  transform: scale(0.97);
}

[role="button"][aria-disabled="true"] {
  opacity: 0.5;
  pointer-events: none;
  cursor: not-allowed;
  background-color: #b0b0b0;
  color: #f2f2f2;
}
```

:::

Atribut `aria-disabled` govori korisnicima čitača ekrana da je dugme za uređivanje onemogućeno i da se sa njim ne može stupiti u interakciju. Ipak, ono nije stvarno onemogućeno. Moraćeš pomoću CSS-a i JavaScript-a da obezbediš odgovarajući izgled i ponašanje.

U mnogim slučajevima, možda ćete koristiti ugrađenu dugmetnu karakteristiku, ali postoje situacije kada ćete morati da koristite specijalizovanu kontroleru. Stoga je važno znati kako preneti status kontrolera na asistivne tehnologije.

Sledeće ARIA stanje je `aria-haspopup`. Ono označava da će interaktivni element otvoriti iskačući element. Može se koristiti samo kada iskačući element ima jednu od sledećih uloga: `menu`, `listbox`, `tree`, `grid` ili `dialog`. Vrednost atributa mora biti jedna od tih uloga ili `true`, što je podrazumevana vrednost za `menu`.

Ovo je primer menija editora fajlova koji koristi `aria-haspopup`:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<button id="menubutton" aria-haspopup="menu" aria-controls="filemenu" aria-expanded="false">File</button>
<ul id="filemenu" role="menu" aria-labelledby="menubutton" hidden>
  <li role="menuitem" tabindex="-1">Open</li>
  <li role="menuitem" tabindex="-1">New</li>
  <li role="menuitem" tabindex="-1">Save</li>
  <li role="menuitem" tabindex="-1">Delete</li>
</ul>
```

```css
#menubutton {
  background-color: #0078d4;
  color: #fff;
  border: none;
  padding: 8px 14px;
  border-radius: 4px;
  cursor: pointer;
  font-size: 14px;
  position: relative;
}

#menubutton:hover,
#menubutton:focus {
  background-color: #005ea2;
  outline: none;
}

#filemenu {
  list-style: none;
  padding: 4px 0;
  margin: 4px 0 0;
  border: 1px solid #ccc;
  border-radius: 4px;
  background-color: #fff;
  width: 160px;
  position: absolute;
  z-index: 1000;
  box-shadow: 0 4px 6px rgba(0,0,0,0.1);
}

#filemenu[hidden] {
  display: none;
}

#filemenu [role="menuitem"] {
  display: block;
  padding: 8px 12px;
  font-size: 14px;
  color: #333;
  cursor: pointer;
}

#filemenu [role="menuitem"]:hover,
#filemenu [role="menuitem"]:focus {
  background-color: #e5f1fb;
  outline: none;
}

#filemenu [role="menuitem"]:focus-visible {
  box-shadow: inset 0 0 0 2px #0078d4;
}

```

:::

Stanje `aria-haspopup` se koristi za prikaz da će dugme menija `File` otvoriti popup meni kada ga aktivirate. Korisnici čitača ekrana mogu čuti ovu dodatnu informaciju kada dođu do dugmeta.

Moraćete da koristite JavaScript za prikazivanje i skrivanje pop-up menija, te implementirati odgovarajuću pomoć tastaturom za interakciju sa menijem. Takođe, molimo vas da znate da se uloga ARIA `menu` odnosi na veoma specifičan tip menija. Obično se odnosi na listu akcija koje korisnik može pokrenuti, slično menijima u desktop aplikacijama. Ne odnosi se na uobičajenu upotrebu "menija" kako mi znamo, kao što su navigacioni meniji. Zapravo, većina menija koje pravite online neće biti ARIA meniji i neće koristiti ni `aria-haspopup`.

Sledeći ARIA atribut koji ćemo diskutovati je `aria-required`. Atribut `aria-required` se koristi da pokaže da je potrebno popuniti određeno polje pre slanja forme.

Ovo je primer korišćenja svojstva `aria-required` za specifični kontroler forme.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div id="name-label">Full Name*</div>
<div role="textbox" contenteditable aria-labelledby="name-label" aria-required="true" id="name"></div>
```

```css
#name-label {
  font-family: system-ui, sans-serif;
  font-size: 0.95rem;
  font-weight: 600;
  margin-bottom: 0.25rem;
  color: #333;
}

[role="textbox"] {
  display: block;
  width: 100%;
  min-height: 2rem;
  padding: 0.5rem 0.75rem;
  border: 1.5px solid #ccc;
  border-radius: 4px;
  font-family: system-ui, sans-serif;
  font-size: 1rem;
  color: #222;
  background-color: #fff;
  line-height: 1.4;
  transition: border-color 0.2s, box-shadow 0.2s;
}

[role="textbox"]:hover {
  border-color: #999;
}

[role="textbox"]:focus {
  outline: none;
  border-color: #0078d4;
  box-shadow: 0 0 0 3px rgba(0, 120, 212, 0.25);
}

[role="textbox"]:empty::before {
  content: attr(data-placeholder);
  color: #aaa;
  pointer-events: none;
}

[role="textbox"][aria-required="true"] {
  border-left: 3px solid #e81123;
  padding-left: calc(0.75rem - 3px);
}
```

:::

Moramo koristiti atribut `contenteditable` kako bi korisnici mogli uneti svoj unos. Takođe koristimo atribut `aria-required` postavljen na `true` da pokaže da je ovaj specifični kontroler forme obavezan.

Da biste učinili da je validador forme kao običan validador, moraćete dodati CSS. Takođe ćete morati da dodate JavaScript kako biste sprečili slanje forme bez sadržaja.

Ako etiketa već sadrži reč `required`, onda biste trebali ostaviti atribut `aria-required`. Ovo osigurava da čitači ekrana izbacuju reč "required" samo jednom.

U mnogim slučajevima, možda ćete koristiti osnovne elemente `label` i `form` zajedno sa atributom `required`. Ali ako želite da kreirate prilagođeni kontroler forme, onda je važno dodati atribut `aria-required` kada je potrebno.

Štaviše, atribut `aria-required` se takođe može koristiti na izvornim unosima formulara, kao što su elementi `input`, `textarea` i `select`. Ovo se često preferira u odnosu na izvorni atribut `required`, jer atribut `required` može imati probleme sa korišćenjem i pristupačnošću, posebno za podrazumevano rukovanje greškama koje pruža pregledač. U konačnici, moraćete da eksperimentišete kako biste odredili koji je atribut najbolji za vaš slučaj.

Konačno stanje ARIA o kojem ćemo razgovarati je `aria-checked`. Ovaj atribut se koristi za prikazivanje da li je element u izabranom stanju. Najčešće se koristi prilikom kreiranja prilagođenih kutija za odabir, radijskih dugmadi, prekidača i lista.

Ovo je primer kako možete koristiti `aria-checked` u specifičnoj padajućoj kontroli za odabir:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<div role="checkbox" aria-checked="true" tabindex="0">Checkbox</div>
```

```css
[role="checkbox"] {
  display: inline-flex;
  align-items: center;
  gap: 0.5rem;
  font-family: system-ui, sans-serif;
  font-size: 1rem;
  cursor: pointer;
  user-select: none;
  color: #222;
}

[role="checkbox"]::before {
  content: "";
  display: inline-block;
  width: 1rem;
  height: 1rem;
  border: 2px solid #666;
  border-radius: 4px;
  background-color: #fff;
  transition: all 0.2s ease;
  box-sizing: border-box;
}

[role="checkbox"]:hover::before {
  border-color: #0078d4;
}

[role="checkbox"]:focus::before {
  outline: 2px solid #0078d4;
  outline-offset: 2px;
}

[role="checkbox"][aria-checked="true"]::before {
  background-color: #0078d4;
  border-color: #0078d4;
  background-image: url("data:image/svg+xml;utf8,<svg xmlns='http://www.w3.org/2000/svg' width='14' height='14' viewBox='0 0 14 14'><path fill='white' d='M5.2 10.4L2 7.2l1.1-1.1 2.1 2.1L10.9 2.5 12 3.6z'/></svg>");
  background-repeat: no-repeat;
  background-position: center;
}

[role="checkbox"][aria-disabled="true"] {
  opacity: 0.5;
  pointer-events: none;
  cursor: not-allowed;
}

```

:::

Lokalni elementi za izbore imaju ugrađeno stanje `checked` koje je obezbeđeno asistivnim tehnologijama. Međutim, ako kreirate prilagođeni kontroler za elemente izbora (select box), moraćete da koristite atribut `aria-checked` kako biste prikazali njegovo stanje.

Kada korisnik interaguje sa specifičnim kontrolerom za izbor, moraćete da koristite stanje `aria-checked` kako biste prikazali novo stanje polja. Kada je polje izabrano, svojstvo `aria-checked` se postavlja na `true`. Kada nije izabrano, postavlja se na `false`.

Prirodni elementi obično imaju odličnu podršku i ugrađene funkcije za pristupačnost.

Međutim, ako morate kreirati prilagođene kontrole, korišćenje ARIA atributa je važno za efikasno prenošenje stanja ovih kontrola asistivnim tehnologijama.

Kao uobičajeno, osigurajte da pregledate svoj rad kako biste bili sigurni da su ARIA atributi korišćeni na odgovarajući način i da prilagođeni kontroler radi na način koji je pogodan i jednostavan za korisnika.
