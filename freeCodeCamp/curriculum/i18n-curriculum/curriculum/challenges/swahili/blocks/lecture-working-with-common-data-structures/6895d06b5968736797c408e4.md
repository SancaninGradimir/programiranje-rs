# --description--

Strukture podataka su osnovni model podataka u nauci o računaru. Sve strukture skladište kolekcije podataka organizovane po redosuku, ali u zavisnosti od njihovog tipa, mogu obavljati različite funkcije u pozadini.

Njihovo osnovno ponašanje može značajno uticati na efikasnost programa, pa hajde da naučimo o statičkim i dinamičkim nizovima i njihovim razlikama kako bi mogao da izabereš najefikasnije rešenje za svoj program.

Tutaanza na safu za data zisizobadilika.

**Neizmenljive strukture podataka** imaju fiksnu veličinu. Čuvaju elemente u kontiguitetnim memorijskim lokacijama.

Veličina statičkog niza određuje se prilikom njegovog kreiranja. Kada se taj deo memorije rezerviše, njegova veličina se više ne menja tokom izvršavanja programa. To je osnovna karakteristika statičkih nizova.

Čuvanje elemenata u susednim memorijskim lokacijama čini proces pristupa podacima efikasnijim, jer program može sačuvati adresu prvog elementa, a zatim koristiti pokazivače za obavljanje jednostavnih proračuna i dohvaćanje drugih elemenata u memoriji.

Zahvaljujući tome, pristup elementima statičkog niza ima konstantnu vremensku složenost `O(1)`, što je veoma efikasno.

Možete koristiti nepromenljivu listu kada znate broj elemenata koji će biti sačuvan unapred. Takođe je korisno kada se vrednosti primaju često, jer je operacija pristupa veoma efikasna.

Međutim, ova struktura podataka ne može ni rasteti ni smanjivati svoju veličinu, pa ako se broj elemenata koje je potrebno sačuvati može menjati, trebalo bi da koristite dinamički niz umesto toga.

Pokušaj povećanja veličine statičkog niza zahteva kreiranje novog niza i kopiranje svih elemenata iz starog u novi, što nije efikasno. U takvim slučajevima dinamički niz je bolji izbor jer taj proces obavlja automatski.

Python nema ugrađene klasične statičke nizove kao strukturu podataka.

Međutim, drugi programski jezici, kao što je Java, to podržavaju. Ovo je primer nepromenljivog niza u Javi koji može da sačuva tri celobrojna broja:

```java
int[] numbers = new int[3]; 
```

Redovi u Python su dinamični, pa ih pregledajmo.

**Dinamički redovi podataka** su jednostavniji jer mogu automatski rasti ili smanjivati dok je program u funkciji.

Radi kroz sistem direktnog skaliranja koji kopira elemente u novi red kada je prethodni red pun. Proces se odvija efikasno jer je veličina novog reda izabrana na način koji čini ove računarsko skupocene operacije ređim.

Pristup elementima dinamičkog niza zahteva konstantno vreme `O(1)`, stoga je ova operacija veoma efikasna.

Umetanje elementa u sredinu redova zahteva liniju `O(n)` jer elementi koji se nalaze posle njega moraju biti pomereni.

Dodavanje elementa na kraj niza ima konstantnu vremensku složenost `O(1)` ako u dinamičkom nizu još ima mesta. Međutim, ako je niz popunjen i mora da se proširi, operacija ima složenost `O(n)`.

Trebalo bi koristiti dinamičke nizove kada ne znate unapred koliko vrednosti ćete morati da sačuvate u nizu. Takođe je korisno kada često dodajete i brišete elemente.

Ugrađena struktura podataka za `list` od Python radi kao dinamički niz. Možete kreirati listu upisivanjem elemenata unutar uglastih zagrada, razdvojenih zarezima.

```python
numbers = [3, 4, 5, 6]
```

Možete pristupiti karakteristici tako što ćete napisati ime parametra koji sadrži listu, praćeno kvadratnim zagradama, a unutar tih zagrada navesti odgovarajuću vrednost/primer.

Indeksi počinju sa 0 za prvi element i povećavaju se za 1 za svaki sledeći element:

```python
numbers[0]  # 3
numbers[1]  # 4
numbers[2]  # 5
numbers[3]  # 6
```

Za korekciju vrednosti, potrebno je samo da unesete novu vrednost:

```python
numbers[2] = 16
```

Možete dodati elemente u listu putem `.append()`:

```python
numbers.append(7)
```

Možete umetnuti komponente u specifičan model koristeći `.insert()`, prosleđujući model kao prvi argument i komponentu kao drugi argument.

```python
numbers.insert(3, 15)
```

Možete ukloniti funkciju u specifičnom modelu na način `.pop()`:

```python
numbers.pop(2)
```

Ako ne navedete modifikator, `.pop()` će ukloniti poslednji element.

Uopšteno, koristi statičke nizove kada unapred znaš broj elemenata i često im pristupaš, a dinamičke nizove kada broj elemenata nije poznat ili se menja tokom vremena.
