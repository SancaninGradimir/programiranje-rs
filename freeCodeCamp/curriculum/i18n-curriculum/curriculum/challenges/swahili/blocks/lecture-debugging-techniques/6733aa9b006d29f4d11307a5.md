# --description--

Ako razvijate programe za JavaScript, sigurno ste naišli na poruku o grešci. Razumevanje uobičajenih poruka o grešci pomoći će vam da efikasnije pronađete greške i razvijete svoje veštine kao moćnijeg programera. Četiri uobičajena tipa poruka o grešci su `SyntaxError`, `ReferenceError`, `TypeError` i `RangeError`.

`SyntaxError` To se dešava kada ste nešto pogrešno napisali u svom kodu, kao što je nedostatak vitičastih zagrada za petlju ili uglastih zagrada. Razmislite o tome kao o gramatičkoj grešci u rečenici. Evo uobičajene greške koju programeri pravią kada kreiraju nizove podataka:

```js
const arr = ["Beau", "Quincy" "Tom"]
```

Svaki element u redovima podataka mora biti razdvojen zarezom ili će doći do poruke o grešci.

Druga česta JavaScript greška je `ReferenceError`. Postoji više vrsta `ReferenceError` grešaka koje nastaju iz različitih razloga. Prva vrsta su greške izazvane korišćenjem promenljivih koje nisu deklarisane.

```js
console.log(price);
```

U ovom primeru, pokušavamo da prikažemo vrednost parametara `price` na konzoli, ali nije navedeno. Ovo će izazvati `ReferenceError`.

Još jedan primer za `ReferenceError` je pokušaj dobijanja vrednosti parametra deklarisanog za `let` ili `const` pre nego što je pomenut:

```js
console.log(b);
const b = 50;
```

Mfano hapo juu utasababisha hitilafu ya `Cannot access 'b' before initialization`.

Treća česta greška je `TypeError`. Ove greške nastaju kada pokušate da izvršite operaciju nad pogrešnim tipom podataka. Evo primera pokušaja korišćenja metode `map` nad objektom:

```js
const developerObj = {
  name: "Jessica",
  country: "USA",
  isEmployed: true
};

developerObj.map()
```

Ovaj primer će izazvati grešku `developerObj.map is not a function` jer se metoda `map` koristi za redove podataka, a ne za objekte.

Hitilafu ya mwisho ya kawaida tutakayochunguza ni `RangeError`.

`RangeError` to se dešava kada vaš kod pokušava da koristi vrednost koja je van opsega koji JavaScript može da obradi. Evo primera postavljanja nevažeće vrednosti za `length` red podataka:

```js
const arr = [];
arr.length = -1; 
```

Pošto `length` red podataka mora biti kompletan broj bez negativnih znakova, postavljanje ga u `-1` uzrokuje `RangeError`.

Dok nastavite da razvijate aplikaciju za JavaScript, molimo vas da razumete ove različite vrste grešaka koje ćete naići i zašto se javljaju.
