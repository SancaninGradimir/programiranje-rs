## Potvrda obrasca za JavaScript

- **API za validaciju granica**: Određeni elementi HTML, kao što su elementi `textarea` i `input`, jasno definišu API validaciju granica. API omogućava vam da potvrdite da vrednost koju je korisnik unio za taj element zadovoljava bilo koju validaciju nivoa HTML koja je navedena, kao što su minimalna dužina ili u skladu sa određenim formatom.
- **Metoda `checkValidity()`**: Ova metoda vraća `true` ako element zadovoljava sve validacije od HTML (u skladu sa njegovim svojstvima), i `false` ako ne zadovoljava.

:::interactive_editor

```html
<form>
  <label>
    Email:
    <input
      id="email"
      type="email"
      required
      pattern=".+\.com$"
      placeholder="example@site.com"
    />
  </label>
</form>

<script>
  const input = document.getElementById("email");

  input.addEventListener("input", (e) => {
    if (!e.target.checkValidity()) {
      e.target.setCustomValidity("You must use a .com email.");
    } else {
      e.target.setCustomValidity("");
    }
  });
</script>
```

:::

- **Putanja za `reportValidity()`**: Ova putanja obaveštava pretraživač da `input` nije validna.

:::interactive_editor

```html
<form>
  <label>
    Email:
    <input
      id="email2"
      type="email"
      required
      pattern=".+\.com$"
      placeholder="example@site.com"
    />
  </label>
</form>

<script>
  const input = document.getElementById("email2");

  input.addEventListener("input", (e) => {
    if (!e.target.checkValidity()) {
      e.target.reportValidity();
    }
  });
</script>
```

:::

- **Svojstvo `validity`**: Ovo svojstvo se koristi za dobijanje ili postavljanje stanja validnosti kontrole forme (poput `<input>`, `<select>`, itd.) i izveštava da li uneti podaci korisnika ispunjavaju kriterijume postavljene za taj element (npr. polja `required`, strukturalna ograničenja, maksimalna dužina, itd.).

:::interactive_editor

```html
<input
  id="age"
  type="number"
  min="18"
  placeholder="Enter age (18+)"
/>

<script>
  const input = document.getElementById("age");

  input.addEventListener("input", (e) => {
    console.log(e.target.validity);
  });
</script>
```

:::

- **Svojstvo `patternMismatch`**: Ovo će biti `true` ako vrednost ne odgovara definisanom standardnom formatu izraza.

## Putanja za `preventDefault()`

- **Opis**: Svaki događaj koji se dešava u DOM ima određeno podrazumevano ponašanje. Događaj klika na proveru menja stanje te provere, podrazumevano. Pritiskanje razmaknice na fokusirano dugme čini to dugme aktivnim. Metoda `preventDefault()` na ovim elementima `Event` sprečava da se to ponašanje desi.

:::interactive_editor

```html
<form id="form">
  <input type="text" placeholder="Try to submit" />
  <button type="submit">Submit</button>
</form>

<p id="status"></p>

<script>
  const form = document.getElementById("form");
  const status = document.getElementById("status");

  form.addEventListener("submit", (event) => {
    event.preventDefault();
    status.textContent = "Form submission prevented.";
  });
</script>
```

:::

## Predaja obrasca

- **Opis**: Postoji tri načina slanja forme. Prvi je kada korisnik klikne na dugme unutar forme koje ima atribut `type` postavljeno na `submit`. Drugi je kada korisnik klikne na dugme `Enter` u bilo kom području `input` koje se može uređivati unutar forme. Treći je putem poziva JavaScript za metode `requestSubmit()` ili `submit()` elementa `form`.
- **Atribut `action`**: Atribut `action` treba da ima URL ili relativnu putanju za trenutni domen. Ova vrednost određuje gde će forma pokušati da pošalje podatke – ako ne postavite atribut `action`, forma će poslati podatke na URL trenutne stranice.

```html
<form action="https://freecodecamp.org" method="GET">
  <input
    type="number"
    name="number"
    placeholder="Enter a number"
  />
  <button type="submit">Submit</button>
</form>
```

- **Sifa za `method`**: Ovaj atribut prihvata standardnu metodu `HTTP`, kao `GET` ili `POST`, i koristi tu metodu prilikom slanja zahteva na URL akciju. Kada nije definisana metoda, formular će koristiti zahtev `GET` podrazumevano. Podaci u formularu će biti zapisani kao parovi `name=value` od URL i dodati na akciju's URL kao parametri pretrigarskog argumenta.

```html
<form action="/data" method="POST">
  <input
    type="number"
    id="input"
    placeholder="Enter a number"
    name="number"
  />
  <button type="submit">Submit</button>
</form>
```

- **Sifa ya `enctype`**: Element/Polje `form` prihvata atribut `enctype`, koji predstavlja tip kodiranja podataka koji će biti korišćen. Ovaj atribut prihvata samo tri vrednosti: `application/x-www-form-urlencoded` (što je podrazumevano, šaljući podatke kao kodiran telo forme pomoću URL), `text/plain` (koji šalje podatke u čistom tekstu, u parovima `name=value` razdvojenim novim redovima), ili `multipart/form-data`, koji je namenjen za rukovanje formama sa otpremanjem fajlova.
