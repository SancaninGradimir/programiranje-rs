# --description--

Mnogi programski jezici i okruženja nude načine za rad sa datotekama i fasciklama. U Node.js to možeš da radiš pomoću modula `fs` (file system).

Ovaj tutorijal će osvatiti Node modul `fs`, kako radi, i tri različita načina na koja možete koristiti metode (methods) koje pruža ovaj modul.

Modul Node `fs` obuhvata načine rada sa fajlovima i direktorijumima, uključujući otvaranje i zatvaranje, čitanje i pisanje, kao i operacije brisanja.

Modul `fs` je standardni modul za Node.js, stoga je dostupan za upotrebu projekta samo ako ste postavili Node.js u vaše okruženje. Da biste koristili ovaj modul, uključite ga na sledeći način:

```js
const fs = require("fs");
```

Kapacitet modula `fs` je što vam omogućava da koristite njegove metode sinhrono i asinhrono. Metode modula `fs` su podrazumevano asinhrone, ali za svaku metodu postoji sinhrona verzija:

```js
fs.writeFile() // Asynchronous file writing
fs.writeFileSync() // Synchronous file writing

fs.readFile() // Asynchronous file reading
fs.readFileSync() // Synchronous file reading

fs.open() // Opens a file
fs.openAsBlob() // Opens as blob
fs.openSync() // Synchronous open
fs.opendir() // Opens directory
fs.opendirSync() // Synchronous directory open
```

Ovo se ovde ne završava. U skladu sa vašim potrebama, možete koristiti ove metode na tri načina:

- uz poziv povratka, na primer `fs.writeFile()` (nije istovremeno)
- uz obećanje ako preferirate sintaksu `async/await`, na primer `fs.promises.writeFile()`
- istovremeno, na primer, `fs.writeFileSync()`

Ovo je osnovna sintaksa za korišćenje sinhronog metoda:

```js
fs.writeFile("filePath", "content", "utf8", (err) => {
  if (err) {
    throw err;
  }
  console.log("File written to!");
});
```

Za verziju obećanja, možete to dovesti iz `fs/promises` ili povezati `promises` na vašu import od `fs`. Ovo je sintaksa:

```js
async function promisesExample() {
  try {
    await fs.promises.writeFile("filePath", "content", "utf8");
    console.log("File written to!");
  } catch (err) {
    console.error("Error:", err);
  }
}

promisesExample();
```

I ovo je živa verzija:

```js
try {
  fs.writeFileSync("filePath", "content", "utf8");
  console.log("File written to!");
} catch (err) {
  console.error("Error:", err);
}
```

Zapamti da je svaki sinhroni metod blokirajući. Drugim rečima, program će zaustaviti izvršavanje i sačekati da se operacija završi pre nego što nastavi sa sledećom linijom koda.

Za brze napomene, male projekte ili jednokratne vežbe, metode sa jednim niti mogu biti prikladne. Međutim, u stvarnim aplikacijama, blokirajući I/O (ulaz/izlaz) je problem jer može zaustaviti druge delove vašeg programa dok Node čeka da sistem fajlova završi svoje operacije. Ovo usporava stvari i utiče na performanse i korisničko iskustvo.

Zato postoje asinhroni načini. One omogućavaju drugim delovima vaše aplikacije da nastave sa radom dok Node obrađuje operacije fajlova u pozadini. Obično se koristi povratni poziv (callback), ali oni mogu postati komplikovani kako vaša aplikacija raste.

Obećanja sa `async/await` rešavaju taj problem. Još i dalje ne sprečava, ali kod izgleda kao standardni jednokratni kod i lakši je za održavanje.

Zbog toga, metoda koja se generalno oslanja na obećanje je preporučena danas, i to ćemo uključiti u ovaj ceo modul.

Puta `writeFile()` vam omogućava da pišete u postojeći fajl. Ako fajl ne postoji, kreira ga u trenutnom direktorijumu, pa piše navedeni sadržaj.

Ovo je kako koristiš metodu `writeFile()`:

```js
const fs = require("fs/promises");

async function writeToFile() {
  try {
    await fs.writeFile(
      "article.md",
      "## Node `fs` Module: The Complete Guide",
      "utf8",
    );
    console.log("File written to!");
  } catch (err) {
    console.error("Error writing to file:", err);
  }
}

writeToFile(); // File written to!
```

Ovo su sadržaji u fajlu article.md, nakon toga:

```md
## Node `fs` Module: The Complete Guide
```

Način za `appendFile()` vam omogućava da dodate u sadržaj postojeće datoteke. Ovako ga koristite:

```js
const fs = require("fs/promises");

async function appendToFile() {
  try {
    await fs.appendFile(
      "article.md",
      "\nIn this article, you will learn all there is to know about the Node fs module...",
      "utf8",
    );
    console.log("File appended to!");
  } catch (err) {
    console.log("Error appending to file:", err);
  }
}

appendToFile(); // File appended to!
```

Putanja za `readFile()` vam omogućava da vidite sadržaj fajla:

```js
const fs = require("fs/promises");

async function readFileContent() {
  try {
    const fileContent = await fs.readFile("article.md", "utf8");
    console.log("File content:", fileContent);
  } catch (err) {
    console.log("File read successfully");
  }
}

readFileContent();

/*
File content: ## Node `fs` Module: The Complete Guide

In this article, you will learn all there is to know about the Node fs module...
*/
```

Ako ne navedete kod karaktera za `utf8`, dobićete sadržaj fajla kao privremeni keš:

```js
async function readFileContent() {
  try {
    const fileContent = await fs.readFile("article.md");
    console.log("File content:", fileContent);
  } catch (err) {
    console.log("File read successfully");
  }
}

readFileContent();

/*

 54 68 65 20 43 6f 6d 70 6c 65 74 65 20 47 75 69 64 65 0a 0a 49 6e 20 74 68 69 73 20 ... 
72 more bytes>
*/
```

U sledećem tutorijalu, naučićete kako da upravljate ovakvim privremenim keš skladištem za Buffer modul Node.js.

Na kraju, putem `unlink()` možete obrisati fajl:

```js
const fs = require("fs/promises");

async function deleteFile() {
  try {
    await fs.unlink("article.ts");
    console.log("File deleted successfully!");
  } catch (err) {
    console.error("Error deleting file:", err);
  }
}

deleteFile();
```

Ako fajl ne postoji, dobićete poruku o grešci sličnu ovoj:

```bash
Error deleting file: Error: ENOENT: no such file or directory, unlink 'article.ts'
```

Ako postoji, fajl se briše, i dobijate poruku o uspehu koja je postavljena.
