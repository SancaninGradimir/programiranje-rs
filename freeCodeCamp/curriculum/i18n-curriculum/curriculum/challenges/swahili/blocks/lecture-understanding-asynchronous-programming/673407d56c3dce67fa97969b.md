# --interactive--

U prethodnim lekcijama, naučili ste o asinhronom programiranju, koje omogućava drugom kodu da nastavi da se izvršava dok čekamo zadatke koji zahtevaju vreme, kao što je dobijanje podataka sa servera, čitanje podataka iz fajla i slično.

`async`/`await`, izgrađen na obećanjima, olakšava pisanje i čitanje asinhronnog koda. Kada stavite ključnu reč `async` ispred funkcije, to znači da će ta funkcija uvek vraćati `Promise`. Ključna reč `await`, koja vam omogućava da sačekate da se `Promise` reši pre nego što nastavite sa sledećim redom koda, može se koristiti samo unutar asinhronih funkcija ili višeslojnih struktura modula. Evo primera koji pokazuje kako `async`/`await` funkcioniše:

:::interactive_editor

```js
async function delayedGreeting(name) {
  console.log("A Messenger entered the chat...");
  await new Promise(resolve => setTimeout(resolve, 2000));
  console.log(`Hello, ${name}!`);
}

delayedGreeting("Alice");
console.log("First Printed Message!");
```

:::

U ovom kodu, definišemo funkciju `async` koja je nazvana `delayedGreeting`. Unutar ove funkcije, koristimo `await` da zaustavimo izvršavanje na 2 sekunde. Nakon kašnjenja, ispisuje pozdrav.

Kada pozovemo ovu funkciju, videćeš da se `First Printed Message!` ispisuje pre pozdrava. To je zato što je funkcija asinhrona — ne blokira izvršavanje ostatka koda.

Jedna od najvećih prednosti `async`/`await` je upravljanje greškama. Ranije, često smo morali da koristimo metodu `.catch()` za obradu grešaka. Sa `async`/`await`, možemo koristiti klauzule `try`/`catch`. Evo primera:

```js
async function fetchUserData() {
  try {
    let response = await fetch(`https://api.example.com/users`);
    let userData = await response.json();
    console.log(userData);
  } catch (error) {
    console.log("Error fetching user data:", error);
  }
}

fetchUserData();
```

U ovom primeru koristimo `async`/`await` za preuzimanje podataka o korisniku iz API-ja. Ključna reč `await` koristi se dva puta: jednom da sačeka završetak `fetch` operacije, a drugi put da sačeka završetak parsiranja JSON-a. Ako tokom ovog procesa dođe do bilo kakve greške, ona će biti obrađena u `catch` bloku.
