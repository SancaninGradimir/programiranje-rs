# --description--

Nakon što je akcija kreirana i poslata, Redux skladište mora da zna kako da reaguje na tu akciju. Ovo je zadatak funkcije `reducer`. Reduceri u Redux-u odgovorni su za promene stanja koje nastaju kao odgovor na akcije. Funkcija `reducer` prima `state` i `action` kao argumente, i uvek vraća novo `state`. Važno je znati da je ovo **isključiva uloga** reducera. Nema nuspojava — ne poziva nikakvu funkciju za API i ne uključuje nikakve skrivene efekte. Reducer je čista funkcija koja uzima stanje i akciju, a zatim vraća novo stanje.

Kanuni nyingine muhimu katika Redux ni kwamba `state` ni ya kusoma tu. Kwa maneno mengine, kitendakazi cha `reducer` lazima **kila mara** kirudishe nakala mpya ya `state` na kisibadilishe hali moja kwa moja. Redux haifanyi utekelezaji wa hali isiyobadilika, hata hivyo, wewe ndiye unawajibika kuitekeleza katika msimbo wa vitendakazi vyako vya reducer. Utazoea hili katika changamoto zijazo.

# --instructions--

Mhariri wa msimbo una mfano wa awali pamoja na mwanzo wa kitendakazi cha `reducer` kwako. Jaza mwili wa kitendakazi cha `reducer` ili ikiwa kitapokea kitendo cha aina ya `'LOGIN'` kirudishe kitu cha hali chenye `login` kikiwa na thamani ya `true`. Vinginevyo, kirudishe hali ya sasa ya `state`. Kumbuka kwamba hali ya sasa ya `state` na kitendo kilichotumwa cha `action` vinapitishwa kwa reducer, hivyo unaweza kupata aina ya kitendo moja kwa moja kwa `action.type`.

```js
assert(loginAction().type === 'LOGIN');
```

```js

```

```js

    store.dispatch(loginAction());
    const afterState = store.getState();
    return initialState.login === false && afterState.login === true;
  })()
);
```

```js

    store.dispatch({ type: '__TEST__ACTION__' });
    let afterTest = store.getState();
    return typeof afterTest === 'object' && afterTest.hasOwnProperty('login');
  })()
);
```
