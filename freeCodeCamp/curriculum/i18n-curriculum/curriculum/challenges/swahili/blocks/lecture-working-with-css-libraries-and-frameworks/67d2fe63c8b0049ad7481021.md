# --description--

Preprocessor za CSS je alat koji dodaje moćne karakteristike kao što su parametri, mixini, interni raspored i nasleđivanje klasa na standardni CSS.

Neki od ovih atributa, kao što su kriterijumi i interno postavljanje, sada su podržani ili im je dodata podrška u originalnom CSS, i ovaj trend bi mogao da se nastavi. Međutim, ranije verzije originalnog CSS nisu podržavale ove atribute.

Zato su preprocesori za CSS počeli da se široko koriste ranije. Oni uzimaju pravila i stilove od CSS koje pišete, zajedno sa dodatom sintaksom, i konvertuju ih u izvorni fajl CSS koji pregledači mogu da razumeju.

Korišćenjem preprocesora CSS, možete organizovati svoj kod CSS na način koji je ponovno upotrebljiv i logičniji. Takođe ćete imati pristup moćnim funkcijama, kao što su mixini, koje ne dolaze direktno sa CSS.

Neki popularni preprocesori za CSS su Sass, Less i Stylus.

Tuzungumzie Sass.

Sass stoji za „Syntactically Awesome Style Sheets“. Kompatibilno je sa svim verzijama CSS i održava ga velika zajednica programera.

Sass podržava značajke kao što su:

- Kriterijumi, ili može sačuvati i ponovno koristiti vrednost u celokupnom stilskom listu.

- Pravila za CSS koja su uključena, ili da stvori vidljiv obrazac u vašoj datoteci.

- Moduli, kako bi mogli da podelite svoje stilove u više fajlova za stil.

- Mixins, da ponovo koristi deklaracije za CSS na celoj vašoj veb stranici.

- Nasleđe, da mnogi birači CSS mogu da dele imovinu.

- Operateri, za osnovne računarske/aritmetičke operacije.

Sass takođe podržava dve sintakse.

Sintaksa SCSS znači Sassy CSS, što je proširena verzija od CSS. To znači da SCSS dodaje osnovnu sintaksu za CSS.

SCSS je najčešće korišćena sintaksa koju ćete naići kada koristite Sass. Zahteva korišćenje okruglih zagrada (`{` i `}`) oko svojstava za CSS i tačka-zarez (`;`) na kraju deklaracija CSS, kao što je originalno CSS.

Ovi fajlovi imaju dodatak od `.scss`. Evo primera:

```scss
$primary-color: #3498eb;

header {
  background-color: $primary-color;
}
```

Ono što vidite ovde iznad je svojstvo definisano u SCSS-u. Ovo svojstvo se koristi u pravilu za CSS dole.

Uporedimo SCSS sa retko korišćenom sintaksom, sintaksom formatiranim razmakima. Ovo je takođe poznato kao "Sass sintaksa" jer je to bila prvobitna Sass sintaksa.

Ova sintaksa zavisi od razmaknih znakova za definisanje pravila. Evo primera:

```scss
$primary-color: #3498eb

header
  background-color: $primary-color
```

Proverite da nema vitičastih zagrada oko pravila `CSS`, niti crtice na kraju izjava za `CSS`. Gore, takođe možete videti parametar. Proverite da njegovo ime počinje sa znakom dolara (`$`) u ovoj sintaksi, što se razlikuje od originalnih parametara `CSS`, ali radi na sličan način.

Sass takođe ima moćnu karakteristiku nazvanu mixins.

Mixins omogućavaju da spojite više svojstava `CSS` i njihove vrednosti pod imenom, te ponovo koristite taj deo koda `CSS` u vašem stilskom listu.

Ovo sprečava da se vaš kod CSS ponavlja često i čini ga lakšim za održavanje, jer ako promenite nešto na tom mestu, promene važe svuda gde koristite mixin.

Pošto mixinovi mogu imati specifična i razumljiva imena, oni mogu pomoći u razumevanju šta svaki deo koda CSS radi, što može učiniti vaše stilove lakim za razumevanje.

Mixini takođe povećavaju stabilnost u stilovima korišćenjem standardnih obrazaca.

Ovo je primer mixina u SCSS sintaksi:

```scss
@mixin center-flex {
  display: flex;
  justify-content: center;
  align-items: center;
}
```

Da biste definisali mixin, počinjete tako što pišete deklaraciju za at-rule `@mixin`, praćenu imenom mixina. U ovom slučaju, mixin se zove `center-flex`. Ima tri svojstva za CSS za centriranje elemenata korišćenjem Flexbox-a.

Kada je vaš mixin definisan, koristićete deklaraciju `@include` at-rule da uključite ta svojstva u pravilo CSS. Samo je potrebno napisati `@include` praćen nazivom mixina. U ovom slučaju, naziv je `center-flex`:

```scss
section {
  @include center-flex;
  height: 500px;
  background-color: #3289a8;
}
```

Ovo je kompletan kod sa mixinom i pravilom za CSS:

```scss
@mixin center-flex {
  display: flex;
  justify-content: center;
  align-items: center;
}

section {
  @include center-flex;
  height: 500px;
  background-color: #3289a8;
}
```

Možete uvesti mixin u pravila CSS kada vam zatreba. Ako trebate napraviti bilo kakve izmene, morate samo promeniti sam mixin, a promene će se desiti svuda automatski. Ovaj primer je u SCSS sintaksi. Proverite da imate vitičaste zagrade i tačku-zarez.

Ovo je isto u sintaksi formatirano sa razmakima, poznato i kao Sass sintaksa, bez vitičastih zagrada ili dvotačka:

```scss
@mixin center-flex
  display: flex
  justify-content: center
  align-items: center

section
  @include center-flex
  height: 500px
  background-color: #3289a8
```

I ovo je kod za CSS koji je preveden, koji će pretraživač prevesti nakon što se fajl promeni u CSS originalni:

```css
section {
  display: flex;
  justify-content: center;
  align-items: center;
  height: 500px;
  background-color: #3289a8;
}
```

Proverite da uključujete tri mixin svojstva gore: `display`, `justify-content`, i `align-items`.

Ovo je samo jednostavan primer onoga što preprocesori za CSS mogu da urade. Imaju mnogo moćnih funkcija koje vredi naučiti i koristiti u vašem svakodnevnom radnom toku.

Međutim, postoje potencijalni nedostaci koje biste trebali znati.

Prvo, preprocessors za CSS moraju da promene pravila za CSS u standardno CSS. Iako je režija ovog procesa konverzije često mala po vremenu i resursima, to je nešto što treba uzeti u obzir za stvarne projekte.

Takođe, mogu uzrokovati probleme pri ispravljanju grešaka, jer pretraživači koriste CSS direktno prevedeno. Utvrđivanje šta izaziva problematičan stil iz proširene sintakse može zahtevati dodatne korake u poređenju sa standardnim CSS.

Međutim, prednosti preprocesora za CSS su obično veće od njihovih nedostataka, posebno za složene projekte. Mogu mnogo pomoći u pisanju čistog, ponovo upotrebljivog, neponavljajućeg i skalabilnog CSS.
