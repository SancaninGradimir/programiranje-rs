# --description--

Obrada događaja je ključni deo svakog veb-sajta koji angažuje korisnika.

React pruža moćan i stabilan način za rukovanje događajima putem svog Sistema sinteetičkih događaja, koji je obloga za prirodne događaje kao što su `click`, `keydown`, i `submit` naučeno u prethodnim lekcijama.

Ovaj omotač kompatibilan sa svim pregledačima obezbeđuje da događaji rade isto u svim pregledačima kako bi se izbegle razlike.

Istražite kako događaji funkcionišu u React da počnete da ih koristite u svojim projektima.

U React-u, rukovalac događaja funkcioniše slično kao izvorni događaji pregledača, ali uz nekoliko izmena.

Umesto korišćenja imena svojstava događaja u malim slovima kao `onclick` na `onsubmit`, React koristi camelCase, kao što su `onClick` na `onSubmit`.

Pored toga, umesto korišćenja tekstualnog niza za označavanje tipa događaja, React očekuje funkciju za rukovaoca događajem.

Funkcija rukovatelja događajem se prosleđuje komponenti kao prop, a tip događaja kao `onClick` ili `onSubmit` koriste se kao propsi u JSX-u.

Ovo je podsetnik kako da se rukuje sa događajem klika u standardnom HTML:

```html
<button onclick="alert('Button clicked!')">Click Me</button>
```

Evo kako to isto možete uraditi u React-u:

```jsx
function handleClick() {
  console.log("Button clicked!");
}

<button onClick={handleClick}>Click Me</button>;
```

U ovom primeru, `handleClick` piše poruku u konzolu kada korisnik klikne na dugme.

Zapamti da ti ne trebaju obične zagrade nakon `handleClick` u atributu `onClick`, jer prosleđuješ referencu funkcije, a ne direktno njeno ime.

U React, funkcije rukovaoca događaja obično počinju sa prefiksom `handle` što ukazuje da su uključene u obradu događaja, kao `handleClick` ili `handleSubmit`.

Kada korisnička akcija pokrene događaj, React prosleđuje objekat Sintetičkog Događaja vašem rukovaocu. Ovaj objekat funkcioniše kao nativni objekat događaja u standardnom JavaScript, pružajući svojstva kao `type`, `target`, i `currentTarget`.

Možete proslediti `event` kao parametar funkciji rukovaoca i napisati u konzolu da proverite njegovu strukturu:

```js
function handleClick(event) {
  console.log(event);
}
```

Da biste sprečili podrazumevano ponašanje, kao što je osvežavanje pretraživača tokom događaja `onSubmit`, na primer, možete pozvati metodu `preventDefault()`:

```jsx
function handleSubmit(event) {
  event.preventDefault();
  console.log("Form submitted!");
}

<form onSubmit={handleSubmit}>
  <input type="text" />
  <button>Submit</button>
</form>;
```

Takođe možete sprečiti događaj da se propagira do roditeljskih komponenti pozivanjem `event.stopPropagation()`.

Ponekad, kada koristite posebne događaje kao komponente za „futa“ i „hariri“, možda želite da prosledite dodatne podatke hendleru događaja. Ovo možete učiniti tako što ćete obaviti hendlera unutar strelične funkcije na istom redu:

```jsx
function handleDelete(id) {
  console.log("Deleting item:", id);
}

<button onClick={() => handleDelete(1)}>Delete Item</button>;
```

Nije dobro koristiti rukovače događaja unutar linije u React jer React efikasno upravlja ponovnim prikazom i izbegava probleme sa performansama.

U standardnom JavaScript, rukovaoci događajima unutar linije mogu uzrokovati probleme sa performansama kreiranjem novih funkcija svaki put kada se ponovo renderuje, jer ne postoji stvarni DOM za poboljšanje procesa.
