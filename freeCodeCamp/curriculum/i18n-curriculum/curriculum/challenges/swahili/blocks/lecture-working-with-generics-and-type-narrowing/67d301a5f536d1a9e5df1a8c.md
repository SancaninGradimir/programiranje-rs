# --description--

Biće trenutaka kada imate vrednost širokog tipa i morate je svesti na specifičniji tip. Na primer, možda morate osigurati da nešto ispunjava interfejs koji ste kreirali. Ili da niz karaktera spada u definisanu listu vrednosti. Postoji nekoliko načina da se ovo postigne.

Prvi način je smanjenje korišćenjem stvarne vrednosti. Razmislite o našem primeru iz poslednje lekcije:

```js
const email = document.querySelector<HTMLInputElement>("#email");
console.log(email.value);
```

Primećujemo grešku kompilatora kada pokušavamo da dobijemo svojstvo `value` od `email`, jer `email` može biti `null`. Međutim, možemo koristiti uslovne informacije da proverimo da li je `email` istinska vrednost pre dobijanja tog svojstva:

```js
const email = document.querySelector<HTMLInputElement>("#email");
if (email) {
  console.log(email.value);
}
```

U ovom ažuriranom primeru, jer `null` nije stvarna vrednost, TypeScript može utvrditi da `email` MORA biti element od `input` unutar sekcije uslova. Stoga ne izaziva grešku kompajlera.

Analiza stvarne vrednosti takođe može da radi u suprotnom smeru:

```js
const email = document.querySelector<HTMLInputElement>("#email");
if (!email) {
  throw new ReferenceError("Could not find email element!")
}
console.log(email.value);
```

Na ovaj način, bacamo grešku ako ``email`` je lažna vrednost. ``null`` je lažna vrednost. Bacanje greške prekida izvršavanje logike ovog koda, što znači da kada dođemo do poziva ``console.log()`` `TypeScript` znače da ``email`` ne može biti ``null``.

Serijski spoj je takođe vrsta sužavanja tipa, pod pretpostavkom da dostupnost svojstva ne može postojati ako je vrednost `email` jednaka `null`.

```js
const email = document.querySelector<HTMLInputElement>("#email");
console.log(email?.value);
```

Ali šta je sa drugim tipom? Pa, takođe možete smanjiti tip koristeći parametar `typeof`. Pogledajte primer kriterijuma koji smo pokazali da može biti niz slova ILI brojeva:

```js
const myVal = Math.random() > 0.5 ? 222 : "222";
console.log(myVal / 10)
```

U ovom primeru, vidimo grešku parsiranja koda jer ne možemo izvesti proračun na vrednosti niza karaktera. Ali možemo koristiti uslovne informacije za proveru `typeof` parametra `myVal`:

```js
const myVal = Math.random() > 0.5 ? 222 : "222";
if (typeof myVal === "number") {
  console.log(myVal / 10);
}
```

Zato što smo koristili glavni termin za `typeof`, TypeScript sada zna da `myVal` mora biti broj, i možemo sigurno izvesti proračun sa njim.

Ali, šta je sa složenijim vrstama stvari? Ako je nešto o čemu se govori i potiče iz klase objekata, možete koristiti glavni termin `instanceof` za smanjenje vrsta. Vraćajući se našem primeru od `querySelector()`:

```js
const email = document.querySelector("#email");
```

Umesto da prosleđujete opšti tip i kažete sa TypeScript, koja je karakteristika, možemo koristiti `instanceof` da smanjimo tip i napišemo sigurniji kod:

```js
const email = document.querySelector("#email");

if (email instanceof HTMLInputElement) {
    console.log(email.value);
}
```

Ovaj put može izgledati slično prethodnom, ali `instanceof` je potvrda u vreme izvršavanja - što znači, ako smo na neki način pogrešili tip od TypeScript, naš kod za JavaScript i dalje potvrđuje da `email` je karakteristika `input`.

Sada, pogledajte primer gde dobijamo nešto od `User` iz API i pokušavamo da ispišemo informacije:

```js
interface User {
    name: string;
    age: number;
}

const printAge = (user: User) => 
  console.log(`${user.name} is ${user.age} years old!`)

const request = await fetch("url")
const myUser = await request.json();
printAge(myUser);
```

Biće nam greška kompilatora kada pokušamo da prođemo `myUser` u funkciju jer, iako znamo da API vraća nešto ispravno, TypeScript ne zna. A metoda `.json()` nema opšti tip.

Najlakši način za rešavanje ovog problema biće promena tipa:

```js
interface User {
    name: string;
    age: number;
}

const printAge = (user: User) => 
  console.log(`${user.name} is ${user.age} years old!`)

const request = await fetch("url")
const myUser = await request.json() as User;
printAge(myUser);
```

Ali svaki put kada promeniš tip, suštinski oslabiš sposobnost TypeScript da detektuje potencijalne greške. Stoga, umesto da menjaš tip, možeš napisati čuvar tipa:

```js
interface User {
    name: string;
    age: number;
}

const isValidUser = (user: unknown): user is User => {
  return !!user && 
    typeof user === "object" &&
    "name" in user &&
    "age" in user;
}
```

Vrsta vraćanja ovde je važan deo definicije ove funkcije. Sintaksa `user is User` pokazuje da naša funkcija vraća booleanski vrednost, što u trenutku `true` znači da vrednost `user` zadovoljava interfejs `User`. Zatim vršimo osnovnu proveru kako bismo osigurali da struktura objekta `user` odgovara – zapamtite korišćenje booleanskog kastovanja (`!!user`) i kastovanje `typeof`. Moramo to učiniti jer `typeof null` vraća `"object"`:

```js
interface User {
    name: string;
    age: number;
}

const isValidUser = (user: unknown): user is User => {
  return !!user && 
    typeof user === "object" &&
    "name" in user &&
    "age" in user;
}

const printAge = (user: User) => 
  console.log(`${user.name} is ${user.age} years old!`)

const request = await fetch("url")
const myUser = await request.json() as User;
if (isValidUser(myUser)) {
    printAge(myUser);
}
```

Sada, kombinovanjem svake naše logike zajedno, više ne dobijamo greške kompilatora i možemo uspješno izgraditi svoj kod.

Sužavanje tipa je moćna karakteristika koja vam pomaže da pišete sigurniji kod, sa manje grešaka – ali zapamtite da TypeScript nisu uvek jednostavni, pa izbegavajte navike kao što je menjanje tipa vrednosti bez sužavanja.
