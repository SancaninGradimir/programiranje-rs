# --interactive--

Operater sekvencijalnog spajanja (`?.`) je važan alat u JavaScript koji vam omogućava siguran pristup svojstvima objekta ili pozivanje metode bez brige da li postoje ili ne. To je kao sigurnosna mreža za rad sa stvarima koje možda imaju nedostajuće delove.

:::interactive_editor

```js
const person = {
  name: "Alice",
  age: 30
};

console.log(person.name); // "Alice"
console.log(person.job); // undefined
```

:::

U ovom primeru, `person.name` postoji, pa beleži `Alice`. Ali `person.job` ne postoji, pa vraća `undefined`.

Dakle, recimo da želimo da dobijemo kvalitet nečega što možda ne postoji:

```js
const person = {
  name: "Alice",
  age: 30
};

console.log(person.address.street); // This will throw an error!
```

Ovaj primer će nam dati `Uncaught TypeError`. Pošto je `person.address` jednako `undefined`, ne možemo dobiti svojstvo `street`. Ovde postaje koristan operater sekvencijalne konkatenacije. Evo primera korišćenja operatera sekvencijalne konkatenacije:

:::interactive_editor

```ts
const user = {
  name: "John",
  profile: {
    email: "john@example.com",
    address: {
      street: "123 Main St",
      city: "Somewhere"
    }
  }
};

console.log(user?.profile?.address?.street); // "123 Main St"
console.log(user?.profile?.phone?.number);   // undefined
```

:::

Korišćenjem operatora sekvencijalnog spajanja, kažemo JavaScript da nastavi sa operacijom samo ako nešto (ili vrednost pre `?.`) postoji i nije `null` ili `undefined`.

Ako je vrednost pre `?.` jednaka `null` ili `undefined`, JavaScript vraća `undefined` umesto da pokuša nastaviti sa operacijom i izbaci grešku.

Zapamtite, operater za sekvencijalnog povezivanja su najvažniji kada ne znate da li postoji atribut ili metoda. Pomaže u sprečavanju grešaka i čini vaš kod robusnijim.
