# --description--

Sledeci tip petlje koji cete nauciti zove se `do...while` petlja. Zove se tako zato sto prvo izvrsi jedan prolaz koda unutar petlje bez obzira na uslov, a zatim nastavlja da se izvrsava dok je uslov u `while` delu `true`.

```js
const ourArray = [];
let i = 0;

do {
  ourArray.push(i);
  i++;
} while (i < 5);
```

Gornji primer radi slicno kao druge petlje i niz postaje `[0, 1, 2, 3, 4]`. Medjutim, ono sto `do...while` razlikuje od ostalih petlji je ponasanje kada uslov ne prodje vec na prvoj proveri. Pogledajmo to u praksi. Evo klasicne `while` petlje koja izvrsava kod dok je `i < 5`:

```js
const ourArray = []; 
let i = 5;

while (i < 5) {
  ourArray.push(i);
  i++;
}
```

U ovom primeru inicijalizujemo `ourArray` kao prazan niz i postavljamo `i` na 5. Kada se izvrsi `while` petlja, uslov je `false` jer `i` nije manje od 5, pa se kod unutar petlje ne izvrsava. Rezultat je da se u `ourArray` ne dodaje nijedna vrednost i niz ostaje `[]` nakon izvrsavanja koda. Sada pogledajte `do...while` petlju:

```js
const ourArray = []; 
let i = 5;

do {
  ourArray.push(i);
  i++;
} while (i < 5);
```

U ovom slucaju, `i` inicijalizujemo na 5, kao i kod `while` petlje. Kada predjemo na sledecu liniju, nema uslova za proveru, pa odmah izvrsavamo kod unutar viticastih zagrada. Dodajemo jedan element u niz i uvecavamo `i` pre provere uslova. Kada proverimo `i < 5` na poslednjoj liniji, vidimo da je `i` sada 6, sto ne prolazi uslov, pa izlazimo iz petlje. Na kraju ovog primera vrednost `ourArray` je `[5]`. Dakle, `do...while` garantuje da ce se kod unutar petlje izvrsiti bar jednom. Hajde da to primenimo i ubacimo vrednost u niz.
