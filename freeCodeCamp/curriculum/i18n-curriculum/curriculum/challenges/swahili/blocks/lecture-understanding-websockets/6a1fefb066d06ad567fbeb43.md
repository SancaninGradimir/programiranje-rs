# --description--

WebSockets omogućavaju klijentu i serveru da komuniciraju u realnom vremenu.

Obično, klijentske aplikacije kao što su web i mobilne aplikacije koriste HTTP. Sa HTTP, klijent šalje zahtev, a server šalje odgovor. Ovo se naziva ciklus zahteva i odgovora, kao što ste naučili.

Nakon što se uspostavi konekcija i razmene podataka, konekcija se zatvara. Ovo je dobro za mnogo stvari, kao što su učitavanje stranica, dohvat ili slanje podataka. Ali spor je i nepouzdano za aplikacije u stvarnom vremenu.

Na primer, zamislite aplikaciju za razgovor. Ako odlučite da koristite HTTP za to, klijent mora stalno da traži podatke od servera, i da se usponi konekcija, stignu podaci, pa se konekcija zatvori nakon svake poruke.

Ovo je problem koji WebSockets rešavaju. Oni stvaraju trajno povezivanje između klijenta i servera. Kada se konekcija uspostavi, obe strane mogu razmenjivati podatke u bilo kom trenutku putem real-time kanala sve dok ne bude poslat signal za zatvaranje sa jedne strane na drugu.

Iako su WebSockets različiti od HTTP, svaka WebSocket konekcija počinje sa HTTP, u procesu koji se zove handshake.

Nakon toga klijent zahteva da se konekcija konvertuje u WebSocket. Kada server prihvati, šalje odgovor `101 Switching Protocols` koji nadograđuje konekciju na WebSocket. Nakon toga, konekcija ostaje otvorena i započinje komunikacija u realnom vremenu.

Tok se vidi ovako:

1. Klijent se povezuje na server preko HTTP
2. Server prihvata i nadograđuje konekciju u WebSocket
3. Koneksija ostaje otvorena
4. Obje strane razmenjuju podatke bilo kada dok koneksija nije zatvorena

Ovaj protok je dvosmeran, nije kao zahtev-odgovor za HTTP.

```md
Client <---> Server
```

Za HTTP, standardni tok prati obrazac zahtev-odgovor:

```md
Client ---> Request
Server ---> Response
```

O tome kako se WebSockets razlikuju od HTTP, HTTP, oni su kao razmena pisama, a WebSocket je kao telefonski poziv. Pišeš pismo, čekaš odgovor, i razgovor se završava. Ali sa telefonskim pozivom, ti i primaoc razgovarate u realnom vremenu.

WebSockets pokreću aplikacije u realnom vremenu kao što su chat aplikacije, multiplayer onlajn igre, uživo sportski feedovi/informacije, dashboardi tržišta akcija i alati za saradnju poput Google Docs-a.
