# --description--

U ovom kursu, naučićete kako da radite sa osnovnim tipovima podataka i kako ti tipovi funkcionišu sa parametrima.

Za tipove osnovnih podataka `string`, `null`, `undefined`, `number`, `boolean`, na `bigint`, TypeScript pruža ključne reči odgovarajućih tipova.

Za ovu lekciju, razmotrićemo osnovne tipove koji se često koriste i neće detaljno pokrivati `bigint`. Tip podataka za `bigint` je tip broja u JavaScript koji se koristi za predstavljanje brojeva većih od standardnog integer tipa.

Ispitajmo svaki od ovih osnovnih tipova počevši od `string`. Evo kako možete napisati niz znakova u TypeScript:

```ts
let firstName: string = "Angie";
```

Za razliku od normalnog JavaScript, parametar `firstName` se očekuje da ima tip `string`. Stoga ako pokušate postaviti novu numeričku vrednost, dobićete grešku kao što je ova:

```ts
let firstName: string = "Angie";
firstName = 9 // Type 'number' is not assignable to type 'string'.
```

Sledeći tip koji ćemo ispitati je tip `number`. Ovde je primer:

```ts
let age: number = 16;
```

Možete uneti kompletan broj i decimalne brojeve za tip `number`. Ali ako pokušate da unesete vrednost koja je tip `bigint`, dobićete grešku kao ovu:

```ts
let age: number = 16;

// Type 'bigint' is not assignable to type 'number'.
age = 123n;
```

Sledeća vrsta koju ćemo istražiti je tip od `boolean`. Evo nekoliko primera:

```ts
let isLoggedIn: boolean = true;
let isAdmin: boolean = false;
```

Dve poslednje vrste su tipovi od `null` i `undefined`.

Kao što zapamtite iz prethodnih lekcija za JavaScript, `null` predstavlja namernu odsutnost vrednosti. `undefined` znači da je parametar deklarisan, ali mu nije dodeljena vrednost.

Videćeš primere kako da radiš sa ovim tipovima kasnije kada naučiš o funkcionalnim tipovima i tipovima unije.

Sada, nakon što smo videli kako tipli deskripcije funkcionišu sa parametrima, postavlja se pitanje da li je potrebno uvek eksplicitno definisati tip za vaše parametre.

Ako imate jednostavna objašnjenja ovakve vrste, onda možda nećete morati da koristite eksplicitni tip.

```ts
let firstName: string = "Angie";
```

Razlog je što će TypeScript pokušati da direktno utvrdi vrstu na osnovu njegove vrednosti. Stoga, ako koristite ovo, TypeScript će znati da je niz karaktera.

```js
let firstName = "Angie";
```

Trebali biste definisati tip jasno za vaše parametre kada TypeScript ne može da utvrdi tip jasno ili kada želite da prisilite određeni tip, u situacijama kao što su funkcionalni parametri ili složeni objekti.
