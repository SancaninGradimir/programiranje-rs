## Biblioteka za sisteme rada JavaScript

- Biblioteke i framework-ovi za JavaScript pružaju brka rešenja za uobičajene probleme i ubrzavaju razvoj dostavljanjem gotovog koda.
- Biblioteke se uglavnom fokusiraju na pružanje rešenja za specifične zadatke, kao što su upravljanje DOM, rukovanje događajima ili obrada AJAX zahteva.
- Neki primeri biblioteka za JavaScript su jQuery i React.
- Framework-ovi, s druge strane, pružaju jasniju strukturu za izgradnju aplikacija. Često dolaze sa setom pravila i konvencija koje programeri moraju pratiti.
- Primeri framework-ova su Angular i Next.js, meta-framework za React.
- **Jednonoćne aplikacije** (SPAs) su web aplikacije koje učitavaju jednu stranicu HTML i dinamički ažuriraju tu stranicu dok korisnik komunicira sa aplikacijom, bez ponovnog učitavanja cele stranice.
- SPAs koriste JavaScript za upravljanje stanjem aplikacije i prikaz sadržaja. To se često radi korišćenjem framework-ova koji pružaju odlična sredstva za izgradnju složenog korisničkog interfejsa.
- Neki problemi povezani sa SPA-ima su:

- Čitači ekrana loše rade sa dinamički ažuriranim sadržajem.
- URL se ne menja kada korisnik navigira unutar aplikacije, što može otežati označavanje, povratak ili deljenje specifičnih stranica.
- Vreme predmemorijalizacije može biti dugo ako je aplikacija velika jer svi resursi moraju biti preuzeti unapred.

## React

- React je popularna biblioteka za JavaScript za izradu korisničkih interfejsa i web aplikacija.
- Glavna koncepcija React je kreiranje ponovno upotrebljivih UI komponenti koje se mogu ažurirati i prikazivati nezavisno kako podaci menjaju.
- React omogućava programerima da opišu kako bi UI trebalo da izgleda u zavisnosti od stanja aplikacije. Zatim, React ažurira i prikazuje odgovarajući deo kada se podaci ili stanje promene.

## Sekcija za React

- Komponente su osnovne građevne jedinice za aplikacije React koje omogućavaju programerima da razbijaju složene korisničke interfejse na male, upravljive delove.
- Korisnički interfejs se piše pomoću JSX-a, sintaksičkog proširenja za JavaScript koje omogućava programerima da pišu kod sličan HTML-u unutar JavaScript-a.
- Komponente su u svojoj srži JavaScript funkcije ili klase koje vraćaju deo UI-ja.

Ovo je primer jednostavnog dela React koji prikazuje poruku pozdrava:

```jsx
function Greeting() {
  const name = 'Anna';
  return <h1>Welcome, {name}!</h1>;
}
```

Da biste koristili ovaj deo, možete jednostavno nazvati:

```jsx
  <Greeting />
```

## Dostavljanje i transportovanje delova za React

- Transferovanje čini deo dostupnim drugim fajlovima. Povraćanje omogućava fajlu da koristi prebačeni deo negde drugde.
- Postoje dve vrste prenosa u React: transferovanje default i transferovanje imena.

**Transportacija default**

- Fajl može imati samo jednu pošiljku od default. Ovo je bolje za fajl koji sadrži jednu glavnu komponentu.
- Delove možete poslati kao pošiljku od default na ovaj način:

```jsx
// City.js
function City() {
  return <p>New York</p>;
}

export default City;
```

- Takođe možete prebaciti na jednu liniju i opis sekcije:

```jsx
export default function City() {
  return <p>New York</p>;
}
```

- Da bi se izvršio transport od default, koristi reč `import` bez zagrade:

```jsx
// App.js
import City from './City';

function App() {
  return (
    <div>
      <h1>My favorite city is:</h1>
      <City />
    </div>
  );
}
```

**Prenos imena**

- Prenos imena omogućava fajlovima da dele mnogo delova. Za razliku od prenosa default, moraju ih prebaciti koristeći originalno ime sa kojim su bili prebačeni.
- Možete preneti mnogo delova iz jednog fajla ovako:

```jsx
// Animals.jsx
export function Cat() {
  return <h2>Mr. Whiskers</h2>;
}

export function Dog() {
  return <h2>Fido</h2>;
}
```

- Da biste izvršili prenos imena, koristite vitičaste zagrade i stvarna prenesena imena:

```jsx
import { Cat, Dog } from './Animals';
```

- Možete promeniti naziv dostave imena prilikom isporuke korišćenjem reči `as`:

```jsx
import { Cat as Kitty } from './Animals';
```

**Kombinacija transporta za default i imena**

- Datoteka može imati jedan prenos default i više prenosa imena istovremeno:

```jsx
// Animals.jsx
export default function Cat() {
  return <h2>Mr. Whiskers</h2>;
}

export function Dog() {
  return <h2>Fido</h2>;
}
```

- Kada dođete na mešovitu pošiljku, default dolazi prvo (bez zagrada), praćena pošiljkom imena (unutar zagrada):

```jsx
import Cat, { Dog } from './Animals';
```

## Pokretanje projekta za React koristeći Vite

- Alati za inicijalizaciju projekata putem CLI-ja nude jednostavan i brz način za pokretanje novih projekata, omogućavajući programeru da se fokusira na pisanje koda umesto upravljanja konfiguracijom.
- Vite je popularan alat za inicijalizaciju projekata i može se koristiti sa React.
- Za kreiranje novog projekta sa Vite-om, možete koristiti sledeću komandu u vašem terminalu:

```bash
npm create vite@latest my-react-app -- --template react
```

Ova komanda kreira novi projekat React nazvan `my-react-app` koristeći šablon React od Vite. U direktorijumu projekta, videćete fajl `package.json` sa zavisnostima projekta i komandama navedenim unutra.

- Da biste pokrenuli projekat, idite u direktorijum projekta i koristite sledeće komande:

```bash
cd my-react-app # path to the project directory
npm install # installs the dependencies listed in the package.json file
```

- Kada se instalira zavisnost, pronaći ćete novu fasciklu u vašem projektu pod nazivom `node_modules`.

- Mapa `node_modules` je mesto gde se svi paketi i biblioteke koje vaš projekat zahteva čuvaju.

- Da biste pokrenuli svoj projekat, koristite sledeću komandu:

```bash
npm run dev
```

- Nakon toga, otvorite svoj pretraživač i idite na `http://localhost:5173` da biste videli kako vaš program React radi.

- Da biste videli kod početne šablone, možete ući u svoj projekat u složku `src` i pronaći ćete fajl `App.jsx`.

## Prosleđivanje prosleđenih parametara (props) u sekcije za React

- Unutar React, props (premljeni parametri) su način za prenošenje podataka od roditeljske komponente ka detinjoj komponenti. Ovaj sistem je potreban za kreiranje ponovo upotrebljivih i dinamičnih UI komponenti.
- Props mogu biti bilo koja vrednost JavaScript. Da biste prenosili props od roditelja ka detetu, dodajete props kao atribute kada koristite detinjoj komponentu unutar JSX-a roditelja. Evo jednostavnog primera:

```jsx
// Parent component
function Parent() {
  const name = 'Anna';
  return <Child name={name} />;
}

// Child component
function Child(props) {
  return <h1>Hello, {props.name}!</h1>;
}
```

Možete proslediti mnogo props-a koristeći *spread* operator ``(...)``, nakon što ih pretvorite u objekat. Evo primera:

```jsx
// Parent component
function Parent() {
  const person = {
    name: 'Anna',
    age: 25,
    city: 'New York'
  };
  return <Child {...person} />;
}
```

U ovom kodu, spread operater ``{...person}`` menja objekat od person-a u privatne props koje se prosleđuju komponenti Child.

## Prikaz po uslovima u React

- Uslovno prikazivanje u React omogućava vam kreiranje dinamičkih korisničkih interfejsa. Koristi se za prikaz različitog sadržaja u zavisnosti od uslova ili stanja unutar vaše aplikacije.
- Postoji nekoliko načina za uslovno prikazivanje sadržaja u React. Jedan uobičajen način je korišćenje ternarnog operatora. Evo primera:

```jsx
function Greeting({ isLoggedIn }) {
  return (
    <div>
      {isLoggedIn ? <h1>Welcome back!</h1> : <h1>Please log in</h1>}
    </div>
  );
}
```

- Drugi način za prikaz sadržaja uslovno je korišćenje logičkog operatora AND (`&&`). Ovo je korisno kada želite da prikažete sadržaj samo ako je određeni uslov ispavljen. Evo primera:

```jsx
function Greeting({ user }) {
  return (
    <div>
      {user && <h1>Welcome, {user.name}!</h1>}
    </div>
  );
}
```

U ovom kodu iznad, element `h1` se prikazuje samo ako je korisnički podatak tačan.

Takođe možete koristiti izraz `if` direktno ovako:

```jsx
function Greeting({ isLoggedIn }) {
  if (isLoggedIn) {
    return <h1>Welcome back!</h1>;
  }
  return <h1>Please sign in</h1>;
}
```

## Prikaz liste u React

- Prikazivanje liste u React je uobičajena operacija prilikom izgradnje korisničkog interfejsa.
- Liste se mogu prikazati korišćenjem JavaScript (JS) array metode `map()` za iteraciju preko niza elemenata i vraćanje novog niza JSX komponenti.
- Na primer, ako imate niz imena koji želite prikazati kao listu, možete uraditi sledeće:

```jsx
function NameList({ names }) {
  return (
    <ul>
      {names.map((name, index) => (
        <li key={`${name}-${index}`}>{name}</li>
      ))}
    </ul>
  );
}
```

- Uvek zapamtite da dodelite jedinstveni ključ svakom elementu popisa kako biste pomogli React u upravljanju zadacima ažuriranja i prikazivanja. Ovim metodama, možete kreirati prilagodljive, efikasne i robusne popise u vašim aplikacijama za React.

## Inline stilovi u React

- Inline stilovi u React omogućavaju vam da direktno koristite CSS stilove na JSX elementima koristeći objekte iz JavaScript.
- Da biste koristili inline stilove u React, možete koristiti atribut 'style' na JSX elementima. Atribut 'style' prima objekat čije ključeve predstavljaju svojstva (properties) iz CSS u camelCase formatu, a vrednosti su odgovarajuće. Evo primera:

```jsx
function Greeting() {
  return (
    <h1
      style={{ color: 'blue', fontSize: '24px', backgroundColor: 'lightgray' }}
    >
      Hello, world!
    </h1>
  );
}

export default Greeting;
```

Takođe možete pružiti te stilove u nečem drugom i navesti ih u atributu `style` na ovaj način:

```jsx
function Greeting() {
  const styles = {
    color: 'blue',
    fontSize: '24px',
    backgroundColor: 'lightgray'
  };

  return <h1 style={styles}>Hello, world!</h1>;
}

export default Greeting;
```

- Inline stilovi podržavaju uslovno stilizovanje, omogućavajući primenu stilova na osnovu prop-a ili stanja. Evo primera kako možete koristiti uslovne stilove na osnovu prop-a:

```jsx
function Greeting({ isImportant }) {
  const styles = {
    color: isImportant ? 'red' : 'black',
    fontSize: isImportant ? '24px' : '16px'
  };

  return <h1 style={styles}>Hello, world!</h1>;
}

export default Greeting;
```

- U ovom gore navedenom kodu, stilovi `color` i `fontSize` su definisani na osnovu svojstva (prop) `isImportant`.
