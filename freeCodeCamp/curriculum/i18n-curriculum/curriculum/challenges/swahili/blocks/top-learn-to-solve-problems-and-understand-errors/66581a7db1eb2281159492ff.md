# --description--

Još jedan važan deo greške je *stack trace*. Ovo vam pomaže da razumete kada je greška bačena u vašoj aplikaciji i koje funkcije su bile pozvane koje su uzrokovale tu grešku. Na primer, ako imate sledeći kod:

```javascript
const a = 5;
const b = 10;

function add() {
  return c;
}

function print() {
  add();
}

print();
```

Naša funkcija `print()` treba da pozove `add()`, koja vraća parametar nazvan `c`, koji trenutno nije deklarisan. Odgovarškom greškom je sledeće:

<img src="https://cdn.statically.io/gh/TheOdinProject/curriculum/284f0cdc998be7e4751e29e8458323ad5d320303/foundations/javascript_basics/understanding_errors/imgs/01.png" width="100%" alt="Greška koja prikazuje stack trace u konzoli za programere">

Stack trace kaže mi da:

1. `c is not defined` je u opsegu `add()`, koji je deklarisan na liniji 5.
1. `add()` je korišćen od strane `print()`, koji je deklarisan na liniji 9.
1. `print()` samostalno je korišćen na liniji 12.

Dakle, stack trace vam omogućava da pratite promene greške do njenog izvora, koji je ovde deklaracija `add()`.

# --questions--

## --text--

Stack trace, kako pomaže u popravljanju grešaka u kodu JavaScript koji je dostavljen?

## --answers--  

Stak tracid pokazuje da funkcija `add()` vraća vrednost `c` pravilno.

---

Stack trace pokazuje koji red u kodu treba da se izmeni kako bi se ispravila sintaktička greška.

---

Stack trace pruža detaljan trag poziva funkcija koje uzrokuju grešku, kako bi pomogao da se utvrdi gde je `c` pogrešno naveden.

---

Stack trace pokazuje samo da `print()` je korišćen, ali ne prikazuje detalje o grešci u `add()`.
