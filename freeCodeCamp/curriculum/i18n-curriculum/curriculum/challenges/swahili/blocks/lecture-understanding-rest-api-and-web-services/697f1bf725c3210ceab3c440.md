# --description--

Usluge mreže i REST API-ji pokreću mnoge aplikacije i sajtove koje koristite svakodnevno. One omogućavaju različitim aplikacijama da komuniciraju i razmenjuju podatke putem mreže, često interneta.

API (Interfejs programskog API-ja) je opšti način za komunikaciju između programa. Web usluga je specifičan tip od API koji koristi mrežne tehnologije, kao što su HTTP, za razmenu podataka preko mreže, obično interneta. Zbog toga, svaka web usluga je API—ali nije svaki API web usluga. REST API-ji su uobičajen tip web usluga koji prate određeni set pravila o tome kako se podaci moraju zahtevati i vraćati.

Ovaj kurs će pokriti napredne mrežne usluge, a sledeći kurs će detaljnije objasniti kako REST API-ji funkcionišu.

Možete zamisliti mrežnu uslugu kao most između aplikacija i aplikacija. Nema korisničkog interfejsa i nije namenjena za direktnu upotrebu. Umesto toga, cilja isključivo na razmenu podataka između aplikacija.

Da bi bilo koji sistem bio smatran mrežnom uslugom, mora posedovati sledeće karakteristike:

* **Pokreće se na serveru**: Web usluga mora biti online kao veb sajt, ali umesto stranica koje korisnici mogu da vide, ona izlaže određene podatke ili funkcije
    
* **Komunikacija između mašina**: Web usluga nije napravljena da je ljudi vide direktno, već za mobilne aplikacije i veb aplikacije da je koriste, kako bi mogle izložiti podatke ili funkcije koje korisnik na kraju vidi
    
* **Ne zavisi od jezika**: Web usluga može biti napisana na bilo kom jeziku kao što su Java ili C#, i svaka aplikacija bi trebalo da može sa njom da interaguje, čak i ako je napisana u drugom jeziku
    
* **Ne zavisi od platforme**: Ne bitno na kojoj platformi mašine rade. Na primer, web usluga koja koristi Linux ne bi trebalo da spreči uređaj koji koristi Windows da sa njom komunicira
    
* **Koristi standardne mrežne protokole**: Web usluga bi obično trebalo da koristi HTTP/HTTPS za razmenu podataka online, FTP za prenos fajlova i SMTP za e-poštu

**REST** na **SOAP** je dva poznata tipa za mrežne usluge, iako postoje druge metode poput **XML-RPC**.

**REST**, poznat kao **uhamishaji wa hali ya uwakilishi (REST)**, koristi se najviše danas. Mnoge moderne aplikacije oslanjaju se na RESTful web servise jer su jednostavni, efikasni i prirodno rade sa mrežom.

REST APIs izlažu resurse, koji su obično predstavljeni putem putanja (paths) kao URL poput `/users` ili `/users/20`. Kada klijent zahteva resurs, server vraća reprezentaciju tog resursa. Ova reprezentacija je često u JSON, ali može biti i u formatu kao XML ili čak običan tekst.

Klijenti mogu da navedu strukturu koju žele koristeći zaglavlja HTTP, kao zaglavlje `Accept`. Primer, klijent može zatražiti odgovor za JSON postavljanjem odgovarajuće vrednosti `Accept`.

```bash
Accept: application/json
```

Možete navesti šta želite XML na ovaj način:

```bash
Accept: application/xml
```

Resurs koji dobijate može izgledati ovako ako je JSON:

```json
{
  "id": 20,
  "name": "John Doe",
  "email": "john@example.com",
  "role": "admin",
  "createdAt": "2025-09-22T10:30:00Z"
}
```

I ovo, ako je XML:

```xml
<user>
  <id>20</id>
  <name>John Doe</name>
  <email>john@example.com</email>
  <role>admin</role>
  <createdAt>2025-09-22T10:30:00Z</createdAt>
</user>
```

SOAP je zasnovan na **Simple Object Access Protocol**. Koristi samo strukturu podataka XML, i najčešće se koristi u poslovnim sistemima koji zahtevaju strože standarde i sigurnost.

Ovo je primer SOAP podataka:

```xml
<soapenv:Envelope xmlns:soapenv="http://schemas.xmlsoap.org/soap/envelope/">
   <soapenv:Body>
      <getUser>
         <userId>20</userId>
      </getUser>
   </soapenv:Body>
</soapenv:Envelope>
```

XML-RPC koristi XML za slanje zahteva i njihovih odgovora. Ovo ga čini laganim, ali bez efikasnosti kao REST.

Što se tiče UDDI, nije usluga za razmenu podataka, već direktorijum koji zavisi od XML gde preduzeća mogu listovati, otkrivati i opisivati svoje online usluge.
