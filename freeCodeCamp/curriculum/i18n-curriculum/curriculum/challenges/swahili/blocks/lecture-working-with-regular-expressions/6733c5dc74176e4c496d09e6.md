# --interactive--

Verifikacija prednjeg i zadnjeg senzora omogućava vam poređenje specifičnih sistema u zavisnosti od prisustva ili odsutnosti okolnih sistema. Postoji četiri tipa ove verifikacije.

Prvi je pozitivni lookahead. On pronalazi obrazac samo ako iza njega sledi drugi obrazac.

Da biste napravili pozitivni lookahead, prvo napišite obrazac koji želite da pronađete. Zatim ga pratite zagradama koje sadrže obrazac koji predstavlja uslov. Posle otvorene zagrade koristite `?=` da označite pozitivni lookahead.

Ovaj obrazac će pronaći reč `free` samo ako je iza nje `code`:

```js
const regex = /free(?=code)/i;
```

Hebu jaribu tabia ya muundo wetu:

:::interactive_editor

```js
const regex = /free(?=code)/i;
console.log(regex.test("freeCodeCamp")); // true
console.log(regex.test("free code camp")); // false
console.log(
  regex.test("I need someone for free to write code for me")
); // false
```

:::

Obratite pažnju da string u kome `free` odmah prati `code` prolazi test.

Ali šta ako želite da pronađete `free` samo kada iza njega NE sledi `code`? Tada možete zameniti `?=` sa `?!` i napraviti negativni lookahead:

```js
const regex = /free(?!code)/i;
```

Hajde da to isprobamo na istom stringu:

:::interactive_editor

```js
const regex = /free(?!code)/i;
console.log(regex.test("freeCodeCamp")); // false
console.log(regex.test("free code camp")); // true
console.log(
  regex.test("I need someone for free to write code for me")
); // true
```

:::

Kao što je očekivano, rezultati su promenjeni. Jedina neuspešna sekvenca karaktera je prva, gde `free` ga odmah prati `code`.

Validacija nazad radi na isti način kao i validacija napred, osim što ne vrši uslovno poređenje bazirano na sledećem obrascu, već uslovno poređenje bazirano na prethodnom obrascu. Pogledajmo pozitivnu obrnuto praćenje (ili: obrnutu validaciju).

Pozitivni lookbehind koristi `?<=` umesto `?=`. Hajde da naš regularni izraz pronađe `code` samo kada mu prethodi `free`:

```js
const regex = /(?<=free)code/i;
```

Kao što je slučaj sa našom naprednom analizom, naša nazadnja analiza poredi prvu sekvencu jer `code` je odmah prethođen sa `free`:

:::interactive_editor

```js
const regex = /(?<=free)code/i;
console.log(regex.test("freeCodeCamp")); // true
console.log(regex.test("free code camp")); // false
console.log(
  regex.test("I need someone for free to write code for me")
); // false
```

:::

Za poređenje `code` kada je HAUTANGULIWI i `free`, možemo koristiti negativni zadnji ugao. Negativni zadnji ugao se definiše promenom `?<=` u `?<!`:

```js
const regex = /(?<!free)code/i;
```

Ovo će uporediti bilo koji događaj od `code` koji NIJE zakazan jednom sa `free`:

:::interactive_editor

```js
const regex = /(?<!free)code/i;
console.log(regex.test("freeCodeCamp")); // false
console.log(regex.test("free code camp")); // true
console.log(
  regex.test("I need someone for free to write code for me")
); // true
```

:::

Zapamtite da `Regex.prototype.test` verifikuje samo da li niz slova odgovara regularnom izrazu. Koristite naš negativni pogled iza sa `String.prototype.match` da vidite kako validacija utiče na to:

```js
const regex = /(?<!free)code/i;
console.log("freeCodeCamp".match(regex)); // null
console.log("free code camp".match(regex)); // ['code', index: 5, input: 'free code camp', groups: undefined]
console.log(
  "I need someone for free to write code for me".match(regex)
); // ['code', index: 33, input: 'I need someone for free to write code for me', groups: undefined]
```

Pogledajte kako naš uobičajeni izraz koristi nazadnu referencu za proveru postojanja `free`, a da ne odgovara `free`. Jedini tekst sadržan u poređenju je `code`.

Prefix i sufiks su veoma važni za uslovno poređenje tekstova bez uticaja na vašu vraćenu vrednost poređenja.
