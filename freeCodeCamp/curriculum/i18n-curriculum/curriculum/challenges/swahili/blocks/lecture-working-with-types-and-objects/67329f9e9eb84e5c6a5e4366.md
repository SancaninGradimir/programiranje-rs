# --interactive--

U prethodnim modulima, navikli ste da radite sa sekvencom izvornih karaktera kao ova:

```js
const greeting = "Hello, World!";
```

Međutim, JavaScript takođe ima string objekte. String objekti i originalni stringovi koriste se za rukovanje tekstom, ali rade drugačije ispod površine. String objekat se kreira korišćenjem konstruktora stringa, koji obuhvata originalnu vrednost unutar objekta. Evo kako da napravite string objekat:

:::interactive_editor

```js
const greetingObject = new String("Hello, World!");

console.log(typeof greetingObject); // "object"
```

:::

Kada koristimo operaciju ``typeof``, možemo videti da je rezultat tip objekta umesto tipa niza karaktera. Jedna od stvari koje ste možda pitali je kako možete koristiti svojstvo kao što je svojstvo ``.length`` na originalnom nizu karaktera.

Kada koristite svojstvo ``length`` na prirodnom nizu znakova, `JavaScript` privremeno pokriva prirodni niz znakova unutar objekta string literala kako bi izvršio operaciju. Zato možete koristiti svojstvo ``length`` sa različitim metodama kao što su ``repeat()``, ``concat()`` i ``slice()``. Ove vrste metoda i svojstava nazivaju se instancijske metode, instancijska svojstva i statičke metode. Detaljno ćete naučiti kako to funkcioniše u budućim modulima.

Glavna razlika između objekta stringa i nativnog niza karaktera je kako se to odnosi na memoriju i performanse. Nativen niz karaktera obično ima bolju efikasnost memorije i brži u poređenju sa objektima stringa. Iako ćete u svom kodu više koristiti nativne nizove karaktera, i dalje je važno razumeti kako objekti stringa funkcionišu.
