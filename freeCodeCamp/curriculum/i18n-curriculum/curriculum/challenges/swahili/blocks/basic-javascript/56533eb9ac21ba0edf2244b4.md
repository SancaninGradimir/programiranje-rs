# --description--

Vrednosti tipa <dfn>string</dfn> u JavaScript-u mogu se pisati jednostrukim ili dvostrukim navodnicima, sve dok pocnete i zavrsite istim tipom navodnika. Za razliku od nekih drugih jezika, jednostruki i dvostruki navodnici rade isto u JavaScript-u.

```js
const doubleQuoteStr = "This is a string"; 
const singleQuoteStr = 'This is also a string';
```

Razlog da izaberete jedan tip navodnika umesto drugog je kada zelite da koristite oba tipa unutar iste niske. To se cesto desava kada cuvate dijalog sa navodnicima ili kada cuvate `<a>` tag sa atributima unutar niske.

```js
const conversation = 'Finn exclaims to Jake, "Algebraic!"';
```

Ipak, to postaje problem ako treba da koristite isti tip navodnika kao spoljasnji i unutar niske. Niska mora da pocinje i zavrsava istim tipom navodnika. Ako se isti navodnik pojavi u sredini, niska se prerano zavrsava i dobijate gresku.

```js
const goodStr = 'Jake asks Finn, "Hey, let\'s go on an adventure?"'; 
const badStr = 'Finn responds, "Let's go!"';
```

Ovde ce `badStr` izazvati gresku.

U gornjem primeru <dfn>goodStr</dfn>, oba tipa navodnika se bezbedno koriste tako sto se navodnik escape-uje pomocu `\`.

**Napomena:** Obrnuta kosa crta `\` nije isto sto i kosa crta `/`. Ne rade istu stvar.

# --instructions--

Prepravite dati string tako da ima jednostruke navodnike na pocetku i kraju, bez escape karaktera.

Trenutno `<a>` tag unutar stringa koristi dvostruke navodnike svuda. Treba da promenite spoljasnje navodnike u jednostruke da biste uklonili escape karaktere.

# --hints--

Treba da uklonite sve obrnute kose crte (`\`).

```js
assert(
  !/\\/g.test(__helpers.removeJSComments(code)) &&
    myStr.match(
      '\\s*<a href\\s*=\\s*"http://www.example.com"\\s*target\\s*=\\s*"_blank">\\s*Link\\s*</a>\\s*'
    )
);
```

Treba da imate dva jednostruka navodnika `'` i cetiri dvostruka navodnika `"`.

```js
assert(__helpers.removeJSComments(code).match(/"/g).length === 4 && __helpers.removeJSComments(code).match(/'/g).length === 2);
```
