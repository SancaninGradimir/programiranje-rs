# --interactive--

## Osnovi nečega

- **Definicija**: Objekt je struktura podataka napravljena od svojstava. Svako svojstvo ima ključeve i vrednosti. Da biste dobili podatke iz objekta, možete koristiti tačka-notaciju ili zagradnu notaciju.

:::interactive_editor

```js
const person = {
  name: "Alice",
  age: 30,
  city: "New York"
};

console.log(person.name);  // Alice
console.log(person["name"]); // Alice
```

:::

Da biste postavili atribut nečega što već postoji, možete koristiti dot notaciju ili zagradnu notaciju zajedno sa operatorom dodeljivanja.

:::interactive_editor

```js
const person = {
  name: "Alice",
  age: 30
};

person.job = "Engineer"
person["hobby"] = "Knitting"
console.log(person);  // {name: 'Alice', age: 30, job: 'Engineer', hobby: 'Knitting'}
```

:::

## Uklanjanje atributa sa objekta

- **Operacija za `delete`**: Ova operacija se koristi za uklanjanje svojstava sa objekta.

:::interactive_editor

```js
const person = {
  name: "Alice",
  age: 30,
  job: "Engineer"
};

delete person.job;

console.log(person.job); // undefined
```

:::

## Proveriti da li nešto ima određena svojstva

- **Njia ya `hasOwnProperty()`**: Ova metoda vraća boolean vrednost (tačno ili netačno) koja ukazuje da li objekat poseduje pomenuto svojstvo.

:::interactive_editor

```js
const person = {
  name: "Alice",
  age: 30
};

console.log(person.hasOwnProperty("name")); // true
console.log(person.hasOwnProperty("job")); // false
```

:::

- **Metoda za `Object.hasOwn()`**: Ovo je moderni, preporučeni način proveravanja da li objekat poseduje sopstveno svojstvo (nije nasleđeno). Sintaksa je `Object.hasOwn(object, propertyName)`. Vraća `true` ako svojstvo postoji na objektu i `false` ako ne postoji — bez obzira na vrednost svojstva. Ovo ga čini sigurnijim od `hasOwnProperty()` i pouzdanijim od `if (obj.prop)` kada vrednosti mogu biti `0`, `false`, `null` ili `undefined`.

:::interactive_editor

```js
const person = {
  name: "Alice",
  age: 30
};

console.log(Object.hasOwn(person, "name")); // true
console.log(Object.hasOwn(person, "job")); // false
```

:::

Evo detaljnog primera koji pokazuje zašto `Object.hasOwn()` je pouzdaniji od direktne provere vrednosti:

:::interactive_editor

```js
const settings = {
  darkMode: false,
  fontSize: 0,
  language: null
};

// Object.hasOwn() correctly sees these properties exist
console.log(Object.hasOwn(settings, "darkMode")); // true (value is false, but exists)
console.log(Object.hasOwn(settings, "fontSize")); // true (value is 0, but exists)
console.log(Object.hasOwn(settings, "theme"));    // false (property was never added)

// Using if() directly is unsafe for falsy values!
if (settings.darkMode) {
  console.log("Dark mode on"); // Does NOT print — misleading!
}

// Object.hasOwn() is the safe way
if (Object.hasOwn(settings, "darkMode")) {
  console.log("darkMode exists, value is:", settings.darkMode); // darkMode exists, value is: false
}
```

:::

- **Opereta ya `in`**: Ovo svojstvo vraća `true` ako postoji na objektu.

:::interactive_editor

```js
const person = {
  name: "Bob",
  age: 25
};

console.log("name" in person);  // true
```

:::

## Dohvatanje svojstava iz interno definisanih elemenata

- **Dobijanje podataka**: Dohvatanje svojstava iz ugnježdjenih objekata podrazumeva korišćenje dot notacije ili bracket notacije, kao što dobijate svojstva iz jednostavnih objekata. Međutim, biće vam potreban niz poziva ovih getter-a kako biste došli do ugnježdene strukture.

:::interactive_editor

```js
const person = {
  name: "Alice",
  age: 30,
  contact: {
    email: "alice@example.com",
    phone: {
      home: "123-456-7890",
      work: "098-765-4321"
    }
  }
};

console.log(person.contact.phone.work); // "098-765-4321"
```

:::

## Tip podataka za primarne i sekundarne podatke

- **Primativni tipovi podataka**: Ovi tipovi podataka uključuju brojeve, nizove karaktera, logičke vrednosti (tačno ili netačno), `null`, `undefined` i simbole. Ovi tipovi se nazivaju "primativni" jer predstavljaju jednu vrednost, a ne objekte. Primativne vrednosti su nepromenljive, što znači da kada se kreiraju, njihova vrednost ne može biti promenjena.
- **Neprimativni tipovi podataka**: U JavaScript, ovo su objekti, uključujući obične objekte, nizove podataka i funkcije. Za razliku od primativnih, neprimativni tipovi mogu skladištiti više vrednosti kao atribute ili komponente.

## Načini za nešto

- **Definicija**: Metode objekta su funkcije povezane sa objektom. Definisane su kao svojstva objekta i mogu dohvatiti i izvršiti podatke tog objekta. Token ``this`` unutar metode se odnosi na sam objekat, omogućavajući pristup njegovim svojstvima.

:::interactive_editor

```js
const person = {
  name: "Bob",
  age: 30,
  sayHello: function() {
    return "Hello, my name is " + this.name;
  }
};

console.log(person.sayHello()); // "Hello, my name is Bob"
```

:::

## Komponenta predmeta

- **Definicija**: U JavaScript, komponenta je specijalni tip funkcije koji se koristi za kreiranje i inicijalizovanje objekata. Naziva se sa rečima `new` i može da inicijalizuje svojstva i metode na novokreiranom objektu. Komponenta `Object()` kreira novi prazan objekat.

```js
new Object()
```

## Rad sa operacijom serijskog povezivanja (`?.`)

- **Definicija**: Ova operacija omogućava vam da sigurno dobijete atribute objekta ili pozovete metodu bez brige da li postoje ili ne.

:::interactive_editor

```ts
const user = {
  name: "John",
  profile: {
    email: "john@example.com",
    address: {
      street: "123 Main St",
      city: "Somewhere"
    }
  }
};

console.log(user.profile?.address?.street); // "123 Main St"
console.log(user.profile?.phone?.number);   // undefined
```

:::

## Rešenje strukture objekta

- **Definicija**: Dekonstrukcija objekta omogućava vam da izvučete vrednosti iz objekata i dodelite ih varijable na kraći i čitljiviji način.

:::interactive_editor

```js
const person = { name: "Alice", age: 30, city: "New York" };

const { name, age } = person;

console.log(name); // Alice
console.log(age);  // 30
```

:::

## Rad sa JSON

- **Definicija**: JSON je skraćenica za JavaScript Object Notation. To je lagani tekstualni format podataka koji se široko koristi za razmenu podataka između servera i mrežnih aplikacija.

```js
{
  "name": "Alice",
  "age": 30,
  "isStudent": false,
  "list of courses": ["Mathematics", "Physics", "Computer Science"]
}
```

- **Putanja za `JSON.stringify()`**: Ova putanja se koristi za konverziju objekta JavaScript u niz karaktera JSON. Ovo je važno kada vam je potrebno sačuvati ili preneti podatke u format koji se lako deli ili prenosi između sistema.

:::interactive_editor

```js
const user = {
  name: "John",
  age: 30,
  isAdmin: true
};

const jsonString = JSON.stringify(user);
console.log(jsonString); // '{"name":"John","age":30,"isAdmin":true}'
```

:::

- **Njia ya `JSON.parse()`**: Ova metoda konvertuje niz znakova od JSON nazad u objekat/strukturu JavaScript. Ovo je važno kada dobijete podatke za JSON sa mrežnog servera ili localStorage i morate te podatke pokrenuti u vašoj aplikaciji.

:::interactive_editor

```js
const jsonString = '{"name":"John","age":30,"isAdmin":true}';
const userObject = JSON.parse(jsonString);

// result: { name: 'John', age: 30, isAdmin: true }
console.log(userObject);
```

:::
