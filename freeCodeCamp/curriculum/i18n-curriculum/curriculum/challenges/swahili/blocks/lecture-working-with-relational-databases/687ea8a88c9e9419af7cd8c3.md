# --description--

Istražite neke osnovne komande koje možete koristiti za unos i pregled podataka u SQL.

Prvo, da biste mogli da pregledate podatke, potrebno je da ih unesete. Zato ćemo početi sa unosom podataka.

Primeri koje ćemo obrađivati u ovoj lekciji zasnivaće se na ovoj tabeli zapisa o psima.

```sql
CREATE TABLE dogs(
  id SERIAL,
  name VARCHAR(100),
  age INTEGER
);
```

Jedwali hili litahifadhi majina na umri wa mbwa mbalimbali.

Ako pretpostavimo da radimo sa PostgreSQL-om, možemo koristiti tip podataka `SERIAL` za `id`. Pošto je `id` kreiran pomoću tipa `SERIAL`, njegova vrednost će biti tipa `INTEGER`, počev od `1`, i automatski će se povećavati pri svakom unosu novog zapisa. Zato ne morate da navodite vrednost za njega prilikom umetanja zapisa.

Počnimo sa unosom zapisa u našu tabelu `dogs` za komandu `INSERT INTO`. Postoji mnogo načina da se ovo uradi. Prvi način je unos jedne vrste i definisanje kolona vrednosti koje će biti postavljene.

Pišete `INSERT INTO`, nakon čega sledi naziv tabele (u ovom slučaju `dogs`), zatim kolone unutar običnih zagrada, a zatim `VALUES` i njihove odgovarajuće vrednosti unutar običnih zagrada.

U ovom primeru unosimo zapis psa čije je `name` vrednost `'Gino'`, a čija je `age` vrednost `3`. Zapamtite da će `id` biti automatski dodeljen:

```sql
INSERT INTO dogs (name, age)
VALUES ('Gino', 3);
```

Ovo je obično sigurnija opcija jer eksplicitno navodite kolone, pa će vrednosti biti dodeljene tim kolonama po redosledu.

Imajte na umu da `'Gino'` mora biti postavljen unutar pojedinačnih navodnika jer je tip `VARCHAR`.

Druga opcija je unos zapisa bez navođenja kolona, kao što sledi:

```sql
INSERT INTO dogs
VALUES ('Gino', 3);
```

Ova sintaksa je validna u SQL, ali je opasna zbog grešaka.

Pošto su vrednosti date prema redosledu kolona, PostgreSQL će pokušati da unese `'Gino'` u kolonu `id` koja očekuje ceo broj. Ovo uzrokuje grešku kao `invalid input syntax for type integer`.

Zbog toga, sigurnije je da se kolone imena navedu jasno prilikom unosa podataka.

Takođe možete uneti više zapisa u jednoj SQL naredbi tako što ih odvojite zarezima.

Ovde, unosimo dva zapisa za pse, jedan za `'Gino'` i drugi za `'Nora'`.

```sql
INSERT INTO dogs (name, age)
VALUES
  ('Gino', 3),
  ('Nora', 2);
```

Kao što možete da unesete zapise, takođe možete tražiti informacije iz baze podataka. U kontekstu baze podataka, upit za pretraživanje je zahtev za podatke.

U SQL, komanda `SELECT` se koristi za zahtev podataka iz jedne ili više tabela. Možete prilagoditi argument pretrage da biste dobili tačne informacije koje vam trebaju.

Ako želite da zahtevate sve informacije iz tabele `dogs`, potrebno je samo napisati `SELECT`, praćen zvezdicom (`*`), zatim `FROM` i ime tabele, `dogs`:

```sql
SELECT *
FROM dogs;
```

U ovoj komandi, zvezdica je wildcard atribut koji predstavlja 'sve kolone'.

Dobićete sve podatke za sve zapise u tabeli:

```sql
 id | name | age 
----+------+-----
  1 | Gino |   3
  2 | Nora |   2
```

Da biste zatražili određene kolone, možete napisati nazive tih kolona u naredbi odmah nakon `SELECT`, odvajajući ih zarezima. U ovom primeru tražimo kolone `name` i `age` iz tabele `dogs`:

```sql
SELECT name, age
FROM dogs;
```

Matokeo yatakuwa na nguzo za `name` na `age` pekee:

```sql
 name | age 
------+-----
 Gino |   3
 Nora |   2
```

Ponekad ćete možda morati da tražite podatke na osnovu određenog uslova. Na primer, da biste pronašli sve pse mlađe od 3 godine, možete koristiti ključnu reč `WHERE` i operator poređenja manje od (`<`):

```sql
SELECT *
FROM dogs
WHERE age < 3;
```

Rezultat će sadržati samo pse čija je starost manja od 3:

```sql
 id | name | age 
----+------+-----
  2 | Nora |   2
```

Ako samo pokušavate da dobijete vrednost `age` za psa `'Gino'`, možete koristiti operator poređenja jednako (`=`):

```sql
SELECT age
FROM dogs
WHERE name = 'Gino';
```

Na hapa ni matokeo:

```sql
 age 
-----
  3
```

Ovo su uobičajeni i jednostavni načini za unos i pregled podataka u SQL, ali postoji mnogo različitih opcija koje možete izabrati, a neke od njih su naprednije. Naučićete o njima u narednim lekcijama.
