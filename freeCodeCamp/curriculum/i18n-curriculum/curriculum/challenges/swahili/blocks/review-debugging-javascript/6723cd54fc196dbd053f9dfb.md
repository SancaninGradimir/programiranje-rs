## Uobičajeni tipovi poruka o grešci

- **SyntaxError**: Ove greške se javljaju kada je nešto netačno u vašem kodu, kao što je nedostajanje zatvarajućih ili otvarajućih zagrada. Razmislite o tome kao o gramatičkoj grešci u rečenici.

```js
const arr = ["Beau", "Quincy" "Tom"]
```

- **ReferenceError**: Postoje različiti tipovi grešaka referenciranja, koje se aktiviraju na različite načine. Prvi tip greške referenciranja su one koje se odnose na nedefinisane varijable. Drugi primer ReferenceError je pokušaj dobijanja vrednosti varijable, deklarisane sa `let` ili `const`, pre nego što je definisana.

:::interactive_editor

```js
console.log(num);
const num = 50;
```

:::

- **TypeError**: Ove greške nastaju kada pokušate da izvršite operaciju sa pogrešnim tipom.

```js
const developerObj = {
  name: "Jessica",
  country: "USA",
  isEmployed: true
};

developerObj.map()
```

- **RangeError**: Ove greške nastaju kada vaš kod pokušava da koristi vrednost koja je van opsega onoga što JavaScript može da obradi.

```js
const arr = [];
arr.length = -1; 
```

## Izjava o `throw`

- **Značenje**: Izjava `throw` u JavaScript se koristi za prikaz greške koju je korisnik prijavio. Greška u programu je neočekivani događaj koji narušava normalan tok programa.

:::interactive_editor

```js
function validateNumber(input) {
  if (typeof input !== "number") {
    throw new TypeError("Expected a number, but received " + typeof input);
  }
  return input * 2;
}

console.log(validateNumber("Naomi"));   // TypeError: Expected a number, but received string
```

:::

## `try...catch...finally`

- **Značenje**: Blok `try` koristi se za pokrivanje koda koji može da izbaci grešku. Deluje kao sigurno mesto za testiranje nečega što može da propadne. Blok `catch` hvata i obrađuje greške koje nastanu u bloku try. Možete koristiti objekat greške unutar catch da istražite šta je pošlo naopako. Blok `finally` izvršava se nakon try i catch, bez obzira da li je greška nastala. Često se koristi za čišćenje zadatke, kao što su zatvaranje fajlova ili oslobađanje resursa.

:::interactive_editor

```js
function processInput(input) {
  if (typeof input !== "string") {
    throw new TypeError("Input must be a string.");
  }

  return input.toUpperCase();
}

try {
  console.log("Starting to process input...");
  const result1 = processInput("hello");
  console.log("Processed result:", result1);  // HELLO
  const result2 = processInput(9);  // throws TypeError
  console.log("Processed result:", result2);  // not executed
} catch (error) {
  console.error("Error occurred:", error.message);
} 
```

:::

## Metode za ispravljanje grešaka

- **Kauli ya `debugger`**: Ova deklaracija vam omogućava da pauzirate svoj kod na specifičnoj liniji kako biste analizirali šta se dešava u programu.

:::interactive_editor

```js
let firstNumber = 5;
let secondNumber = 10;

debugger; // Code execution pauses here
let sum = firstNumber + secondNumber;

console.log(sum);
```

:::

- **Breakpoints**: Breakpoints omogućavaju da zaustavite izvršavanje vašeg koda na određenoj liniji koju izaberete. Nakon zaustavljanja, možete pregledati varijable, evaluirati matematički izraz i proveriti stack poziva.
- **Watchers**: Watch expressions vam omogućavaju da pratite vrednosti varijabli ili matematičke izraze dok se kod izvršava, čak i ako su van trenutnog opsega (scope).
- **Profiling**: Profiling pomaže u identifikovanju delova koji zahtevaju mnogo vremena, omogućavanjem snimanja snimaka ekrana i beleženje korišćenja procesora (CPU), poziva funkcija i vremena izvršenja.
- **`console.dir()`**: Ova metoda se koristi za prikaz listu interakcija svojstava specifičnog objekta JavaScript. Prikazano je u strukturi stabla (tree structure) koja se može proširiti da bi se videla sva svojstva definisana unutra.

```js
console.dir(document);
```

- **`console.table()`**: Ova metoda prikazuje podatke u tabelarnom formatu u konzoli. Prima jedan obavezan argument, koji mora biti niz podataka ili objekat, i jedan opcionalni argument za prikazivanje koje svojstvo (kolone) treba da prikaže.
