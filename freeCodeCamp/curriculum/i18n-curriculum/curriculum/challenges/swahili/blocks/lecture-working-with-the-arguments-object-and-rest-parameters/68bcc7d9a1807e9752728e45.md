# --interactive--

U prethodnoj lekciji, naučili ste kako da radite sa objektom `arguments` koji je objekat sličan nizu podataka koji sadrži vrednosti argumenata prosleđenih funkciji.

:::interactive_editor

```js
function logArgs() {
  for (const arg of arguments) {
    console.log(arg);
  }
}

logArgs(1, 2, 3);
// result:
// 1
// 2
// 3
```

:::

Iako je ovo legitimni način za dobijanje i rad sa promenljivom skupom argumenata iz funkcije, moderne aplikacije koje koriste JavaScript umesto toga koriste sintaksu rest parametra.

Evo ažuriranog primera koji koristi REST parametarsku sintaksu umesto `arguments`:

:::interactive_editor

```js
function logArgs(...args) {
  for (const arg of args) {
    console.log(arg);
  }
}

logArgs(1, 2, 3);
// result:
// 1
// 2
// 3
```

:::

Ovaj ažurirani primer više ne zahteva direktne reference na objekat `arguments`. Umesto toga, poslednji parametar u definiciji funkcije ima tri tačke ispred sebe. To uzrokuje da ovaj rest parametar bude postavljen unutar objekta `Array`. Možete se na ovaj rest parametar pozvati bilo kojim imenom koje želite. Samo osigurajte da je poslednji parametar u definiciji funkcije, kao što je ovde:

```js
function exampleFunction(a, b, ...restOfArgs) {
  // some code here
}

function anotherFunction(x, y, ...theArgs) {
  // some code here
}
```

Postoji manje ograničenja sa kojima se susrećete kada koristite sintaksu rest parametra. Jedno ograničenje je da definicija funkcije može imati samo jedan rest parametar. Stoga će sledeći primer ovde izgledati kao nevažeći.

```js
// Won't work

function badFunction(...args, ...moreArgs) {
  // some code here
}
```

Još jedno ograničenje je da zarezima na kraju nisu dozvoljeni nakon atributa 'rest': `function exampleFunction(a, b, ...restOfArgs, )`.

Još jedan uslov je da parametar rest ne može imati podrazumevanu vrednost. Inače, biće izbačeno `SyntaxError`.

```js
function badFunction(...args = [1,2]){
  // some code here
}
```

Šta je razlika između objekta `arguments` i parametara za `rest`?

Glavna razlika je u tome što objekat `arguments` nije pravi niz, pa ne koristi metode kao `includes`, `pop` i `push`. Međutim, rest parametar je primer `Array`. Stoga možete koristiti važeće metode nizova koje su konstruisane bez potrebe za njihovom prethodnom konverzijom u pravi niz.

:::interactive_editor

```js
function hasCat(...args) {
  return args.includes("cat");
}

console.log(hasCat("dog", "chicken", "cat")); // true
console.log(hasCat("dog", "chicken", "horse")); // false
```

:::
