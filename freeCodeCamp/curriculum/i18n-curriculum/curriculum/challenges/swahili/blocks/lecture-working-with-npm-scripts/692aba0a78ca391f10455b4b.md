# --description--

U razvoju moderne mreže, možete podeliti svoj kod u mnogo fajlova koji se zovu "moduli."

Možete zamisliti modul kao nezavisan i ponovno upotrebljiv deo koda.

Integrisanjem modula u druge module, možete ponovno koristiti njihovu funkcionalnost dok zadržavate kod u različitim fajlovima, što je lako za održavanje, posebno za velike i složene veb aplikacije.

U Node.js, svaki fajl se tretira kao moduli.

Node ima dva različita sistema za rad sa modulima: CommonJS i ES.

CommonJS je originalni i podrazumevani način rada sa paketima u Node.js okruženju. Module učitava sinhrono, što znači da mora da sačeka da se jedan modul potpuno učita pre nego što počne učitavanje sledećeg ili nastavi izvršavanje koda.

I dalje se koristi mnogo, ali polako se zamenjuje novom ES sintaksom.

CommonJS koristi funkciju `require()` za uvoz modula, uzimajući svoj relativni put kao argument.

Na primer, recimo da imamo modul nazvan `math.js`, gde objašnjavamo osnovne matematičke operacije.

Koristićemo funkciju za množenje samo da bismo primer učinili jednostavnim i preglednim.

```js
function multiply(a, b) {
  return a * b;
}
```

Kako možemo ponovo koristiti ovu funkciju izvan našeg trenutnog modula? Podrazumevano, funkcije i parametri deklarisani u modulu Node.js su **privatni** — drugi moduli ne mogu da im pristupe osim ako ih eksplicitno ne učinimo javnim. Postoji nekoliko načina za to.

Jedan uobičajen način je postavljanje vrednosti za `module.exports`. Ovo predstavlja ono što će modul vratiti kada se koristi negde drugde, i daje drugim modulima pristup funkcijama, parametrima ili stavkama koje ste odlučili da eksponujete.

U ovom primeru, postavljamo element sa atributom `multiply` za `module.exports`. Vrednost tog atributa je referenca na funkciju `multiply` koju smo prethodno deklarisali. Ovo će eksportovati funkciju, tako da je možemo pozvati iz drugog modula ako nam zatreba.

```javascript
function multiply(a, b) {
  return a * b;
}

module.exports = {
  multiply: multiply
};
```

Recimo da pokušavamo da uvezemo modul `math.js` u drugi modul. Pozivamo funkciju `require()` i prosleđujemo relativnu putanju kao argument. Zatim, postavljamo nešto vraćeno od `require()` u parametar `math`.

```javascript
const math = require('./math');
```

Proverite vrednost `math` u konzoli štampanjem pomoću `console.log()`:

```javascript
console.log(math);
```

Ovo su rezultati. To je objekat sa svojstvom `multiply` i funkciju kao vrednost:

```javascript
{ multiply: [Function: multiply] }
```

Ako pozovemo ovu funkciju sa `math.multiply()`, prosleđujući brojeve `6` i `7` kao argumente, rezultat je `42`. Funkcija radi uspešno i još uvek je u različitoj datoteci! Dakle, naše izjava za dohvaćanje je radila dobro.

```javascript
const math = require('./math');

console.log(math.multiply(6, 7)); // 42
```

Ali postoje drugi načini za prenošenje funkcije, parametra ili stvari.

Na primer, ako treba da pošaljete samo jednu funkciju, parametar ili stavku, možete je postaviti direktno kao vrednost za `module.exports`:

```javascript
function multiply(a, b) {
  return a * b;
}

module.exports = multiply;
```

Ako je potrebno da prenesete više funkcija/argumenta, samo ih morate staviti u `module.exports`, razdvojeno zarezima.

Ovde možete videti da prebačeni element ima dve karakteristike: `subtract` i `multiply`. Njihove vrednosti su reference na odgovarajuće funkcije, pa ćete imati pristup obe u bilo kom modulu koji učitava `math.js`:

```javascript
function subtract(a, b) {
  return a - b;
}

function multiply(a, b) {
  return a * b;
}

module.exports = {
  subtract: subtract,
  multiply: multiply
};
```

Wakati kitu cha `module.exports` ima mnogo karakteristika, biće vam opcija da izaberete da li želite da unesete sve ili da dovedete karakteristiku po karakteristiku.

U ovom primeru, imamo ceo predmet transportovan putem `math.js`, pa ćemo imati pristup `subtract` i `multiply`:

```javascript
const math = require('./math');

console.log(math.subtract(6, 7));  // -1
console.log(math.multiply(6, 7));  // 42
```

Ako želimo da koristimo samo `multiply` umesto toga, moramo samo napisati njegovo ime unutar zagrade, nakon glavne reči `const`. U osnovi, koristi se sintaksa dodeljivanja rešavanja strukture da bi se dobila njegova vrednost iz prenetog objekta. Zatim, možemo se direktno osvrnuti na njegovo ime u našem kodu:

```javascript
const { multiply } = require('./math');

console.log(multiply(6, 7));
```

Ovo radi i za mnogo stvari. Samo treba da ih razdvojite zarezom unutar zagrada. Ovde, unosićemo funkcije za `subtract` i `multiply`:

```javascript
const { subtract, multiply } = require('./math');
```

Po podrazumevanju, Node.js smatraće sledeće kao module CommonJS:

* Mafaili yenye nyongeza ya `.cjs`.
    
* Datoteke sa ekstenzijom `.js` kada najbliža nadređena datoteka `package.json` sadrži polje najvišeg nivoa `"type"` sa vrednošću `"commonjs"`.
    
* Datoteke sa ekstenzijom `.js` ili bez ekstenzije kada najbliža nadređena datoteka `package.json` nema polje najvišeg nivoa `"type"` ili kada ne postoji nijedna `package.json` datoteka u nadređenim direktorijumima.
    
* Mafaili yenye nyongeza isiyo `.mjs`, `.cjs`, `.json`, `.node`, au `.js`.
    

Iako ES moduli postepeno zamenjuju CommonJS module, njihovo razumevanje je i dalje važno za programere koji rade sa postojećim Node.js kodom i velikim brojem npm paketa.
