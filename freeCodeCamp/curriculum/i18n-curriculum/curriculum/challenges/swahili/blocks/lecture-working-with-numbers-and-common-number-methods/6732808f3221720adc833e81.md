# --interactive--

`parseFloat()` na `parseInt()` su dva važna načina u JavaScript za konverziju sekvence slova u brojeve. Ove metode su važne posebno kada obrađujete unos korisnika ili obradujete podatke koji dolaze u formi sekvence slova, ali treba da se tretiraju kao numeričke vrednosti.

Počinjemo sa `parseFloat()`. Ova funkcija analizira sekvencu znakova argumenata i vraća broj tipa floating-point. Dizajnirana je da ekstrahuje broj od početka sekvence znakova, čak i ako ta sekvenca sadrži nenumeričke znakove kasnije. Zapamtite da su float brojevi oni koji imaju decimalnu tačku. Tako radi `parseFloat()`:

:::interactive_editor

```js
console.log(parseFloat("3.14"));     // 3.14
console.log(parseFloat("3.14 abc")); // 3.14
console.log(parseFloat("3.14.5"));   // 3.14
console.log(parseFloat("abc 3.14")); // NaN
```

:::

Kao što vidite, `parseFloat()` počinje da analizira od početka sekvence znakova i nastavlja sve dok ne naiđe na karakter koji ne može biti deo floating-point broja. Ako ne može da pronađe validan broj na početku sekvence, vraća `NaN` (Nije Broj).

`parseInt()`, s druge strane, analizira sekvencu karaktera argumenata i vraća kompletan broj. Kao `parseFloat()`, počinje od početka sekvence karaktera, ali zaustavlja se na prvi ne-numerički karakter. Ovo je kako `parseInt()` radi:

:::interactive_editor

```js
console.log(parseInt("42"));       // 42
console.log(parseInt("42px"));     // 42
console.log(parseInt("3.14"));     // 3
console.log(parseInt("abc123"));   // NaN
```

:::

`parseInt()` Zaustavlja analizu na prvom ne-numeričkom karakteru koji naiđe. Za float brojeve, vraća samo ceo deo broja. Ako ne može pronaći validan ceo broj na početku sekvence, vraća `NaN`.

Oba puta imaju nekoliko karakteristika koje treba uzeti u obzir. Ignorišu prazan prostor na početku:

:::interactive_editor

```js
console.log(parseFloat("  3.14"));  // 3.14
console.log(parseInt("  42"));      // 42
```

:::

Obrađuju operatore za dodavanje i uklanjanje na početku sekvence:

:::interactive_editor

```js
console.log(parseFloat("+3.14"));  // 3.14
console.log(parseInt("-42"));      // -42
```

:::

Dobro je znati da, iako su ove metode moćne, imaju određena ograničenja. Na primer, one ne obrađuju sve vrste brojeva, kao što je naučna notacija, direktno. Za potrebe složenije analize, možda ćete morati da koristite dodatne metode ili biblioteke.

Kao zaključak, `parseFloat()` i `parseInt()` su važni alati za konverziju sekvenci slova u brojeve u JavaScript. Razumevanje kako funkcionišu i njihove specifične karakteristike omogućava vam da efikasnije rukujete numeričkim podacima u vašim aplikacijama, posebno kada obrađujete unos korisnika ili eksterne izvore podataka.
