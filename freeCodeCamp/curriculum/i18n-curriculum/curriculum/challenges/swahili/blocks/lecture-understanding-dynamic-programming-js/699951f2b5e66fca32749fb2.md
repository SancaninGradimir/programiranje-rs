# --description--
  
Metoda stanja je algoritamska tehnika koja rešava složna problema razbijanjem na manje probleme i skladištenjem rezultata kako bi se izbegle ponavljajuće proračune. Ova metoda transformiše probleme koji bi inače zahtevale vreme transformacije jednačina u probleme koje se mogu rešiti u polinomialnom vremenu.

## Glavni principi programa klimatskih promena

Program za uslovne promene radi tamo gde postoji dva važna uslova u problemu.

- **Ponovljivi manji problemi**: Slični, manji problemi često se javljaju prilikom rešavanja većeg problema. Umesto da brojimo ove male probleme često, čuvamo njihovo rešenje.
    
- **Optimalna struktura problema**: Najbolje rešenje za problem ima optimalna rešenja za manje probleme. To znači da možemo izgraditi najbolje rešenje kombinovanjem najboljih rešenja manjih delova.

Istražite ove koncepte koristeći uobičajen problem „penjanja nivoa“.

## Problem korišćenja standardne rekurzije

Razmislite o problemu penjanja stepenicama: ako se penjete stepenicama sa koracima `n` i možete da odjednom popnete 1 ili 2 stepena. Koliko različitih načina ima da se stigne na vrh?

```js
function climbStairsRecursive(n) {
  // Base cases: 1 way for 1 step, 2 ways for 2 steps
  if (n <= 2) {
    return n;
  }

  // Recursive rule: to reach step n, you can come from step (n-1) or step (n-2)
  return climbStairsRecursive(n - 1) + climbStairsRecursive(n - 2);
}
```

Ova implementacija ima vremensku složenost transformacije jednačine zbog mnogo ponavljajućih proračuna. Prilikom izračunavanja `climbStairsRecursive(5)`, dešava se sledeće:

- `climbStairsRecursive(5)` poziva `climbStairsRecursive(4)` na `climbStairsRecursive(3)`
    
- `climbStairsRecursive(4)` poziva `climbStairsRecursive(3)` na `climbStairsRecursive(2)`
    
- Sada `climbStairsRecursive(3)` se broji **dvaput**
    
- `climbStairsRecursive(3)` poziva `climbStairsRecursive(2)` na `climbStairsRecursive(1)`
    
- `climbStairsRecursive(2)` se broji **ukupno 3 puta**

Samo za `n=5`, izvršavamo 9 poziva funkcija kada nam je potrebno samo 5 jedinstvenih kalkulacija. Kako `n` raste, ponavljanje toga nadilazi jednačinu - `climbStairsRecursive(30)` bi zahtevalo milione poziva funkcija! Vremenska složenost postaje `O(2^n)`, što ga čini neefikasnim i nepraktičnim za velike vrednosti `n`.

## Rešenja za program promene stanja

Program za promenu stanja eliminiše ponavljajuće proračune kroz dve glavne metode:

### Napomene (Top-down metoda)

Memorija skladišti rezultate skupih poziva funkcija i vraća sačuvane rezultate kada se ponovi isti unos.

```js
function climbStairsMemo(n, memo = {}) {
  // If already calculated, return cached value
  if (memo[n] !== undefined) {
    return memo[n];
  }

  // Base cases: 1 way for 1 step, 2 ways for 2 steps
  if (n <= 2) {
    return n;
  }

  // Calculate once and store in memo for future use
  memo[n] =
    climbStairsMemo(n - 1, memo) +
    climbStairsMemo(n - 2, memo);

  return memo[n];
}
```

Memoizacija je efikasnija jer se svaka jedinstvena vrednost od `1` do `n` izračunava samo jednom. Kada nam je potreban `climbStairsMemo(3)` ponovo, umesto da ga ponovo izračunavamo (što bi pokrenulo više rekurzivnih poziva), mi samo pretražujemo našu memorijsku rečnik za `O(1)`.

Pratiti izvršavanje `climbStairsMemo(5)` korišćenjem metode odozgo-nadole da vidimo kako memorija smanjuje ponavljajući rad:

```md
Call: climbStairsMemo(5)
  memo = {} (empty)
  
  Call: climbStairsMemo(4) 
    memo = {} (empty)
    
    Call: climbStairsMemo(3)
      memo = {} (empty)
      
      Call: climbStairsMemo(2) → returns 2 (base case)
      Call: climbStairsMemo(1) → returns 1 (base case)
      
      Result: 2 + 1 = 3
      memo = {3: 3} (stored!)
    
    Call: climbStairsMemo(2) → returns 2 (base case)
    
    Result: 3 + 2 = 5
    memo = {3: 3, 4: 5} (stored!)
  
  Call: climbStairsMemo(3) → returns 3 (FROM MEMO - no recursion!)
  
  Result: 5 + 3 = 8
  memo = {3: 3, 4: 5, 5: 8}
```

**Poređenje efikasnosti**

- **Standardna rekursija**: Izvršava 9 poziva funkcija i ponavljajuće proračune
    
- **Memorizacija**: Izvršava samo 5 jedinstvenih proračuna, a zatim ponovo koristi sačuvane rezultate
    
- **Vremenska složenost**: Smanjena je sa `O(2^n)` na `O(n)` jer vršimo samo jedinstvene proračune `n`
    
- **Prostorna složenost**: `O(n)` za skladištenje memorije i stack poziva funkcija
    
- **Stvarni uticaj**: `climbStairsMemo(30)` se smanjuje sa miliona poziva funkcija na samo 30 jedinstvenih proračuna!

### Tabele (Metoda od donjeg nivoa ka višem)

Tabela gradi rešenje odozdo prema gore, popunjavajući tabelu rešenjima manjih problema:

```js
function climbStairsTabulation(n) {
  if (n <= 2) {
    return n;
  }

  // Create an array to store solutions from 0 to n
  const dp = new Array(n + 1).fill(0);

  // Base cases
  dp[1] = 1; // 1 way to reach step 1
  dp[2] = 2; // 2 ways to reach step 2

  // Build up the solution iteratively
  for (let i = 3; i <= n; i++) {
    // Ways to reach step i = ways to reach (i-1) + ways to reach (i-2)
    dp[i] = dp[i - 1] + dp[i - 2];
  }

  return dp[n];
}
```

Ovaj proces eliminiše rekurziju tako što gradi rešenje sekvencijalno, od najjednostavnijih problema do cilja.

Primenićemo bottom-up pristup praktično da vidimo kako gradimo rešenje korak po korak. Ovo je kontinuirana konstrukcija za `climbStairsTabulation(5)`:

```md
Initial state:
dp = [0, 1, 2, 0, 0, 0]
     [0, 1, 2, 3, 4, 5] ← indices (step numbers)

Step by step construction:

i = 3:
  dp[3] = dp[2] + dp[1] = 2 + 1 = 3
  dp = [0, 1, 2, 3, 0, 0]
  
i = 4:
  dp[4] = dp[3] + dp[2] = 3 + 2 = 5
  dp = [0, 1, 2, 3, 5, 0]
  
i = 5:
  dp[5] = dp[4] + dp[3] = 5 + 3 = 8
  dp = [0, 1, 2, 3, 5, 8]

Final result: dp[5] = 8
```

**Glavne prednosti tabela**

- **Nema režije rekurzije**: Za razliku od memorije, ne postoji stos poziva rekurzije.
    
- **Očekivana implementacija**: Izračunavamo vrednosti u fiksnom redosledu (1, 2, 3, 4, 5...).
    
- **Cache-friendly**: Sekvencijalni pristup podacima poboljšava korišćenje memorije.
    
- **Lako optimizovati**: Može smanjiti prostornu složenost na `O(1)` jer nam trebaju samo poslednje dve vrednosti.

```js
function climbStairsOptimized(n) {
  if (n <= 2) {
    return n;
  }

  // Store only the last two results to save space
  let prev2 = 1; // ways to reach step 1
  let prev1 = 2; // ways to reach step 2

  for (let i = 3; i <= n; i++) {
    const current = prev1 + prev2;

    // Shift values forward
    prev2 = prev1;
    prev1 = current;
  }

  return prev1;
}
```

**Poređenje efikasnosti**

- **Standardna rekurzija**: 9 poziva funkcija za `n=5`, rast promena jednačine.
    
- **Tabela**: Dodaje 3 jednostavne za `n=5`, linearni rast.
    
- **Vremenska složenost**: `O(n)` umesto `O(2^n)`.
    
- **Prostorna složenost**: `O(n)` za niz podataka, ili `O(1)` za optimizaciju.
    
- **Očekivana performansa**: Nema rizika od preljeva steka za velike ulazne podatke.

Oba metoda smanjuju vremensku složenost od `O(2^n)` transformacije jednačine do `O(n)` linije, značajno poboljšavajući ono što pravi razliku između rešavanja problema u milisekundama i čekanja godina za velike ulazne podatke.

## Stvarna upotreba

Programi promena stanja imaju mnogo primena u računarstvu i više:

- **Optimizacija putanje podataka**: GPS sistemi koriste algoritme transicije stanja da pronađu najkraći put između lokacija.
    
- **Obrađivanje teksta**: Alati za proveru pravopisa i funkcija automatskog dovršavanja često se oslanjaju na algoritme stanja prebacivanja kako bi izračunali editans (rastojanje uređivanja) između reči.
    
- **Kreiranje finansijskih modela**: Investicione strategije i optimizacija portfolija koriste metode transicije stanja.
    
- **Alokacija resursa**: Knapsak problem i njegove varijante javljaju se u planiranju rasporeda, budžetiranju i upravljanju resursima.

## Primer prakse: Problem razmene valute

Problem razmene valute je uobičajni izazov u programiranju koji, kada se reši korišćenjem dinamičkog programiranja, pokazuje dva glavna principa DP-a: optimalnu strukturu problema i preklapajuće podprobleme.

Problem konverzije valute pita: "Koliko minimalnih jedinica valute je potrebno da se napravi ciljani iznos?"

Evo jednog rešenja korišćenjem programa za promenu stanja:

```js
function minCoins(amount, coins) {
  // Create dp array from 0 to amount, filled with Infinity
  const dp = new Array(amount + 1).fill(Infinity);

  // Base case: 0 coins needed to make amount 0
  dp[0] = 0;

  // Build up from 1 to target amount
  for (let i = 1; i <= amount; i++) {
    // Try each coin denomination
    for (let coin of coins) {
      // Only use coin if it is not larger than current amount
      if (coin <= i) {
        // Choose the minimum: current value OR solution for remaining amount + 1 coin
        dp[i] = Math.min(dp[i], dp[i - coin] + 1);
      }
    }
  }

  // Return result if possible, -1 if impossible (infinity)
  return dp[amount] === Infinity ? -1 : dp[amount];
}

/*
 Example usage:
coins = [1, 3, 4], amount = 6

dp[0] = 0
dp[1] = dp[0] + 1 = 1
dp[2] = dp[1] + 1 = 2
dp[3] = min(dp[2]+1, dp[0]+1) = min(2+1, 0+1) = 1
dp[4] = min(dp[3]+1, dp[1]+1, dp[0]+1) = min(1+1, 1+1, 0+1) = 1
dp[5] = min(dp[4]+1, dp[2]+1, dp[1]+1) = min(1+1, 2+1, 1+1) = 2
dp[6] = min(dp[5]+1, dp[3]+1, dp[2]+1) = min(2+1, 1+1, 2+1) = 2

Result: 2 coins (3 + 3)
*/
```

Evo kako algoritam konverzije valute programa promena stanja funkcioniše korak po korak za `coins = [1, 3, 4]`, `amount = 6`:

```md
Initial state:
dp = [0, ∞, ∞, ∞, ∞, ∞, ∞]
     [0, 1, 2, 3, 4, 5, 6] ← amounts

Building up the solution:

For amount = 1:
  Try coin 1: dp[1] = min(∞, dp[0] + 1) = min(∞, 0 + 1) = 1
  dp = [0, 1, ∞, ∞, ∞, ∞, ∞]

For amount = 2:
  Try coin 1: dp[2] = min(∞, dp[1] + 1) = min(∞, 1 + 1) = 2
  dp = [0, 1, 2, ∞, ∞, ∞, ∞]

For amount = 3:
  Try coin 1: dp[3] = min(∞, dp[2] + 1) = min(∞, 2 + 1) = 3
  Try coin 3: dp[3] = min(3, dp[0] + 1) = min(3, 0 + 1) = 1
  dp = [0, 1, 2, 1, ∞, ∞, ∞]

For amount = 4:
  Try coin 1: dp[4] = min(∞, dp[3] + 1) = min(∞, 1 + 1) = 2
  Try coin 3: dp[4] = min(2, dp[1] + 1) = min(2, 1 + 1) = 2
  Try coin 4: dp[4] = min(2, dp[0] + 1) = min(2, 0 + 1) = 1
  dp = [0, 1, 2, 1, 1, ∞, ∞]

For amount = 5:
  Try coin 1: dp[5] = min(∞, dp[4] + 1) = min(∞, 1 + 1) = 2
  Try coin 3: dp[5] = min(2, dp[2] + 1) = min(2, 2 + 1) = 2
  Try coin 4: dp[5] = min(2, dp[1] + 1) = min(2, 1 + 1) = 2
  dp = [0, 1, 2, 1, 1, 2, ∞]

For amount = 6:
  Try coin 1: dp[6] = min(∞, dp[5] + 1) = min(∞, 2 + 1) = 3
  Try coin 3: dp[6] = min(3, dp[3] + 1) = min(3, 1 + 1) = 2
  Try coin 4: dp[6] = min(2, dp[2] + 1) = min(2, 2 + 1) = 2
  dp = [0, 1, 2, 1, 1, 2, 2]

Final result: dp[6] = 2 (achieved with coins 3 + 3)
```

Ovo rešenje pokazuje dva glavna principa programa za izračunavanje novca. Ima manji ponavljajući problemi jer je pronalaženje najmanjeg broja novčića za iznos od 6 jedinica zahteva poznavanje rešenja za iznose 5, 3 i 2. Ovi manji problemi se javljaju i prilikom računanja drugih iznosa. Ima optimalnu strukturu problema jer optimalno rešenje za bilo koji iznos uključuje optimalna rešenja za manje iznose. Ako znamo da je najmanji broj novčića za iznos od 3 jedinice jedan, onda je jedan način da se stvori iznos od 6 jedinica korišćenjem tog rešenja zajedno sa jednom novčićem vrednosti 3.

Bez DP-a, morali bismo testirati sve kombinacije valuta – broj varijacija jednačine. Sa DP-om, gradimo rešenje sekvencijalno:

- **Vremenska složenost**: `O(amount × number of coins)` umesto transformacije jednačine.
    
- **Prostorna složenost**: `O(amount)` za red od `dp`.
    
- **Nema ponavljanja posla**: Svaki mali problem (dobijanje nekoliko novčića za svaki iznos) rešava se samo jednom.
    
- **Ponovno upotrebljivi rezultati**: Kada znamo nekoliko novčića za iznos od 3, koristimo ovo znanje za veći iznos koji može da profitira.

## Kada koristiti program za promenu stanja?

Programa promene stanja je efikasan tamo:

- Problem se može razbiti na manje, ponavljajuće probleme.
    
- Problem ima optimalnu strukturu problema.
    
- Standardno rešenje putem rekurzije zahtevalo bi ponavljajuće proračune.
    
- Morate poboljšati vremensku složenost na uštrb prostorne složenosti.

Standardni sistemi programiranja prebacivanja stanja uključuju probleme optimizacije (pronalaženje minimalne/maksimalne vrednosti), brojbeni probleme (broj načina za postizanje nečega) i probleme odlučivanja koji se mogu razbiti na manje odluke.

Program transformacije stanja menja kompleksne probleme u upravljive probleme skladištenjem i ponovnom korišćenjem rešenja manjih problema. Razumevanje ove metode otvara vrata za efikasno rešavanje mnogih matematičkih izazova.
