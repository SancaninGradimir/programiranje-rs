## WebSockets protiv HTTP

- **HTTP** prati obrazac zahtev-odgovor: klijent šalje zahtev, server šalje odgovor, nakon čega se konekcija zatvara.
- **WebSockets** uspostavljaju持久nu, dvosmernu konekciju između klijenta i servera koja ostaje otvorena dok jedna od strana ne pošalje signal za zatvaranje.
- HTTP je dobar za učitavanje stranica ili dohvaćanje podataka, ali je spor i neefikasan za aplikacije u realnom vremenu.
- Svaka WebSocket konekcija počinje sa **rukojanjem HTTP**: klijent traži nadogradnju (upgrade), a server šalje odgovor od `101 Switching Protocols`.

### Tok WebSocket konekcije

1. Mteja se povezuje sa serverom preko HTTP kako bi tražio poboljšanje.
2. Server prihvata i šalje odgovor `101 Switching Protocols`.
3. Povezanje ostaje otvoreno.
4. Obje strane razmenjuju podatke u bilo kom trenutku dok se konekcija ne zatvori.

### Komunikacije HTTP naspram WebSocket-a

- **HTTP**: `Client → Request → Server → Response` (jedan ciklus, zatim se zatvara)
- **WebSocket**: `Client ↔ Server` (trajan, dvosmerni)

### Primer razumevanja

- **HTTP** je kao razmena pisama: ti napišeš jedno, čekaš odgovor, i razgovor se završava.
- **WebSockets** je kao telefonski poziv: obe strane razgovaraju u realnom vremenu na potreban period.

### Stvarne upotrebe WebSockets-a

- Aplikacija za razgovor
- Višigračni online naslovi
- Ažuriranja rezultata uživo sportskih događaja
- Dashboardi za tržište akcija
- Alati za saradnju (npr. Google Docs)

## Struktura Pub/Sub

- **Pub/Sub** (Objavljanje/Pretplata) je obrazac poruka koji omogućava komponentama sistema da komuniciraju bez direktnog znanja o jedni drugima.
- Postoje tri glavne komponente: **objavljivači**, **teme** i **pretplatnici**.
  - **Objavljivač** šalje poruku na temu.
  - **Tema** (takođe se naziva kanal ili grupa) funkcioniše kao posrednik.
  - **Pretplatnik** prima poruke sa teme kojoj je pretplaćen.

### Tok poruka za Pub/Sub

1. Korisnik se registruje za temu.
2. Autor šalje poruku toj temi.
3. Sistem šalje poruku svakom korisniku koji je registrovan za tu temu.

### Direktna komunikacija naspram Pub/Sub

- **Bez Pub/Sub-a**: `Application A → Application B` — bliska veza; promena B može uništiti A.
- **Sa Pub/Sub-om**: `Application A → Topic → Subscribers` — A samo objavljuje; ne mora da zna ko prima.

### Stvarne primene Pub/Sub

- Aplikacije za razgovor sa više soba
- Višigračni online igre
- Sistemi obaveštavanja/notifikacija
- Live kontrolne table (dashboardi)
- Aplikacije bazirane na događajima

### WebSockets protiv Pub/Sub

- **WebSockets** pruža kanal za komunikaciju (put/autoput).
- **Pub/Sub** pruža obrazac poruke koji određuje ko prima specifičnu poruku (saobraćajni sistem).
- Mnoge aplikacije u realnom vremenu koriste obe zajedno: WebSockets uspostavlja otvorenu konekciju, dok Pub/Sub rutira poruku do odgovarajućih pretplatnika.

## Rad sa WebSockets-ima u Node.js

- Node.js nema ugrađeni WebSocket server. Biblioteka `ws` je standardni eksterni paket koji se koristi za ovo.
- Instaliraj sa: `npm install ws`

### Kreiranje WebSocket servera

```js
const WebSocket = require("ws");

const server = new WebSocket.Server({ port: 3000 });
```

### Događaji za WebSocket

| Događaj | Okidač |
|---|---|
| `connection` | Novi klijent se povezuje sa serverom |
| `message` | Povezani klijent koristi podatke |
| `close` | Povezanje se zatvara |
| `error` | Došlo je do greške na vezi |

### Slanje poruke klijentu

```js
socket.send("Hello from the server");
```

### Slanje poruke svim povezanima klijentima

- `server.clients` ima sve povezane klijente.
- Proverite `readyState === WebSocket.OPEN` pre slanja kako biste izbegli greške za odvojene klijente.

```js
server.on("connection", (socket) => {
  socket.on("message", (message) => {
    server.clients.forEach((client) => {
      if (client.readyState === WebSocket.OPEN) {
        client.send(message.toString());
      }
    });
  });
});
```

- **Slanje poruka velikom broju korisnika istovremeno** je model slanja poruka velikom broju korisnika koji su povezani zajedno — koristi se u aplikacijama za razgovor, uživo obaveštenjima, ažuriranjima igara i događajima višegrališnih igara.
