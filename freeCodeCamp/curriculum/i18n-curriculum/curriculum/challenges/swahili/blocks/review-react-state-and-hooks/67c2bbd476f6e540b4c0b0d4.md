## Rad sa događajima u React

- **Sintetički sistem događaja**: Ovo je način React za rukovanje događajima. Deluje kao omotač nad izvornim događajima, kao što su događaji `click`, `keydown`, i `submit`. Event handler u React koristi konvenciju imenovanja 'camel case'. (Primer `onClick`, `onSubmit`, itd.)

Ovde je primer korišćenja atributa `onClick` za element `button` u React:

```jsx
function handleClick() {
  console.log("Button clicked!");
}

<button onClick={handleClick}>Click Me</button>;
```

U React, funkcije rukovaoca događaja obično počinju sa predfiksom `handle` što pokazuje da su uključene u obradu događaje, kao što su `handleClick` ili `handleSubmit`.

Kada korisnička akcija izazove događaj, React prosleđuje objekat Sintetičkog Događaja (Synthetic Event) vašem rukovaocu. Ovaj objekat funkcioniše kao nativni Objekat Događaja u standardnom JavaScript, pružajući svojstva kao što su `type`, `target`, i `currentTarget`.

Da biste sprečili podrazumevano ponašanje, kao što je osvežavanje pretraživača tokom događaja `onSubmit`, na primer, možete pozvati metodu `preventDefault()`:

```jsx
function handleSubmit(event) {
  event.preventDefault();
  console.log("Form submitted!");
}

<form onSubmit={handleSubmit}>
  <input type="text" />
  <button>Submit</button>
</form>;
```

Takođe možete pokriti funkcionalnost rukovaoca unutar strelične funkcije kao što je ovo:

```jsx
function handleDelete(id) {
  console.log("Deleting item:", id);
}

<button onClick={() => handleDelete(1)}>Delete Item</button>;
```

## Rad sa stanjem i hook-om za `useState`

- **Definicija stanja**: U React, stanje je nešto sa podacima za komponentu. Kada se stanje ažurira, komponenta će ponovo renderovati. React tretira stanje kao nepromenljivo (immutable), što znači da ga ne biste trebali menjati direktno.
- **Hook za `useState()`**: Hook za `useState` je funkcija koja vam omogućava deklarisanje varijabli stanja u funkcionalnim komponentama. Evo osnovne sintakse:

```js
const [stateVariable, setStateFunction] = useState(initialValue);
```

U parametru stanja imate sledeće:

- `stateVariable` Čuva trenutnu vrednost stanja
- `setStateFunction` (Funkcija za postavljanje vrednosti) ažurira parametar stanja
- `initialValue` Postavlja početno stanje

Ovde je kompletan primer sekcije `Counter`:

```jsx
import { useState } from "react";

function Counter() {
  const [count, setCount] = useState(0);

  return (
    <div>
      <h2>{count}</h2>

      <button onClick={() => setCount(count - 1)}>Decrement</button>
      <button onClick={() => setCount(count + 1)}>Increment</button>
    </div>
  );
}

export default Counter;
```

## Prikazivanje sa sekcijama za React

- **Definicija**: U React, prikazivanje je proces u kojem se delovi pojavljuju u korisničkom interfejsu (UI), obično u pretraživaču. Proces prikaza ima tri koraka: pokretanje, prikazivanje i autorizovanje.

Ovaj korak se dešava kada React detektuje da se nešto promenilo i da je potrebno ažuriranje UI. Ove promene često proizlaze iz ažuriranja stanja ili prosleđenih parametara.

Kada dođe do pokretačkog koraka, `React` ulazi u fazu prikaza. Ovde, `React` ponovo skenira vaše sekcije i određuje šta treba da se prikaže. Da bi to učinio, `React` koristi laganu kopiju stvarnog `DOM` nazvanu virtuelnim `DOM`. Sa virtuelnim `DOM`, `React` može brzo da proveri šta treba da se promeni u sekciji.

Korak odobravanja je mesto gde React uzima izmene pripremljene iz virtuelnog DOM i postavlja ih u stvarni DOM. Drugim rečima, ovde vidite konačan rezultat na ekranu.

## Ažuriranje stavki i redova podataka po statusu

- **Ažuriranje stavki u stanju**: Ako je potrebno ažurirati nešto u nekom stanju, prvo bi trebalo kreirati novi predmet ili kopirati postojeći, a zatim postaviti stanje za taj novi predmet. Svaki predmet postavljen u stanje treba tretirati kao nepromenljiv (immutable). Evo primera postavljanja imena, starosti i grada korisnika. Funkcija `handleChange` se koristi za obradu ažuriranja podataka korisnika:

```jsx
import { useState } from "react";

function Profile() {
  const [user, setUser] = useState({ name: "John Doe", age: 31, city: "LA" });

  const handleChange = (e) => {
    const { name, value } = e.target;

    setUser((prevUser) => ({...prevUser, [name]: value}));
  };

  return (
    <div>
      <h1>User Profile</h1>
      <p>Name: {user.name}</p>
      <p>Age: {user.age}</p>
      <p>City: {user.city}</p>

      <h2>Update User Age </h2>
      <input type="number" name="age" value={user.age} onChange={handleChange} />

      <h2>Update User Name </h2>
      <input type="text" name="name" value={user.name} onChange={handleChange} />

      <h2>Update User City </h2>
      <input type="text" name="city" value={user.city} onChange={handleChange} />
    </div>
  );
}

export default Profile;
```

- **Ažuriranje redova podataka u stanju**: Kada ažurirate redove podataka u nekom stanju, važno je da ih ne menjate direktno koristeći metode kao `push()` ili `pop()`. Umesto toga, trebalo bi da kreirate novi red kada ažurirate stanje:

```js
const addItem = () => {
  const newItem = {
    id: items.length + 1,
    name: `Item ${items.length + 1}`,
  };

  // Creates a new array
  setItems((prevItems) => [...prevItems, newItem]);
};
```

Ako želite ukloniti stavke iz niza, trebate koristiti metodu `filter()`, koja vraća novi niz nakon filtriranja stavki koje želite ukloniti:

```js
const removeItem = (id) => {
  setItems((prevItems) => prevItems.filter((item) => item.id !== id));
};
```

## Referenciranje vrednosti korišćenjem Refs

- **Atribut za `ref`**: Možete dobiti čvor (node) DOM u React koristeći atribut `ref`. Evo primera koji pokazuje `ref` kako bi se istakao element (ili: fokusirao na) `input`. Atribut `current` se koristi za dobijanje trenutne vrednosti `ref`:

```jsx
import { useRef } from "react";

const Focus = () => {
  const inputRef = useRef(null);

  const handleFocus = () => {
    if (inputRef.current) {
      inputRef.current.focus();
    }
  };

  return (
    <div>
      <input ref={inputRef} type="text" placeholder="Enter text" />
      <button onClick={handleFocus}>Focus Input</button>
    </div>
  );
};

export default Focus;
```

## Rad sa kukom za `useEffect`

- **Hook za `useEffect()`**: U React, efekat je bilo šta što se dešava izvan procesa prikazivanja komponente. To jest, bilo šta što React ne funkcioniše automatski kao deo prikaza interfejsa korisnika (UI). Uobiđi primeri uključuju dohvaćanje podataka, ažuriranje naslova kartice pretraživača, čitanje ili pisanje u keš pretraživača, dobijanje lokacije korisnika i slično. Ove operacije interaguju sa spoljnim svetom i poznate su kao sporedni efekti. React pruža hook `useEffect` kako bi se omogućilo rukovanje tim sporednim efektima. `useEffect` vam omogućava pokretanje funkcije nakon što je komponenta prikazana ili ažurirana.

```js
import { useEffect } from "react";

useEffect(() => {
 // Your side effect logic (usually a function) goes here
}, [dependencies]);
```

Funkcija efekta se izvršava nakon što je sekcija prikazana, kada argument opcionog `dependencies` određuje kada će taj efekat biti izvršen.

Zapamtite da `dependencies` može biti red „reaktivnih vrednosti“ (stanje, parametri, funkcije, kontekstualne varijable, itd.), prazan red ili potpuno nepostojeći. Ovo je kako sve ove opcije upravljaju/kontrolišu kako `useEffect` radi:

- Ako je `dependencies` red koji sadrži jednu ili više reaktivnih vrednosti, efekat će se izvršiti svaki put kada se oni promene.

- Ako je `dependencies` prazan red, `useEffect` će raditi samo jednom kada se sekcija prikaže prvi put.

- Ako ignorišete `dependencies`, efekat će biti primenjen pri svakom prikazivanju ili ažuriranju dela.

## Kako kreirati prilagođene hook-ove

- **Prilagođeni hookovi**: Prilagođeni hook vam omogućavaju da izvučete ponovo upotrebljivu logiku iz nekog mesta, kao što je dobijanje podataka, upravljanje stanjem, menjanje stanja i sporedni efekti, poput praćenja statusa mreže. U React, svi ugrađeni hookovi počinju sa reči `use`, stoga vaš prilagođeni hook bi trebalo da prati isti ugovor.

Evo primera kako kreirati hook za `useDebounce`:

```js
function useDebounce(value, delay) {
  const [debouncedValue, setDebouncedValue] = useState(value);

  useEffect(() => {
    const handler = setTimeout(() => {
      setDebouncedValue(value);
    }, delay);

    return () => {
      clearTimeout(handler);
    };
  }, [value, delay]);

  return debouncedValue;
}

export { useDebounce };
```
