# --description--

U Express, možete kreirati **modularnog rukovaoca rute podataka** koristeći metodu `app.route()`. Ovo je korisno kada želite da obrađujete više ruta za HTTP kao što su GET, POST ili PUT za istu rutu podataka bez ponavljanja te rute svaki put.

Umesto pisanja različitih definicija putanje podataka kao `app.get("/user")`, `app.post("/user")`, i `app.put("/user")`, `app.route()`, to vam omogućava da ih spojite u jedan deo generisanog koda.

Putanja `app.route()` deluje kao kratak način za definisanje više *handler-a* sa jednim krajnjim tačkom. Ovo je posebno važno u API za RESTful, gde jedan resurs (kao `/user`) često podržava mnogo operacija.

Evo primera koji simulira API za RESTful upravljanje korisnicima:

```javascript
const express = require("express");
const app = express();
const port = 3000;

app.use(express.json()); // Parse JSON request bodies

// Use app.route() to group handlers for the same path
app
  .route("/user")
  .get((req, res) => {
    res.send("Fetching user data");
  })
  .post((req, res) => {
    const { name } = req.body;
    res.send(`Creating a user with name: ${name}`);
  })
  .put((req, res) => {
    const { name } = req.body;
    res.send(`Updating the user to: ${name}`);
  })
  .delete((req, res) => {
    res.send("Deleting the user");
  });

app.listen(port, () => {
  console.log(`Server running at http://localhost:${port}`);
});
```

U gornjem primeru, `app.route("/user")` opisuje jedan put podataka putem `/user`. Zatim povezujemo više puta od HTTP, `.get()`, `.post()`, `.put()`, i `.delete()`, obrađujući različite tipove zahteva na isti način.

Evo pregleda različitih metoda i kako mogu funkcionisati u jednostavnoj RESTful aplikaciji:

* GET `/user`: Preuzima i vraća podatke o korisniku.
    
* POST `/user`: Kreira novog korisnika koristeći podatke iz tela zahteva.
    
* PUT `/user`: Inasasisha mtumizi aliyepo kwa data zilizopo katika mwili wa ombi.
    
* DELETE `/user`: Inafuta mtumizi.
    

Korišćenje `app.route()` pomaže da kod bude organizovan kada obrađujete više HTTP metoda za istu rutu. Umesto da više puta ponavljate istu rutu, povezane obrađivače grupišete na jednom mestu.

To smanjuje ponavljanje i čini kod lakšim za čitanje i održavanje. Kada su svi obrađivači za jednu rutu definisani na jednom mestu, jasnije je kako se ta krajnja tačka ponaša.

Kako vaša aplikacija raste i kada postaju složenije putanje podataka, `app.route()` pruža jednostavan način za grupisanje povezanih logika. To je posebno važno prilikom dizajniranja API resursa (RESTful), gde jedan resurs često podržava više operacija kao što su GET, POST, PUT i DELETE.
