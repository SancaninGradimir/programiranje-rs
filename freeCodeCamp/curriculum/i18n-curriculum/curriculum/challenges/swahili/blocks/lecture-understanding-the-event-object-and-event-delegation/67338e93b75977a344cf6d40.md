# --description--

Propagacija događaja, ili distribucija, odnosi se na način na koji se događaj širi do roditeljskih elemenata kada je pokrenut. Na primer, razmislite o ovom kodu:

```html
<p>
  <span>Click me~!</span>
</p>
```

Komponenta za `p` ovde će biti uzeta kao roditeljska komponenta za `span`.

Kada kliknete na element `span`, kao što je uputeno, element `span` postaje cilj događaja za `click`. Međutim, taj događaj takođe se propagira do roditelja – element `p` može primiti i koristiti taj događaj po potrebi.

Ali šta tačno ovo znači? Pa, možete priložiti slušaoca događaja na element za `p`:

```js
const p = document.querySelector("p");
p.addEventListener("click", (event) => {
  console.log(event.target);
});
```

Nakon toga, kada kliknete na element `span`, videćete tekst `Click me~!` koji se piše u konzolu.

Događaj se propagira do roditeljskog elementa `p`, koji ga koristi u listeneru događaja da bi prikazao svrhu događaja.

Pogledajte kako je cilj i dalje element od `span`. Ovo je zato što je element `span` primio prvi klik.

Da bismo proverili kako sve funkcioniše, dodati ćemo naš kod:

```js
const p = document.querySelector("p");
const span = document.querySelector("span");
p.addEventListener("click", (event) => {
  console.log("P listener: ");
  console.log(event.target);
});
span.addEventListener("click", (event) => {
  console.log("Span listener: ");
  console.log(event.target);
});
```

Da vam dam ideju kako se događaj odvija, ovde je ono što ćete videti u konzoli nakon klika na element/funkciju `span`:

```html
"Span listener: "
<span>Click me~!</span>
"P listener: "
<span>Click me~!</span>
```

Sada ćemo videti šta se dešava kada blokira distribuciju događaja za `stopPropagation()`. Spomenućemo ga u slušaocu događaja za `span`:

```js
const p = document.querySelector("p");
const span = document.querySelector("span");
p.addEventListener("click", (event) => {
  console.log("P listener: ");
  console.log(event.target);
});
span.addEventListener("click", (event) => {
  console.log("Span listener: ");
  console.log(event.target);
  event.stopPropagation();
});
```

Zatim kliknite ponovo `span`:

```html
"Span listener: "
<span>Click me~!</span>
```Ovaj put, ne vidimo slušaoca za `p` pokrenutog. Događaj nikada nije bio aktiviran za karakteristiku `p`, jer smo rekli da ne sme da se širi dok ga obrađuje podfunkcija `span`.

Emitovanje događaja se može smatriti suprotnim. To je proces uhvaćanja događaja i prosleđivanja ga drugoj komponenti.

Kada se vratimo na naš kod, poboljšaćemo ga da klikne na element `span` kako bi promenio njegovu boju u crvenu:

```js
const p = document.querySelector("p");
const span = document.querySelector("span");
p.addEventListener("click", (event) => {});
span.addEventListener("click", (event) => {
  event.target.style.color = "red";
});
```

Ali šta ako imaš dvadeset komponenti za `span`? Ili možda koristiš JavaScript da kreiraš više komponenti za `span` brzo?

Umesto da prilažete listener događaja svakom elementu `span`, možete koristiti listener na komponenti `p` za sve. Drugim rečima, možete prebaciti upravljanje klikovima za `span` na roditeljsku komponentu `p`.

Naš kod sada može izgledati ovako:

```js
const p = document.querySelector("p");
p.addEventListener("click", (event) => {
  event.target.style.color = "red";
});
```

Pogledajte kako nismo povezali nijedvog slušaoca sa elementom `span`. Ispravno ste prebacili upravljanje događajem na element `p`. Ali da li to radi?

Dodajte nekoliko dodatnih komponenti za `span` i da vidimo:

```html
<p>
  <span>Click me~!</span>
  <span>Click me~!</span>
  <span>Click me~!</span>
  <span>Click me~!</span>
</p>
```

Sada, svaki put kada kliknemo `span`, tekst tog elementa će biti crven.

I tako, za jednog slušaoca događaja dozvolili smo da događaj `click` pređe iz komponenti `span` u roditeljski element `p`, i premestili logiku događaja `click` na komponentu `p`.

Distribucija i emitovanje događaja mogu biti kompleksna tema, posebno kada se radi sa visoko ugnježdenim elementima kao što su tabele. Preporučuje se da istražite više i isprobate neke od skripti koje smo ovde napisali.
