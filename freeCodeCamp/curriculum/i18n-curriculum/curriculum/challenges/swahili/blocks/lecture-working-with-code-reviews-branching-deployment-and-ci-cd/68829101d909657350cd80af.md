# --description--

GitHub Pages je veoma dobro rešenje za objavljivanje statičkih veb-sajtova ili aplikacija kojima nije potreban serverski deo za izvršavanje logike. Drugim rečima, aplikacije koje u potpunosti rade na strani klijenta, odnosno u korisnikovom pregledaču, mogu se u potpunosti objaviti na ovoj platformi.

Zaidi ya hayo, GitHub Pages je potpuno besplatan za projekte sa otvorenim izvorom. Postoji mnogo dobrih razloga da učinite svoj projekat otvorenim izvorom, a Pages dodaje još jedan razlog.

Ali kako deployiraš projekat na GitHub Pages? To zavisi od toga kako tvoja aplikacija funkcioniše.

Počinjemo sa osnovnom stranicom za HTML. Ovo je primer koji možete postaviti u svoj postojeći repozitorijum ako želite:

```html
<!DOCTYPE html>
<html>
  <head>
    <title>freeCodeCamp</title>
    <link rel="stylesheet" href="./style.css" />
  </head>
  <body>
    <h1>Welcome to freeCodeCamp's Lessons!</h1>
    <p>Let's have some fun learning about Git and GitHub!</p>
  </body>
</html>
```

Sada treba da posetiš svoj repozitorijum i klikneš na karticu "Settings" da bi otvorio njegova podešavanja. Zatim izaberi "Pages" u bočnom meniju.

U početku će biti dostupno samo nekoliko opcija, ali će se to promeniti kada počneš sa podešavanjem. Prvo treba da kažeš GitHub-u odakle želiš da objaviš sajt.

Preferiraćemo da ostavimo ovo kao "Deploy from a branch", jer ne koristimo radni tok i uopšte nismo konfigurisali akcije za GitHub. Vi ćete više naučiti o ovome u sledećoj lekciji.

Sledeće, postavi granu koju distribuiraćemo od `main` ili distribuirati ono što je tu.

Zatim ćeš moći da biraš da li će se sajt objaviti iz korenskog direktorijuma projekta ili iz fascikle `docs`. Ovu mogućnost koriste projekti poput freeCodeCamp-a, gde se dokumentacija nalazi zajedno sa aplikacijom, pa se na GitHub Pages objavljuje samo sadržaj fascikle `docs`. Za sada ostavi opciju `/` (korenski direktorijum) i klikni na dugme "Save" da sačuvaš promene.

Kada sačuvate, videćete novo polje za konfigurisanje specifičnog domena. Ako već imate domen i želite da ga koristite, možete pratiti dokumentacioni link za to. Za ovaj tutorijal, međutim, ostavićemo ga otvorenim.

Moći ćeš da vidiš i adresu na kojoj je objavljen tvoj GitHub Pages sajt. Ako nemaš prilagođeni domen, adresa će biti `<username>.github.io`, gde je `<username>` ime tvog naloga (ili organizacije ako objavljuješ iz repozitorijuma organizacije). Repozitorijum će biti dostupan na putanji koja odgovara njegovom imenu, na primer `<username>.github.io/<repo-name>`. Kada posetiš taj URL, videćeš svoj sajt!

Sada, kada gurnete nove promene u granu `main`, GitHub Pages će automatski ažurirati naš veb sajt koji je deployovan sa najnovijim promenama. Ovaj proces može potrajati nekoliko minuta, u zavisnosti od opterećenja servera – i možda ćete morati da izvršite hard refresh kako biste obrisali keš.

Takođe, uspešno smo implementirali osnovnu web aplikaciju na GitHub Pages. Takođe možete uključiti fajlove CSS i JavaScript u vaše Page deployovanje, pa tako možete kreirati lepe i kooperativne sajtove. I kada naučite sistem radnog toka kao što su React ili Angular, možete ih čak implementirati i na GitHub Pages!
