# --interactive--

Grupa za hvatanje omogućava vam da „uhvatite“ deo odgovarajuće sekvence karaktera ili ga koristite na bilo koji način koji vam je potreban. Grupe za hvatanje definišu se standardnim zagradama koje okružuju uzorak za hvatanje, bez početnih karakteristika kao što je predgled (lookahead).

Hajde da izdvojimo `code` iz stringa `freeCodeCamp`. Da bismo to uradili, okružićemo `code` običnim zagradama i označiti ga kao grupu za hvatanje:

```js
const regex = /free(code)camp/i;
```

Da bismo potvrdili ovaj obrazac, možemo ga testirati protiv niza karaktera `freecodecamp`:

:::interactive_editor

```js
const regex = /free(code)camp/i;
console.log(regex.test("freecodecamp")); // true
```

:::

Ali ovo ne koristi našu grupu za hvatanje. Umesto toga, pogledajmo rezultat korišćenja metode `match`:

```js
const regex = /free(code)camp/i;
console.log("freecodecamp".match(regex));
// [
//   'freecodecamp',
//   'code', <--
//   index: 0,
//   input: 'freecodecamp',
//   groups: undefined
// ]
```

Ovde vidimo da niz koji vraća `match` sadrži drugi element, koji predstavlja deo stringa koji je uhvaćen našom grupom za hvatanje.

Obratite pažnju kako grupa za hvatanje odgovara celom obrascu `code`, dok bi klasa karaktera odgovarala samo jednom karakteru iz skupa `c`, `o`, `d` i `e`.

Ali kako možemo da ovo koristimo? Dobro, grupe za hvatanje se često koriste prilikom menjanja sadržaja niza znakova. Pripremaćemo kod da to uradi. Zamenićemo `freecodecamp` sa `paidcodeworld`:

:::interactive_editor

```js
const regex = /free(code)camp/i;
console.log("freecodecamp".replace(regex, "paidcodeworld"));
```

:::

Ovo radi samo od sebe, ali šta ako ne znamo koliko `o` su u `code`? Ako nam je potreban okidač za jedan ili više `o`:

:::interactive_editor

```js
const regex = /free(co+de)camp/i;
console.log("freecoooooooodecamp".replace(regex, "paidcodeworld"));
```

:::

Dobijamo `paidcodeworld` kao naš rezultat. Želimo sačuvati broj `o`, pa nam je potrebno ponovo koristiti ono što je uhvaćeno pomoću regularnog izraza.

Ovde ulazi povratna referenca. Umesto da ručno postavimo deo `code` u niz za zamenu, možemo direktno referencirati uhvaćenu grupu.

U pozivu funkcije `replace`, na grupu za hvatanje se poziva pomoću znaka dolara (`$`) i broja grupe. U našem slučaju to je `$1`, pošto je `code` uhvaćen u prvoj grupi za hvatanje:

:::interactive_editor

```js
const regex = /free(co+de)camp/i;
console.log("freecoooooooodecamp".replace(regex, "paid$1world")); // paidcoooooooodeworld
```

:::

Sada smo uspešno sačuvali nepoznat broj karaktera za `o` dok menjamo `freecodecamp` u `paidcodeworld`. Međutim, povratne reference nisu ograničene samo na poziv za zamenu (replace). Možete ih koristiti direktno u regularnom izrazu.

Ovo će vam omogućiti da uporedite uzorak koji je prethodno snimljen kasnije u prirodnom govoru.

Recimo da želimo da uparimo `freecodecamp` dva puta, i jednak broj od `o`, ali bilo gde u nizu slova.

Prvo, moramo ih razdvojiti našim wildcard karakterom, i omogućiti bilo koji broj znakova u skladu sa tim wildkardom:

```js
const regex = /free(co+de)camp.*free(co+de)camp/i;
```

Ova trenutna upotreba neće osigurati da je broj znakova `o` jednak, međutim. Da bismo to postigli, moramo promeniti drugu grupu za hvatanje i referencirati prvu grupu.

U opštem govoru, povratna referenca se prikazuje sa kosom crtom praćenom brojem uhvatne grupe:

:::interactive_editor

```js
const regex = /free(co+de)camp.*free\1camp/i;
console.log(regex.test("freecooooodecamp is great i love freecooooodecamp")); // true
console.log(regex.test("freecooooodecamp is great i love freecodecamp")); // false
```

:::

Dakle, vidimo da niz karaktera sa tačnim brojem `o` se poklapa, dok niz sa dva različita broja `o` ne poklapa.

Ova sintaksa je dobra, ali može brzo da postane komplikovana kada se vratite na više grupa za hvatanje. Srećom, umesto korišćenja brojeva, možete vašim grupama dati imena.

Da biste definisali nazvani grupni uhvat, dodajete znak pitanja (`?`) praćen nazivom okružen manjim i većim znakovima na početku grupe. Neka bude našu grupnu hvatanje `code`:

```js
const regex = /free(?<code>co+de)camp.*free\1camp/i;
```

Sada možemo ispraviti našu povratnu referencu u regularni izraz kako bi se odnosila na ovu grupu. Imenovana povratna referenca počinje sa kosom crtom praćeno karakterom `k` u JavaScript. Zatim dodajete ime, ponovo okruženo znakovima manjim od (`<`) i većim od (`>`). Pogledajte to:

```js
const regex = /free(?<code>co+de)camp.*free\k<code>camp/i;
```

Sada ako pogledamo naš poziv za `test()`, možemo da vidimo da i dalje prolazimo:

:::interactive_editor

```js
const regex = /free(?<code>co+de)camp.*free\k<code>camp/i;
console.log(regex.test("freecooooodecamp is freecooooodecamp")); // true
```

:::

Kada koristimo našu nazvanu grupu za hvatanje u pozivu `replace()`, unosite znak dolara u niz slova, praćen imenom okruženim znakovima manjeg i većeg od:

:::interactive_editor

```js
const regex = /free(?<code>co+de)camp/i;
console.log("freecooooodecamp".replace(regex, "paid$<code>camp")); // paidcooooodecamp
```

:::

Na kraju, ponekad želiš da kreiraš grupu slova, ali ti ne trebaju uhvaćeni rezultati.

Pretpostavimo da želimo da pronađemo `freecodecamp` ili `freecandycamp`. Možete napraviti dva obrasca razdvojena OR operatorom:

```js
const regex = /freecodecamp|freecandycamp/i;
```

Ali to može postati veoma dugačko za složenije regularne izraze. Umesto toga, možete koristiti grupu koja ne hvata kako biste obuhvatili deo koji želite da razdvojite OR operatorom:

```js
const regex = /free(?:code|candy)camp/i;
```

Grupa koja ne hvata ne čuva podudaranje `code|candy` u memoriji. Međutim, pomaže u pravljenju alternativnih obrazaca bez uticaja na čitljivost ili performanse.
