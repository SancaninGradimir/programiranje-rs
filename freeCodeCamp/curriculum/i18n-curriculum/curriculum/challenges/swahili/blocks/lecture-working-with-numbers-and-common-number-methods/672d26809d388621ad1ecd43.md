# --interactive--

U JavaScript, `NaN` predstavlja "Nije broj". To je posebna vrednost koja predstavlja rezultat broja koji ne može biti predstavljen ili nije definisan. `NaN` je atribut globalnog objekta, i takođe se tretira kao tip broja u JavaScript, što na početku može izgledati protiv očekivanja.

`NaN` Ovo je obično rezultat operacije koja bi trebalo da vrati broj, ali ne može da pruži smislenu numeričku vrednost. Primer:

:::interactive_editor

```js
let result = 0 / 0;
console.log(result); // NaN
```

:::

U ovom slučaju, deljenje nule sa nulom je matematički nedefinisano, stoga JavaScript vraća `NaN`. Jedna jedinstvena karakteristika `NaN` je da ne odgovara ničemu, čak ni samom sebi:

:::interactive_editor

```js
console.log(NaN === NaN); // false
```

:::

Ovo jedinstveno ponašanje čini izazovnim proveravati da li je vrednost `NaN` korišćenjem standardnih komparatora. Da bi se rešio ovo, JavaScript pruža funkciju `isNaN()`. Svojstvo funkcije `isNaN()` koristi se za određivanje da li je vrednost `NaN` ili ne. Međutim, važno je razumeti kako `isNaN()` funkcioniše, jer ponekad može da proizvede neočekivane rezultate. Evo kako `isNaN()` funkcioniše:

:::interactive_editor

```js
console.log(isNaN(NaN));       // true
console.log(isNaN(undefined)); // true
console.log(isNaN({}));        // true

console.log(isNaN(true));      // false
console.log(isNaN(null));      // false
console.log(isNaN(37));        // false

console.log(isNaN("37"));      // false: "37" is converted to 37
console.log(isNaN("37.37"));   // false: "37.37" is converted to 37.37
console.log(isNaN(""));        // false: empty string is converted to 0
console.log(isNaN(" "));       // false: string with a space is converted to 0

console.log(isNaN("blabla"));  // true: "blabla" is not a number
```

:::

Kao što vidite, `isNaN()` pokušava prvo da konvertuje parametar u broj. Ako ne može biti konvertovan, vraća `true`. Ovo ponašanje može izazvati neočekivane rezultate, posebno prilikom obrade sekvence karaktera koja može biti prisiljena da postane broj.

Zbog ove moguće promene, ES6 (šesta verzija JavaScript, izdata 2015.) uveo je `Number.isNaN()`. Ova metoda ne pokušava da konvertuje argument u broj pre testiranja. Vrati samo `true` ako je vrednost tačno `NaN`:

:::interactive_editor

```js
console.log(Number.isNaN(NaN));        // true
console.log(Number.isNaN(Number.NaN)); // true
console.log(Number.isNaN(0 / 0));      // true

console.log(Number.isNaN("NaN"));      // false
console.log(Number.isNaN(undefined));  // false
console.log(Number.isNaN({}));         // false
console.log(Number.isNaN("blabla"));   // false
```

:::

`Number.isNaN()` pruža pouzdaniji način od proveravanja vrednosti `NaN`, posebno u slučajevima kada eksplicitno konvertovanje tipa može uzrokovati neočekivane rezultate za globalnu funkciju `isNaN()`. U praksi, prilikom rukovanja operacijama brojeva ili unosom korisnika koji bi trebalo da bude broj, često je važno proveriti `NaN` kako bi se greške ili neočekivani unosi pravilno (ili elegantno) obradili. Na primer:

:::interactive_editor

```js
let a = 0;
let b = 0;
let result = a / b;

if (Number.isNaN(result)) {
  result = "Error: Division resulted in NaN";
}

console.log(result); // "Error: Division resulted in NaN"
```

:::

U ovom primeru, koristimo `Number.isNaN()` da uhvatimo slučaj gde operacija deljenja izaziva `NaN`, i omogućava nam da adekvatno rešimo ovu situaciju. Razumevanje `NaN` i kako pravilno proveriti je važno za pisanje robusnog koda za JavaScript, posebno prilikom rukovanja matematičkim operacijama ili analize unosa korisnika.
