# --description--

Node.js Nema ugrađenog WebSocket servera. Programeri obično koriste eksterne pakete kao `ws` da implementiraju WebSocket protokol i upravljaju komunikacijom u realnom vremenu između klijenata i servera.

Biblioteka `ws` je lagana, brza i jednostavna za upotrebu. Implementira WebSocket protokol kako biste mogli kreirati i upravljati WebSocket konekcijom.

Možete ga instalirati koristeći npm:

```bash
npm install ws
```

Nakon postavljanja, kreirajte WebSocket server uvozi brojne biblioteke `ws`, pozovite njenu funkciju `Server` i prosledite port:

```js
const WebSocket = require("ws");

const server = new WebSocket.Server({
  port: 3000,
});
```

Ovo inicijalizuje WebSocket server koji sluša konekcije na port `3000`.

Kada se klijent poveže, server šalje događaj `connection`.

```js
server.on("connection", (socket) => {
  console.log("Client connected");
});
```

Zamisli ovaj događaj kao način da server kaže: 'novi korisnik je pridružio'.

Parametar ``socket`` predstavlja konekciju između servera i tog klijenta. Koristi se za slanje i primanje poruka.

Povezani klijent kada pošalje podatke, server šalje događaj `message`.

```js
server.on("connection", (socket) => {
  socket.on("message", (message) => {
    console.log(message.toString());
  });
});
```

U ovom primeru, server sluša dolazne poruke i piše ih na konzolu.

Možete poslati podatke nazad kupcu koristeći metod `send()`:

```js
socket.send("Hello from the server");
```

Ovo šalje poruke putem otvorene WebSocket konekcije.

Dok je konekcija otvorena, klijent i server mogu nastaviti razmenu poruka u realnom vremenu.

Osim `connection` i `message`, WebSocket konekcija takođe šalje događaje za `close` i `error`.

`close` To se dešava kada se konekcija prekine:

```js
socket.on("close", () => {
  console.log("Disconnected");
});
```

`error` nastaje kada postoji problem:

```js
socket.on("error", (error) => {
  console.error(error);
});
```

Takođe, ovi događaji su osnova za mnoge WebSocket aplikacije.

Uobičajena potreba u aplikacijama za poruke je broadcastovanje. Broadcastovanje znači slanje poruke više povezanih korisnika umesto samo jednog.

Zamislite sočijalni prostor sa tri korisnika, imenovana **John Doe**, **Jane Doe** i **Wes Smith**.

Ako John Doe pošalje poruku, server možda mora da pošalje tu poruku i Jane Doe i Wesu Smithu.

Da bi to uradio, server čuva podatke korisnika koji su se prijavili i njihovu aktivnost kada stigne poruka.

Pre slanja poruke, molimo proverite da li je konekcija klijenta još uvek otvorena:

```js
if (client.readyState === WebSocket.OPEN) {
  // do something
}
```

Svaki WebSocket konekcija ima atribut `readyState` koji određuje njen trenutni status.

`WebSocket.OPEN` To znači da je konekcija aktivna i spremni ste za slanje i primanje podataka.

Za proveru ovoga, server sprečava slanje poruka klijentima koji su se odjavili ili čija je konekcija još u procesu uspostavljanja:

```js
server.on("connection", (socket) => {
  socket.on("message", (message) => {
    server.clients.forEach((client) => {
      if (client.readyState === WebSocket.OPEN) {
        client.send(message.toString());
      }
    });
  });
});
```

U ovom primeru, svaki povezani klijent prima dolaznu poruku. Ovo se zove broadcastovanje.

Mnoge aplikacije u realnom vremenu koriste broadcastovanje za distribuciju podataka kao što su poruke u chatu, live obaveštenja, informacije o igrama i događaji višeg igrača.

Za biblioteku `ws`, lako je izgraditi WebSocket servere u Node.js.

Potrebno je kreirati server, pratiti događaje, razmenjivati poruke i emitovati ažuriranja povezanih klijentima.

Ova zaključanja su osnova za mnogo programa u realnom vremenu.
