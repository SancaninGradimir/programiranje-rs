# --description--

U Express-u, **rute** su URL obrasci koji određuju kako server odgovara na dolazne zahteve. Kada korisnik zatraži određeni URL, Express uparuje taj URL sa rutom i izvršava odgovarajući obrađivač.

Rute se definišu pomoću metoda kao što su `app.get()`, `app.post()` i slično. One mogu biti statične, poput `/home`, ili dinamičke, gde se parametri koriste za preuzimanje vrednosti iz URL-a, kao što je `/user/:id`.

U ovom modu ćemo razmotriti trajne puteve podataka. Evo jednog primera:

```javascript
// Static route
app.get("/home", (req, res) => {
  res.send("Welcome to the homepage!");
});
```

U ovom primeru, server sluša GET zahteve za `/home` i vraća jednostavan porukasti odgovor.

Putanja za `app.get()` prima dva argumenta:

* Prvi argument, `"/home"`, predstavlja statičku rutu (ili URL) koju server osluškuje.
    
* Drugi argument je funkcija povratnog poziva koja se izvršava kada se pošalje GET zahtev toj ruti.
    

Unutar toka, `res.send()` šalje odgovor nazad korisniku. U ovom slučaju, šalje tekst `Welcome to the homepage!`, koji se prikazuje u pretraživaču.

Evo još jednog primera trajne putanje podataka:

```javascript
// Static route
app.get("/menu/drinks", (req, res) => {
  res.send("Welcome to the drinks menu!");
});
```

U ovom slučaju server osluškuje GET zahteve za URL `/menu/drinks` i šalje odgovor kada se zatraži ta ruta.

Ovakve rute možete koristiti kada vaš sajt ima više povezanih stranica grupisanih pod zajedničkom putanjom. Na primer, sajt restorana može organizovati stranice svog menija ovako:

* `/menu/food`
    
* `/menu/drinks`
    
* `/menu/desserts`
    

Ovo pomaže da zadržite vaše URL jasnim i smislenim, te takođe čini vaše putanje podataka lakim za organizovanje i održavanje dok vaša aplikacija raste.

U trening okruženju, pregledaće dati putanje podataka promena sa parametrima.
