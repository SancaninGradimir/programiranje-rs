## Uvod u kontrolu verzija

- **Opis**: Sistem za kontrolu verzija omogućio vam da pratite i upravljate promenama u vašem projektu. Primeri sistema za kontrolu verzija koji se koriste u softveru su Git, SVN, ili Mercurial.

## Pružaoci usluga kontrole verzija bazirani na oblaku

- **Katalog pružalaca usluga kontrole verzija zasnovanih na oblaku**: GitHub na GitLab su popularni primeri pružalaca usluga kontrole verzija baziranih na oblaku koji omogućavaju razvojnim timovima da sarađuju i upravljaju bazama podataka.

## Postavljanje i konfigurisanje Git

- **Postavljanje Git**: Da proverite da li Git već postoji na vašem računaru, možete pokrenuti sledeću komandu u terminalu:

```sh
git --version
```

Ako vidite broj verzije, to znači da je Git postavljen. Ako nije, onda ćete morati da ga postavite.

Za sisteme Linux, Git često dolazi predinstalirano u mnogim distribucijama. Ako nemate Git predinstaliran, trebalo bi da ga instalirate pomoću komandi menadžera paketa kao `sudo apt-get install git` ili `sudo pacman -S git`.

Za korisnike Mac-a, možete instalirati Git putem Homebrew za `brew install git`, ili možete preuzeti instalacioni paket sa veb lokacije Git.

Za Windows, možete preuzeti instalacioni paket sa sajta Git. Ili, ako ste instalirali Chocolatey, možete pokrenuti `choco install git.install` u PowerShell-u. Zapamtite da na Windows-u možda želite preuzeti Git Bash kako biste imali okruženje šelela slično Unix-u.

Da biste potvrdili da je instalacija uspešna, ponovo pokrenite komandu `git --version` u terminalu.

- **Podešavanja za Git**: ``git config`` se koristi za postavljanje parametara konfiguracije koji su povezani sa načinom na koji Git radi na vašem računaru. Da biste videli vaše trenutne parametre konfiguracije i gde su sačuvani u vašem sistemu, možete pokrenuti sledeću komandu:

```sh
git config --list --show-origin
```

Trenutno bi trebalo da vidite samo sistemsku konfiguraciju ako ste Git postavili prvi put.

Da biste postavili vaše korisničko ime, možete pokrenuti sledeću komandu:

```sh
git config --global user.name "Jane Doe"
```

Bendera ya `--global` koristi se ovde za postavljanje korisničkog imena za sve projekte na vašem sistemu koji koristi Git. Ako je potrebno ukloniti korisničko ime za određeni projekat, onda možete pokrenuti tu komandu u direktorijumu tog projekta bez bendere (ili: *bez opcije*) `--global`.

Da biste postavili adresu e-pošte korisnika, možete izvršiti sledeću komandu:

```sh
git config --global user.email janedoe@example.com
```

Druga konfiguracija koju možete postaviti je preferirani editor Git, koristite ga. Evo primera kako postaviti preferirani editor na Emacs:

```sh
git config --global core.editor emacs
```

Ako ne želiš da postaviš omiljeni uređivač, onda Git će koristiti podrazumevani uređivač tvog sistema.

## Program za Otvoreni izvor naspram zatvorenog izvora

- **Definicija**: „Otvoreni kod“ znači da ljudi mogu videti objavljeni kod, predložiti izmene, prijaviti probleme i čak pokrenuti izmenjenu verziju. „Zatvoreni kod“ znači da su jedine osobe koje mogu videti i interagovati sa projektom one kojima je eksplicitno dozvoljeno.

## GitHub

- **Ufafanuzi**: GitHub je rešenje zasnovano na oblaku koje pruža skladištenje projekata sa kontrolom verzija u objektu nazvan "baza podataka", i omogućava funkcionalnosti za saradnju da koriste te projekte.
- **GitHub CLI**: Ovaj alat se koristi za izvršavanje specifičnih funkcija GitHub bez napuštanja komandne linije. Ako ga niste instalirali, možete pronaći uputstva o tome kako da to učinite u dokumentaciji GitHub - ali bi trebalo da je dostupno u menadžeru paketa vašeg sistema.
- **GitHub Pages**: GitHub Pages je opcija za hostovanje statičkih sajtova, ili aplikacija koje ne zahtevaju backend server za obradu logike. Drugim rečima, aplikacije koje koriste samo klijentsku stranu, ili u pregledaču korisnika, mogu biti potpuno hostovane na ovoj platformi.
- **GitHub Actions**: GitHub Actions je funkcionalnost koja vam omogućava da pokrenete automatizovane serije zadataka direktno u vašu bazu podataka GitHub, uključujući izgradnju (build), testiranje i implementaciju (deployment) vašeg koda.

## Komande za Standarde za Git

- **`git init`**: Ovo će inicijalizovati praznu bazu podataka Git ili Git da počne da prati promene ovog projekta. Kada inicijalizujete praznu bazu podataka Git za projekat, dodaće se nova skrivena direktorijumska putanja `.git`. Ova putanja `.git` sadrži ključne informacije za Git da upravlja vašim projektom.
- **`git status`**: Ova komanda se koristi za prikaz trenutnog stanja vaše radne direktorijumske putanje – Koristićete ovu komandu često u vašem radnom procesu.
- **`git add`**: Ova komanda se koristi za dodavanje vaših promena u stanište (staging area). Sve što se nalazi u staništu će biti dodato sledećoj snimci (commit). Ako želite da dodate sve nepripremljene promene, možete koristiti `git add .` Period (`.`) je alias za trenutnu direktorijumsku putanju.
- **`git commit`**: Ova komanda se koristi za kreiranje snimka (commit) vaših promena. Snimak je fotografija stanja vašeg projekta u tom trenutku. Kada izvršite `git commit`, otvoriće se vaš omiljeni editor definisan u konfiguraciji Git. Kada otvorite editor, možete napisati detaljnu poruku o promenama. Takođe, možete poslati kratku poruku koristeći komandu `git commit -m`, kao što je sledeće:

```sh
git commit -m "short message goes here"
```

- **`git log`**: Ovo će listati sve prethodne komite i ključne informacije kao što su autor, datum commita, poruka commita i hash commita. Hash commita je duga niz koji se koristi kao jedinstveni identifikator commita.
- **`git remote add`**: Ova komanda se koristi za konfigurisanje udaljene veze sa vašom udaljenom bazom podataka.
- **`git push`**: Ova komanda se koristi za slanje vaših promena na udaljenu bazu podataka.
- **`git pull`**: Ova komanda se koristi za povlačenje najnovijih promena sa vaše udaljene baze podataka na vašu lokalnu bazu podataka.
- **`git clone`**: Ova komanda će izvršiti kloniranje baze podataka. To znači da ćete imati kopiju baze podataka. Ova kopija uključuje istoriju baze podataka, sve fajlove/foldere i komite na vašem lokalnom uređaju.
- **`git remote -v`**: Ova komanda će prikazati listu udaljenih repozitorija povezanih sa vašom lokalnom bazom podataka Git.
- **`git branch`**: Ova komanda će listati sve lokalne grane.
- **`git fetch upstream`**: Ova komanda govori Git da preuzme najnovije promene sa vašeg udaljenog izvora (koji je izvorna baza podataka).
- **`git merge upstream/main`**: Ova komanda govori Git da integriše najnovije promene iz grane `main` sa udaljenog izvora u vašu trenutnu granu.
- **`git reset`**: Ova komanda vam omogućava da resetujete trenutno stanje grane. Prolazak flag-a `--hard` govori Git da prisili lokalne fajlove da odgovaraju stanju grane. Ovo osigurava da počinjete sa čistim radnim stanjem.
- **`git rebase`**: Rebase u Git je način pomeranja ili spajanja niza commita sa jedne grane na drugu.

## Rad sa ograncima

- **Opis**: Grana u Git je zasebna radna oblast gde možete napraviti izmene. Grana `main` često pokazuje glavnu ili produkcionu granu u stvarnoj aplikaciji. Timovi programera kreiraju mnoge grane za nove funkcije i popravke grešaka, a zatim te izmene ponovo spajaju (merge) u granu `main`.
- **Kreiranje nove grane**: Da biste kreirali novu granu, možete pokrenuti sledeću komandu:

```sh
git branch feature
```

Da biste prešli na tu granu, možete izvršiti sledeću komandu:

```sh
git checkout feature
```

Mnogi razvojci će koristiti kratku komandu za kreiranje i prebacivanje na sledeću granu:

```sh
git checkout -b new-branch-name
```

Nova i alternativna komanda je komanda za `git switch`. Evo primera kreiranja i prebacivanja na novu granu:

```sh
git switch -c new-branch-name
```

- **Strategije grana**: Vaša grana `main` je vaša podrazumevana grana i obično je stabilna. Stoga je bolje kreirati nove grane odatle za stvari kao što su popravke grešaka, nove funkcije ili druge različite zadatke.
- **Konflikti spajanja**: Ovo se dešava kada Git pokušava da spoji promene sa različitih grana, ali ne zna koje promene zadržati. Ovo se obično dešava kada postoje sukobne promene u istom delu fajla.

## Status za datoteku koja se prati sa Git

- **"Neterminisan"**: To znači da je fajl nov u bazi podataka, i da Git nikada nije "video" ranije.
- **"Izmenjen"**: Ovaj fajl je postojao u prethodnoj distribuciji, i ima promene koje nisu uključene u vrednost distribucije.
- **"Ignorisano"**: Možda nećete videti ignorisane fajlove u Git, ali vaš IDE može imati indikator za njih. Ignorisani fajlovi nisu uključeni u operacije Git, obično zato što su navedeni u fajlu `.gitignore`.
- **"Izbrisan"**: Izbrisan fajl je suprotno od neterminisanog fajla – to je fajl koji je ranije postojao, a zatim je uklonjen.
- **"Promenjeno ime"**: Fajl sa promenjenim imenom je fajl čiji se sadržaj nije promenio, ali mu je promenjeno ime ili lokacija. U nekim slučajevima, fajl može biti smatran kao da je promenjeno ime čak i ako ima male promene.

## Fajlovi za `.gitignore`

- **Definicija**: Fajl `.gitignore` je poseban tip fajla povezan sa aktivnostima Git. Ime ukazuje da se ovaj fajl koristi za reći Git da prestane praćenje stvari, i to je uobičajena upotreba. Ali ono što se radi jeste reći Git da prestane praćenje fajlova.

## Rad sa bazama podataka

- **Definicija**: Baza podataka je kao kontejner projekta – ako radiš na aplikaciji, čuvaš sve fajlove te aplikacije zajedno u bazi podataka. Baze podataka mogu biti lokalne na tvom računaru, ili udaljene na uslugama kao GitHub.
- **Javna naspram Privatne baze podataka**: Javna baza podataka može biti pregledana i preuzeta od strane bilo koga. Privatna baza podataka dostupna je samo tebi, osim ako ne dozvoliš otvoreni pristup drugima.
- **Kreiranje baze podataka na GitHub**: Da biste kreirali novu bazu podataka na GitHub, možete kliknuti na dugme `"New Repository"` i pratiti vodič GitHub za konfigurisanje nove baze podataka.
- **Pogonjenje lokalne baze podataka na GitHub**: Ako imate lokalni projekat na računaru, možete "pogurati" tu bazu podataka ka GitHub. Evo pregleda korak po korak procesa:

1. Inicijalizujte prazan Git repozitorijum u direktorijumu projekta (`git init`).
2. Napravite promene na svom projektu.
3. Pokrenite komandu `git status` da vidite sve promene koje prati Git.
4. Pripremite svoje promene (`git add`).
5. Izvršite snimanje vaših promena (`git commit`).
6. Konfigurišite udaljeni repozitorijum (`git remote add`).
7. Potaknite svoje promene na GitHub (`git push`).

## Povlačenje promena

- **Pull Request**: Pull request je zahtev za spajanje promena sa vaše grane na odredišnu granu. Pull request je proces koji se koristi kada želite da doprinesete izmenama koda u projektu. Ovaj način omogućava menadžerima projekta da preglede vaše promene. Mogu ostaviti komentare, postaviti pitanja i predložiti revizije. Nakon što se proces pregleda završi, može biti odobren i spojen (merged) u glavnu granu.

## Doprineti drugim projektima

- **Proces**: Postoji hiljade projekata kojima možete doprineti. Ovo je osnovni proces kako da doprinesete drugoj bazi podataka:

1. Pročitajte dokumentaciju za doprinose
2. Pronađite dostupan problem koji možete da rešite
3. Napravite fork repozitorijuma/baze podataka
4. Kopirajte vorkovani repozitorijum/bazu podataka
5. Kreiraj novu granu (branch)
6. Izradite izmene u skladu sa problemom
7. Izradite PR (Pull Request)
8. Sačekajte pregled tog PR-a

## Rad sa SSH ključevima i GPG

- **GPG Ključevi**: GPG ključevi, ili Gnu Privacy Guard, obično se koriste za potpisivanje fajlova ili distribucija. Osoba može koristiti vaš javni GPG ključ da potvrdi da li je digitalni potpis fajla generisan vašim ključem i da sadržaj fajla nije izmenjen ili kompromitovan.

Da biste kreirali GPG ključ, moraćete da pokrenete:

```sh
gpg --full-generate-key
```

- **SSH ključevi**: SSH ključevi, ili Secure Shell, uobičajeno se koriste za autentifikaciju udaljenih konekcija na serveru - preko alata `ssh`. Takođe možete koristiti SSH ključeve za potpisivanje paketa.

Sa SSH ključem, pokovaćeš:

```sh
ssh-keygen -t ed25519 -C "your_email@example.com"
```

`ed25519` To je savremeni algoritam za potpisivanje javnih ključeva.

- **Potpisivanje distribucija sa GPG ključevima**: Da potpiš svoje distribucije sa svojim GPG ključem, moraćeš da učitaš svoj javni ključ, a ne privatni ključ, na svoj nalog GitHub. Da sprovedeš svoje javne ključeve, moraćeš da pokreneš sledeću komandu:

```sh
gpg --list-secret-keys --keyid-format=long
```

Zatim, dobijte javni ključ i koristite:

```sh
gpg --armor --export "<key id>"
```Zatim, uzmite kratki ID koji ste dobili prilikom listanja ključeva i pokrenite ovu komandu da ga postavite kao vaš ključ za potpisivanje git:

```sh
git config --global user.signingkey <your_gpg_key_id>
```

Zatim, možete proslediti flag `-S` sa svojom komandom `git commit` da potpišete određeni sertifikat - biće vam potrebno uneti lozinku. Inače, ako želite da potpišete svaki sertifikat direktno, možete postaviti konfiguraciju za automatski potpis na `true`:

```sh
git config --global commit.gpgsign true
```

- **Potpisivanje grana pomoću SSH ključeva**: Potpisivanje SSH ključem, što je nova funkcija u GitHub, zahtevaće da počnete sa učitavanjem ključeva na vaš nalog GitHub. Zatim ćete morati da podesite status potpisivanja za Git koristeći SSH:

```sh
git config --global gpg.format ssh
```

Zatim, prilikom postavljanja ključa za potpisivanje, navedićete putanju do fajla umesto ID-a:

```sh
git config --global user.signingkey <path_to_your_ssh_keys>
```
