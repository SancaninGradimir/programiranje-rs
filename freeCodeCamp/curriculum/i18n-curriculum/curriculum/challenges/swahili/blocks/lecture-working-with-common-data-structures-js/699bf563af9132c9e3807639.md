# --description--

Strukture podataka su osnovne strukture podataka u računarskom znanju. Sve strukture skladište kolekcije podataka organizovane po redosledu, ali u zavisnosti od njihovog tipa, mogu obavljati različite funkcije u pozadini.

Njihove osnovne karakteristike mogu imati značajan uticaj na efikasnost programa, zato naučimo o promenljivim i nepromenljivim nizovima podataka i njihovim razlikama, kako biste mogli da izaberete najefikasniji za svoj program.

Početi ćemo sa redovima nemenjivih podataka.

**Imutabilni nizovi podataka** imaju fiksnu veličinu. Skladište elemente u neprekidnim memorijskim lokacijama.

Veličina nepromenljivog niza određuje se kada se niz inicijalizuje. Kada se taj određeni deo memorije jednom rezerviše, ostaje stabilan i ne može se menjati dok program radi. Ovo je važna osobina nepromenljivih nizova.

Čuvanje elemenata u susednim memorijskim područjima čini proces dohvaćanja podataka efikasnijim jer program može sačuvati adresu prvog elementa, a zatim koristiti pokazivače da obavi jednostavne proračune i dohvatiti druge elemente u memoriji.

Zahvaljujući tome, pristup vrednostima nepromenljivog niza traje konstantno vreme `O(1)`, što je veoma efikasno.

Možete koristiti nepromenljiv niz kada znate unapred koliko će biti elemenata sačuvano. Takođe je korisno i kada se vrednosti primaju često, jer je operacija pristupa veoma efikasna.

Međutim, ova struktura podataka ne može ni rasteti ni smanjivati svoju veličinu, pa ako se broj elemenata koje je potrebno sačuvati može menjati, trebalo bi da koristite dinamički niz umesto toga.

Pokušaj povećanja veličine nepromenljivog niza zahteva kreiranje novog niza i kopiranje svih elemenata iz starog u novi, što je neefikasno. U tom slučaju, promenljivi niz će biti bolji jer obrađuje ovaj proces direktno.

JavaScript ne sadrži standardne nizove niskog nivoa kao što su C ili Java, ali možete simulirati niz fiksne veličine koristeći nešto unutar `Array`:

```js
let numbers = new Array(3); // fixed size of 3
numbers[0] = 10;
numbers[1] = 20;
numbers[2] = 30;
```

Iako JavaScript dozvoljava menjanje veličine, za ovaj koncept predstavlja red fiksne dužine.

Redovi u JavaScript su podrazumevano dinamični, pa hajde da ih pregledamo.

**Dinamički redovi podataka** su jednostavniji jer mogu automatski rasti ili smanjivati dok je program u funkciji.

Radi kroz direktan mehanizam skaliranja koji kopira elemente u novi red kada je originalni red pun. Ovaj proces se izvodi efikasno jer je veličina novog reda odabrana na učinkovit način, što smanjuje broj izvršavanja ovih računski zahtevnih operacija.

Pristup elementima dinamičkog niza ne zahteva konstantno vreme `O(1)`, stoga je ova operacija veoma neefikasna.

Umetanje elementa u sredinu redova zahteva liniju `O(n)` jer elementi koji se nalaze posle njega moraju biti pomereni.

Dodavanje elementa na kraj niza traje konstantno vreme `O(1)` ako još postoji prostor u promenljivom nizu, ali ako je niz popunjen i mora da se promeni njegova veličina, ova operacija ima složenost `O(n)`.

Trebalo bi da koristiš dinamičke nizove kada ne znaš unapred koliko vrednosti ćeš morati da sačuvaš u nizu. Takođe je korisno i kada stalno dodaješ i brišeš elemente.

Kreiranje reda u JavaScript, pišete elemente unutar uglastih zagrada, razdvojene zarezima:

```js
let numbers = [3, 4, 5, 6];
```

Ovo kreira dinamički niz sa četiri elementa.

Možete dobiti element u nizu tako što ćete napisati ime promene, praćeno uglastim zagradama sa primerom.

Indeksi počinju sa 0 za prvi element i povećavaju se za 1 za svaki sledeći element.

```js
numbers[0]; // 3
numbers[1]; // 4
numbers[2]; // 5
numbers[3]; // 6
```

Za ispravljanje vrednosti, samo je morate ponovo postaviti:

```js
numbers[2] = 16
```

Sada, red postaje `[3, 4, 16, 6]`.

Da biste dodali element na kraj reda, koristite metodu `.push()`:

```js
numbers.push(7)
```

Sada red postaje `[3, 4, 16, 6, 7]`:

Uključivanje elemenata u specifičan model, JavaScript nema `.insert()` kao Python, ali možete koristiti način `.splice()`. Sintaksa je sledeća:

```js
array.splice(index, deleteCount, newElement)
```

Evo primera:

```js
numbers.splice(3, 0, 15);
```

Ovaj primer znači:

- Počni na modelu 3
- Ukloni elemente 0
- Postavi na 15

Ako je red bio `[3, 4, 16, 6, 7]`, sada postaje `[3, 4, 16, 15, 6, 7]`.

Možete ukloniti funkciju u specifičnom modelu koristeći isti metod od `.splice`:

```js
numbers.splice(2, 1);
```

To znači početi sa modelu 2 i ukloniti element 1.

Da biste uklonili poslednji element, koristite `.pop()`:

```js
numbers.pop();
```

Postoje mnoge druge ugrađene metode redova koje možete pronaći u dokumentaciji i redovima na JavaScript.
