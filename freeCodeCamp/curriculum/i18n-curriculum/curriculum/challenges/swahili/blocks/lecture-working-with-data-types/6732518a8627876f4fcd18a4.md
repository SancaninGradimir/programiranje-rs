# --interactive--

Operator `typeof` u JavaScript je jednostavan, ali moćan alat koji vam omogućava da vidite tip podataka varijable ili vrednosti. Uvek vraća niz karaktera koji pokazuje taj tip.

Istražite nekoliko primera:

:::interactive_editor

```js
let num = 42;
console.log(typeof num); // "number"
```

:::

U ovom prvom primeru, kreirali smo varijablu nazvanu `num` i dodelili joj vrednost broja `42`. Kada koristite operator `typeof` na varijabli nazvanoj `num`, vraća niz karaktera `number`.

Evo još jednog primera korišćenja operatera `typeof` na varijabli nazvanoj `isUserLoggedIn`:

:::interactive_editor

```js
let isUserLoggedIn = true;
console.log(typeof isUserLoggedIn); // "boolean"
```

:::

Kada koristite operator `typeof` na varijabli `isUserLoggedIn`, vraća niz znakova `boolean` jer je toj varijabli dodata vrednost tačno ili netačno `true`.

Korišćenje operatera `typeof` može biti korisno posebno kada tražite greške ili pokušavate da razumete kakve podatke koristite u svom kodu.

Međutim, postoji veoma poznata greška u JavaScript kada je reč o pitanju `null`.

Pogledajmo primer:

:::interactive_editor

```js
let exampleVariable = null;
console.log(typeof exampleVariable); // "object"
```

:::

U ovom primeru, imamo varijablu nazvanu `exampleVariable` i dodelili smo joj vrednost `null`. Ali kada koristimo operator `typeof`, vraća niz karaktera `object`.

Ovo se smatra greškom u JavaScript, koja postoji od samog početka. Uzrok ovog ponašanja leži u tome kako JavaScript je prvobitno kreirano.

Kada je jezik prvi put implementiran, vrednosti kao što su `null` bile predstavljene kao specifičan tip objekta, i uzrokujući ove nepredvidive rezultate.

Nažalost, ovo je postalo deo jezika, i iako je zbunjujuće/konfuzno, to je nešto što ćete morati da uzmete u obzir.
