## Algoritmi u Big O notaciji

- **Algoritmi**: Skup nedvosmislenih instrukcija za rešavanje problema ili izvršavanje zadatka. Algoritam mora se sastojati od konačnog broja koraka, a svaki korak mora biti tačan i nedvosmislen.

- **Big O Notacija**: Opisuje najgori slučaj performanse, ili stopu rasta, algoritma kako veličina ulaza raste. Razmatra kako rast potrošnje resursa u odnosu na veličinu ulaza, zanemarujući konstante faktore i niže redove članova.

### Normalna složenost tokom vremena

- **O(1) - Konstantno vreme**: Algoritam zahteva isto vreme bez obzira na veličinu ulaza.

:::interactive_editor

```js
function checkEvenOrOdd(number) {
  if (number % 2 === 0) {
    return "Even";
  } else {
    return "Odd";
  }
}

console.log(checkEvenOrOdd(4)) // "Even"
console.log(checkEvenOrOdd(5)) // "Odd"
```

:::

- **O(log n) - Logaritamsko vreme**: Vreme raste polako kako se povećava ulaz. Često se javlja u algoritmima koji ponavljaju smanjenje veličine problema na pola (kao što je Binarna pretraga).

- **O(n) - Linearno vreme**: Vreme izvršavanja raste proporcionalno sa veličinom ulaza.

```js
for (const grade of grades) { // grades is an array
  console.log(grade);
}
```

- **O(n log n) - Logaritamska složenost vremena**: Prose vremenska složenost efikasnih algoritama sortiranja kao što su Merge Sort i Quick Sort.

- **O(n²) - Kvadratično vreme**: Vreme izvršavanja raste kvadratno. Često se javlja u ugnježdenim petljama.

:::interactive_editor

```js
const n = 3;

for (let i = 0; i < n; i++) {
  for (let j = 0; j < n; j++) {
    console.log("Hello, World!");
  }
}
```

:::

### Kompleksnost prostora

- **O(1) - Konstantni prostor**: Algoritam koristi konstantnu količinu memorije bez obzira na veličinu ulaza.
- **O(n) - Linearni prostor**: Potrošnja memorije raste proporcionalno veličini ulaza.
- **O(n²) - Kvadratni prostor**: Potrošnja memorije raste kvadratno u odnosu na veličinu ulaza.

## Metode za rešavanje problema

- **Razumevanje problema**: Pročitajte problematsku izjavu nekoliko puta. Identifikujte ulaz, očekivani rezultat i način transformacije ulaza u rezultate.

- **Pseudokod**: Opis logike algoritma na visokom nivou koja ne zavisi od bilo kog jezika. Koristi uobičajen jezik pomešan sa programskim strukturama kao što su `IF`, `ELSE`, `FOR`, `WHILE`.

```md
GET original_string
SET reversed_string = ""
FOR EACH character IN original_string:
  ADD character TO THE BEGINNING OF reversed_string
DISPLAY reversed_string
```

- **Ugaoni slučajevi**: Poseban, validan unos koji se dešava na granicama onoga što algoritam treba da obrađuje. Uvek razmatrajte i testirajte ugaone slučajeve.

## Redovi podataka

- **Trajno skladište podataka**: Ima veličinu postavljenu tokom inicijalizacije. Komponente se čuvaju u susednim memorijskim oblastima. Veličina ne može biti promenjena tokom izvršavanja programa.

- **Dinamički nizovi podataka**: Mogu rasti ili smanjiti se automatski tokom izvršavanja programa. One obrađuju promene veličine automatskim kopiranjem u veći niz kada je potrebno.

### Implementacija dinamičkih redova u JavaScript

:::interactive_editor

```js
let numbers = [3, 4, 5, 6];
// Access elements
console.log(numbers[0])  // 3

// Update elements
numbers[2] = 16
console.log([...numbers]);  // [3, 4, 16, 6]

// Add elements
numbers.push(7)
console.log([...numbers]);  // [3, 4, 16, 6, 7] 

// Add at a specific index
numbers.splice(3, 0, 15);
console.log([...numbers]);  // [3, 4, 16, 15, 6, 7]

// Remove elements
numbers.splice(2, 1);  // Remove at specific index
console.log([...numbers]);  // [3, 4, 15, 6, 7]

numbers.pop()   // Remove last element
console.log([...numbers]); // [3, 4, 15, 6]
```

:::

### Vremenska složenost za dinamičke nizove

- **Pristup**: O(1)
- **Dodavanje na kraj**: O(1) u proseku, O(n) tokom skaliranja veličine
- **Dodavanje/Umetanje u sredinu**: O(n)
- **Brisanje**: O(n) za sredinu, O(1) na kraju

## Ustani

- **Stack**: Struktura podataka tipa Last-In, First-Out (LIFO). Elementi se dodaju i uklanjaju samo sa vrha.

- **Operacija pusha**: Dodavanje elementa na vrh steka. Vremenska složenost: O(1).

- **Операција pop**: Uklanjanje elementa sa vrha steka. Vremenska složenost: O(1).

:::interactive_editor

```js
// Using JavaScript array as a stack

let stack = [];

// Push operations
stack.push(1);
stack.push(2);
stack.push(3);
console.log([...stack]);  // [1, 2, 3]

// Pop operations
let topElement = stack.pop();
console.log(topElement);  // 3
console.log([...stack]);  // [1, 2]
```

:::

## Foleni

- **Red**: Struktura podataka tipa First-In, First-Out (FIFO). Elementi se dodaju na zadnji kraj i uklanjaju sa prednjeg kraja.

- **Enqueue operacija**: Dodavanje elementa na zadnji deo redova. Vremenska složenost: O(1).

- **Dequeue operacija**: Uklanjanje elementa sa prednje strane redne liste. Vremenska složenost: O(1).

:::interactive_editor

```js
// Using JavaScript array as a queue
let queue = [];

// Enqueue operations
queue.push(1);
queue.push(2);
queue.push(3);
console.log([...queue]);  // [1, 2, 3]

// Dequeue operations
let firstElement = queue.shift();
console.log(firstElement);  // 1
console.log([...queue]);  // [2, 3]
```

:::

## Povezane liste

- **Povezana lista**: Linearna struktura podataka gde svaki čvor sadrži podatke i referencu ka sledećem čvoru. Čvorovi su povezani kao lanac.

### Samostalni povezani liste

- **Struktura**: Svaki čvor ima podatke i jednu referencu na sledeći čvor.
- **Traversiranje**: Može se kretati samo napred, od glave do repa.
- **Glavni čvor**: Prvi čvor u listi, obično je jedini čvor dostupan direktno.
- **Repni čvor**: Poslednji čvor u listi, pokazuje na `null`.

### Operacije sa vremenskom složenošću

- **Ubacivanje na početak**: O(1)
- **Ubacivanje na kraj**: O(n) - mora da prođe do kraja
- **Ubacivanje u sredinu**: O(n) - mora da prođe do pozicije
- **Brisanje na početak**: O(1)
- **Brisanje na kraj**: O(n) - mora da prođe kako bi pronašao prethodni čvor (node)
- **Brisanje u sredinu**: O(n) - mora da prođe kako bi pronašao čvor

### Dvostruke liste

- **Struktura**: Svaki čvor ima podatke i dve reference: sledeći čvor i prethodni čvor.
- **Traversiranje**: Može se kretati u oba pravca.
- **Memorija**: Zahteva više memorije nego samo spojene liste zbog dodatne reference.

## Kada koristiti svaku strukturu podataka

- **Niz podataka (dinamički niz)**: Kada vam je potreban sekvencijalni i indeksirani pristup, a ne znate veličinu unapred
- **Stak**: Za LIFO operacije (uklanjanje elemenata/pop, evaluacija izraza, poništavanje)
- **Red**: Za FIFO operacije (planiranje zadataka, pretraga u širinu/BFS)
- **Povezana lista**: Kada vam je potrebno često dodavanje/uklanjanje na početku, nepoznata veličina i nije potreban slučajni pristup
