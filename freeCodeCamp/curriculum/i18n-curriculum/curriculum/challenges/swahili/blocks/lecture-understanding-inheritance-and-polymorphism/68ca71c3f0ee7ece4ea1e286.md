# --description--

U početnom kursu, naučili ste o postavljanju prefiksa sa pojedinačnim i dvostrukim podznaknicima na svojstva elemenata.

Da bih vam przypomenuo razliku između njih, jedan donji crtačić (`_`) je konvencija koja znači da je atribut namijenjen za interni rad unutar klase i ne bi trebalo direktno pristupiti mu izvan te klase. Dva donja crtačića (`__`), s druge strane, sprečavaju da se taj atribut pristupi direktno izvan klase.

Ovde je primer koji pokazuje kako ove dve funkcionišu:

```py
class Example:
    def __init__(self):
        self._internal = 'I can be accessed from outside the class, but should not'
        self.__private = 'You cannot access me directly from outside the class'

obj = Example()

print(obj._internal) # I can be accessed from outside the class, but should not
print(obj.__private)  # AttributeError: 'Example' object has no attribute '__private'
```

Postavljanje prefiksa od dva donjevu crte na svojstvo elementa uzrokuje proces name mangling-a za Python, gde Python tajno menja ime tog svojstva dodavanjem donjevu crte i imena klase kao prefiks, menjajući `__attribute` u `_ClassName__attribute`.

Da biste videli ovo kako se dešava, kreirate primer klase objekata i koristite specifično svojstvo `__dict__` tog primera, koje je rečnik sa atributima tog objekta:

```py
class Example:
    def __init__(self, internal, private):
        self._internal = internal
        self.__private = private

example1 = Example(
    'I can be accessed from outside the class, but should not',
    'I cannot be accessed directly from outside the class'
)

print(example1.__dict__)
```

Rezultat će biti:

```py
{
  '_internal': 'I can be accessed from outside the class, but should not',
  '_Example__private': 'I cannot be accessed directly from outside the class'
}
```

Kao što vidite, atribut `__private` je sačuvan kao `_Example__private`. To znači da još uvek možete pristupiti tom atributu izvan klase na ovaj način:

```py
class Example:
    def __init__(self, internal, private):
        self._internal = internal
        self.__private = private

example1 = Example(
    'I can be accessed from outside the class, but should not',
    'I cannot be accessed directly from outside the class'
)
example2 = Example(
    'I should not be accessed from outside the class',
    'But I can be accessed from outside the class with name mangling'
)

print(example1._Example__private) # I cannot be accessed directly from outside the class
print(example2._Example__private) # But I can be accessed from outside the class with name mangling
```

Zašto pa Python vrši name mangling?

Glavna svrha name mangling-a je sprečavanje slučajne kolizije svojstava klase i metoda prilikom korišćenja nasleđivanja. Evo jasno razumljivog primera:

```py
class Parent:
    def __init__(self):
        self.__data = 'Parent data'

class Child(Parent):
    def __init__(self):
        super().__init__()
        self.__data = 'Child data'

c = Child()
print(c.__dict__) # {'_Parent__data': 'Parent data', '_Child__data': 'Child data'}
```

Može se videti da klasa `Parent` i `Child`, koja nasleđuje od nje, svaka ima svoja svojstva za `_class__data`. To je moguće uz name mangling. Inače, `Child` bi slučajno proširio podatke Roditelja.

Ovo bi se dogodilo ako Python ne dozvoljava name mangling, tj. ako ne koristi prefiks od dvostrukog donjeg crte na sve atribute obe klase:

```py
class Parent:
   def __init__(self):
       self.data = 'Parent data'

class Child(Parent):
   def __init__(self):
       super().__init__()
       self.data = 'Child data'

c = Child()
print(c.__dict__)  # {'data': 'Child data'}
```

Dakle, koji treba da koristiš za postavljanje prefiksa svojstva između jedne poddonje linije (underscore) (`_`) i dve poddonje linije (underscore) (`__`)? Zavisi. Ako je svojstvo samo za interno korišćenje unutar klase, koristi jednu poddonju liniju (underscore).

Ali ako radiš sa nasleđenom klasom, treba da koristiš dvostruke donje linije kako bi svojstvo od roditelja nije slučajno promenjeno.
