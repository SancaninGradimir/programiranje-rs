# --description--

React Server Components (RSCs) je novi model koji je promenio način na koji programeri za React rešavaju stvari. Sa RSC-ovima, više posla se prebacuje na server, što ima mnogo prednosti.

Istražimo šta su server komponente, kako funkcionišu, i šta je uzrokovalo njihovo uvođenje.

React Server Components su React komponente koje se renderuju isključivo na serveru, pri čemu se klijentu šalje samo konačni HTML. To znači da ove komponente mogu direktno da pristupaju resursima na serveru i značajno smanje količinu JavaScript-a koja se šalje pregledaču.

Aplikacije za React su obično koristile sistem "client component" koji obrađuje sve u standardnoj aplikaciji React, kao što je prikazivanje, interakcija i sporedni efekti. Termin "client component" nije bio uobičajen sve dok nisu uvedeni React server components nedavno.

Međutim, sistem za klijentske komponente ima nedostatke kao što su veliki paketi od JavaScript i duga inicijalna vremena učitavanja.

Sistemi React poput Next.js-a i Gatsbyja pronašli su metode za prebacivanje nekih procesa na server radi rešavanja tih problema, ali nijedan od njih nije imao zvanične standarde. Ako ste ikada koristili bilo koji od ovih sistema, možda ste čuli za `getServerSideProps` i `getServerData`.

Zatim su došli React Server Components, koji omogućavaju pokretanje nekih delova u potpunosti na serveru kako bi se mogle obavljati stvari kao što je dohvaćanje podataka i izračunavanje pre nego što bilo koji kod bude izvršen u pregledaču korisnika.

Komponente servera su prvo postale popularne i lako dostupne u Next.js-u. Druge sisteme kao što su Remix i Gatsby prate, a postoji eksperimentalni dodatak za Vite pod nazivom `vite-plugin-react-server` koji vam omogućava da gradite server komponente.

Kako rade komponente servera?

Jedan od najboljih načina za prikaz mogućnosti React Server Components jeste kroz dohvatanje podataka.

U client komponentama za React standardnih, omogućava pretraživaču da obrađuje zahteve od API. Pošto je pristup podacima sporedni efekat, poziva API u hook-u `useEffect`.

Takođe je dobra praksa postaviti status parametre kao što su učitavanje (loading), podaci (data) i greška (error) kako biste mogli da prikažete da se podaci učitavaju, da prikažete podatke kada su spremni ili da prikažete grešku u vašoj aplikaciji.

Za React Server Components, možete prebaciti ceo deo na server i dobiti podatke tamo bez korišćenja `useState` ili `useEffect`:

```jsx
const Users = async () => {
  const res = await fetch("https://jsonplaceholder.typicode.com/users");
  const users = await res.json();

  return (
    <>
      <h1 className="text-4xl text-center mt-6">Users</h1>
      <ul className="text-center mt-3">
        {users.map((user) => (
          <li key={user.id}>{user.name}</li>
        ))}
      </ul>
    </>
  );
};

export default Users;
```

Zato što se React Server Komponente pokreću samo na serveru, možete dobiti podatke iz API i prikazati ih odmah. Takođe, pošto se pristup podacima dešava na serveru, blizu izvora, vaša aplikacija može bolje da radi, posebno za korisnike sa sporom mrežnom vezom.

Jedna velika stvar na koju treba obratiti pažnju je da sav kod server komponenti ostaje na serveru i nije poslat u pretraživač. To znači da ne možete koristiti React sa njima, i da nemaju pristup Web API-jima ili slušaocima događaja pretraživača. Pa, kako možete dodati interaktivnost?

U Next.js ruteru, sve komponente su server components po defaultu. Ako želite dodati interaktivnost, morate postaviti tu sekciju kao client component koristeći komandu `"use client"`.

Pretpostavimo da želite da učinite prethodni primer klient komponentom. Evo kako to možete uraditi:

```jsx
"use client";

import { useState, useEffect } from "react";

const Users2 = () => {
  const [status, setStatus] = useState({
    users: [],
    loading: true,
    error: null,
  });

  async function fetchUsers() {
    try {
      const res = await fetch("https://jsonplaceholder.typicode.com/users");
      const data = await res.json();
      setStatus((prevStatus) => ({
        ...prevStatus,
        users: data,
        loading: false,
      }));
    } catch (err) {
      setStatus((prevStatus) => ({
        ...prevStatus,
        error: err.message,
        loading: false,
      }));
    }
  }

  useEffect(() => {
    fetchUsers();
  }, []);

  if (status.loading) {
    return <p>Loading Users...</p>;
  }
  if (status.error) {
    return <p>Error getting users: {status.error}</p>;
  }

  return (
    <>
      <h1 className="text-4xl text-center mt-6">Users</h1>
      <ul className="text-center mt-3">
        {status.users.map((user) => (
          <li key={user.id}>{user.name}</li>
        ))}
      </ul>
    </>
  );
};

export default Users2;
```

Ako želite dodati interakciju kao što su događaji klikova, taj deo takođe mora biti klijentska komponenta:

```jsx
"use client";

import { useState } from "react";

const Counter = () => {
  const [count, setCount] = useState(0);

  return (
    <>
      <h1>Counter</h1>
      <button onClick={() => setCount(count + 1)}>Increment</button>
      <h2>{count}</h2>
      <button onClick={() => setCount(count - 1)}>Decrement</button>
    </>
  );
};

export default Counter;
```

Ako ne navedete direktivu `use client` za taj deo, dobićete grešku sa porukom koja kaže: "Uvozite server komponentu koja zahteva `useState`. Ovaj hook od React radi samo u client component. Da biste to rešili, postavite fajl (ili njegov roditelj) sa direktivom `"use client"`."

Glavne prednosti koje dolaze sa React Server Components su što pristup podacima postaje lakši, kod postaje lakši za čitanje, a složenost na strani klijenta se smanjuje.
