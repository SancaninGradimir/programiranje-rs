## Uobičajene greške u Python

- **SyntaxError**: Greška koju Python izbacuje kada vaš kod ne prati njegove sintaksne pravila. Na primer, kod `print("Hello there"` će izazvati sintaksnu grešku sa porukom `SyntaxError: '(' was never closed`, jer kod nedostaje zatvarajuća zagrada.
- **NameError**: Python izbacuje `NameError` kada pokušavate da pristupite parametru ili funkciji koje niste definisali. Na primer, ako imate liniju `print(username)` u vašem kodu bez prethodno definisanog parametra `username`, dobićete NameError sa porukom `NameError: name 'username' is not defined`.
- **TypeError**: Ovo je greška koju Python baca kada izvršavate operacije na dve ili više nekompatibilnih tipova podataka. Na primer, ako pokušate da dodate niz karaktera na broj, dobićete grešku `TypeError: can only concatenate str (not "int") to str`.
- **IndexError**: Dobići ćete `IndexError` ako pokušate da pristupite indeksu koji ne postoji u listi ili bilo kojoj drugoj sekvenci kao što su tupupla i niz karaktera. Na primer, u nizu karaktera `Hello world`, indeks poslednjeg karaktera je `11`. Ako nastavite da pristupate karakterima na ovaj način, `greet = "hello world"; print(greet[12])`, dobićete grešku sa porukom `IndexError: string index out of range`.
- **AttributeError**: Python izbacuje ovu grešku kada pokušavate da koristite metodu ili atribut koji ne postoji u objektu tog tipa. Na primer, pozivanje `.append()` na niz karaktera kao što je `"hello".append("!")` će rezultovati greškom sa porukom `AttributeError: 'str' object has no attribute 'append'`.

## Efikasne metode za ispravljanje grešaka u Python

- **Korišćenjem funkcije `print`**: Unos informacija za `print()` na različitim mestima u vašem kodu tokom debagovanja pomaže vam da vidite vrednosti varijabli i kako teče vaš kod.
- **Korišćenjem ugrađenog Debuggera Python (`pdb`)**: Python pruža modul `pdb` za debagovanje. To je deo standardne biblioteke Python, pa je uvek dostupan za upotrebu. Za `pdb`, možete postaviti *breakpoint* na funkciju `set_trace()` kako biste mogli da prolazite kroz kod korak po korak i inspekujete varijable interaktivno.
- **Vežbanje korišćenja alata za debagovanje u IDE-u**: Integrisana razvojna okruženja (IDE) i uređivači koda kao što su Pycharm i VS Code pružaju alate za debagovanje sa *breakpoints*-ovima, izvršavanjem korak po korak, inspekcijom varijabli i drugim funkcijama za debagovanje.

## Rešavanje greške

- **`try...except`**: Ovo se koristi za izvršavanje dela koda koje može izazvati grešku. Deo koda `try` je deo gde očekujete da će greška nastupiti, dok deo `except` hvata specifičnu grešku i izvršava se ako ta greška nastupi. Evo primera:

  ```py
  try:
    print(22 / 0)
  except ZeroDivisionError:
    print('You can\'t divide by zero!')
    # You can't divide by zero!
  ```

Takođe možete povezati više delova `except` radi rukovanja većim brojem vrsta grešaka:

  ```py
  try:
    number = int(input('Enter a number: '))
    print(22 / number)
  except ZeroDivisionError:
    print('You cannot divide by zero!')
    # You cannot divide by zero! prints when you enter 0
  except ValueError:
    print('Please enter a valid number!')
    # Please enter a valid number! prints when you enter a string  
  ```

- **`else` na `finally`**: Ovi delovi dodaju `try...except`. Ako ne dođe do greške, deo `else` radi. Deo `finally` funkcioniše uvek bez obzira na to da li su se greške desile ili ne.

  ```py
  try:
    result = 100 / 4
  except ZeroDivisionError:
    print('You cannot divide by zero!') # This will not run
  else:
    print(f'Result is {result}') # Result is 25.0
  finally:
    print('Execution complete!') # Execution complete!
  ```

- **Objekat izuzetka (Exception Object)**: Ovo vam omogućava da uhvatite sam izuzetak radi efikasnijeg rukovanja greškama i direktnog ispisivanja poruke o izuzetku. Da biste dobili objekat izuzetka, morate koristiti ključnu reč `as`. Evo primera:

  ```py
  try:
      value = int('This will raise an error')
  except ValueError as e:
      print(f'Caught an error: {e}')
      # Caught an error: invalid literal for int() with base 10: 'This will raise an error'
  ```

- **Izjava za `raise`**: Ovo vam omogućava da ručno bacite grešku. Možete ga koristiti za bacanje greške kada je određeni uslov ispunjen. Evo primera:

  ```py
  def divide(a, b):
      if b == 0:
          raise ZeroDivisionError('You cannot divide by zero')
      return a / b
  ```## Indikacija greške

Izjava `raise` je takođe važna kada kreirate svoje prilagođene greške, jer ga možete koristiti za bacanje greške sa specifičnom porukom. Evo primera:

  ```py
  class InvalidCredentialsError(Exception):
      def __init__(self, message="Invalid username or password"):
          self.message = message
          super().__init__(self.message)

  def login(username, password):
      stored_username = "admin"
      stored_password = "password123"
      
      if username != stored_username or password != stored_password:
          raise InvalidCredentialsError()
      
      return f"Welcome, {username}!"
  ```  

Evo kako možete koristiti funkciju `login` zajedno sa greškom `InvalidCredentialsError`:

  ```py
  # failed login attempt
  try:
      message = login("user", "wrongpassword")
  except InvalidCredentialsError as e:
      print(f"Login failed: {e}")
  else:
      print(message)

  # successful login attempt
  try:
      message = login("admin", "password123")
  except InvalidCredentialsError as e:
      # This block is not executed because the login was successful
      print(f"Login failed: {e}")
  else:
      # The else block runs if the 'try' block completes without an exception
      print(message)
  ```

Izraz `raise` takođe se može koristiti sa glavnim terminom `from` za povezivanje grešaka, pokazujući odnos između različitih grešaka:

  ```py
  def parse_config(filename):
    try:
        with open(filename, 'r') as file:
            data = file.read()
            return int(data)
    except FileNotFoundError:
        raise ValueError('Configuration file is missing') from None
    except ValueError as e:
        raise ValueError('Invalid configuration format') from e

  config = parse_config('config.txt')
  ```
