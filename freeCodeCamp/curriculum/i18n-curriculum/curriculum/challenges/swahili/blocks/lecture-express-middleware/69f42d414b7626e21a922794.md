# --description--

Middleware za obradu grešaka u Express je poseban tip međusobnog softvera koji se koristi za rukovanje greškama koje nastaju tokom ciklusa zahtev-odgovor. Važno je za detekciju nepredviđenih grešaka, njihovo logovanje i pružanje odgovarajućih odgovora klijentu.

Kao što ste videli u prethodnoj lekciji o obradi grešaka, middleware za obradu grešaka za Express ima četiri parametra: `err`, `req`, `res`, i `next`. Ovaj middleware se inicijalizuje kada se greška baci u funkciju `next()`.

Evo ponovnog podsećanja na sintaksu:

```js
app.use((err, req, res, next) => {
  console.error(err.message);
  res.status(500).send("Internal Server Error");
});
```

Ovo se prikazuje u jednostavnom programu za Express:

```js
const express = require('express')
const app = express()

// A simple route that throws an error
app.get('/', (req, res) => {
  throw new Error('Something went wrong!')
})

// Error-handling middleware
app.use((err, req, res, next) => {
  console.error(err.stack); // Logs the error stack
  res.status(500).send('Something went wrong!')
})

app.listen(3000, () => {
  console.log('Server running on http://localhost:3000')
})
```

* Data path handler: Data path handler namerno baca grešku kako bi simulirao problem.
* Error handling middleware: Ovaj middleware je registrovan preko `app.use()`. Hvata greške koje se bacaju putem `next(err)` i šalje odgovor HTTP 500 klijentu sa opštom porukom o grešci.
* `err`: Ovo je objekat greške koji se prosleđuje middleware-u.

Middleware za rukovanje greškama osigurava da korisnici i dalje dobijaju ispravan odgovor kada nešto pukne u vašoj aplikaciji. Važno je uvek uključiti ovaj middleware na kraju svog steka (ili sloja) middleware-a kako bi uhvatio bilo kakve greške.
