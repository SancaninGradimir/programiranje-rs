# --interactive--

Lančanozivanje metoda je tehnika u kojoj pozivate više metoda sekvencijalno, jedna za drugom. Možete koristiti lančanozivanje metoda za mnogo tipova vrednosti u JavaScript, uključujući nizove znakova (stringove), nizove podataka i objekte. Iako su nizovi znakova osnovne vrednosti, JavaScript ih privremeno stavlja u objekat od tipa `String` kada koristite metodu za niz znakova.

Pogledaćemo primer koristeći različite metode sekvenciranja karaktera:

:::interactive_editor

```js
const result = "  Hello, World!  "
  .trim()
  .toLowerCase()
  .replace("world", "JavaScript");

console.log(result); // "hello, JavaScript!"
```

:::

U ovom primeru počinjemo sa niskom znakova i izvršavamo tri operacije uzastopno: uklanjamo suvišne razmake, pretvaramo tekst u mala slova i zamenjujemo nisku `world` sa `JavaScript`. Svaka metoda vraća novu nisku, koja postaje ulaz za naredni poziv metode.

Serijska konekcija puteva može značajno poboljšati čitanje koda kada radi sa kompleksnim operacijama.

Na primer, razmotri sledeći primer koji koristi metode `filter`, `map` i `reduce`:

:::interactive_editor

```js
const transactions = [
  { amount: 100, type: "credit" },
  { amount: 20, type: "cash" },
  { amount: 150, type: "credit" },
  { amount: 50, type: "cash" },
  { amount: 75, type: "credit" }
];

const totalCreditWithBonus = transactions
  .filter((transaction) => transaction.type === "credit")
  .map((transaction) => transaction.amount * 1.1)
  .reduce((sum, amount) => sum + amount, 0);

console.log(totalCreditWithBonus); // 357.5
```

:::

U ovom primeru imamo niz transakcija gde svaka stavka sadrži iznos i tip plaćanja – kreditnu karticu ili gotovinu.

Prvo filtriramo transakcije i pravimo novi niz koji sadrži samo transakcije izvršene kreditnom karticom. Zatim nad rezultatom primenjujemo metodu `map`, pri čemu svaki iznos množimo sa `1.1`, što predstavlja bonus od `10%`.

Zatim rezultat prosleđujemo metodi `reduce`, koja sabira sve iznose i dobija vrednost `357.5`.

Iako ulančavanje metoda može učiniti kod kraćim i lakšim za čitanje, važno je da ga koristiš promišljeno.

Vrlo duga sekvenca može biti teška za popravljanje grešaka, jer nije odmah jasno koji korak u sekvenci može uzrokovati problem. Često je dobra metoda podeliti veoma dugu sekvencu na više manjih koraka radi bolje jasnoće i lakšeg popravljanja grešaka.

Možeš ulančavati metode i nad objektom. U tom slučaju svaka metoda vraća `this`, što se odnosi na trenutni objekat i omogućava da sledeća metoda u lancu nastavi rad nad njim.

:::interactive_editor

```js
const calculator = {
  total: 0,
  add(n) {
    this.total += n;
    return this;
  },
  multiply(n) {
    this.total *= n;
    return this;
  },
  subtract(n) {
    this.total -= n;
    return this;
  },
  getResult() {
    return this.total;
  }
};

const result = calculator.add(5).multiply(2).subtract(3).getResult();
console.log(result); // 7
```

:::

Na ovaj način, svaka putanja vraća isto, tako da možete nastaviti povezivati pozive jedan za drugim.
