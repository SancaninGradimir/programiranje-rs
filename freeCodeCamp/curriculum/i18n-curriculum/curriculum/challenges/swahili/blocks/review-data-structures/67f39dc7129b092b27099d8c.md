## Algoritmi u Big O notaciji

- **Algoritmi**: Skup nedvosmislenih instrukcija za rešavanje problema ili izvršavanje zadatka. Algoritam mora se sastojati od konačnog broja koraka, a svaki korak mora biti tačan i nedvosmislen.

- **Big O Notacija**: Opisuje najgori slučaj performanse, ili stopu rasta, algoritma kako raste veličina ulaza. Razmatra kako rast potrošnje resursa sa povećanjem veličine ulaza, izbegavajući uzimanje u obzir konstantnih faktora i parametara nižeg nivoa.

### Normalna složenost tokom vremena

- **O(1) - Konstantno vreme**: Algoritam zahteva isto vreme bez obzira na veličinu ulaza.

```python
def check_even_or_odd(number):
    if number % 2 == 0:
        return 'Even'
    else:
        return 'Odd'
```

- **O(log n) - Logaritamsko vreme**: Vreme raste polako kako ulaz raste. Često se koristi u algoritmima koji smanjuju veličinu problema po delovima (kao Binarna pretraga).

- **O(n) - Linearno vreme**: Vreme izvršavanja raste proporcionalno sa veličinom ulaza.

```python
for grade in grades:
    print(grade)
```

- **O(n log n) - Logaritamska složenost vremena**: Prose vremenska složenost efikasnih algoritama sortiranja kao što su Merge Sort i Quick Sort.

- **O(n²) - Kvadratična složenost**: Vreme izvršavanja raste kvadratno. Često se javlja u ugnježdene petlje.

```python
for i in range(n):
    for j in range(n):
        print("Hello, World!")
```

### Kompleksnost prostora

- **O(1) - Konstantan prostor**: Algoritam koristi konstantnu količinu memorije bez obzira na veličinu ulaza.
- **O(n) - Linearan prostor**: Potrošnja memorije raste proporcionalno sa veličinom ulaza.
- **O(n²) - Kvadratan prostor**: Potrošnja memorije raste kvadratno sa veličinom ulaza.

## Metode za rešavanje problema

- **Razumevanje problema**: Pročitajte problematsku izjavu nekoliko puta. Identifikujte ulaz, očekivani rezultat i način transformacije ulaza u rezultate.

- **Pseudokod**: Opis logike algoritma na visokom nivou koja ne zavisi od bilo kog jezika. Koristi uobičajen jezik pomešan sa programskim strukturama kao što su `IF`, `ELSE`, `FOR`, `WHILE`.

```md
GET original_string
SET reversed_string = ""
FOR EACH character IN original_string:
  ADD character TO THE BEGINNING OF reversed_string
DISPLAY reversed_string
```

- **Ugaoni slučajevi**: Poseban, validan unos koji se dešava na granicama onoga što algoritam treba da obrađuje. Uvek razmatrajte i testirajte ugaone slučajeve.

## Redovi podataka

- **Slojovi trajnih podataka**: Veličina je postavljena tokom inicijalizacije. Elementi se skladište u susednim memorijskim oblastima. Veličina ne može biti promenjena tokom izvršavanja programa.

- **Dinamički nizovi podataka**: Mogu rasti ili smanjiti se automatski tokom izvršavanja programa. One obrađuju promene veličine automatskim kopiranjem u veći niz kada je potrebno.

### Lista za Python (redovi sa dinamičkim podacima)

```python
numbers = [3, 4, 5, 6]

# Access elements
numbers[0]  # 3

# Update elements
numbers[2] = 16

# Add elements
numbers.append(7)
numbers.insert(3, 15)  # Insert at specific index

# Remove elements
numbers.pop(2)  # Remove at specific index
numbers.pop()   # Remove last element
```

### Vremenska složenost za dinamične podatke

- **Pristup**: O(1)
- **Dodavanje na kraj**: O(1) u proseku, O(n) tokom skaliranja veličine
- **Dodavanje/Umetanje u sredinu**: O(n)
- **Brisanje**: O(n) za sredinu, O(1) na kraju

## Ustani

- **Stack**: Struktura podataka tipa Last-In, First-Out (LIFO). Elementi se dodaju i uklanjaju samo sa vrha.

- **Operacija pusha (Push)**: Dodavanje elementa na vrh steka. Vremenska složenost: O(1).

- **Korak za uklanjanje (Pop)**: Uklanjanje elementa sa vrha steka. Vremenska složenost: O(1).

```python
# Using Python list as stack
stack = []

# Push operations
stack.append(1)
stack.append(2)
stack.append(3)

# Pop operations
top_element = stack.pop()  # Returns 3
```

## Foleni

- **Red**: Struktura podataka tipa First-In, First-Out (FIFO). Elementi se dodaju na zadnji kraj i uklanjaju sa prednjeg kraja.

- **Korak ubacivanja (Enqueue)**: Dodavanje elementa na zadnji deo reda. Vremenska složenost: O(1).

- **Operacija uklanjanja (Dequeue)**: Uklanja element sa prednje strane reda. Vremenska složenost: O(1).

```python
from collections import deque

# Using deque for efficient queue operations
queue = deque()

# Enqueue operations
queue.append(1)
queue.append(2)
queue.append(3)

# Dequeue operations
first_element = queue.popleft()  # Returns 1
```

## Povezane liste

- **Povezana lista**: Linearna struktura podataka gde svaki čvor sadrži podatke i referencu ka sledećem čvoru. Čvorovi su povezani kao lanac.

### Samostalni povezani liste

- **Struktura**: Svaki čvor ima podatke i referencu na sledeći čvor.
- **Traversal**: Može se kretati samo napred od glave do repa.
- **Glavni čvor**: Prvi čvor u listi, obično je jedini čvor koji se može direktno dohvatiti.
- **Repni čvor**: Poslednji čvor u listi, prikazuje `None`.

### Etapi i vremenski izazovi

- **Umetanje na početku**: O(1)
- **Umetanje na kraju**: O(n) - Mora se proći do kraja
- **Umetanje na sredini**: O(n) - Mora se proći do pozicije
- **Brisanje na početku**: O(1)
- **Brisanje na kraju**: O(n) - Mora se proći da bi se dobio prethodni čvor
- **Brisanje na sredini**: O(n) - Mora se proći da bi se dobio čvor

### Dvostruke liste

- **Struktura**: Svaki čvor ima dva podatka i dve reference: sledeći čvor i prethodni čvor.
- **Traversiranje**: Može se kretati u oba pravca.
- **Memorija**: Zahteva više memorije nego samo povezane liste zbog dodatne reference.

## Mapovi za hash na skupu

### Karte i karte za hash

- **Mapa (Apstraktni tip podataka)**: Upravlja kolekcijama parova ključ-vrednost. Svaki ključ mora biti jedinstven, ali vrednosti mogu biti ponovljene.

- **Hash mapa**: Stvarni implementacija mape ADT korišćenjem metode heširanja. Koristi funkciju heširanja da kreira vrednosti heša za ključeve, koje određuju mesto skladištenja u osnovnom nizu.

### Rečnici za Python (Mapa za hash)

```python
# Creating dictionaries
my_dictionary = {
    "A": 1,
    "B": 2, 
    "C": 3
}

# Alternative creation
my_dictionary = dict(A=1, B=2, C=3)

# Access and modify
value = my_dictionary["A"]  # 1
my_dictionary["A"] = 4      # Update value
del my_dictionary["A"]      # Remove key-value pair

# Check membership
"C" in my_dictionary

# Get keys, values, items
my_dictionary.keys()
my_dictionary.values()
my_dictionary.items()
```

### Kompleksnost vremena za hash mape

- **Prosečan slučaj**: O(1) za ubacivanje, dohvaćanje, brisanje
- **Najgori slučaj**: O(n) kada postoji mnogo kolizija hasha

### Skup

- **Skup (Set)**: Neuređene kolekcije jedinstvenih elemenata. Bez duplikata, ne čuva specifičan redosled.

- **Samo nepromenljivi elementi**: Skupovi mogu sadržati samo nepromenljive tipove podataka (brojevi, niz znakova, tuple) jer moraju ostati fiksne vrednosti haša.

```python
# Creating sets
numbers = {1, 2, 3, 4}
empty_set = set()  # Must use set(), not {}

# Add and remove elements
numbers.add(5)
numbers.remove(4)      # Raises KeyError if not found
numbers.discard(4)     # No error if not found

# Set operations
set_a = {1, 2, 3, 4}
set_b = {2, 3, 4, 5, 6}

# Union, intersection, difference, symmetric difference
set_a.union(set_b)                    # or set_a | set_b
set_a.intersection(set_b)             # or set_a & set_b
set_a.difference(set_b)               # or set_a - set_b
set_a.symmetric_difference(set_b)     # or set_a ^ set_b

# Subset and superset checks
set_a.issubset(set_b)
set_a.issuperset(set_b)
set_a.isdisjoint(set_b)

# Membership testing
5 in numbers
```

### Vremenska složenost za skup

- **Prosečan slučaj**: O(1) za dodavanje, brisanje, proveru članstva
- **Najgori slučaj**: O(n) usled kolizija haša

## Kolizije haša

- **Kolizija hasha**: Dešava se kada dva različita ključa generišu istu vrednost hasha.

- **Metode za rešavanje kolizija**:
  - **Lančano adresiranje (Chaining)**: Naredni prikaz pokazuje spojenu listu koja čuva sve elemente sa istom vrednošću hasha.
  - **Otvoreno adresiranje (Open Addressing)**: Pronađi sledeći dostupan slot koristeći prethodno postavljenu sekvencu/probe.

## Kada koristiti svaku strukturu podataka

- **Lista**: Kada vam je potreban usmeren pristup, sa modelom i kada ne znate veličinu unapred
- **Stak**: Za LIFO operacije (ponovna akcija/otkazivanje, evaluacija izraza, povratak)
- **Red**: Za FIFO operacije (planiranje zadataka, pretraga po širini)
- **Povezana lista**: Kada vam je potrebno često dodavanje/uklanjanje na početku, nepoznata veličina, nije potreban nasumičan pristup
- **Hash mape**: Za brzi pristup parovima ključ-vrednost, brojanje događaja, privremeno skladištenje
- **Set**: Za proveru jedinstvenosti, matematičke operacije na skupovima, uklanjanje duplikata
