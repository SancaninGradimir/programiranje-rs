# --interactive--

Ranije ste naučili kako semantičko označavanje i atributi ARIA čine sadržaj pristupačnim. U stvarnoj aplikaciji, sadržaj često nije statičan, i stranice se stalno ažuriraju putem promena preko JavaScript. Kada se ovo dogodi, važno je osigurati da su ove promene prikazane i u HTML. Ovo omogućava čitačima ekrana da pruže tačne informacije o ažuriranim stanjima korisnicima. Bez ovoga, osobe koje se oslanjaju na asistivne alate možda neće znati da se nešto promenilo, ili mogu primiti stare ili netačne informacije.

Istražimo uobičajen slučaj gde JavaScript menja stanje elementa, i kako možete postaviti vaš HTML da bude pristupačan ažuriranjem atributa ARIA.

Zamisli da imaš dugme koje otvara i zatvara padajući meni:

:::interactive_editor

```html
<button id="menuButton" aria-expanded="false" aria-controls="menuList">
  Menu
</button>

<ul id="menuList" hidden>
  <li>List item 1</li>
  <li>List item 2</li>
</ul>
<script src="index.js"></script>
```

```js
const button = document.getElementById("menuButton");
const menu = document.getElementById("menuList");

button.addEventListener("click", () => {
  const expanded = button.getAttribute("aria-expanded") === "true";
  button.setAttribute("aria-expanded", String(!expanded));
  menu.hidden = expanded;
});
```

:::

HTML kreira dugme koje kontroliše padajući meni. Dugme koristi `aria-expanded` da prikaže da li je meni otvoren, i `aria-controls` za povezivanje sa menijem referenciranjem `id` menija.

JavaScript čeka klik na dugme za meni. Kada se klikne, menja vrednost `aria-expanded` i ažurira atribut `hidden` u meniju. Ovo održava da su vidljivost menija i informacije o pristupačnosti usklađene, tako da korisnici sa vizuelnim sposobnostima i korisnici čitača ekrana znaju da li je meni otvoren ili zatvoren.

O ovim ARIA atributima ćeš učiti više u narednim lekcijama. Za sada zapamti da, kada JavaScript promeni stanje elementa, treba ažurirati i odgovarajuće ARIA atribute kako bi pomoćne tehnologije ispravno prikazivale stanje na stranici.
