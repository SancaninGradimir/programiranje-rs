# --description--

Primer zahtev-odgovor za HTTP je osnovni komunikacioni model koji pokreće World Wide Web (WWW). Ima mnogo delova koji ga čine efikasnim, tako da svi možemo dobiti opremu iz mreže.

Istražimo šta je primer zahtev-odgovor za HTTP, delove koje ga čine, i kako ti delovi sarađuju da postignu efikasnost u tome što pokreće mrežu.

Primer zahtev-odgovor za HTTP je komunikacioni model u mrežnoj interakciji gde klijent, obično veb pregledač, šalje **zahtev od HTTP** serveru, a server obrađuje taj zahtev i vraća **odgovor za HTTP**.

Ovo su delovi koji čine primer zahteva i odgovora za HTTP:

* **Klijent**: Pretraživač, mobilna aplikacija ili bilo koja druga aplikacija koja šalje zahteve HTTP na server i interpretira primljene odgovore HTTP.
    
* **Zahtev**: Poruka koju klijent šalje serveru. Sadrži metodu, URL, zaglavlja i opcionalno telo.
    
* **Metoda HTTP**: Akcija izvršavanja — `GET` za dobijanje podataka, `POST` za slanje podataka, `PATCH` i `PUT` za modifikovanje podataka, i `DELETE` za brisanje podataka.
    
* **URL (Uniform Resource Locator)**: Resurs tražen od servera.
    
* **Zaglavlja**: Metadata koja se šalje uz zahtev ili odgovor. Primer, `Content-Type` i `Authorization`.
    
* **Telo**: Podaci koji se šalju uz zahteve `POST`, `PUT`, `PATCH`, `DELETE`, često u formatu JSON ili podatke forme.
    
* **Server**: Backend sistem koji obrađuje zahtev i generiše odgovor. Može biti napisan na bilo kom programskom jeziku koji podržava server-side programiranje.
    
* **Odgovor**: Poruka koja se vraća sa servera. Sadrži kod stanja, zaglavlja i opcionalno telo.
    
* **Kod statusa ili kod odgovora**: Numerički broj koji ukazuje na uspeh, grešku ili preusmeravanje.
    
* **Telo odgovora**: Stvarni podaci koji se vraćaju, obično u formatu HTML, JSON ili XML.

Dakle, kako sve ove komponente rade zajedno u ciklusu zahtev-odgovor za HTTP?

1. **Klijent Pokreće Zahtev**
    
    * Pretraživač, mobilna aplikacija ili klijent za API šalje zahtev HTTP namenjen dostizanju servera.
        
    * Zahtev uključuje metode kao `GET`, `POST`, `PUT`, `PATCH`, `DELETE`, URL, zaglavlja i ponekad telo.
        
2. **Zahtev Se Šalje Na Server**
    
    * Zahtev putuje preko mreže ka serveru, backend sistemu napisanom na programskom jeziku.
        
    * Server prima zahtev i vraća važne informacije sa njega.
        
3. **Server Obrađuje Zahtev**
    
    * Server proverava putanju zahteva i URL.
        
    * Može biti potrebno da se potvrdi autentikacija identiteta (preko zaglavlja kao što je autorizacija/authorization).
        
    * Ako je potrebno, on interagiruje sa bazom podataka kako bi dobio ili modifikovao podatke.
        
4. **Server Priprema Odgovor**
    
    * Server kreira odgovor sa kodom stanja kao što su 200, 404, 500 i slično.
        
    * Odgovor uključuje zaglavlja, na primer `Content-Type: application/json`.
        
    * Ako je primenljivo, dodaje se telo odgovora u JSON, HTML ili XML.
        
5. **Odgovor Se Šalje Klijentu**
    
    * Server šalje odgovor nazad preko mreže.
        
    * Klijent prima odgovor i obrađuje podatke.
        
6. **Klijent Obrađuje Odgovor**
    
    * Ako je odgovor uspešan (status kod 200), klijent prikazuje podatke, na primer prikazuje podatke na veb stranici.
        
    * Ako postoji greška (`404 Not Found`, `500 Internal Server Error` i slično), klijent može prikazati poruku o grešci.
