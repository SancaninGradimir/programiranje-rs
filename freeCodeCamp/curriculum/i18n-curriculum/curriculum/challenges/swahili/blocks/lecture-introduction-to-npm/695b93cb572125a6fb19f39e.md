# --description--

Kada instalirate zavisnosti koristeći npm, posebno u postojećem projektu, npm proverava vašu datoteku ``package.json`` kako bi videla koji su paketi potrebni vašem projektu.

Svaka zavisnost u datoteci `package.json` ima broj verzije poput ovog:

```bash
"lodash": "^4.17.21"
```

Kao što ste ranije naučili, **caret znak (^)** omogućava npm-u da postavi novu manje verziju i ispravke kao `4.18.0` ili `4.17.22`. Ova mogućnost je korisna, ali takođe može izazvati probleme.

Ako dva programera pokrenu `npm install` u različito vreme, npm može instalirati malo različite verzije istog paketa. Čak i male razlike mogu izazvati greške ili neočekivano ponašanje.

Datoteka `package-lock.json` rešava ovaj problem. Ona se automatski kreira prilikom prvog pokretanja `npm install` i zaključava tačne verzije svake instalirane zavisnosti — uključujući i sve njihove podzavisnosti.

Ovo osigurava da svi koji rade na projektu uvek dobiju iste verzije paketa.

Evo jednostavne `package.json` datoteke sa paketom `lodash` kao zavisnošću:

```bash
{
  "name": "package-lock-json",
  "version": "1.0.0",
  "description": "",
  "main": "index.js",
  "scripts": {
    "test": "echo \"Error: no test specified\" && exit 1"
  },
  "keywords": [],
  "author": "",
  "license": "ISC",
  "type": "commonjs",
  "dependencies": {
    "lodash": "^4.17.21"
  }
}
```

Ovde je odgovarajući deo iz `package-lock.json`:

```bash
"node_modules/lodash": {
  "version": "4.17.21",
  "resolved": "https://registry.npmjs.org/lodash/-/lodash-4.17.21.tgz",
  "integrity": "sha512-v2kDEe57..."
}
```

Angalia tofauti:

* U datoteci `package.json`, `lodash` je naveden kao `^4.17.21` — fleksibilno pravilo verzionisanja.
    
* Katika `package-lock.json`, `lodash` imefungwa kwa toleo `4.17.21`.
    

Haijalishi mtu anapokimbia `npm install`, atapata toleo hilo halisi.

Na ovaj način, bez obzira gde neko trči `npm install`, dobiće istu verziju Lodasha svaki put.

Zapamtite da ovo takođe odnosi na zavisnosti zavisnosti, odnosno bilo kakve zavisnosti od zavisnosti koje koristite za svoj projekat.

Takođe primeti da, iako `dependencies` u `node_modules/express` koriste caret (`^`) za opsege verzija, svaka pojedinačna podzavisnost, kao što je `node_modules/etags`, zaključana je na određenu verziju.

Ukratko, fajl `package-lock.json` osigurava da svi rade sa istim informacijama i obezbeđuje konzistentnost između mašina za programere, CI servera, distribucije softvera i drugih okruženja.

Važno je uvek da umetnete svoj fajl `package-lock.json` u Git kako bi tima delila istu konfiguraciju. Takođe, nemoj ručno menjati fajl `package-lock.json`—npm će ga automatski ažurirati svaki put kada se zavisnost promeni.
