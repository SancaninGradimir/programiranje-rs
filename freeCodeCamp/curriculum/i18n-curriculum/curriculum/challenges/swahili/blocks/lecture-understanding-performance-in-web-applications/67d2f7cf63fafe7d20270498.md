# --description--

Iako su alati poput Lighthouse-a i Chrome DevTools-a dobri, ponekad vam je potreban specifičan uvid kreiran za vaš sajt.

Evo gde dolaze Performance Web APIs!

U ovom tutorijalu, objasnoći ćemo kako da se koriste tri važna Web API-ja za merenje i analizu brzine vaše veb stranice.

Ali prvo, Performance Web APIs, šta su to?

Performance Web APIs omogućavaju programerima da prate efikasnost učitavanja i odgovaranja za veb stranice, direktno iz koda. Ove API omogućavaju merenje vremena učitavanja stranice, praćenje kašnjenja pri prikazivanju i interakciji, i analizu vremena izvršavanja za JavaScript.

Sa ovim API-jima, možete izgraditi svoje alate za monitorovanje performansi bez zavisnosti od tuđih programa!

Istražite tri moćna Web API-ja koja možete koristiti danas.

Prvo, `performance.now()`.

API Ovo vam daje vremenske mere sa visokom preciznošću (u milisekundama) da biste izmerili koliko vremena je potrebno za učitavanje različitih delova vaše veb stranice.

Recimo da želite da testirate kako funkcija radi brzo:

```js
const start = performance.now();  
// Run some code here  
const end = performance.now();  

console.log(`Execution time: ${end - start}ms`);
```

Ovo je preciznije od korišćenja `Date.now()` jer meri vreme u milisekundama i sa mikrosekundnom preciznošću, izbegavajući probleme sa promenom satova. Možete ga koristiti za praćenje vremena izvršavanja skripte, kašnjenja odgovora na događaje i performansi crtanja.

Sledeće, Merilo performansi API.

API ovo vam daje sažetak svakog koraka učitavanja stranice, od DNS upita do `DOMContentLoaded`.

Želiš li da izmeriš koliko vremena tvoja stranica traje za potpuno učitavanje?

```js
let [navigationTiming] = performance.getEntriesByType("navigation");

if (navigationTiming instanceof PerformanceNavigationTiming) {
  // Calculate time from navigation start to DOM content loaded
  const pageLoadTime =
    navigationTiming.domContentLoadedEventEnd - navigationTiming.startTime;

  console.log("DOM Content Loaded Time:", pageLoadTime, "ms");
}
```

Ključne metrike koje možete pratiti sa API su: vreme pretrage DNS-a – ili rečima, brzina konekcije, Vreme prvog bajta (TTFB) – ili rečima, brzina odgovora servera, i na `DOMContentLoaded` – ili rečima, kada je stranica spremna za interakciju.

Ako je vreme učitavanja vaše stranice sporo, API ovo pokazuje tačno gde se kašnjenje dešava!

Konačno, hajde da istražimo `PerformanceObserver`.

API ovo prati operativne događaje kao što su strukturalne promene, duge zadatke i interakciju korisnika.

Da li želite pratiti dugačke zadatke za JavaScript?

```js
const observer = new PerformanceObserver((list) => {  
  list.getEntries().forEach((entry) => {  
    console.log(`Long task detected: ${entry.duration}ms`);  
  });  
});  

observer.observe({ type: "longtask", buffered: true });
```

Na API, šta ovo može da prati? Može da prati duge zadatke – ili rečima, JavaScript koji sprečava prikaz stranice, promenu rasporeda za otkrivanje problema sa interfejsom korisnika (UI), i Kašnjenje prvog unosa (FID) – ili kako stranica brzo reaguje na unos korisnika.

Ako želite da pratite performanse u realnom vremenu, API ovo je velika promena.

Pa, koji treba da koristi API? Evo brnog poređenja:

| Performance API | Koristi se za |
| :--- | :--- |
| `performance.now()` | Merenje sa preciznošću funkcije i skripte |
| Performance Timing API | Merenje performansi učitavanja cele stranice |
| Performance Observer | Praćenje interakcija u realnom vremenu i prikazivanje |

Povezujući ove API-je, možete izgraditi svoje alate za testiranje performansi i pratiti šta vaša web stranica najviše zahteva.
