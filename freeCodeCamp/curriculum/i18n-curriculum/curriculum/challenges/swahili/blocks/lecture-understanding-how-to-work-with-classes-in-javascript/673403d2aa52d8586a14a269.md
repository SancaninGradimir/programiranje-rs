# --interactive--

U programiranju, nasleđivanje vam omogućava da definišete klase objekata koje nasleđuju atribute i metode iz drugih klasa.

Na primer, `Car` je vrsta `Vehicle`, stoga možete definisati klasu `Vehicle` sa opštijim atributima i metodama koje se odnose na sve vrste vozila, a zatim definisati klasu `Car` koja nasleđuje atribute i metode iz klase `Vehicle`.

Ovaj odnos "je" je karakteristika nasleđivanja. U ovom primeru, `Car` je specifičnija vrsta od `Vehicle`.

Postoje dva glavna koncepta koja morate znati kada započinjete rad sa nasleđem: roditeljska klasa i detska klasa.

Roditeljska klasa je klasa koja funkcioniše kao mapa drugih klasa. Definiše svojstva i metode koje nasleđuju druge klase.

Klasa deteta je klasa koja nasleđuje svojstva i putanje druge klase. Klasa deteta takođe može povećati funkciju svoje roditeljske klase dodavanjem novih svojstava i putanja.

U našem početnom primeru, `Vehicle` će biti roditeljska klasa klase `Car` dok će klasa `Car` biti detetška klasa klase `Vehicle`.

Darasa la mzazi linaweza kuwa na madarasa mengi ya watoto.

U JavaScript, koristimo ključnu reč `extends` za implementaciju nasleđa. Ova reč ukazuje da je klasa podklasa neke druge klase.

U ovom primeru, gde su prikazane samo prve linije definicije klasa, možeš videti da klasa `Car` proširuje klasu `Vehicle`, što znači da nasleđuje klasu `Vehicle`.

```js
class Vehicle {
  // Implementation of Vehicle...
}

class Car extends Vehicle {
  // Implementation of Car...
}
```

Pretpostimo da klasa `Vehicle` ima dve osobine: `brand` i `year`. Možete ovo videti u ažuriranom kodu.

`Car` ima metodu nazvanu `honk`. Međutim, napomena je da klasa `Car` nema svoj član/komponentu. Možemo videti da li nasleđuje te atribute.

```js
class Vehicle {
  constructor(brand, year) {
    this.brand = brand;
    this.year = year;
  }
}

class Car extends Vehicle {
  honk() {
    console.log("Honk! Honk!");
  }
}
```

Da biste definisali primer za `Car`, potrebno je da prosledite dva argumenta: `brand` i `year`. U ovom slučaju, marka je `freeCodeCamp Motors` i godina je `2019`.

```js
let myCar = new Car("freeCodeCamp Motors", 2019);
```

Ako pokušate da dobijete ova svojstva i pozovete metodu koristeći notaciju tačke:

```js
console.log(myCar.brand);
console.log(myCar.year);
myCar.honk();
```

Ovde je kompletan primer:

Zaista, rezultati su tačni, stoga ove osobine i metode su definisane u primeru `Car` i uštedeli ste ponavljanje mnogo koda nasleđujući ove osobine iz klase `Vehicle`.

U ovom primeru, klasa deteta nije zahtevala nikakve dodatne atribute. Zato klasa nije zahtevala komponentu, već samo metodu.

Ako želite dodati dodatna svojstva/atribute, moraćete da definisati komponentu.

Da vidimo primer.

```js
class Vehicle {
  constructor(brand, year) {
    this.brand = brand;
    this.year = year;
  }
}

class Car extends Vehicle {
  constructor(brand, year, numDoors) {
    super(brand, year);
    this.numDoors = numDoors;
  }
}
```

Ovog puta, dodaćemo funkcionalnost `numDoors` za razred učenika `Car`.

Sada možete videti da ova klasa ima komponentu u ažuriranom kodu. Unutar komponente, postoji poziv na `super()`, prosleđujući argumente `brand` i `year`.

`super()` poziva komponentu superklase, stoga korišćenjem `super(brand, year)` definišete atribute superklase u klasi deteta. Ovo je drugi termin koji možete koristiti—superklasa je roditeljska klasa, a podklasa je detetška klasa.

Zatim, linija `this.numDoors = numDoors` definiše novu osobinu koja pripada isključivo klasi `Car`, broj vrata (`numDoors`).

Ovo je način za dodavanje klase za `Vehicle`, dodavanjem jedinstvenog atributa na instance `Car`.

To je primer širenja, jedan od osnovnih prednosti nasleđa.

Sada, ako kreirate primer klase `Car`, videćete da ima tri svojstva: `brand`, `year`, i `numDoors`. Takođe ćete morati da prosledite potrebne argumente kada kreirate primer.

```js
let myCar = new Car("freeCodeCamp Motors", 2019, 4);
```

Ako ispišete vrednosti ovih svojstava za `console.log()`:

```js
console.log(myCar.brand);
console.log(myCar.year);
console.log(myCar.numDoors);
```

Ovde je kompletan primer:

Glavna prednost nasleđivanja je ponovna upotreba koda, podela, proširivost i bolja struktura koda.

Pri implementaciji nasleđivanja klasa, možete ponovo koristiti kod napisan za roditeljsku klasu u detinjarskoj klasi, čime izbegavate ponavljanje koda.
