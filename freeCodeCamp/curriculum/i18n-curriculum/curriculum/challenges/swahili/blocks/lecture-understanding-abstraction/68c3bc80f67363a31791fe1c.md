Već smo razgovarali o enkapsulaciji, nasleđivanju i polimorfizmu, hajde da diskutujemo ključni koncept koji sledi u programiranju usmerenom na objekte – apstrakcija.

Abstrahovanje je proces sakrivanja složenih detalja implementacije i prikazivanja samo ključnih karakteristika objekta ili sistema. Razmislite o tome kao da se fokusirate na ono što objekat radi, umesto na to kako to radi.

Apstrakcija nije ograničena samo na Python. To je programski koncept koji se može implementirati u mnogim jezicima koji podržavaju objektno orijentisano programiranje.

Da bi se pokazala apstrakcija, zamisli da voziš automobil. Kada si na sedištu za vozača, koristiš ključne komponente kao što su volan, menjač i pedale gasa i kočnice. Ne moraš da znaš složene detalje o tome kako radi motor, kako transmisija menja brzine ili fiziku iza kočionog sistema, jer su sve to složeni detalji implementacije.

To je abstrahovanje u akciji! Daje vam pojednostavljeni interfejs za interakciju sa složenim sistemom.

U ponovnom primeru automobila, pojednostavljeni interfejs su volan, kočnice i gaspedal, dok je kompleksni sistem sam automobil.

O tome kako Python implementira apstrakciju, radi to putem modula `abc`.

Ovaj modul pruža klasu za `ABC` (što znači "abstract base class") i dekorator `@abstractmethod`.

`ABC` je klasa namenjena za nasleđivanje, ali ne možete kreirati objekte direktno iz nje. Definiše zajednički interfejs metoda i svojstava koje njegove podklase moraju da implementiraju.

S druge strane, apstraktni način je način deklarisan u Apstraktnoj Osnovnoj Klasu (ABC) korišćenjem dekoratora `@abstractmethod`. Može da nema nijednu implementaciju ili može imati podrazumevanu osnovnu implementaciju. Međutim, svaka podklasa mora da prepravi (override) bude smatrana konkretnom i da se može instancirati, čak i ako je dostavljena osnovna implementacija.

Ovo je osnovna sintaksa apstraktne klase u Python:

```py
from abc import ABC, abstractmethod

# Define an abstract base class
class AbstractClass(ABC):
    @abstractmethod
    def abstract_method(self):
        pass

# Concrete subclass that implements the abstract method
class ConcreteClassOne(AbstractClass):
    def abstract_method(self):
        print('Implementation in ConcreteClassOne')

# Another concrete subclass
class ConcreteClassTwo(AbstractClass):
    def abstract_method(self):
        print('Implementation in ConcreteClassTwo')
```

Ovo je osnovni primer:

```py
from abc import ABC, abstractmethod

class Animal(ABC): # Inherits from abstract base class
   @abstractmethod # Abstract method decorator
   def make_sound(self):  # The method subclasses must override
       pass

# Concrete class that will override the abstract method
class Dog(Animal):
   def make_sound(self):
       print('Woof!')

# Another concrete class that will override the abstract method
class Cat(Animal):
   def make_sound(self):
       print('Meow!')

# Another concrete class that will override the abstract method
class Monkey(Animal):
   def make_sound(self):
       print('Ooh ooh aah aah!')

# Create instances of each concrete class
animals = [Dog(), Cat(), Monkey()]

# Loop through the instances to call the make_sound method
for animal in animals:
   animal.make_sound()

# Output:
# Woof!
# Meow!
# Ooh ooh aah aah!
```

U ovom primeru:

- Uvezemo klasu `ABC` i `abstractmethod` iz modula `abc`.
- Zatim kreiramo klasu `Animal` koja nasleđuje od `ABC`, i kreiramo apstraktnu metodu `make_sound` unutar nje koju svaka podklasa `Animal` mora da prepravi.
- Kreiramo konkretne klase `Dog`, `Cat` i `Monkey`, koje moraju da preprave apstraktnu metodu `make_sound`.
- Kreiramo instancu konkretnih klasa i pozivamo njihovu metodu `make_sound` da pokaže kako svaka implementira apstraktnu metodu `make_sound` na svoj način.

Zapamtite da ne možete kreirati primer klase Animal. Evo šta se dešava kada pokušate to:

```py
dog = Animal() 
# TypeError: Can't instantiate abstract class Animal 
# without an implementation for abstract method 'make_sound'
```

Problem se tiče podklasa koje ne pružaju implementaciju apstraktne metode. Čak i ako deklariš podklasu za Animal, ne možeš da implementiraš/prepišeš make_sound. Evo primera koji to pokazuje:

```py
class Bird(Animal):
    pass

bird = Bird()
# TypeError: Can't instantiate abstract class Bird 
# without an implementation for abstract method 'make_sound'
```

Evo još jednog primera, ovog puta sa svojstvom modela koje mu možete dodeliti stvarne podatke:

```py
from abc import ABC, abstractmethod

# The blueprint for any toy that can speak
class TalkingToy(ABC):
   def __init__(self, name):
       self.name = name
   @abstractmethod
   def speak(self):
       pass

class RobotToy(TalkingToy):
   def speak(self):
       print(f'{self.name} says beep boop! I am a robot!')

class TeddyBearToy(TalkingToy):
   def speak(self):
       print(f"{self.name} says hug me! I'm cuddly!")

class DinosaurToy(TalkingToy):
   def speak(self):
       print(f'{self.name} says ROOOOAR!')

# Create toys
rusty = RobotToy('Rusty')
fluffy = TeddyBearToy('Fluffy')
rex = DinosaurToy('Rex')

toys = [rusty, fluffy, rex]
for toy in toys:
   toy.speak()

# Output:
# Rusty says beep boop! I am a robot!
# Fluffy says hug me! I'm cuddly!
# Rex says ROOOOAR!
```

U ovom primeru:

- Osnovna apstraktna klasa `TalkingToy` definiše šemu za svaku igračku koja može da govori.
- Podklase `RobotToy`, `TeddyBearToy` i `DinosaurToy` implementiraju metodu `speak` na svoj način.
- Kada kreiramo instance ovih podklasa i pozovemo metodu `speak`, svaka igračkica govori na svoj jedinstven način.

Zaključno, abstrahovanje u Python pojednostavljuje složene sisteme povećanjem ponovne upotrebe.

Jeste li videli kako možete ponovo koristiti jednu metodu iz apstraktne klase u mnogim podklasama, pritom prisiljavajući svaku podklasu da pruži svoje specifično ponašanje.

Ovaj pristup osigurava da je vaš kod organizovan, efikasan i lak za održavanje, posebno kada vaša aplikacija raste.
