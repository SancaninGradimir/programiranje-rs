# --description--

Ako uzmemo 47, obrnemo ga i dodamo, 47 + 74 = 121, što je palindrom.

Daje svim brojevima palindrom brzo na taj način. Na primer,

<div style="margin-left: 4em;">
  349 + 943 = 1292,<br>
  1292 + 2921 = 4213<br>
  4213 + 3124 = 7337<br>
</div>

Yaani, 349 ilichukua mizunguko mitatu kufikia palindromu.

Iako to još nije dokazano, veruje se da neki brojevi, poput 196, nikada neće proizvesti palindrom. Broj koji ne postane palindrom postupkom obrtanja cifara i sabiranja naziva se Lišrelov broj. Zbog prirode ovog problema i za potrebe ovog zadatka, pretpostavićemo da je broj Lišrelov dok se ne dokaže suprotno. Takođe je poznato da će za svaki broj manji od deset hiljada važiti jedno od sledećeg: (i) postaje palindrom u najviše pedeset iteracija ili (ii) niko do sada, uprkos svoj raspoloživoj računarskoj snazi, nije uspeo da ga pretvori u palindrom. Zapravo, 10677 je prvi broj za koji je pokazano da zahteva više od pedeset iteracija pre nego što postane palindrom: 4668731596684224866951378664 (53 iteracije, 28 cifara).

Zanimljivo je da postoje palindromski brojevi koji su Lišrelovi brojevi; prvi takav primer je 4994.

Koliko brojeva ima za Lychrel ispod `num`?

**Napomena:** Tekst je blago izmenjen 24. aprila 2007. kako bi se naglasila pretpostavljena priroda Lišrelovih brojeva.

```js
const countLychrelNumbers = (size) => {
  const numReverse = (num) => {
    return Number(num.toString().split('').reverse().join(''));
  };
  const isPalin = (num) => {
    if (numReverse(num) === num) {
      return true;
    }
    return false;
  };
  let total = 0;
  for (let i = 1; i < size; i++) {
    let loopCount = 1;
    let sum = i;
    while (loopCount < 50) {
      sum = sum + numReverse(sum);
      if (isPalin(sum)) {
        break;
      } else {
        loopCount++;
      }
    }
    if (loopCount === 50) {
      total++;
    }
  }
  return total;
}
```
