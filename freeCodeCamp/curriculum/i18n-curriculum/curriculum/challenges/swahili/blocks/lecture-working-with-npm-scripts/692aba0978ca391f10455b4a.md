# --description--

Naučimo se o npm bazi podataka, gde možete da podelite vaše pakete JavaScript sa celim svetom.

**npm registry** je repozitorijum javnih i privatnih paketa.

Razvojci i organizacije mogu objaviti pakete na ovoj bazi podataka kako bi podelili svoje alate sa svojim zajednicama i članovima tima, a u mnogim slučajevima, i svih drugih programera Node.js.

Paketi mogu biti javni ili privatni. Javni paketi su dostupni svima. Privatni paketi su dostupni samo određenim korisnicima ili članovima organizacije.

U npm registru, možete podeliti ono što ste izgradili sa svetom samo nekoliko klikova. Zvuči super, zar ne?

Da vidimo kako možete da objavite paket korak po korak.

## Korak 1: Kreiraj nalog

Prvi korak je kreiranje besplatnog npm naloga. Da biste kreirali svoj nalog:

* Pošalji [https://www.npmjs.com/signup](https://www.npmjs.com/signup).
    
* Popunite obrazac. Biće vam potrebno da unesete korisničko ime, e-poštu i lozinku.

Molimo vas da znate da će vaša e-mail adresa biti dodata u metapodatke paketa kada ih objavljujete, pa je tako može biti javno vidljiva.

## Korak 2: **Prijavi se**

Nakon što imate npm nalog, moraćete da uđete u svoj terminal i pokrenete ovu komandu:

```bash
npm login
```

Zatim ćete morati da verifikujete podatke vašeg naloga.

## **Korak 3: Kreiraj fajl za** `package.json`

Pre nego što odštampate vaš paket, moraćete da kreirate fajl sa imenom `package.json` ovom komandom:

```bash
npm init
```

(Ako je vaš projekat već spreman i imate ovaj fajl, možete direktno otići na **Korak 4**).

Moraćete da unesete ove informacije u vaš fajl `package.json`:

* **name**: Ovo je obično ime vašeg projekta/direktorijuma. Ako planirate da vaš paket učinite javnim, ovo ime mora biti jedinstveno, osim ako ne koristite opseg (o tome ćemo razgovarati uskoro).
    
* **version**: Treba početi sa `"1.0.0"`, podrazumevanom vrednošću. Ovo predstavlja početnu verziju vašeg paketa i možete ga ažurirati kako menjate paket.
    
* **main**: Tačka ulaska vašeg projekta. Podrazumevano je `index.js`.
    
* **author**: Možete napisati svoje ime ili ime koje želite da koristite.
    
* **license**: Izaberite specifičnu licencu. Primeri su MIT i ISC.
    
* **description**: Kratak sažetak vašeg paketa.

Kada izaberete ime vašeg paketa, uverite se da nije već korišćeno u bazi podataka npm-a. Da biste to uradili, možete otići na sledeću URL, menjajući `<your-package-name>` vašim imenom paketa:

```bash
https://www.npmjs.com/package/<your-package-name>
```

Ako nema paketa u URL, ime paketa je dostupno i možete ga koristiti.

Međutim, ako je ime već zauzeto i želiš da ga sačuvaš, možeš koristiti ono što se naziva **scope** za objavljivanje svog paketa.

### **Opseg**

Kada kreirate račun korisnika ili organizaciju u npm bazi podataka, dobijate opseg sa imenom vašeg korisnika ili imenom vaše organizacije.

Opseg pomaže u izbegavanju sukoba imena jer možete objaviti svoj paket pod svojim opsegom, čak i ako već postoji drugi paket sa istim imenom.

Ime paketa sa opsegom počinje imenom opsega. Na primer, u `package.json`, ćete naći ovu paru ključ-vrednost:

```bash
"name": "@<scope-name>/<your-package-name>"
```

Možete izvršiti ove izmene direktno u fajlu ako je potrebno, ili možete kreirati fajl `package.json` i opsega od samog početka dodavanjem `--scope=@<scope-name>` u komandu `npm init`.

```bash
npm init --scope=@<scope-name>
```

Naziv opsega može biti vaše korisničko ime ako koristite lični nalog, ili naziv vaše organizacije ako koristite korporativni nalog.

## Korak 4: Kreiraj README.md

README tvog paketa je Markdown fajl gde opisujete šta vaš paket radi i njegovu predviđenu upotrebu.

Primjeri važnog sadržaja koji biste možda željeli da uključite su:

* Kusudi lake.
    
* Jinsi ya kukisakinisha.
    
* Vitegemezi muhimu na mambo ya kuzingatia.
    
* Jinsi ya kuanza.
    
* Vipande vya mfano vya msimbo.
    
* Miongozo ya michango (ikiwa unakubali michango).
    
* Veza ka datoteci dnevnika izmena (changelog) u kojoj pratiš promene između različitih verzija.
    

Struktura README-a je lako prilagodljiva. Možete ga izmeniti tako da odgovara vašim potrebama.

## **Korak 5:** `.npmignore`

Fail `.npmignore` govori npm bazi koje fajlove treba uključiti i koje ne treba uključiti prilikom objavljivanja vašeg paketa. Vrlo je važno sprečiti deljenje osetljivih informacija kao što su privatni ključevi, lozinke i lični podaci.

U ovom fajlu, možete napisati obrasce koji pokazuju koji fajlovi, koje foldere i koje vrste fajlova nisu uključene. Prati iste pravila za podudarnost obrazaca kao i fajlovi `.gitignore`.

Ako si radio sa Git, možete da pretpostavite da je ova datoteka veoma slična `.gitignore`, zar ne? To je zato što tako jeste. Prema dokumentaciji npm-a, "Ako nema fajla `.npmignore`, ali postoji fajl `.gitignore`, onda npm ignoriše stvari koje odgovaraju fajlu `.gitignore`."

Pia, takođe bi trebalo da znate da npm traži fajlove `.npmignore` i `.gitignore` u svim podfoldernima vašeg paketa, a ne samo u glavnom folderu.

Po podrazumevanju, ove putanje se ignorišu direktno, pa nema potrebe da ih unosite u `.npmignore`:

* `.*.swp`
    
* `._*`
    
* `.DS_Store`
    
* `.git`
    
* `.gitignore`
    
* `.hg`
    
* `.npmignore`
    
* `.npmrc`
    
* `.lock-wscript`
    
* `.svn`
    
* `.wafpickle-*`
    
* `config.gypi`
    
* `CVS`
    
* `npm-debug.log`

Paпка `node_modules` takođe se ignoriše, osim uključene zavisnosti.

## **Korak 6: Objavite svoj paket**

Pre nego što objavite svoj paket u npm repozitorijum, trebalo biste ga testirati izvršavanjem ove komande:

```bash
npm install <full-path-to-package-directory>
```

Kada budete spremni za štampanje, komanda će zavisiti od tipa paketa: sa opseгом naspram bez opsega.

Ako vaš paket nema opsega, pokrenite ovu komandu:

```bash
npm publish
```

Ako vaš paket ima opseg, biće objavljen kao privatni paket podrazumevano (ako je ova funkcija dostupna na vašem nalogu). Ako želite da učinite paket javnim, moraćete da dodate `--access public` u komandu.

```bash
npm publish --access public
```

Da biste videli vaš javni profil, idite na URL:

```bash
https://npmjs.com/package/<your-package-name>
```

Trebalo bi da vidite status vašeg paketa (javnog ili privatnog) ispod imena paketa.

## **Korak 7 (Opciono): Instalirajte vaš paket**

Sada si sve pripremio/pripremila, možeš instalirati svoj paket u projekat Node.js sa ovom komandom:

```bash
npm install <your-package-name>
```

## Ažuriraj svoj paket

Ako trebate napraviti izmene u vašem paketu nakon objavljivanja, možete ažurirati verziju paketa u `package.json`.

Ovo je standardni ugovor o dostavljanju detaljne verzije:

* **Prva verzija:** `1.0.0`.
    
* **Verzija popravka (Patch):** Povećajte treći broj. Na primer, ažuriranje sa `1.0.0` na `1.0.1`.
    
* **Manja verzija (Minor):** Povećajte srednji broj i vratite poslednji broj na nulu. Na primer, ažuriranje sa `1.0.0` na `1.1.0`.
    
* **Glavna verzija (Major):** Povećajte prvi broj i vratite srednji i poslednji broj na nulu. Na primer, ažuriranje sa `1.1.0` na `2.0.0`.
    
Odlično. Sada znate kako da objavite svoj paket u npm repozitorijumu i kako da ažurirate za popravku grešaka i dodavanje novih funkcija.
