# --description--

Strategija je način autentifikacije korisnika. Možeš koristiti strategiju da dozvoliš korisnicima da se autentifikuju na osnovu informacija sačuvanih lokalno (ako si ih prethodno registrovao) ili od različitih provajdera kao što su Google ili GitHub. Za ovaj projekat, koristićemo Passport middleware. Passport pruža kompletan set strategija koje podržavaju autentifikaciju koristeći korisničko ime i lozinku, GitHub, Google, i više.

`passport-local@~1.0.0` je već dodata kao zavisnost. Uvezi je na svom serveru ovako:

```javascript
const LocalStrategy = require('passport-local');
```

Reci passport-u da **koristi** instancu `LocalStrategy` sa nekoliko podešenih postavki. Uveri se da je ovo (i sve odavde) zatvoreno unutar konekcije sa bazom podataka jer zavisi od toga!:

```javascript
passport.use(new LocalStrategy((username, password, done) => {
  myDataBase.findOne({ username: username }, (err, user) => {
    console.log(`User ${username} attempted to log in.`);
    if (err) return done(err);
    if (!user) return done(null, false);
    if (password !== user.password) return done(null, false);
    return done(null, user);
  });
}));
```

Ovo opisuje proces koji se koristi kada pokušavaš da autentifikuješ nekoga lokalno. Prvo, pokušava da pronađe korisnika u tvojoj bazi podataka po unetom korisničkom imenu. Zatim, proverava da li se lozinka poklapa. Konačno, ako nije bilo grešaka koje si proverio (npr. pogrešna lozinka), `user` objekat se vraća i korisnik je autentifikovan.

Mnoge strategije se podešavaju koristeći različite postavke. Generalno, lako ih je inicijalizovati prema README u repozitorijumu te strategije. Dobar primer je GitHub strategija gde ne moraš da brineš o korisničkom imenu ili lozinci jer će korisnik biti poslat na GitHub stranicu za autentifikaciju da se verifikuje. Sve dok su prijavljeni i pristaju, GitHub vraća njihov profil za tvoju upotrebu.

U sledećem koraku, postavićeš kako da pozoveš strategiju autentifikacije da autentifikuje korisnika na osnovu podataka iz forme.

Pošalji svoju stranicu kada vidiš da si uradio dobro. Ako naiđeš na greške, možeš <a href="https://forum.freecodecamp.org/t/advanced-node-and-express/567135#authentication-strategies-6" target="_blank" rel="noopener noreferrer nofollow">pogledati projekat urađen do ovog koraka</a>.
