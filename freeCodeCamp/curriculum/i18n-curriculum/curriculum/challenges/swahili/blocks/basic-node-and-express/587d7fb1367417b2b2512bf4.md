# --description--

Middleware može biti montirana na specifičnoj ruti koristeći `app.METHOD(path, middlewareFunction)`. Middleware takođe može biti ulančana u nizu unutar definicije rute.

Tazama mfano ufuatao:

```js
app.get('/user', function(req, res, next) {
  req.user = getTheUserSync();  // Hypothetical synchronous operation
  next();
}, function(req, res) {
  res.send(req.user);
});
```

Ovaj pristup je važan za deljenje server operacija u male, modularne delove. To donosi bolju organizaciju programa, i mogućnost ponovnog korišćenja koda na različitim mestima. Ovaj pristup se takođe može koristiti za validaciju podataka. Svaki deo niza middleware može da zaustavi izvršenje trenutnog lanca i prebaci kontrolu na specifično napravljene funkcije za rukovanje greškama. Ili možete prebaciti kontrolu na sledeću odgovarajuću rutu, za rukovanje specifičnim događajima. Videćemo kako u deo za naprednu Express upotrebu.

# --instructions--

U ruti `app.get('/now', ...)` povežite middleware funkciju sa finalnim handlerom. U middleware funkciji treba da dodate trenutno vreme u objekat zahteva pod ključem `req.time`. Možete koristiti `new Date().toString()`. U handleru, odgovorite sa JSON objektom, koristeći format `{time: req.time}`.

**Napomena:** Test neće proći ako ne kreirate lanac middleware. Ako postavite funkciju na drugo mesto, test će pasti, čak iako je rezultat tačan.
