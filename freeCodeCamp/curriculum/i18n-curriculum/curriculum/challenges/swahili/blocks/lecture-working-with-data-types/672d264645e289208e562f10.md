# --description--

JavaScript je jezik sa dinamički tipiziranim tipovima, što znači da ne morate specificirati tip podataka parametra kada ga deklarišete. Umesto toga, tip se određuje na osnovu vrednosti dodeljene parametru tokom izvršavanja programa. Ovo vam omogućava da promenite tip parametra u celom programu.

Na primer:

```js
let example = "Hello";
example = 42;
```

U ovom primeru, imamo parametar nazvan `example` sa tipom podataka niz znakova (string). Ali zatim ažuriramo vrednost u broj umesto toga.

Sposobnost dinamičkog tipiziranja čini JavaScript lakšim i tolerantnijim za brzo pisanje koda, ali može da uvede greške koje je teško otkriti, posebno kada program raste.

U jezicima sa unapred definisanim tipovima kao što su C# ili C++, moraš da deklariraš tip podataka parametra kada ga definišeš, i taj tip se ne može promeniti.

Na primer, ako deklarišete parametar kao `integer`, možete dobiti samo cele brojeve. Ako pokušate da mu date drugi tip, programa će vratiti grešku.

Evo primera u jeziku C#:

```csharp
int data = 42; // data must always be an integer
data = "Hello"; // This would cause an error in C#
```

Razlika između dinamičkog tipiziranja i static tipiziranja leži u efikasnosti naspram sigurnosti vašeg koda. Jezici sa dinamičkim tipiziranjem nude efikasnost, ali po cenu grešaka koje mogu nastati tokom izvršenja.

Jezici sa static tipizacijom postavljaju stroga pravila koja mogu sprečiti određene greške, ali zahtevaju više obaveštenja unapred i pružaju nisku efikasnost prilikom promene tipa.

Ovo je još jedan primer kreiranja parametra sa tipom `number`, a zatim ga kasnije konverzije u tip `string`:

```js
let data = 100;  // Initially a number
data = "New data";  // Dynamically changes to a string
```

Kada se kuca na static, promene ovakve nisu dozvoljene, jer će tip podataka biti definisan.

Konačno, dinamičko tipiziranje JavaScript omogućava parametrima da menjaju tip slobodno, što pruža efikasnost, ali može uzrokovati neočekivane greške tokom izvršavanja.

Jezici sa static tipiziranjem kao Java zahtevaju rano određivanje tipova parametara, što pomaže u otkrivanju grešaka pre nego što program počne, ali pruža manju efikasnost.
