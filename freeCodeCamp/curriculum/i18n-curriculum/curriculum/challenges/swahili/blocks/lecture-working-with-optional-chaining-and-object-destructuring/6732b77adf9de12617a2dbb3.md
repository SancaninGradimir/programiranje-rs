# --interactive--

Rešavanje strukture objekata je moćna karakteristika u JavaScript koja omogućava izvlačenje vrednosti iz objekata i postavljanje ih u parametre kraćim i čitljivijim načinom.

To je deo karakteristike ES6 (ECMAScript 2015) i postalo je važan alat za mnoge razvijatelje JavaScript.

Design pattern može pojednostaviti vaš kod, posebno kada koristite složene komponente ili kada morate vratiti više vrednosti istovremeno.

Po svojoj suštini, objektno orijentisano rešenje se tiče izvlačenja vrednosti iz objekata i postavljanja ih u različite parametre. Umesto da dobijete atribute objekta direktno, možete pružiti više atributa u jednoj deklaraciji. To može učiniti vaš kod čistijim i efikasnijim.

Počnimo sa primerom da pokažemo kako rešenje za strukturu objekata funkcioniše:

:::interactive_editor

```js
const person = { name: "Alice", age: 30, city: "New York" };

const { name, age } = person;

console.log(name); // Alice
console.log(age);  // 30
```

:::

U ovom primeru, izvlačimo atribute `name` i `age` iz objekta `person` i postavljamo ih u parametre sa istim imenima.

Jedna od prednosti modelovanja objekata je što možete dodeliti vrednosti parametrima sa različitim imenima. Ovo je posebno važno kada koristite stvari čija imena svojstava mogu stvoriti konflikt sa postojećim parametrima ili kada želite da koristite drugačije ime:

:::interactive_editor

```js
let person = { name: "Alice", age: 30, city: "New York" };

let { name: personName, age: personAge } = person;

console.log(personName); // Alice
console.log(personAge); //  30
```

:::

U ovom slučaju, dodeljujemo atribut `name` i postavljamo ga u parametar nazvan `personName`, i radimo isto sa `age` i `personAge`.

Rešavanje strukture objekata takođe vam omogućava da postavite podrazumevane vrednosti. Ako svojstvo nije prisutno u objektu koji rešavate, možete postaviti rezervnu vrednost:

:::interactive_editor

```js
let person = { name: "Alice", age: 30, city: "New York" };
let { name, age, country = "Unknown" } = person;

console.log(country); // Unknown
```

:::

Ovde, pošto `country` nije u našem `person`, uzima podrazumevanu vrednost `Unknown`.

Drugi uobičajeni slučaj je rešavanje strukture objekata organizovanih unutar. Možete pristupiti svojstvima unutar drugih objekata koristeći druge zagrade:

:::interactive_editor

```js
const recipe = {
  name: "Chocolate Cake",
  ingredients: {
    flour: "2 cups",
    sugar: "1 cup"
  }
};

// Extract `flour` from `ingredients`
const { ingredients: { flour } } = recipe;

console.log(flour); // "2 cups"
```

:::

Ovo je isto kao da se dobije direktan kredit:

```js
const flour = recipe.ingredients.flour;
console.log(flour); // "2 cups"
```

Sada, hajde da pričamo o kratkoj sintaksi u objektnom dizajnu modela. Kada kreirate objekte, posebno kada imena atributa odgovaraju imenima parametara, možete koristiti kratku sintaksu:

:::interactive_editor

```js
let name = "Bob";
let age = 25;

let person = { name, age };

console.log(person); // { name: "Bob", age: 25 }
```

:::

Gornji kod uzima atribute čija imena odgovaraju našim parametrima i dodeljuje im vrednosti tih parametara.

Ova kratka sintaksa je važna posebno kada vraćate stavke iz funkcije ili kada kreirate objekte sa mnogo svojstava:

:::interactive_editor

```js
function createPerson(name, age) {
  return { name, age };
}

let person = createPerson("Charlie", 35);
console.log(person); // { name: "Charlie", age: 35 }
```

:::

Rešenja objektnog modelovanja i sažeta sintaksa objekata su moćne karakteristike koje mogu učiniti vaš kod kraćim i lakšim za čitanje.

Posebno je važno kada koristite složene strukture podataka, ili kada morate prosleđivati više parametara funkcijama.

Kada nastavite da koristite JavaScript, otkrićete mnogo situacija gde ove tehnike mogu pojednostaviti vaš kod i učiniti ga čitljivijim.
