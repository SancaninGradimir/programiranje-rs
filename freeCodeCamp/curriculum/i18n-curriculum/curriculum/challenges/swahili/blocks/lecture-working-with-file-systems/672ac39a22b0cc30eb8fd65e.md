# --description--

Kada razvijate mrežne aplikacije, važno je da razmislite o organizaciji vaših fajlova i foldera. Ovo će pomoći da vaš kod bude dobro strukturiran i pojednostavi održavanje.

Uzmimo primer strukture foldera za HTML i CSS projekat:

```bash
.
├── /assets
│   ├── /images
│   │   ├── logo.png
│   │   ├── banner.jpg
│   │   └── icons.svg
│   ├── /fonts
│   │   ├── custom-font.woff
│   │   └── custom-font.woff2
├── /css
│   ├── main.css
│   ├── about.css
│   └── contact.css
├── index.html
├── about.html
├── contact.html
└── README.md
```

Na samom vrhu primera nalazi se tačka koja predstavlja koreni direktorijum.

Kornev direktorijum je najviši nivo direktorijuma u sistemu datoteka, koji se koristi kao početna tačka za sve putanje fajlova i sadrži sve ostale direktorijume i fajlove unutar sebe. Direktorijum je drugi naziv za mapu.

Unutar korenog direktorijuma, prvi prikazani direktorijum je direktorijum `assets`. Reč `assets` se odnosi na bilo koje fajlove koji se koriste u projektu, kao što su slike, fontovi ili drugi resursi.

U ovom primeru, postoje dva direktorijuma unutar direktorijuma `assets` koji se zovu `images` i `fonts`.

Sledeći direktorij će biti direktorij za `css`. Ponekad se ovo naziva i direktorij za stilove (*stylesheets*). Ovde čuvate sve vaše fajlove za CSS.

Ispod direktorijuma `css` nalaze se HTML fajlovi. Fajl `index.html` predstavlja početnu stranicu, dok `about.html` i `contact.html` predstavljaju stranice O nama i Kontakt.

Na kraju, tu je fajl `README.md`. Fajlovi `README` se često koriste da bi pružili važne informacije o projektu, kao što je šta projekat radi, kako se koristi i druge bitne detalje.

Nastavak `.md` predstavlja Markdown, što je jednostavan jezik oznaka koji se često koristi za pisanje dokumentacije.

Organizovanjem projekta u foldere za asete, CSS i HTML fajlove, olakšavate pronalaženje onoga što tražite. Takođe olakšavate drugim programerima da doprinesu vašem projektu.

Važno je znati da ovo je samo jedan primer kako možete organizovati vaše fajlove i foldere. Postoji mnogo različitih načina za kreiranje strukture projekta.
