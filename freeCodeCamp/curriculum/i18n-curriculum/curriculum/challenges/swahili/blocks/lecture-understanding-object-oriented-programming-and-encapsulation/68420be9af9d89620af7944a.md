# --description--

Program koji koristi klase objekata, poznat i kao OOP, je paradigma programiranja u kojoj programeri tretiraju sve u svom kodu kao stvarni objekat iz stvarnog sveta.

Klasa objekata je kao mapa za kreiranje objekata. Svaki objekat kreiran iz klase ima atribute koji definišu podatke i metode koje definišu ponašanje tih objekata.

U prethodnoj lekciji, naučili ste kako da kreirate klase. Ovo je podsetnik za sintaksu:

```py
class ClassName:
   def __init__(self, parameters):
       attribute = value

   def method_name(self):
       # method logic
```

Ovo je primer klase koja koristi posebnu metodu `__init__` za inicijalizaciju atributa brenda i boja svaki put kada se objekat kreira korišćenjem te klase:

```py
class Car:
   def __init__(self, brand, color):
       self.brand = brand
       self.color = color

# create two objects from the Car class
car1 = Car('Toyota', 'red')
car2 = Car('Lambo', 'green')

print('Car 1 Brand:', car1.brand) # Car 1 Brand: Toyota
print('Car 1 Color:', car1.color) # Car 1 Color: red

print('Car 2 Brand:', car2.brand) # Car 2 Brand: Lambo
print('Car 2 Color:', car2.color) # Car 2 Color: green
```

Program koji koristi objekte ima četiri glavna principa koji mu pomažu da efikasno organizuje i upravlja kodom. Ti principi su enkapsulacija svojstava objekta (encapsulation), nasleđivanje, polimorfizam i apstrakcija.

Preostali deo ovog tutorijala će se baviti kako funkcioniše vezivanje atributa elemenata.

Upakovanje atributa funkcije je spajanje atributa funkcije i metoda objekta u jednu jedinicu, klasu.

Kada definišete svojstva objekta, možete sakriti interni stanje objekta iza jednostavnog skupa javnih metoda i svojstava koja funkcionišu kao interfejs. Iza tog interfejsa nalaze se privatna svojstva i metode koje kontrolišu kako se podaci menjaju i ko ih može videti.

Pretpostavimo da želite pratiti saldo novčanika. Želite omogućiti ljudima da unose ili izvlače novac iz novčanika, ali niko ne bi trebalo da pristupi saldu direktno.

U tom slučaju, možete učiniti da `deposit()` i `withdraw()` budu javne metode, i sakriti saldo pod atributom `_balance`:

```py
class Wallet:
   def __init__(self, balance):
       self._balance = balance # For internal use by convention

   def deposit(self, amount):
       if amount > 0:
           self._balance += amount # Add to the balance safely

   def withdraw(self, amount):
       if 0 < amount <= self._balance:
           self._balance -= amount # Remove from the balance safely
```

Po konvenciji, postavljanje jednopreseknog prefiksa na atribute elementa i metode znači da su namenjeni za interno korišćenje. Niko ne bi trebalo da ih razmatra direktno izvan klase jer to krši pravila definisanja atributa elementa i može izazvati grešku.

Iako je prefiks jedne linije samo dogovor/sporazum, postavljanje prefiksa dve linije na svojstva elementa i metodu efikasno sprečava da se pristupi izvan njihove klase, čineći te svojstva i metode privatnim.

```py
class Wallet:
   def __init__(self, balance):
       self.__balance = balance # Private attribute

   def deposit(self, amount):
       if amount > 0:
           self.__balance += amount # Add to the balance safely

   def withdraw(self, amount):
       if 0 < amount <= self.__balance:
           self.__balance -= amount # Remove from the balance safely

account = Wallet(500)
print(account.__balance) # AttributeError: 'Wallet' object has no attribute '__balance'
```

Da biste dobili trenutnu vrednost `__balance`, možete definisati putanju za `get_balance`. Na primer:

```py
class Wallet:
   def __init__(self, balance):
       self.__balance = balance

   def deposit(self, amount):
       if amount > 0:
           self.__balance += amount

   def withdraw(self, amount):
       if 0 < amount <= self.__balance:
           self.__balance -= amount
  
   def get_balance(self):
       return self.__balance


acct_one = Wallet(100)
acct_one.deposit(50)
print(acct_one.get_balance()) # 150

acct_two = Wallet(450)
acct_two.withdraw(28)
print(acct_two.get_balance()) # 422

acct_two.deposit(150)
print(acct_two.get_balance()) # 572
```

Takođe možete definisati privatnu metodu za `__validate` da proverite da li je svaki iznos za depozit ili povlačenje pozitivan broj.

```py
class Wallet:
   def __init__(self):
       self.__balance = 0

   def __validate(self, amount):
       if amount < 0:
           raise ValueError('Amount must be positive')

   def deposit(self, amount):
       self.__validate(amount)
       self.__balance += amount

   def withdraw(self, amount):
       self.__validate(amount)
       if amount > self.__balance:
           raise ValueError('Insufficient funds')
       self.__balance -= amount

   def get_balance(self):
       return self.__balance

acct_one = Wallet()
acct_one.deposit(3)
print(acct_one.get_balance()) # 3

acct_one.deposit(50)
print(acct_one.get_balance()) # 53

acct_one.deposit(-4)  # ValueError: Amount must be positive
acct_one.withdraw(-8) # ValueError: Amount must be positive
acct_one.withdraw(58) # ValueError: Insufficient funds
```

Kao što vidite, ruta `__validate` je privatna, i radi u pozadini na javnim rutama od `deposit()` i `withdraw()` kako bi osigurala da je iznos uvek validan.

U sledećoj lekciji, naučićete više o tome kako svojstva elemenata sa prefiksom od dve linije funkcionišu.

Ukratko, implementacija svojstava komponente zatvara interne podatke iza eksponovanih javnih interfejsa. Na taj način štiti vaše klase od međuižanstvenog uticaja i omogućava centralizovano razmatranje autentifikacije. Možete da modifikujete ili dodate svoj kod slobodno, znajući da eksterni kod pogađa samo interfejse koje eksponujete.
