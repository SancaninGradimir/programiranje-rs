# --description--

Poslednji deo strategije je da se bavi profilom koji se vraća sa GitHub-a. Trebamo da pronađemo korisnika u bazi podataka ako postoji, ili da ga kreiramo ako ne postoji, popunimo podatke iz profila, a zatim vratimo korisnika. GitHub nam daje *id* unutar svakog profila koji možemo koristiti za pretragu kako bismo identifikovali korisnika (što je već implementirano). Ispod je primer implementacije koji možete koristiti u svom projektu - ide unutar funkcije koja je drugi argument nove strategije, tačno ispod mesta gde se trenutno nalazi `console.log(profile);`:

```js
myDataBase.findOneAndUpdate(
  { id: profile.id },
  {
    $setOnInsert: {
      id: profile.id,
      username: profile.username,
      name: profile.displayName || 'John Doe',
      photo: profile.photos[0].value || '',
      email: Array.isArray(profile.emails)
        ? profile.emails[0].value
        : 'No public email',
      created_on: new Date(),
      provider: profile.provider || ''
    },
    $set: {
      last_login: new Date()
    },
    $inc: {
      login_count: 1
    }
  },
  { upsert: true, new: true },
  (err, doc) => {
    return cb(null, doc.value);
  }
);
```

`findOneAndUpdate` vam omogućava da pronađete dokument i ažurirate ga. Ako dokument ne postoji, biće umetnut i vraćen kroz funkciju povratnog poziva. U ovom primeru, svaki put postavljamo `last_login`, povećavamo `login_count` za `1`, i popunjavamo polja samo kada se novi dokument umetne (novi korisnik). Primetite korišćenje podrazumevanih vrednosti. Povremeno vraćeni profil neće imati sva polja popunjena ili korisnik će postaviti da je privatno. U tom slučaju, to rukujete da biste sprečili grešku.
