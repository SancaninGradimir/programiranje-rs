# --description--

- **JavaScript** dešava se sekvencijalno i čeka da prethodna operacija završi pre nego što nastavi sa sledećom.
- **JavaScript** omogućava više operacija da se izvršavaju u pozadini bez blokiranja glavne niti.
- **Niti** su sekvenca instrukcija koje se mogu izvršiti nezavisno od glavnog toka programa.
- **Handler događaja** su funkcije koje se prosleđuju kao argumenti drugim funkcijama i izvršavaju se nakon završetka operacije ili kao rezultat događaja.

## Engine za JavaScript u okruženju izvršavanja JavaScript

- **Motor JavaScript** je program koji izvršava kod JavaScript u web pregledaču. Radi kao prevodilac koji uzima vaš kod, konvertuje ga u instrukcije koje računar može da razume i izvrši shodno tome.
- V8 je primer motora JavaScript napravljenog od strane Google-a.
- **Okruženje za izvršavanje JavaScript** je okruženje u kojem se kod JavaScript dešava. Uključuje motor JavaScript koji obrađuje i izvršava kod, kao i dodatne komponente poput web pregledača ili Node.js.

## API od Fetch

- API Fetch omogućava mrežnim aplikacijama da izvrše mrežne zahteve, obično radi dohvaćanja ili slanja podataka na server. Pruža metod `fetch()` koji možete koristiti za slanje ovih zahteva.
- Možete dobiti tekst, slike, audio, JSON i druge vrste podataka koristeći metod API Fetch-a.

## Putanje za HTTP za API u Fetch

API Fetch podržava različite metode za HTTP da se poveže sa serverom. Najčešće korišćene metode su:

- **GET**: Koristi se za dobijanje podataka sa servera. Po defaultu, API od Fetch koristi metodu `GET` za dobijanje podataka.

```js
fetch('https://api.example.com/data')
```

Da bi se koristili dostupni podaci, mora biti konvertovano u strukturu JSON koristeći metodu `.json()`:

```js
fetch('https://api.example.com/data')
  .then(response => response.json())
  .then(data => console.log(data))
```

U ovom kodu, `fetch()` vraća Promise, i handler za `.then()` konvertuje rešeni rezultat u format JSON.

- **POST**: Koristi se za slanje podataka na server. Metoda `POST` se koristi za kreiranje novog resursa na serveru.

```js
fetch('https://api.example.com/users', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    name: 'John Doe',
    email: 'john@example.com'
  })
})
```

U ovom primeru, šaljemo zahtev za `POST` kreiranje novog korisnika. Specifikovali smo metodu kao `POST`, postavljanjem odgovarajućih zaglavlja i uključivanjem tela sa podacima koje želimo poslati. Telo mora biti niz znakova (string), pa koristimo `JSON.stringify()` da konvertujemo naš objekat u string sa JSON.

- **PUT**: Koristi se za ažuriranje podataka na serveru. Metoda `PUT` se koristi za ažuriranje postojećih resursa na serveru.

```js
fetch('https://api.example.com/users/45', {
  method: 'PUT',
  headers: {
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    name: 'John Smith',
    email: 'john@example.com'
  })
})
```

U ovom primeru, ažuriramo ID od `45` koji je definisan na kraju URL. Koristili smo metodu `PUT` u kodu i takođe smo definisali podatke kao telo koje će se koristiti za ažuriranje identifikovanih podataka.

- **DELETE**: Koristi se za brisanje podataka na serveru. Metoda ``DELETE`` koristi se za brisanje resursa sa servera.

```js
fetch('https://api.example.com/users/45', {
  method: 'DELETE'
})
```

U ovom primeru, šaljemo zahtev za `DELETE` brisanje korisnika sa ID-om `45`.

## Obećanje i povezivanje obećanja

- **Obećanje** je objekat koji predstavlja završetak ili neuspeh asinhronih operacija i vrednost koja iz njih proizlazi. Vrednost obećanja poznata je tek kada se asinhrona operacija završi.
- Evo primera kreiranja jednostavnog obećanja:

```js
const promise = new Promise((resolve, reject) => {
  setTimeout(() => {
    resolve('Data received successfully');
  }, 2000);
});
```

- Putanja `.then()` se koristi u Promise-u da pokaže šta treba da se desi kada je Promise izvršen, dok se `.catch()` koristi za rukovanje bilo kakvim greškama koje nastanu.
- Evo primera korišćenja `.then()` i `.catch()` sa Promise-ima:

```js
promise
  .then(data => {
    console.log(data);
  })
  .catch(error => {
    console.error(error);
  });
```

U navedenom primeru, putanja ``.then()`` se koristi za pisanje podataka primljenih od Ahadija, dok putanja ``.catch()`` se koristi za pisanje bilo kakvih grešaka koje nastanu.

- **Lančevanje obećanja**: Jedna od moćnih karakteristika Obećanja je što možemo spojiti više asinhronih operacija zajedno. Svaki `.then()` može vratiti novo Obećanje, omogućavajući nam izvršavanje sekvence asinhronih operacija jedna za drugom.
- Evo primera lančevanja obećanja:

```js
fetch('https://api.example.com/data')
  .then(response => response.json())
  .then(data => {
    console.log(data);
    return fetch('https://api.example.com/other-data');
  })
  .then(response => response.json())
  .then(otherData => {
    console.log(otherData);
  })
  .catch(error => {
    console.error(error);
  });
```

U ovom primeru, prvo dobijamo podatke iz URL jedan, zatim dobijamo podatke iz URL drugi na osnovu prvog odgovora, i na kraju zapisujemo drugi primljeni podatak.

Mehanizam `catch` će rukovati svim greškama koje se dogode tokom procesa. To znači da ne morate dodavati upravljanje greškama za svaki korak, što značajno pojednostavljuje vaš kod.

## Korišćenje `async/await` za rukovanje obećanjima

`async`/`await` To olakšava pisanje i čitanje asinhronog koda, koji je baziran na Obećanjima (Promises).

- **async**: Ključna reč `async` se koristi za definisanje asinhrone funkcije. Funkcija `async` vraća Obećanje (Promise), koje je rešeno vrednošću koju vraća funkcija `async`.
- **await**: Ključna reč `await` se koristi unutar funkcije `async` za suspenzovanje izvršavanja funkcije dok Obećanje (Promise) nije rešeno. Može se koristiti samo unutar funkcije `async`.
- Evo primera korišćenja `async/await`:

```js
async function delayedGreeting(name) {
  console.log("A Messenger entered the chat...");
  await new Promise(resolve => setTimeout(resolve, 2000));
  console.log(`Hello, ${name}!`);
}

delayedGreeting("Alice");
console.log("First Printed Message!");
```

U ovom primeru, funkcija `delayedGreeting` je funkcija `async` koja čeka 2 sekunde pre štampanja pozdravne poruke. Ključna reč `await` koristi se za zaustavljanje izvršavanja funkcije dok `Promise` nije rešeno.

- Moja najveća prednost/korist od `async/await` je upravljanje greškama putem sekcija `try/catch`. Evo primera:

```js
async function fetchData() {
  try {
    const response = await fetch('https://api.example.com/data');
    const data = await response.json();
    console.log(data);
  } catch (error) {
    console.error(error);
  }
}

fetchData();
```

U ovom primeru, deo ``try`` ima kod koji može da izazove grešku, a deo ``catch`` rukuje greškom ako dođe do nje. Ovo čini upravljanje greškama jednostavnim i razumljivim.

## Svojstvo `async`

- Funkcija `async` opisuje pretraživač koji preuzima skriptu fajl asinhrono dok nastavlja da analizira dokument HTML.
- Kada se skripta preuzme, analiza HTML je privremeno zaustavljena, izvršava se skripta, nakon čega nastavlja analiza HTML.
- Trebalo bi da koristite `async` za samostalnu skriptu gde red izvršavanja nije garantovan.

## Karakteristika `defer`

- Karakteristika `defer` takođe preuzima asinhronu skriptu, ali odlaže izvršavanje skripte do nakon što je dokument HTML potpuno analiziran.
- Skripte `defer` čuvaju redosled izvršavanja kako je prikazano u dokumentu HTML.

- Važno je napomenuti da sva svojstva `async` i `defer` nisu uzeta u obzir za *inline* skriptu i rade samo za eksterne skripte.

- Kada su funkcije za `async` i `defer` zajedno, funkcija `async` dobija prioritet.

## API za Geolokaciju

- API za geolokaciju pruža način da veb stranica traži lokaciju korisnika.

- Primer ispod pokazuje način `getCurrentPosition()` od API, koji se koristi za dobijanje trenutne lokacije korisnika.

```js
navigator.geolocation.getCurrentPosition(
  (position) => {
    console.log("Latitude: " + position.coords.latitude);
    console.log("Longitude: " + position.coords.longitude);
  },
  (error) => {
    console.log("Error: " + error.message);
  }
);
```

U ovom kodu, pozivamo `getCurrentPosition` i dajemo mu funkciju koja će biti pozvana kada je prilika uspešno dostupna.

Predmet `position` ima različite informacije, ali ovde smo izabrali samo `latitude` i `longitude`.

Ako postoji problem sa dobijanjem `position`, onda će greška biti ispisana na konzolu.

- Važno je poštovati privatnost korisnika i tražiti njihovu lokaciju samo kada je neophodno.
