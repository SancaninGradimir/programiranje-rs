# --description--

Formular je osnova svake veb aplikacije jer omogućava obradu unosa korisnika, prikupljanje podataka i pokretanje akcija.

U React, forme se kontrolišu korišćenjem stanja (state) ili referenci (refs), dajući vam potpunu kontrolu nad njihovim ponašanjem i validacijom. Ova dva načina kontrole formulara nazivaju se "controlled" i "uncontrolled" input.

Istražimo šta znače kontrolisani i nekontrolisani ulazi.

Kontroilisani unos je metoda koja je najviše slična obrascu React za rukovanje poljima unosa forme. Za kontrolisane unose, skladištite vrednost polja unosa u *state* i ažurirajte ga kroz događaje (events) od `onChange`. Ovo vam daje potpunu kontrolu nad podacima forme i omogućava trenutnu validaciju i prikaz uslova.

Proces funkcioniše ovako: React upravlja stanjem forme korišćenjem hook-a `useState`, i čuva svaku promenu. Kada korisnik kuca u polje za unos, dešava se događaj `onChange`, stanje se ažurira, a React ponovo renderuje to polje sa novom vrednošću.

```jsx
import { useState } from "react";

function App() {
  const [name, setName] = useState("");

  const handleChange = (e) => {
    setName(e.target.value);
  };

  const handleSubmit = (e) => {
    e.preventDefault();
    console.log(name);
  };

  return (
    <>
      <form onSubmit={handleSubmit}>
        <label htmlFor="name">Your name</label> <br />
        <input value={name} id="name" onChange={handleChange} type="text" />
        <button type="submit">Submit</button>
      </form> 
    </>
  );
}

export default App;
```

Prednost za controlled inputs je sledeća:

- Trenutna dostupnost podataka forme.

- Možete izvršiti trenutnu verifikaciju.

- Možete onemogućiti dugme za slanje na osnovu uslova.

- Možete kontrolisati vrednost unosa programski.

S druge strane, uncontrolled inputs su se više vide u tradicionalnim formularima od HTML. Stoga, umesto da obrađuju ulaz putem hook-a `useState`, uncontrolled inputs u HTML čuvaju svoje interne stanje uz pomoć DOM.

Pošto DOM upravlja vrednostima unosa, ono što vam je potrebno je povući vrednosti ulaznih delova koristeći referencu. Ova metoda zahteva manje koda i radi bolje jer reference ne izazivaju da se React ponovo renderuje.

Ovde je primer nekontrolisanih ulaza:

```jsx
import { useRef } from "react";

function App() {
  const nameRef = useRef();

  const handleSubmit = (e) => {
    e.preventDefault();
    console.log(nameRef.current.value);
  };

  return (
    <form onSubmit={handleSubmit}>
      <label htmlFor="name">Your</label>{" "}
      <input type="text" ref={nameRef} id="name" />
      <button type="submit">Submit</button>
    </form>
  );
}

export default App;
```

Jedna od veoma poznatih prednosti nekontrolisanih ulaza je što zahtevaju manje koda. Takođe bolje funkcionišu i osećaju se prirodno za one koji počinju da koriste React a znaju HTML.

Dakle, koji treba da se koristi između kontrolisanih i nekontrolisanih ulaza?

Koristite kontrolisani inpute kada želite ažuriranja forme u realnom vremenu, trenutnu validaciju ili kada želite uskladiti vrednosti unosa i stanje (state). Oni pružaju bolju kontrolu, ali zahtevaju češće ponovno renderovanje.

Koristite nekontrolisane ulaze kada želite jednostavniji formular, kada samo želite dobiti vrednost prilikom slanja, ili kada koristite kod koji nije React.

Bez obzira da li koristite kontrolisane ili nekontrolisane unose, ovde su najbolje prakse koje biste trebali uzeti u obzir kada kreirate formu u React:

- Uvek blokiraj slanje forme putem default.

- Uverite se da ste validirali unos pre slanja.

- Uvek pružajte jasan povratni odgovor korisnicima o statusu učitavanja, greškama autentifikacije ili drugim srodnim stanjima.
