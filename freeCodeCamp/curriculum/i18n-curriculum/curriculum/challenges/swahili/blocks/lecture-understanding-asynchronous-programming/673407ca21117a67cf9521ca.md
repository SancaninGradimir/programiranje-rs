# --description--

`Promise` predstavlja status ili neuspešnost operacije koja nije u realnom vremenu. Prvo je u stanju čekanja. Zatim se može promeniti u stanje izvršeno kada operacija uspe, ili u stanje odbijeno kada operacija ne uspe. Evo primera kreiranja `Promise`:

```js
const aPromise = new Promise((resolve, reject) => {
  setTimeout(() => {
    resolve("Operation successful!");
  }, 1000);
});
```

U ovom primeru, kreiramo obećanje koje izvršava asinhronu operaciju koristeći `setTimeout`. Nakon jedne sekunde, obećanje se rešava porukom `Operation successful!`.

Drugi način rada sa obećanjima je korišćenje metoda `.then` i `.catch`.

Metoda `.then()` koristi se kod Promise objekata da odredi šta treba da se desi kada se Promise uspešno izvrši, dok se `.catch()` koristi za obradu grešaka koje se pojave. Pogledajmo kako možemo koristiti `.then()` i `.catch()` sa našim Promise objektom:

```js
aPromise.then((result) => {
  console.log(result);  // Outputs: "Operation successful!"
}).catch((error) => {
  console.error(error);
});
```

U ovom kodu funkcija prosleđena metodi `.then()` biće pozvana sa razrešenom vrednošću Promise objekta. Ako dođe do greške, umesto toga biće pozvana funkcija prosleđena metodi `.catch()`.

Sada hajde da razgovaramo o ulančavanju obećanja (promise chaining). Jedna od najmoćnijih osobina obećanja jeste to što možemo povezati više asinhronih operacija. Svaki `.then()` može da vrati novo obećanje, omogućavajući vam da izvršavate niz asinhronih operacija jednu za drugom. Evo primera:

```js
fetch('https://api.example.com/data')
  .then(response => response.json())
  .then(data => {
    console.log(data);
    return fetch('https://api.example.com/data2');
  })
  .then(response => response.json())
  .then(data2 => console.log(data2))
  .catch(error => console.error('Error:', error));
```

U ovom primeru izvršavamo dva API poziva jedan za drugim. Prvi `.then()` parsira odgovor kao JSON. Drugi `.then()` ispisuje podatke i šalje drugi API zahtev. Treći `.then()` parsira odgovor drugog API poziva, a četvrti ispisuje te podatke. Ako se bilo gde u ovom lancu pojavi greška, biće uhvaćena u završnom `.catch()`.

Važno je napomenuti da će `.catch()` uhvatiti greške iz bilo kog prethodnog koraka u lancu. To znači da nije potrebno dodavati obradu grešaka za svaki pojedinačni korak, što može znatno pojednostaviti tvoj kod.
