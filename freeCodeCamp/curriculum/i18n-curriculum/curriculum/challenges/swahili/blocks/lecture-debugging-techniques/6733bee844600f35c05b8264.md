# --interactive--

U prethodnoj lekciji, naučili ste kako da bacate izuzetke u svoje programe. U ovoj lekciji, gledaćemo kako da se ove greške elegantno rukuju unutar elementa `try…catch…finally`.

Funkcija `try` se koristi za obavotavanje koda koji može da baci grešku. Deluje kao sigurno mesto za testiranje nečega što može da propadne.

Funkcionalnost `catch` uhvata i obrađuje greške koje nastaju u funkcionalnosti `try`. Možete koristiti objekat `Error` unutar `catch` da istražite šta je uzrokovalo grešku.

Komponenta `finally` se izvršava nakon komponenti `try` i `catch`, bez obzira na to da li je došlo do greške ili ne. Često se koristi za zadatke čišćenja, kao što su zatvaranje fajlova ili oslobađanje resursa.

Ovo je primer korišćenja funkcije `try…catch`:

:::interactive_editor

```js
function processInput(input) {
  if (typeof input !== "string") {
    throw new TypeError("Input must be a string.");
  }

  return input.toUpperCase();
}

try {
  console.log("Starting to process input...");
  const result = processInput(9);
  console.log("Processed result:", result);
} catch (error) {
  console.error("Error occurred:", error.message);
} 
```

:::

U ovom primeru, imamo funkciju nazvanu `processInput` koja prvo proverava da li `input` nije tip `string`. Ako je to slučaj, onda bacamo grešku. Inače, vraćamo rezultat korišćenjem metode `toUpperCase` na `input`.

Pozivamo funkciju unutar bloka `try`. Pošto poziv funkcije baca grešku, ona će biti uhvaćena u bloku `catch`, a poruka o grešci će biti prikazana u konzoli.

Greška poslata unutar elementa `catch` je objekat od `Error` koji sadrži informacije o toj grešci. U ovom slučaju, koristimo atribut `message` koji prikazuje informacije razumljive za čoveka (korisnika).

Koristimo `console.error` zato što je namenjen upravo za beleženje grešaka. U većini modernih pregledača, izlaz funkcije `console.error()` prikazuje se crvenom bojom u konzoli.

Izjava za `finally` se dešava bez obzira na to da li je izuzetak (exception) bačen ili ne.

```js
try {
  // Code that might throw an error
} catch (error) {
  // Code to handle the error
} finally {
  // Code that runs regardless of whether an error occurred or not
}
```

Dobar primer upotrebe bloka `finally` je rad sa datotekama. U JavaScript-u možete otvoriti datoteku, koristiti blok `try` za upis podataka u nju, a ako dođe do greške, blok `catch` će je obraditi. Nakon toga možete koristiti blok `finally` da zatvorite datoteku.
