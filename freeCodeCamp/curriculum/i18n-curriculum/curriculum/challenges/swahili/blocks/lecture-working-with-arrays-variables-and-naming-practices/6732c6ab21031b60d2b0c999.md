# --description--

Dodeljivanje imena promenljivama i funkcijama je važan deo pisanja čistog koda, koji je razumljiv i jednostavan za održavanje. Dobre konvencije imenovanja čine vaš kod samodokumentovanim, smanjujući potrebu za mnogo komentara i olakšavajući drugim programerima, uključujući vas sebe kasnije, razumevanje vašeg koda.

Počinjemo sa opštim konvencijama imenovanja u JavaScript. U prethodnim lekcijama, naučili ste o korišćenju camel case za imena parametara. Za parametre sa vrednostima tačno ili netačno (boolean), uobičajena je praksa korišćenja prefiksa kao što su `is`, `has`, ili `can`. Ovo čitaocu odmah govori da je parametar vrednosti tačno ili netačno:

```js
let isLoading = true;
let hasPermission = false;
let canEdit = true;
```

Imena funkcija bi trebalo jasno da pokazuju šta ta funkcija radi. Često je korisno početi glagolom:

```js
function getUserData(){
  /* ... */
}

function calculateTotal(){
  /* ... */
}

function validateInput(){
  /* ... */
}
```

Za funkcije koje vraćaju vrednost istinito ili neistinito, često nazvane predikati, možete koristiti iste prefikse kao i `is`, `has`, ili `can`:

```js
function isValidEmail(email) {
  /* ... */
}

function hasRequiredFields(form) {
  /* ... */
}
```

Ako postoje funkcije koje dohvaćaju podatke, uobičajeno je početi sa rečju `get`:

```js
function getProductDetails(productId) {
  /* ... */
}

function getUserProfile(userId) {
  /* ... */
}
```

Kada ima funkcija koje unose podatke, uobičajeno je početi sa rečju `set`:

```js
function setUserPreferences(preferences) {
  /* ... */
}

function setPageTitle(title) {
  /* ... */
}
```

Za funkcije obrađivača događaja, možete postaviti prefiks `handle` ili sufiks `handler`:

```js
function handleClick(){
  /* ... */
}

function onSubmit(){
  /* ... */
}

function keyPressHandler(){
  /* ... */
}
```

Обрадавач događaja je akcija koja se dešava nakon događaja, kao što je kliktanje na dugme. Ovo ćete naučiti u narednim lekcijama.

Kada imenujete parametre petlje i petlje, uobičajeno je koristiti samo slovo kao `i`, `j`, ili `k`, ali za ugnježdenim petljama ili složenijim petljama mogu pomoći opisna imena:

```js
for (let i = 0; i < array.length; i++) {
  /* ... */
}

for (let studentIndex = 0; studentIndex < students.length; studentIndex++) {
  /* ... */
}
```

Kada imenujete kolonu podataka, razmislite o korišćenju pluralnih imenica kako biste pokazali da atribut ima više stavki:

```js
const colors = ['red', 'green', 'blue'];
const userNames = ['Alice', 'Bob', 'Charlie'];
```

Zapamti, cilj je da tvoj kod bude što više samodosmatiran/dokumentovan. Dobra praksa je da ako trebaš dodati komentare koji objašnjavaju šta radi neki parametar ili funkcija, možda ćeš morati razmisliti o davanju drugog imena koje ima više objašnjenja.

Na kraju, imajte čvrste konvencije u svom kodu ili timu. Ako vaš tim ima postavljene konvencije imenovanja, pridržavajte se toga. Doslednost čini kod lakšim za čitanje i održavanje za sve uključene.
