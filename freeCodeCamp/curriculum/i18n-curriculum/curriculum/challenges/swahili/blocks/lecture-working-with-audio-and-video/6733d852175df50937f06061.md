# --description--

Za prikaz zvuka i videa, ponekad želite snimiti zvuk ili video. API za Media Capture and Streams, ili API za MediaStream, to vam omogućava.

Da bi koristio ovaj API, potrebno je da kreiraš objekat `MediaStream`. To možeš uraditi pomoću konstruktora, ali on neće biti povezan sa korisničkim uređajima. Umesto toga, svojstvo `mediaDevices` globalnog objekta `navigator` nudi metod `getUserMedia()` koji možeš koristiti.

Ovaj metod prihvata objekat `constraints` koji opisuje koje vrste medijskog sadržaja želiš da dobiješ. Objekat ima svojstva `audio` i `video` koja predstavljaju audio i video tokove. Ova svojstva mogu biti `false` ako ne želiš taj tip toka, `true` ako ga želiš ili objekti koji definišu dodatna ograničenja.

Na primer, možda vam je potrebna specifična video rezolucija:

```js
window.navigator.mediaDevices.getUserMedia({
  audio: true,
  video: {
    width: {
      min: 1280,
      ideal: 1920,
      max: 3840
    },
    height: {
      min: 720,
      ideal: 1080,
      max: 2160
    }
  }
});
```

Ovaj deo objašnjava nizu i visoku rezoluciju za video striming. Funkcija `ideal` opisuje željenu rezoluciju – a on će pružiti rezoluciju najbližu vašoj idealnoj rezoluciji.

Čim kreirate svoj MediaStream (ako korisnik prihvati direktan zahtev za pristup njihovim uređajima), možete koristiti podatke strema na bilo koji način koji vam je potreban.

Zapamtite da `getUserMedia()` vraća Ohverovanje (Promise), što znači da ćete morati koristiti funkciju reacciona kako biste koristili tok, ili koristiti sintaksu async/await. Više ćete saznati o Ohverovanju i asinhronom programiranju u narednim lekcijama.

Ovo je osnovni primer koji prikazuje tok kamere korisnika na stranici.

```js
const video = document.querySelector("video");
const stream =
  await window.navigator.mediaDevices.getUserMedia({ video: true });
video.srcObject = stream;
await video.play();
```

Važno je znati da API ovo ne daje pristup snimanju ekrana. Naučićete se o API u sledećoj lekciji.
