# --description--

U Express, `express.Router()` je moćna funkcija koja vam pomaže da organizujete vaše putanje podataka u male, modularne delove. Ovo je posebno važno kada vaša aplikacija raste.

Umesto da sve rute definišete u jednoj velikoj datoteci, možete ih podeliti u više datoteka korišćenjem rutera. To čini kod preglednijim, lakšim za održavanje i proširivanje.

Put za `express.Router()` kreira novi generator puta.

Ruter funkcioniše kao mini Express aplikacija. Može da obrađuje svoje rute i middleware, ali nije kompletna Express aplikacija. Kada definišete rute u ruteru, možete ih povezati sa glavnom aplikacijom pomoću `app.use()`.

Drugim rečima, ruter vam omogućava da grupišete povezane rute, a zatim ih kasnije dodate u glavnu aplikaciju.

Korišćenje planera ruta nudi nekoliko prednosti:

**Kutenganisha masuala**

Rute možete definisati u posebnim datotekama, što poboljšava organizaciju i čitljivost aplikacije.

**Matumizi tena ya msimbo**  
Ruteri se mogu ponovo koristiti u različitim delovima aplikacije ili čak u drugim projektima.

**Čistija glavna datoteka**
Umesto da glavnu datoteku `app.js` ili `server.js` zatrpate velikim brojem definicija ruta, možete ih premestiti u rutere i glavnu datoteku zadržati fokusiranom na konfiguraciju i podešavanja.

Pogledajmo jednostavan primer u kome koristimo `express.Router()` za definisanje ruta pod putanjom `/users`.

Prvo, ovde je fajl za `app.js`:

```javascript
const express = require("express");
const app = express();
const port = 3000;

// Importing the user routes
const userRoutes = require("./userRoutes");

// Using the userRoutes router for /users path
app.use("/users", userRoutes);

app.listen(port, () => {
  console.log(`Server running at http://localhost:${port}`);
});
```

Evo `userRoutes.js`:

```javascript
const express = require("express");
const router = express.Router();

// GET request for /users
router.get("/", (req, res) => {
  res.send("List of users");
});

// POST request for /users
router.post("/", (req, res) => {
  res.send("Creating a new user");
});

// GET request for /users/:id
router.get("/:id", (req, res) => {
  const userId = req.params.id;
  res.send(`User details for user with id: ${userId}`);
});

module.exports = router;
```

Sada hajde da pažljivo ispitamo kako sve radi/funkcioniše.

U `app.js`, unosićemo `express` da kreiramo instancu programa Express i postavimo ga u parametar nazvan `app`.

Zatim, dohvatamo putanju za `userRoutes` iz `userRoutes.js` koristeći `app.use("/users", userRoutes)` da povežemo tu putanju. Ovo obaveštava Express da, ako zahtev počinje sa `/users`, koristi putanju za `userRoutes`.

Zatim u `userRoutes.js`, kreiramo novi ruter koristeći `express.Router()` i definišemo putanje podataka koristeći `router.get()`, `router.post()`, i `router.get("/:id")`.

Imajte na umu da su putanje podataka definisane ovde zasnovane na `/users` jer je ruter povezan tamo. Stoga:

*   `router.get("/")` → `GET /users`: Odgovor na spisak korisnika
    
*   `router.post("/")` → `POST /users`: Odgovor na poruku o kreiranju novog korisnika
    
*   `router.get("/:id")` → `GET /users/:id`: Odgovor na podatke o korisniku koristeći dostavljen ID korisnika

Korišćenjem `express.Router()`, glavni fajl ostaje čist i uredno konfigurisan. Sve putanje povezane sa korisnicima su objedinjene u `userRoutes.js`.

Kako se vaša aplikacija razvija, možete kreirati druge rute za druge dijelove, kao što su `/products`, `/orders` ili `/auth`.

Svaka komponenta zadržava svoju strukturu u svojoj datoteci, što čini program lak za održavanje i proširenje.
