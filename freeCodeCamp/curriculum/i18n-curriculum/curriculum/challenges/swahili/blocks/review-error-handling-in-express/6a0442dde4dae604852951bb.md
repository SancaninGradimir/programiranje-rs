## Kodovi stanja za odgovor na HTTP

- **Status kodovi** su kratka poruka broja koja obaveštava klijenta šta se dogodilo nakon slanja zahteva (na primer, 200, 404 ili 500).

### 1xx — Informacije

- **100 Nastavi** — usluga obaveštava klijenta da pošalje preostali deo zahteva.
- **101 Promena protokola** — usluga menja protokol (ovo se često vidi sa WebSockets-ima).

### 2xx — Uspeh

- **200 U redu** — zahtev je uspešan.
- **201 Kreiran** — novi resurs je uspešno kreiran.
- **204 Bez sadržaja** — zahtev je uspešan, ali nema tela odgovora (često se koristi za `DELETE`).

### 3xx — Nova smernica

- **301 Trajno premešteno** — resurs ima novi trajni URL.
- **302 Pronađeno** — resurs je privremeno u drugačijoj URL.
- **304 Nije promenjeno** — klijent može da koristi svoju keširanu verziju; ništa se nije promenilo.

### 4xx — Greške klijenta

- **400 Ombi baya** — Zahtev je loše napisan ili nije prihvatljiv.
- **401 Hauruhusiwi** — Potrebna je verifikacija identiteta.
- **403 Imekwisha ruhusiwa** — Potvrđeno je, ali nema dozvolu.
- **404 Haipatikani** — Zahtevani resurs ne postoji.

### 5xx — Greške servera

- **500 Greška na strani servera** — Opšta greška sa strane servera.
- **502 Ložauljeta (Bad Gateway)** — Klijent je primio nevažeći odgovor sa drugog servera.
- **503 Usluga nije dostupna** — Server je preopterećen ili privremeno nedostupan.

### Korišćenje status kodova u Express

- `res.status(400).send("Bad Request")` — šalje 400 i poruku čistog teksta.
- `res.status(200).json({ message: "Success" })` — šalje 200 i telo JSON.
- `res.status(500).send("Internal Server Error")` — šalje 500 i poruku.

## Rešavanje greške u Express

- **Intermediarni program za rukovanje izuzecima** ima četiri parametra: `(err, req, res, next)`. Signatura ovih četiri parametra ukazuje Express da je ovo handler (ili rukovaoc) za izuzetke.
- Postavite intermediarni program za rukovanje izuzecima **na kraju** niza intermediarnih programa, nakon svih metoda.

```javascript
app.use((err, req, res, next) => {
  console.error(err.message);
  res.status(500).send("Internal Server Error");
});
```

### Rešavanje ne-real-time grešaka u Express 4

- Asinhrone greške nisu direktno uhvaćene u Express 4.
- Umotajte asinhronu logiku u `try/catch` i pozovite `next(err)` da prosledite grešku rukovađaju grešaka.

```javascript
app.get("/user", async (req, res, next) => {
  try {
    const user = await getUserFromDatabase();
    res.send(user);
  } catch (err) {
    next(err);
  }
});
```

- Učitavanje `asyncHandler` izbegava ponavljanje `try/catch` na svakom putu:

```javascript
function asyncHandler(fn) {
  return function (req, res, next) {
    Promise.resolve(fn(req, res, next)).catch(next);
  };
}
```

### Rukovodi se sa asinhonymo greškama u Express 5

- U Express 5, asinhroni greške i odbijena obećanja se direktno šalju handleru za greške — nema potrebe za `next(err)` ručno.

### Rešavanje grešaka 404

- Dodaj globalni interceptor koji se izvršava **nakon rute, ali pre rukovaoca grešaka** za obradu neusklađenih ruta:

```javascript
app.use((req, res, next) => {
  res.status(404).send("Sorry, that route does not exist.");
});
```

### Najbolje prakse za rešavanje grešaka

- Uvek implementiraj centralni modul za rukovanje krajnjim greškama.
- Rukuj sa 404 greškom drugačije od drugih grešaka.
- U produkciji, interniši detalje o greškama — nikada ne šalji traker grešaka korisnicima.

## Ispravljanje grešaka i dodavanje zapisa u Express

- **Ugrađeno debugovanje**: Postavite varijablu okruženja `DEBUG` da biste omogućili rezultate ugrađenog debugovanja za Express.
  - `DEBUG=express:* node index.js` — uključuje sve poruke debugovanja za Express.
  - `DEBUG=express:router node index.js` — prikazuje samo rezultate debugovanja vezane za rutiranje.
  - PowerShell: `$env:DEBUG = "express:*"; node index.js`
  - CMD: `set DEBUG=express:* && node index.js`
- **`morgan`**: Popularni alat za zapisivanje logova aplikacija HTTP (putanja, URL, kod statusa, vreme odgovora).

```javascript
const morgan = require('morgan');
app.use(morgan('dev'));
```

- **Debagovanje** prati kako Express obrađuje aplikacije i middleware bez unosa produkcionih logova.
- **Logovanje** snima ono što se dešava u aplikaciji — važno je otkriti greške i razumeti saobraćaj (trafiku).

## Zdravstveni pregled i dostojanstveno sahranjivanje

### Provera zdravlja

- **Zdravstveni endpoint** izvještava da li je aplikacija dobro pokrenuta.
- Load balancers i izvršioci (npr., Kubernetes, AWS Elastic Beanstalk) pozivaju ovaj endpoint da bi utvrdili da li je usluga u dobrom stanju.
- Odgovor **200** znači dobro zdravlje; neprimljivanje odgovora uzrokuje automatski restart.

```javascript
app.get('/health', (req, res) => {
  res.status(200).send('OK');
});
```

### Zatvaranje sa poštovanjem

- **Elegantno gašenje** omogućava programu da završi postojeće zahteve pre prekida, umesto da odmah prekine konekciju.
- Slušaj signal `SIGTERM` (kada proces treba da se završi) i izvrši poziv `server.close()`:

```javascript
process.on('SIGTERM', () => {
  server.close(() => {
    console.log('Server closed. Cleaning up...');
  });
});
```

- Prati signal `SIGINT` (kada je poslat pritiskom Ctrl+C tokom lokalnog razvoja) i implementiraj istu logiku zatvaranja kao `SIGTERM`.

### Ne čuvati stanje programa pre zatvaranja

- Uklonite **503** iz provere zdravlja tokom deploymenta kako bi balanser opterećenja ne blokirao novi saobraćaj ka aplikaciji.

```javascript
let isShuttingDown = false;

app.get('/health', (req, res) => {
  if (isShuttingDown) {
    res.status(503).send('Shutting down');
  } else {
    res.status(200).send('OK');
  }
});

process.on('SIGTERM', () => {
  isShuttingDown = true;
  server.close(() => {
    console.log('Graceful shutdown complete');
  });
});
```
