# --description--

U prethodnim lekcijama, naučili smo kako da izgradimo male komponente u React kao što je ovo:

```jsx
function Greeting() {
  const developerName = "Jessica";
  return <h1>Hi {developerName}!</h1>;
}
```

Možemo da odaberemo da organizujemo ovu sekciju unutar drugog roditeljskog dela ili korenskog dela, kao što je ovo:

```jsx
function App() {
  return <Greeting />;
}

function Greeting() {
  const developerName = "Jessica";
  return <h1>Hi {developerName}!</h1>;
}
```

Iako će ovaj kod dobro funkcionisati i prikazati rezultate `Hi Jessica!` na ekranu, to nije previše dinamičan deo.

Da li bismo hteli da prikažemo različito ime kao Naomi, Tom ili Oliver? Ovde ulaze prosleđeni parametri (props) za React. Props, koji su skraćenica od *properties*, je način na koji rodenski komponenti prosleđuju podatke u dete komponentu. Props mogu biti bilo kog tipa: niz znakova (string), broj, vrednost tačno ili netačno (boolean), objekti ili nizovi podataka.

Ažurirajte naš početni primer kako bi sada prihvatio parametar `name`:

```jsx
function App() {
  return <Greeting name="Jessica" />;
}

export default App;

function Greeting(props) {
  console.log(props);
  return <h1>Hi {props.name}!</h1>;
}
```

Za deo deteta koji se zove `Greeting`, sada koristimo `props.name` umesto da direktno postavimo ime `"Jessica"`. Takođe snimamo `props` u konzoli koja prikazuje nešto.

Zatim, unutar parent komponente `App`, prosleđujemo vrednost na prop-u `name` da bi bilaći. Rezultat će biti isti na ekranu kao i ranije, ali smo sada kreirali fleksibilniju komponentu.

Sada imamo mogućnost ponovnog korišćenja detetkog dela više puta i prosleđivanja različitih imena svaki put:

```jsx
function App() {
  return (
    <>
      <Greeting name="Naomi" />
      <Greeting name="Tom" />
      <Greeting name="Jessica" />
      <Greeting name="Oliver" />
    </>
  );
}
```

Takođe možete izabrati da koristite uzorak rešenja na prosleđene parametre kako bi bilo lakše za čitanje. Evo kako možete prepisati deo `Greeting`:

```jsx
function Greeting({ name }) {
  return <h1>Hi {name}!</h1>;
}
```

Ovaj kod postiže iste rezultate, ali čini jasno koji parametar ta sekcija očekuje da primi.

React takođe pruža poseban prop pod nazivom `children`. Sav JSX koji postaviš između otvarajuće i zatvarajuće oznake komponente prosleđuje se toj komponenti kao `children`.

Na primer:

```jsx
function Card({ children }) {
  return <div className="card">{children}</div>;
}

function App() {
  return (
    <Card>
      <h2>Hello</h2>
    </Card>
  );
}
```

U ovom slučaju, JSX unutar `<Card>...</Card>` postaje prop za `children`. Rezultat prikazan na DOM će biti:

```html
<div class="card">
  <h2>Hello</h2>
</div>
```

Ovaj sistem se često koristi za dizajn sekcije, gde sekcija obuhvata druge elemente korisničkog interfejsa (UI).

Ponekad, možete imati mnogo svojstava koja morate proslediti kao prosljeđene parametre. Umjesto da ih prosleđujete jedno po jedno, možete koristiti spread operator (`...`), nakon što ih pretvorite u objekat.

Ovo je primer nove sekcije koja se zove `DeveloperCard`:

```jsx
function DeveloperCard({ name, age, country }) {
  return (
    <div className="developer-card">
      <h1>Developer: {name}</h1>
      <p>Age: {age}</p>
      <p>Country: {country}</p>
    </div>
  );
}
```

Ovaj deo od `DeveloperCard` prima tri prosleđena parametra: `name`, `age`, i `country`.

U roditeljskom delu `App`, možemo koristiti spread sintaksu za prosleđivanje svih svojstava iz objekta kao pojedinačnih argumenata u dete del:

```jsx
function App() {
  const developerObj = {
    name: "Alice",
    age: 30,
    country: "USA",
  };

  return (
    <div className="App">
      <DeveloperCard {...developerObj} />
    </div>
  );
}
```

Ovo je važno posebno kada radite sa redovima podataka i prosleđujete mnogo skupova parametara koji se predaju komponentama. Na primer, možete imati listu programera gde svaki red ima istu strukturu, ali predstavlja različitu osobu.

Naučićete više o tome kako prikazati listu u redovima/nizu u narednim lekcijama.

Korišćenje prosleđenih parametara u React čini vaše komponente dinamičnijim i ponovo upotrebljivijim, omogućavajući vam da izgradite složeniji UI. Međutim, važno je napomenuti da prosleđeni parametri ne mogu se menjati, što znači da se ne mogu modifikovati nakon renderovanja komponente. Ako morate da rukujete unosom korisnika i menjate podatke, trebalo bi da koristite `state` umesto toga. Naučićete više o upravljanju stanjem (state) u narednim lekcijama.
