# --description--

Prikazivanje liste je osnovna vežba u web aplikacijama React, i koristi se za prikaz podataka korisnicima. U React, metoda `map` se koristi za konverziju niza podataka u niz JSX elemenata koji se mogu prikazati na UI-u.

Ovo je primer sekcije nazvane `FruitList` koja prikazuje listu voća:

```jsx
function FruitList() {
  const fruits = ['Apple', 'Banana', 'Cherry', 'Date'];
  return (
    <ul>
      {fruits.map(fruit => <li>{fruit}</li>)}
    </ul>
  );
}
```

U ovom primeru, funkcija `map` iterira kroz svaki element u nizu `fruits`. Za svako voće, kreira novi element `li` sa imenom tog voća. Novi niz elemenata `li` zatim se prikazuje unutar roditeljskih etiketa (parent labels) `ul`.

Međutim, prilikom prikazivanja liste u React, važno je ne zaboraviti parametre `key` za svaki element u listi. Ključ mora uvek biti jedinstven i pomaže React da utvrdi koji su elementi promenjeni, dodati ili uklonjeni, što je važno za efikasno prikazivanje i ažuriranje liste.

Ako zaboravite ključ, React će prikazati upozorenje u konzoli, ali neće baciti veliku grešku. Program i dalje može da prikaže i funkcioniše, ali možete naići na male neusklađenosti, posebno kada se lista menja. Ove greške mogu biti teške za debagovanje jer interfejs (UI) može izgledati normalno na početku.

Uobičajena praksa je korišćenje modela niza kao key-a, kao što sledi:

```jsx
{fruits.map((fruit, index) => <li key={index}>{fruit}</li>)}
```

Iako je ovo upozorenje, generalno se smatra neadekvatnom metodom. Korišćenje svojstva kao ključa može izazvati probleme kada se lista ponovo sortira, sortiraju ili filtriraju. React koristi ključ za praćenje elemenata. Ako se redosled liste promeni, React može pogrešno koristiti stanje komponente ili ne uspešno ažurirati DOM jer ključevi (svojstva) ostaju isti čak i ako su podaci unutar tog svojstva promenjeni.

Najbolja praksa je korištenje stabilnog i jedinstvenog identifikatora za svaki element. Ovo je obično ID iz vaše baze podataka, kao što su UUID ili ID baze podataka.

Ako podaci nemaju jedinstveni ID, možete kreirati jedan kada se podaci generišu (primer, korišćenjem `crypto.randomUUID()` ili biblioteka kao `uuid`), ili koristite kombinaciju delova koji osiguravaju jedinstvenost. Međutim, trebalo bi da izbegavate kreiranje ključeva tokom prikazivanja (primer, `key={Math.random()}`), jer će ovo uzrokovati React ponovno kreiranje elemenata DOM svaki put i resetovanje njihovog stanja.

Hajde da promenimo naš primer ili uključimo ključeve:

```jsx
function FruitList() {
  const fruits = ["Apple", "Banana", "Cherry", "Date"];
  return (
    <ul>
      {fruits.map((fruit, index) => (
        <li key={`${fruit}-${index}`}>{fruit}</li>
      ))}
    </ul>
  );
}
```

U ovom izmenjenom primeru kreiramo jedinstveni key za svaki element liste kombinovanjem naziva voća i njegovog indeksa. To obezbeđuje da svaki element liste ima različit key, što pomaže React-u da efikasno upravlja i ažurira listu kada se elementi dodaju, uklanjaju ili menjaju redosled.

React takođe vam omogućava da prikažete složenije strukture. Na primer, možete imati niz objekata koji predstavljaju korisnike, a svaki sa mnogo svojstava koja želite da prikažete:

```jsx
function UserList() {
  const users = [
    { id: "user-001-employee", name: "Alice", email: "alice@example.com" },
    { id: "user-002-employee", name: "Bob", email: "bob@example.com" },
    { id: "user-003-employee", name: "John", email: "john@example.com" },
  ];
  return (
    <div>
      {users.map((user) => (
        <div key={user.id}>
          <h3>{user.name}</h3>
          <p>{user.email}</p>
        </div>
      ))}
    </div>
  );
}
```

U ovom primeru, kreiramo složeniji JSX model za svakog korisnika, prikazujući njihovo ime i e-mail adresu. Koristimo `id` korisnika kao `key`, što je najbolja praksa.

U zaključku, prikazivanje liste u React uključuje konverziju nizova podataka u JSX elemente, obično koristeći funkciju `map`.
