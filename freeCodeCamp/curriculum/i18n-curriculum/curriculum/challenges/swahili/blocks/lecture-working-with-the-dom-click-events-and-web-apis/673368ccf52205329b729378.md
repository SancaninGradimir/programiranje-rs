# --interactive--

Saznajte o `innerText`, `textContent`, i `innerHTML`.

Ovo su osobine koje možete pronaći u JavaScript-u za dobijanje ili menjanje sadržaja HTML elementa. Iako u početku mogu izgledati slično, imaju važne razlike. Izbor odgovarajuće osobine zavisi od vaše konkretne potrebe, pa počnimo.

Tuanze na `innerText`.

`innerText` predstavlja vidljivi tekstualni sadržaj HTML elementa i njegovih potomaka. Ova osobina ne uključuje skriveni tekst niti HTML oznake, već samo prikazani tekst.

Na primer, ovde možete videti `div` element sa dva pasusa:

```html
<div id="container">
  <p>Hello, World!</p>
  <p>I'm learning JavaScript</p>
</div>
```

Ako dobijemo referencu na ovaj HTML element u našem JavaScript kodu pomoću metode `getElementById()`, možemo pristupiti njegovom svojstvu `innerText`.

:::interactive_editor

```html
<div id="container">
  <p>Hello, World!</p>
  <p>I'm learning JavaScript</p>
</div>
<script src="index.js"></script>
```

```js
const container = document.getElementById("container");
console.log(container.innerText);
```

:::

Ovo je unutrašnji sadržaj ovog elementa:

```md
Hello, World!
I'm learning JavaScript
```

Ova osobina vraća tekstualni niz koji sadrži tekst unutar elementa, uključujući tekst iz njegovih potomaka.

Treba da znate da `innerText` vraća samo tekst koji je vidljiv u trenutku kada se vrednost traži. Ako je potomak skriven, njegov tekst neće biti vidljiv.

Evo primera gde je drugi pasus sakriven:

:::interactive_editor

```html
<div id="container">
  <p>Hello, World!</p>
  <p hidden>I'm learning JavaScript</p>
</div>
```

:::

Ako pokušamo ponovo da ispišemo `innerText`, rezultat sada neće sadržati tekst drugog pasusa:

:::interactive_editor

```html
<div id="container">
  <p>Hello, World!</p>
  <p hidden>I'm learning JavaScript</p>
</div>
<script src="index.js"></script>
```

```js
const container = document.getElementById("container");
console.log(container.innerText);
```

:::

Možete umetnuti `innerText` elementa HTML ovako, ali ovo će modifikovati postojeći tekst i dodati funkciju prekoračivanja linije (`br`) za svaki prekid linije:

:::interactive_editor

```html
<div id="container">
  <p>Hello, World!</p>
  <p>I'm learning JavaScript</p>
</div>
<script src="index.js"></script>
```

```js
const container = document.getElementById("container");
container.innerText = "JavaScript is awesome!";
```

:::

Pošto ``innerText`` se bavi vizuelnim aspektima, dobijanje njegove vrednosti izaziva proces koji se naziva "reflow", koji ponovo računa pozicije određenih elemenata na veb stranici. Ovaj proces može biti težak za računar, stoga ga treba izbegavati ako je to moguće.

Dobro. Sada hajde da pričamo o `textContent`.

`textContent` Obnavlja standardni tekstualni sadržaj elementa, uključujući sav tekst unutar njegovih potomaka.

Ključna razlika između `innerText` i `textContent` je u tome što `textContent` uvek vraća sav tekstualni sadržaj elementa sa HTML i njegove potomke, bez obzira na to da li su vidljivi ili skriveni.

Ovde imamo isti primer u HTML:

```html
<div id="container">
  <p>Hello, World!</p>
  <p>I'm learning JavaScript</p>
</div>
```

Ako pokušate da dobijete ovaj atribut, videćete tekst elementa i njegovog potomka kao rezultate, čuvajući doslednost i poziciju:

:::interactive_editor

```html
<div id="container">
  <p>Hello, World!</p>
  <p>I'm learning JavaScript</p>
</div>
<script src="index.js"></script>
```

```js
const container = document.getElementById("container");
console.log(container.textContent);
```

:::

Ako element HTML nije vidljiv, kao što možete da vidite ovde gde smo sakrili drugi pasus, njegov tekst će i dalje biti uključen u ovaj atribut:

:::interactive_editor

```html
<div id="container">
  <p>Hello, World!</p>
  <p hidden>I'm learning JavaScript</p>
</div>
<script src="index.js"></script>
```

```js
const container = document.getElementById("container");
console.log(container.textContent);
```

:::
  
`textContent` takođe uključuje sadržaj komponenti kao `script` i `style`.

Ako pokušate da promenite vrednost `textContent` na čvoru, on će pronaći sve svoje potomke čvorove i zameni ih jednim tekstualnim čvorom sa novom sekvencijom:

:::interactive_editor

```html
<div id="container">
  <p>Hello, World!</p>
  <p hidden>I'm learning JavaScript</p>
</div>
<script src="index.js"></script>
```

```js
const container = document.getElementById("container");
container.textContent = "New content";
```

:::

Na hatimaye, tuzungumzie jinsi `textContent` na `innerText` zinavyotofautiana na `innerHTML`.

Zapamtite da za `innerHTML` možete umetnuti sadržaj HTML unutar elementa. Ovo je pomoć pri unosu novog HTML u DOM putem promena.

Međutim, imajte na umu da ovo predstavlja bezbednosni rizik ako nemate kontrolu nad nizom, kao što je niz koji sadrži podatke koje je uneo korisnik. Ako su ti podaci zlonamerni, mogu izazvati ozbiljne bezbednosne probleme.

Da biste ovo izbegli, preporučuje se korišćenje osobine `textContent` za ubacivanje običnog teksta.

Osobine `innerText`, `textContent` i `innerHTML` u JavaScript-u pružaju različite načine za pristupanje i upravljanje sadržajem HTML elemenata. Trebalo bi da razumete razlike između ovih osobina ako želite efikasno da radite sa HTML sadržajem u JavaScript-u.
