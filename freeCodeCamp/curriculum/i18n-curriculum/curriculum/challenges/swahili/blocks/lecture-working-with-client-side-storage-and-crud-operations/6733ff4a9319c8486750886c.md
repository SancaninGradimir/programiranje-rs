# --description--

API za Web Storage pruža način da pretraživači skladište parove ključ-vrednost unutar pregledača, omogućavajući programerima da sačuvaju podatke koji se mogu koristiti prilikom učitavanja različitih stranica i sesija.

Dva glavna dela Web Storage API-ja su `localStorage` i `sessionStorage`.

`localStorage` je deo API, odnosno Online skladište (Cloud Storage), koje omogućava podacima da ostanu trajni čak i nakon zatvaranja prozora pretraživača ili osvežavanja stranice. Ovi podaci ostaju dostupni sve dok ih eksplicitno ne obriše program ili korisnik.

`sessionStorage` je drugi deo Web Storage API-ja koji čuva podatke privremeno tokom sesije stranice, što znači da su podaci dostupni sve dok je kartica ili prozor pregledača otvoren. Međutim, za razliku od `localStorage`-a, podaci u `sessionStorage`-u se brišu kada se kartica ili prozor zatvori. Više o `sessionStorage`-u naučićete u narednoj lekciji.

Uobičajena upotreba `localStorage` uključuje čuvanje korisničkih podešavanja, kao što su tema ili jezičke preferencije, pamćenje podataka formulara tokom sesija u pretraživaču i skladištenje malih delova informacija radi poboljšanja performansi web aplikacija.

Keširanje (caching) se odnosi na skladištenje podataka koji se često pristplažu u privremeni prostor za skladištenje, poznat kao keš (cache), kako bi sledeći zahtevi za te podatke mogli biti ispunjeni brže bez potrebe za ponovnim izračunavanjem ili dohvaćanjem sa sporog izvora podataka, kao što su baza podataka ili eksterni server.

Neke od najčešćih metoda za `localStorage` uključuju metode `setItem`, `getItem`, `removeItem` i `clear`.

Evo primera korišćenja metode `setItem()` koja čuva par ključ-vrednost u `localStorage`.

```js
localStorage.setItem('username', 'JaneDoe');
```

Ako želimo da preuzmemo vrednost određenog ključa iz `localStorage`, možemo koristiti metodu `getItem()`.

```js
let username = localStorage.getItem('username');
console.log(username); // JaneDoe
```

Da dohvatite element iz `localStorage` koristeći njegov ključ, možete koristiti metodu `removeItem()`.

```js
localStorage.removeItem('username');
```

Da biste obrisali sve podatke iz `localStorage`-a, možete koristiti metodu `clear()`.

```js
localStorage.clear();
```

Sada, razmotrimo primer gde koristimo `localStorage` za čuvanje teme koju je korisnik preporučio.

```js
// Store the user's theme preference
localStorage.setItem('theme', 'dark');

// Retrieve the stored theme preference
const userTheme = localStorage.getItem('theme');
console.log(userTheme); // 'dark'

// Remove the theme preference
localStorage.removeItem('theme');

// Clear all localStorage data
localStorage.clear();
```

U ovom primeru:

- Prvo sačuvavamo izbor teme (`dark`) za korisnika.

- Zatim dobijamo tu scenu i prikazujemo je na konzoli.

- Na kraju, pokazaćemo kako da uklonite specifičan element ili obrišete sve sačuvane podatke.

`localStorage` je veoma važno za čuvanje malih segmenata podataka koji moraju da traju između sesija, ali je važno znati da `localStorage` ne bi trebalo koristiti za skladištenje osetljivih informacija, kao što su lozinke, jer može uzrokovati sigurnosne rizike.
