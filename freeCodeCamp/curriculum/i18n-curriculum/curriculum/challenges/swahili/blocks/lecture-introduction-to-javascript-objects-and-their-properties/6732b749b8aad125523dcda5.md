# --interactive--

U JavaScript, funkcije i metode su oba načina za obmotavanje ponovno upotrebljivog koda, ali imaju značajne razlike u načinu definisanja, korišćenja i kontekstu njihovog rada. Razumevanje ovih razlika je važno za pisanje koda JavaScript koji je efikasan i dobro strukturisan.

Kao što ste naučili u prethodnim modulima, funkcije su ponovo upotrebljivi delovi koda koji obavljaju određene zadatke:

:::interactive_editor

```js
function greet(name) {
    return "Hello, " + name + "!";
}
console.log(greet("Alice")); // "Hello, Alice!"
```

:::

Metode objekata, s druge strane, su funkcije povezane sa objektom. Definisane su kao atribute objekta i mogu dohvatiti i izvršiti podatke tog objekta. Evo primera objekta sa metodama:

:::interactive_editor

```js
const person = {
    name: "Bob",
    age: 30,
    sayHello: function() {
        return "Hello, my name is " + this.name;
    }
};

console.log(person.sayHello()); // "Hello, my name is Bob"
```

:::

U ovom primeru, `sayHello` je metoda objekta `person`. Ključna reč `this` omogućava metodi `sayHello` da dobije parametre objekta nazvanog `person`. Naučićete više o ključnoj reči `this` u narednim časovima.

Razlika između funkcija i metoda je način na koji se koriste. Funkcije se pozivaju pomoću svog imena, dok se metode pozivaju korišćenjem zapisa sa tačkom na objektu kojem pripadaju. Na primer, funkciju `greet` pozivamo kao `greet("Alice")`, dok metodu `sayHello` pozivamo kao `person.sayHello()`.

Druga važna razlika je kontekst njihovog rada. Obične funkcije imaju svoj opseg, ali nemaju ugrađenu referencu na određeni objekat. Metode su, međutim, povezane sa svojim objektom i mogu pristupiti njegovim svojstvima i drugim metodama koristeći ključnu reč `this`.

Važno je zapamtiti da metode pomažu u organizovanju koda unutar logičkih objekata, dok se funkcije koriste za opštiji kod koji se može ponovo koristiti.

Razumevanje razlike između funkcija i metoda objekata je važno dok napredujete u učenju JavaScript-a. Iako u početku mogu izgledati slično, prepoznavanje kada koristiti svaku od njih pomoći će vam da pišete bolje organizovan, efikasniji i kvalitetniji kod.
