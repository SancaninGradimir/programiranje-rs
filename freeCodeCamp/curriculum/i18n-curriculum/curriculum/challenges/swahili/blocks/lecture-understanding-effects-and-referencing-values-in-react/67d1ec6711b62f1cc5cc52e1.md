# --description--

U React, efekat je bilo šta što se događa izvan procesa prikazivanja komponente. To jest, bilo šta što React ne održava automatski kao deo prikaza UI-ja.

Tipični primeri uključuju dobijanje podataka, ažuriranje naslova kartice pretraživača, čitanje ili pisanje u lokalno skladište pregledača, dobijanje lokacije korisnika i mnoge druge stvari. Ove operacije interaguju sa spoljnim svetom i poznate su kao nuspojave (side effects).

React uklanja funkciju od `useEffect` ili omogućava rukovanje tim nuspojavama. `useEffect` omogućio je pokretanje funkcije nakon što je sekcija prikazana ili ažurirana.

Istražite kako funkcionalnost `useEffect` radi i zašto je važna za moderno razvijanje React.

Za korištenje funkcije `useEffect`, prvo je potrebno dovesti (import):

```js
import { useEffect } from "react";
```

Zatim ga koristiš kao funkciju, ovako:

```js
useEffect(() => {
  // Your side effect logic (usually a function) here
}, [dependencies]);
```

Funkcija efekta se izvršava nakon što je sekcija prikazana, dok opcioni argument `dependencies` određuje kada će se taj efekat izvršiti.

Imajte na umu da `dependencies` može biti niz "zavisnih vrednosti" (state, props, functions, variables, itd.), prazan niz ili potpuno izostavljen. Evo kako sve te opcije kontrolišu kako `useEffect` radi:

- Ako je `dependencies` red koji sadrži jednu ili više zavisnih vrednosti, efekat će se izvršiti svaki put kada se promene.

- Ako je `dependencies` prazan red, `useEffect` će biti izvršen samo jednom kada se sekcija prikaže prvi put.

Ako izostavite `dependencies`, efekat će se pokretati svaki put kada se komponenta prikaže ili ažurira.

Na primer, u ovoj aplikaciji ``Counter``, ne prosleđujemo argument ``dependencies``, stoga će se efekat izvršiti kada je sekcija prikazana i svaki put kada se ažurira:

```jsx
import { useState, useEffect } from "react";

const Counter = () => {
  const [count, setCount] = useState(0);

  useEffect(() => {
    console.log("Component renders");
  });

  return (
    <div
      style={{
        display: "flex",
        alignItems: "center",
        flexDirection: "column",
      }}
    >
      <h2>{count}</h2>
      <div>
        <button onClick={() => setCount(count + 1)}>Increase</button>
        <button onClick={() => setCount(count - 1)}>Decrease</button>
      </div>
    </div>
  );
};

export default Counter;
```

Ali ako prosledimo prazan red kao zavisnost, efekat će biti pokrenut samo prilikom prvog prikazivanja:

```js
useEffect(() => {
  console.log('Component renders');
}, []);
```

Ako prolazite kroz stanje `count` kao zavisnost, efekat će se izvršiti kada je komponenta prvi put prikazana, i takođe kada `count` promeni:

```js
useEffect(() => {
  document.title = `The current count is ${count}`;
  console.log('component renders');
}, [count]);
```

Napomena: Ako efekat koji pokrećete traje duže od životnog ciklusa prikazivanja komponente, možda ćete želeti druga funkcija za "čišćenje" te funkcije nakon što se komponenta prikaže ili ažurira.

Na primer, ako tvoja funkcija efekta koristi `setInterval()`, postavlja slušaoca događaja kao `window.addEventListener()`, ili se povezuje sa serverom, biće ti potrebna funkcija za čišćenje da pokrene `clearInterval()`, `window.removeEventListener()`, i odjakaže se od servera, respektivno.

Ovo je sintaksa za vraćanje funkcije za čišćenje iz funkcije `useEffect`:

```js
useEffect(() => {
  // Your side effect logic here
  return () => {
    // Cleanup logic here (optional)
  };
}, [dependencies]);
```

Na primer, ako dodajete slušač događaja skrolovanja (scroll), možete ga očistiti uklanjanjem tog slušača u vašoj funkciji za čišćenje:

```js
useEffect(() => {
  const handleScroll = () => {
    // Handle scroll logic
  };
  window.addEventListener("scroll", handleScroll);

  return () => {
    window.removeEventListener("scroll", handleScroll);
  };
}, []);
```
