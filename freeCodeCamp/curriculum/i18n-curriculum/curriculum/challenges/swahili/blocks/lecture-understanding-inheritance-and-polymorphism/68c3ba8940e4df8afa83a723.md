# --description--

Nasleđivanje je glavni koncept koji sledi u objektno orijentisanom programiranju (OOP) koje ćemo naučiti.

Istražimo detaljnije ovu koncepciju i kako vam omogućava pisanje ponovno upotrebljivog koda.

Kroz nasleđivanje, podklasa može koristiti atribute i metode bazne klase (ili roditeljske klase). To vam omogućava ponovno korišćenje koda, kreiranje jasne strukture klase i prilagođavanje ponašanja bez prepisivanja celog koda. Možete se prilagoditi dodavanjem postojećih metoda ili ih preklapanjem u podklasi.

Ovo je osnovna sintaksa nasleđivanja:

```py
class Parent:
    # Parent attributes and methods

class Child(Parent):
    # Child inherits, extends, and/or overrides where necessary
```

Iako klasa `Child` nasleđuje od klase `Parent`, mora da prođe Roditeljsko na Dete.

Ovaj način se naziva direktno nasleđe, jer dete nasleđuje samo od jedne klase roditelja.

Evo primera:

```py
class Animal:
    def __init__(self, name):
        self.name = name

    def sound(self):
        return f'{self.name} makes a sound'

class Dog(Animal):
    bark = 'woof! woof!! woof!!!'

jack = Dog('Jack')
print(jack.sound())  # Jack makes a sound
print(jack.bark)  # woof! woof!! woof!!!
```

Možete videti da možemo ponovo koristiti svojstvo elementa ``self.name`` i metodu ``sound()`` iz roditeljske klase ``Animal`` u detinjarskoj klasi ``Dog``.

Promenite putanju `sound()` od roditeljske klase `Animal` u ćerkoljsku klasu `Dog` kako bismo mogli da koristimo `sound()` vrednost klase `bark`:

```py
class Animal:
    def __init__(self, name):
        self.name = name

    def sound(self):
        return f'{self.name} makes a sound.'

class Dog(Animal):
    bark = 'woof! woof!! woof!!!'

    # Override sound() to use bark class variable
    def sound(self):
        return f'{self.name} barks {self.bark}'

jack = Dog('Jack')
print(jack.sound())  # Jack barks woof! woof!! woof!!!
```

Ako želite da sačuvate vrednost return od `sound()` kada kasnije dodajete vrednost klase bark, možete dodati `sound()` koristeći funkciju `super()`:

```py
class Animal:
    def __init__(self, name):
        self.name = name

    def sound(self):
        return f'{self.name} makes a sound'

class Dog(Animal):
    bark = 'woof! woof!! woof!!!'

    # Call Animal.sound(), then append bark
    def sound(self):
        base = super().sound()
        return f'{base}, then {self.name} barks {self.bark}'

jack = Dog('Jack')
print(jack.sound())  # Jack makes a sound, then Jack barks woof! woof!! woof!!!
```

U ovom primeru, `base` je rezultat pozivanja metode `sound()` iz klase `Animal`, zatim dodajemo specifično ponašanje klase `Dog`. Na ovaj način, možete dodati funkcionalnost roditeljske klase `Animal` uz zadržavanje njenog prvobitnog ponašanja.

Takođe postoji često nasleđe, gde dete može naslediti više od jedne karakteristike roditelja.

Ovo je osnovna sintaksa za često nasleđivanje:

```py
class Parent:
    # Attributes and methods for Parent

class Child:
    # Attributes and methods for Child

class GrandChild(Parent, Child):
    # GrandChild inherits from both Parent and Child
    # GrandChild can combine or override behavior from each
```

Jednostavan način demonstracije nasleđa je često primer žabe, koja može da hoda po zemlji i pliva u vodi:

```py
class Walker:
    def walk(self):
        return 'I can walk on land'

class Swimmer:
    def swim(self):
        return 'I can swim in water'

# Amphibian inherits from both Walker and Swimmer
class Amphibian(Walker, Swimmer):
    def __init__(self, name):
        self.name = name

    def introduce(self):
        return f"I'm {self.name} the frog. {self.walk()} and {self.swim()}."

frog = Amphibian('Freddy')
print(frog.introduce())
# Output: I'm Freddy the frog. I can walk on land and I can swim in water.
```
