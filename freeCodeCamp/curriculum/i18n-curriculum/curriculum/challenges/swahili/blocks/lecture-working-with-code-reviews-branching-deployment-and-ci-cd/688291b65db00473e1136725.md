# --description--

CI znači Kontinuirana integracija, a CD znači Kontinuirana isporuka ili Kontinuirano razmenjivanje (deployment). Zajedno, ovi procesi vam omogućavaju da distribuirate promene na vašu aplikaciju automatski nakon što osigurate da te promene pravilno funkcionišu.

U procesu CI možeš pokrenuti linter i testove nad kodom. CI sistemi se obično pokreću prilikom otvaranja Pull Request-a kako bi proverili da izmene nisu pokvarile aplikaciju, kao i na grani `main` kako bi se osiguralo da je spremna za objavljivanje.

Proces CD takođe se može pokrenuti na zahtev za povlačenje promena, ako inicijalizujete početno okruženje kako biste mogli da vidite te promene direktno. Ali takođe ćete pokrenuti na vašoj grani `main` da biste direktno implementirali promene kada budu odobrene i spojene.

Ali kako ovo radi tačno sa GitHub Actions? Prvo, moramo razumeti šta su zapravo GitHub Actions. GitHub Actions obezbeđuju privremene izvršiljače (zamislite da je to privremni računar) koji vam omogućavaju da izvršite specifične komande.

Actions se definišu u YAML datotekama. Možeš imati više datoteka za različite akcije. Na primer, jednu za CI, a drugu za CD.

Istražite jednostavan primer CI:

```yml
name: Node.js CI
on:
  push:
    branches:
      - main
  pull_request:
    branches:
      - main

jobs:
  lint:
    name: Lint and Test
    runs-on: ubuntu-latest

    steps:
      - name: Checkout Source Files
        uses: actions/checkout@v4

      - name: Use Node.js v22
        uses: actions/setup-node@v4
        with:
          node-version: 22

      - name: Setup pnpm
        uses: pnpm/action-setup@v2
        with:
          version: 10

      - name: Install Dependencies
        run: pnpm install

      - name: Lint Source Files
        run: pnpm run lint

      - name: Verify Build
        run: pnpm run build

      - name: Run Tests
        run: pnpm run test
```

Ovde ima dosta toga, pa hajde da to razložimo. Pogledajmo prvi deo:

```yml
name: Node.js CI
on:
  push:
    branches:
      - main
  pull_request:
    branches:
      - main
```

`name` objašnjava ime same akcije. Ovo se vidi na mestima kao deo provere stanja PR-a. Atributi `on` objašnjavaju ono što se naziva "okidači radnog toka" - ovo su događaji koji će pokrenuti izvršavanje vaše akcije.

U ovom primeru akcija se pokreće kada se commit-ovi pošalju na `main` i kada je Pull Request usmeren ka grani `main`. To znači da se CI izvršava kada neko otvori Pull Request, a zatim ponovo nakon spajanja.

Odeljak `jobs` je mesto gde definišeš šta tvoja akcija zapravo radi. Ključ `lint:` je opciono interno ime posla. Svojstvo `name` određuje javni naziv tog posla. Svojstvo `runs-on` određuje okruženje u kojem će se akcija izvršavati – u ovom slučaju koristi se najnovija podržana verzija Ubuntu-a.

Svojstvo `steps` je mesto gde se dešava sva logika. Ovaj odeljak definiše konkretne korake koje akcija treba da izvrši. Pogledajmo prvi deo:

```yml
      - name: Checkout Source Files
        uses: actions/checkout@v4
```

Ovaj korak ima dve karakteristike, `name` koje bi korak trebalo da ima, i svojstvo `uses`. Ovo svojstvo `uses` je posebno jer upućuje izvršiocu akcije da dobije javnu akciju od GitHub i pokrene, umesto da izvršava komandu. Akcija `actions/checkout` rukuje kopiranjem i pripremom baze podataka unutar vašeg izvršioca.

```yml
      - name: Use Node.js v22
        uses: actions/setup-node@v4
        with:
          node-version: 22

      - name: Setup pnpm
        uses: pnpm/action-setup@v2
        with:
          version: 10
```

Sledeće dve komponente takođe koriste akcije drugih ljudi. Ove korake pripremaju Node i pnpm menadžer paketa. Sve ima ključ `with` koji možete koristiti za prosleđivanje argumenata tim drugim akcijama. Razmislite o ovome kao o prosleđivanju argumenata funkciji.

Poslednji delovi su mesto gde se zaista dešavaju čuda.

```yml
      - name: Install Dependencies
        run: pnpm install

      - name: Lint Source Files
        run: pnpm run lint

      - name: Verify Build
        run: pnpm run build

      - name: Run Tests
        run: pnpm run test
```

Imamo deo `Install Dependencies` koji instalira npm pakete. Naš deo `Lint Source Files` pokreće linter, deo `Verify Build` osigurava da kod TypeScript može biti kompajliran, i deo `Run Tests` pokreće jedinice testove.

Jedno važno što treba znati o akcijama je da, po podrazumevanju, ako jedan od ovih koraka ne uspe, preostali će proći i ceo proces će biti označen kao uspešan.

Gde postavljate vaše fajlove YAML da biste mogli da izvršite ove akcije? Trebalo bi da budu postavljeni u direktorijum `.github/workflows` unutar vaše baze podataka kako bi GitHub pristupio i koristio.

Postoji još mnogo toga što actions mogu da urade, i podstičem te da pročitaš dokumentaciju i istražiš postojeće actions u projektima kao što je baza podataka freeCodeCamp. Ali za sada, treba da imaš dobro razumevanje osnova GitHub Actions i CI/CD!
