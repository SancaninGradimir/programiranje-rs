## Izmena baze podataka

Ovo je proces strukturiranja relacione baze podataka radi smanjenja redundancije podataka i poboljšanja integriteta.

Prednosti uključuju:

- Smanjenje redundantnih podataka, čime se štedi skladišni prostor i smanjuje inkonsistentnost.
- Osiguravanje integriteta podataka korišćenjem primarnih i stranih ključeva.
- Činjenjem baza podataka lakim za održavanje i razumevanje.

### Standardne forme

- **1NF (Prva Normalna Forma)**
  - Svaka ćelija ima jednu vrednost (atomsku).
  - Svaki zapis je jedinstven (osigurano primarnim ključem).
  - Redosled redova/kolona nije važan.
  - Primer: Premestite više brojeva telefona iz tabele `students` u drugu tabelu `student_phones`.

- **2NF (Druga normalna forma)**
  - Ispunjava uslove 1NF.
  - Nema **delimične zavisnosti**: svaki neključni atribut mora zavisiti od punog primarnog ključa sa više komponenti.
  - Primer: Podelite tabelu `orders` u `order_header` i `order_items` kako biste izbegli da atributi zavise samo od dela ključa.

- **3NF (Treća normalna forma)**
  - Ispunjava zahteve 2NF.
  - Nema **tranzitivne zavisnosti**: atributi koji nisu ključni ne smeju zavisiti od drugih atributa koji nisu ključni.
  - Primer: Premestite `city_postal_code` u tabelu `cities` umesto da ga čuvate za svaku porudžbinu.

- **BCNF (Boyce-Codd normalna forma)**
  - Ispunjava uslove 3NF.
  - Svaki determinant (leva strana funkcionalne zavisnosti) mora biti primarni ključ.

**Napomena**: Ciljajte na 3NF u višestrukoj strukturi radi dobrog balansa između integriteta i performansi.

## Glavni koncepti za SQL

- SQL je skraćenica za Structured Query Language, jezik za komunikaciju sa relacijskim bazama podataka.
- **Osnovne komande** → `SELECT`, `INSERT`, `UPDATE`, `DELETE`, `CREATE TABLE`, `ALTER TABLE`, itd.
- **Joins** → Kombinuje podatke iz više tabela (`INNER JOIN`, `LEFT JOIN`, `RIGHT JOIN`, `FULL JOIN`).

## Izvršavanje komandi za SQL u Bash-u

Možete izvršiti komande za SQL direktno sa komandne linije koristeći klijenta interfejsa komandne linije (CLI) za `psql` za PostgreSQL ili slične alate za druge baze podataka.

Na primer, izvršavanje fajla SQL u PostgreSQL-u:

```bash
psql -U username -d database_name -c "SELECT * FROM students;"
```

Takođe možete izvršiti MySQL komande direktno:

```bash
mysql -u username -p database_name -e "SELECT * FROM students;"
```

### Pokretanje SQL iz datoteke

```bash
# PostgreSQL
psql -U username -d database_name -f script.sql

# MySQL
mysql -u username -p database_name < script.sql
```

### Ubacivanje SQL u Bash skriptu

```bash
#!/bin/bash
DB_USER="school_admin"
DB_NAME="school"

# Insert student data
psql -U "$DB_USER" -d "$DB_NAME" -c \
"INSERT INTO students (name, age, major) VALUES ('Alice', 20, 'CS');"
```

### Upotreba parametara u SQL

```bash
#!/bin/bash
DB_USER="school_admin"
DB_NAME="school"
STUDENT_NAME="Bob"
AGE=21

psql -U "$DB_USER" -d "$DB_NAME" -c \
"INSERT INTO students (name, age) VALUES ('$STUDENT_NAME', $AGE);"
```

**Napomena**: Očistite parametre kako biste izbegli injekciju SQL.

## Dobijanje i korišćenje izlaza komandi za SQL u Bash-u

Kada izvršavate komande za SQL putem `psql`, možete **uhvatiti** i **obrađivati** vrednosti koje vraćaju u vašim Bash skriptama.

### Održavanje jedne vrednosti

```bash
#!/bin/bash
DB_USER="school_admin"
DB_NAME="school"

# Get total student count
STUDENT_COUNT=$(psql -U "$DB_USER" -d "$DB_NAME" -t -A -c \
"SELECT COUNT(*) FROM students;")

echo "Total students: $STUDENT_COUNT"
```

Rezultati → 42

### Dobijanje mnogih kolona

```bash
#!/bin/bash
DB_USER="school_admin"
DB_NAME="school"

# Get top 3 students' names and ages
RESULTS=$(psql -U "$DB_USER" -d "$DB_NAME" -t -A -F"," -c \
"SELECT name, age FROM students LIMIT 3;")

echo "Top 3 students:"
echo "$RESULTS"
```

Rezultati

```bash
Alice,20
Bob,21
Charlie,22
```

### Prolazak kroz ciklus na osnovu rezultata argumenata

```bash
#!/bin/bash
DB_USER="school_admin"
DB_NAME="school"

# Get student names and majors
psql -U "$DB_USER" -d "$DB_NAME" -t -A -F"," -c \
"SELECT name, major FROM students;" | while IFS="," read -r name major
do
    echo "Student: $name | Major: $major"
done
```

Struktura rezultata

```bash
Student: Alice | Major: CS
Student: Bob   | Major: Math
Student: Carol | Major: Physics
```

## Šiva za SQL

To je ranjivost sajberbezbednosti gde hakeri ubacuju zlonamerni kod SQL u polja za unos kako bi kontrolisali bazu podataka.

Ovo može izazvati opasne radnje kao što su:

- Zaobilazak verifikacije identiteta.
- Ukradba osetljivih podataka.
- Izmena ili brisanje zapisa.

Primer napada iglom SQL:

```sql
SELECT * FROM users WHERE username = ' " " OR "1"="1" -- ' AND password = 'anything';
```

Ovaj argument će vratiti sve korisnike jer je uslov `OR "1"="1"` uvek tačan, omogućavajući hakera da prođu proveru prijave.

### Sprečavanje injekcije SQL

1. **Koristi pripremljene upite (prepared statements):** Oni razdvajaju SQL kod od podataka i sprečavaju SQL injection. Evo primera (Node.js i pg):

    ```js
    client.query('SELECT * FROM users WHERE username = $1 AND password = $2', [username, password]);
    ```

2. **Validacija unosa**: Čistite i validirajte sve korisničke unose kako biste osigurali da odgovaraju očekivanom formatu.

3. **Princip minimalnih privilegija**: Koristite račune baze podataka sa najmanjim nivoom dozvola potrebnih za aplikaciju.

**Napomena**: Nikada ne uklanjajte administratorska prava sa aplikacionih naloga.

## Problem N+1

Problem N+1 nastaje kada aplikacija izvrši jedan upit da dobije listu stavki (N), a zatim za svaku stavku izvršava još jedan upit kako bi dobila povezane podatke, što rezultira upitom N+1.

**Zašto je loše?**

- Svaki upit povećava opterećenje mreže i obradu.
- Mnogi mali upiti su sporiji od jednog optimizovanog upita.

### Primer strukture N+1

```sql
-- 1: Get list of orders
SELECT * FROM orders LIMIT 50;

-- N: For each order, get customer
SELECT * FROM customers WHERE customer_id = ...;
```

**Rešenje**: Koristite `JOINs` ili druge operacije seta.

```sql
SELECT 
  orders.order_id,
  orders.product,
  orders.quantity,
  customers.customer_id,
  customers.name,
  customers.email,
  customers.address
FROM orders
JOIN customers 
  ON orders.customer_id = customers.customer_id
WHERE orders.order_id IN (SELECT order_id FROM orders LIMIT 50);
```

Uvek traži prilike da povežeš povezane podatke u jednu tezu/argument.
