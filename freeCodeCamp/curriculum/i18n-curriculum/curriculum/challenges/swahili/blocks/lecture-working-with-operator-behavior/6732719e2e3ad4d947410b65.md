# --interactive--

Ako napišete matematički izraz sa više operatora u JavaScript, JavaScript, kako odlučuje koji će biti evaluiran prvi? Ovde ulazi redosled prioriteta operatora. Detaljno ćemo proučiti redosled prioritetnosti operatora sa primerima u kodu, kao i šta se dešava kada operateri imaju jednak prioritet.

Redos operacija određuje redos kojim se operacije izvršavaju u matematičkom izrazu. Operatori sa višim prioritetom računaju se pre onih sa nižim prioritetom. Razmislite o redu operacija kao što je u aritmetici, gde se deljenje i množenje vrše pre sabiranja i oduzimanja.

Bez praćenja ovog pravila, osnovni proračuni bi vam dali netačan rezultat. JavaScript radi na isti način. Ima svoja pravila o tome koja operacija dolazi prva, druga i tako dalje. Na primer, pogledajte izraz ispod:

:::interactive_editor

```js
const result = 2 + 3 * 4;

console.log(result); // 14
```

:::

Ako JavaScript bi merilo ovaj izraz od leve ka desnoj strani, očekivalo bi se `2 + 3 = 5`, zatim `5 * 4 = 20`. Međutim, pošto množenje ima viši prioritet od sabiranja, JavaScript merilo deo `3 * 4` prvo, što rezultira `2 + 12 = 14`.

Ponekad želite da određeni delovi vašeg izraza budu procenjeni prvo, bez obzira na pravila prioriteta. Možete koristiti standardne zagrade, `()`, za to. Sve što je unutar standardnih zagrada procenjuje se prvo, bez obzira na šta. Hajde da učinimo da deo prethodnog primera `2 + 3` bude procenjen prvo:

:::interactive_editor

```js
const result = (2 + 3) * 4;

console.log(result); // 20
```

:::

U prethodnom primeru, obične zagrade uzrokuju da JavaScript pozove `2 + 3` prvo, zatim dodaje rezultat sa `4`. Ovo vam daje `20` umesto `14`.

Distributivna operacija takođe ima viši prioritet od dodavanja ili oduzimanja:

:::interactive_editor

```js
const result = 2 + 6 / 3;

console.log(result); // 4
```

:::

Ako JavaScript analizira ovaj izraz od leve ka desno, očekivali biste `2 + 6 = 8`, a zatim `8 / 3 = 2.67`. Međutim, pošto je razdvajanje prioritetnije od dodavanja, JavaScript prvo proverava razdvajanje: `6 / 3 = 2`, zatim dodaje `2 + 2`, i dobija rezultat `4`.

Dakle, u množenju i deljenju, ove operacije se uvek izvršavaju pre sabiranja i oduzimanja, osim ako ne koristite zagrade da promenite redosled. Šta se dešava ako operateri imaju isti prioritet? JavaScript koristi asocijativnost operatera (associativity) da bi odredio redosled računanja.

Operacionalni odnos je ono što govori JavaScript kako da primeni operator od leve ka desnoj ili od desne ka levoj. Za većinu operatera, kao što su sabiranje i množenje, odnos je od leve ka desnoj. Stoga, JavaScript izvršava ove operacije od apsolutno najlevije strane izraza do desne:

:::interactive_editor

```js
const result = 10 - 2 + 3;

console.log(result); // 11
```

:::

Prvo, `10 - 2 = 8`, pa `8 + 3 = 11`. JavaScript se kreće od leve ka desnoj strani u ovom slučaju. Neki operateri, kao što je delilac vrednosti (`=`), su u odnosu od desna ka levo. To znači da se desna strana izraza prvo procenjuje:

:::interactive_editor

```js
let a, b;
a = b = 5;

console.log(a); // 5
console.log(b); // 5
console.log(a + b); // 10
```

:::

U prethodnom primeru, JavaScript postavlja vrednost `5` na `b` prvo, zatim postavlja `b` (koji je sada `5`) na `a`.

Operacija snage takođe je u vezi sa relacijom od desna ka levom:

:::interactive_editor

```js
const result = 2 ** 3 ** 2;

console.log(result); // 512
```

:::

Prvo, JavaScript meri `3 ** 2`, što je jednako `9`, zatim meri `2 ** 9`, što je jednako `512`. Ako operater snage ima odnos od leve ka desnoj, JavaScript bi merio `2 ** 3` prvo dobijajući `8`, zatim `8 ** 2` dobijajući `64`.
