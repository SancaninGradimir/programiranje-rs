# --description--

U React, inline stilovi se koriste za primenu stilova CSS direktno na komponente React unutar vašeg JSX koda umesto da ih definišete u zasebnim fajlovima od CSS.

React koristi JavaScript objekte za inline stilove umesto uobičajenih CSS stringova. To znači da umesto pisanja stilova kao u CSS datoteci, praviš JavaScript objekat čiji su ključevi CSS svojstva u camelCase obliku, a vrednosti su CSS vrednosti kao stringovi.

Evo primera kako možete koristiti stilove unutar linije za deo `Button`:

```jsx
function Button({ buttonText }) {
  const defaultStyles = {
    backgroundColor: "#007BFF",
    color: "white",
    border: "none",
    borderRadius: "4px",
    padding: "10px 20px",
    fontSize: "16px",
    fontWeight: "bold",
    cursor: "pointer",
    transition: "background-color 0.3s ease",
  };

  return <button style={defaultStyles}>{buttonText}</button>;
}
```

U ovom primeru, definišemo stil objekat koji se zove `defaultStyles`. Zatim primenjujemo ove stilove na element dugmeta koristeći atribut `style`. React osigurava korišćenje ovih stilova na elementu kada je prikazan.

Takođe možete izabrati da prosledite nešto direktno u atribut `style`. Evo modifikovanog primera koji izgleda ovako:

```jsx
function Button({ buttonText }) {
  return (
    <button
      style={{
        backgroundColor: "#007BFF",
        color: "white",
      }}
    >
      {buttonText}
    </button>
  );
}
```

Pogledajte dvostruke vitičaste zagrade `{{}}` u atributu `style`. Spoljašnje zagrade prikazuju JavaScript izraz unutar JSX-a, dok unutrašnje zagrade definišu JavaScript objekat. Ova sintaksa omogućava da ubacite JavaScript vrednosti direktno u JSX atribute.

Ponekad želite da prenesete objekat direktno ako ima samo nekoliko svojstava, kao što je prikazano ovde. Inače, bolje je prenositi ime objekta kao u prvom primeru.

Važno je napomenuti da, iako su imena svojstava za CSS obično napisana sa kebab-case case, kao `font-size`, u inline stilovima od React, koristimo camelCase case, kao `fontSize`. Ovo je zato što stilski objekat je nešto od JavaScript, i imena sa kebab-case case nisu dozvoljena kao ključevi svojstava za JavaScript bez korišćenja navodnika.

Velika prednost inline stilova unutar React je što podržava stilove koji se menjaju u zavisnosti od stanja elementa ili prosleđenih parametara. Primer:

```jsx
function DynamicButton({ isActive }) {
  const buttonStyles = {
    backgroundColor: isActive ? "green" : "red",
    color: "white",
    padding: "10px 15px",
    border: "none",
    cursor: "pointer",
  };

  return <button style={buttonStyles}>Login</button>;
}
```

U ovom primeru, pozadinska boja dugmeta se menja u zavisnosti od svojstva `isActive`. Ovaj tip adaptivnog stilizovanja može biti moćan za kreiranje interfejsa korisnika koji su kolaborativni i prilagodljivi uređaju.

Ukratko, inline stilovi u React pružaju moćan način za korišćenje i kontrolisanje stilova direktno unutar vaših komponenti. Koriste objekte JavaScript umesto niza karaktera CSS, zahtevaju imena svojstava u 'camelCase' formatu case i mogu lako ubaciti dinamičke vrednosti. To je ključni alat u programerskom repertoaru React, posebno za kreiranje visoko prilagođenih i interaktivnih korisničkih interfejsa.
