# --interactive--

Naučio si da su klase poput opštih mapa koje možeš koristiti za kreiranje stvari.

Ove mape zahtevaju opšti način za referenciranje objekta koji je kreiran, pristupačan ili modifikovan unutar koda. Ovo je posebno slučaj kada se koristi ključna reč `this`. On se odnosi na kontekst u kojem mora biti izvršen kod. Ako se koristi unutar metode, vrednost `this` biće referenca na objekat povezan sa tom metodom. Na ovaj način, možete pristupiti svojstvima i metodama objekta unutar klase i ponovo koristiti tu metodu za različite stvari.

Na primer, ovde imamo klasu od `Dessert`. U ovom trenutku, definišemo svojstvo `hasPeanuts`.

```js
class Dessert {
  constructor(hasPeanuts) {
    this.hasPeanuts = hasPeanuts;
  }
}
```

Proverite da koristimo ključnu reč `this` unutar komponente. Zapamtite da se komponenta instancira direktno kada se kreira novi objekat klase.

Ključna reč `this` govori programu da `hasPeanuts` treba da bude svojstvo objekta koji se trenutno kreira.

Desni deo informacija o raspodeli vrednosti postavlja vrednost parametra `hasPeanuts` za svojstvo `hasPeanuts` novog objekta.

```js
this.hasPeanuts = hasPeanuts;
```

Ovo je u kontekstu komponente, ali takođe možete koristiti ključnu reč `this` unutar drugog puta.

Ovo je naš kurs od `Dessert` sa malim promenama. Sada imamo dve karakteristike: `name` i `price`.

```js
class Dessert {
  constructor(name, price) {
    this.name = name;
    this.price = price;
  }

  showPrice() {
    console.log(`The price of ${this.name} is $${this.price}.`);
  }
}
```

Takođe imamo novi način za `showPrice`. Na ovom načinu, štampamo deskriptivnu poruku na konzoli koja sadrži ime i cenu desertnog artikla.

Obrati pažnju kako koristimo ključnu reč `this` da pristupimo svojstvima objekta unutar metode `showPrice`.

Koja će biti vrednost ključne reči `this` u kontekstu ove metode? Ona će se odnositi na objekat nad kojim je metoda pozvana. U ovom slučaju kreiramo novu instancu klase `Dessert` sa imenom `"Brownie"` i cenom `$5.99`:

```js
const brownie = new Dessert("Brownie", 5.99);
```

Ako pozovemo putanju `.showPrice()` na ovom objektu:

```js
brownie.showPrice();
```

Ovo je kompletan primer:

:::interactive_editor

```js
class Dessert {
  constructor(name, price) {
    this.name = name;
    this.price = price;
  }

  showPrice() {
    console.log(`The price of ${this.name} is $${this.price}.`);
  }
}

const brownie = new Dessert("Brownie", 5.99);
brownie.showPrice();
console.log(brownie);
```

:::

Imena na oznake trenutnog objekta (`brownie`) su izmenjena u sekvencu slova.

Sve ovo je zahvaljujući ključnoj riječi `this`, koja vam daje opšti način za referenciranje nečega trenutnog unutar klase. Ovo vam omogućava da ponovo koristite tu metodu za različite stvari.

Ovo je u kontekstu načina stvari, ali ključna reč `this` takođe će imati vrednost u slobodnim funkcijama i streličnim funkcijama.

Njegova vrednost unutar slobodne funkcije obično pominje globalnu stvar (u ne-strogoj situaciji) ili `undefined` (u strogoj situaciji).

Vrednost ključne reči `this` je nešto drugačija u streličitim funkcijama. Streličite funkcije nasleđuju vrednost `this` iz okruženja u kojem su definisane. One nisu vezane za svoj kontekst `this`. To znači da vrednost `this` zavisi od konteksta u kojem je streličita funkcija definisana, a ne od mesta ili načina njenog korišćenja. To čini streličite funkcije korisnim za pozivanje, vraćanje ili očuvanje konteksta.

Međutim, zbog toga, vrednost `this` ne upućuje na trenutni objekat u putanjama objekata, pa je ovo nešto što treba da uzmete u obzir kada koristite strelice funkcije.

Isto pravilo važi i za streličaste funkcije definisane unutar drugih funkcija. One nasleđuju vrednost `this` iz svog roditeljskog opsega.

Ključna reč `this` je moćan alat za pristup i upravljanje objektima. Razumevanje njenog rada je ključno za savladavanje objektno orijentisanog programiranja u JavaScript-u.
