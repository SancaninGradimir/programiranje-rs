# --description--

U prethodnoj lekciji, naučili ste kako da radite sa aliasima tipova i videli ste primere kako se deklarišu tipovi podataka.

Kao što znate, jedan način da se označi tip nečega je korišćenje glavnog termina `type`, kao što sledi:

```ts
type ID = number | string;

type User = {
  id: ID;
  name: string;
};
```

Drug način za navođenje vrste stvari je korišćenje deklaracije `interface`, kao što sledi:

```ts
type ID = number | string;

interface User {
  id: ID;
  name: string;
}
```

`type` i `interface` su slični, ali glavna razlika je u tome što interfejsi koriste ključnu reč `extends` za proširivanje drugih interfejsa ili tipova, dok aliasi tipa `type` koriste tipove preseka (`&`) za njihovo spajanje.

Istražite nekoliko primera:

```ts
interface User {
  id: number | string;
  name: string;
}

interface Admin extends User {
  role: string;
}

const adminUser: Admin = {
  id: 101,
  name: "Alice",
  role: "superadmin"
};
```

U ovom primeru, `User` počinje sa `id` na `name`. Interfejs za `Admin` dodaje `User`, što znači da uključuje `id` i `name` i dodaje novu osobinu nazvanu `role`.

Ako želite dodati `type`, koristićete tip intersekcije (`&`) kao što sledi:

```ts
type User = {
  id: number | string;
  name: string;
};

type Admin = User & {
  role: string;
};

const adminUser: Admin = {
  id: 101,
  name: "Alice",
  role: "superadmin"
};
```

`Admin` povezuje tip `User` i novi tip objekta koji uključuje `role`. Rezultat je oblik sličan primeru interfejsa, koji je objekat sa `id`, `name`, i `role`.

Sve `type` na `interface` mogu opisati oblik objekta. Ako niste sigurni koji da koristite, zapamtite da `interface` je specifično dizajniran za oblike objekata, dok je `type` jednostavniji i može predstavljati tipove unija, primitives, i složenije kombinacije tipova.
