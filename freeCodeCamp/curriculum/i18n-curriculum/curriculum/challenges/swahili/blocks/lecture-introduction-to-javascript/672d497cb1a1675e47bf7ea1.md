# --interactive--

Katika JavaScript, varijable funkcionišu kao kontejneri za skladištenje podataka koje možete pristupiti i modifikovati u celoj svojoj aplikaciji.

Možete zamisliti varijable kao kutije koje čuvaju vrednosti. Varijablama, možete sačuvati stvari kao brojeve ili tekst i referisati ove vrednosti kad god vam zatrebaju u vašem programu.

Jedan način za deklarisanje varijable unutar JavaScript je korišćenje ključne reči `let`. Naučićete više o ključnoj reči `let` zajedno sa drugim načinima deklarisanja varijabli u budućim lekcijama.

Ovo je primer korišćenja `let` za deklarisanje varijable nazvane `age`:

```js
let age;
```

Trenutno, varijabla `age` nema dodeljenu vrednost. Ako pokušate da je koristite, vratiće `undefined`, što znači da nema vrednosti.

Evo primera.

**PAŽNJA**: `console.log()` je funkcija koja prikazuje informacije u konzoli, delu vašeg web pregledača koji se koristi za pronalaženje grešaka u kodu. Naučićete više o `console.log()` u narednim lekcijama. Takođe, oznake `//` se koriste za dodavanje komentara u vaš kod. Komentari su napomene samo za vas ili druge programere i koje se ne uzimaju u obzir kada se kod izvršava.

:::interactive_editor```js
let age;
console.log(age); // undefined
```

:::

Ili kuweka thamani kwa variable utahitaji kutumia opereta ya mgawo kama ifuatavyo:

```js
let age = 25;
```

Sada kada koristite varijablu `age`, vraća vrednost `25`.

:::interactive_editor

```js
let age = 25;
console.log(age); // 25
```

:::

Opereta ya mgawo inaonekana kama alama ya sawa (`=`) lakini haikagui usawa. Utajifunza kuhusu opereta sahihi za kukagua usawa katika mafundisho yajayo.

Operatori dodeljivanja se koriste za dodelu vrednosti varijabli. Ovaj proces dodeljivanja vrednosti varijabli poznat je kao inicijalizacija.

Jedna prednost korišćenja ključne reči ``let`` za deklarisanje variables je što možete ponovo dodeliti vrednosti njima. U programu, ponovno dodeljivanje vrednosti znači dodeljuje novu vrednost variable koja već ima vrednost.

Ovo je primer za ponovno postavljanje vrednosti varijable `age`.

:::interactive_editor

```js
let age = 25;
console.log(age); // 25
age = 30;
console.log(age); // 30
```

:::

Sada promenljiva `age` ima vrednost `30`. Proverite da ključna reč `let` nije potrebna ponovo jer je promenljiva `age` već deklarisana, pa nema potrebe da se deklarše drugi put.

Kada ponovo dodeljujete vrednost, potrebno je samo da se referišete na ime varijable. Ponovno dodeljivanje vrednosti je važno jer vam omogućava da ažurirate i promenite vrednost sačuvanu u varijabli dok vaš program radi. Dobar primer ovoga je ažuriranje poena u igri.

Imeovanje varijabli može delovati jednostavno, ali postoje neka pravila i najbolje prakse kako biste osigurali da je vaš kod čitljiv i funkcioniše dobro.

Imena vaših varijabli bi trebalo da opisuju podatke koje predstavljaju. Na primer, umesto korišćenja imena kao `x`, više opisivo ime kao `age` ili `points` čini vaš kod lakšim za razumevanje.

```js
// Bad variable names
let x = 10;
let y = "John";

// Good variable names
let age = 10;
```

Varijable u JavaScript moraju početi sa slovom, donjom crtom (`_`), ili znakom dolara (`$`). Ne mogu početi brojem.

```js
// Valid variable names
let age;
let _score;
let $total;

// Invalid variable names
let 1stPlace; // starts with a number
```

Imena promenljivih uzimaju u obzir mala i velika slova, što znači da se reč `age` sa svim malim slovima i reč `Age` sa velikim slovom `A` smatraju različitim promenljivama.

:::interactive_editor

```js
let age = 25;
let Age = 30;
console.log(age); // 25
console.log(Age); // 30
```

:::

Zato je važno pratiti konvenciju imenovanja kao što je camelCase. CamelCase je kada prva reč ima sve mala slova, a svaka sledeća reč počinje velikim slovom.

Evo primera korišćenja konvencije camelCase za varijablu:

```js
let thisIsCamelCase;
let anotherExampleVariable;
let freeCodeCampStudents;
```

Postoje određene ključne reči u JavaScript koje ne možete koristiti kao imena promenljivih, kao što su `let`, `const`, `function`, ili `return`, jer su rezervirane samim jezikom.

Takođe biste trebali izbegavati korišćenje specijalnih znakova kao što su uzvičnik (`!`) ili znak '@' (`@`) u imenima promenljivih. Bolje je da imena promenljivih budu čitljiva koristeći slova, brojeve, donje crte ili znake dolara.

Prateći ove smernice, vaš kod će biti čistiji i lakši za održavanje kako raste u složenosti.
