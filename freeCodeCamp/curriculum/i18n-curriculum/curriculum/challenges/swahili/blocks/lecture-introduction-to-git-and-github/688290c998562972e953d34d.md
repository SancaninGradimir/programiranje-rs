# --description--

Naučili ste kako da sačuvate promene i gurnete ih na vaš udaljeni repozitorijum, ali je sve bilo u vašoj glavnoj grani `main`. Ovo može da radi za mali projekat, ali kada doprinosite drugim projektima sa otvorenim izvorom, ili kada vaš projekat raste, grane su veoma važne.

Na primer, ako napravite fork repozitorijuma i sačuvate izmene na grani `main` svog forka, možete završiti sa izmenama u svom forku koje originalni repozitorijum nije prihvatio. To može znatno otežati usklađivanje vaše grane sa prihvaćenim izmenama iz originalnog repozitorijuma. Takođe, kada radite na sopstvenom projektu, prebacivanje između različitih vežbi postaje mnogo teže ako ne koristite grane. U tom slučaju morali biste ručno da pratite koje izmene pripadaju kojoj funkcionalnosti ili kojoj grešci.

Ali šta je tačno grana? Zamisli svoju granu `main` kao glavnu ulicu, gde je svaki snimak auto koji ulazi na put. Možda odlučiš da odeš da nešto proveriš, poput kreiranja nove funkcije, pa ti treba izaći sa puta na kratko vreme da vidiš.

Grana je odvojena linija razvoja od glavne istorije projekta, u kojoj možeš slobodno da radiš na novoj funkcionalnosti ili ispravci greške. Ako si zadovoljan promenama, možeš ih spojiti sa granom `main`. Ako nisi zadovoljan, možeš obrisati granu i vratiti se na `main` bez prihvatanja tih promena.

Pre nego što napraviš novu granu, proveri postojeće grane komandom `git branch`. Izlaz može izgledati ovako:

```sh
* main
```

Trenutno imamo samo jednu granu, našu podrazumevanu granu `main`. Zvezdica (`*`) pokazuje da je to grana na kojoj se trenutno nalaziš. O tome ćeš uskoro naučiti više.

Možete kreirati novu granu komandom `git branch`, navodeći argument za ime nove grane:

```sh
git branch feature
```

Ovo kreira novu granu pod nazivom `feature`. Hajde da ponovo proverimo grane pomoću komande `git branch`:

```sh
  feature
* main
```

Sada možemo da vidimo svoju novu granu, ali se i dalje nalazimo na grani `main`. Potrebno je da se prebacimo na novu granu tako što ćemo je "izabrati". Kada izaberete novu granu, Git učitava datoteke u stanje u kojem su se nalazile u tom trenutku.

Tuchague tawi letu jipya kwa:

```sh
git checkout feature
```

au

```sh
git switch feature
```

I koristite `git branch` ponovo da vidite:

```sh
* feature
  main
```

Primećujete da je čvor sada prešao na granu koju ste trenutno „izabrali“.

Često kada kreirate novu granu, želite da pređete na nju odmah. Umesto da kreirate granu, a zatim je prebacujete dve komande, možete kreirati i prebaciti granu samo jednom komandom pomoću:

```sh
git checkout -b feature
```

au

```sh
git switch -c feature
```

Sada smo kreirali i izabrali novu granu. Ova grana je kopija grane `main`, ima sav isti kod kao i `main` kada ste kreirali granu. Sada možemo dodati bilo koji kod na ovu granu bez uticaja na granu `main`. Napustili smo glavnu granu da bismo kreirali funkciju. Ako proverite status `git status`, videćete nešto slično ovome:

```sh
On branch feature
nothing to commit, working tree clean
```

Ima novi fajl `feature.md` za pokretanje ovog:

```sh
echo "This is our new feature" > feature.md
```

Sada će situacija biti ovako:

```sh
On branch feature
Untracked files:
  (use "git add <file>..." to include in what will be committed)
	feature.md

nothing added to commit but untracked files present (use "git add" to track)
```

Sledeće, je vreme da sačuvate svoju novu funkciju. Prvo, učitajte fajl u područje za pripremu:

```sh
git add feature.md
```

Zatim, sačuvajte promene:

```sh
git commit -m "my new feature"
```

Konačno, gurnite vašu granu sa `feature` do vaše baze podataka:

```sh
git push -u origin feature
```

Zastavica `-u` povezuje vašu lokalnu granu `feature` sa udaljenom granom, tako da se buduće guranje može izvršiti samo komandom `git push`.

Sledeće ćemo naučiti kako da napravimo pull request za našu novu funkcionalnost!
