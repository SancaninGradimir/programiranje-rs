# --interactive--

Element `button` se koristi za izvršavanje određene akcije kada se inicijalizuje. Ovo je primer elementa `button` sa tekstom dugmeta `Start Game`.

```html
<button>Start Game</button>
```

Drugi primeri korišćenja elementa `button` uključuju slanje forme, prikazivanje modala ili otvaranje i zatvaranje bočnog menija. Element `button` ima svojstvo `type` koje kontroliše ponašanje dugmeta prilikom inicijalizacije. Prva moguća vrednost za svojstvo `type` biće tip `button`. Evo primera korišćenja elementa `button` sa tipom `button` i tekstom `Show Alert`:

```html
<button type="button">Show Alert</button>
```

Po podrazumevanju, dugme ništa neće uraditi kada se aktivira. Međutim, možete dodati kod JavaScript da učinite dugme interaktivnim, kao što je prikazivanje upozorenja u ovom slučaju.

Kliknite na dugme `Show Alert` da biste videli upozorenje koje se pojavljuje na ekranu. Da biste interagovali sa ovim primerom, moraćete da omogućite interaktivni uređivač.

**NAPOMENA**: Ovaj interaktivni primer koristi JavaScript, ali nema potrebe da brineš o razumevanju JavaScript koda. O JavaScript-u ćeš učiti u budućim modulima.

:::interactive_editor

```html
<button type="button">Show Alert</button>
<script src="index.js"></script>
```

```js
const btn = document.querySelector("button");
btn.addEventListener("click", () => alert("You clicked on the alert button"));
```

:::

Drugu moguću vrednost za atribut `type` je vrednost `submit`. Ovde je primer korišćenja elementa `button` tipa `submit`.

:::interactive_editor

```html
<form action="">
  <label for="email">Email address:</label>
  <input type="email" id="email" name="email" />
  <button type="submit">Submit form</button>
</form>
```

:::

Unutar polja `form`, postoji polje `label` i `input` za korisnikovu e-mail adresu. Kada korisnik klikne na dugme za slanje, podaci će biti poslati na server i obrađeni. Moguća treća vrednost za atribut `type` je vrednost `reset`. Evo primera polja `form` sa dugmićima za resetovanje i slanje.

Omogući interakcijski uređivač i interaguj sa poljem za e-mail unos u prozoru prikaza, unosiv lažnu e-mail adresu. Zatim klikni na dugme za resetovanje da vidiš svoj e-mail uklonjen iz tog polja.

:::interactive_editor

```html
<form action="">
  <label for="email">Email address:</label>
  <input type="email" id="email" name="email" />
  <button type="reset">Reset form</button>
  <button type="submit">Submit form</button>
</form>
```

:::

U ovom izmenjenom primeru, elementi `label` i `input` koriste se za prikupljanje korisnikove imejl adrese. Kada korisnik klikne na dugme za resetovanje, svi uneti podaci biće obrisani. Važno je napomenuti da dugmad za resetovanje uglavnom nisu dobra ideja jer korisnici mogu slučajno obrisati svoje podatke. Takođe, previše dugmadi u obrascu može preopteretiti korisnički interfejs.

Drugi način za kreiranje dugmadi u HTML je korišćenje funkcije `input`. Funkcija `input` takođe ima svojstvo `type` sa mogućim vrednostima od `submit`, `reset` i `button`. Evo primera korišćenja funkcije `input` koja ima `type` postavljeno na `button`:

:::interactive_editor

```html
<input class="start-btn" type="button" value="Start Game" />
<script src="index.js"></script>
```

```js
document.addEventListener("DOMContentLoaded", () => {
  const btn = document.querySelector(".start-btn");
  btn.addEventListener("click", () => {
    const paraEl = document.createElement("p");
    const bodyEl = document.querySelector("body");

    bodyEl.appendChild(paraEl);
    paraEl.textContent = "The game has started!!!";
  });
});
```

:::

Svojstvo `value` se koristi za prikaz teksta dugmeta. Pa, koja je razlika između korišćenja komponenti `input` i `button`? Komponente `input` su komponente bez sadržaja, što znači da ne mogu imati čvorove dece, kao što je tekst, i mogu imati samo početnu oznaku. S druge strane, komponenta `button` nudi više slobode jer možete unutra organizovati tekst, slike i simbole.
