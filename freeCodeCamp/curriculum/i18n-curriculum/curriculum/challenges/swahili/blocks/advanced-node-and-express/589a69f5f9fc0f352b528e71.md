# --description--

Poslednji deo za postavljanje tvoje GitHub autentifikacije je da kreiraš samu strategiju. `passport-github@~1.1.0` je već dodat kao zavisnost, pa ga uvezi u svom `auth.js` fajlu kao `GitHubStrategy` ovako: `const GitHubStrategy = require('passport-github').Strategy;`. Ne zaboravi da uvezeš i podesiš `dotenv` ili koristiš tvoje environment promenljive.

Da bi inicijalizovao GitHub strategiju, moraš reći Passportu da koristi `GitHubStrategy` koju si upravo kreirao, koja prihvata 2 argumenta: objekat (sa `clientID`, `clientSecret`, i `callbackURL`) i funkciju koja se zove kada je korisnik uspešno autentifikovan, koja će odrediti da li je korisnik novi i koja polja treba unapred sačuvati u objektu korisnika u bazi podataka. Ovo je uobičajeno za mnoge strategije, ali neke mogu zahtevati dodatne informacije kako je objašnjeno u README-u te specifične GitHub strategije. Na primer, Google takođe zahteva *scope* koji određuje koji tip informacija zahteva tvoj zahtev da se vrati i traži od korisnika da odobri taj pristup.

Strategija koju implementiraš sada autentifikuje korisnike korišćenjem GitHub naloga i OAuth 2.0 tokena. Client ID i secret koji su dobijeni tokom kreiranja aplikacije se prosleđuju kao opcije pri kreiranju strategije. Strategija takođe zahteva `verify` povratni poziv, koji prima access token i opcionalni refresh token, zajedno sa `profile`-om koji uključuje profil autentifikovanog GitHub korisnika. Povratni poziv `verify` mora biti nazvan `cb` izlažeći korisnika da bi se završila autentifikacija.

Tvoja nova strategija treba sada da izgleda ovako:

```js
passport.use(new GitHubStrategy({
  clientID: process.env.GITHUB_CLIENT_ID,
  clientSecret: process.env.GITHUB_CLIENT_SECRET,
  callbackURL: /*INSERT CALLBACK URL ENTERED INTO GITHUB HERE*/
},
  function(accessToken, refreshToken, profile, cb) {
    console.log(profile);
    //Database logic here with callback containing your user object
  }
));
```

Tvoja autentifikacija još neće uspeti, i zapravo će ti dati besmislenu grešku o bazi podataka i povratnom pozivu, ali treba da pišeš tvoj GitHub profil u konzoli dok pokušavaš!

Pošalji svoju stranicu kada vidiš da si uradio dobro. Ako naiđeš na greške, možeš <a href="https://forum.freecodecamp.org/t/advanced-node-and-express/567135#implementation-of-social-authentication-ii-4" target="_blank" rel="noopener noreferrer nofollow">pogledati projekat urađen do sada</a>.
