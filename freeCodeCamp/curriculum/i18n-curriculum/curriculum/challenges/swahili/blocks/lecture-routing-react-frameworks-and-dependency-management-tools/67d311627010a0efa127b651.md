# --description--

U početnoj obuci, naučili ste da je React jednopage aplikacija. Jednopage aplikacije su programi koji uključuju jednu datoteku od HTML i koristeći JavaScript da ažuriraju bilo koji sadržaj na stranici putem direktnih promena.

Pa, šta se dešava kada želite dodati mnogo "stranica" u vašu aplikaciju za React? Kako ćete prelistati te različite komentare/sekcije?

Tamo React Router ulazi.

React Router je biblioteka treće strane koja vam omogućava dodavanje rute podataka u vaše aplikacije za React. Da biste počeli, moraćete instalirati React Router u postojeći projekat React kao što sledi:

```sh
npm i react-router
```

Kada pregledate fajl la `package.json`, videćete da `react-router` je dodato u listu zavisnosti:

```json
"dependencies": {
  "react": "^18.3.1",
  "react-dom": "^18.3.1",
  "react-router": "^7.2.0"
}
```

Zatim unutar tvog fajla sa `main.jsx` ili `index.jsx`, biće ti potrebno import `BrowserRouter` i da prikažeš `BrowserRouter` oko tvoje sekcije `App`:

```jsx
import { StrictMode } from "react";
import { createRoot } from "react-dom/client";
import { BrowserRouter } from "react-router";
import App from "./App.jsx";

import "./index.css";

createRoot(document.getElementById("root")).render(
  <StrictMode>
    <BrowserRouter>
      <App />
    </BrowserRouter>
  </StrictMode>
);
```

Da biste omogućili putanje podataka u vašoj aplikaciji, moraćete da ažurirate svoju deklaraciju `import` kako biste uključili delove `Routes` i `Route`, kao što je prikazano:

```js
import { BrowserRouter, Routes, Route } from "react-router";
```

Zatim unutar `BrowserRouter`, dodajte sekcije za `Routes` i `Route`:

```jsx
createRoot(document.getElementById("root")).render(
  <StrictMode>
    <BrowserRouter>
      <Routes>
        <Route path="/" element={<App />} />
      </Routes>
    </BrowserRouter>
  </StrictMode>
);
```

`path` i `element` se koriste za povezivanje URL sa UI sekcijom. U ovom slučaju, konfigurišemo rutu početne stranice koja je predstavljena sekcijom `App`.

Nije uobičajeno u velikim programima imati mnogo komentara i strukturirane putanje podataka kao ove:

```jsx
<Routes>
  <Route index element={<Home />} />
  <Route path="about" element={<About />} />

  <Route path="products">
    <Route index element={<ProductsHome />} />
    <Route path=":category" element={<Category />} />
    <Route path=":category/:productId" element={<ProductDetail />} />
    <Route path="trending" element={<Trending />} />
  </Route>
</Routes>
```

Parametri za `index` u ovim primerima namenjeni su da predstavljaju podrazumevanu putanju za određeni deo putanje. Stoga deo `Home` će biti prikazan na korenskoj putanji (`/`) kada je deo `ProductsHome` prikazan na putanji `products`.

Takođe ste možda primetili da organizujemo nekoliko puteva unutar drugih puteva, kao što sledi:

```jsx
<Route path="products">
  <Route path="trending" element={<Trending />} />
</Route>
```

Ovo znači da put za manji put će biti spojen sa putem glavnog puta. U ovom primeru, `path` za omiljene proizvode biće `products/trending`.

Kama `path` počinje sa dve tačke (`:`), to znači da predstavlja deo modifikacije u putanji podataka:

```jsx
<Route path=":category" element={<Category />} />
```

U ovom primeru imamo sekciju izmene nazvanu `category`. Kada korisnik navigira na URL kao `products/brass-instruments`, onda će komentar promeniti u sekciju od `Category` i možete dobiti odgovarajuće podatke putem izmene.

Možete dobiti vrednost dela promene korišćenjem hook-a `useParams` u manjem delu kao što je sledeće:

```jsx
import { useParams } from "react-router";

export default function Category() {
  let params = useParams();
   {/* Accessing the category param: params.category */}
   {/* rest of code goes here */}
}
```

Dinamičke putanje su korisne jer vam omogućavaju kreiranje dinamičkih i ponovo upotrebljivih sekcija koje prikazuju različit sadržaj u zavisnosti od parametara koji se nalaze u URL. Umesto definisanja statične liste putanja za svaki mogući način, možete koristiti dinamičke sekcije za prikaz raznolikog sadržaja u zavisnosti od zahteva korisnika.
