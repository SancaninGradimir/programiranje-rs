# --description--

**Prioritetna cola** je vrsta apstraktnog tipa podataka (ADT) koja funkcioniše kao cola ili stos, ali sa jednom važnom razlikom.

Kao što ste možda već upoznati, standardni redovi prate model FIFO (First In, First Out), gde je prvi element dodan u red, to i prvi koji se uklanja iz reda.

Stak prati model LIFO (Poslednje ulazak, prvo izlazak), gde se element koji je poslednji dodan u stak, prvi uklanja iz tog staka.

Oni razmatraju samo redosled unosa elemenata.

Međutim, redovi prioriteta uzimaju u obzir „prioritet“ elemenata. Prioritet se koristi za određivanje koji element treba da bude obrađen sledeći/kasnije.

Obično se prvo isporučuje funkcionalnost sa visokim prioritetom, ali druga implementacija takođe može da izabere da prvo isporuči funkcionalnost sa niskim prioritetom. To će zavisiti od zahteva vaše aplikacije.

Prioritetni redovi pružaju veliku pomoć u stvarnim primenama, kao što su pronalaženje najkraćeg puta između dve lokacije, planiranje zadataka u operativnim sistemima, simulacija saobraćaja, kompresija podataka i upravljanje mrežama.

U praksi, prioritetne cola se obično implementiraju korišćenjem strukture podataka niza.

**Rundo** je struktura podataka drveta sa veoma specifičnom svojstvom nazvanom **svojstvo Runda**. Ovo svojstvo određuje odnos između svakog čvora i njegovih podčvorova, u zavisnosti od tipa Runda.

Postoje dve glavne vrste runda:

* Veliki kup (max-heap)
    
* Mali kup (min-heap)

U **velikom krugu**, vrednost svakog čvora je veća ili jednaka vrednosti njegovih dečijih čvorova.

U ovom primeru možeš da vidiš strukturu stabla sa čvorovima 8, 7, 5, 2 i 1. Primeti da je čvor 7 veći od čvorova 2 i 1, u skladu sa svojstvom hipa. Isto važi i za sve ostale čvorove.

Svojstvo heap-a je važno jer osigurava da najviši (ili najniži) element (u zavisnosti od tipa heap-a) uvek ostaje na vrhu, što značajno pojednostavljuje uklanjanje tog elementa.

U praksi, heap je obično implementiran kao niz podataka kako bi se efikasno pristupilo roditeljskim i detilji čvorovima.

Korišćenje nizova podataka pojednostavljuje logiku pristupa ovim vrednostima ili "čvorovima" jer, iza scene, ako struktura nastavlja da ima oblik potpunog binarnog stabla, implementacija niza zahteva samo jednostavne aritmetičke operacije na osnovu njihovih parametara kako bi pronašla gde su elementi u memoriji.

JavaScript Nema ugrađenog modula za stos, ali možete implementirati manji stos koristeći niz podataka.

Ovdje je osnovna implementacija malog paketa u JavaScript:

```js
class MinHeap {
  constructor(compare = (a, b) => a - b) {
    this.data = [];
    this.compare = compare;
  }

  peek() {
    return this.data[0];
  }

  push(value) {
    this.data.push(value);
    this.#bubbleUp(this.data.length - 1);
  }

  pop() {
    if (this.data.length === 0) return undefined;

    const top = this.data[0];
    const last = this.data.pop();

    if (this.data.length > 0) {
      this.data[0] = last;
      this.#bubbleDown(0);
    }

    return top;
  }

  pushPop(value) {
    if (this.data.length === 0) return value;

    if (this.compare(this.data[0], value) < 0) {
      const top = this.data[0];
      this.data[0] = value;
      this.#bubbleDown(0);
      return top;
    }

    return value;
  }

  heapify(arr) {
    this.data = arr.slice();
    for (let i = Math.floor(this.data.length / 2) - 1; i >= 0; i--) {
      this.#bubbleDown(i);
    }
  }

  #bubbleUp(i) {
    while (i > 0) {
      const p = Math.floor((i - 1) / 2);
      if (this.compare(this.data[i], this.data[p]) >= 0) break;
      [this.data[i], this.data[p]] = [this.data[p], this.data[i]];
      i = p;
    }
  }

  #bubbleDown(i) {
    const n = this.data.length;

    while (true) {
      let smallest = i;
      const l = 2 * i + 1;
      const r = 2 * i + 2;

      if (l < n && this.compare(this.data[l], this.data[smallest]) < 0) smallest = l;
      if (r < n && this.compare(this.data[r], this.data[smallest]) < 0) smallest = r;

      if (smallest === i) break;

      [this.data[i], this.data[smallest]] = [this.data[smallest], this.data[i]];
      i = smallest;
    }
  }
}
````

Da biste koristili ovaj stos, možete kreirati prazan stos. Ovo će biti osnovna struktura podataka za stos:

```js
const myHeap = new MinHeap();
```

Da bi dodao elemente u hip, pozivaš `push()`. On će automatski postaviti element na odgovarajuće mesto kako bi se očuvalo svojstvo hipa:

```js
myHeap.push(9);
```

Da biste dobili element sa najvišim prioritetom (u ovom slučaju, najmanja vrednost), pozivate `pop()`:

```js
myHeap.pop();
```

`pushPop()` Kombinuje obje operacije u jedan poziv.

Ovo je bolje od pozivanja u različitim serijama, posebno kada je skup veliki, jer izvršava samo jednu operaciju ponovnog uređenja:

```js
myHeap.pushPop(15);
```

Ako već imate red podataka i želite ga pretvoriti u stub, možete pozvati `heapify()`:

```js
myHeap.heapify([9, 2, 7, 1]);
```

Aliako trenutno, planiramo elemente po njihovoj vrijednosti, zar ne?

Da li bismo želeli da ih organizujemo po „prioritetu“ umesto toga?

Možete ovo da uradite tako što ćete sačuvati redove podataka sa ovom strukturom: `[priority, element]`.

U JavaScript-u se nizovi ne upoređuju automatski element po element prilikom određivanja prioriteta, zato hipu prosleđuješ funkciju za poređenje koja prvo upoređuje prioritet.

Molimo zapamtite da, u ovom slučaju, manji brojevi će predstavljati viši prioritet. To znači da će element sa prioritetom 1 imati **viši prioritet** od elementa sa prioritetom 3:

```js
const myHeap = new MinHeap((a, b) => a[0] - b[0]);

myHeap.push([3, "A"]);
myHeap.push([2, "B"]);
myHeap.push([1, "C"]);
```

Ako vam je potrebno da elementi sa jednakim prioritetom budu prikazani u definisanom poretku, možete razmotriti uključivanje jedinstvenog identifikatora kao sekundarnog atributa vašeg elementa kako biste rešili veze (break ties), kao što je ovo `[priority, counter, element]`.

Na primer, možete uporediti prvo po prioritetu, a zatim po primeru:

```js
let counter = 0;
const myHeap = new MinHeap((a, b) => (a[0] - b[0]) || (a[1] - b[1]));

myHeap.push([3, counter++, "A"]);
myHeap.push([2, counter++, "B"]);
myHeap.push([2, counter++, "C"]);
```

Sada hajde da pričamo o efikasnosti gomile.

Prosečna i najgora slučaj vremenska složenost za umetanje i brisanje malih ili velikih vrednosti iz skupa (zavisno od tipa skupa) je logaritamska, `O(log n)`, jer je broj operacija često jednak veličini skupa, što je log(n).

Prosečno i najgore vreme za operaciju „progleda“ je konstantno, O(1). Pregled podrazumeva pronalaženje najmanje ili najveće vrednosti (u zavisnosti od tipa strukture) bez njenog uklanjanja.

Operacija "heapify", gde se heap gradi iz neuređenog niza, ima linearnu vremensku složenost, O(n), u proseku i u najgorem slučaju.

Slično tome, pretraživanje i brisanje bilo kog elementa imaju linearnu složenost vremena u proseku i najgoru složenost od O(n), jer mogu zahtevati proći kroz ceo niz.

Šta im je potrebno mesta?

Kompleksnost prostora niza je linearna, O(n), gde je `n` broj elemenata koji sadrži. Zahteva skladištenje elemenata i mali dodatni režijski utrag za sam objekat niza podataka.

Prioritetne kolige i pila su veoma važne u računarskoj nauci. Omogućavaju vam da brzo dohvatite i koristite najvažnije elemente iz liste. Ova efikasnost je važna za mnoge računarske programe koji obavljaju kritične zadatke u stvarnom svetu, kao što je pronalaženje najbržeg puta na mapi.
