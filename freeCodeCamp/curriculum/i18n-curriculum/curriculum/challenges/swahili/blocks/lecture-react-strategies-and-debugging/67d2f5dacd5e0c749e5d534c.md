# --description--

Pregledači imaju ugrađene alate za programere koje možeš da koristiš za otklanjanje grešaka u HTML-u, CSS-u i JavaScript-u.

Međutim, nije baš dobro za pronalaženje i popravljanje grešaka u aplikacijama za React. Stoga, tim React je kreirao alat nazvan "React Developer Tools" (poznat i kao React DevTools) kako bi mogao da pregleda, pronađe greške i testira performanse aplikacija za React.

React DevTools je dostupan kao ekstenzija za pretraživač za Chrome, Edge i Firefox. Ako koristite Chrome ili Edge, idite na Chrome Web Store, potražite "React Developer Tools", i dodajte ga u svoj pretraživač.

I ako koristiš Firefox, idi na stranicu Dodatci za Firefox, potraži taj alat i dodaj ga u svoj pregledač.

Ako koristiš Safari, možeš da instaliraš React DevTools iz npm-a pokretanjem `npm install -g react-devtools` ili `yarn global add react-devtools`.

Nakon konfigurisanja i omogućavanja `React DevTools`, kada otvorite aplikaciju `React` u svom pretraživaču, zatim otvorite alate za developera vašeg pretraživača, pronaći ćete dve dodatne kartice: Elementi i Profil.

Sekcijski tab prikazuje svaku sekciju u strukturi drveta. Stoga, možete:

- Kutazama mti wa sehemu za programu.
- Pregled i izmena prosleđenih prop-ova, stanja i vrednosti konteksta u realnom vremenu.
- Pregled izvornog koda svake izabrane komponente.
- Kurekodi data ya sehemu kwenye konsoli.
- Ispitivanje DOM elemenata komponente.

S druge strane, kartica Profila pomaže vam da snimite i analizirate performanse sekcija kako biste identifikovali nepotrebno prikazivanje, videli vreme posvećenosti i stvari koje možete poboljšati.

Evo jednostavne aplikacije koja pokazuje kako možeš da ispituješ komponente, njihove prop-ove i stanje. Ovo je isti kod koji smo koristili u prethodnoj lekciji o prosleđivanju prop-ova:

```jsx
import { useState } from "react";

export default function App() {
  const greeting = "Hello, Prop Drilling!";
  const response = "I'm not here to play!";

  return <Parent greeting={greeting} response={response} />;
}

const Parent = ({ greeting, response }) => {
  return <Child greeting={greeting} response={response} />;
};

const Child = ({ greeting, response }) => {
  return <Grandchild greeting={greeting} response={response} />;
};

const Grandchild = ({ greeting, response }) => {
  const [count, setCount] = useState(0);

  return (
    <>
      <h1>{greeting}</h1>
      <h2>{response}</h2>

      <button onClick={() => setCount(count + 1)}>Increase Count</button>
      <h2>Count: {count}</h2>
      <button onClick={() => setCount(count - 1)}>Decrease Count</button>
    </>
  );
};
```

Ako pogledaš karticu Components u React DevTools-u, videćeš hijerarhijsko stablo komponenti. Komponenta `App` nalazi se na vrhu, a ispod nje su komponente `Parent`, `Child` i `Grandchild`.

Ako izabereš bilo koju od ovih komponenti, videćeš njene prop-ove i stanje. Ako izabereš komponentu `Parent`, videćeš prop-ove `greeting` i `response`, čije su vrednosti `Hello, Prop Drilling!` i `I'm not here to play!`.

Možete videti prosleđene parametre i ažurirati status u realnom vremenu, pa ga promeniti ako je potrebno. Na primer, možete izabrati deo `Grandchild` i promeniti parametar `greeting` sa `Hello, Prop Drilling!` na `Hello, Welcome to Prop Drilling!`, i videti te promene odmah na stranici.

Da biste snimili podatke u sekciji na konzoli, pregledajte odgovarajuće elemente DOM i pogledajte izvorni kod te sekcije. Ikone u gornjem desnom uglu omogućavaju da to uradite. Ako izaberete sekciju `Grandchild` i kliknete na dugme `Log the component data to the console`, snimiće prebačene parametre, stanje (state), hookove, čvorove (nodes) i druge podatke u konzolu.

Česta greška koju možete naići u React naziva se neusklađenost prosleđujućih parametara.

Na primer, recimo da u delu `Child`, slučajno si prosledio `reply` kao parametar umesto `response`:

```jsx
const Child = ({ greeting, response }) => {
  return <Grandchild greeting={greeting} reply={response} />;
};
```

Zapamtite da `Grandchild` očekuje parametar od `response`. Jerativo, jer deo dobija različit parametar, ne može prikazati taj tekst na stranici, i umesto toga dodaje `h2` prazno u DOM. Umesto toga, videćete samo element sa `h1` i tekstom `Hello, Prop Drilling!`, zajedno sa dugmići i drugim tekstovima koji su već dostupni na stranici. Element `h2` je i dalje tu, ali zato što je prazan, ne možete ga videti bez inspekcionisanja DOM.

Da biste ovo ispravili, možete pregledati proces prosleđivanja parametara iz sekcije `Parent` do `Child` i direktno urediti ime parametra. Kada stignete do kartice Sekcija, izaberite sekciju `Child` i promenite parametar od `reply` u `response`, videćete element `h2` na stranici sa tekstom `I'm not here to play!`.
