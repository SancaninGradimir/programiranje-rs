# --description--

U ovoj lekciji naučićete kako da radite sa tipovima funkcija. Počnimo sa objašnjenjem tipova parametara.

Ovde je primer osnovne funkcije:

```ts
function circleArea(radius) {
  return Math.PI * radius * radius;
}
```

Trenutno, TypeScript će prikazati poruku za `Parameter 'radius' implicitly has an 'any' type`. Tip od `any` pokazuje da vrednost može imati bilo koji tip. Naučićete više o ovom tipu u sledećoj lekciji.

Da biste uklonili grešku i eksplicitno postavili tip parametra, možete ga napisati na način kao što je ovo:

```ts
function circleArea(radius: number) {
  return Math.PI * radius * radius;
}
```

Ako pokušate da pozovete funkciju `circleArea` sa različitim nebrojnim tipom, TypeScript će prikazati grešku kao što je ovo:

```ts
function circleArea(radius: number) {
  return Math.PI * radius * radius;
}

// Argument of type 'string' is not assignable to parameter of type 'number'
circleArea("3");
```

Ako pokušate da pružite više argumenata nego što funkcija očekuje, TypeScript će prikazati grešku:

```ts
function circleArea(radius: number) {
  return Math.PI * radius * radius;
}

// Expected 1 arguments, but got 2.
circleArea(3, 4);
```

Ako pružite previše malo argumenata, onda TypeScript takođe će prikazati grešku:

```ts
function circleArea(radius: number) {
  return Math.PI * radius * radius;
}

// Expected 1 arguments, but got 0.
// An argument for 'radius' was not provided.
circleArea();
```

Možete učiniti parametre opcionim koristeći oznaku `?` pored parametra kao što je ovo:

```ts
function area(radius: number, height?: number): number {
  const baseArea = Math.PI * radius * radius;

  if (height !== undefined) {
    const sideArea = 2 * Math.PI * radius * height;
    return baseArea + sideArea;
  }

  return baseArea;
}
```

Imajte na umu da opcioni parametri uvek moraju biti posle obaveznih parametara. Postavljanje obaveznog parametra nakon opcionog izazvaće TypeScript grešku.

Provera uslova unutar funkcije postoji u slučaju da argument `height` nije prosleđen.

Ako želite da postavite tip povratne vrednosti, možete dodati anotaciju tipa povratne vrednosti ovako:

```ts
function circleArea(radius: number): number {
  return Math.PI * radius * radius;
}
```

Opis tipa povratne vrednosti piše se nakon liste parametara. Za opise tipova promenljivih, TypeScript često može sam da detektuje tip povratne vrednosti. Zato koristite jasne opise tipa povratne vrednosti kada želite da učinite namenu funkcije jasnijom ili sprečite neželjene promene vrednosti povratne vrednosti.

Sledeći primer koji ćemo pogledati imaće asinhronu funkciju i obećanje. Ako vaša funkcija vraća obećanje, možete koristiti tip kao što je `Promise`:

```ts
async function getCityTemperature(city: string): Promise<number> {
  const response = await fetch(`https://api.example.com/weather?city=${city}`);
  const data = await response.json(); 
  return data.temperature;
}
```

U ovom stanju, ako pokušate da uklonite tip povratne vrednosti, TypeScript može da detektuje povratnu vrednost kao `Promise<any>`. Zato je ovo dobro stanje za postavljanje tipa povratne vrednosti.

Sad, videli smo samo primere funkcija koje imaju imena. Ali, kako tipovi rade sa funkcijama bez imena?

Ovde je primer korišćenja anonimne funkcije u pozivu povratka koji se prosleđuje za `forEach`:

```ts
const languages = ["JavaScript", "Python", "TypeScript"];

languages.forEach((lang) => {
  console.log(`${lang.toUpperCase()} has ${lang.length} characters`);
});
```

Kigezo cha `lang` nema specifikacije tipa. Umesto toga, TypeScript koristi kontekstualnu klasifikaciju, što znači da prepoznaje tip iz konteksta u kojem se pojavljuje.
