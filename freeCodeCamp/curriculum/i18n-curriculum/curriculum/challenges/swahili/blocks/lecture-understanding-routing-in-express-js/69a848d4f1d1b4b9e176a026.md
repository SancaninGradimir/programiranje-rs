# --description--

Sada si kreirao osnovnu aplikaciju za Express; vreme je da pažljivo pregledaš rutiranje i kako funkcioniše.

U Express, rutiranje je način na koji definišete tačke završetka mrežne aplikacije. Možete zamisliti putanje podataka kao pravila koja govore programu kako da odgovori kada korisnik poseti određeni URL.

Ponovo ćemo koristiti naš jednostavan program za Express:

```javascript
const express = require("express");
const app = express();
const port = 3000;

app.get("/", (req, res) => {
  res.send("Hello World!");
});

app.listen(port, () => {
  console.log(`Example app listening on port ${port}`);
});
```

U ovom primeru, program određuje putanju korena (`/`) putem URL. Kada klijent pošalje GET zahtev na ovu putanju, server odgovara tekstom `Hello World!`. Zatim program počinje da sluša dolazne zahteve na port `3000`.

Ali postoje i drugi načini za putanju podataka.

Express omogućava obradu različitih vrsta zahteva za HTTP koristeći različite metode rutiranja. Najčešće su:

* `app.get()` – inapata data
    
* `app.post()` – inaunda data mpya
    
* `app.put()` – inasasisha data iliyopo
    
* `app.delete()` – inafuta data
    

Svaka ruta određuje kako aplikacija treba da odgovori na određeni HTTP zahtev za određenu URL putanju.

Evo nekoliko jednostavnih primera:

```javascript
app.post("/submit", (req, res) => {
  res.send("Form submitted!");
});

app.put("/update", (req, res) => {
  res.send("Item updated!");
});

app.delete("/delete", (req, res) => {
  res.send("Item deleted!");
});
```

Svaki put kada Express primi zahtev sa određenom HTTP metodom i URL putanjom, izvršava odgovarajuću funkciju kako bi poslao odgovor klijentu.

Ove rute možete testirati pomoću alata kao što su Postman ili `curl`, koji vam omogućavaju slanje različitih tipova HTTP zahteva vašem serveru.

To je rutiranje u Express - radi o određivanju kako vaša aplikacija reaguje na različite zahteve.
