## `Audio` Komponenta na standardnom putu

- **Opis**: Komponenta `Audio`, kao i druge komponente, je posebna funkcija koja se poziva ključnom rečju `new`. Vraća `HTMLAudioElement`, koji zatim možete koristiti za reprodukciju zvuka korisniku ili ga dodati u DOM kako bi korisnik mogao sam da ga pregleda. Kada pozivate komponentu, opcionalno možete proslediti URL kao argument. URL treba da predstavlja izvor audio fajla koji želite da reprodukujete. Ili, ako morate promeniti izvor na drugi način, možete postaviti vrednost URL-a u svojstvo `src` vraćenog audio elementa.
- **Metoda za `play()`**: Ova metoda se koristi od strane komponenti `audio` ili `video` za pokretanje medija.

:::interactive_editor

```html
<audio id="audio" controls>
  <source src="https://www.w3schools.com/html/horse.mp3" type="audio/mpeg">
</audio>

<button id="playBtn">Play Audio</button>

<script>
  const audio = document.getElementById("audio");
  const playBtn = document.getElementById("playBtn");

  playBtn.addEventListener("click", () => {
    audio.play();
  });
</script>
```

:::

- **Putanja za `pause()`**: Ova putanja se koristi od strane komponenti `audio` ili `video` za pauziranje reprodukcije medija. Čuva trenutnu poziciju, tako da ako se `play()` pozove, počinje od te pozicije.

:::interactive_editor

```html
<audio id="myAudio" controls>
  <source src="https://www.w3schools.com/html/horse.mp3" type="audio/mpeg">
</audio>

<br>

<button id="pauseBtn">Pause Audio</button>

<script>
  document.addEventListener("DOMContentLoaded", () => {
    const audio = document.getElementById("myAudio");
    const pauseBtn = document.getElementById("pauseBtn");

    pauseBtn.addEventListener("click", () => {
      audio.pause();
    });
  });
</script>
```

:::

- **Njia ya `addTextTrack()`**: Ova putanja vam omogućava da odredite način povezivanja teksta sa medijel objektom – što je posebno korisno za dodavanje titlova na video.
- **Njia ya `fastSeek()`**: Ova putanja vam omogućava da pomerite poziciju reprodukcije na određeni vremenski period unutar medija.

Komponenta za `Audio()` ima karakteristike kao što su putevi. Ove karakteristike uključuju:

- `currentTime`: za dobijanje trenutnog vremena reprodukcije zvuka
- `loop`: za reprodukovanje zvuka kontinuirano, automatskim ponavljanjem kada dođe do kraja
- `muted`: za utišavanje zvuka medijalnog elementa bez obzira na postavljeni nivo jačine zvuka

## Različiti tipovi za zvuk i video

- **MIME tip**: MIME tip, koji označava Multipurpose Internet Mail Extensions, je standardizovani način za prikazivanje tipa fajla programski. MIME tip može da programu, kao što je vaš pretraživač, kaže kako da rukuje određenim fajlom. U slučaju zvuka i videa, MIME tip ukazuje na medijski format koji se može ugraditi na web stranicu.
- **Atribut `source`**: Koristi se za određivanje tipa fajla i izvora – i može uključivati više različitih tipova koristeći više atributa izvora. Kada to uradite, pretraživač će odlučiti najbolji format da koristi za trenutno okruženje korisnika.
- **MP3**: Ovo je vrsta digitalnog formata fajla koji se koristi za skladištenje muzike, zvuka ili drugih audio snimaka. To je komprimovana verzija snimanja zvuka koja smanjuje veličinu fajla, čineći ga lakšim za skladištenje i deljenje. MP3 ima široku podršku u pretraživačima i MIME tip `audio/mpeg`.
- **MP4**: MP4 je vrsta digitalnog formata fajla koji se koristi za skladištenje videa i zvuka. Koristi se kao kontejner koji drži video (slike) i zvuk (muziku ili govor) u jednom fajlu. MP4 može imati MIME tip `audio/mp4` ILI `video/mp4`, zavisno od toga da li je fajl samo video ili samo audio.
- Ostali tipovi su `WMV` povezani sa Windows Media Player aplikacijom, `OGG`, `MKV`, `AVI` i mnogi drugi.

## codecs

- **Opis**: Codec, skraćenica od "encoder/decoder" (enkoder/dekoder), je algoritam ili program koji može konvertovati zvuk i video između analognog i digitalnog formata. Codeci se mogu navesti kao deo MIME tipa. Osnovna sintaksa za navođenje kodeka je dodavanje tačke nakon tipa medija, zatim `codecs=` i kodek.

## API za HTMLMediaElement

- **Opis**: API od `HTMLMediaElement` se koristi za kontrolu ponašanja audio i video elemenata na vašoj stranici. On proširuje osnovni interfejs za `HTMLElement`, tako da imate pristup osnovnim funkcijama, metodama i ovim događajima podrške kao što su `waiting`, `ended`, `canplay`, `canplaythrough`, itd. Primeri metoda uključuju `play()`, `fastSeek()`, `pause()`, i `canPlayType()` za proveru da li pretraživač može da reprodukuje audio fajl.

## API za Media Capture na Streamovima

- **Opis**: API za Media Capture na Streams, ili API za MediaStream, koristi se za snimanje zvuka i videa sa vašeg uređaja. Da biste koristili API, morate kreirati objekat od `MediaStream`. To možete učiniti komponentom, ali bi bio direktno vezan za uređaje korisnika. Umesto toga, svojstvo `mediaDevices` objekta `global` navigator ima metodu `getUserMedia()` koju možete koristiti.

```js
window.navigator.mediaDevices.getUserMedia({
  audio: true,
  video: {
    width: {
      min: 1280,
      ideal: 1920,
      max: 3840
    },
    height: {
      min: 720,
      ideal: 1080,
      max: 2160
    }
  }
});
```

## API za Screen Capture

- **Opis**: API za Screen Capture koristi se za snimanje ekrana korisnika. API je dostupan pozivanjem metode ``getDisplayMedia()`` objekta ``mediaDevices`` i korišćenjem vrađenog medijskog toka.

## API za MediaStream Recording

- **Opis**: API za MediaStream Recording radi paralelno sa API MediaStreams, što vam omogućava snimanje MediaStreama (ili čak `HTMLMediaElement` direktno).

## API od Media Source Extensions

- **mada**: API Media Source Extensions je ono što vam omogućava da direktno prenesete sliku kamere korisnika u video element sa atributom `srcObject`.

## API iz Web Audio

- **Opis**: API Web Audio upravlja svim zvukovima koji se čuju na mreži. API ovo uključuje ključne komponente kao što su `AudioBuffer` (predstavlja privremeni keš koji čuva audio podatke) ili `AudioContext`, koji se koristi za predstavljanje dijagrama obrade zvuka.
