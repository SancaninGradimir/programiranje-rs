# --description--

Sortiranje liste elemenata je uobičajena tema u računarstvu. Postoje dva glavna algoritma o kojima treba da znate za pretraživanje: linearno pretraživanje i binarno pretraživanje.

Linearna pretraga počinje od početka liste i prolazi kroz svaki element dok ne pronađe vrednost koju tražite.

Ako je vrednost koju tražite dostupna, vraćen je model koji se nalazi u spisku. Ako vrednost nije dostupna, vraća se `-1`. Vraćamo `-1` jer nije validan model u mnogim programskim jezicima.

Evo kako izgleda kod za linearnu pretragu:

```javascript
function linearSearch(arr, target) {
    for (let i = 0; i < arr.length; i++) {
        if (arr[i] === target) {
            return i;
        }
    }
    return -1;
}
```

Ako je lista koju tražimo `[13, 4, 7, 9, 10]` i vrednost koju pretražujete je `9`, funkcija će vratiti `3` jer `9` postoji u modelu `3`.

Ako promenimo vrednost koju tražite na `5`, funkcija će vratiti `-1` jer `5` nije u listi.

Iako je ovo jednostavan algoritam, nije najefikasniji. Ako imate veliku listu elemenata, linearna pretraga može potrajati dugo da pronađe vrednost koju tražite.

Vremenska složenost linearnog pretraživanja je `O(n)`, jer vreme potrebno za pretragu u nizu raste linearno u zavisnosti od veličine niza.

Teškoća linijskog pretraživačkog prostora je `O(1)` jer ne zahteva dodatni prostor za pretraživanje u listi.

Binarna pretraga je veoma efikatan algoritam za pretraživanje u velikoj listi podataka. Uslov ovde je da lista mora biti sortirana po rastućem redosledu.

Binarna pretraga radi tako što deli listu na pola i proverava da li se vrednost koju tražiš nalazi u sredini liste. Ako je vrednost u sredini liste, vraća se indeks te vrednosti. Inače, algoritam proverava da li se vrednost nalazi u levom ili desnom delu liste.

Nastavlja da deli preostale delove liste na pola dok ne pronađe željenu vrednost. Ako vrednost nije u listi, vraća `-1`.

Evo kako izgleda binarni tražilica kod:

```javascript
function binarySearch(arr, target) {
    let low = 0;
    let high = arr.length - 1;

    while (low <= high) {
        const mid = Math.floor((low + high) / 2);

        if (arr[mid] === target) {
            return mid;
        } else if (arr[mid] < target) {
            low = mid + 1;
        } else {
            high = mid - 1;
        }
    }

    return -1;
}
```

Počinje identifikovanjem obrasca `low` i `high`. Ovo predstavlja red liste koji tražimo unutar njega.

Zatim proveravamo stanje `low` da li je manje ili jednako `high`. Ako je `low` veće od `high`, pronašli smo celu listu i vrednost nije dostupna. U tom slučaju, zaustavljamo pretragu i vraćamo `-1`.

Ako je element `low` manji ili jednak elementu `high`, izračunavamo srednji element liste, `mid`. Zatim proveravamo da li se vrednost koju tražite nalazi u srednjem elementu. Ako je tu, vraćamo srednji element.

U suprotnom slučaju, proveravamo da li je vrednost u sredini manja od vrednosti koju tražite. Ako jeste, ažuriramo donji indeks na srednji indeks plus jedan. To znači da ćemo pretraživati desnu polovinu liste.

Na kraju, ako ne postoji drugi uslov je `True`, ažuriramo model `high` u model jedan položaj levo. To znači da ćemo tražiti levu polovinu liste.

Nastavićemo da ponavljamo ovaj proces sve dok ne dobijemo vrednost ili odlučimo da ta vrednost nije na listi.

Vremenska složenost binarnog pretraživanja je `O(log n)` jer vreme potrebno za pretragu u nizu raste logaritamski u odnosu na veličinu niza.

Težina (kompleksnost) binarnog pretruma je `O(1)` jer ne zahteva dodatni prostor za pretraživanje u listi.

Binarna pretraga i linearna pretraga mogu se koristiti za različite izazove koje ćete naići u računarskoj nauci. Važno je razumeti razliku između ova dva algoritma i kada koristiti svaki od njih.
