# --interactive--

Konstrukcija za strukturu niza podataka je karakteristika u JavaScript koja vam omogućava da izvučete vrednosti iz nizova podataka i dodelite ih varijablama na kraći i čitljiviji način. Pruža jednostavnu sintaksu za pristup elementima niza podataka i dodelu ih u različite varijable.

Ova tehnika je vrlo korisna kada koristite nizove podataka i funkcije koje vraćaju više vrednosti. Evo primera korišćenja rešenja strukture niza podataka:

:::interactive_editor

```js
let fruits = ["apple", "banana", "orange"];

let [first, second, third] = fruits;

console.log(first);  // "apple"
console.log(second); // "banana"
console.log(third);  // "orange"
```

:::

U ovom primeru, imamo niz podataka nazvan `fruits` sa tri elementa. Koristeći rešenje strukture niza podataka, stavljamo prvi element u parametar `first`, drugi element u `second`, i treći element u `third`. Ovo nam omogućava pristup pojedinačnim elementima niza bez korišćenja numeričkog indeksa.

Ovako bi izgledalo kada bi koristio indekse za pristup svakom elementu umesto destrukturiranja niza:

:::interactive_editor

```js
const fruits = ["apple", "banana", "orange"];

const first = fruits[0];
const second = fruits[1];
const third = fruits[2];

console.log(first); // "apple"
console.log(second); // "banana"
console.log(third); // "orange"
```

:::

Destrukturiranje niza takođe ti omogućava da preskočiš elemente koji te ne zanimaju korišćenjem zareza. Na primer:

:::interactive_editor

```js
let colors = ["red", "green", "blue", "yellow"];
let [firstColor, , thirdColor] = colors;

console.log(firstColor); // "red"
console.log(thirdColor); // "blue"
```

:::

U ovom primeru preskačemo drugi element niza `colors` pomoću dodatnog zareza. Tako se `red` dodeljuje promenljivoj `firstColor`, a `blue` promenljivoj `thirdColor`, dok se `green` u potpunosti preskače.

Još jedna moćna karakteristika dizajna niza podataka je mogućnost korišćenja podrazumevanih vrednosti. Ako niz ima manje elemenata nego parametara koje želite postaviti, možete navesti podrazumevane vrednosti:

:::interactive_editor

```js
let numbers = [1, 2];
let [a, b, c = 3] = numbers;

console.log(a); // 1
console.log(b); // 2
console.log(c); // 3
```

:::

Ovde postavljamo podrazumevanu vrednost `3` za `c` jer red `numbers` nema treći element.

Sada pogledajmo REST sintaksu, koja se prepoznaje po tri tačke (`...`). Ona ti omogućava da preostale elemente niza koji nisu raspakovani smestiš u novi niz. Evo kako funkcioniše:

:::interactive_editor

```js
let fruits = ["apple", "banana", "orange", "mango", "kiwi"];
let [first, second, ...rest] = fruits;

console.log(first);  // "apple"
console.log(second); // "banana"
console.log(rest);   // ["orange", "mango", "kiwi"]
```

:::

U ovom primeru, `first` i `second` preuzimaju prva dva elementa niza `fruits`, dok `rest` preuzima sve preostale elemente kao novi niz. REST sintaksa mora biti poslednji element u obrascu destrukturiranja.

Dizajnerski obrazac za niz podataka je moćna karakteristika koja može učiniti vaš kod kraćim i lakšim za čitanje. Posebno je važan kada koristite nizove podataka, i kada želite da izvučete specifične elemente iz niza.
