# --description--

Trenutno, ne možete znati ko je povezao sa vašom web socket. Kada `req.user` ima korisnički objekat, to je samo kada vaš korisnik interaguje sa web serverom, a za web socket-ove nemate `req` (request) i zato nemate korisničke podatke. Jedan način da rešite problem znanja ko je povezao sa vašim web socket-om je da parsirate i interpretirate cookie koji drži passport sesiju i pretvorte je u korisnički objekat. Srećom, postoji paket na NPM-u koji je specifično za ovo što čini ovu tešku vežbu lakšom!

`passport.socketio@~3.7.0`, `connect-mongo@~3.2.0`, i `cookie-parser@~1.4.5` su već dodati kao zavisnosti. Uvezi ih kao `passportSocketIo`, `MongoStore`, i `cookieParser` redom. Takođe, trebamo da inicijalizujemo novi store za sesije, iz `express-session` koje smo koristili ranije. Treba da izgleda ovako:

```js
const MongoStore = require('connect-mongo')(session);
const URI = process.env.MONGO_URI;
const store = new MongoStore({ url: URI });
```

Sada trebamo samo da kažemo Socket.IO da koristi ovo i postavi opcije. Osigurajte da je ovo dodato pre postojećeg socket koda a ne unutar postojećeg connection listener-a. Za vaš server, treba da izgleda ovako:

```js
io.use(
  passportSocketIo.authorize({
    cookieParser: cookieParser,
    key: 'express.sid',
    secret: process.env.SESSION_SECRET,
    store: store,
    success: onAuthorizeSuccess,
    fail: onAuthorizeFail
  })
);
```

Imajte na umu da konfigurisanje Passport autentifikacije za Socket.IO je veoma slično načinu na koji smo konfigurisali `session` middleware za API. To je zato što su oba dizajnirana da koriste isti metod autentifikacije - uzmi ID sesije iz cookie-a i autentifikuj.

Ranije, kada smo konfigurisali `session` middleware, nismo otkrili ime cookie-a za sesiju (`key`). To je zato što je `session` paket koristio podrazumevanu vrednost opcije. Sada smo dodali drugi paket koji zahteva pristup istoj vrednosti iz cookie-ova, trebamo da eksplicitno postavimo `key` vrednost u oba dela konfiguracije.

Osigurajte da dodajete `key` sa imenom cookie-a u `session` middleware koji odgovara Socket.IO ključu. Takođe, dodajte referencu na `store` u opcije, blizu mesta gde postavljate `saveUninitialized: true`. Ovo je ključno za saopštavanje Socket.IO-u koja sesija je povezana.

<hr>

Sada, definišite callback funkcije za `success` i `fail`:

```js
function onAuthorizeSuccess(data, accept) {
  console.log('successful connection to socket.io');

  accept(null, true);
}

function onAuthorizeFail(data, message, error, accept) {
  if (error) throw new Error(message);
  console.log('failed connection to socket.io:', message);
  accept(null, false);
}
```

Korisnički objekat je sada dostupan na vašem socket objektu kao `socket.request.user`. Na primer, sada možete da dodate sledeće:

```js
console.log('user ' + socket.request.user.username + ' connected');
```

To će pisati u konzoli servera ko je povezao!
