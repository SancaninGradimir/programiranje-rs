# --description--

U React, rendering je proces gde se delovi prikazuju u korisničkom interfejsu (UI), obično je pregledač.

React uzima sav vaš kod od JavaScript, JSX, na CSS, određuje kako bi trebalo da izgleda, a zatim ga prikazuje u korisničkom interfejsu.

Proces renderovanja ima tri koraka: trigger, render i commit. Hajde da ih analiziramo detaljnije.

Trigger akcija se dešava kada React detektuje da se nešto promenilo i da je korisnički interfejs možda potreban za ažuriranje. Ove promene često proizlaze iz ažuriranja stanja ili prosleđuju parametre.

Na primer, prepoznavanje da je vreme za večeru može dovesti do toga da uđete na kuhinju i započnete kuvanje.

U primeru od `Counter` ispod, klikatanje na dugme za povećanje ili smanjenje uzrokuje React da prikaže novu vrednost od `count`:

```jsx
import { useState } from "react";

function Counter() {
  const [count, setCount] = useState(0);

  return (
    <div>
      <h1>{count}</h1>
      <button onClick={() => setCount(count - 1)}>Decrement</button>
      <button onClick={() => setCount(count + 1)}>Increment</button>
    </div>
  );
}

export default Counter;
```

Kada se okidač pokrene, React ulazi u fazu renderovanja. Ovde React ponovo izračunava vaše komponente i odlučuje šta treba prikazati.

Da bi ovo uradili, React koristi laganu kopiju DOM "originala". Ovo se zove virtuelni DOM. Korišćenjem virtuelnog DOM, React može brzo proveriti šta treba da se promeni u sekciji.

Zamislite ovu fazu kao trenutak kada ste u kuhinji, prikupili sastojke i počinjete da kuvate večeru.

Za komponentu `Counter`, faza renderovanja je trenutak kada React ponovo poziva funkciju sa novom vrednošću `count`. React ponovo izračunava kako deo `<h1>{count}</h1>` treba da izgleda na osnovu ažurirane vrednosti `count`, ali nećete videti promene na ekranu sve do sledeće faze — commit.

Faza commit-a je trenutak kada React preuzima pripremljene promene iz virtualnog DOM-a i postavlja ih u pravi DOM. Drugim rečima, ovo je faza kada vidite konačan rezultat na ekranu.

Da bi ovo uradio, React upoređuje virtuelni DOM sa stvarnim DOM, detektuje samo delove koji zahtevaju ažuriranje, i primenjuje te promene na stvarni DOM radi ažuriranja korisničkog interfejsa.

Možete zamisliti ovaj korak kao kada stavljate kuvanu hranu na sto, čineći ga da izgleda kao što to radi React kada postavlja ažuriranje na pravi DOM.

U vezi sa dijelom `Counter`, korak commita je kada se nova vrijednost `count` postavlja u polje `h1`, i možete vidjeti promjene na stranici.

Ovaj treći proces je veoma brz jer React smanjuje direktno izvršavanje DOM tako što prvo izračunava promene u virtuelnom DOM, a zatim ažurira samo delove koji moraju da se menjaju u stvarnom DOM.
