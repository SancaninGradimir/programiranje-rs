# --description--

Kao podsetnik, ovaj projekat je napravljen na osnovu osnovnog projekta preuzetog iz <a href="https://github.com/freeCodeCamp/boilerplate-infosec/" target="_blank" rel="noopener noreferrer nofollow">GitHub</a>.

Cross-site scripting (XSS) je vrsta uobičajenog napada gde se zlonamerni kod injektuje u ranljive stranice, sa ciljem krađe osetljivih podataka kao što su kolačići sesije ili lozinke.

Osnovno pravilo za smanjenje rizika od XSS napada je jednostavno: "Nikada ne verujte korisničkom ulazu." Kao programer, uvek biste trebali sanitizovati sve spoljne ulaze. Ovo uključuje podatke iz formulara, URL iz parametara pretrage GET-a, pa čak i iz tela POST-a. Sanitizacija znači da trebate tražiti i zapisivati karaktere koji mogu biti opasni, kao što su &lt;, >.

Savremeni pretraživači mogu pomoći u smanjenju rizika korišćenjem najboljih programskih praksi. Često se ovo može konfigurisati putem HTTP zaglavlja (HTTP headers).

Kichwa cha HTTP cha X-XSS-Protection je osnovna zaštita. Pretraživači detektuju injektovane skripte koristeći heuristički filter. Ako je ovo zaglavlje omogućeno, pretraživač modifikuje kod skripte, čineći ga bezopasnim. Trenutno ima ograničenu podršku.

# --instructions--

Koristi `helmet.xssFilter()` za čišćenje unosa koji se šalje na vaš server.

# --hints--

Centralni program helmet.xssFilter() treba biti pravilno postavljen

```js
const response = await fetch(code + '/_api/app-info');
if (!response.ok) {
  throw Error(await response.text());
}
const data = await response.json();
assert.include(data.appStack, 'xXssProtection');
assert.property(data.headers, 'x-xss-protection');
```
