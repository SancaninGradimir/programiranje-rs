# --description--
  
Dinamičko programiranje je algoritamska tehnika koja rešava kompleksne probleme razbijanjem na manje probleme i skladištenjem rezultata kako bi se izbegle ponovljene proračune. Ova metoda menja probleme koji bi inače zahtevali eksponencijalno vreme u probleme rešive iz polinomske vremena.

## Glavni principi za dinamički program

Uslovni program radi na mestima gde postoji dva važna uslova u problemu.

- **Ponovljivi manji problemi**: Slični, manji problemi često se javljaju prilikom rešavanja većeg problema. Umesto da brojimo ove male probleme često, čuvamo njihovo rešenje.
    
- **Optimalna struktura problema**: Najbolje rešenje za problem ima optimalna rešenja za manje probleme. To znači da možemo izgraditi najbolje rešenje kombinovanjem najboljih rešenja manjih delova.

Istražite ove koncepte koristeći uobičajen problem „penjanja nivoa“.

## Problem korišćenja standardne rekurzije

Razmisli o problemu sa stepenicom: penješ se po stepenicama `n` i možeš da popneš 1 ili 2 stepena odjednom. Koliko različitih načina možeš stići do vrha?

```python
def climb_stairs_recursive(n):
    """Recursive approach"""
    if n <= 2:
        return n  # Base cases: 1 way for 1 step, 2 ways for 2 steps
    # To reach step n, we can come from step (n-1) or step (n-2)
    return climb_stairs_recursive(n-1) + climb_stairs_recursive(n-2)
```

Ova implementacija ima vremensku složenost transformacije jednačina zbog mnogo ponavljajućih se proračuna. Tokom računanja `climb_stairs(5)`, ovo se dešava:

- `climb_stairs(5)` poziva `climb_stairs(4)` na `climb_stairs(3)`
    
- `climb_stairs(4)` poziva `climb_stairs(3)` na `climb_stairs(2)`
    
- Sada `climb_stairs(3)` se računa **dva puta**
    
- `climb_stairs(3)` poziva `climb_stairs(2)` na `climb_stairs(1)`
    
- `climb_stairs(2)` se računa **ukupno 3 puta**

Samo sa `n=5`, izvršavamo 9 poziva funkcija kada nam je potrebno samo 5 jedinstvenih proračuna. Kako `n` raste, ponavljanje ovo prelazi granice promena jednačine - `climb_stairs(30)` bi zahtevalo milione poziva funkcija. Vremenska složenost postaje `O(2^n)`, što ga čini neefikasnim i nepraktičnim za velike vrednosti `n`.

## Rešenje za dinamični program

Dinamički program uklanja ponavljajuće proračune kroz dve glavne metode:

### Napomene (Top-down metoda)

Memorija skladišti rezultate skupih poziva funkcija i vraća sačuvane rezultate kada se ponovi isti unos.

```python
def climb_stairs_memo(n, memo={}):
    """Dynamic programming with memoization"""
    # Check if we've already calculated this value
    if n in memo:
        return memo[n]  # Return cached result - O(1) lookup!
    
    # Base cases
    if n <= 2:
        return n
    
    # Calculate once and store in memo for future use
    memo[n] = climb_stairs_memo(n-1, memo) + climb_stairs_memo(n-2, memo)
    return memo[n]
```

Memoizacija je efikasnija jer se svaka jedinstvena vrednost od `1` do `n` računa samo jednom. Kada nam treba `climb_stairs(3)` ponovo, umesto da ga recalculiramo (što bi pokrenulo više rekurzivnih poziva), mi samo tražimo u našem memorijskom rečniku za period `O(1)`.

Pratićemo izvršenje `climb_stairs(5)` korišćenjem metode odozgo nadole da bismo videli kako memorija razdvaja ponavljajuću funkciju:

```md
Call: climb_stairs_memo(5)
  memo = {} (empty)
  
  Call: climb_stairs_memo(4) 
    memo = {} (empty)
    
    Call: climb_stairs_memo(3)
      memo = {} (empty)
      
      Call: climb_stairs_memo(2) → returns 2 (base case)
      Call: climb_stairs_memo(1) → returns 1 (base case)
      
      Result: 2 + 1 = 3
      memo = {3: 3} (stored!)
    
    Call: climb_stairs_memo(2) → returns 2 (base case)
    
    Result: 3 + 2 = 5
    memo = {3: 3, 4: 5} (stored!)
  
  Call: climb_stairs_memo(3) → returns 3 (FROM MEMO - no recursion!)
  
  Result: 5 + 3 = 8
  memo = {3: 3, 4: 5, 5: 8}
```

**Poređenje efikasnosti**

- **Redovna rekurzija**: Izvršava 9 poziva funkcija i ponavljajuće proračune
    
- **Memorizacija**: Izvršava samo 5 jedinstvenih proračuna, a zatim ponovo koristi sačuvane rezultate
    
- **Vremenska složenost**: Smanjena je sa `O(2^n)` na `O(n)` jer izvršavamo samo jedinstvene proračune `n`
    
- **Prostorna složenost**: `O(n)` za skladištenje memorije i stog poziva
    
- **Stvarni efekat**: `climb_stairs(30)` se smanjuje sa miliona poziva funkcija na približno 30 jedinstvenih malih proračuna

### Tabele (Metoda od donjeg nivoa ka višem)

Tabela gradi rešenje od dna ka vrhu, popunjavajući tabelu rešenjima malih problema:

```python
def climb_stairs_tabulation(n):
    """Dynamic programming with tabulation"""
    if n <= 2:
        return n
    
    # Create array to store results for all steps from 0 to n
    dp = [0] * (n + 1)
    dp[1] = 1  # 1 way to reach step 1
    dp[2] = 2  # 2 ways to reach step 2
    
    # Build up the solution iteratively
    for i in range(3, n + 1):
        # Ways to reach step i = ways to reach (i-1) + ways to reach (i-2)
        dp[i] = dp[i-1] + dp[i-2]
    
    return dp[n]
```

Ovaj proces eliminiše rekurziju tako što gradi rešenje sekvencijalno, od najjednostavnijih problema do cilja.

Istražite metodologiju od gore ka dole praktičnom korišćenjem da vidite kako gradimo rešenje po redosledu. Ovo je sekvencijalna izgradnja za `climb_stairs(5)`:

```md
Initial state:
dp = [0, 1, 2, 0, 0, 0]
     [0, 1, 2, 3, 4, 5] ← indices (step numbers)

Step by step construction:

i = 3:
  dp[3] = dp[2] + dp[1] = 2 + 1 = 3
  dp = [0, 1, 2, 3, 0, 0]
  
i = 4:
  dp[4] = dp[3] + dp[2] = 3 + 2 = 5
  dp = [0, 1, 2, 3, 5, 0]
  
i = 5:
  dp[5] = dp[4] + dp[3] = 5 + 3 = 8
  dp = [0, 1, 2, 3, 5, 8]

Final result: dp[5] = 8
```

**Glavne prednosti tabela**

- **Nema režije rekurzije**: Za razliku od memorije, nema steka poziva rekurzije.
    
- **Predvidivo izvršavanje**: Izračunavamo vrednosti po fiksnom redosledu (1, 2, 3, 4, 5...).
    
- **Efikasan u korišćenju memorije**: Sekvencijalni pristup nizu podataka povećava efikasnost korišćenja memorije.
    
- **Lako za optimizaciju**: Može smanjiti prostornu složenost na `O(1)` jer nam trebaju samo poslednje dve vrednosti.

```python
def climb_stairs_optimized(n):
    if n <= 2:
        return n
    
    prev2, prev1 = 1, 2  # Only store last two values
    for i in range(3, n + 1):
        current = prev1 + prev2
        prev2, prev1 = prev1, current
    return prev1
```

**Poređenje efikasnosti**

- **Standardna rekursija**: 9 poziva funkcija za `n=5`, rast promena jednačine.
    
- **Tabela**: Jednostavno sabiranje od 3 za `n=5`, linearni rast.
    
- **Vremenska složenost**: `O(n)` umesto `O(2^n)`.
    
- **Prostorna složenost**: `O(n)` za niz podataka, ili `O(1)` za optimizaciju.
    
- **Predvidljiva performansa**: Nema rizika od prepunjenja steka kod velikog ulaza.

Oba metoda smanjuju vremensku složenost sa polinomialnog ``O(2^n)`` na linearno ``O(n)``, značajno poboljšavanjem koje omogućava rešavanje problema u milisekundama umesto čekanja godina za velike ulazne podatke.

## Stvarna upotreba

Dinamički program ima mnogo upotreba u računarskoj nauci i više:

- **Optimizacija putanje podataka**: GPS sistemi koriste dinamičke algoritme za pronalaženje najkraćeg puta između lokacija.
    
- **Obrada teksta**: Alati za proveru pravopisa i funkcije automatskog dovršavanja teksta često se oslanjaju na dinamičke algoritme za izračunavanje udaljenosti promena između reči.
    
- **Finansijsko modelovanje**: Investicione strategije i optimizacija portfolija često koriste dinamičke algoritme.
    
- **Alokacija resursa**: Problem rančkara (knapsack problem) i njegove varijacije javljuju se u planiranju rasporeda, budžetiranju i upravljanju resursima.

## Primer prakse: Problem razmene valute

Problem razmene valuta je uobičajni programski izazov koji, kada se reši pomoću dinamičkog programiranja, ilustruje dva glavna principa DP-a: optimalnu podstrukturu i preklapajuće podprobleme.

Problem konverzije valute pita, "Koliko minimalnih jedinica valute je potrebno da se napravi ciljani iznos?"

Ovde je jedno rešenje korišćenjem dinamičkog programa:

```python
def min_coins(amount, coins):
    """Find minimum number of coins needed to make the given amount"""
    # Initialize dp array with "infinity" - represents impossible to make
    dp = [float('inf')] * (amount + 1)
    dp[0] = 0  # Base case: 0 coins needed for amount 0
    
    # For each amount from 1 to target amount
    for i in range(1, amount + 1):
        # Try each coin denomination
        for coin in coins:
            if coin <= i:  # Can only use coin if it doesn't exceed current amount
                # Update minimum: current minimum vs (coins for remaining amount + 1)
                dp[i] = min(dp[i], dp[i - coin] + 1)
    
    # Return result if possible, -1 if impossible
    return dp[amount] if dp[amount] != float('inf') else -1

# Example usage:
# coins = [1, 3, 4], amount = 6
# dp[6] = min(dp[5]+1, dp[3]+1, dp[2]+1) = min(3+1, 1+1, 2+1) = 2
# Result: 2 coins (3 + 3)
```

I ovde je kako algoritam za konverziju valute promenljive aplikacije radi korak po korak za `coins = [1, 3, 4]`, `amount = 6`:

```md
Initial state:
dp = [0, ∞, ∞, ∞, ∞, ∞, ∞]
     [0, 1, 2, 3, 4, 5, 6] ← amounts

Building up the solution:

For amount = 1:
  Try coin 1: dp[1] = min(∞, dp[0] + 1) = min(∞, 0 + 1) = 1
  dp = [0, 1, ∞, ∞, ∞, ∞, ∞]

For amount = 2:
  Try coin 1: dp[2] = min(∞, dp[1] + 1) = min(∞, 1 + 1) = 2
  dp = [0, 1, 2, ∞, ∞, ∞, ∞]

For amount = 3:
  Try coin 1: dp[3] = min(∞, dp[2] + 1) = min(∞, 2 + 1) = 3
  Try coin 3: dp[3] = min(3, dp[0] + 1) = min(3, 0 + 1) = 1
  dp = [0, 1, 2, 1, ∞, ∞, ∞]

For amount = 4:
  Try coin 1: dp[4] = min(∞, dp[3] + 1) = min(∞, 1 + 1) = 2
  Try coin 3: dp[4] = min(2, dp[1] + 1) = min(2, 1 + 1) = 2
  Try coin 4: dp[4] = min(2, dp[0] + 1) = min(2, 0 + 1) = 1
  dp = [0, 1, 2, 1, 1, ∞, ∞]

For amount = 5:
  Try coin 1: dp[5] = min(∞, dp[4] + 1) = min(∞, 1 + 1) = 2
  Try coin 3: dp[5] = min(2, dp[2] + 1) = min(2, 2 + 1) = 2
  Try coin 4: dp[5] = min(2, dp[1] + 1) = min(2, 1 + 1) = 2
  dp = [0, 1, 2, 1, 1, 2, ∞]

For amount = 6:
  Try coin 1: dp[6] = min(∞, dp[5] + 1) = min(∞, 2 + 1) = 3
  Try coin 3: dp[6] = min(3, dp[3] + 1) = min(3, 1 + 1) = 2
  Try coin 4: dp[6] = min(2, dp[2] + 1) = min(2, 2 + 1) = 2
  dp = [0, 1, 2, 1, 1, 2, 2]

Final result: dp[6] = 2 (achieved with coins 3 + 3)
```

Ovo rešenje pokazuje dva glavna principa dinamičkog programiranja. Ima preklapajuće podprobleme jer pronalaženje najmanjeg broja novčića za iznos od 6 zahteva poznavanje rešenja za iznose 5, 3 i 2. Ovi preklapajući podproblemi javljaju se i prilikom izračunavanja drugih iznosa. Ima optimalnu strukturu problema jer najbolje rešenje za bilo koji iznos uključuje najbolja rešenja za manje iznose. Ako znamo da je najmanji broj novčića za iznos od 3 jednak 1, onda je jedan način za postizanje iznosa od 6 korišćenje tog rešenja zajedno sa dodatnim novčićem vrednosti 3.

Bez DP-a, morali bismo testirati sve kombinacije valuta – broj varijacija jednačine. Sa DP-om, gradimo rešenje sekvencijalno:

- **Vremenska složenost**: `O(amount × number of coins)` umesto transformacija jednačine.
    
- **Prostorna složenost**: `O(amount)` za niz (ili red) `dp`.
    
- **Nema ponavljanja rada**: Svaki mali problem (dobijanje nekoliko novčića za svaku količinu) rešava se samo jednom.
    
- **Ponovo upotrebljivi rezultati**: Kada znamo nekoliko novčića za iznos 3, koristimo to znanje za mnogo veći iznos koji može da profitira.

## Pošalji mi dinamički program

Dinamički programi su efikasni tamo:

- Problem se može podeliti na manje, ponavljajuće subprobleme.
    
- Problem ima optimalnu strukturu problema.
    
- Standardno rešenje rekursijom bi zahtevalo ponavljanje proračuna.
    
- Morate poboljšati vremensku složenost na štetu prostorne složenosti.

Dinamički sistemi programiranja uključuju probleme optimizacije (pronalaženje minimalne/maksimalne vrednosti), probleme brojanja (broj načina za postizanje nečega) i probleme odlučivanja koji se mogu razložiti na manje odluke.

Adaptivni program pretvara teške probleme u jednostavne tako što čuva i ponovno koristi rešenja manjih problema uzastopno. Razumevanje ove metode otvara vrata za rešavanje mnogih matematičkih izazova efikasno.
