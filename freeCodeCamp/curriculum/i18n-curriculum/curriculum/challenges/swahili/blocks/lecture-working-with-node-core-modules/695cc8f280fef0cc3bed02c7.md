# --description--

JavaScript je prvobitno napravljen za rad u veb-pregledačima, gde je njegova glavna uloga bila da veb-stranice učini interaktivnim. Zbog toga se rani JavaScript uglavnom fokusirao na obradu teksta u obrascima i izmenu Modela objekta dokumenta (DOM).

Međutim, nisu svi podaci na vebu tekst. Datoteke, slike i video-zapisi predstavljaju binarne podatke, koji zahtevaju drugačiji način obrade. U pregledačima se takvim podacima obično bave posebni delovi pregledača, a ne sam JavaScript.

Savremeni pregledači koriste motore za renderovanje i motore `JavaScript` za upravljanje ovim zadacima — na primer, Blink (sa V8) u Chrome-u, WebKit (sa JavaScriptCore) u Safariju, i Gecko (sa SpiderMonkey) u Firefox-u.

Node.js ne radi u pregledaču, pa mu je bio potreban sopstveni način rada sa binarnim podacima, naročito prilikom rada sa ulazom i izlazom datoteka (I/O) i TCP tokovima, gde podaci stižu u delovima. Tu na scenu stupa modul `Buffer`.

Modul `Buffer` u Node.js omogućava rad sa binarnim podacima kao što su datoteke, slike ili mrežni tokovi. Pomoću njega možeš da čuvaš i menjaš binarne podatke direktno u memoriji.

Kao modul za `fs`, `Buffer` je jedan od glavnih modula za Node.js, stoga ne morate ga instalirati zasebno pre korišćenja.

Da bi ga koristio, prvo uvezi modul koristeći destrukturiranje:

```js
const { Buffer } = require("buffer");
```

Zatim pozovi metode koje pruža `Buffer`. Na primer, `Buffer.from()` omogućava kreiranje bafera iz tekstualnog niza, niza podataka ili drugih sirovih podataka.

Evo kako da napraviš `Buffer` iz tekstualnog niza i niza podataka:

```js
// Create a buffer from a string
const myStrBuffer = Buffer.from("freeCodeCamp");
console.log(myStrBuffer); // <Buffer 66 72 65 65 43 6f 64 65 43 61 6d 70>

// Create a buffer from an array of numbers
const myNumBuffer = Buffer.from([
  70, 82, 69, 69, 67, 79, 68, 69, 67, 65, 77, 80,
]);

console.log(myNumBuffer); // <Buffer 46 52 45 45 43 4f 44 45 43 41 4d 50>
```

Iako je moguće koristiti neke metode modula `Buffer` bez njegovog uvoza, druge nisu dostupne dok ga izričito ne uvezeš. Zato se preporučuje da uvek uvezeš `Buffer` kada ga koristiš u svojim projektima.

Možeš pristupiti pojedinačnim elementima `Buffer` objekta kao da je niz:

```js
console.log(myNumBuffer[0]); // 70
console.log(myStrBuffer[0]); // 102
```

Takođe možete koristiti način `toString()` u privremenom skladištu da vidite kako zaista izgleda:

```js
console.log(myStrBuffer.toString()); // freeCodeCamp
console.log(myNumBuffer.toString()); // FREECODECAMP
```

`Buffer.alloc()` omogućava vam da kreirate novi kontejner određenog veličine (broj bajtova). Svaki bajt unutar njega se direktno popunjava sa `0`:

```js
const someBuffer = Buffer.alloc(10);
console.log(someBuffer); // <Buffer 00 00 00 00 00 00 00 00 00 00>
```

Vidite da je privremena keš memorija inicijalizovana na nulu, u skladu sa veličinom poslatom na putanju `alloc()`.

Možete nastaviti da koristite način `Buffer.write()` za pisanje u ovaj privremeni repozitorijum:

```js
someBuffer.write("Hello fCC");

console.log(someBuffer); // <Buffer 48 65 6c 6c 6f 20 66 43 43 00>
console.log(someBuffer.toString()); // Hello fCC
```

Ako upišete više podataka nego što skladište može da primi, biće odsečeno:

```js
someBuffer.write("Hello freeCodeCamp");

console.log(someBuffer); // <Buffer 48 65 6c 6c 6f 20 66 72 65 65>
console.log(someBuffer.toString()); // Hello free
```

Konačno, možete koristiti `Buffer.byteLength()` da pokažete broj bajtova potreban za skladištenje niza znakova za određeno kodiranje:

```js
console.log(Buffer.byteLength("Hello freeCodeCamp")); // 18
```

Drugi načini za `Buffer` uključuju:

- `Buffer.isBuffer()`: proverava da li je objekat bafer.
- `Buffer.compare()`: upoređuje dva bafera i vraća njihov redosled pri poređenju.
- `Buffer.concat()`: spaja više bafera u jedan.
