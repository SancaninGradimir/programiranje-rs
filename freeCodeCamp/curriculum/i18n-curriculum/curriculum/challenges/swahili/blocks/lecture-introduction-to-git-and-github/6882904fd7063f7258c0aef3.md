# --description--

Pre nego što pokrenete vaš repozitorijum koda (repository) na GitHub, biće vam potrebna autentifikacija. Svi SSH i GPG ključevi koriste par javnog i privatnog ključa. To znači da čuvate privatni ključ na svom lokalnom računaru, koji koristite za autentifikaciju vašeg identiteta. Javni ključ delite sa ljudima ili uslugama koje moraju da verifikuju vašu autentikaciju. Drugim rečima, potreban je privatni ključ za izvršavanje akcije, a javni ključ se koristi za verifikaciju te akcije.

GPG ključevi, ili Gnu Privacy Guard, obično se koriste za potpisivanje fajlova ili commit-ova. Osoba može koristiti vaš GPG javni ključ da potvrdi da li je potpis fajla od vašeg ključa i da sadržaj u fajlu nije izmenjen ili kompromitovan. SSH ključevi, ili Secure SHell, obično se koriste za autentifikaciju udaljene veze na server - putem alata `ssh`. Međutim, kao što ćete naučiti u ovom kursu, možete koristiti i SSH ključeve za potpisivanje commit-ova.

Za kreiranje GPG ključa, biće vam potrebno da pokrenete:

```sh
gpg --full-generate-key
```

Kwa funguo ya SSH, utaendesha:

```sh
ssh-keygen -t ed25519 -C "your_email@example.com"
```

Obavezno navedite svoju pravu adresu e-pošte.

Obe ove komande će tražiti lozinku za zaštitu ključeva. Ona nije obavezna, ali se toplo preporučuje.

Kod SSH ključeva dobićeš dve datoteke — jednu sa privatnim, a drugu sa javnim ključem. Datoteka javnog ključa završava se ekstenzijom `.pub`, kako slučajno ne bi podelio privatni ključ. Datoteke će se nalaziti u direktorijumu `~/.ssh`.

GPG ključevi se, međutim, nalaze u skrivenom i šifrovanom direktorijumu. Da prikažeš svoje javne ključeve, pokreni:

```sh
gpg --list-secret-keys --keyid-format=long
```

Zatim, da prikažeš javni ključ, koristi:

```sh
gpg --armor --export "<key id>"
```

Ali šta možeš da uradiš sa ovim novim ključevima? Već smo govorili o potpisivanju commit-ova, a za to možeš koristiti jedan od ovih ključeva.

Da bi potpisivao commit-ove GPG ključem, potrebno je da na GitHub nalog otpremiš javni, a ne privatni ključ. Zatim uzmi kratki ID ključa koji si ranije dobio i pokreni ovu komandu da ga postaviš kao Git ključ za potpisivanje:

```sh
git config --global user.signingkey <your_gpg_key_id>
```

Zatim, možete proslediti flag `-S` u vašu komandu `git commit` da potpiš specifičan commit – biće vam potrebno uneti svoju lozinku. Inače, ako želite automatski potpisati svaki commit, možete podesiti konfiguraciju za auto-potpisivanje na `true`:

```sh
git config --global commit.gpgsign true
```

Da biste potpisali sa SSH ključem, koji je relativno nova funkcija u GitHub, moraćete da počnete učitavanjem ključeva na vaš nalog/račun GitHub. Zatim ćete morati da podesite podešavanje potpisivanja za Git koristeći SSH:

```sh
git config --global gpg.format ssh
```

Zatim, prilikom postavljanja signaturskog ključa, ćete proslediti putanju fajla umesto ID:

```sh
git config --global user.signingkey <path_to_your_ssh_keys>
```

Konačno, možete omogućiti automatsko potpisivanje kao što radite sa GPG ključem.

Za jednu od ovih implementiranih metoda, sada ste spremni za commit-ove sa potvrdom koji će dobiti posebnu oznaku na GitHub.
