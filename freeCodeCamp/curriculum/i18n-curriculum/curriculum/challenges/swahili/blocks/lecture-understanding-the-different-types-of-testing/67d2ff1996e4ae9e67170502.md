# --description--

Kada dizajnirate svoje aplikacije, često ćete imati niz malih funkcija koje se bave samo jednom zadatkom. Ovo je poznato kao „princip jedne odgovornosti“ (Single Responsibility Principle). Kada imate niz malih funkcija, najbolje je testirati te funkcije kako biste osigurali da sve radi onako kako je očekivano. U ovom tutorijalu, ćemo pogledati kako kreirati unit testove koristeći popularni Jest testing framework.

U ovom primeru napravićemo funkciju koja vraća novi preuređeni niz karaktera:

```js
export function getFormattedWord(str) {
  if (!str) return '';
  return str.charAt(0).toUpperCase() + str.slice(1);
}
```

Ova funkcija `getFormattedWord` ima parametar koji se zove `str` i prvo proverava da li je `str` prazan. Ako jeste, vraća se prazan niz. U suprotnom, vraća se novi niz u kome je prvo slovo napisano velikim slovom. Izvozimo ovu funkciju kako bismo mogli da je koristimo u datoteci za testiranje.

U posebnoj datoteci `getFormattedWord.test.js` možemo napisati nekoliko testova kako bismo potvrdili da funkcija radi ono što treba.

Na vrhu datoteke `getFormattedWord.test.js`, prvo treba da uvezemo funkciju ovako:

```js
import { getFormattedWord } from "./getFormattedWord.js";
```

Zatim moramo instalirati paket Jest-a koristeći `npm i jest`. Zatim možemo dodati test da proverimo da li je reč kreirana malim slovima, `hello`, pravilno formatirana u `Hello`, koja počinje velikim slovom `H`:

```js
test('capitalizes the first letter of a word', () => {
  expect(getFormattedWord('hello')).toBe('Hello');
});
```

Funkcija za `expect` koristi se za testiranje vrednosti. Spojen je sa matcherom, koji je funkcija koja proverava da li vrednost radi kako se očekuje. U ovom slučaju, matcher je `toBe()`. Jest ima različite tipove matcher-a da bi pomogli u testiranju booleana, nizova znakova (stringova), brojeva i više.

Jedan od načina za pokretanje testova jeste dodavanje npm skripte u datoteku `package.json`. Evo primera skripte za Jest radni okvir:

```js
"scripts": {
  "test": "jest"
},
```

Zatim samo pokrenite `npm run test` u vašem terminalu da pokrenete svoje testove. Ako prođu vaši testovi, trebalo bi da vidite nešto slično ovome u terminalu:

```sh
PASS ./getFormattedWord.test.js
  ✓ capitalizes the first letter of a word (1 ms)

Test Suites: 1 passed, 1 total
Tests:       1 passed, 1 total
Snapshots:   0 total
Time:        0.103 s, estimated 1 s
Ran all test suites.
```

Da bismo videli kako test ne prolazi, možemo namerno pokvariti funkciju na sledeći način:

```js
export function getFormattedWord(str) {
  if (!str) return '';
  return "This is incorrect";
}
```

Sada kada pokrenete komandu `npm run test`, pojaviće se poruka o grešci jer je test očekivao drugačiji rezultat:

```sh
FAIL ./getFormattedWord.test.js
  ✕ capitalizes the first letter of a word (1 ms)

  ● capitalizes the first letter of a word

  expect(received).toBe(expected) // Object.is equality

  Expected: "Hello"
  Received: "This is incorrect"

    2 |
    3 | test('capitalizes the first letter of a word', () => {
    4 |   expect(getFormattedWord('hello')).toBe('Hello');
                                            ^
    5 | });
    6 |

    at Object.toBe (getFormattedWord.test.js:4:37)

Test Suites: 1 failed, 1 total
Tests:       1 failed, 1 total
Snapshots:   0 total

```

Sada možemo da ažuriramo našu funkciju i vratimo je na izvorno ovde:

```js
export function getFormattedWord(str) {
  if (!str) return '';
  return str.charAt(0).toUpperCase() + str.slice(1);
}
```

Dodajte još jednu meru za pregled slučaja gde nije dat argument za `getFormattedWord`:

```js
test("returns an empty string when no argument is provided", () => {
  expect(getFormattedWord()).toBe("");
});
```

Postoje drugi parametri koje možete dodati kako biste nešto učinili stabilnijim, ali ova prvih dva parametra predstavljaju dobar uvod u funkcionalno testiranje.

Kada je reč o testiranju aplikacija za JavaScript, neki uobičajeni okviri za testiranje uključuju Jest, Mocha i Vitest.

Funkcionalno testiranje je važno jer će vam pomoći da otkrijete više grešaka u vašem softveru, i osigurati da sve radi kako treba. Takođe se može koristiti kao vrsta dokumentacije za vaš program jer ima za cilj da pruži očekivano ponašanje vašeg koda.
