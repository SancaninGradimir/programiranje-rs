## Osnovi CSS

- **CSS je šta?**: Cascading Style Sheets (CSS) je jezik za stilove koji se koristi za primenu stila na elemente HTML. CSS se koristi za boje, pozadinske slike, rasporede i više.
- **Osnovna struktura pravila CSS**: Pravilo CSS ima dva glavna dela: selektor i blok deklaracija. Selektor je format koji se koristi u CSS za identifikovanje i ciljanje specifičnih elemenata HTML radi stilizovanja. Blok deklaracija postavlja set stila za određeni selektor, ili više selektora.
- **Element `meta name="viewport"`**: Ovaj element `meta` daje instrukcije pretraživaču kako da kontroliše dimenzije i skaliranje stranice na različitim uređajima, posebno mobilnim telefonima i tabletima.
- **Podrazumevani stilovi pretraživača**: Svaki element HTML će imati podrazumevane stilove pretraživača. To obično uključuje stvari kao što su podrazumevani margini i padding.

## CSS Unutar linije, unutar zaglavlja, i van

- **CSS U Liniji**: Ove stilove se pišu direktno unutar elementa HTML koristeći atribut `style`. Često se koristi CSS unutar linije zbog podjele funkcionalnosti/zadataka.
- **CSS U Zaglavlju**: Ove stilove se pišu unutar oznaka `<style>` unutar sekcije `head` dokumenta HTML. Ovo može biti korisno za kreiranje kratkih primera koda, ali se obično koristi CSS unutar zaglavlja.
- **CSS Eksternalno**: Ove stilove se pišu u zasebnu datoteku CSS i povezuju sa dokumentom HTML koristeći element/funkciju `link` u sekciji `head`. Za većinu projekata, koristiće se eksterni fajl CSS umesto internog ili inline.

## Raditi sa svojstvima `width` i `height`

- **Sifa za `width`**: Ovo svojstvo opisuje širinu elementa. Ako ne definišete širinu, podrazumevana je `auto`. Ovo omogućava pregledaču da odredi širinu elementa na osnovu sadržaja, roditelja i tipa prikaza.
- **Sifa za `min-width`**: Ovo svojstvo opisuje minimalnu širinu elementa.
- **Sifa za `max-width`**: Ovo svojstvo opisuje maksimalnu širinu elementa.
- **Sifa za `height`**: Ovo svojstvo opisuje visinu elementa. Takođe, podrazumevana je visina `auto`, što znači da će se prilagoditi sadržaju.
- **Sifa za `min-height`**: Ovo svojstvo opisuje minimalnu visinu elementa.
- **Sifa za `max-height`**: Ovo svojstvo opisuje maksimalnu visinu elementa.

## Različite vrste za mešavinu CSS

- **Kombinacija predaka (` `)&nbsp;\*nafasi moja kati ya vichaguzi**: Ova kombinacija se koristi za ciljanje elemenata koji su predaci pomenutog roditeljskog elementa.
- **Kombinacija deteta (`>`)**: Ova kombinacija se koristi za odabir elemenata koji su direktni potomci pomenutog roditeljskog elementa.
- **Kombinacija neposrednog brata (`+`)**: Ova kombinacija bira sledeći element koji je brat pomenutog elementa.
- **Kombinacija sledećeg brata (`~`)**: Ova kombinacija bira sve elemente koji su braća pomenutog elementa i koji dolaze nakon njega.

## Elementi Nivoa Unutrašnje Linije, Nivo Bloka i Nivo Linija-Bloka

- **Inline Elementi**: Inline elementi zauzimaju samo potrebnu širinu i ne započinju novi red. Ovi elementi se uklapaju unutar sadržaja, omogućavajući tekstu i drugim inline elementima da se prikazuju zajedno sa njima. Uobičajeni inline elementi su `span`, `anchor` i `img`.
- **Block Elementi**: Block elementi po podrazumevanju zauzimaju svu dostupnu širinu, protežući se preko širine svog kontejnera. Uobičajeni block elementi su `div`, `p` i `section`.
- **Line-Block Elementi**: Možete postaviti element na `inline-block` korišćenjem atributa `display`. Ovi elementi funkcionišu kao inline elementi, ali mogu imati `width` i `height` kao block elemente.

## Ivica i unutrašnji prostor

- **Sifa za `margin`**: Ova svojstvo se koristi za postavljanje prostora izvan elementa, između granice elementa i okolnih elemenata.
- **Skraćeno pisanje za `margin`**: Možete postaviti vrednosti 1–4 za podešavanje stranica ivice. Jedna vrednost koristi se za sve četiri strane; dve vrednosti postavljaju `top` i `bottom`, zatim `right` i `left`; tri vrednosti postavljaju `top`, jednako (`right` i `left`), zatim `bottom`; četiri vrednosti postavljaju `top`, `right`, `bottom`, `left`.
- **Sifa za `padding`**: Ovo svojstvo se koristi za postavljanje prostora unutar elementa, između sadržaja i njegove granice.
- **Skraćeno pisanje za `padding`**: Možete postaviti vrednosti 1–4 za podešavanje stranica unutrašnjeg prostora. Jedna vrednost koristi se za sve četiri strane; dve vrednosti postavljaju `top` i `bottom`, zatim `right` i `left`; tri vrednosti postavljaju `top`, jednako (`right` i `left`), zatim `bottom`; četiri vrednosti postavljaju `top`, `right`, `bottom`, `left`.

## Jedinstvenost CSS

- **Specifičnost inline CSS-a**: Inline CSS ima najveću specifičnost jer se primenjuje direktno na element. Ima prednost u odnosu na interne i eksterne stilove. Vrednost specifičnosti za inline stilove je (1, 0, 0, 0).
- **Specifičnost internog CSS-a**: Interni CSS definiše se unutar elementa `style` u delu `head` HTML dokumenta. Ima manju specifičnost od inline stilova. Interni i eksterni CSS imaju isti nivo specifičnosti, koji zavisi od selektora, a ne od mesta na kome se nalaze.
- **Specifičnost eksternog CSS-a**: Eksterni CSS se povezuje pomoću elementa `link` u delu `head` i piše se u zasebnim `.css` datotekama. Kao i interni CSS, njegova specifičnost zavisi od korišćenih selektora. Kada dva pravila imaju istu specifičnost, redosled u izvornom kodu određuje pobednika — pravilo koje se pojavljuje kasnije ima prednost. Eksterni CSS pruža bolju održivost za velike projekte.
- **Univerzalni selektor (`*`)**: Ovo je posebna vrsta CSS selektora koja odgovara svakom elementu u dokumentu. Često se koristi za primenu stilova na sve elemente na stranici, što može pomoći pri resetovanju ili ujednačavanju stilova između različitih pregledača. Univerzalni selektor ima najnižu specifičnost među svim selektorima. Doprinosi vrednošću 0 svim delovima specifičnosti (0, 0, 0, 0).
- **Selektori tipa**: Ovi selektori ciljaju elemente na osnovu naziva njihove HTML oznake. Imaju nižu specifičnost od drugih selektora. Vrednost specifičnosti za selektor tipa je (0, 0, 0, 1).
- **Selektori klase**: Ovi selektori označavaju se tačkom (`.`) iza koje sledi naziv klase. Vrednost specifičnosti za selektor klase je (0, 0, 1, 0). To znači da selektori klase imaju prednost nad selektorima tipa, ali ih mogu nadjačati ID selektori i inline stilovi.
- **ID selektori**: ID selektori označavaju se znakom tarabe (`#`) iza kojeg sledi naziv ID-ja. Imaju veoma visoku specifičnost, veću od selektora tipa i klase, ali manju od inline stilova. Njihova vrednost specifičnosti je (0, 1, 0, 0).
- **Ključna reč `!important`**: Koristi se da bi pravilo stila dobilo najveći prioritet i nadjačalo sve druge deklaracije istog svojstva. Kada se koristi, primorava pregledač da primeni navedeni stil bez obzira na specifičnost drugih selektora. Koristi je pažljivo jer može otežati održavanje i otklanjanje grešaka u CSS-u.
- **Cascade algoritam**: Algoritam koji određuje koja će se CSS pravila primeniti kada više stilova cilja isti element. On obezbeđuje da se primeni najprikladniji stil prema jasno definisanim pravilima.
- **CSS nasleđivanje**: Proces kojim se stilovi prenose sa roditeljskih elemenata na njihove potomke. Nasleđivanje omogućava da definišeš stilove na višem nivou stabla dokumenta i primeniš ih na više elemenata bez pojedinačnog navođenja.

## Reč kreativnosti

- **Raspored**: Ovo je način na koji su vidljivi elementi raspoređeni na stranici ili ekranu kako bi preneli poruku. Ovi elementi mogu uključivati tekst, slike i prazan prostor.
- **Poravnanje**: Ovo je način na koji su elementi postavljeni u odnosu jedan prema drugom. Pravilno korišćenje poravnanja pomaže da dizajn izgleda čist i dobro organizovan.
- **Kompozicija**: To je čin aranžiranja elemenata radi stvaranja lepog dizajna. Ona određuje kako se elementi kao što su slike, tekst i oblici međusobno odnose i doprinose kompoziciji na umetnički način.
- **Balans**: Ovo je način na koji je vizuelna težina raspoređena unutar dizajna. Dizajneri pokušavaju da postignu balans kroz simetrične ili asimetrične aranžmane.
- **Dimenzija**: Odnosi se na poređenje dimenzija ili veličine jednog elementa u odnosu na drugi.
- **Vizuelna hijerarhija**: Uspostavlja redosled važnosti elemenata u dizajnu. Cilj je osigurati da su najvažnije informacije primećene prvenstveno.
- **Kontrast**: To je proces stvaranja jasnih razlika između elemenata. Može se postići promenom boje, veličine, oblika, teksture ili bilo kojeg drugog vidljivog atributa. Snažan kontrast takođe poboljšava čitljivost.
- **Prazan prostor (Negativni prostor)**: To je prazno mesto u dizajnu. To je oblast koja okružuje elemente.
- **UI (Korisnički interfejs)**: UI obuhvata vidljive i interaktivne elemente koje korisnici mogu videti na svojim ekranima, kao što su ikone, slike, tekst, meniji, linkovi i tasteri.
- **UX (Korisničko iskustvo)**: UX se tiče toga kako se korisnici osećaju kada koriste proizvod ili uslugu. Proizvod sa dobrim korisničkim iskustvom je jednostavan za razumevanje, korišćenje, efikasan, pristupačan i prijatan.
- **Kreativni brief**: To je dokument koji definiše ciljeve, objektivne i zahteve projekta. To je mapa koja vodi kroz kreativni proces i osigurava da finalni proizvod zadovoljava potrebe klijenta.
- **Vektorski dizajn**: Podrazumeva stvaranje digitalne umetnosti korišćenjem matematičkih formula za definisanje linija, oblika i boja.
- **Prototipovanje**: Odnosi se na proces kreiranja modela interakcije proizvoda ili korisničkog interfejsa.

## Osnove dizajna korisničkog interfejsa

- **Dobar kontrast između pozadinske i prednje boje**: Važno je osigurati da pozadinska i prednja boja imaju dobar kontrast kako bi tekst bio čitljiv. WCAG smernice za pristupačnost sadržaja na mreži preporučuju odnos kontrasta od najmanje 4,5:1 za običan tekst i 3:1 za veliki tekst.
- **Jasan vizuelni linijski tok u dizajnu**: Jaka vidljiva linija može pružiti jasan put za oko da prati, osiguravajući da informacije koje predstavljate budu korišćene u namernom redosledu.
- **Responsivne slike (Praznice koje se prilagođavaju uređaju)**: Responsivne slike su slike koje menjaju veličinu u zavisnosti od veličine ekrana na kojem se gledaju. Ovo je važno jer osigurava da vaše slike izgledaju dobro na svim uređajima, od računara do mobilnih telefona.
- **Progresivno poboljšanje**: Ovo je dizajnerska tehnika koja osigurava da svi korisnici, bez obzira na pretraživač ili uređaj, mogu pristupiti ključnom sadržaju i funkcionalnosti aplikacije.
- **Dizajn usmeren na korisnika**: Ovo je pristup koji stavlja krajnjeg korisnika u prvi plan, od njegovih potreba do preferencija i ograničenja. Cilj dizajna usmerenog na korisnika je kreiranje web stranice koja je razumljiva, laka za korišćenje i prijatna za interakciju sa korisnicima.
- **Istraživanje korisnika**: Ovo je sistematska studija ljudi koji koriste vaš proizvod. Cilj je merenje potreba, ponašanja i izazova korisnika.
- **Testiranje korisnika**: Ovo se odnosi na vežbe prikupljanja podataka od korisnika kada interaguju sa vašom aplikacijom.
- **A/B testiranje**: Ovo je proces slanja nove funkcije malom, nasumično izabranoj grupi korisnika. Zatim možete koristiti analitičke podatke da utvrdite da li je ta funkcija korisna.
- **Potrebe korisnika**: Ovo se odnosi na priče ili kriterijume koje vaša aplikacija treba da prati. Potrebe korisnika mogu biti definisane istraživanjem korisnika, ili industrijskim standardima. Takođe mogu biti definisane povratnim informacijama zainteresovanih strana (stakeholderi).
- **Postepila otkrivanja**: Ovo je dizajnerski obrazac koji se koristi za prikazivanje samo relevantnog sadržaja korisnicima na osnovu njihove trenutne aktivnosti i skrivanje ostalog. To se radi kako bi se smanjio kognitivni teret

## Najbolje metode za kreativnost

- **Tamni režim**: Ovo je posebna funkcija u web aplikacijama gde možete promeniti šemu boja sa svetle na tamnu. Trebalo bi da koristite boje niskog kontrasta u tamnom režimu. Boje niskog kontrasta su one koje nisu previše jarke i imaju nizak nivo uticaja.
- **Putanja (Breadcrumbs)**: Ovo je navigaciona pomoć koja korisniku pokazuje gde se nalazi unutar hijerarhije sajta. Najbolje je postaviti putanju na vrh stranice kako bi je korisnici lako pronašli. Takođe, želite da osigurate da su *breadcrumbs* dovoljno veliki za lakše čitanje, ali ne preveliki pa da zauzimaju previše prostora na stranici.
- **Sekcija kartica**: Vaša sekcija kartica bi trebalo da bude jednostavan u dizajnu, bez vizuelnog preopterećenja ili prikazivanja previše informacija. Za medije, obavezno izaberite slike i video snimke visokog kvaliteta kako biste poboljšali korisničko iskustvo (UX).
- **Beskonačno skrolovanje**: Ovo je dizajnerski obrazac koji učitava više sadržaja dok korisnik skroluje dole po stranici. Trebalo bi da razmotrite korišćenje dugmeta „Učitaj više“ jer daje korisniku kontrolu kada želi da vidi više sadržaja. Takođe, možete dodati i dugme za povratak kako bi korisnici imali mogućnost vraćanja na prethodnu stranicu bez potrebe za skrolovanjem sve do samog vrha.
- **Modalni dijalog**: Ovo je vrsta *pop-up* prozora koji se prikazuje preko postojećeg sadržaja stranice. Obično su pozadinski elementi tamnije boje kako bi korisnik fokusirao pažnju na modalni sadržaj. Takođe, dobra praksa je dozvoliti korisniku da klikne izvan modala kako bi zatvorio prozor. Kada koristite element HTML od `dialog`, dobićete ugrađene funkcije i prednosti pristupačnosti.
- **Indikator napretka za formular/registraciju**: Ovo je način prikazivanja korisnicima gde se nalaze u nekom procesu. Može se koristiti u formularima, registraciji i procesima podešavanja. Va dizajn bi trebalo da bude jednostavan, lako dostupan i da omogućava povratak na prethodne korake.
- **Korpa za kupovinu**: Korpa je mesto gde korisnik vidi proizvode koje je izabrao na e-trgovinskoj platformi. Vaša korpa bi trebalo da bude uvek vidljiva korisniku, koristite standardnu ikonu kao što su korpa, torba ili vrećica, i da ima jasno definisano dugme za akciju (CTA) kako bi korisnici mogli nastaviti sa plaćanjem.

## Alati za kreativnost koji se često koriste

- **Figma**: Ovo je cloud alat koji se specijalizuje za dizajn korisničkog interfejsa i korisničko iskustvo (UI/UX). Omogućava timovima za dizajn i razvoj da sarađuju sa bilo kog mesta, nudeći ugrađene funkcije uključujući osnovni vektorski dizajn, direktno postavljanje, sistem komentara i još mnogo toga.
- **Sketch**: Ovo je popularni alat za dizajn koji se koristi zbog svog intuitivnog i jednostavnog interfejsa, idealan za programere koji žele brzo kreirati prototipe. Takođe ga koriste dizajneri za zadatke kao što su kreiranje UI-ja, logotipa i web rasporeda.
- **Adobe XD**: Ovo je alat za vektorski dizajn i prototipiranje za UI/UX dizajn, poznat po svojoj odličnoj integraciji sa drugim Adobe programima kao što su Photoshop, Illustrator i After Effects.
- **Canva**: Ovaj alat vam omogućava da kreirate različite vrste vizuelnog sadržaja, uključujući bannere, naslovne slike, prezentacije, kratke video snimke i još mnogo toga. Njegov jednostavan i prijateljski dizajn je idealan za početnike.

## Kompletnih mera

- **`px` (Pikseli)**: Ova kompletna jedinica je mera veličine postavljena u CSS, koja pruža preciznu kontrolu nad merama. To znači da je 1px uvek jednako 1/96 inča.
- **`in` (Inči)**: Ova kompletna jedinica je jednaka 96px.
- **`cm` (Centimetar)**: Ova kompletna jedinica je jednaka 25.2/64 inča.
- **`mm` (Milimetar)**: Ova kompletna jedinica je jednaka 1/10 centimetra.
- **`q` (Četvrt milimetra)**: Ova kompletna jedinica je jednaka 1/40 centimetra.
- **`pc` (Pika)**: Ova kompletna jedinica je jednaka 1/6 inča.
- **`pt` (Tačka)**: Ova kompletna jedinica je jednaka 1/72 inča.

## Povezani parametri

- **Asilimia**: Ove povezane dimenzije omogućavaju definisanje veličine, razmera i drugih svojstava kao deo roditeljskog elementa. Na primer, ako postavite `width: 50%;` za element, zauzimaće pola širine roditeljskog kontejnera.
- **Dimenzija `em`**: Ove dimenzije su povezane sa veličinom fonta elementa. Ako koristite `ems` za svojstvo `font-size`, veličina teksta će biti povezana sa veličinom roditeljskog fonta.
- **Dimenzija `rem`**: Ove dimenzije su povezane sa veličinom fonta korenovog elementa, koji je element `html`.
- **Dimenzija `vh`**: `vh` je postavljeno za `"viewport height"` i `1vh` odgovara 1% visine vidljivog područja.
- **Dimenzija `vw`**: `vw` je postavljeno za `"viewport width"` i `1vw` odgovara 1% širine vidljivog područja.

## Rad od `calc`

- **Kazi ya `calc()`**: Za funkciju `calc()`, možete izvršiti proračune direktno unutar vaših stilova kako biste odredili vrednosti svojstava na dinamičan način. To znači da možete kreirati dinamički korisnički interfejs i prilagoditi ga računanjem dimenzija na osnovu veličine vidokruga ili drugih elemenata.

## Mock klasa za akcije korisnika

- **Definicija Pseudo-klase**: Ovo su specifični selektori za CSS koji vam omogućavaju da izaberete element na osnovu njegovog stanja ili određene pozicije.
- **Pseudo-klasa za akcije korisnika**: Ovo su specifični selektori koji omogućavaju menjanje izgleda elemenata na osnovu interakcije korisnika, poboljšavajući ukupno korisničko iskustvo.
- **Pseudo-klasa `:active`**: Ova pseudo-klasa vam omogućava da izaberete stanje akcije elementa, kao što je klik na dugme.
- **Pseudo-klasa `:hover`**: Ova pseudo-klasa definiše stanje prikazivanja elementa kada ga miš pređe (hover).
- **Pseudo-klasa `:focus`**: Ova pseudo-klasa postavlja stilove kada element dobije fokus, obično putem navigacije mišem ili kada korisnik klikne na polje forme.
- **Pseudo-klasa `:focus-within`**: Ova pseudo-klasa se koristi za postavljanje stilova elementu kada on ili njegov predak imaju fokus.

## Klasa simuliranog ulaza

- **Pseudo-klasa za input**: Ove klase se koriste za ciljanje elemenata HTML od `input` u zavisnosti od njihovog stanja pre i posle interakcije korisnika.
- **Pseudo-klasa `:enabled`**: Ova pseudo-klasa se koristi za ciljanje dugmadi forme ili drugih trenutno aktiviranih elemenata.
- **Pseudo-klasa `:disabled`**: Ova pseudo-klasa omogućava stilizovanje onemogućenog elementa.
- **Pseudo-klasa `:checked`**: Ova pseudo-klasa se koristi za prikaz korisniku da je element izabran.
- **Pseudo-klasa `:valid`**: Ova pseudo-klasa cilja polja za unos koja zadovoljavaju kriterijume validacije.
- **Pseudo-klasa `:invalid`**: Ova pseudo-klasa cilja polja za unos koja ne zadovoljavaju kriterijume validacije.
- **Pseudo-klase `:in-range` i `:out-of-range`**: Ove pseudo-klase primenjuju stilove na elemente u zavisnosti od toga da li su njihove vrednosti unutar ili van definisanog opsega.
- **Pseudo-klasa `:required`**: Ova pseudo-klasa cilja elemente `input` sa atributom `required`. Ovo ukazuje korisniku da mora popuniti to polje pre slanja forme.
- **Pseudo-klasa `:optional`**: Ova pseudo-klasa primenjuje stilove na elemente za unos koji nisu obavezni i mogu ostati prazni.
- **Pseudo-klasa `:autofill`**: Ova pseudo-klasa primenjuje stilove na polja za unos koja pregledač automatski popunjava sa sačuvanim podacima.

## Simulirana klasa o mestu

- **Pseudoklasa Mahali**: Ove pseudoklase se koriste za stilizovanje linkova i ciljanih elemenata unutar trenutnog dokumenta.
- **Pseudoklasa `:any-link`**: Ova pseudoklasa je kombinacija klasa `:link` i `:visited`. Dakle, poklapa se sa bilo kojim anhkor elementom koji ima atribut href, bez obzira da li je kliknut ili ne.
- **Pseudoklasa `:link`**: Ova pseudoklasa vam omogućava da ciljate sve linkove koji nisu kliknuti na web stranici. Možete je koristiti za stilizovanje različitih linkova pre nego što korisnik klikne.
- **Pseudoklasa `:local-link`**: Ova pseudoklasa cilja linkove koji vode ka istom dokumentu. Ovo može pomoći kada želite da razlikujete interne i eksterne linkove.
- **Pseudoklasa `:visited`**: Ova pseudoklasa cilja link koji je korisnik kliknuo.

## Teorija boje

- **Definicija Teorije Boja**: Ovo je lekcija o tome kako boje interaguju i utiču na naše percepcije. Obuhvata odnos boja, harmoniju boja i psihološki efekat boja.
- **Primarne boje**: Ove boje su žuta, plava i crvena; to su primarne boje od kojih potiču sve ostale boje.
- **Sekundarne boje**: Dobijaju se mešanjem jednakih količina dve primarne boje. Zelena, narandžasta i ljubičasta su primeri sekundarnih boja.
- **Tertijarne boje**: Dobijaju se mešanjem primarne i susedne sekundarne boje. Žuto-zelena, plavo-zelena i plavo-ljubičasta su primeri tertijarnih boja.
- **Tople boje**: Ove boje su crvena, narandžasta i žuta; one unose osećaj udobnosti, topline i vitalnosti.
- **Hladne boje**: Ove boje su plava, zelena i ljubičasta; one unose osećaj smirenosti, mira i profesionalnosti.
- **Kolorokrug**: Kolorokrug je kružni prikaz koji pokazuje kako se boje međusobno nadovezuju. To je važan alat za dizajnere jer im pomaže da izaberu kombinacije boja.
- **Analogne šeme boja**: Ove šeme boja stvaraju osećaj harmonije i smirenosti. Imaju slične boje koje su susedne na kolorokrugu.
- **Komplementarne šeme boja**: Ove šeme boja stvaraju snažan kontrast i vizuelni efekat. Njihove boje su na suprotnim stranama kolorokruga, u odnosu jedna na drugu.
- **Triadijske šeme boja**: Ove šeme imaju intenzivne boje. Sastavljene su od boja koje su približno jednako udaljene jedna od druge. Povezane, one čine jednakostranični trougao na kolorokrugu.
- **Monohromatske šeme boja**: Ovom šemi se sve boje dobijaju od jedne primarne boje menjanjem svetline, tamnje i intenziteta. Ovo unosi osećaj jedinstva i harmonije uz stvaranje varijacije.

## Različite metode za rad sa bojama u CSS

- **Navedene boje**: Ove su deklarisane nazive boja koje pretraživači prepoznaju. Primeri su `blue`, `darkred`, `lightgreen`.
- **Funkcija `rgb()`**: RGB predstavlja Crveno, Zeleno i Plavo — osnovne boje svetlosti. Ove tri boje se mešaju sa različitom intenzitetom kako bi stvorile mnogo boja. Funkcija `rgb()` omogućava vam da definišete boju korišćenjem RGB modela boja.
- **Funkcija `rgba()`**: Ova funkcija dodaje četvrti parametar — alfa (alpha) — koji kontroliše prozirnost boje.
- **Funkcija `hsl()`**: HSL predstavlja Ton (Hue), Saturaciju i Svetlinu (Lightness) — tri ključne komponente koje opisuju boju.
- **Funkcija `hsla()`**: Ova funkcija dodaje četvrti parametar — alfa (alpha) — koji kontroliše prozirnost boje.
- **Heksaadecimalni**: Heksadecimalan broj (skraćeno od heksaadecimalnog broja) je sekvenca od šest karaktera koja se koristi za predstavljanje boje u RGB modelu boja. "Hex" se odnosi na sistem brojanja sa bazom 16, koji koristi brojeve od 0 do 9 i slova A do F.

## Karakteristika za `box-shadow`

- **Objašnjenje**: Svojstvo `box-shadow` postavlja jednu ili više senki oko elementa.
- **Vrednosti udaljenosti**: Morate definisati vrednosti za horizontalnu (`offset-x`) i vertikalnu (`offset-y`) os. Pozitivna vrednost `offset-x` pomera senku udesno, dok negativna vrednost pomera ulevo. Pozitivna vrednost `offset-y` pomera senku nadole, dok negativna vrednost pomera nagore. Ako je vrednost `0`, nije potrebno navoditi dimenziju.
- **Radijus zamućenja (Blur Radius)**: Ova opcionalna vrednost kontroliše kako senka izgleda zamućena. Ako nije definisana, podrazumevano je `0`, što stvara oštre ivice. Veća vrednost čini senku mekšom.
- **Radijus širenja (Spread Radius)**: Ova opcionalna vrednost kontroliše koliko se senka širi ili smanjuje. Ako nije definisana, podrazumevano je `0`.
- **Boja senke**: Možete odrediti boju korišćenjem imena boja, hex vrednosti, `rgb()`, `rgba()`, `hsl()`, ili funkcija `hsla()`.
- **Tekst za `inset`**: Dodavanje teksta `inset` postavlja senku unutar elementa umesto spolja.
- **Postavljanje više senki u kutiji**: Možete postaviti više senki razdvajanjem zarezima. Senke se postavljaju slojevito, od prednjeg ka zadnjem.

### Sintaksa

```css
box-shadow: offset-x offset-y blur-radius spread-radius color;
```

## Karakteristike CSS Promene

- **Opis**: Ova funkcija vam omogućava da koristite različite transformacije za elemente, kao što su rotiranje, promena veličine, savijanje ili pomeranje (od) u 2D ili 3D prostoru.

- **Transformacije i Pristupačnost**: Ako koristite transformacije za skrivanje ili prikaz sadržaja, osigurajte da je sadržaj i dalje dostupan za čitače ekrana i navigaciju pomoću tastature. Skriveni sadržaj treba da bude stvarno skriven, kao što je korišćenjem `display: none` ili `visibility: hidden`, umesto samo vizuelnog pomeranja van ekrana.

## Primer kutije

- **Definicija**: U primeru kutije CSS, svaki element je okružen kutijom. Ova kutija ima četiri dela: oblast sadržaja, `padding`, `border`, `margin`.
- **Oblast Sadržaja**: Oblast sadržaja je najunutrašnji deo kutije. To je prostor koji drži stvarni sadržaj elementa, kao što su tekst ili slika.
- **`padding`**: Unutrašnji razmak je područje koje sledi odmah nakon oblasti sadržaja. To je prostor između oblasti sadržaja i granice elementa.
- **`border`**: Granica je spoljna ivica ili linija elementa u primeru kutije CSS. To je vidljiva granica elementa.
- **`margin`**: Padding je prostor van granice elementa. Osigurava razdaluku između elementa i drugih okružujućih elemenata.

## Padanje ruba

- **Definicija**: Ova karakteristika se javlja kada vertikalne ivice susednih elemenata preklapaju, uzrokujući jednu ivicu koja je jednaka i veća od dve druge. Ova karakteristika se odnosi samo na vertikalne ivice (gore i dole), a ne na horizontalne ivice (levo i desno).

## Vrednosti atributa za `content-box` i `border-box`

- **Atribut za `box-sizing`**: Ovaj atribut se koristi za određivanje kako su konačni `width` i `height` izračunati za element HTML.
- **Vrednost za `content-box`**: U primeru od `content-box`, `width` i `height` postavljeni za element određuju dimenzije sadržajnog područja, ali ne uključuju `padding`, `border` ili `margin`.
- **Vrednost za `border-box`**: Za `border-box`, `width` i `height` elementa uključuje sadržajno područje, `padding` i `border`, ali ne uključuje `margin`.

## Karakteristika filtra CSS

- **Opis**: Ova karakteristika se može koristiti za kreiranje različitih efekata kao što su zamućenje, promena boja i podešavanje kontrasta.
- **Funkcija `blur()`**: Ova funkcija koristi Gaussian blur za element. Iznos je definisan u pikselima i predstavlja radius zamućenja.
- **Funkcija `brightness()`**: Ova funkcija podešava svetlinu elementa. Vrednost od 0% učiniće element potpuno tamnim, dok će vrednosti preko 100% povećati svetlinu.
- **Funkcija `grayscale()`**: Ova funkcija konvertuje element u sivu boju. Iznos je definisan kao procenat, gde je 100% potpuno sivo, a 0% daje nepromenjenu sliku.
- **Funkcija `sepia()`**: Ova funkcija koristi sepiju za element. Kao siva, koristi vrednost u procentima.
- **Funkcija `hue-rotate()`**: Ova funkcija koristi rotaciju boja za element. Vrednost je definisana u stepenima i predstavlja rotaciju oko točkića boja.

## Uvod u CSS Flexbox i Primer Fleksa

- **Definicija**: CSS flexbox je model rasporeda elemenata jednosmernog tipa koji omogućava aranžiranje elemenata u redovima i kolonama unutar kontejnera.
- **Flex Primer**: Ovaj primer objašnjava kako se flex elementi raspoređuju unutar flex kontejnera. Svaki flex kontejner ima dve ose: glavnu osu i popernu osovu.

## Karakteristika od `flex-direction`

- **Opis**: Ova svojstvo postavlja smer glavne ose. Podrazumevana vrednost `flex-direction` je `row`, što postavlja sve flex elemente u jedan red, prema smeru jezika vašeg pregledača (od leve ka desnoj ili od desne ka levoj).
- **`flex-direction: row-reverse;`**: Ovo menja raspored elemenata u redu.
- **`flex-direction: column;`**: Ovo će postaviti flex elemente vertikalno umesto horizontalno.
- **`flex-direction: column-reverse;`**: Ovo menja raspored vertikalnih flex elemenata.

## Karakteristika za `flex-wrap`

- **Opis**: Ovo svojstvo određuje kako se fleksibilne komponente prilagođavaju unutar fleks kontejnera da bi odgovarale dostupnom prostoru. `flex-wrap` može uzeti tri vrednosti: `nowrap`, `wrap`, i `wrap-reverse`.
- **`flex-wrap: nowrap;`**: Ovo je podrazumevana vrednost. Fleksibilne komponente neće se prelomiti na novi red, čak i ako njihova širina prelazi širinu kontejnera.
- **`flex-wrap: wrap;`**: Ovo svojstvo će prelomiti komponente kada njihova širina prelazi širinu kontejnera.
- **`flex-wrap: wrap-reverse;`**: Ovo svojstvo će prelomiti fleksibilne komponente u smeru unazad.
- **Svojstvo `flex-flow`**: Ovo svojstvo je skraćenica za `flex-direction` i `flex-wrap`.

## Svojstvo `justify-content`

- **Opis**: Ova svojstvo postavlja elemente dece koje stoje na glavnoj osi fleksibilnog kontejnera.
- **`justify-content: flex-start;`**: U ovom slučaju, flex elementi će biti raspoređeni od početka glavne ose. Ovo može biti po redu ili vertikalno.
- **`justify-content: flex-end;`**: U ovom slučaju, flex elementi su raspoređeni na kraju glavne ose, po redu ili vertikalno.
- **`justify-content: center;`**: Ovo postavlja flex elemente u sredinu glavne ose.
- **`justify-content: space-between;`**: Ovo će rasporediti elemente ravnomerno na glavnoj osi.
- **`justify-content: space-around;`**: Ovo će rasporediti flex elemente ravnomerno unutar glavne ose, dodajući prostor pre prvog i posle poslednjeg elementa.
- **`justify-content: space-evenly;`**: Ovo raspoređuje elemente ravnomerno na glavnoj osi.

## Atribut za `align-items`

- **Opis**: Ovo svojstvo se koristi za distribuciju komponenti na poprečnu osu. Napomena: Poprečna osa je vertikalna u odnosu na glavnu osu.
- **`align-items: center;`**: Ovo se koristi za postavljanje delova u sredini poprečne ose.
- **`align-items: flex-start;`**: Ovo postavlja delove počevši od početka poprečne ose.
- **`align-items: stretch;`**: Ovo se koristi za ispravljanje fleksibilnih delova na poprečnoj osi.

## Uvod u tipografiju

- **Definicija**: Tipografija je umetnost izbora odgovarajućeg fonta i dizajna kako bi tekst izgledao dobro i bio lako čitljiv. "Tip" se odnosi na način na koji su pojedinačna slova kreirana i raspoređena.
- **Definicija Typeface-a (tipfač):** Typeface je opšti dizajn i stil celog seta slova, brojeva i simbola. To je kao mapa fontove porodice.
- **Definicija Fonta:** Font je specifična varijanta typeface-a sa određenim karakteristikama, kao što su veličina, težina (debljina), stil i širina.
- **Serif Typeface:** Ovaj typeface ima tradicionalni stil i male linije na krajevima slova. Serif typefacei se često koriste za štampane materijale, poput knjiga.
- **Sans Serif Typeface:** Ovaj typeface ima moderniji izgled, bez malih linija na krajevima slova. Sans serif typefacei se često koriste u digitalnom dizajnu jer su lakši za čitanje na ekranu. Primeri su Helvetica, Arial i Roboto.
- **Težina Fonta (Font Weight):** Ovo je debljina slova, uključujući lagano, normalno, podebljano i debelo/crno.
- **Stil Fonta:** Ovo je orijentacija i nagib slova, kao što su *italic* i *oblique*.
- **Širina Fonta (Font Width):** Ovo je jednaka razdaljina koju slovo zauzima, kao što su *condensed* (skupljeno) i *expanded* (prošireno).
- **Osnovna Linija (Baseline):** Ovo je maštovita linija od koje se većina slova oslanja.
- **Visina Kapitala (Cap Height):** Ovo je visina velikih slova, merena od osnovne linije do vrha.
- **X Visina (X-height):** Ovo je prosečna visina malog slova, bez računanja delova koji se podižu ili spuštaju.
- **Ascenderi (Podignuti delovi):** To su delovi malih slova koji se podižu iznad X visine, kao što su gornji delovi slova 'h', 'b', 'd' i 'f'.
- **Descenderi (Spušteni delovi):** To su delovi malih slova koji se spuštaju ispod osnovne linije, kao što su rep slova 'y', 'g', 'p' i 'q'.
- **Kerning:** Ovo je podešavanje razmaka između specifičnih parova slova radi poboljšanja čitljivosti i estetike. Na primer, smanjenje razmaka između slova A i V.
- **Tracking:** Ovo je podešavanje razmaka između svih slova u određenom bloku teksta. To je rastojanje između slova. Utiče na to kako tekst izgleda gušće ili otvorenije.
- **Leading (Linijski razmak):** Ovo je vertikalni razmak između redova teksta. Meri se od osnovne linije jednog reda do osnovne linije sledećeg reda.
- **Najbolje prakse u tipografiji:** Trebalo bi da izaberete fontove koji su jasni i laki za korišćenje kako biste svoj dizajn učinili lakim za razumevanje. Ovo je posebno važno za glavni tekst vaše veb stranice. Korisnici imaju mnogo veću verovatnoću interakcije sa vašim sadržajem ako je font lako čitljiv. Trebalo bi da koristite samo dva ili tri fonta kako biste stvorili vizuelnu harmoniju. Korišćenje previše fontova može učiniti tekst teškim za čitanje i oslabiti identitet vaše marke. To takođe može uticati na korisničko iskustvo povećanjem vremena učitavanja stranice. Možete koristiti veličinu fonta kako biste stvorili vizuelnu hijerarhiju za naslove, podnaslove, pasuse i druge elemente. Na primer, glavni naslov na veb stranici bi trebalo da ima najveći font, praćen manjim podnaslovima sa manjom veličinom fonta. Ovo će svakom elementu u hijerarhiji dati specifičnu veličinu fonta koja pomaže korisnicima da vizuelno navigiraju po dizajnu.

## Svojstvo CSS od `font-family`

- **Definicija**: Font family je grupa fontova koji dele zajednički dizajn. Svi fontovi pripadajući jednoj porodici potiču iz jednog osnovnog tipografskog dizajna, ali se razlikuju u stilu, debljini i širini. Možete navesti više font porodica po prioritetu, odozgo nadole, razdvajajući ih zarezima. Ovi zamenski fontovi će funkcionisati kao alternativni izbor. Uvek biste trebali uključiti opštu porodicu fontova na kraju liste `font-family`.
- **Uobičajene porodice fontova**: Evo nekih uobičajenih font porodica koje se koriste u CSS: serif, sans-serif, monospace, cursive, fantasy

## Sigurni fontovi za internet

- **Objašnjenje**: Ovi fontovi su mali skup fontova koji je vrlo verovatno instaliran na računaru ili uređaju. Kada pregledač treba da prikaže font, pokušava da pronađe fajl fonta u korisničkom sistemu. Međutim, ako font nije dostupan, obično se vraća na podrazumevani sistemski font.

- **Web-bezbedni fontovi**: 
  - Sans-serif fontovi se često koriste u web razvoju jer nemaju male "noge" ili linije na kraju slova, pa su laki za čitanje na ekranu. Primeri bezbednih sans-serif fontova su: Arial, Verdana i Trebuchet MS.
  - Naspram toga, serif fontovi imaju male "noge" na kraju slova, pa se često koriste za tradicionalno štampanje. Bezbedni serif fontovi uključuju: Times New Roman i Georgia.

## Pravila za At i Pravila od `@font-face`

- **Definicija**: `@` pravila su deklaracije koje daju instrukcije pretraživaču. Možete ih koristiti za definisanje različitih elemenata u stilskom listu, kao što su media queries, keyframes, font faces i slično.
- **`@font-face`**: Ovo vam omogućava da definišete specifične fontove navođenjem fajla fonta, formata i drugih važnih svojstava, kao što su težina i stil. Da bi pravilo `@font-face` bilo važeće, takođe je potrebno definisati svojstvo `src`. Ovo svojstvo sadrži reference na font resurse.
- **Font Formati**: Za svaki font resurs, možete navesti i format. Ovo je opcionalno. To je indikacija za pretraživač o formatu fonta. Ako nije naveden format, resurs će biti preuzet, a format detektovan nakon preuzimanja. Ako je format nevažeći, resurs neće biti preuzet. Mogući font formati su `collection`, `embedded-opentype`, `opentype`, `svg`, `truetype`, `woff` i `woff2`.
- **`woff` i `woff2`**: `woff` znači "Web Open Font Format". To je široko korišćeni font format razvijen od strane Mozilla u saradnji sa Type Supply, LettError i drugim organizacijama. Razlika između `woff` i `woff2` je algoritam koji se koristi za obradu podataka.
- **OpenType**: Ovo je format varijabilnih fontova kreiran od strane Microsofta i Adobe-a, koji korisnicima omogućava pristup dodatnim funkcijama unutar fonta. Često se koristi u velikim operativnim sistemima.
- **`tech()`**: Ovo se koristi za specifikovanje tehnologije font resursa. Ovo je opcionalno.

## Rad sa eksternim fontovima

- **Definicija**: Eksterne fontove su fajlovi fontova koji nisu direktno uključeni u fajlove vašeg projekta. Obično se čuvaju na zasebnom serveru. Kada uključite ove eksterne fontove u svoj projekat, možete koristiti element sa `link` ili deklaraciju sa `@import` unutar vašeg CSS.
- **Google Fonts**: Ovo je usluga Google-a koja pruža kolekciju fontova, mnogi od kojih su posebno kreirani za web razvoj.
- **Font Squirrel**: Ovo je drugi popularni resurs koji možete koristiti da dodate specifične eksterne fontove vašim projektima.

## Svojstvo `text-shadow`

- **Opis**: Ovo svojstvo se koristi za postavljanje senki tekstu. Morate navesti horizontalni i vertikalni razmak senke od teksta, respektivno. Ove vrednosti su neophodne. Pozitivne vrednosti X i Y će pomeriti senku udesno i dole, respektivno, dok će negativne vrednosti pomeriti je ulevo i gore.
- **Boja senke**: Možete prilagoditi boju senke tako što ćete ovu vrednost postaviti pre ili posle razmaka. Ako boja nije navedena, pregledač će automatski odrediti boju, pa je obično najbolje eksplicitno definisati.
- **Radijus zamućenja (Blur Radius)**: Radijus zamućenja je opciono, ali čini senku mekšom i privlačnijom. Vrednost po defaultu za radijus zamućenja je nula. Veća vrednost znači veće zamućenje, što čini senku svetlijom.
- **Postavljanje više senki teksta**: Tekst može imati više od jedne senke. Senke će biti postavljene u slojevima, od prednjeg ka zadnjem, a prva senka je na vrhu.

Ovaj alat takođe ima funkciju Simulacija bočne slepilo (Color Blind Simulator) koja pokazuje kako vaš sajt ili aplikacija izgleda osobama sa problemima u vidu boja.

## Drvo pristupa

Struktura pristupačnosti je model koji koriste asistivne tehnologije, kao što su čitači ekrana, kako bi razumele i interagovale sa sadržajem na veb stranici.

## Rad od `max()`

Funkcija `max()` vraća najveću vrednost iz skupa vrednosti odvojenih zarezima:

```css
img {
  width: max(250px, 25vw);
}
```

U gornjem primeru, širina slike će biti 250px ako je širina viewport-a manja od 1000 piksela (jer je 250px veći od 25% širine viewport-a). Ako je širina viewport-a veća od 1000 piksela, širina slike će biti 25vw. To je zato što je 25vw jednako 25% širine viewport-a.

## Najbolja metoda sa CSS i na pristup

- **`display: none;`**: Korišćenje `display: none;` znači da čitači ekrana i druge asistivne tehnologije ne mogu pristupiti ovom sadržaju, jer nije uključen u stab pristupačnosti. Stoga je važno koristiti ovu metodu samo kada želite potpuno ukloniti sadržaj iz vizuelne prezentacije i pristupačnosti.
- **`visibility: hidden;`**: Ovaj atribut i vrednost skrivaju sadržaj vizuelno, ali ostavljaju ga u toku dokumenta, što znači da i dalje zauzima prostor na stranici. Ove elemente takođe neće čitati ni čitači ekrana jer su uklonjeni iz stabla pristupačnosti.
- **Darasa CSS sa `.sr-only`**: Ovo je uobičajena tehnika koja se koristi za skrivanje sadržaja vizuelno, a da ostane dostupan čitačima ekrana.

```css
.sr-only {
  position: absolute;
  width: 1px;
  height: 1px;
  padding: 0;
  margin: -1px;
  overflow: hidden;
  clip: rect(0, 0, 0, 0);
  white-space: nowrap;
  border: 0;
}
```

- **`scroll-behavior: smooth;`**: Ova svojstva i vrednost omogućavaju linijsko ponašanje.
- **Sifa ya `prefers-reduced-motion`**: Medijski atribut koji se može koristiti za detekciju korisničke preferencije za kružne crteže.

```css
@media (prefers-reduced-motion: no-preference) {
  * {
    scroll-behavior: smooth;
  }
}
```

U gornjem primeru, pomeranje linije je omogućeno ako korisnik nema preferenciju za dijagramima kola postavljene na svom uređaju.

## Skrivanje sadržaja na osnovu atributa HTML

- **Atribut za `aria-hidden`**: Koristi se za skrivanje elementa od osoba koje koriste asistivne tehnologije, kao što su čitači ekrana. Na primer, može se koristiti za skrivanje dekorativnih slika koje ne pružaju nikakav smislen sadržaj.
- **Atribut za `hidden`**: Ovaj atribut je podržan od strane mnogih modernih pregledača i skriva sadržaj iz pogledu i iz stabla pristupačnosti. Može se lako promeniti korišćenjem JavaScript.

:::interactive_editor

```html
<p aria-hidden>This paragraph is visible for sighted users, but is hidden from assistive technology.</p>
<p hidden>This paragraph is hidden from both sighted users and assistive technology.</p>
```

:::

## Problem pristupa svojstvu `placeholder`

Korišćenje placeholder teksta nije dobro za pristupačnost. Često korisnici mešaju tekst naslovnica sa stvarnom vrednošću unosa – misle da je već vrednost u unosu.

## Rad sa selektorima atributa i elemenata

- **Opis**: Selektor za `attribute` omogućava vam da ciljate elemente HTML na osnovu njihovih atributa, kao što su atribut `href` ili `title`.
- **Atribut `title`**: Ovaj atribut pruža dodatne informacije o elementu.

## Ciljanje elemenata po atributima za `lang` i `data-lang`

- **Atribut `lang`**: Ovaj atribut se koristi u HTML za određivanje jezika sadržaja unutar elementa. Možda želite stilizovati različite elemente u zavisnosti od napisanih jezika, posebno na višejezičnom sajtu.
- **Atribut `data-lang`**: Specifični atributi podataka kao što je atribut `data-lang` često se koriste za skladištenje dodatnih informacija u elementima, kao što je određivanje jezika korišćenog unutar određenog dela teksta.

## Rad sa selektorem atributa, Svojstva liste sa atributom `type`

- **Svojstvo `type`**: Kada radite sa sortiranim listama u HTML, svojstvo `type` vam omogućava da utvrdite tip broja koji se koristi, kao što su obični brojevi, abeceda ili rimski brojevi.

## Rad sa plutajućim decimalnim brojevima (Floats)

- **Definicija**: Flotacija se koristi za uklanjanje elementa iz normalnog toka stranice i pozicioniranje ga na levoj ili desnoj strani njegovog kontejnera. Ovo uzrokuje da tekst prati sadržaj koji je u stanju flotacije.
- **Brisanje flotacije**: Atribut `clear` koristi se za određivanje da li element treba da bude pomeren ispod sadržaja koji je u stanju flotacije. Kada postoji mnogo elemenata u stanju flotacije jedan pored drugog, mogu nastati problemi sa preklapanjem i kolaps rasporeda. Stoga je metoda `clearfix` kreirana da reši ovaj problem.

## Pozicije za Static, Relative do Absolute

- **Statička pozicija**: Ovo je standardni tok dokumenta. Elementi se postavljaju odozgo nadole i sekvencijalno od leve ka desne.
- **Relativna pozicija**: Ovo vam omogućava da koristite atribute `top`, `left`, `right` i `bottom` da postavite poziciju elementa unutar standardnog toka dokumenta. Takođe možete koristiti relativnu poziciju da elementi preuzmu druge elemente na stranici.
- **Apsolutna pozicija**: Ovo vam omogućava da izvučete element iz standardnog toka dokumenta i da on radi nezavisno od drugih elemenata.

## Prostor za Fixed i Sticky

- **Fiksna pozicija**: Element postavljen na `position: fixed` uklanja se iz normalnog toka dokumenta i pozicionira se prema prikaznom prozoru, što znači da ostaje na mestu čak i kada korisnik skroluje stranicu. Ovo se koristi za elemente kao što su zaglavlja ili navigacioni barovi koji moraju biti vidljivi u svakom trenutku.
- **Lepkasta pozicija (Sticky)**: Ovaj tip pozicioniranja funkcioniše kao kombinacija relativnog i fiksnog pozicioniranja. Inicijalno, element se ponaša kao da je relativno pozicioniran, ostajući unutar toka dokumenta. Međutim, kada korisnik skroluje do određene tačke, on dodiruje prikazni prozor (obično na vrhu) i počinje da funkcioniše kao fiksni. Ovo je idealno za kreiranje elemenata poput lepljivih navigacionih barova, koji postaju fiksni tek kada korisnik skroluje do određene tačke.

## Rad sa karakteristikom `z-index`

- **Definicija**: Svojstvo `z-index` u CSS se koristi za kontrolisanje vertikalnog raspoređivanja elemenata sa specifičnim razmakom koji se preklapaju na stranici.

## Adaptivni dizajn mreže zavisno od uređaja

- **Definicija**: Osnovni princip responsivnog dizajna je sposobnost sajta da menja raspored i sadržaj u zavisnosti od veličine ekrana i kapaciteta uređaja sa kojeg se gleda.
- **Fluidne mreže**: One koriste relativne dimenzije, kao što su procenati (%), umesto fiksnih jedinica poput piksela, omogućavajući sadržaju da menja veličinu i pozicioniranje u zavisnosti od veličine ekrana.
- **Responsivne slike**: One su postavljene tako da menjaju veličinu unutar elemenata koji ih okružuju, osiguravajući da ne pređu svoje granice na manjim ekranima.

## Upiti medija

- **Opis**: Ovo omogućava programerima da postave različite stilove u zavisnosti od karakteristika uređaja, posebno širine vidokruga.
- **Tip medija `all`**: Pogodno za sve uređaje. Ovo je podrazumevano ako nije specificiran tip medija.
- **Tipovi medija `print`**: Cilja štampače i dokumente pregledane u režimu predgleda štampanja.
- **Tipovi medija `screen`**: Cilja pre svega ekrane.
- **`aspect-ratio`**: Ovo opisuje odnos između širine i visine vidokruga.
- **`orientation`**: Koristi se za prikaz da li je uređaj u horizontalnom ili vertikalnom orijentaciji.
- **`resolution`**: Koristi se za opisivanje rezolucije izlaznog uređaja u tačkama po inču (dpi) ili tačkama po centimetru (dpcm).
- **`hover`**: Koristi se za testiranje da li primarni metod unosa može interagovati sa elementima.
- **`prefers-color-scheme`**: Koristi se za detekciju da li je korisnik zahtevao svetlu ili tamnu temu boja.
- **Logički operateri i Media Query-ji**: Operater `and` koristi se za kombinovanje više medijskih karakteristika, dok se `not` i `only` mogu koristiti za negiranje ili razdvajanje media query-ja.

## Standardna ograničenja medija

- **Definicija**: Media query su specifični parametri u dizajnu sajta gde se raspored i sadržaj menjaju kako bi odgovarali različitim veličinama ekrana. Postoje opšti parametri koje možete koristiti za ciljanje mobilnih telefona, tableta i monitora računara. Međutim, nije razumno pokušavati da prate svaku veličinu ekrana za različite uređaje.
- **Mali uređaji (mobilni telefoni)**: do 640px
- **Srednji uređaji (tableti)**: od 641px do 1024px
- **Veliki uređaji (desktop računari)**: 1025px i više

## Prvi telefonski poziv

- **Definicija**: Mobile-first pristup je filozofija dizajna i razvojna strategija u responsivnom web dizajniranju koja daje prioritet kreiranju veb sajtova za mobilne uređaje pre nego što se dizajnira za velike ekrane.

## Specifične karakteristike za CSS (Parametri CSS)

- **Definicija**: Specifični atributi za CSS, poznati i kao parametri za CSS, su elementi definisani od autorima CSS koji sadrže specifične vrednosti za ponovnu upotrebu u celom dokumentu. To je moćna karakteristika koja omogućava stilskim listama da budu efikasnije, lakše za održavanje i prilagodljive. Specifični atributi su posebno važni pri kreiranju fleksibilnih struktura. Možete definisati skup atributa za različite teme.

## Propisi za `@property`

- **Definicija**: Pravilo `@property` je moćna karakteristika CSS koja omogućava programerima da definišu specifične atribute za veću kontrolu nad njihovim ponašanjem, uključujući kako se menjaju i njihove početne vrednosti.

```css
@property --property-name {
  syntax: '<type>';
  inherits: true | false;
  initial-value: <value>;
}
```

- **`--property-name`**: Ovo je ime specifičnog atributa koji definišete. Kao svi specifični atributi, mora početi sa dve linije/karaktera.
- **`syntax`**: Ovo opisuje tip atributa, koji mogu biti stvari kao `<color>`, `<length>`, `<number>`, `<percentage>`, ili složeniji tipovi.
- **`inherits`**: Ovo opisuje da li atribut treba da nasledi vrednost od roditeljskog elementa.
- **`initial-value`**: Ovo postavlja podrazumevanu (default) vrednost atributa.
- **Alternativni način**: Kada koristite specifičan atribut, možete pružiti alternativnu vrednost korišćenjem funkcije `var()`, kao što radite sa standardnim atributima za CSS.

## Temelji za CSS Grid

- **Definicija**: CSS Grid je dvodimenzionalni sistem rasporeda koji se koristi za kreiranje složenih rasporeda na veb stranicama. Grid se sastoji od redova i kolona sa razmacima između njih. Da bi definisao grid raspored, postavi svojstvo `display` na `grid`.
- **Jedinica `fr` (fraction)**: Ova jedinica predstavlja deo raspoloživog prostora unutar grid kontejnera. Možeš je koristiti za pravljenje fleksibilnih grid rasporeda.
- **Pravljenje razmaka između linija**: Postoje tri načina za pravljenje razmaka u CSS Grid-u. Koristi `column-gap` za razmake između kolona, `row-gap` za razmake između redova ili skraćeno svojstvo `gap` za oba istovremeno.
- **Funkcija `repeat()`**: Ova funkcija služi za ponavljanje delova liste linija. Umesto `grid-template-columns: 1fr 1fr 1fr;` možeš koristiti funkciju `repeat()`.
- **Eksplicitni grid**: Broj redova i kolona možeš definisati pomoću svojstava `grid-template-columns` ili `grid-template-rows`.
- **Implicitni grid**: Kada se elementi postave izvan definisanog grida, redovi i kolone se automatski kreiraju za te elemente.
- **Funkcija `minmax()`**: Ova funkcija služi za postavljanje minimalne i maksimalne veličine grid linije.
- **Automatsko postavljanje u Grid-u**: Pregledač automatski raspoređuje elemente pomoću algoritma za automatsko postavljanje. Svojstvo `grid-auto-flow` kontroliše ovo ponašanje pomoću vrednosti kao što su `row`, `column` ili `dense`.
- **Pozicioniranje pomoću linija**: Svi grid rasporedi imaju linije. Za određivanje početka elementa koriste se `grid-column-start` i `grid-row-start`, a za kraj `grid-column-end` i `grid-row-end`. Možeš koristiti i skraćena svojstva `grid-column` i `grid-row`.
- **`grid-template-areas`**: Ovo svojstvo služi za imenovanje oblasti u grid rasporedu.
- **Poravnanje**: Koristi `justify-items` i `align-items` za kontrolu poravnanja elemenata unutar njihovih grid oblasti. Koristi `justify-self` i `align-self` za pojedinačne grid elemente kako bi zamenio poravnanje definisano na kontejneru.

## Osnovi animacije za CSS

- **Opis**: Animacije CSS omogućavaju vam da kreirate zanimljive i vizuelne efekte na web stranicama bez potrebe za JavaScript ili složenim programima. One pružaju način za glatki prelaz elemenata između različitih stilova tokom definisanog perioda.
- **Pravilo za `@keyframes`**: Ovo pravilo opisuje korake i stilove animacije. Opisuje stilove koje element treba da ima u različitim delovima animacije.
- **Svojstvo `animation`**: Ovo je skraćeno svojstvo (shorthand property) koje se koristi za postavljanje animacije.
- **`animation-name`**: Ovo opisuje ime pravila `@keyframes` koje će biti korišćeno.
- **`animation-duration`**: Ovo postavlja vremenski period koji animacija treba da prođe do završetka.
- **`animation-timing-function`**: Ovo opisuje kako animacija teče tokom vremena (kao ease, linear, ease-in-out).
- **`animation-delay`**: Ovo postavlja odlaganje pre nego što animacija počne.
- **`animation-iteration-count`**: Ovo postavlja koliko puta animacija treba da se ponovi.
- **`animation-direction`**: Ovo određuje da li animacija radi napred, unazad ili se razmenjuje.
- **`animation-fill-mode`**: Ovo opisuje kako element treba da bude stilizovan pre i posle animacije.
- **`animation-play-state`**: Ovo vam omogućava da pauzirate i nastavite animaciju.

## Pristup Media Query-u za `prefers-reduced-motion`

- **Media Query za `prefers-reduced-motion`**: Jedno od glavnih pitanja pristupa i animacije je da mogu uzrokovati nelagodu ili čak štetu nekim korisnicima. Osobe sa vestibularnim poremećajima ili osećajem pokreta mogu dobiti vrtoglavicu, mučninu ili glavobolju kada su izložene određenim pokretima na ekranu. Media query `prefers-reduced-motion` omogućava web programerima da detektuju da li je korisnik tražio animaciju ili blage efekte pokreta u sistemu.
