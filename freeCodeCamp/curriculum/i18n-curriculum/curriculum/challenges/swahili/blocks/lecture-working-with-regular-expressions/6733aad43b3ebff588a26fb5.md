# --interactive--

Regularni izrazi, ili regex, je funkcija podržana od strane mnogih različitih programskih jezika.

Regularni izraz je posebna sintaksa za kreiranje "obrazca", koji se zatim može koristiti za validaciju sekvenci karaktera, ekstrakciju teksta i više.

Pregledajmo osnovnu, uobičajenu frazu:

```js
const regex = /freeCodeCamp/;
```

Primetite kako u JavaScript-u definišete regularni izraz tako što obrazac stavljate između dve kose crte (`/`). Pazite da ovo ne pomešate sa komentarima, gde tekst dolazi posle dve kose crte.

Ovaj uobičajeni izraz odgovara tekstu `freeCodeCamp`, sa velikim slovima `C`, bilo gde u nizu karaktera. Ali kako to zaista možete uraditi?

Ovo nas vodi do našeg prvog puta – puta za `test()`. Put za `test()` nalazi se u elementima `RegExp`, koji su elementi koji predstavljaju uobičajeni izraz (kao što smo upravo definisali).

Metoda `test()` prihvata string koji proverava da li odgovara regularnom izrazu. Na primer, hajde da testiramo string `e`:

:::interactive_editor

```js
const regex = /freeCodeCamp/;
const test = regex.test("e");
console.log(test);
```

:::

Možete videti da smo pozvali metodu `test()` nad našim novim regularnim izrazom i prosledili string `e` kao argument.

Metoda `test()` vratila je `false` zato što string `e` ne odgovara obrascu `freeCodeCamp`. Iako obrazac `freeCodeCamp` sadrži slovo `e`, regularni izrazi funkcionišu drugačije.

Hajde da pogledamo još nekoliko primera. Odvojite trenutak da ih pažljivo proučite:

:::interactive_editor

```js
const regex = /freeCodeCamp/;
console.log(regex.test("freeCodeCamp"));
console.log(regex.test("freeCodeCamp is great"));
console.log(regex.test("I love freeCodeCamp"));
console.log(regex.test("freecodecamp"));
console.log(regex.test("FREECODECAMP"));
console.log(regex.test("free"));
console.log(regex.test("code"));
console.log(regex.test("camp"));
```

:::

Primetite da su prva tri reda vratila `true`. Svi ovi stringovi negde u sebi sadrže tekst `freeCodeCamp`.

Redovi 5 i 6 vraćaju ``false``. Iako ima tekst ``freecodecamp``, slova ne odgovaraju. Standardna upotreba uzima u obzir slova po defaultu.

Konačno, iako poslednja tri reda imaju deo strukture, sekvenca slova ne obuhvata celu strukturu.

Putanja za `test()` vraća `boolean`, pokazujući da li niz slova odgovara potpuno normalnom izrazu.

Ali šta ako želite više informacija od toga? String ima metodu `match()`. Ona prihvata regularni izraz, mada možete proslediti i običan string koji će biti pretvoren u regularni izraz.

`match()` vraća red podataka po abecednom poretku. Šta je red podataka? Dobro, hajde da istražimo:

```js
const regex = /freeCodeCamp/;
const match = "freeCodeCamp".match(regex);
console.log(match);
```

Ako pokrenemo ovo, dobijamo vraćeni red! Ali to je neverovatan red za pregled. Ima dodatne karakteristike:

```js
// [
//   'freeCodeCamp',
//   index: 0,
//   input: 'freeCodeCamp',
//   groups: undefined
// ]
```

Funkcija `groups` će prikazati sve kreirane grupe. Naučićete njegovo značenje u sledećoj lekciji.

Svojstvo `index` vam govori koji je karakter pronađen unutar niza znakova na mestu podudaranja. U našem slučaju, pronađeno je na početku niza znakova.

Sifa `input` ti govori niz slova kojim je putovan po `match()`.

Hajde da isprobamo još nekoliko primera, i da vidimo kako se rezultati menjaju:

```js
const regex = /freeCodeCamp/;
console.log("freeCodeCamp".match(regex)); // ['freeCodeCamp', index: 0, input: 'freeCodeCamp', groups: undefined]
console.log("freeCodeCamp is great".match(regex)); // ['freeCodeCamp', index: 0, input: 'freeCodeCamp is great', groups: undefined]
console.log("I love freeCodeCamp".match(regex)); // ['freeCodeCamp', index: 7, input: 'I love freeCodeCamp', groups: undefined]
console.log("freecodecamp".match(regex)); // null
console.log("FREECODECAMP".match(regex)); // null
console.log("free".match(regex)); // null
```

Trebalo bi znati da prvih tri reda treba da daju rezultate, pa hajde da ih proverimo:

```js
// ['freeCodeCamp', index: 0, input: 'freeCodeCamp', groups: undefined]
// ['freeCodeCamp', index: 0, input: 'freeCodeCamp is great', groups: undefined]
// ['freeCodeCamp', index: 7, input: 'I love freeCodeCamp', groups: undefined]
```

Da li je ovo ono što ste očekivali? Možete videti kako `input` i `index` su se promenili prema datoj sekvenci slova, i gde je podudarnost pronađena unutar sekvence slova.

Tri druge linije koje ne odgovaraju, `null` umesto reda.

Sada kada možemo testirati i pronaći podudarnosti sekvenci karaktera našim uobičajenim govorom, šta ako želimo da promenimo sadržaj unutar sekvence karaktera? Možda je neko napisao `freecodecamp` sa svim malim slovima, a mi želimo da direktno izmenimo slovo za njih.

Prvo, moramo ispraviti naš uobičajeni izraz da odgovara tipu malog slova od `freecodecamp`, i kreirati naš test niz:

```js
const regex = /freecodecamp/;
const str = "freecodecamp is rly kewl";
```

String takođe ima metodu `replace()` koja prihvata dva argumenta: regularni izraz za pretragu (ili običan string ako vam nisu potrebne regex mogućnosti) i string kojim će se poklapanje zameniti (ili funkciju koja će biti pozvana za svako poklapanje).

Dakle, ako želimo da zamenimo `freecodecamp` pravilno napisanim tekstom:

:::interactive_editor

```js
const regex = /freecodecamp/;
const str = "freecodecamp is rly kewl";
const replaced = str.replace(regex, "freeCodeCamp");
console.log(replaced);
```

:::

Možete videti da `replace()` vraća ažurirani string u kojem je pronađeni obrazac zamenjen.

Regularni izrazi i sve metode povezane sa njima mogu delovati komplikovano. Međutim, imaćete priliku da ih detaljnije istražite tokom ovog kursa.
