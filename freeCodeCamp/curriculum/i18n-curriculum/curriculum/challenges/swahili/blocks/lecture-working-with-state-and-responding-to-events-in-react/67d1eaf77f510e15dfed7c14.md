# --description--

State je jedan od najvažnijih temelja za React i druge front-end sisteme. To je kao mozak komponente, što znači da čuva privremene i promenljive podatke te kontroliše kako se komponente prikazuju i funkcionišu.

Istražite u čemu je stanje i kako `useState` hook omogućava da radi sa njim.

Stanje predstavlja dinamičke podatke u vašem delu React, kao vrednost iz unosa korisnika, podatke dostupne iz API, ili stavku u listi zadataka.

Svaki put kada se state promeni, React ponovo prikazuje komponentu bez ponovnog učitavanja stranice kako bi prikazao promene u korisničkom interfejsu. Ova sposobnost menjanja čini aplikaciju interaktivnom.

`useState` hook je funkcija koja omogućava deklarisanje varijabli stanja u delovima funkcije.

Pre hook-ova, state se mogao koristiti samo u klasnim komponentama. Ali od uvođenja hook-ova u React 16.8, možete koristiti state u funkcionalnim komponentama pomoću `useState` hook-a.

Da biste koristili `useState` hook, morate ga dobiti (import) iz React:

```js
import { useState } from "react";
```

Takođe možete dovesti React samostalno i dobiti pristup `useState` hook kao karakteristiku:

```js
import React from "react";
```

Evo kako možete deklarisati state promenljivu prilikom uvođenja `useState`:

```js
const [stateVariable, setStateFunction] = useState(initialValue);
```

Evo kako možete deklarisati state promenljivu prilikom uvođenja React-a:

```js
const [stateVariable, setStateFunction] = React.useState(initialValue);
```

U parametru stanja imate sledeće:

- `stateVariable` inashikilia thamani ya sasa ya state
- `setStateFunction` (funkcija za postavljanje) ažurira state promenljivu.
- `initialValue` huweka state ya awali

Zapamtite da je state u komponenti React privatan i razdvojen za svaku instancu komponente. To znači da ako renderujete istu komponentu dva puta, state jedne komponente ne menja state druge. Ovo takođe znači da ako želite da podelite state između komponenti, morate podići taj state do običnog roditelja i proslediti ga kao propse (props).

Druga stvar je da se hook-ovi moraju pozivati na višoj nivou komponente, pre glavne reči `return`, kako bi se stanje i efekat održali stabilnim u svim prikazima. To znači da ne možete koristiti stanje unutar petlja, uslova ili ugnježdene funkcije.

Evo primera upravljanja stanjem korišćenjem `useState` hooka u sekciji `Counter`:

```jsx
// Importing the useState hook
import { useState } from "react";

function Counter() {
  const initialValue = 0;

  // The state variable and setter function
  const [count, setCount] = useState(initialValue);

  return (
    <div>
      {/* Display current state value */}
      <h2>{count}</h2>

      <button onClick={() => setCount(count - 1)}>Decrement</button>
      <button onClick={() => setCount(count + 1)}>Increment</button>
    </div>
  );
}

export default Counter;
```

U gornjem kodu, imamo `useState` hook preuzet iz React. U sekciji od `Counter`, `count` predstavlja trenutno stanje kada `setCount` je funkcija za postavljanje i ažuriranje stanja. Trenutna vrednost stanja (state) je `0`. Deklaracija `return` ima brojač i dve dugme za smanjenje i povećanje broja za `1`.

Moguće je upravljati više delova stanja pozivanjem `useState` hook više puta. Ovo je posebno važno kada imate neovisne parametre stanja:

```jsx
function UserProfile() {
  const [isOnline, setIsOnline] = useState(false);
  const [notifications, setNotifications] = useState(0);

  // The rest of the component logic
}
```

Takođe možete pozvati `useState` hook često kada upravljate sa više stanja koja se ažuriraju različito, kao što su delovi forme:

```jsx
function SignUpForm() {
  const [name, setName] = useState("");
  const [username, setUsername] = useState("");
  const [email, setEmail] = useState("");

  // The rest of the component logic
}
```

Međutim, u ovom slučaju, bolje je spojiti sve stanje jer su svi deo jednog obrasca:

```jsx
function SignUpForm() {
  const [formData, setFormData] = useState({
    name: "",
    username: "",
    email: "",
  });

  // The rest of the component logic
}
```

Tako je stanje i način na koji možete koristiti `useState` hook.
