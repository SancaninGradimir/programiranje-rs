# --description--

U prethodnim lekcijama ste naučili kako da radite sa osnovnim tipovima, kao i union tipovima i interfejsima.

Ali TypeScript ima posebne tipove koje treba da znate. Počnimo sa pregledom tipa `any`.

Vrsta `any` se koristi za predstavljanje bilo kakve vrste vrednosti.

```ts
let randomValue: any;

randomValue = 42;           

randomValue = "Hello";     
 
randomValue = true;        

randomValue = { name: "Alice" }; 
```

Za ovakav tip, trebalo bi biti pažljiv jer se lako može zloupotrijebiti i koristiti pogrešno. Ne želite da sve napišete u tipu `any` samo da smirite poruku o grešci TypeScript. Ovo narušava svrhu korišćenja TypeScript i povećanja sigurnosti tipa od samog početka.

Zamena za `any` će biti tip `unknown`. `unknown` je isto kao `any`, ali za tip `unknown`, treba da uradite proveru tipa promena pre korišćenja.

Evo primera:

```ts
function doubleValue(value: unknown) {
  if (typeof value === "number") {
    console.log(value * 2);
  } else if (typeof value === "string") {
    console.log(value + value);
  }
}

doubleValue(10);    
doubleValue("Hi "); 
doubleValue(true);   
```

U ovom primeru, provera tipa osigurava da se operacije izvršavaju samo sa vrednostima ispravnog tipa. Vrednosti koje ne odgovaraju očekivanom tipu nisu proverene.

Sledeća vrsta koju ćemo pogledati je vrsta `never`. Ova vrsta predstavlja nešto što nikada nije bilo.

U većini slučajeva, pisaćete opis tipa koristeći `never`. Umjesto toga, najverovatnije ćete videti tip `never` pojavljen u poruci greške kao što je ova:

```ts
function processValue(value: string | number) {
  if (typeof value === "string") {
    console.log("String value:", value.toUpperCase());
  } else if (typeof value === "number") {
    console.log("Number value:", value * 2);
  } else {
    console.log(value); 
  }
}
```

U okviru sekcije `else` iz gornjeg primera, `value` ima tip `never`. Ovo je zato što grana `else` više nije dostupan i nema preostalih tipova. Ako biste hteli da pozovete funkciju sa vrednostima različitim od `string` ili `number`, biće prikazana greška poput ove:

```ts
function processValue(value: string | number) {
  if (typeof value === "string") {
    console.log("String value:", value.toUpperCase());
  } else if (typeof value === "number") {
    console.log("Number value:", value * 2);
  } else {
    console.log(value); 
  }
}

// Argument of type 'boolean' is not assignable to parameter of type 'string | number'.
processValue(true); 
```

Poslednji tip koji ćemo pregledati je tip `void`. Ako imate funkciju koja ne vraća vrednost, to će biti tip `void`.

Evo primera:

```ts
type Status = "loading" | "success" | "error";

type Handler = {
  status: Status;
  onChange: (newStatus: Status) => void; 
};
```

U ovom primeru, svojstvo za `onChange` za tip `Handler` ne vraća ništa, zato se `void` koristi.
