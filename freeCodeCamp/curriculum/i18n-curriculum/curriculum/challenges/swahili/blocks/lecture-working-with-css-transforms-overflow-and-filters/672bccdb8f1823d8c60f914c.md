# --interactive--

Reset za CSS je list stilova koji se izdaje da ukloni podrazumevane stilove koje web pretraživači postavljaju za elemente HTML. Na primer, možda ste primetili da određeni elementi, kao što su paragrafi i naslovi sekcija, već imaju podrazumevane ivice čak i ako ih eksplicitno definišete u vašim običnim stilovima.

Ovo ćeš primetiti i kod raznih svojstava mnogih HTML elemenata. Različiti pregledači mogu primenjivati različite podrazumevane stilove na HTML elemente. Podrazumevani stilovi u Google Chrome-u možda neće biti isti kao u Mozilla Firefox-u ili Microsoft Edge-u. To može dovesti do nedoslednog izgleda između pregledača, što treba izbegavati koliko god je moguće.

Da biste rešili ovo, reset za CSS uklanja podrazumevane stilove. Uklanjanjem svih ili nekih podrazumevanih stilova, možete dobiti stabilan temelj za svoj dizajn i smanjiti razlike koje mogu nastati u pretraživačima i uređajima. Uklanjanje podrazumevanih stilova takođe može pojednostaviti proces stilizovanja jer ćete videti samo stilove koje ste eksplicitno definisali u svojim standardnim listama stilova. Postoje dve glavne metode za resetovanje CSS: možete kreirati sopstvene/lokalne resetove CSS ili koristiti tuđe resetove CSS.

Uobičajeni CSS reseti su listovi stilova koje praviš od početka kako bi odgovarali potrebama tvog projekta. Na taj način možeš kontrolisati koje će podrazumevane stilove resetovati i imati veću fleksibilnost. Međutim, potrebno je i vreme za razvoj i održavanje tih listova stilova. Evo primera poznatog CSS pravila koje resetuje margine i unutrašnji razmak za sve HTML elemente:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css">
<h1>Example Heading</h1>
<p>This is a paragraph.</p>
```

```css
* {
  margin: 0;
  padding: 0;
}
```

:::

Ovo pravilo se obično piše na samom početku CSS datoteke. Selektor `*` je univerzalni selektor koji odgovara svim HTML elementima, pa će svi imati podrazumevanu marginu `0` i podrazumevani unutrašnji razmak `0` na sve četiri strane. To ti daje čistu početnu osnovu, nakon čega možeš primenjivati željene stilove pomoću konkretnijih CSS selektora niže u datoteci.

Možeš koristiti ovaj pristup da izabereš bilo koji HTML element i vratiš njegove podrazumevane stilove. Jednostavno napravi prilagođeni CSS stylesheet, koristi odgovarajuće CSS selektore za elemente i postavi stilove.

Međutim, ovo može biti veoma dugotrajan proces. Ako želite da uštedite vreme, možete koristiti i CSS reset stilove trećih strana. Ovi stilovi su već napravljeni, pa ih možete preuzeti i direktno uključiti u svoj projekat. Dobar primer CSS reseta treće strane je `Normalize.css`. Ovaj stil ujednačava stilove za mnoge HTML elemente, a pritom zadržava važne podrazumevane stilove, naročito one povezane sa pristupačnošću. Takođe ispravlja uobičajene greške i razlike u prikazu stilova.

Druga mogućnost je `sanitize.css`. To je CSS biblioteka koju možeš koristiti kako bi podrazumevani stilovi bili dosledni u svim glavnim modernim pregledačima. Ova biblioteka je razvijena uporedo sa `Normalize.css`, pa se obe razvijaju zajedno. Takođe sadrži zasebne stylesheet-ove koje možeš preuzeti za posebne namene, poput stilizovanja formulara i tipografije.

Dostupno je mnogo opcija, ali treba da izabereš one koje najbolje odgovaraju potrebama tvog projekta. Takođe možeš kombinovati oba pristupa koristeći CSS reset biblioteke trećih strana zajedno sa prilagođenim resetima.

Kada radiš sa CSS resetima, važno je voditi računa i o pristupačnosti. Tvoja veb aplikacija treba da bude dostupna svima. Zato ne bi trebalo da resetuješ stilove koji mogu pomoći čitačima ekrana ili drugim asistivnim tehnologijama. Takođe treba uzeti u obzir uticaj ovih dodatnih stylesheet-ova na performanse aplikacije, jer moraju biti preuzeti pre nego što se primene tvoji prilagođeni stilovi.

Uklanjanjem podrazumevanih stilova, CSS resetovi vam pružaju praznu osnovu za primenu sopstvenog dizajna. To dovodi do doslednijeg i ujednačenijeg korisničkog iskustva u različitim pregledačima i na različitim uređajima.
