# --description--

Crypto je još jedan glavni modul izgrađen unutar Node.js. Uključuje alate za stvari kao što su hešovanje (hashing), enkripcija, dekripcija i kreiranje digitalnih potpisa, sve korišćeno za zaštitu osetljivih informacija i obezbeđivanje vašeg programa.

Međutim, `crypto` pruža samo osnovne niskonivojske alate, a ne gotovo bezbednosno rešenje. Pisanje sopstvenog koda za šifrovanje ili autentifikaciju može biti rizično ako nisi pažljiv. U većini slučajeva bolje je koristiti proverene biblioteke kao što su `bcrypt` za heširanje lozinki ili `jsonwebtoken` (JWT) za prijavljivanje i rad sa tokenima.

Međutim, i dalje je važno razumeti kako neke metode u kripto modulu funkcionišu.

Da bi koristio ove metode, potrebno je da uvezeš modul `crypto`:

```js
const crypto = require("crypto");
```

Neke metode služe za transformaciju podataka, kao što su heširanje i šifrovanje, dok su druge namenjene upravljanju ključevima i bezbednosti, poput generisanja nasumičnih vrednosti i kreiranja tajni.

Prvo istražimo metode хеширања и šifrovanja podataka.

Metod `createHash()` kreira hash objekat koristeći algoritme kao što su `sha256`, `sha512` ili `md5`. To je jednosmerna operacija, pa je nije moguće obrnuti.

`createHash()` je koristan za heširanje lozinki i pravljenje otisaka datoteka. Koristi se tako što proslediš algoritam, zatim metodom `update()` uneseš podatke, a na kraju metodom `digest()` i odgovarajućim kodiranjem dobiješ hash vrednost:

```js
const crypto = require("crypto");

const hashedPassword = crypto
  .createHash("sha256")
  .update("myStrongPassword")
  .digest("hex");

console.log("createHash result:", hashedPassword);
// createHash result: f92c9cfa0ead1bcec05ca75888a4074ba994ad237e5e2a8c7cc6a620378c061d
```

`createHmac()` radi gotovo isto što i `createHash()`, ali dodaje nivo bezbednosti tako što koristi tajni ključ. Samo onaj ko poseduje taj ključ može da potvrdi hash. Koristan je za autentifikaciju i proveru integriteta podataka:

```js
const crypto = require("crypto");

const hashedMessage = crypto
  .createHmac("sha256", "secretkey")
  .update("important-secret-message")
  .digest("hex");

console.log("createHmac result:", hashedMessage);
// createHmac result: da48d6f026b6036286b1fb872c63264130d5cc4271f3a213bb6ddca5a023e77e
```

Metode za `createCipheriv()` i `createDecipheriv()` obavljaju šifrovanje i dešifrovanje podataka. Sve primaju algoritme, ključeve i `iv`, koji je deo nasumičnih ili jedinstvenih podataka koji se koriste na početku procesa šifrovanja:

```js
createCipheriv(algorithm, key, iv);
createDecipheriv(algorithm, key, iv);
```

Za dešifrovanje moraš koristiti isti ključ. U suprotnom će dešifrovanje biti neuspešno i dobićeš grešku ili nečitljiv tekst umesto originalne poruke.

Evo ovih dve kada se pokrenu:

```js
const crypto = require("crypto");

// A key must match the algorithm length. Here AES-256 is 32 bytes
const key = Buffer.from("12345678901234567890123456789012");

// A fixed IV, 16 bytes for AES
const iv = Buffer.from("1234567890123456");

const cipher = crypto.createCipheriv("aes-256-cbc", key, iv);

let encrypted = cipher.update("Hello campers!", "utf8", "hex");
encrypted += cipher.final("hex");

console.log("Encrypted data:", encrypted);
// Encrypted data: 4ee93aa398ab44e3540e4a67ca96bc8c

// Decrypt the "Hello campers!" message
const decipher = crypto.createDecipheriv("aes-256-cbc", key, iv);
let decrypted = decipher.update(encrypted, "hex", "utf8");
decrypted += decipher.final("utf8");

console.log("Decrypted data:", decrypted);
// Decrypted data: Hello campers!
```

Drugi način kripto za transformaciju podataka je `sign()` na `verify()`.

`sign()` Kreira digitalni potpis iz određenih podataka koristeći privatne ključeve. Ovaj potpis potvrđuje da podaci dolaze od vlasnika privatnog ključa i nisu izmenjeni.

`verify()` Zatim proverava taj potpis, i neuspešno je ako podaci ili potpis ne odgovaraju.

Sada, istražimo kripto metode za generisanje nasumičnih vrednosti i kreiranje tajni.

`randomBytes()` prima parametar `size` i generiše kriptografski bezbedne tokene. Zato je pogodan za generisanje UUID identifikatora. Takođe predstavlja bolju alternativu za `Math.random()`, koji nije dovoljno bezbedan za tokene i ključeve.

```js
console.log("Random Bytes:", crypto.randomBytes(16));
// Random Bytes: <Buffer 01 88 aa 1e 2c 38 48 39 26 e1 6b a9 d8 c5 ed 49>
```

Rezultat je buffer podrazumevan. Kako ste naučili u tutorijalu o Buffer modu, možete konvertovati taj Buffer u string korišćenjem metode `toString()`:

```js
console.log("Random Bytes:", crypto.randomBytes(16).toString("hex"));
// Random Bytes: a6154ef5a296fa176ad0f332bd94d712
```

Hoja ya `'hex'` katika `toString('hex')` ovde govori Node da napiše binarne podatke iz Buffer-a kao sekvencu heksadecimalnih karaktera.

Putanja za `randomInt()` prima `min` i `max` da generiše kompletan, siguran slučajni broj između njih. Bitno je za OTP-ove i nasumičan izbor.

```js
console.log("Random Int:", crypto.randomInt(0, 100)); // 89
```

Takođe, poboljšanje za `Math.random()` i `Math.floor()` je u tome što ova metoda koristi siguran slučajni kriptografski algoritam 'ispod haube', pa hakeri ne mogu predvideti slučajni broj koji će nastati.

Alternativan način je `createSecretKey()`. Prima *buffer* i proizvodi zakrivljene sirove bajtove u `KeyObject`:

```js
const crypto = require("crypto");

const secret = crypto.createSecretKey(crypto.randomBytes(32));
console.log(secret); // SecretKeyObject [KeyObject] { [Symbol(kKeyType)]: 'secret' }
```

Zatim možete koristiti metodom `export` da pošaljete `KeyObject` to:

```js
const secret = crypto.createSecretKey(crypto.randomBytes(32));
console.log(secret.export());
// <Buffer 53 06 a1 c7 75 69 8b 38 8b a4 b2 f7 1b bc b8
// ae e2 d1 bf 67 af 1a 6a 0a 6e a0 29 62 bb 52 52 32>
```

Na kraju pozovi `toString()` nad baferom da vidiš njegov tekstualni prikaz:

```js
const secret = crypto.createSecretKey(crypto.randomBytes(32));
console.log(secret.export().toString('hex'));
// 32dfe5917668580160986f1623bf8152913329c71163be9c3404a110cd78efd6
```

Mbali na hizi, kuna:

- `createPublicKey()` na `createPrivateKey()` zinazokuwezesha kufanya kazi na funguo zilizozalishwa mahali pengine
- `createDiffieHellman()` omogućava dvema stranama da generišu zajedničku tajnu bez njenog direktnog slanja.
- `Certificate()` služi za rad sa sertifikatima koji se koriste u HTTPS-u, tako da možeš da analiziraš, izvoziš i proveravaš njihov sadržaj.
