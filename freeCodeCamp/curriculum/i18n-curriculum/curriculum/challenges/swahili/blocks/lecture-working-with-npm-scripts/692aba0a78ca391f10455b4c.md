# --description--

Sada, nakon razumevanja CommonJS sistema modula, hajde da istražimo ES sistem modula.

ES moduli, ili ECMAScript moduli, su formalna struktura, standardizovana za JavaScript kako je definisano specifikacijama ECMAScripta. Oni koriste deklaraciju od `import` i `export`, a u Node.js, učitavaju se asinhrono, što je bolje.

Node.js koristi fajlove kao ES module za ES u sledećim uslovima:

* Datoteke sa ekstenzijom `.mjs`.

* U datotekama sa ekstenzijom `.js` kada najbliži fajl `package.json` ima polje visokog nivoa `"type"` sa vrednošću `"module"`.

Ovde imamo funkciju `multiply`, isto što ste ranije videli, ali sada ćemo koristiti ESM sistem. Zbog toga, dodajemo dodatak za `.mjs` oba modula.

Da bi eksportovao funkciju `multiply` iz modula `math.mjs`, koristi ključnu reč `export`, nakon koje sledi naziv funkcije, promenljive ili objekta koji želiš da eksportuješ unutar vitičastih zagrada:

```javascript
function multiply(a, b) {
  return a * b;
}

export { multiply };
```

Funkciju možeš eksportovati i direktno tako što ćeš staviti ključnu reč `export` ispred ključne reči `function`. Ovo se naziva imenovani eksport. Jedan modul može imati više imenovanih eksportovanja:

```javascript
export function multiply(a, b) {
  return a * b;
}
```

Zatim, da bi importovao ovaj modul, koristi ključnu reč `import`, nakon koje slede vitičaste zagrade. Obrati pažnju da koristimo `import` umesto `require()`. To je glavna razlika između CommonJS i ES modula.

Unutar vitičastih zagrada navodimo naziv funkcije, promenljive ili objekta koji želimo da importujemo. U ovom slučaju to je `multiply`. Zatim koristi ključnu reč `from` i relativnu putanju do modula. Ovaj kod će se uspešno izvršiti i rezultat će biti `42`.

```javascript
import { multiply } from './math.mjs';

console.log(multiply(6, 7)); // 42
```

U ES sistemu možeš imati jedan podrazumevani eksport po modulu. U tom slučaju nije potrebno koristiti vitičaste zagrade prilikom importovanja.

Evo primera u kojem je funkcija `multiply` postavljena kao podrazumevani eksport za ceo modul `math.mjs`:

```javascript
export default function multiply(a, b) {
  return a * b;
}
```

Sada nam ne trebaju vitičaste zagrade, jer će funkcija biti uključena i postavljena na podrazumevanu vrednost:

```javascript
import multiply from './math.mjs';
```

Ako ne koristite ekstenzije fajlova ili fajl ``package.json`` da specificirate za `Node.js` vrstu sistemskih modula koje želite koristiti, on će pregledati izvorni kod tražeći ES modul sinaksu. Ako se ta sinaksa pronađe, tretiraće kod kao ES modul; inače, tretiraće ga kao CommonJS modul.

Kada je CommonJS bio korišćen kao podrazumevani standard za Node.js u svojim ranim danima, dok programeri koriste ESM zbog njegove efikasnosti i kompatibilnosti sa modernom JavaScript, važno je znati njihove razlike i kako da koristite oba.
