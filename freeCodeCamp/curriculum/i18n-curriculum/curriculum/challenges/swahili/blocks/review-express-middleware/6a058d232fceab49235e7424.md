## Middleware šta je?

- **Middleware** je funkcija koja se izvršava tokom ciklusa zahteva na serveru.
- Middleware ima pristup **objektu zahteva** (`req`), **objektu odgovora** (`res`) i **funkciji `next`**.
- Uobičajena upotreba: logovanje zahteva, obrada grešaka, analiza dolaskih podataka i serviranje statičkih fajlova.
- Middleware se izvršava u redosledu njegovog dodavanja (registracije) u aplikaciju.

### Kako middleware radi

1. Zahtev se šalje na server.
2. Funkcije *middleware*-a obrađuju zahtev.
3. Ako *middleware* pozove `next()`, kontrola se prenosi na sledeći *middleware* ili rukovaoca putanje podataka.
4. Ako *middleware* ne pozove `next()`, ciklus zahtev-odgovor prestaje.

```javascript
app.use((req, res, next) => {
  console.log('Request URL:', req.url);
  console.log('Request Method:', req.method);
  next();
});
```

- `app.use()` dodaje centralnu komponentu generalno u program.

## Program srednjeg nivoa

- **Srednji programski sloj** je povezan sa primerom programa Express.
- Registrovano korišćenjem `app.use()` ili specijalnih putanja podataka kao što su `app.get()`, `app.post()`, itd.
- Dešava se za svaki dolazni zahtev osim ako nije blokiran specijalnom putanjom podataka.
- Upotreba: logovanje, verifikacija identiteta i rukovanje greškama u više puteva podataka.

```javascript
app.use((req, res, next) => {
  console.log('Request received at:', new Date());
  next();
});
```

## Program za pronalaženje putanja srednjeg nivoa

- **Intermediarni program na nivou rutera** zatvoren je sa primerom `express.Router()` umesto glavnog programa.
- Koristi se koristeći `router.use()` i utiče samo na putanje podataka koje obrađuje taj ruter.
- Pomaže u razdvajanju programa na nezavisne module putanja podataka.
- Instalirajte ruter sa `app.use('/path', router)` ili postavite opseg međuprograma za specifične putanje podataka.

```javascript
const router = express.Router();

router.use((req, res, next) => {
  console.log('Request made to /menu route');
  next();
});

router.get('/drinks', (req, res) => {
  res.send('Welcome to the drinks menu!');
});

app.use('/menu', router);
```

## Middleware za obradu grešaka

- **Middleware za obradu grešaka** je poseban middleware koji se koristi za hvatanje grešaka tokom ciklusa zahtev-odgovor.
- Ima **četiri parametra**: `err`, `req`, `res`, i `next`. Potpis ovih četiri parametra pokazuje za Express da se radi o obradiocu grešaka.
- Poziva se kada greška prođe kroz `next(err)`.
- Trebalo bi ga postaviti **na kraj** lanca middleware-a, nakon svih drugih middleware komponenti i ruta podataka.

```javascript
app.use((err, req, res, next) => {
  console.error(err.stack);
  res.status(500).send('Something went wrong!');
});
```

## Centralni program razvijen unutar i od strane drugih likova

### Centralno razvijeni interni program

- **`express.json()`**: Detektovanje dolaznih aplikacija koje sadrže JSON.

```javascript
app.use(express.json());
```

- **`express.static()`**: serviranje statičkih fajlova kao slika, CSS, i JavaScript.

```javascript
app.use(express.static('public'));
```

- **`express.urlencoded()`**: analizira dolazne zahteve sa opterećenjem URL-encoded (struktura poslata putem formulara HTML); čini analizirane podatke dostupnim na `req.body`.

```javascript
app.use(express.urlencoded({ extended: true }));
```

### Centralni program za druge likove

- **`cors`**: dozvoljava Cross-Origin Resource Sharing.

```javascript
app.use(cors());
```

- **`morgan`**: beležava zahteve za HTTP.

```javascript
app.use(morgan('tiny'));
```

- Funkcije centralnog programa koje su ugrađene i one drugih komponenti dodaju se po redosledu pozivanja `app.use()`.
