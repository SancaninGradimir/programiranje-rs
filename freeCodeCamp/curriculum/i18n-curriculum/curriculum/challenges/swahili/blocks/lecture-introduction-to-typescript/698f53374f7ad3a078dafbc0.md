# --description--

U ovom tutorijalu, naučićete kako da dodate tipove objekata i redova podataka.

Prvo proverimo redove podataka.

Ovo je primer niza programskih jezika:

```js
const programmingLanguages = ["Java", "C++", "Python"];
```

Ako želite da eksplicitno postavite taj tip niza kao niz stringova, možete ga napisati kao `string[]`:

```ts
const programmingLanguages:string[] = ["Java", "C++", "Python"];
```

Kao što ste ranije naučili, TypeScript će biti u stanju da prepozna tip kao niz sekvence karaktera ako napišete samo ovako:

```js
const programmingLanguages = ["Java", "C++", "Python"];
```

Međutim, eksplicitno određivanje tipova će pomoći u situaciji kada pokušavate da dodate drugi tip u red kao ovaj:

```ts
// Type 'number' is not assignable to type 'string'.(2322)
const programmingLanguages:string[] = ["Java", 9, "Python"];
```

Drug način za postavljanje tipa kolone je korišćenje ove sintakse:

```ts
const programmingLanguages:Array<string> = ["Java", "C++", "Python"];
```

Podrazumevano, ove dve sintakse su iste, a izbor između njih je uglavnom stvar lične preference.

Sada pogledajmo kako se postavljaju tipovi za objekte.

Ovde je primer nečega doslovnog:

```js
const developer = {
  firstName: "Jessica",
  isEmployed: true
}
```

Ako želiš eksplicitno definisati tipove atributa za nešto, možeš koristiti ovu sintaksu:

```ts
const developer: { firstName: string, isEmployed: boolean } = {
  firstName: "Jessica",
  isEmployed: true
}
```

Ako pokušate dodati ili ukloniti svojstvo sa objekta, onda ćete dobiti grešku kao ovu:

```ts
// Property 'isEmployed' is missing in type '{ firstName: string; }' but required in type '{ firstName: string; isEmployed: boolean; }'
const developer: { firstName: string, isEmployed: boolean } = {
  firstName: "Jessica",
}
```

Možete učiniti da su svojstva objekta opciona koristeći `?` nakon imena svojstva kao što je ovo:

```ts
const developer: { firstName: string, isEmployed?: boolean } = {
  firstName: "Jessica",
}
```

TypeScript može nastati greška jer je svojstvo od `isEmployed` postavljeno kao opcionalno.

Dakle, sve instance objekata imale su ograničene atribute. To znači da smo jasno naveli svaki atribut koji objekat može imati:

```ts
const developer: { firstName: string, isEmployed: boolean } = {
  firstName: "Jessica",
  isEmployed: true
}
```

Ali šta ako ne znate sva imena svojstava unapred? Šta ako želite nešto što može imati bilo koji broj svojstava, ali sve vrednosti moraju pratiti određeni tip?

Zapravo, postoje dva načina za postizanje ovoga. Prvi način je korišćenje `Record`, koji je vrsta generičkog utiliteta ugrađenog unutar TypeScript.

Ovo je osnovna sintaksa:

```ts
Record<Keys, ValueType>
```

Prvi tip predstavlja tip ključa, dok drugi tip predstavlja tip vrednosti.

Ovo je primer korišćenja `Record` za `userRoles`:

```ts
const userRoles: Record<string, string> = {
  admin: "full-access",
  editor: "limited-access",
  viewer: "read-only"
};
```

Ovo polje `userRoles` može imati bilo koje ključeve sa nizom karaktera, a sve vrednosti moraju biti niz karaktera. Ako pokušate da postavite vrednost koja nije niz karaktera, doći ćete do greške:

```ts
// Type 'number' is not assignable to type 'string'
const userRoles: Record<string, string> = {
  admin: 1
};
```

Drugi način za zapisivanje iste ideje je korišćenje uzorka potpisa:

```ts
const obj: { [key: string]: string } = {};
```

Sintaksa iznad kaže da svako svojstvo koje ima ključeve tipa string mora imati vrednost tipa string. Evo primera objekta za `settings`:

```ts
const settings: { [key: string]: string } = {
  theme: "dark",
  language: "en",
  layout: "grid"
};
```

Kao što je ranije, ako imate neispravne tipove vrednosti, TypeScript će prikazati poruku o grešci:

```ts
// Type 'boolean' is not assignable to type 'string'
const settings: { [key: string]: string } = {
  darkMode: true
};
```

Sve `Record` na osnovnom modelu potpisa su iste. Stoga, izbor među njima je obično pitanje ličnog ukusa.
