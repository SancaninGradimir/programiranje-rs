# --description--

Da li vam se ikada desilo da je vaš program Express pao bez jasne poruke o grešci? Debugovanje grešaka i snimanje može pomoći da vidite šta se tačno dešava iza kulisa, kako biste problem pronašli i rešili brže.

U ovom tutorijalu, ćemo pogledati kako Express rukuje ispravljanjem grešaka i logovanjem—i kako ga možete koristiti za poboljšanje vašeg razvojnog procesa.

## Ispravljanje ugrađenih grešaka sa Express

Express već ima alat za popravljanje grešaka koji se zove `debug`. Nije potrebno instalirati ništa da biste ga koristili.

Za ispravljanje grešaka, ono što morate da uradite je postaviti varijablu okruženja za `DEBUG` kada pokrenete svoj program.

Evo primera:

```js
DEBUG=express:* node index.js
```

Ova komanda omogućava sve poruke za ispravljanje grešaka ispod polja imena `express:*`. Početi ćete da vidite rezultate kao:

```js
express:router:route dispatching GET /
express:router:layer processing GET /
```

Ovaj uvid vam pomaže da pratite distribuciju putanja podataka, pozive međusobnog softvera i više. To je odličan način za razumevanje toka zahteva unutar vaše aplikacije.

Želite li da budete specifičniji? Možete smanjiti polje imena:

```js
DEBUG=express:router node index.js
```

Ovo će prikazati samo zapise povezane sa logikom putnika za Express.

Kwenye Windows, sintaksia ni tofauti kidogo kwa PowerShell:

```js
$env:DEBUG = "express:*"; node index.js
```

Na CMD, izgleda ovako:

```js
set DEBUG=express:* && node index.js
```

Ideja je u redu, to je drugačiji način postavljanja parametara okruženja.

Sada proverimo zapis u Express.

Za snimanje zahteva za HTTP, mnogi programeri koriste srednju aplikaciju kao `morgan`.

Evo kako to koristiš:

```bash
npm install morgan
```

Zatim dodajte u vaš program:

```js
const express = require('express')
const morgan = require('morgan')
const app = express()

app.use(morgan('dev'))s
```

Sada, svaki dolazni zahtev će biti zapisano na terminalu i detalji kao što su, URL, statusni broj, i vreme odgovora.

I zašto je ovo važno?

Debugovanje vam pruža uvid kako Express rukuje aplikacijama, middleware-om i rutama bez punjenja vaših produkcionih zapisa.

Logovanje vam daje zapis onoga što se dešava u aplikaciji, što je važno za pronalaženje grešaka, razumevanje saobraćaja i rano otkrivanje problema.

Zajedno, pružamo vam kontrolu i transparentnost u vašem procesu razvoja.
