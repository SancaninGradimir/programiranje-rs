# --description--

U JavaScript, `Map` je ugrađeni objekat koji skladišti parove ključ-vrednost, kao što je običan objekat. Međutim, razlikuje se od običnih objekata JavaScript jer dozvoljava sve vrste ključeva, uključujući objekte i funkcije.

`WeakMap` je kolekcija parova ključ-vrednost, kao `Map`, ali koristi slabe reference za svoje ključeve. Ključevi moraju biti objekti, dok vrednosti mogu biti bilo kog tipa.

Razmotrite odnos između `WeakMap` i `Map` kao onaj između `WeakSet` i `Set`, kako ste naučili u prethodnoj lekciji.

Za kreiranje `Map`, koristite komponentu `Map()` generisanu glavnom rečju `new`:

```js
const myFirstMap = new Map();
```

Možete inicijalizovati Map sa vrednošću:

```js
const myTreesMap = new Map([
 [{ type: 'deciduous' }, 'Maple tree'],
 [['forest', 'grove'], 'Pine tree'],
 [42, 'Oak tree'],
 [true, 'Birch tree'],
 [function() { return 'I am a function key'; }, 'Willow tree'],
]);
```

Ako počnete `Map` sa vrednošću, možete koristiti metodu za `set()` povećanje vrednosti:

```js
const myTreesMap = new Map();
myTreesMap.set({ type: 'deciduous' }, 'Maple tree');
myTreesMap.set([1, 2], 'Pine tree');
myTreesMap.set(42, 'Oak tree');
myTreesMap.set(true, 'Birch tree');
myTreesMap.set(function() { return "I'm a function key"; }, 'Willow tree');

console.log(myTreesMap);
```

Ovo je kako `Map` izgleda na konzoli:

```js
/*

  [[Entries]]
    0:{Object => "Maple tree"}
      key: {type: 'deciduous'}
      value: "Maple tree"
    1:{Array(2) => "Pine tree"}

```

Drugi način koji možete koristiti za rad sa `Map` je:

- `get(key)` Dohvatiti vrednost povezanu sa `key` pomenutom.
- `has(key)` Proveriti da li `key` postoji u `Map`.
- `delete(key)` Ukloniti par ključ-vrednost iz `Map`.
- `clear()` Ukloniti sve parove ključ-vrednost.
- `entries()` Proveriti unos za `Map` (ponovo vraća unos u `MapIterator`).
- `forEach()` Ponoviti unos za `Map`.
- `size` Prikazati broj parova ključ-vrednost u `Map`.

Postoji komponenta `WeakMap()` koju možete koristiti za kreiranje `WeakMap`:

```js
const myFirstWeakMap = new WeakMap();
```

Načini za `set()`, `get()`, `has()`, na `delete()` su sve dostupni za korišćenje i sa `WeakMap`. Na primer, ovako možete staviti stvari u `Map` koristeći način `set()`:

```js
const myTreeWeakMap = new WeakMap();

myTreeWeakMap.set({ id: 1 }, 'Maple tree');
myTreeWeakMap.set({ id: 2 }, 'Pine tree');
myTreeWeakMap.set({ id: 3 }, 'Oak tree');
myTreeWeakMap.set({ id: 4 }, 'Birch tree');
myTreeWeakMap.set({ id: 5 }, 'Willow tree');

console.log(myTreeWeakMap);
```

Ovo se `WeakMap` prikazuje u konzoli:

```js
/*
WeakMap {{…} => 'Willow tree', {…} => 'Maple tree', {…} => 'Pine tree', {…} => 'Oak tree'}
  [[Entries]]
    No properties
  [[Prototype]]: WeakMap
*/
```

Ovo je razlika između `Map` i `WeakMap`:

| **Element** | **Map** | **WeakMap** |
| --- | --- | --- |
| **Tip ključa** | Ključevi mogu biti bilo kog tipa podataka, uključujući nizove karaktera, brojeve, objekte ili čak funkcije. | Ključevi moraju biti objekti. |
| **Upotreba** | Koristite `Map` kada vam je potrebno povezati podatke sa bilo kojim tipom ključa. | Koristite `WeakMap` samo kada vam je potrebno povezati podatke sa objektima. |
| **Iterabilnost** | Možete iterirati `Map` koristeći `forEach()`, `keys()`, `values()` ili `entries()`. | `WeakMap` nije iterabilan. |
| **Svojstvo veličine** | `Map` ima svojstvo `size` za dobijanje broja parova ključ-vrednost. | `WeakMap` nema svojstvo `size`. |

U tabeli iznad, možete videti pregled razlika, uključujući tipove ključa, primenu, ponavljanje i karakteristike veličine. Molimo vas da odvojite vreme za čitanje sadržaja ove tabele kako biste saznali više o njihovim razlikama.

## --text--

Koja od ovih razliku `Map` i `WeakMap` u JavaScript?

## --answers--

`Map` dozvoljava samo stvari poput ključeva, dok `WeakMap` dozvoljava ključeve bilo kog tipa.
