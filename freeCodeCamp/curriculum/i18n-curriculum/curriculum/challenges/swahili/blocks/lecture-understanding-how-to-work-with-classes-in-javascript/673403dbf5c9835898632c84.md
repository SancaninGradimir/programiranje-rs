# --interactive--

Statička svojstva i metode pripadaju samoj klasi, a ne pojedinačnim instancama te klase. Možeš im pristupiti direktno pomoću imena klase, bez kreiranja instance. Definišu se unutar klase kako bi objedinjavale povezanu funkcionalnost.

Možete definisati putanju za static tako što ćete napisati glavnu reč `static` pre imena putanje.

```js
class MyClass {
  static staticMethod() { ... }
}
```

Zatim, možete pozvati metodu static direktno u klasi, koristeći notaciju tačka i prosleđujući sve potrebne argumente:

```js
MyClass.staticMethod();
```

Obrati pažnju da ovu metodu možeš pozvati bez kreiranja instance klase. To je jedna od glavnih osobina statičkih metoda.

Evo primera. Pretpostavimo da pravimo aplikaciju za filmove i želimo da možemo da upoređujemo filmove na osnovu njihovih ocena.

Možemo smatrati ovu komparativnu metodu kao pristup višeg nivoa koji nije specifičan za bilo koje kino:

```js
if (movieA.rating < movieB.rating) {
  console.log(`${movieB.title} has a higher rating.`);
} 
```

To je opštija metoda koja je povezana sa klasom `Movie`.

Radi lakšanja i dobrog održavanja, biće korisno definisati ga unutar klase `Movie` ili postaviti sve povezane metode blizu svake.

Zbog toga je ovo dobar kandidat za statičku metodu. Možeš je videti odmah ispod konstruktora:

:::interactive_editor

```js
class Movie {

  constructor(title, rating) {
    this.title = title;
    this.rating = rating;
  }

  static compareMovies(movieA, movieB) {
    if (movieA.rating > movieB.rating) {
      console.log(`${movieA.title} has a higher rating.`);
    } else if (movieA.rating < movieB.rating) {
      console.log(`${movieB.title} has a higher rating.`);
    } else {
      console.log("These movies have the same rating.");
    }
  }

}

let movieA = new Movie("Movie A", 80);
let movieB = new Movie("Movie B", 45);
console.log(movieA);
```

:::

Statička metoda je definisana ključnom rečju `static` i zove se `compareMovies`. Prima dva parametra: `movieA` i `movieB`. Oni predstavljaju instance klase `Movie`.

Uporedićemo ih na osnovu njihovih ocena, u opsegu od `0` do `100`. Ova logika je implementirana pomoću uslovnih izraza i ispisuje odgovarajuću poruku u zavisnosti od toga koji film ima veću ocenu.

Kada je put definisan, možete ga pozvati u klasi. Ali prvo, morate imati spreman argument i deklarisani su u vašem programu.

U ovom slučaju, metoda uzima dva filmska događaja kao argumente, pa definišemo ove događaje ovde:

```js
let movieA = new Movie("Movie A", 80);
let movieB = new Movie("Movie B", 45);
```

Možeš videti da `movieA` ima veću ocenu od `movieB`. Pogledajmo rezultat ove metode.

Da biste pozvali metodu, samo je potrebno koristiti notaciju tačke na samoj klasi.
Pišete ime klase (`Movie`), praćeno tačkom, zatim naziv metode static (`compareMovies`).

Zatim, prosleđujete/unesete izjavu unutar običnih zagrada. U ovom slučaju, potrebni su dva filmska događaja koje put zahteva.

```js
Movie.compareMovies(movieA, movieB);
```

Ovo je ažurirani primer:

:::interactive_editor

```js
class Movie {

  constructor(title, rating) {
    this.title = title;
    this.rating = rating;
  }

  static compareMovies(movieA, movieB) {
    if (movieA.rating > movieB.rating) {
      console.log(`${movieA.title} has a higher rating.`);
    } else if (movieA.rating < movieB.rating) {
      console.log(`${movieB.title} has a higher rating.`);
    } else {
      console.log("These movies have the same rating.");
    }
  }

}

let movieA = new Movie("Movie A", 80);
let movieB = new Movie("Movie B", 45);
Movie.compareMovies(movieA, movieB);
console.log(movieA);
```

:::

Metode za static takođe pomažu u implementaciji "factory" metoda. Factory metoda je način kojim definišete, zajedno sa komponentom, kako biste kreirali objekte na osnovu specifičnih parametara.

Ovde je primer sa klasom `Pizza`. Metoda static `createMargherita` je fabrička metoda koju možete pozvati da biste kreirali instancu pizze Margherita sa njenim tipom i cenom koji su već postavljeni.

```js
class Pizza {
  constructor(type, price) {
    this.type = type;
    this.price = price;
  }

  static createMargherita() {
    return new this("Margherita", 6.99);
  }
}
```

Ovo takođe predstavlja veoma važnu stvar o static.

Vrednost ključne reči `this` u metodama static je sama klasa, jer metoda static pripada klasi.

Stoga možemo koristiti `this` da kreiramo novi događaj klase za `Pizza`.

Ukoliko pozovete ovu metodu u klasi `Pizza` sama i postavite vraćeni događaj kao parametar, kao što je ovde primer:

```js
let myPizza = Pizza.createMargherita();
```

Možete ga koristiti bilo gde želite u vašem kodu. Na primer, možete ga ispisati na konzolu:

```js
console.log(myPizza);
```

Ovo su rezultati:

```js
Pizza { type: 'Margherita', price: 6.99 }
```

Takođe možete koristiti dot notaciju za pozivanje njegovih putanja i pristup njihovim svojstvima, kao što je ovo:

```js
console.log(myPizza.type);
```

Ovo je kompletan primer:

```js
class Pizza {
  constructor(type, price) {
    this.type = type;
    this.price = price;
  }

  static createMargherita() {
    return new this("Margherita", 6.99);
  }
}

let myPizza = Pizza.createMargherita();
console.log(myPizza);
console.log(myPizza.type);
```

Pored toga, takođe možete definisati karakteristike static sa glavnom rečju `static`.

U ovom primeru, imamo karakteristiku od static `numberOfPizzasSold`.

```js
class Pizza {
  static numberOfPizzasSold = 0;

  constructor(type) {
    this.type = type;
    Pizza.numberOfPizzasSold++;
  }
}
```

Ni static zato što ne poseduje bilo koji događaj sa pizzom, to je svojstvo same klase.

Ima početnu vrednost od `0` i ažurira se svaki put kada se kreira novi događaj.

Ako kreirate dva događaja pizze, vrednost će biti ažurirana dvaput:

```js
let pizza1 = new Pizza("Margherita");
let pizza2 = new Pizza("Neapolitan");
```

Da biste pristupili vrednosti svojstva static, samo morate koristiti notaciju tačke na samoj klasi, jer je svojstvo član te klase.

```js
class Pizza {
  static numberOfPizzasSold = 0;

  constructor(type) {
    this.type = type;
    Pizza.numberOfPizzasSold++;
  }
}

let pizza1 = new Pizza("Margherita");
let pizza2 = new Pizza("Neapolitan");

console.log(Pizza.numberOfPizzasSold);
```

U ovom slučaju, rezultat je `2` zato što su bile prodane dve pice.

Ovo su osnove karakteristika i metoda za static u JavaScript. Razumevanje članova od static je važno za kreiranje ponovo upotrebljivih i efikasnih klasa.

Koja je glavna razlika između putanje u static i događajne putanje u JavaScript?

## --answers--

Metode za static mogu se koristiti samo unutar klase, dok događajne metode mogu se koristiti van klase.
