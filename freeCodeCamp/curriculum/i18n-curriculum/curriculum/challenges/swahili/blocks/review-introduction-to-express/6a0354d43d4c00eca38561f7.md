## Usluge mreže i API

- **API (Application Programming Interface)**: Opšta metoda za komunikaciju između programa.
- **Mrežna usluga**: Poseban tip API koji koristi mrežne tehnologije kao HTTP za razmenu podataka preko mreže. Svaka mrežna usluga je API, ali ne svako API je mrežna usluga.
- **Ključne karakteristike mrežne usluge**: radi na serveru, omogućava komunikaciju između mašina, ne zavisi od jezika, ne zavisi od platforme i koristi standardne mrežne protokole (HTTP/HTTPS, FTP, SMTP).
- **REST (pregledni prenos stanja (REST))**: Najčešći tip mrežne usluge danas. Eksponuje resurse identifikovane metodama URL kao `/users` ili `/users/20`. Odgovori su obično u JSON, ali mogu biti i XML ili običan tekst.
- **SOAP (Simple Object Access Protocol)**: Koristi samo XML. Često se koristi u korporativnim sistemima koji zahtevaju strože standarde i sigurnost.
- **XML-RPC**: Koristi XML za kreiranje zahteva i odgovora. Lagan je, ali nije fleksibilan kao REST.
- **`Accept` zaglavlje**: Koristi ga klijent da kaže serveru koji željeni format odgovora, na primer `Accept: application/json`.

## Struktura od REST

- **Bez stanja**: Svaki zahtev mora da pošalje sve informacije koje server treba da obradi. Server ne pamti prethodne zahteve.
- **Resurs**: Komad podataka koji se može dohvatiti ili modifikovati, identifikovan po URL (npr. `/products/1`).
- **Metode za HTTP u REST**:
  - `GET` — dohvati resurs
  - `POST` — kreiraj novi resurs
  - `PUT` / `PATCH` — ažuriraj postojeći resurs
  - `DELETE` — obriši resurs

## Brojevi stanja za HTTP

Nakon obrade zahteva, server odgovara sa **status kodom** koji ukazuje rezultat:

- `200` **(OK)**: Zahtev je uspešan
- `201` **(Created)**: Krešiv resurs je kreiran
- `400` **(Bad Request)**: Klijent je poslao nevažeće podatke
- `404` **(Not Found)**: Zahtevani resurs ne postoji
- `500` **(Internal Server Error)**: Došlo je do greške na serveru

### Kako zahtev REST radi

1. Klijent šalje zahtev sa metodom, URL, opcionalnim zaglavljima i opcionalnim telom.
2. Server validira zahtev i identifikuje resurs.
3. Server izvršava akciju (dobavi, kreiraj, ažuriraj ili obriši).
4. Server konstruiše odgovor sa statusnim kodom i opcionalnim telom (obično JSON).
5. Server šalje odgovor — nije sačuvano stanje klijenta.
6. Klijent proverava statusni kod i obrađuje povratne podatke.

## Monoliths protiver microservices

- **Monolith**: Jedna velika baza koda gde su UI, baza podataka i API zajedno. Dobro funkcioniše za male aplikacije, ali je teško održavati i skalirati kako raste.
- **Microservices**: Arhitektura gde je velika aplikacija podeljena na male, nezavisne servise. Svaki servis obrađuje jednu poslovnu funkcionalnost (npr. proizvod, porudžbina, plaćanje) i može imati svoj kod, svoju bazu podataka i svoj tim.
- **Jinsi microservices zinavyowasiliana**: Kroz API ili sisteme poruka (messaging systems).
- **Faida za microservices**: Brzo skaliranje, dobro izolovanje grešaka, i sloboda korišćenja različitih jezika ili framework-ova za svaki servis.
- **Changamoto za microservices**: Teže otklanjanje grešaka (debugging), složeno upravljanje infrastrukturom, i visok mrežni overhead između servisa.

## Express.js šta je to?

- **Express**: Mala i jednostavna mrežna biblioteka bazirana na Node.js koja pojednostavljuje izgradnju web servera i API.
- **Instaliraj Express**: `npm install express`
- **Šta možete izgraditi sa Express**: veb aplikacije, RESTful API, backend za SPA, servisi sa mnogo middleware-a, i sve što koristi HTTP na strani servera.
- **Middleware (Sredinski softver)**: Kod koji se izvršava između prijema zahteva i slanja odgovora. Koristi se za rukovanje greškama, validaciju unosa, modifikovanje podataka zahteva ili odgovora, i više. Mnoge middleware funkcije mogu se povezati zajedno.
- **Dodaci (Plugins)**: Eksterni npm moduli koji dodaju Express, obično implementirani kao middleware.

## Creiranje osnovnog programa od Express

```javascript
const express = require("express");
const app = express();
const port = 3000;

app.get("/", (req, res) => {
  res.send("Hello World!");
});

app.listen(port, () => {
  console.log(`Example app listening on port ${port}`);
});
```

- `require("express")` — uzima biblioteku od Express.
- `express()` — fabrička funkcija koja kreira i vraća novi instanci za/od programa Express.
- `app` — glavni objekat servera sa pristupom putanjama kao što su `.get()`, `.post()` i `.listen()`.
- `port` — mrežni endpoint na koji program sluša. Port `3000` je uobičajen za lokalni razvoj.
- `app.listen(port, callback)` — pokreće server i počinje da sluša dolazne zahtevke od HTTP.

## Definisanje puta u Express

- **Put podataka**: Pravilo koje govori programu kako da odgovori korisniku kada traži specifičan URL na određeni način putem HTTP.
- **Putovi podataka**: `app.get()`, `app.post()`, `app.put()`, `app.delete()` — svaki obrađuje zahteve sa odgovarajućim putem HTTP.
- **Put podataka za static**: Fiksirano kao URL sa `/home` ili `/menu/drinks`.
- **Dinamički put podataka (parametar puta podataka)**: Placeholder u URL koji hvata vrednost, povezan sa `:`, na primer `/post/:postId`.
- **`req.params`**: Objekt sa uhvaćenim dinamičkim vrednostima iz URL. Na primer, zahtev od `/post/42` vraća `req.params.postId === "42"`.

### Handleri putanja podataka

- **Handler rute podataka** je callback funkcija `(req, res) => { ... }` koja se izvršava kada zahtev odgovara ruti podataka.
- `req` — sadrži informacije o dolaznom zahtevu (parametri, sekvenca argumenata pretrage, zaglavlja, telo).
- `res` — koristi se za slanje odgovora klijentu.

## Načini za odgovor

| Metoda | Funkcija |
| --- | --- |
| `res.send()` | Šalje odgovor — običan tekst, HTML, JSON, ili binarni podaci. |
| `res.json()` | Šalje odgovor za JSON; direktno modifikuje objekat od JavaScript. |
| `res.status(code)` | Postavlja statusni kod za HTTP; obično je povezan sa `res.send()` ili `res.json()`. |
| `res.redirect(url)` | Preusmerava klijenta na drugi URL. |
| `res.render(view, data)` | Prikazuje prikaz šablona (template engine) i šalje rezultujući HTML. |

## Povezivanje rukovalaca putanje podataka za `app.route()`

- `app.route(path)` povezuje rukovaocači više ruta za HTTP na jedan način kako bi se izbeglo ponavljanje.

```javascript
app
  .route("/user")
  .get((req, res) => { res.send("Fetching user data"); })
  .post((req, res) => { res.send("Creating a user"); })
  .put((req, res) => { res.send("Updating the user"); })
  .delete((req, res) => { res.send("Deleting the user"); });
```

## Konfigurisanje puta za modul za `express.Router()`

- `express.Router()` kreira ruter — kao mali modul Express — unutar kojeg možete definisati putanje podataka, a zatim ih postaviti u glavnu aplikaciju.
- Postavite ruter za `app.use(path, router)`. Putanje podataka unutar rutera zavise od metode postavljanja.
- **Prednost**: razdvaja odgovornosti, čini glavni fajl čistijim i omogućava ponovnu upotrebu i lako proširenje putanja podataka.

```javascript
// userRoutes.js
const express = require("express");
const router = express.Router();
router.get("/", (req, res) => { res.send("List of users"); });
router.get("/:id", (req, res) => { res.send(`User ${req.params.id}`); });
module.exports = router;

// app.js
app.use("/users", userRoutes); // mounts the router at /users
```

## Za pristup fajlovima static

- **static fajlovi**: Slično slikama, CSS i JavaScript sa klijentske strane isporučuju se direktno bez serverske obrade.
- `express.static(directory)` — ugrađeni middleware koji služi fajlove iz određenog foldera.
- Ako traženi fajl nije dostupan, Express direktno vraća 404.
- Nije potrebno definisati putanju podataka za static fajlove.

```javascript
app.use(express.static(path.join(__dirname, "public")));
```

- Možete pristupiti iz više direktorijuma pozivanjem `app.use()` često, opciono i metodom postavljanja:

```javascript
app.use(express.static(path.join(__dirname, "public")));
app.use("/images", express.static(path.join(__dirname, "images")));
```
