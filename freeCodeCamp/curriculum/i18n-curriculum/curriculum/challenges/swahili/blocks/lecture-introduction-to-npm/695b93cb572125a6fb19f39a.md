# --description--

U prethodnom tutorijalu, naučili ste šta je zavisnost. Ovde, naučićete više o zavisnostima i stvarima koje treba uzeti u obzir kada tražite eksterni paket za vaš projekat.

Zavisnost je još jedan deo koda koji vaš projekat zahteva ili koristi. Ovo su obično eksterni paketi koje su drugi programeri kreirali i podelili u npm registri.

Korišćenje zavisnosti štedi mnogo vremena. Umesto da sve gradiš sam, možeš instalirati paket koji već rešava uobičajni problem — stvari poput kreiranja mrežnog servera, upravljanja rutama (routing), dobijanja podataka, upravljanja autentifikacijom identiteta i više.

U vašoj datoteci za `package.json`, svi paketi na koje projekat zavisi su navedeni pod ključem `dependencies`:

```json
{
  "name": "package-dependencies",
  "version": "1.0.0",
  "description": "a demo on how package dependencies work",
  "main": "index.js",
  "scripts": {
    "test": "echo \"Error: no test specified\" && exit 1"
  },
  "keywords": [
    "Node",
    "NPM"
  ],
  "author": "Kolade Chris",
  "license": "ISC",
  "dependencies": {
    "chalk": "^5.6.2",
    "lodash": "^4.17.21"
  }
}
```

U primeru fajla `package.json` navedeno iznad, `chalk` i `lodash` su navedeni kao zavisnosti projekta. To znači da se projekt oslanja na ove pakete za rad. Ako nisu instalirani—i ako nisu navedeni u fajlu `package.json`—svaka funkcionalnost koja ih koristi će biti pokvarena.

Postoji i druga vrsta zavisnosti koja se naziva `devDependencies`. To su paketi koji su potrebni samo tokom razvoja ili testiranja projekta, ali ne i kada se aplikacija izvršava u produkcionom okruženju. Uobičajeni primeri su:

* Nodemon za automatsko ponovno pokretanje projekta kada dođe do izmena.
    
* Jest au Mocha kwa ajili ya upimaji
    
* ESLint kwa ukaguzi wa msimbo
    

Kao što očekujete, `devDependencies` se pojavljuje ispod sekcije `devDependencies` u fajlu `package.json`:

```json
{
  "name": "package-dependencies",
  "version": "1.0.0",
  "description": "a demo on how package dependencies work",
  "main": "index.js",
  "scripts": {
    "test": "echo \"Error: no test specified\" && exit 1"
  },
  "keywords": [
    "Node",
    "NPM"
  ],
  "author": "Kolade Chris",
  "license": "ISC",
  "dependencies": {
    "chalk": "^5.6.2",
    "lodash": "^4.17.21"
  },
  "devDependencies": {
    "jest": "^30.2.0",
    "nodemon": "^3.1.10"
  }
}
```

Kada biraš pakete koje ćeš koristiti u projektu, treba uzeti u obzir nekoliko važnih stvari.

Bezbednost uvek mora doći na prvo mesto. Čak i jedan nesiguran paket može uzrokovati probleme celom projektu. Takođe treba da razmotrite performanse, jer veliki ili loše optimizovan paket može odložiti vaš projekat.

Takođe je korisno birati pakete koji su popularni i dobro održavani. Takvi paketi obično imaju aktivnu zajednicu, dobru dokumentaciju i redovna ažuriranja. Jedan od jednostavnih načina da proveriš popularnost jeste da pogledaš broj nedeljnih preuzimanja na npm-u. Na primer, paket `chalk` se preuzima više od 300.000 puta nedeljno:

<img src="https://cdn.freecodecamp.org/curriculum/lecture-transcripts/what-are-package-dependencies-and-what-are-some-considerations-for-choosing-an-external-package-1.webp" alt="screenshot from npm of chalk package statistics">

Održavanje je takođe važno. Paketi koji su dobro održavani se redovno ažuriraju i bolje rade sa novim verzijama drugog softvera.

Na kraju, ne zaboravite dokumentaciju. Jasna i detaljna dokumentacija olakšava razumevanje kako koristiti paket i ušteda mnogo vremena kada ga uključujete u svoj projekat.
