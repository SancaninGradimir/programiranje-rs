# --interactive--

U prošlom tutorijalu, naučili ste kako da radite sa funkcijama, koje su ponovo upotrebljivi delovi koda koji pomažu da vaš kod bude modularniji, lakši za održavanje i efikasniji. Svi prethodni primeri koristili su standardnu sintaksu funkcija, kao što je ovo:

```js
function greetings(name) {
  console.log("Hello, " + name + "!");
}
```

Međutim, drugi način za pisanje funkcija u JavaScript je kreiranje izraza strelične funkcije (arrow function). Evo kako možete da promenite originalni primer koristeći sintaksu strelične funkcije umesto toga:

```js
const greetings = (name) => {
  console.log("Hello, " + name + "!");
};
```

U ovom modifikovanom primeru, kreiramo razliku `const` koja se zove `greetings` i dajemo joj funkciju bez imena. Većina sintakse će izgledati normalno, osim što nedostaje glavna reč `function` i dodaju se strelica (`=>`) između parametra `name` i tela funkcije. Ako vaša lista parametara ima samo jedan parametar, onda možete ukloniti obične zagrade kao što je ovo:

```js
const greetings = name => {
  console.log("Hello, " + name + "!");
};
```

Ako tvoja funkcija sa strelicom nema parametara, onda moraš da koristiš standardne zagrade kao ovo:

```js
const greetings = () => {
  console.log("Hello");
};
```

Kada učiš funkcije prvi put, morao si da obuhvatiš telo funkcije uglastim zagradama. Ali ako je telo tvoje funkcije samo jedna linija koda, možeš da ukloniš uglaste zagrade ovako:

```js
const greetings = name => console.log("Hello, " + name + "!");
```

Važno je znati da uklanjanje običnih zagrada i kružnih zagrada iz standardne funkcionalske sintakse neće raditi. Doći ćete do greške ako pokušate nešto poput ovoga:

```js
// This will produce syntax errors 
function greetings name console.log("Hello, " + name + "!");
```

Ove vrste jednopredajnih funkcija rade samo ako koristite sintaksu strelične funkcije. Još jedan važan koncept je izraz za `return`. Evo primera korišćenja sintakse strelične funkcije za izračunavanje površine:

:::interactive_editor

```js
const calculateArea = (width, height) => {
  const area = width * height;
  return area;
};

console.log(calculateArea(5, 3)); // 15
```

:::

Kreiramo razliku unutar funkcije nazvane `area`, a zatim vraćamo tu razliku. Ali možemo malo da očistimo naš kod i vratimo samu kalkulaciju:

:::interactive_editor

```js
const calculateArea = (width, height) => {
  return width * height;
}; 

console.log(calculateArea(5, 3)); // 15
```

:::

Ako pokušate da uklonite vitičaste zagrade i unesete izračunavanje na jednu liniju, dobićete poruku `Uncaught SyntaxError: Unexpected token 'return'`:

```js
const calculateArea = (width, height) => return width * height;
```

Razlog za ovu grešku je zato što morate ukloniti izraz `return`. Kada uklonite taj izraz `return`, greška će nestati, a funkcija će i dalje nastaviti da vraća proračune na indirektan način.

```js
const calculateArea = (width, height) => width * height;
```

Dakle, kada bi trebalo da koristite sintaksu strelične funkcije? Dobro, zavisi od toga. Mnogi programeri često koriste to u svojim ličnim projektima. Međutim, kada radite u timu, izbor obično zavisi od toga da li postojeći kod koristi obične funkcije ili strelične funkcije. U narednim lekcijama, diskutovaćemo kada koristiti strelične funkcije i kada izbegavati.
