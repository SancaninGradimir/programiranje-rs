# --description--

U SQL-u, primarni ključevi i strani ključevi koriste se za uspostavljanje odnosa između tabela.

Tuanze na funguo kuu.

Glavni ključ je jedna kolona ili skup kolona koji jedinstveno identifikuju svaki zapis (red) u tabeli. Ovaj ograničenje osigurava da nijedan zapis u tabeli neće imati iste vrednosti za glavni ključ.

Zbog toga tabela može imati samo jedan primarni ključ.

Vrednost u koloni primarnog ključa takođe ne može biti `NULL`, tako da će uvek imati važeću vrednost.

U PostgreSQL-u, da biste kolonu postavili kao primarni ključ svoje tabele, potrebno je samo da dodate `PRIMARY KEY` nakon tipa podataka:

```sql
column_name data_type PRIMARY KEY
```

Hii ni mfano wa jedwali la `students`:

```sql
CREATE TABLE students (
  student_id SERIAL PRIMARY KEY,
  name VARCHAR(100)
);
```

Svaki student će imati svoj `student_id`, a ovaj identifikator će biti primarni ključ tabele. Tip `SERIAL` je ovde koristan jer osigurava da će `student_id` uvek imati jedinstvenu vrednost.

Kompozitni primarni ključ se koristi kada tabela nema jedan jedinstveni atribut za identifikaciju redova. U ovom slučaju, možete koristiti kombinaciju od dva ili više atributa kao primarni ključ koji zajedno garantuje jedinstvenost.

Da biste to uradili, navedite koje redove imaju kompozitni primarni ključ kada kreirate tabelu kao ovu:

```sql
CREATE TABLE table_name (
  column1 data_type column_constraint,
  column2 data_type column_constraint,
  column3 data_type column_constraint,
  ...
  PRIMARY KEY (column1, column2)
);
```

Zamislite da imate tabelu sa kolonama `student_id` i `course_id` koje vam govore koji studenti pohađaju različite kurseve.

Svaki učenik će biti upisan na nekoliko predmeta, i svaki predmet će imati više učenika. Stoga ne postoji jedinstveni red – ali će uvek biti jedan učenik upisan na više od jednog predmeta. Zbog toga možete koristiti kombinaciju ovih petelja/redova kao primarni ključ:

```sql
CREATE TABLE course_enrollments (
  student_id INT,
  course_id INT,
  ...
  PRIMARY KEY (student_id, course_id),
);
```

Primarni ključevi su važni za osiguravanje jedinstvenosti i integriteta podataka, poboljšanje dostupnosti podataka, i uspostavljanje odnosa sa drugim tabelama u relacionoj bazi podataka.

Ovo nas vodi ka drugom tipu ključeva koje ćemo naučiti u ovoj lekciji: strani ključevi.

Strani ključ je red (ili skup redova) u tabeli koji referencira primarni ključ druge tabele. Tabela može imati više stranih ključeva.

U ovom primeru, imamo dve tabele, `customers`, i `orders`:

```sql
CREATE TABLE customers (
  customer_id SERIAL PRIMARY KEY,
  first_name VARCHAR(100) NOT NULL,
  ...
);

CREATE TABLE orders (
  order_id SERIAL PRIMARY KEY,
  customer_id INTEGER,
  ...
  FOREIGN KEY (customer_id) REFERENCES customers(customer_id)
);
```

Primarni ključ tabele `customers` je `customer_id` i primarni ključ tabele `orders` je `order_id`.

Međutim, proverite i da postoji red sa `customer_id` u tabeli `orders`.

Zašto čuvamo podatke korisnika u tabeli `orders`?

Ovo radi kako bi se uspostavila veza između tabela `customers` i `orders`. Svaka porudžbina će sačuvati identifikator klijenta koji je podneo.

Da bismo uspostavili odnos između tabela, napravićemo kolonu `customer_id` u tabeli `orders` kao strani ključ koji se odnosi na kolonu `customer_id` u tabeli `customers`:

```sql
FOREIGN KEY (customer_id) REFERENCES customers(customer_id)
```

Vrednosti u koloni stranog ključa moraju odgovarati vrednostima primarnog ključa u povezanoj tabeli ili mogu biti `NULL` (ako strani ključ dozvoljava `NULL` vrednosti).

Kwa nini hii ni msaada?

Osiguravanjem da se ove kolone u tabelama potpuno poklapaju, primarni i strani ključevi sprečavaju kreiranje "siročadi" zapisa, odnosno zapisa koji se odnose na nepostojeći zapis u drugoj tabeli.

Primarni i strani ključevi predstavljaju osnovu relacionih baza podataka. Oni omogućavaju funkciju "odnosa" u ovim bazama podataka, omogućavajući vam da precizno modelujete podatke iz stvarnog sveta.
