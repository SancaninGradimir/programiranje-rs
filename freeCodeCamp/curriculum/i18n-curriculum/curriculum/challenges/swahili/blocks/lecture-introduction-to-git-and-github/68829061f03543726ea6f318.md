# --description--

Razgovarali smo malo o Git i GitHub, sada možemo početi da sve povežemo zajedno. Da bismo to uradili, potreban vam je repozitorijum. Ali šta je repozitorijum? Možete zamisliti repozitorijum kao kontejner projekta – ako radite na aplikaciji, sačuvali biste fajlove te aplikacije zajedno u repozitorijumu.

Repozitorijumi mogu biti lokalni na tvom računaru ili udaljeni na servisima kao što je GitHub. Možeš početi od lokalnog ili udaljenog repozitorijuma — rezultat je isti, zato izaberi tok rada koji ti najviše odgovara. Ako počinješ od udaljenog repozitorijuma, možeš ga napraviti direktno na GitHub-u prateći ova uputstva:

Posetite <a href="https://github.com" target="_blank">sajt GitHub</a>, uverite se da ste prijavljeni na svoj nalog, zatim kliknite na ikonu plus znaka gore desno na navigacionoj traci. Videćete će mnogo opcija, ali ona koja vam treba je "New Repository". Izaberite ga da biste otvorili interfejs za kreiranje novog repozitorijuma/repositoryja.

Prva opcija koju ćeš videti omogućava izbor šablona. Možda još nemaš nijedan, ali šablon je posebna vrsta repozitorijuma koja služi kao osnova za nove projekte. Koristan je jer može sadržati dokumentaciju ili šablone za prijavu problema.

Sledeći korak je izbor vlasnika repozitorijuma. Možete izabrati svoj korisnički nalog ili neku od organizacija kojima pripadate ili kojima imate pristup. Zatim možete dati naziv repozitorijumu – uglavnom je najbolje koristiti naziv koji ukratko opisuje projekat koji pravite. Kao dobra praksa, izbegavajte razmake u nazivu repozitorijuma. GitHub će ih automatski zameniti crticama (-).

Sledeće, možete opciono dodati detalje koji će se pojaviti na početnoj stranici rezervacije.

Takođe možeš izabrati da li će repozitorijum biti javan ili privatan. Javni repozitorijum može svako da pregleda i preuzme, dok privatnom možeš pristupiti samo ti i korisnici kojima izričito odobriš pristup.

Ako ne koristiš šablon, možeš odmah generisati početne datoteke repozitorijuma. Datoteka `README` služi za dokumentaciju i opis projekta. Datoteka `.gitignore` određuje koje će datoteke Git ignorisati, poput zavisnosti, datoteka okruženja i drugih koje ne želiš da pratiš. GitHub nudi gotove šablone za `.gitignore` prema jeziku ili okruženju. Licenca je pravni dokument koji određuje ko može koristiti tvoj softver i pod kojim uslovima.

Kada podesite sve onako kako želite, kliknite na dugme "Create Repository" da biste završili proces. Bićete automatski preusmereni na početnu stranicu svog novog repozitorijuma!

Odavde možeš kliknuti na dugme **Code** da dobiješ opcije za kloniranje repozitorijuma na svoj računar.

Za ovu lekciju možeš zanemariti kartice **Codespaces** i **Copilot**. Fokusiraćemo se na tri opcije na kartici **Local**. Prve dve, HTTPS i SSH, koriste se na isti način — komandom `git clone` i odgovarajućim URL-om kloniraš udaljeni repozitorijum na računar. Razlika je u načinu povezivanja i autentifikacije.

Za kopiranje javnog repozitorijuma, nećete morati da verifikujete identitet. Ali za kopiranje privatnog repozitorijuma, ili za push-ovanje promena u javne ili privatne repozitorijume, biće vam potrebna verifikacija. Daljka URL koja koristi HTTPS se koristi za autentikaciju putem vašeg korisničkog imena GitHub i lozinke. Međutim, zbog sigurnosnih problema, ovo više nije pouzdano, pa je potrebno koristiti korisničko ime i token pristupa (access token). Ipak, ovo nije siguran način kao SSH, koji koristi vaše privatne i javne ključeve. Prednost je u tome što omogućava direktan pristup, umesto da se svaki put izdaje token. Postoji drugačiji tutorijal koji detaljno objašnjava šta je SSH i kako funkcioniše, ali možete pročitati dokumentaciju GitHub o tome kako da konfigurišete SSH.

Ako kopirate podatke/skladište, možete ovo pokrenuti u terminalu:

```sh
git clone git@github.com:your-username/your-repository-name
```

Kreiraće se folder na vašem računaru sa imenom vašeg skladišta ići će da uključi sve fajlove sa vašeg skladišta unutra. Odavde možete `cd` (promeni direktorijum) otići na vaše kopirano skladište za:

```sh
cd name-of-cloned-repo
```

Pored toga, lokalna kopija repozitorijuma imaće remote koji pokazuje na GitHub repozitorijum. Da vidiš svoje remote repozitorijume, pokreni sledeću komandu:

```sh
git remote -v
```

Druga opcija za kopiranje vašeg skladišta je korišćenje GitHub CLI. Ovaj alat se koristi za obavljanje specifičnih zadataka GitHub bez napuštanja komandne linije. Ako ga niste instalirali, možete pronaći uputstva o tome kako ga instalirati u dokumentaciji GitHub – ali bi trebalo da bude dostupan u menadžeru paketa vašeg sistema.

Ako kopiram repozitorijum koristeći CLI alat, koristio bih:

```sh
gh repo clone your-username/your-repository-name
```

Rezultati ovih komandi su isti: dobićete kopiju vaše rezervne memorije na vašem računaru.

Takođe možete koristiti GitHub CLI da kreirate skladište:

```sh
gh repo create
```

Ako preskočite bilo koji argument, CLI će vam pružiti wizard za interakciju ili konfigurisanje vašeg skladišta. Ovaj wizard će vam pružiti sve opcije kao web interfejs. Sa ovim alatom, takođe ćete dobiti opciju da odmah klonirate novo skladište, pa nećete morati da prolazite kroz ručne korake pomenute ranije.

Sada imaš osnovno razumevanje kako funkcionišu repozitorijumi i kako da napraviš jedan kako bi pravilno verzionisao svoj rad. U narednim lekcijama naučićeš kako da postaviš postojeći lokalni repozitorijum na GitHub i kako da šalješ izmene u udaljene repozitorijume.
