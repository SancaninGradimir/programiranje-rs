# --description--

U početnoj obuci, naučili ste kako da kreirate novu bazu podataka na GitHub. Ali šta ako imate postojeći projekat unutar računara koji želite dodati na GitHub? U ovoj obuci, naučićete kako da push-ujete postojeće lokalne baze podataka na GitHub i povučete promene sa udaljenih baza podataka.

Počnite kreiranjem nove baze podataka na GitHub bez bilo kakvih fajlova. Nakon što uspešno završite, bićete preusmereni na stranicu koja sadrži URL neophodno za kopiranje vaše baze podataka.

Sada otvori terminal i izaberi postojeći projekat koji želiš da postaviš na GitHub. U ovoj lekciji koristi se lokalni projekat pod nazivom `super-awesome-game`, koji sadrži sledeće datoteke:

```sh
jessicawilkins super-awesome-game >> ls
README.md  index.html script.js  styles.css
```

Da bi započeo praćenje verzija ovog projekta pomoću Git-a, pokreni komandu `git init` u direktorijumu projekta. Time će biti kreiran prazan Git repozitorijum koji će početi da prati izmene u projektu. Ako je sve prošlo uspešno, videćeš sličan izlaz u terminalu:

```sh
jessicawilkins super-awesome-game >> git init
Initialized empty Git repository in /Users/jessicawilkins/workspace/freeCodeCamp/super-awesome-game/.git/
```

Nakon inicijalizacije Git repozitorijuma, biće dodat skriveni direktorijum `.git`. Da bi ga video, pokreni komandu `ls -a`, koja prikazuje sve datoteke i direktorijume, uključujući skrivene:

```sh
jessicawilkins super-awesome-game >> ls -a
.          ..         .git       README.md  index.html script.js  styles.css
```

Veoma je važno da ne obrišeš direktorijum `.git`, jer ćeš u suprotnom izgubiti celu Git istoriju projekta.

Kada je Git repozitorijum inicijalizovan, možeš pokrenuti komandu `git status`. Ova komanda prikazuje trenutno stanje radnog direktorijuma i koristićeš je veoma često. Rezultat bi trebalo da izgleda približno ovako:

```sh
jessicawilkins super-awesome-game >> git status
On branch main

No commits yet

Untracked files:
  (use "git add <file>..." to include in what will be committed)
	README.md
	index.html
	script.js
	styles.css

nothing added to commit but untracked files present (use "git add" to track)
```

`On branch main` znači da se nalazite na grani `main`. Grana u Git-u predstavlja zaseban prostor za rad u kojem možete praviti izmene. Grana `main` najčešće predstavlja glavnu ili produkcionu granu stvarnog projekta. Timovi programera obično kreiraju više grana za nove funkcionalnosti i ispravke grešaka, a zatim te izmene spajaju nazad u granu `main`. Više o tom procesu naučićete u sledećoj lekciji.

`Untracked files` znači da je Git pronašao datoteke koje trenutno ne prati.

Datoteka može imati jedno od pet stanja:

1. **Nepraćena (Untracked):** Datoteka je nova u repozitorijumu i Git je još nije počeo da prati.
2. **Izmenjena (Modified):** Datoteka je postojala u prethodnom commit-u, ali sadrži izmene koje još nisu sačuvane u novom commit-u.
3. **Ignorisana (Ignored):** Ignorisane datoteke možda neće biti vidljive u Git-u, ali ih IDE može označiti. Git ih ne uključuje u svoje operacije, najčešće zato što su navedene u datoteci `.gitignore`.
4. **Obrisana (Deleted):** Datoteka je ranije postojala, ali je uklonjena.
5. **Preimenovana (Renamed):** Sadržaj datoteke nije promenjen, ali je promenjeno njeno ime ili lokacija. U nekim slučajevima Git može smatrati datoteku preimenovanom čak i ako sadrži manje izmene.

Ako imaš promene koje nisu sačuvane u tvojoj bazi podataka, prvo moraš "prebaciti na pripremni korak". "Pripremni prostor" je specifična datoteka u tvom direktorijumu `.git` koja sadrži podatke koji će biti uključeni u tvoj sledeći *commit*. Commit znači snimak trenutnog stanja vaše baze podataka.

Ako želite da prebacite/kopirate fajl direktno, možete koristiti sledeću komandu:

```sh
git add name-of-file
```

Ako želite da sačuvate sve promene koje nisu u stacionarnom području (staging area), onda možete koristiti `git add .` Tačka (`.`) je alternativno ime za trenutni direktorijum. Drugi programeri upozoravaju da se ne koristi `git add .` jer možete uključiti fajlove koje ne nameravate. Ako se to dogodi, možete "ukloniti iz stacionarnog područja" koristeći sledeću komandu:

```sh
git reset name-of-file
```

Generalno, ako projekat prati fajlove dodate korišćenjem `git status` pre nego što ih prebaci u fazu pripreme, biće u redu.

Čim postavite vaše fajlove u staging područje, moraćete ih *commitovati*. Možete *commitovati* svoje promene pokretanjem komande `git commit`. Ovo će otvoriti vaš omiljeni editor postavljen u konfiguraciji Git. Kada se editor otvori, možete uneti detaljnu poruku o svojim promenama. Takođe možete izabrati da unesete kratku poruku koristeći komandu `git commit -m` kao što je sledeće:

```sh
git commit -m "short message goes here"
```

Kada koristite zastavu `-m`, i dalje ćete želeti da pružite objašnjavu poruku kako biste dali kontekst promenama. Evo primera dobrih poruka koje možete koristiti:

```sh
git commit -m "feat(api): implement JWT-based authentication"
git commit -m "fix(auth): correct token expiration time"
git commit -m "refactor: simplify login flow logic"
```

`feat(api)` na `fix(auth)` slede stil Conventional Commits koji se koristi za pružanje konteksta o vrstama promena u tom commitu.

Nakon što zatvorite svoj zadatak i pokrenete `git status` ponovo, trebalo bi da vidite sledeće rezultate u terminalu:

```sh
On branch main
nothing to commit, working tree clean
```

Da biste videli sve vaše prethodne komitise za ovaj projekat, možete pokrenuti komandu ``git log``. Ovo će listati sve prethodne komitise i važne informacije kao što su autor, datum komita, poruka komita i hash komita. Hash komita je dugačak niz koji se koristi kao jedinstveni identifikator za komit.

Čim završiš svoj rad, sada možeš da pushuje svoje promene na GitHub. Prvo ćeš morati da uspostaviš udaljenu konekciju do svoje udaljene baze podataka. Naučio/la si ovo u prethodnoj obuci, ali evo podsetnika korišćenjem SSH-a:

```sh
git remote add origin git@github.com:your-github-username/name-of-repo.git
```

Poslednji korak je da pokrenete svoje promene na GitHub koristeći ovu komandu:

```sh
git push -u origin main
```

Brend za `-u` je prečica za `--set-upstream`. Kreira konekciju između vaše lokalne grane `main` i udaljene grane `main` na `origin`. Morate ga koristiti samo jednom za svaku novu granu. Nakon toga, možete samo pokrenuti `git push` bez navođenja imena grane.

Nakon uspešnog izvršenja, trebalo bi da vidite odgovarajuće rezultate u terminalu:

```sh
Enumerating objects: 3, done.
Counting objects: 100% (3/3), done.
Delta compression using up to 8 threads
Compressing objects: 100% (2/2), done.
Writing objects: 100% (3/3), 253 bytes | 253.00 KiB/s, done.
Total 3 (delta 0), reused 0 (delta 0), pack-reused 0 (from 0)
To https://github.com/jdwilkin4/super-awesome-game.git
 * [new branch]      main -> main
branch 'main' set up to track 'origin/main'.
```

Takođe bi trebalo da vidiš nove izmene na stranici svog GitHub repozitorijuma nakon osvežavanja stranice.

Ako sarađuješ sa drugima na projektu i moraš povući nove promene iz baze podataka, možeš koristiti komandu `git pull`. Naučićete više o timskom radu i saradnji u sledećoj lekciji.
