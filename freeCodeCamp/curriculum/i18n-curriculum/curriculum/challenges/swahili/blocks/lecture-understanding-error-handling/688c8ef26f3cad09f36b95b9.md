# --description--

Ispravljanje grešaka je ključna veština za svakog programera Python. Razumevanje osnovnih metoda može pomoći u efikasnoj identifikaciji i popravljanju problema.

Ispravljanje grešaka (debugging) je proces pronalaženja i rešavanja grešaka u kodu. Uključuje pregledanje koda, razumevanje njegovog toka i korišćenje alata za otkrivanje uzroka problema.

U ovom tutorijalu, pregledaćemo uobičajene metode za otklanjanje grešaka koje možete koristiti u vašem sledećem projektu za Python.

## Korišćenje funkcije `print()` sa nizom znakova u f-stringovima

Prvo, korišćenje funkcije `print()` i f-string sekvenci znakova u različitim delovima vašeg koda može pomoći u razumevanju protoka i stanja promenljivih. Primer:

```py
def add(a, b):
    result = a + b
    print(f'Adding {a} and {b} gives {result}')
    return result
```

Prikazivanjem vrednosti za `a`, `b`, i `result`, možete potvrditi da funkcija radi kako je očekivano.

## Ispravljanje grešaka u saradnji sa modulom `pdb`

Zatim možeš koristiti ugrađeni Python modul `pdb` za interaktivno otklanjanje grešaka:

```py
import pdb

def divide(a, b):
    pdb.set_trace()
    return a / b

print(divide(10, 2))
```

Kada postavite točku praćenja (trace) za funkciju `set_trace()`, možete korak po korak prolaziti kroz kod, pregledati parametre i razumeti ponašanje programa.

Ako pokreneš gornji kod, videćeš izlaz koji prikazuje putanju do datoteke koju izvršavaš, liniju koja je pozvala funkciju `set_trace()`, sledeću liniju koda i interaktivni `pdb` prompt:

```py
> /Users/fcc/Desktop/debugging.py(5)divide()
-> return a / b
(Pdb)
```

Ako na promptu otkucaš `help`, videćeš listu dostupnih komandi:

```py
(Pdb) help

Documented commands (type help <topic>):
========================================
EOF    c          d        h         list      q        rv       undisplay
a      cl         debug    help      ll        quit     s        unt      
alias  clear      disable  ignore    longlist  r        source   until    
args   commands   display  interact  n         restart  step     up       
b      condition  down     j         next      return   tbreak   w        
break  cont       enable   jump      p         retval   u        whatis   
bt     continue   exit     l         pp        run      unalias  where    

Miscellaneous help topics:
==========================
exec  pdb
```

Zatim možete koristiti te komande da popravite greške u svom kodu.

Na primer, ako želite da pregledate tipove elemenata u vašem kodu u tom trenutku, možete koristiti komandu `whatis`:

```py
(Pdb) whatis a
<class 'int'>
(Pdb) whatis divide
Function divide
```

Kao što možete da vidite, kada radi `.set_trace()`, tip parametra `a` je ceo broj, a `divide` je funkcija.

Da biste nastavili sa izvršavanjem vašeg koda, možete koristiti komandu `continue`, ili jednu od njegovih alternativnih naziva, `cont` ili `c`:

```py
(Pdb) continue
5.0
```

## Alat za ispravljanje grešaka za IDE

Integrisana razvojna okruženja (IDEs) nude napredne alate za debagovanje, kao što su tačke za zaustavljanje (breakpoints), korak po korak izvršavanje i inspekcija varijabli.

## Korišćenje debugera u VS Code-u

Ako koristite VS Code, možete postaviti tačke za zaustavljanje u svom kodu i pokrenuti debagera da pauzirate izvršavanje na tim mestima. Evo kako da popravite greške funkcije kao što je `divide`:

**Korak 1: Pripremi svoj kod**

Kreiraj fajl pod nazivom `main.py` sa sledećim sadržajem:

```py
def divide(a, b):
    result = a / b
    return result

print(divide(10, 2))
print(divide(15, 3))
```

**Korak 2: Postavite znak za zaustavljanje**

1. Kliknite u prazninu između redova (levna ivica) blizu linije 2 (`result = a / b`) da postavite znak za zaustavljanje

2. Crvena tačka će se pojaviti, što ukazuje da je postavljen zaustavni znak

**Korak 3: Počnite sa ispravljanjem grešaka**

1. Kliknite `F5` i idite Run > Start Debugging

2. Odaberi "Python File" kada je zatraženo

3. Korektor grešaka će zaustaviti izvršenje na vašoj tački prekida

**Korak 4: Proverite parametre**

- Prođi preko parametara da vidiš njihove trenutne vrednosti

- Koristite panel sa parametrima na levoj strani da biste videli sve interne parametre

- Koristite Debugging Console za evaluaciju matematičkih izraza

**Korak 5: Prođi kroz kod korak po korak**

Koristite alat za ispravljanje grešaka da biste uradili:

- **Nastavi (F5)**: Nastavite izvršavanje do sledeće tačke za zaustavljanje

- **Napredovanje korak gore (F10)**: Selektuj trenutni red i pređi na sledeći

- **Uđi unutra (F11)**: Uđi u poziv funkcije

- **Toka nje (Shift+F11)**: Napusti trenutni modul

Alati za debagovanje u IDE-u pružaju interfejs za pregled stanja vašeg programa, što olakšava identifikovanje i otklanjanje problema u poređenju sa korišćenjem samo komandi štampanja.

Posjedovanjem ovih osnovnih tehnika za otklanjanje grešaka – korišćenjem komandi za `print()`, modula `pdb`, i alata IDE-a – možete identifikovati i rešiti probleme efikasno u vašem kodu Python. Svaka metoda ima svoje mesto: komande za `print()` za brzu proveru, `pdb` za istraživanje saradnje, i IDE alat za otklanjanje grešaka za vizuelnu inspekciju.
