## Biblioteka za sisteme JavaScript

- Biblioteke i okviri za JavaScript pružaju brka rešenja za uobičajene probleme i povećavaju brzinu razvoja obezbeđivanjem gotovog koda.
- Biblioteke se obično više fokusiraju na pružanje rešenja za specifične zadatke, kao što je manipulisanje DOM, rukovanje događajima ili upravljanje AJAX zahtevima.
- Nekoliko primera biblioteka za JavaScript su jQuery i React.
- S druge strane, okviri pružaju definisanu strukturu za izgradnju aplikacija. Često dolaze sa setom pravila i konvencija koje programeri moraju pratiti.
- Primeri okvira uključuju Angular i Next.js, koji je meta okvir za React.
- **Single-page applications** (SPAs) su web aplikacije koje učitavaju jednu stranicu HTML i dinamički ažuriraju tu stranicu dok korisnik interaguje sa aplikacijom bez ponovnog učitavanja cele stranice.
- SPAs koriste JavaScript za upravljanje stanjem aplikacije i prikaz sadržaja. Ovo se često postiže korišćenjem okvira koji pružaju odlične alate za izgradnju kompleksnih korisničkih interfejsa.
- Neki izazovi povezani sa SPA-ima su:

- Čitači ekrana imaju poteškoća sa dinamički ažuriranim sadržajem.
- URL ostaje isti kada korisnik navigira unutar aplikacije, što može otežati označavanje, povratak ili deljenje specifičnih stranica.
- Početno vreme učitavanja može biti dugo ako je aplikacija velika jer su svi resursi potrebni za učitavanje odjednom.

## React

- React je popularna biblioteka za izgradnju korisničkih interfejsa i veb aplikacija, koristeći JavaScript.
- Glavna koncepcija za React je kreiranje ponovo upotrebljivih UI komponenti koje se mogu ažurirati i prikazati nezavisno kako podaci promenu.
- React omogućava programerima da opišu kako bi trebalo da izgleda korisnički interfejs u zavisnosti od stanja aplikacije. React zatim ažurira i prikazuje odgovarajuću komponentu kada se podaci ili stanje promene.

## Sekcija za React

- Komponente su osnovni blokovi za aplikacije React koji omogućavaju programerima da razbijaju složene korisničke interfejse na manje, upravljive komponente.
- Korisnički interfejs je definisan pomoću JSX-a, koji predstavlja proširenje JavaScript sintakse i omogućava programerima da pišu kod sličan HTML-u unutar JavaScript-a.
- Komponenta je u osnovi funkcija JavaScript-a ili klasa koja vraća deo korisničkog interfejsa.

Ovo je primer jednostavnog dela React koji daje poruku pozdrava:

```jsx
function Greeting() {
  const name = 'Anna';
  return <h1>Welcome, {name}!</h1>;
}
```

Za korišćenje ovog dela, možete samo nazvati:

```jsx
  <Greeting />
```

## Donesi delove za transportovanje React

- Izvoz (export) čini deo dostupnim drugim fajlovima. Uvoz (import) omogućava fajlu da koristi izvezen deo negde drugde.
- Postoje dve vrste izvoza u React: podrazumevani izvoz (default exports) i imenovani izvoz (named exports).

**Podrazumevani transport**

- Datoteka može imati samo jedan podrazumevani izvoz. Ovo je pogodno za datoteku koja nosi jednu glavnu sekciju.
- Možete izvesti sekciju kao podrazumevani izvoz na ovaj način:

```jsx
// City.js
function City() {
  return <p>New York</p>;
}

export default City;
```

- Takođe možete prebaciti na jednu liniju i opis sekcije:

```jsx
export default function City() {
  return <p>New York</p>;
}
```

- Da biste postavili podrazumevani transport, koristite reč `import` bez okruga:

```jsx
// App.js
import City from './City';

function App() {
  return (
    <div>
      <h1>My favorite city is:</h1>
      <City />
    </div>
  );
}
```

**Prenos imena**

- Prenos po imenu omogućava fajlu da deli više delova. Za razliku od podrazumevanog prenosa, moraju biti dati sa istim imenom sa kojim su preneseni.
- Na ovaj način možete preneti više delova iz jednog fajla:

```jsx
// Animals.jsx
export function Cat() {
  return <h2>Mr. Whiskers</h2>;
}

export function Dog() {
  return <h2>Fido</h2>;
}
```

- Da biste dobili listu prenesenih imena, koristite kvastrične zagrade i puna imena koja su prenesena:

```jsx
import { Cat, Dog } from './Animals';
```

- Možete promeniti naziv dostave/shipping ime prilikom dostavljanja koristeći reč `as`:

```jsx
import { Cat as Kitty } from './Animals';
```

**Kombinacija Dostave Zadane Opcije i Imena**

- Fajl može imati jedan podrazumevani prenos i prenos više imena istovremeno:

```jsx
// Animals.jsx
export default function Cat() {
  return <h2>Mr. Whiskers</h2>;
}

export function Dog() {
  return <h2>Fido</h2>;
}
```

- Kada unesete kombinaciju dostave, podrazumevana dostava dolazi prva (bez zagrada), praćena dostavom po imenu (unutar zagrada):

```jsx
import Cat, { Dog } from './Animals';
```

## Postavljanje projekta React koristeći Vite

## Prenesite prope u sekcije React

- U React, props (skraćenica za properties) je način prenošenja podataka od roditeljske komponente ka dječijoj komponenti. Ovaj mehanizam je ključan za kreiranje ponovno upotrebljivih i prilagodljivih UI komponenti.
- Props mogu biti bilo koja vrijednost JavaScript. Da biste proslijedili props od roditeljske ka dječijoj komponenti, dodajte props kao atribute kada koristite dječiju komponentu u JSX-u roditelja. Evo jednostavnog primjera:

```jsx
// Parent component
function Parent() {
  const name = 'Anna';
  return <Child name={name} />;
}

// Child component
function Child(props) {
  return <h1>Hello, {props.name}!</h1>;
}
```

Možete proslediti mnogo propova koristeći operater širenja `(...)`, nakon što ga pretvorite u nešto. Evo primera:

```jsx
// Parent component
function Parent() {
  const person = {
    name: 'Anna',
    age: 25,
    city: 'New York'
  };
  return <Child {...person} />;
}
```

U ovom kodu, spread operacija `{...person}` konvertuje objekat person u privatne propse koje se prosleđuju komponenti Child.

## Uslovno prikazivanje (conditional rendering) u React

- Uslovno prikazivanje u React omogućilo je kreiranje dinamičkih korisničkih interfejsa. Koristi se za prikaz različitog sadržaja u zavisnosti od određenog stanja ili *state*-a unutar vaše aplikacije.
- Postoji nekoliko načina za uslovno prikazivanje sadržaja u React. Jedan uobičajen način je korišćenje ternarnog operatora. Evo primera:

```jsx
function Greeting({ isLoggedIn }) {
  return (
    <div>
      {isLoggedIn ? <h1>Welcome back!</h1> : <h1>Please log in</h1>}
    </div>
  );
}
```

- Drug način je korišćenje logičkog AND operatora (`&&`). Ovo je korisno kada želite da prikažete sadržaj samo ako je određeni uslov ispavljen. Evo primera:

```jsx
function Greeting({ user }) {
  return (
    <div>
      {user && <h1>Welcome, {user.name}!</h1>}
    </div>
  );
}
```

U gornjem kodu, element `h1` će biti prikazan samo ako je uslov korisnika istinit.

Takođe možete koristiti direktnu izjavu za `if` na ovaj način:

```js
function Greeting({ isLoggedIn }) {
  if (isLoggedIn) {
    return <h1>Welcome back!</h1>;
  }
  return <h1>Please sign in</h1>;
}
```

## Prikaz liste u React

- Prikazivanje liste u React je uobičajena operacija prilikom izgradnje korisničkih interfejsa.
- Liste se mogu prikazati korišćenjem metode JS data niza `map()` kako bi se iteriralo kroz niz objekata i vratio novi niz JSX elemenata.
- Na primer, ako imate niz imena koji želite prikazati kao listu, možete to učiniti ovako:

```jsx
function NameList({ names }) {
  return (
    <ul>
      {names.map((name, index) => (
        <li key={`${name}-${index}`}>{name}</li>
      ))}
    </ul>
  );
}
```

- Uvek zapamtite da pružite jedinstveni ključ za svaki element liste kako bi pomoglo React u upravljanju zadacima ažuriranja i prikazivanja. Ovim metodama, možete kreirati dinamične, bolje i efikasne liste u vašim aplikacijama za React.

## Inline stilovi (inline styles) u React

- Inline stili u React nisu dozvoljavali direktnu upotrebu stilova CSS na JSX elementima korišćenjem JavaScript.
- Da biste koristili inline stilove u React, koristite `style` atribut na JSX elementima. Atribut `style` prima objekat gde su ključevi svojstva CSS u camelCase formatu, a vrednosti su odgovarajuće. Evo primera:

```js
function Greeting() {
  return (
    <h1
      style={{ color: 'blue', fontSize: '24px', backgroundColor: 'lightgray' }}
    >
      Hello, world!
    </h1>
  );
}

export default Greeting;
```

Takođe možete pružiti stilove na drugačijem predmetu i spomenuti ih u atributu `style` ovakvim načinom:

```jsx
function Greeting() {

  const styles = {
    color: 'blue',
    fontSize: '24px',
    backgroundColor: 'lightgray'
  };

  return <h1 style={styles}>Hello, world!</h1>;
}

export default Greeting;
```

- Inline stilovi pomažu dinamičkim stilovima omogućavajući vam da koristite stilove na osnovu propsa ili state-a. Evo primera kako možete koristiti stilove na osnovu prop-a:

```jsx
function Greeting({ isImportant }) {

  const styles = {
    color: isImportant ? 'red' : 'black',
    fontSize: isImportant ? '24px' : '16px'
  };

  return <h1 style={styles}>Hello, world!</h1>;
}

export default Greeting;
```

- U gornjem kodu, stilo[vi] za `color` i `fontSize` su postavljeni prema svojstvu (prop) od `isImportant`.

## Rad sa događajima u React

- **Sistem sintetičkih događaja (Synthetic Event System)**: Ovo je način za React obradu događaja. Radi kao omotač nad prirodnim događajima, poput događaja `click`, `keydown` i `submit`. Događajni hendleri u React koriste konvenciju imenovanja camelCase. (Npr. `onClick`, `onSubmit`, itd.)

Ovo je primer korišćenja atributa `onClick` za element `button` u React:

```jsx
function handleClick() {
  console.log("Button clicked!");
}

<button onClick={handleClick}>Click Me</button>;
```

U React, funkcije za rukovanje događajima često počinju sa prefiksom `handle` koji pokazuje da obrađuju događaje, kao što su `handleClick` ili `handleSubmit`.

Kada korisnička akcija pokrene događaj, `React` prosleđuje objekat Sintetičkog Događaja vašem handler-u. Ovaj objekat funkcioniše kao nativni Objekat Događaja u standardnom `JavaScript`, pružajući svojstva kao što su ``type``, ``target``, i ``currentTarget``.

Da biste sprečili podrazumevano ponašanje, kao što je osvežavanje pretraživača tokom događaja `onSubmit`, na primer, možete pozvati metodu `preventDefault()`:

```jsx
function handleSubmit(event) {
  event.preventDefault();
  console.log("Form submitted!");
}

<form onSubmit={handleSubmit}>
  <input type="text" />
  <button>Submit</button>
</form>;
```

Takođe možete dodeliti funkciju rukovatelja na streličku (arrow) kao što je ovo:

```jsx
function handleDelete(id) {
  console.log("Deleting item:", id);
}

<button onClick={() => handleDelete(1)}>Delete Item</button>;
```

## Rad sa stanjem i `useState` Hook

- **Definicija stanja**: U React, stanje je nešto što nosi podatke komponente. Kada se stanje ažurira, komponenta će se ponovo renderovati. React tretira stanje kao nepromenljivo (imutabilno), pa ga ne bi trebalo menjati direktno.
- **`useState()` Hook**: `useState` je funkcija koja vam omogućava da eksponujete varijable stanja u funkcionalnim komponentama. Ovo je osnovna struktura:

```js
const [stateVariable, setStateFunction] = useState(initialValue);
```

U parametru stanja imate sledeće:

- `stateVariable` zadržava trenutnu vrednost stanja
- `setStateFunction` (funkcija ažuriranja) ažurira parametar stanja
- `initialValue` postavlja početno stanje

Ovo je kompletan primer za deo `Counter`:

```jsx
import { useState } from "react";

function Counter() {
  const [count, setCount] = useState(0);

  return (
    <div>
      <h2>{count}</h2>

      <button onClick={() => setCount(count - 1)}>Decrement</button>
      <button onClick={() => setCount(count + 1)}>Increment</button>
    </div>
  );
}

export default Counter;
```

## Prikaz i dijelovi za React

- **Definicija**: U React, prikazivanje je proces u kojem se delovi pojavljuju na korisničkom interfejsu (UI), obično u pretraživaču. Proces prikazivanja ima tri faze: pokretanje, renderovanje i izvršavanje.

Proces renderovanja se dešava React kada detektuje da se nešto promenilo i da je UI možda potreban za ažuriranje. Ove promene često proizlaze iz ažuriranja stanja (state) ili propova.

Kada se okidač dogodi, React ulazi u fazu renderovanja. Ovde, React ponovo procjenjuje vaše komponente i određuje šta prikazati. Da bi to učinio, React koristi laganu kopiju stvarnog DOM nazvanu virtuelni DOM. Korišćenjem virtuelnog DOM, React može brzo proveriti šta treba da se promeni u komponenti.

Faza izvršenja je mesto gde React uzima promene pripremljene iz virtuelnog DOM i postavlja ih na stvarno DOM. Drugim rečima, ovde vidiš konačan rezultat na ekranu.

## Ažuriranje stavki i redova podataka u stanju

- **Ažuriranje stvari u stanju (state)**: Ako treba da ažuriraš nešto u stanju, prvo moraš kreirati novi objekat ili kopirati postojeći, a zatim postaviti stanje za taj novi objekat. Sve što je postavljeno u stanje treba tretirati kao nepromenljivo direktno. Ovo je primer postavljanja korisničkog imena, starosti i grada. Funkcija `handleChange` se koristi za obradu ažuriranja podataka korisnika:

```jsx
import { useState } from "react";

function Profile() {
  const [user, setUser] = useState({ name: "John Doe", age: 31, city: "LA" });

  const handleChange = (e) => {
    const { name, value } = e.target;

    setUser((prevUser) => ({...prevUser, [name]: value}));
  };

  return (
    <div>
      <h1>User Profile</h1>
      <p>Name: {user.name}</p>
      <p>Age: {user.age}</p>
      <p>City: {user.city}</p>

      <h2>Update User Age </h2>
      <input type="number" name="age" value={user.age} onChange={handleChange} />

      <h2>Update User Name </h2>
      <input type="text" name="name" value={user.name} onChange={handleChange} />

      <h2>Update User City </h2>
      <input type="text" name="city" value={user.city} onChange={handleChange} />
    </div>
  );
}

export default Profile;
```

- **Ažuriranje reda podataka u stanju**: Kada ažurirate red podataka u stanju (state), važno je da ga ne menjate direktno koristeći metode kao `push()` ili `pop()`. Umesto toga, trebalo bi kreirati novi red kada ažurirate stanje:

```jsx
const addItem = () => {
  const newItem = {
    id: items.length + 1,
    name: `Item ${items.length + 1}`,
  };

  // Creates a new array
  setItems((prevItems) => [...prevItems, newItem]);
};
```

Ako želite da uklonite elemente iz niza podataka, koristite metodu `filter()`, koja vraća novi niz nakon filtriranja onoga što želite da uklonite:

```jsx
const removeItem = (id) => {
  setItems((prevItems) => prevItems.filter((item) => item.id !== id));
};
```

## Referenciranje vrednosti korišćenjem refs-a

- **Svojstvo `ref`**: Možete dobiti čvor DOM u React koristeći svojstvo `ref`. Ovo je primer za prikazivanje `ref` kako bi se istakao element `input`. Svojstvo `current` se koristi za dobijanje trenutne vrednosti tog `ref`:

```jsx
import { useRef } from "react";

const Focus = () => {
  const inputRef = useRef(null);

  const handleFocus = () => {
    if (inputRef.current) {
      inputRef.current.focus();
    }
  };

  return (
    <div>
      <input ref={inputRef} type="text" placeholder="Enter text" />
      <button onClick={handleFocus}>Focus Input</button>
    </div>
  );
};

export default Focus;
```

## Rad sa `useEffect` Hook

- **`useEffect()` Hook**: U React, efekat je bilo šta što se dešava izvan procesa renderovanja komponente. To znači bilo šta što React neće biti direktno obrađeno kao deo renderovanja korisničkog interfejsa (UI). Uobiđi primeri uključuju dohvaćanje podataka, ažuriranje naslova kartice pretraživača, čitanje ili pisanje u lokalno skladište pretraživača, dobijanje lokacije korisnika i slično. Ove operacije interaguju sa spoljnim svetom i poznate su kao nuspojave (side effects). React pruža hook `useEffect` kako bi vam omogućio da rukujete tim nuspojavama. `useEffect` omogućava izvršavanje funkcija nakon što je komponenta renderovana ili ažurirana.

```js
import { useEffect } from "react";

useEffect(() => {
 // Your side effect logic (usually a function) goes here
}, [dependencies]);
```

Funkcija efekta se izvršava nakon što je sekcija prikazana, dok opcioni argument ``dependencies`` kontroliše kada će efekat biti pokrenut.

Napomena da `dependencies` može biti niz od "akcionih vrednosti" (state, props, funkcije, parametri, itd.), prazan niz ili potpuno neispravan. Ovako te opcije kontrolišu kako `useEffect` radi:

- Ako je `dependencies` red koji sadrži barem jednu dinamičku vrednost, efekat će biti pokrenut svaki put kada se izmene.

- Ako je `dependencies` prazan red, `useEffect` će se izvršiti samo jednom kada se sekcija osveži prvi put.

- Ako nedostaje `dependencies`, efekat će biti primenjen svaki put kada se deo popravi ili ažurira.

## Kako kreirati prilagođene hook-ove

- **Prilagođeni hookovi**: Prilagođeni hook omogućava vam da izvučete ponovno upotrebljivu logiku iz nekog dela, kao što su dohvaćanje podataka, upravljanje stanjem, menjanje stanja i sporedni efekti poput praćenja statusa online. U React, svi kreirani hookovi počinju rečju `use`, stoga vaš prilagođeni hook mora pratiti taj ugovor.

Ovo je primer za kreiranje hook-a `useDebounce`:

```jsx
function useDebounce(value, delay) {
  const [debouncedValue, setDebouncedValue] = useState(value);

  useEffect(() => {
    const handler = setTimeout(() => {
      setDebouncedValue(value);
    }, delay);

    return () => {
      clearTimeout(handler);
    };
  }, [value, delay]);

  return debouncedValue;
}

export { useDebounce };
```

## Rad sa formularima u React

- **Kontrolisani ulaz (Controlled Inputs)**: Ovo je kada sačuvate vrednost polja za unos u stanje (state) i ažurirate ga preko događaja `onChange`. To vam daje potpunu kontrolu nad podacima forme i omogućava trenutnu validaciju i prikaz uslova.

```jsx
import { useState } from "react";

function App() {
  const [name, setName] = useState("");

  const handleChange = (e) => {
    setName(e.target.value);
  };

  const handleSubmit = (e) => {
    e.preventDefault();
    console.log(name);
  };

  return (
    <>
      <form onSubmit={handleSubmit}>
        <label htmlFor="name">Your name</label> <br />
        <input value={name} id="name" onChange={handleChange} type="text" />
        <button type="submit">Submit</button>
      </form>
    </>
  );
}

export default App;
```

- **Nedokontrolisani ulazi (Uncontrolled Inputs)**: Umesto obrade unosa putem hook-a `useState`, nedokontrolisani unos u HTML čuva svoj interni status uz pomoć DOM. Pošto DOM kontroliše vrednosti unosa, potrebno je da preuzmete vrednosti delova unosa koristeći `ref`.

```jsx
import { useRef } from "react";

function App() {
 const nameRef = useRef();

 const handleSubmit = (e) => {
   e.preventDefault();
   console.log(nameRef.current.value);
 };

 return (
   <form onSubmit={handleSubmit}>
     <label htmlFor="name">Your</label>{" "}

 );
```

## Rad sa `useActionState` Hook

- **Akcije na serveru (Server Actions)**: Ovo su funkcije koje se izvršavaju na serveru kako bi omogućile upravljanje formularima direktno na serveru, bez potrebe za endpointima API. Evo primera iz Next.js aplikacije:

```js
"use server";

async function submitForm(formData) {
 const name = formData.get("name");
 return { message: `Hello, ${name}!` };
}
```

Uputstva za `"use server"` označavaju ovu funkciju kao server akciju.

- **`useActionState` Hook**: Ovaj hook ažurira stanje (state) na osnovu rezultata slanja forme. Ovo je osnovna struktura za hook `useActionState`:

```js
const [state, action, isPending] = useActionState(actionFunction, initialState, permalink);
```

- `state` je trenutno stanje koje vraća akcija.
- `action` je funkcija koja pokreće server akciju.
- `isPending` je boolean koji ukazuje da li se akcija trenutno izvršava ili ne.
- Parametar `actionFunction` je sama server akcija.
- `initialState` je parametar koji predstavlja početno stanje stanja pre izvršavanja akcije.
- `permalink` je opcioni niz karaktera koji nosi jedinstveni URL stranice koju forma menja.

## Preuzimanje podataka u React

- **Opcije za dohvaćanje podataka**: Postoji mnogo različitih načina za dohvaćanje podataka u React. Možete koristiti naivni Fetch API, ili alate trećih strana kao što su Axios ili SWR.
- **Najčešće korišćene varijable stanja prilikom dohvaćanja podataka**: Bez obzira na metodu koju izaberete za dohvaćanje podataka u React, postoji nekoliko varijabli stanja koje biste trebali pratiti. Prvo je sami podatak. Drugo je praćenje da li se podaci još uvek dohvaćaju. Treće je varijabla stanja koja će uhvatiti sve greške koje se mogu desiti tokom dohvaćanja podataka.

```js
const [data, setData] = useState(null);
const [loading, setLoading] = useState(true);
const [error, setError] = useState(null);
```

Pošto je uzimanje podataka nuspojava, bolje je koristiti `Fetch API` unutar hook-a `useEffect`.

```js
useEffect(() => {
  const fetchData = async () => {
    try {
      const res = await fetch("https://jsonplaceholder.typicode.com/posts");
      
      if (!res.ok) {
        throw new Error("Network response was not ok");
      }

      const data = await res.json();
      setData(data);
    } catch (err) {
      setError(err);
    } finally {
      setLoading(false);
    }
  };

  fetchData();
}, []);
```

Zatim možete prikazati poruku učitavanja ako nije uspešno preuzetih podataka, poruku o grešci ako je došlo do greške, ili rezultate.

```jsx
if (loading) {
  return <p>Loading...</p>;
}

if (error) {
  return <p>{error.message}</p>;
}

return (
  <ul>
    {data.map((post) => (
      <li key={post.id}>{post.title}</li>
    ))}
  </ul>
);
```

## Rad sa `useOptimistic` Hook

- **`useOptimistic` Hook**: Ovaj hook se koristi da bi se osiguralo da korisnički interfejs (UI) nastavi da reaguje dok čeka akciju async da se završi u pozadini. Pomaže u upravljanju "optimistic updates" na UI-u, strategijom gde se trenutno ažuriraju podatci na UI-u na osnovu očekivanih rezultata akcije, kao što je čekanje odgovora servera.

Evo osnovne strukture:

```js
const [optimisticState, addOptimistic] = useOptimistic(actualState, updateFunction);
```

- `optimisticState` je privremeno stanje koje se ažurira trenutno radi boljeg korisničkog iskustva.
- `addOptimistic` je funkcija koja se koristi za ažuriranje optimističnog stanja pre nego što stvarni status promeni.
- `actualState` je stvarna vrednost stanja izvedena iz rezultata akcije, kao što je preuzimanje podataka sa servera.
- `updateFunction` je funkcija koja kontroliše kako bi optimistično stanje trebalo da se ažurira kada se pozove.

Ovo je primer korišćenja `hook`-a `useOptimistic` u delu `TaskList`:

```js
"use client";

import { useOptimistic } from "react";

export default function TaskList({ tasks, addTask }) {
  const [optimisticTasks, addOptimisticTask] = useOptimistic(
    tasks,
    (state, newTask) => [...state, { text: newTask, pending: true }]
  );

  async function handleSubmit(e) {
    e.preventDefault();
    const formData = new FormData(e.target);

    addOptimisticTask(formData.get("task"));

    addTask(formData);
    e.target.reset();
  }

  return <>{/* UI */}</>;
}
```

- **`startTransition`**: Ovo se koristi za prikaz dela interfejsa korisnika (UI) i postavljanje ažuriranja stanja kao nehitnog prelaska/tranzicije. Ovo omogućava da UI nastavi odgovarati tokom teških ažuriranja. Evo osnovne strukture:

```js
startTransition(action)
```

`action` izvršava ažuriranje stanja ili pokreće logiku vezanu za tranziciju. Ovo osigurava da hitna ažuriranja UI-a (kao što su kucanje ili klikanje) nisu blokirana.

- **`useMemo` Hook**: Ovaj hook se koristi za skladištenje izračunatih vrednosti. Evo primera čuvanja rezultata sortiranja velikog niza podataka. `expensiveSortFunction` će se izvršiti samo kada `largeArray` promeni:

```js
const memoizedSortedArray = useMemo(
  () => expensiveSortFunction(largeArray),

);
```

## React Router

- **Uvod**: React Router je biblioteka drugih komponenti koja vam omogućava dodavanje rutiranja u vaše aplikacije za React. Da biste počeli, moraćete da instalirate React Router u postojeći projekat React na sledeći način:

```bash
npm i react-router
```

Zatim u fajlu `main.jsx` ili `index.jsx`, moraćete postaviti strukturu putanje ovako:

```jsx
import { StrictMode } from "react";
import { createRoot } from "react-dom/client";
import { BrowserRouter, Routes, Route } from "react-router";
import App from "./App.jsx";

import "./index.css";

createRoot(document.getElementById("root")).render(
  <StrictMode>
    <BrowserRouter>
      <Routes>
        <Route path="/" element={<App />} />
      </Routes>
    </BrowserRouter>
  </StrictMode>
);
```

`path` na `element` koristi se za spajanje URL sa elementima UI-ja. U ovom primeru, postavljamo putanju (link) za početnu stranicu koja vodi ka sekciji `App`.

- **Mnogi komentari i konfiguracija puta**: Uobičajeno je u velikim programima imati mnogo komentara i konfiguraciju rute kao ovde:

```jsx
<Routes>
  <Route index element={<Home />} />
  <Route path="about" element={<About />} />

  <Route path="products">
    <Route index element={<ProductsHome />} />
    <Route path=":category" element={<Category />} />
    <Route path=":category/:productId" element={<ProductDetail />} />
    <Route path="trending" element={<Trending />} />
  </Route>
</Routes>
```

Prop `index` u ovim primerima predstavlja podrazumevani način za segment putanje. Stoga će deo `Home` biti prikazan u korenskom putu `/`, dok će deo `ProductsHome` biti prikazan u putanji `/products`.

- **Ugnježđivanje ruta (Nesting Routes)**: Možete definisati jednu rutu unutar druge rute, što uzrokuje da putanja detalne rute bude dodata na putanju roditeljske rute.

```jsx
<Route path="products">
  <Route path="trending" element={<Trending />} />
</Route>
```

U gornjem primeru, path za proizvode koji su u trendu će biti products/trending.

- **Динамички сегменти (Dynamic Segments)**: Динамички сегмент је место где било који део пута `URL` је динамичан.

```jsx
<Route path=":category" element={<Category />} />
```

U ovom primeru imamo dinamički deo koji se zove `category`. Kada korisnik ode na URL kao products/brass-instruments, pogled će se promeniti u deo `Category` i možete uzeti relevantne podatke na osnovu tog dela.

- **`useParams` Hook**: Ovaj hook se koristi za dobijanje dinamičkih parametara iz puta (path) URL.

```js
import { useParams } from "react-router";

export default function Category() {
  let params = useParams();
   {/* Accessing the category param: params.category */}
   {/* rest of code goes here */}
}
```

## Prop Drilling

- **Definicija**: Prop drilling je proces prosleđivanja prop-ova od roditeljskog komponenta do duboko ugnježdene detske komponente, čak i ako neke detske komponente ne zahtevaju te prop-ove.

## Upravljanje stanjem

- **Context API**: Context znači mesto gde rodenski komponent čini da su informacije dostupne detilnim komponentama bez nego direktnog prosleđivanja putem props-ova. `createContext` se koristi za kreiranje kontekst objekta koji predstavlja kontekst koji će druge komponente čitati. `Provider` se koristi za distribuciju vrednosti konteksta detilnim komponentama.

```js
import { useState, createContext } from "react";

const CounterContext = createContext();

const CounterProvider = ({ children }) => {
  const [count, setCount] = useState(0);

  return (
    <CounterContext.Provider value={{ count, setCount }}>
      {children}
    </CounterContext.Provider>
  );
};

export { CounterContext, CounterProvider };
```

- **Redux**: Redux upravlja quản state managementom pružajući centralni store i strogu kontrolu nad ažuriranjima stanja. Koristi predvidljiv obrazac sa akcijama (actions), reducerima (reducers) i middleware-om. Akcije su nosioci informacija koji šalju podatke iz vaše aplikacije u Redux store, često pokrenuti interakcijom korisnika. Reduceri su funkcije koje definišu kako bi se stanje trebalo promeniti na osnovu tih akcija, osiguravajući da je stanje ažurirano na nepromenljiv način (immutable). Middleware, s druge strane, funkcioniše kao most između dispatch-ovanja akcije i reducer-a, omogućavajući vam dodavanje funkcionalnosti Redux-u (npr. logovanje događaja, rukovanje operacijama async) bez menjanja osnovnog toka.

- **Zustand**: Ovo rešenje za upravljanje stanjem je pogodno za male do srednje aplikacije. Radi korišćenjem `hook`-a `useStore` da dohvati stanje direktno u komponentama i stranicama. Ovo vam omogućava da menjate i dobijate podatke bez potrebe za actions, reducers ili providerima.

## Sekcija za React Server

- **Definicija**: Delovi React Server su delovi React koji se prikazuju samo na serveru, i šalju samo završni HTML klijentu. To znači da ti delovi mogu direktno pristupiti resursima servera i značajno smanjiti količinu JavaScript koja se šalje u pretraživač.

## Razlika između stvarnih i vidljivih performansi

- **Percipirane performanse**: Ovo je kako korisnici osećaju performanse veb sajta. To je način na koji ga procenjuju po njegovoj odgovornosti i pouzdanosti. Ovo je subjektivna mera, pa je teško meriti, ali je izuzetno važna jer korisničko iskustvo određuje uspeh ili neuspeh sajta.
- **Stvarna performanse**: Ovo su merljive i stvarne performanse veb sajta. Meri se korišćenjem metrika kao što su vreme učitavanja stranice, vreme odgovora servera i vreme renderovanja/prikaza. Ove mere pogođene su od mnogo faktora vezanih za mrežu i sam kod.

## Metoda za poboljšanje vidljivih performansi

- **Opterećenje po zahtevu (Lazy Loading)**: Ova tehnika smanjuje inicijalno vreme učitavanja tako što odlaže učitavanje resursa koji nisu neophodni u pozadini.
- **Smanjite kašnjenje fontova**: Ako vaš sajt koristi specifične fontove, trebalo bi da smanjite i kašnjenje učitavanja fontova, jer to može izazvati skakanje ili prikazivanje zamenskih fontova dok se primarni font ne učita. Preporuka je korišćenje zamenskog fonta koji liči na glavni font, kako bi promena bila minimalna.
- **Koristite indikatore učitavanja**: Prikaz indikatora učitavanja za proces koji traje dugo čim korisnik klikne na element može učiniti da se korisnik oseća povezano sa procesom i da vreme čekanja izgleda kraće.

## Glavni koncepti performansi

- **Redos izvora (Source order)**: Ovo se odnosi na način na koji elementi HTML su postavljeni u dokumentu. Ovo određuje šta se učitava prvo i može značajno uticati na performanse i Pristupačnost.

Neki od najboljih metoda za organizaciju izvora uključuju:

- Postavljanje važnog sadržaja kao što su naslovi sekcija, navigacija ili glavni tekst na vrh strukture `HTML`.
- Odlaganje nebitnih skripti, kao što su one za analitiku, ili džetova trećih strana, kako bi ne blokirale renderovanje.
- Korišćenje progresivnog poboljšanja (progressive enhancement), kako bi se osiguralo da osnovno iskustvo radi čak i pre učitavanja stilova i skripti. Progresivno poboljšanje je način izgrađivanja veb sajtova i aplikacija baziran na tome da stranica prvo funkcioniše sa `HTML`.

Evo primera dobro organizovanog izvora, koristeći najbolje metode koje smo prošli:

```html
<h1>Welcome to FastSite!</h1>
<p>Critical information loads first.</p>
<script src="slow-script.js" defer></script>
```

- **Put kritičnog renderovanja (Critical Rendering Path)**: Ovo je niz koraka koje pregledač prati kako bi pretvorio kod u sliku na ekranu.
- **Latencija (Latency)**: Ovo je vreme koje zahtev treba da pređe između pregledača i servera. Drugim rečima, velika latencija je isto kao spori sajt.

Neki načini smanjenja kašnjenja uključuju:

- Korišćenje CDN-ova, ili Content Delivery Networks, za dostavu fajlova sa bliskih lokacija.
- Omogućavanje kompresije korišćenjem Gzip-a radi smanjenja veličine fajlova.
- Optimizacija slika i korišćenje lazy loading-a.

```html
<img src="placeholder.jpg" data-src="real-image.jpg" loading="lazy">
```

## Obnavljanje INP

- **Definicija**: INP (Interaction to Next Paint) meri kako stranica generalno reaguje merenjem vremena od kada korisnik izvrši akciju, kao što je klikatanje ili pritiskanje dugmeta, do trenutka kada pregledač poboljša prikaz. Manji INP ukazuje na stranicu koja reaguje brže.

Ovo su načini za poboljšanje INP:

- Smanjite opterećenje glavnog thread-a razdvajanjem dugih zadataka JavaScript.
- Koristite `requestIdleCallback()` za nekritičke skripte. Ovo će zadatak staviti u red da se izvrši kada je pregledač u periodu mirovanja (idle).
- Odložite ili učitajte leniško teške resurse, kao što je prethodno objašnjeno.
- Optimizujte ručkice događaja. Ako se ovi ručkici pokreću često ili obavljaju te zadatke, to može usporiti stranicu i povećati INP. Rešenje je korišćenje debounsing-a. Debouncing osigurava da se zadatak izvrši tek nakon što korisnik prestane sa kucanjem tokom kratkog kašnjenja – na primer 300ms. Ovo sprečava nepotrebne proračune i poboljšava performanse.

## Kako funkcioniše prikaz u pregledaču

- **Kako funkcioniše renderovanje**: Prvo, pretraživač parsira HTML i gradi DOM. Zatim, pretraživač obrađuje CSS, konstruišući CSS Object Model, ili CSSOM. Ovo je druga drvo struktura koja određuje kako bi elementi trebalo da budu stilizovani. Na kraju, pretraživač crta piksele na ekranu, prikazujući svaki element prema izračunanim stilovima i rasporedu (layout). Na kompleksnim stranicama, ovo može uključivati više slojeva spojena zajedno kako bi se dobio konačni vizuelni rezultat.

## Kako performanse utiču na održivost

- **Osnovno objašnjenje**: Internet doprinosi sa približno 2% globalnih emisija ugljen-dioksida—što je jednako avio-industriji! Svaki prebačeni bajt zahteva struju, od centara podataka do korisničkih uređaja. Veliki fajlovi i neefikasni skriptovi znače veće potrošnje energije. Veb sa visokim performansama nije samo brz, već smanjuje i nepotrebnu obradu i potrošnju energije.

## Načini za smanjenje vremena učitavanja stranice

- **Optimizujte medijske resurse**: Velike slike i video snimci su glavni razlozi dugog vremena učitavanja. Optimizovanjem ovih resursa, možete značajno povećati brzinu vašeg sajta. To uključuje stvari kao što su komprimovanje slika, korišćenje modernih formata poput WebP i primena lazy loading za resurse.
- **Koristite keš pretraživača**: Keš omogućava pretraživačima da sačuvaju delove vašeg sajta unutar zemlje, smanjujući tako vreme učitavanja za povratnike.
- **Smanjite i komprimujte fajlove**: Smanjenje veličine vaših fajlova može povećati brzinu preuzimanja. To uključuje smanjenje veličine prebačenih fajlova i komprimovanje fajlova sa CSS i JavaScript.

## Poboljšanje "vreme do upotrebljivosti"

- **Definicija**: "time to usable" je vreme od trenutka kada korisnik traži stranicu do trenutka kada može u potpunosti da sa njom interaguje. Da biste poboljšali "time to usable", možete implementirati *lazy loading* za vaše resurse ili smanjiti resurse koji blokiraju renderovanje.

## Važne metrike za merenje performansi

- **First Contentful Paint au FCP**: Mjeri kako se prvi deo sadržaja—tekst ili slika—brzo pojavljuje na ekranu. Dobar FCP je ispod 1,8 sekunde, a loš FCP je više od 3 sekunde. Možete proveriti svoj FCP koristeći Chrome DevTools i pregledavajući tab za performanse (performance).
- **Total Blocking Time**: Ovo pokazuje koliko je glavni thread bio blokiran teškim zadacima JavaScript. Ako je Total Blocking Time (TBT) visok, korisnici doživljavaju spor interfejs. Da biste poboljšali TBT, razdelite dugačke zadatke i odložite nebitne skripte.
- **Bounce Rate**: Ovo je procenat posetioca koji napuštaju bez interakcije. Ako vaš sajt ima visok *bounce rate*, moguće je da je razlog to što su vaše stranice spore.
- **Unique Users**: Ova metrika prati koliko jedinstvenih korisnika posećuje vaš sajt. Da biste videli *Bounce Rate* i *Unique Users*, možete koristiti Google Analytics. To će vam omogućiti da pratite ove metričke podatke i poboljšate angažovanje.

## Alati za merenje performansi

- **Chrome DevTools**: Chrome DevTools je alat ugrađen u Google Chrome koji vam omogućava da analizirate i debugujete performanse u realnom vremenu. DevTools će prikazati vreme učitavanja, potrošnju CPU-a i kašnjenje renderovanja. Izuzetno je koristan za testiranje First Contentful Paint, ili FCP, što predstavlja način na koji korisnik vidi prvi sadržaj. Ako vam sajt izgleda sporo, DevTools će vam pomoći da identifikujete usko grlo (bottlenecks).
- **Lighthouse**: Ovo je automatski alat koji proverava performanse, SEO i Pristupačnost (Accessibility).
- **WebPageTest**: Ovaj alat vam omogućava da testirate kako se vaš sajt učitava iz različitih lokacija i uređaja. Pruža detaljan pregled Speed Index vašeg sajta, Total Blocking Time, i drugih ključnih metrika performansi. Ako želite da znate kako pravi korisnici pristupaju vašem sajtu širom sveta, WebPageTest je alat za to.
- **PageSpeed Insights**: Ovaj alat analizira vaš sajt i predlaže brka poboljšanja za mobilne telefone i računare. Kaće vam šta usporava vaš sajt i daje specifične preporuke kao što su optimizacija slika, uklanjanje skripti koje blokiraju renderovanje, i smanjenje vremena odgovora servera. PageSpeed Insights je brz i jednostavan način da vidite kako Google procenjuje performanse vašeg sajta.
- **Real User Monitoring**: RUM alati prave praćenje stvarnog ponašanja korisnika, pokazujući kako stvarni posetioci pristupaju vašem sajtu. Popularni RUM alati uključuju Google Analytics, koji prati vreme učitavanja stranice i *bounce rates*, kao i New Relic ili Datadog, koji prate probleme performansi u realnom vremenu. Ako želite podatke od stvarnih korisnika, RUM alati su neophodni.

## Rad sa Performance Web API-jima

- **Definicija**: Performance Web API-ji omogućavaju programerima da prate kako se web stranica učitava i reaguje direktno u kodu. Ovi API-ji vam omogućavaju da merite vreme učitavanja stranice, pratite kašnjenje renderovanja i interakcije, te analizirate vreme izvršavanja JavaScript.
- **`performance.now()`**: API Ovo vam daje vremenske oznake visoke preciznosti (u milisekundama) za merenje vremena koje je potrebno da se različiti delovi vaše veb stranice učitaju.

```js
const start = performance.now();  
// Run some code here  
const end = performance.now();  

console.log(`Execution time: ${end - start}ms`);
```

- **Performance Timing API**: API Ovo vam daje pregled svakog koraka učitavanja stranice, od DNS upita do `DOMContentLoaded`.

```js
const timing = performance.timing;  

const pageLoadTime = timing.loadEventEnd - timing.navigationStart;  
console.log(`Page load time: ${pageLoadTime}ms`);
```

- **`PerformanceObserver`**: API Ovo prati događaje performansi kao što su layout shifts, dugotrajni zadaci i interakcije korisnika.

```js
const observer = new PerformanceObserver((list) => {  
  list.getEntries().forEach((entry) => {  
    console.log(`Long task detected: ${entry.duration}ms`);  
  });  
});  

observer.observe({ type: "longtask", buffered: true });
```

## Metoda za poboljšanje performansi CSS

- **Animacija CSS**: Animiranje nekih svojstava za CSS, kao što su dimenzije, pozicija i raspored (layout), uzrokuje proces nazvan "reflow", gde pregledač ponovo izračunava poziciju i geometriju određenih elemenata na stranici. Ovo zahteva repaint, što je računski skupo. Stoga se savetuje smanjiti broj animacija za CSS koliko je moguće ili barem dati korisniku opciju da ih uključi ili isključi.

## Metode za poboljšanje performansi JavaScript

- **Razdvajanje koda (Code Splitting)**: Razdvajanje vašeg koda JavaScript na module koji su neophodni i oni koji nisu neophodni takođe je važno. Na ovaj način, možete unapred učitati one neophodne što je moguće brže i odložiti one manje neophodne kako bi se stranice prikazale što je moguće brže.
- **Izmena DOM**: Zapamtite da izmena DOM znači proces dinamičke promene sadržaja stranice za JavaScript u interakciji sa Document Object Model (DOM). Izmena DOM je računski zahtevna. Smanjenje količine izmena DOM u vašem kodu JavaScript poboljšaće performanse.

## Sistemi za CSS

- **Sistemi za CSS**: Sistemi CSS mogu povećati brzinu vašeg rada, kreirati dosledan vizuelni stil na veb stranici, osigurati da vaš dizajn izgleda isto u više pregledača, i organizovati vaš kod CSS sa dobrim rasporedom.
- **Popularni sistemi za CSS**: Neki popularni sistemi CSS su Tailwind CSS, Bootstrap, Materialize i Foundation.
- **Mogući nedostaci**:
  - CSS koji pruža sistem može kolidirati sa vašim specifičnim CSS.
  - Vaš sajt može izgledati slično drugim sajtovima koji koriste isti okvir.
  - Veliki sistemi mogu uzrokovati probleme sa performansama.

## Dva tipa sistema za CSS

- **Utility-first sistemi CSS**: Ovi sistemi imaju male klase sa specifičnom svrhom, kao što je postavljanje margina, paddinga ili pozadinske boje. Ove male klase možete primeniti direktno na elemente HTML kako je potrebno. Tailwind CSS se smatra utiliti-prvim sistemom CSS.

Ovo je primer korišćenja Tailwind CSS za stilizovanje dugmeta.

```html
<button class="bg-blue-500 text-white font-bold py-2 px-4 rounded-full hover:bg-blue-700">
  Button
</button>
```

- **Komponentno-zavisni sistemi CSS**: Ovi sistemi imaju gotove komponente sa već definisanim stilovima koje možete dodati na vaš sajt. Komponente su dostupne u zvaničnoj dokumentaciji sistema CSS, i možete ih kopirati i zalepiti u svoj projekat. Bootstrap se smatra komponentno-zavisnim sistemom CSS.

Ovo je primer korišćenja Bootstrapa za kreiranje liste grupa. Umesto korišćenja malih klasa na elementima HTML, dodaćete ceo deo, uključujući strukturu HTML.

```html
<div class="card" style="width: 25rem;">
  <ul class="list-group list-group-flush">
    <li class="list-group-item">HTML</li>
    <li class="list-group-item">CSS</li>
    <li class="list-group-item">JavaScript</li>
  </ul>
</div>
```

## Tailwind CSS

Tailwind je utility-first framework CSS. Umesto pisanja specifičnih pravila za CSS, gradiš svoje dizajne kombinovanjem malih utilitarnih klas direktno na vašem HTML.

### Utility alati za responsivni dizajn (Responsive Design)

Tailwind koristi `sm:`, `md:`, i `lg:` za primenu stilova na različitim veličinama ekrana.

```html
<div class="w-full md:w-1/2 lg:flex-row">
  Responsive layout
</div>
```

### Utilityji za Flexbox

Klasama kao `flex`, `flex-col`, `justify-around`, na `items-center` čini ga lakšim za kreiranje dinamičkih modela.

```html
<div class="flex flex-col md:flex-row justify-around items-center">
  <p>Column on small screens</p>
  <p>Row on medium and larger screens</p>
</div>
```

### Utilities za Grid

Tailwind uključuje utilitete za CSS Grid, kao što su `grid`, `grid-cols-1`, i `md:grid-cols-3`.

```html
<div class="grid grid-cols-1 md:grid-cols-3 gap-8">
  <div class="bg-gray-100 p-4">Column 1</div>
  <div class="bg-gray-100 p-4">Column 2</div>
  <div class="bg-gray-100 p-4">Column 3</div>
</div>
```

### Alati za prostor

Utility-ji poput `mt-8`, `mx-auto`, `p-4`, na `gap-4` pomažu u kreiranju stabilnog prostora bez pisanja CSS.

```html
<div class="mt-8 p-4 bg-indigo-600 text-white">
  Spaced content
</div>
```

### Alati za tipografiju (Tipografija)

Alati kao `uppercase`, `font-bold`, `font-semibold`, i na `text-4xl` kontrolišu izgled teksta.

Možete postaviti promenljivu veličinu fonta na breakpointima, kao što su `text-3xl` i `md:text-5xl`.

```html
<h1 class="text-3xl md:text-5xl font-semibold text-center">
  Responsive Heading
</h1>
```

### Boje i stanja za hover

Tailwind nudi mnoge boje, kao što su `text-red-700`, `bg-indigo-600`, i `bg-gray-100`.

Klase kao `hover:bg-pink-600` olakšavaju interakcione efekte.

```html
<a href="#" class="bg-pink-500 hover:bg-pink-600 text-white px-4 py-2 rounded-md">
  Hover Me
</a>
```

### Granice, Prstenovi, Efekti

- **Borders**: `border-2 border-red-300` dodaje granice sa specifičnom debljinom i bojom.
- **Rings**: `ring-1 ring-gray-300` stvara efekte slične konturi (outline), često se koristi za fokus ili kartice.
- **Trokutači i skaliranje**: Klase kao što su `rounded-md`, `rounded-xl`, i `scale-105` dodaju dobar izgled.

```html
<div class="p-6 rounded-xl ring-2 ring-fuchsia-500 scale-105">
  Highlighted card
</div>
```

### Gradienti

Tailwind podržava gradientne utilitete kao `bg-gradient-to-r from-fuchsia-500 to-indigo-600`.

```html
<div class="p-4 text-white bg-gradient-to-r from-fuchsia-500 to-indigo-600">
  Gradient background
</div>
```

## CSS Predprocesori

- **CSS preprocessor**: CSS preprocessor je alat koji proširuje standardni CSS. Parsira kod sa proširenom sintaksom u originalni fajl CSS. Može pomoći u pisanju čistog, ponovno upotrebljivog, neponavljajućeg i skalabilnog CSS za velike projekte.
- **Funkcionalnosti**: Neke funkcionalnosti koje mogu biti kreirane pomoću CSS preprocessors su varijable, mixini, ugnježdenje (nesting) i nasleđivanje selektora.
- **Popularni CSS preprocessors**: Neki od popularnih preprocesora su Sass, Less i Stylus.
- **Mogući nedostaci**:
  - Kompilacija pravila iz CSS u standardni CSS može povećati opterećenje.
  - Prevedeni kod može biti težak za modifikovanje.

## Sass

- **Sass**: Ovo je jedan od najpoznatijih CSS preprocesora. Sass stoji za "Syntactically Awesome Style Sheets."
- **Funkcionalnosti koje podržava Sass**: Sass podržava funkcionalnosti kao što su varijable, ugnježđena pravila CSS, moduli, mixini, nasleđivanje i osnovni aritmetički operateri

## Dve sintakse koje Sass podržava

- **SCSS Sintaksa**: SCSS (Sassy CSS) proširuje osnovnu sintaksu CSS. Ovo je najčešće korišćena sintaksa za Sass. SCSS fajlovi imaju ekstenziju `.scss`.

```sass
$primary-color: #3498eb
```

### Mixins

- **Mixins**: Mixins omogućavaju da kombinujete mnoge svojstva za CSS i njihove vrednosti pod imenom, te ponovo koristite taj blok koda CSS u vašem stilskom listu (stylesheet).

Ovo je primer definisanja mixina u SCSS sintaksi. U ovom primeru, mixin se zove `center-flex`. Ima tri svojstva CSS za centriranje stvari korišćenjem flexbox-a.

```scss
@mixin center-flex {
```

```scss
section {
```

## Ručni i automatski testovi

- **Ručno testiranje**: Tokom ručnog testiranja, tester prolazi kroz svaki deo softvera i testira različite funkcije kako bi se osigurao da radi ispravno. Ako se tokom procesa testiranja otkrije bilo kakva greška, tester će to prijaviti timu za razvoj softvera da bi je popravili.
- **Automatsko testiranje**: Tokom automatskog testiranja, možete napisati drugačiji program koji proverava da li vaš softver radi kako je očekivano.

## Testiranje jedinica (Unit Testing)

- **Unit Testing**: U unit testingu, testiraš svaku funkciju kako bi osigurao da sve radi onako kako je očekivano. Unit testovi takođe mogu služiti kao vrsta dokumentacije za tvoju aplikaciju jer predstavljaju očekivano ponašanje tvog koda.
- **Single Responsibility Principle**: Ovaj princip predlaže da svaka funkcija bude mala i odgovorna samo za jednu stvar.
- **Mifumo ya kawaida ya majaribio na JavaScript**: Neki popularni frameworkovi za testiranje su Jest, Mocha, i Vitest. Jest je popularan okvir (framework) za unit testove.

Ovo je primer jedinica testiranja korišćenjem Jest-a.

Prvo, možete kreirati funkciju koja vraća novu sekvencu karaktera:

```javascript
export function getFormattedWord(str) {
  if (!str) return '';
  return str.charAt(0).toUpperCase() + str.slice(1);
}
```

U drugoj datoteci od `getFormattedWord.test.js`, možete napisati testove kako biste potvrdili da funkcija radi kako je očekivano. Datoteka `getFormattedWord.test.js` će izgledati ovako:

```javascript
import { getFormattedWord } from "./getFormattedWord.js";
import { test, expect } from "jest";

test('capitalizes the first letter of a word', () => {
  expect(getFormattedWord('hello')).toBe('Hello');
});
```

- **Funkcija za `expect`**: Funkcija `expect` se koristi za testiranje vrednosti.
- **Matcher**: Matcher je funkcija koja proverava da li vrednost radi kako se očekuje. U gornjem primeru, matcher je `toBe()`. Jest ima različite matchere.

Da biste koristili Jest, prvo morate instalirati paket `jest` koristeći `npm i jest`. Takođe ćete morati da dodate skript u vaš fajl `package.json` na sledeći način:

```js
"scripts": {
  "test": "jest"
},
```

Zatim, možete pokrenuti komandu `npm run test` da pokrenete svoje testove.

## Životni ciklus razvoja softvera

- **Različite faze životnog ciklusa razvoja softvera**:
  - **Faza planiranja**: Programski tim prikuplja predložene radne zahteve od zainteresovanih strana.
  - **Faza dizajna**: Programski tim razlaže zahteve i određuje najbolje metode rešenja.
  - **Faza implementacije**: Programski tim razlaže zahteve u manje zadatke i izvršava ih.
  - **Faza testiranja**: Ovo uključuje ručno i automatsko testiranje novih zadataka. Ponekad, tim testira aplikaciju kroz ceo razvojni ciklus kako bi otkrio i ispravio sve eventualne probleme.
  - **Faza razmene (Deployment)**: Tim distribuira nove izmene u okruženju za izgradnju ili testiranje.
  - **Faza održavanja**: Ovo uključuje popravku bilo kakvih problema koji se pojave od strane klijenata u proizvodnoj aplikaciji.

- **Različiti primeri životnog ciklusa razvoja softvera**:
  - **Primer Waterfall**: Waterfall je model u kojem mora biti završena svaka faza ciklusa pre nego što počne sledeća faza.
  - **Primer Agile**: Agile se fokusira na iterativni razvoj podeljavanjem posla u sprintove.

## BDD na TDD

- **TDD**: Test-driven development je metodologija koja naglašava pisanje testova prvo. Pisanje testova pre izgradnje komponenti pruža trenutnu povratnu informaciju programerima tokom procesa razvoja.
- **BDD**: Behavior-driven development je pristup koji povezuje niz testova sa poslovnim ciljevima. Testovi u BDD-u treba da budu napisani jezikom razumljivim tehničkim i ne-tehničkim osobama. Primer te sintakse je Gherkin.
- **BDD Test Frameworks**: Primjeri BDD test frameworkova uključuju Cucumber, JBehave i SpecFlow.

## Asertacije (Assertions) u jediničnim testovima

- **Autentifikacija**: Autentifikacija se koristi za testiranje da li kod radi kako je očekivano.
- **Biblioteke za autentifikaciju**: Chai je popularna biblioteka za autentifikaciju. Ostale biblioteke za autentifikaciju za JavaScript su `should.js` i `expect.js`.

Ovo je primer asertacije korišćenjem Chai-a koji proverava da li je vrednost return iz funkcije `addThreeAndFour` jednaka broju 7:

```js
assert.equal(addThreeAndFour(), 7);
```

- **Najbolja praksa**: Bez obzira na biblioteku za autentifikaciju koju koristite, trebalo bi da napišete jasne poruke o uspehu i neuspehu koje pomažu u razumevanju koji testovi ne prolaze i zašto.

## Mocking, Faking, i Stubbing

- **Mocking**: Mockovanje je proces zamene stvarnih podataka sa lažnim podacima koji imitiraju ponašanje pravih komponenti. Na primer, možete *mockovati* odgovor API u testovima umesto da često šaljete zahtev za preuzimanje podataka putem API.
- **Stubbing**: Stubovanje: Stubi su entiteti koji vraćaju unapred definisane odgovore ili uzorak podataka za očekivano ponašanje u aplikaciji. Na primer, možete *stubovati* ponašanje konekcije sa bazom podataka u vašim testovima bez oslanjanja na stvarnu bazu podataka.
- **Faking**: Fejkovi su lagani izdanci stvarnih komponenti bez njihove složenosti ili nuspojava. Na primer, možete *fejkovati* bazu podataka skladištima podataka u memoriji umesto interakcije sa pravom bazom podataka. Ovo će vam omogućiti da simulirate operacije baze podataka u memoriji, što je brže od rukovanja stvarnom bazom podataka.

## Funkcionalno testiranje (Functional Testing)

- **Functional Testing**: Funkcionalno testiranje proverava da li programski elementi i funkcije rade kako je očekivano. Cilj funkcionalnog testiranja je testiranje celog sistema protiv mnogo scenarija.
- **Non-Functional Testing**: Nefunkcionalno testiranje se bavi aspektima kao što su performanse i pouzdanost.
- **Smoke testing**: Smoke testiranje je početna provera sistema na osnovne ili kritične probleme pre nego što počnu detaljniji testovi.

## Testiranje od kraja do kraja (End-to-End Testing)

- **End-to-End Testing**: End-to-end testing, ili E2E, testira stvarne scenarije iz perspektive korisnika. Testiranje od kraja do kraja pomaže da se osigura da vaša aplikacija radi kako treba i da je predvidiva za korisnike. Međutim, zahteva mnogo vremena za podešavanje, dizajniranje i održavanje.
- **Okviri za testiranje od kraja do kraja**: Playwright je popularan okvir za testiranje od kraja do kraja razvijen od strane Microsofta. Drugi primeri alata za testiranje od kraja do kraja uključuju Cypress, Selenium i Puppeteer.

## Testiranje upotrebljivosti (Usability Testing)

- **Usability Testing**: Usability testing je proces davanja stvarnih korisnika da koriste aplikaciju kako bi se otkrili problemi dizajna, korisničkog iskustva ili funkcionalnosti u samoj aplikaciji. Usability testing se fokusira na to koliko lako korisnici mogu koristiti aplikaciju.
- **Četiri uobičajene vrste usability testinga**:
  - **Explorative**: Explorative usability testing uključuje interakciju korisnika sa različitim funkcijama aplikacije kako bi se razumeo način njihovog rada.
  - **Comparative**: Comparative testing je proces poređivanja korisničkog iskustva vaše aplikacije sa drugim sličnim aplikacijama na tržištu.
  - **Assessment**: Assessment testing je testiranje kojim istražujete koliko je aplikacija laka za korišćenje.
  - **Validation**: Validation testing je identifikovanje ozbiljnih problema koji bi mogli sprečiti korisnika da pravilno koristi aplikaciju.
- **Alati za usability testing**: Primeri alata za usability testing su Loop11, Maze, Userbrain, UserTesting i UXTweak.

## Testiranje kompatibilnosti (Compatibility Testing)

- **Compatibility Testing**: Cilj kompatibilnost testiranja je osigurati da vaša aplikacija radi u različitim računarskim okruženjima.
- **Različite vrste kompatibilnost testiranja**:
  - **Backwards Compatibility**: Backwards kompatibilnost je kada aplikacija radi sa ranijim verzijama.
  - **Forwards Compatibility**: Forwards kompatibilnost je kada aplikacija i sistemi rade sa budućim verzijama.
  - **Hardware Compatibility**: Hardware kompatibilnost je sposobnost aplikacije da radi ispravno na različitim konfiguracijama hardvera.
  - **Operating Systems Compatibility**: Operating system kompatibilnost je sposobnost aplikacije da radi na različitim operativnim sistemima, kao što su macOS, Windows i Linux kao Ubuntu i Fedora.
  - **Network Compatibility**: Mrežna kompatibilnost znači da aplikacija može raditi u različitim mrežnim uslovima, kao što su različite brzine mreže, protokoli, sigurnosne podešavanja itd.
  - **Browser Compatibility**: Pretraživačka kompatibilnost znači da web aplikacija može raditi stabilno u različitim pretraživačima, kao što su Google Chrome, Safari, Firefox itd.
  - **Mobile Compatibility**: Važno je osigurati da vaše aplikacije rade na raznim uređajima Androida i iOS-a, uključujući telefone i tablete.

## Testiranje performansi (Performance Testing)

- **Performance Testing**: U performans testiranju, testiraš brzinu, responzivnost, skalabilnost i stabilnost aplikacije pod različitim opterećenjima rada. Cilj je identifikovati sve vrste problema sa performansama.
- **Različite vrste performance testinga**:
  - **Load Testing**: Load testing određuje kako sistem funkcioniše tokom normalnih i visokih opterećenja.
  - **Stress Testing**: Stress testing je kada testiraš svoju aplikaciju pod ekstremnim opterećenjima i vidiš kako tvoj sistem upravlja tim opterećenjem.
  - **Soak Testing (Endurance Testing)**: Soak testing ili endurance testing je vrsta load testiranja gde testiraš sistem sa velikim opterećenjem tokom dužeg perioda.
  - **Spike Testing**: Spike testing je kada naglo povećavaš i smanjuješ opterećenje i analiziraš kako sistem reaguje na te promene.
  - **Breakpoint Testing (Capacity Testing)**: Breakpoint testing ili capacity testing je kada postepeno povećavaš opterećenje sve dok sistem ne počne da otkazuje ili mu performanse padnu.

## Testiranje sigurnosti (Security Testing)

- **Testiranje sigurnosti (Security Testing)**: Testiranje sigurnosti pomaže u identifikaciji slabosti i nedostataka.
- **Pravila sigurnosti (Security Principles)**:
  - **Konfidencijalnost (Confidentiality)**: Štiti od otkrivanja osetljivih informacija neovlašćenim primaocima.
  - **Integritet (Integrity)**: Uključuje sprečavanje da loših korisnika izmene podatke korisnika.
  - **Autentifikacija (Authentication)**: Uključuje proveru identiteta korisnika kako bi se osiguralo da mu je dozvoljeno korišćenje sistema.
  - **Autorizacija (Authorization)**: To je proces određivanja koje akcije su ovlašćeni korisnici dozvoljeni da izvrše ili koji delovi sistema im je dozvoljeno pristupiti.
  - **Dostupnost (Availability)**: Osigurava da su informacije i usluge dostupne ovlašćenim korisnicima kada im zatrebaju.
  - **Nepokretnost/Negacija (Non-Repudiation)**: Osigurava da i pošiljalac i primaoc imaju dokaz o slanju i verifikaciji identiteta pošiljaoca. Štiti od toga da pošiljalac negira slanje informacija.
- **Uobičajene sigurnosne pretnje**:
  - **Cross-Site Scripting (XSS)**: XSS napadi nastaju kada haker ubaci zlonamerni skriptovi na web stranicu i izvrši ih u pregledaču žrtve.
  - **SQL Injection**: SQL injection omogućava lošim korisnicima da ubace zlonamerni kod u bazu podataka.
  - **Denial-of-Service (DoS) Attack**: DoS napad se dešava kada loši korisnici preplave veb sa velikim brojem zahteva ili saobraćajem, što uzrokuje usporavanje ili čak pad servera, čineći sajt nedostupnim za korisnike.
- **Kategorije alata za testiranje sigurnosti**:
  - **Static Application Security Testing (SAST)**: Ovi alati pregledaju izvorni kod programa kako bi identifikovali sigurnosne slabosti.
  - **Dynamic Application Security Testing (DAST)**: Ovi alati interaguju sa prednjim delom programa kako bi otkrili sigurnosne propuste. DAST nema pristup izvornom kodu.
- **Penetration Testing (pentest)**: Penetracioni testiranje je vrsta sigurnosnog testiranja koja uključuje simulaciju kibernetičkih napada na aplikaciju radi identifikacije bilo kakvih slabosti u sistemu.

## A/B testiranje

- **A/B Testiranje**: A/B testiranje podrazumeva poređenje dve verzije stranice ili aplikacije i ispitivanje koja od verzija radi bolje. Takođe je poznato kao *bucket* ili *split testing*. A/B testiranje vam omogućava donošenje odluka vođenih podacima i stalno poboljšanje korisničkog iskustva.
- **Alati za A/B Testiranje**: Primeri alata za A/B testiranje su GrowthBook i LaunchDarkly.

## Testovi za Alfa i Beta

Nakon završetka početnog razvoja i testiranja programa, važno je da program prođe kroz testiranje stvarnih korisnika. Ovde dolaze alfa i beta testiranja.

- **Alpha Testing**: Alpha testing se sprovodi od strane specifične grupe testera koja prolazi kroz program kako bi osigurala da nema grešaka pre nego što bude izbačen na tržište. Alpha testing je deo acceptance testing-a i koristi metode white i black box testing-a.
- **Beta Testing**: Beta testing se dešava kada je program pušten realnim korisnicima. Korisnici mogu da interaguju sa programom i pruže povratne informacije. Beta testing je takođe vrsta user acceptance testing-a.
- **Acceptance Testing**: Acceptance testing osigurava da program ispunjava poslovne potrebe i zahteve korisnika pre nego što bude pušten.
- **Black Box Testing**: Black box testing se fokusira samo na očekivano ponašanje programa.
- **White Box Testing**: White box testing uključuje testera koji zna unutrašnje komponente i vrši testiranje na njima.

## Regresiona analiza

- **Regresija**: Regresija se odnosi na situaciju kada nove promene slučajno pokvare postojeće funkcionalnosti.
- **Regresiono testiranje**: Regresiono testiranje pomaže u otkrivanju problema regresije. U regresionom testiranju, ponavljate testove performansi na delovima vaše aplikacije kako biste osigurali da sve i dalje radi onako kako je očekivano.
- **Alati za regresiono testiranje**: Alati koje možete koristiti za regresiono testiranje su Puppeteer, Playwright, Selenium i Cypress.
- **Metode regresionog testiranja**:
  - **Unit regresiono testiranje**: Ovo je kada imate spisak stvari koje moraju biti testirane svaki put kada se vrše velike promene ili popravke na programu.
  - **Parcijalno regresiono testiranje**: Ovo uključuje ciljane metode kako bi se osiguralo da nove promene nisu pokvarile specifične funkcije programa.
  - **Kompleтно regresiono testiranje**: Ovo pokreće testove na svim funkcijama u kodu. Ovo je opcija koja zahteva mnogo vremena i najdetaljnija je.
- **Retestiranje**: Retestiranje se koristi za proveru poznatih problema i osiguranje da su rešeni. Za razliku od regresionog testiranja, retestiranje traži poznate probleme, dok regresiono testiranje traži nove probleme koji su nastali usled nedavnih promena u kodu.

## TypeScript šta je to?

- **JavaScript**: JavaScript je jezik sa promenljivim tipovima podataka. To znači da parametri mogu primiti bilo koju vrednost tokom izvršavanja. Izazov jezika sa promenljivim tipovima jeste što nedostatak sigurnosti tipa može dovesti do grešaka.

Na primer, čak i ako vaša funkcija za JavaScript očekuje red podataka, i dalje ga možete pozvati sa brojem:

```javascript
const getRandomValue = (array) => {
  return array[Math.floor(Math.random() * array.length)];
}

console.log(getRandomValue(10));
```

`console` onaj primer iznad će biti `undefined`.

- **TypeScript**: TypeScript proširuje jezik JavaScript dodavanjem tipova podataka, što pomaže u otkrivanju grešaka uzrokovanih neusklađenim tipovima pre pokretanja vašeg koda.

Na primer, možete definisati tip parametra za `array` kao što sledi:

```typescript
const getRandomValue = (array: string[]) => {
  return array[Math.floor(Math.random() * array.length)];
}
```

Ova definicija tipa govori TypeScript da parametar za `array` mora biti niz znakova. Zatim, kada pozovete `getRandomValue` i date mu broj, dobićete grešku nazvanu greška kompajlera.

- **Kompajler**: Prvo morate prevesti kod od TypeScript u standardni format JavaScript. Kada pokrenete kompajler, TypeScript će evaluirati vaš kod i izdati grešku za bilo kakve probleme gde tipovi ne odgovaraju.

## Tip podataka u TypeScript

- **Osnovni tipovi podataka u TypeScript**: Za osnovne tipove podataka `string`, `null`, `undefined`, `number`, `boolean`, i `bigint`, TypeScript daje ključne reči tipova.

```typescript
let str: string = "Naomi";
let num: number = 42;
let bool: boolean = true;
let nope: null = null;
let nada: undefined = undefined;
```

- **Redovi podataka**: Moguće je definisati redicu specifičnog tipa sa dve različite sintakse.

```typescript
const arrOne: string[] = ["Naomi"];
const arrTwo: Array<string> = ["Camperchan"];
```

- **Vitu**: Možete definisati kompletnu strukturu objekta.

```typescript
const obj: { a: string, b: number } = { a: "Naomi", b: 42 };
```

Ako želiš nešto sa bilo kojim ključevima, ali sve vrednosti moraju biti sekvenca slova, postoje dva načina definisanja:

```typescript
const objOne: Record<string, string> = {};
const objTwo: { [key: string]: string } = {};
```

- **Još jedan važan tip u TypeScript**:
  - **`any`**: ``any`` ukazuje da vrednost može biti bilo kog tipa. Govori kompajleru da prestane da brine o tipu tog parametra.
  - **`unknown`**: ``unknown`` govori TypeScript da mu je važan tip vrednosti, ali ne zna koji. ``unknown`` je preferiraniji od ``any``.
  - **`void`**: Ovo je poseban tip koji ćete koristiti samo kada definišete funkciju. Funkcije bez vrednosti return koriste tip return od `void`.
  - **`never`**: Predstavlja tip koji nikada neće postojati.

- **Ključna reč za `type`**: Ova reč je kao `const`, ali umesto deklarisanja parametra, možete deklarisati tip.

Pogodno je za deklarisanje specifičnih tipova, kao što su union tipi ili tipovi koji sadrže samo određene vrednosti:

```typescript
type stringOrNumber = string | number;
type bot = "camperchan" | "camperbot" | "naomi";
```

- **`interface`**: Interfejzi su kao klase za tipove. Mogu da implementiraju ili prošire druge interfejse, specifični su za tipove objekata, i obično se preporučuju osim ako vam nije potrebna posebna funkcionalnost koju pruža deklaracija `type`.

```typescript
interface wowie {
  zowie: boolean;
  method: () => void;
}
```

- **Definisanje tipa return**: Takođe možete definisati _tip povratne vrednosti_ funkcije.

Primer ispod definiše vrednost return kao niz znakova. Ako pokušate da vratite nešto drugo, TypeScript će proizvesti grešku kompajlera.

```typescript
const getRandomValue = (array: string[]): string => {
  return array[Math.floor(Math.random() * array.length)];
}
```

## Generici

- **Definisati generički tip**: Možete definisati generički tip i navesti ga u vašoj funkciji. To se može smatrati kao poseban parametar koji pružate funkciji kako biste kontrolisali ponašanje definicije tipa funkcije.

Evo primera definisanja generičkog tipa za funkciju:

```typescript
const getRandomValue = <T>(array: T[]): T => {
  return array[Math.floor(Math.random() * array.length)];
}
const val = getRandomValue([1, 2, 4])
```

Sintaksa za `<T>` govori TypeScript da definišete generički tip `T` za funkciju. `T` je uobičajeno ime za generički tip, ali možete koristiti bilo koje ime.

Zatim, kažeš `TypeScript` da parametar za ``array`` je niz vrednosti koje odgovaraju generičkom tipu, i da vraćena vrednost je jedan element istog tipa.

- **Definisanje tipa argumenta u pozivu funkcije**: Možete proslediti tip argument u pozivu funkcije koristeći uglaste zagrade između imena funkcije i njenih parametara.

Evo primera prosljeđivanja argumenta tipa u pozivu funkcije:

```typescript
const email = document.querySelector<HTMLInputElement>("#email");
console.log(email.value);
```

Ovo obaveštava TypeScript da će funkcija koju očekujete biti polje za unos.

## Sužavanje tipa

- **Sužavanje po istinitosti (Narrowing by Truthiness)**: U ovom primeru, dobićete grešku kompajlera kada pokušavate da dobijete svojstvo `value` od `email` jer `email` _može_ biti `null`.

```typescript
const email = document.querySelector<HTMLInputElement>("#email");
console.log(email.value);
```

Možete koristiti izjavu uslova da proverite da li je `email` _kweli_ pre dobijanja funkcije:

```typescript
const email = document.querySelector<HTMLInputElement>("#email");
if (email) {
  console.log(email.value);
}
```

Provera činjenica takođe može da funkcioniše na različite načine:

```typescript
const email = document.querySelector<HTMLInputElement>("#email");
if (!email) {
  throw new ReferenceError("Could not find email element!")
}
console.log(email.value);
```

Bacanjem greške, završava se izvršavanje logike ovog koda, pa kada dođe do poziva `console.log()`, TypeScript zna da `email` _ne može_ biti `null`.

- **Optional Chaining**: Optional chaining `?.` je takođe vrsta type narrowing-a, na istom principu da pristup svojstvu ne može doći ako je vrednost `email` jednaka `null`.

```typescript
const email = document.querySelector<HTMLInputElement>("#email");
console.log(email?.value);
```

- **Opereta ya `typeof`**: Možete koristiti uslove za proveru _tipa parametra_ koristeći operetu `typeof`.

```typescript
const myVal = Math.random() > 0.5 ? 222 : "222";
if (typeof myVal === "number") {
  console.log(myVal / 10);
}
```

- **Ključna reč za `instanceof`**: Ako nešto dolazi iz klase, možete koristiti ključnu reč `instanceof` da specifikujete tip.

```typescript
const email = document.querySelector("#email");

if (email instanceof HTMLInputElement) {
    console.log(email.value);
}
```

- **Casting**: Kada `TypeScript` ne može direktno da utvrdi tip vrednosti, kao rezultat iz metode ``request.json()`` u primeru ispod, dobićete grešku kompajlera. Jedan način za rešavanje ovoga je izvršiti *type casting*, ali to smanjuje sposobnost `TypeScript` da detektuje potencijalne greške.

```typescript
interface User {
    name: string;
    age: number;
}

const printAge = (user: User) => 
  console.log(`${user.name} is ${user.age} years old!`)

const request = await fetch("url")
const myUser = await request.json() as User;
printAge(myUser);
```

- **Type Guard**: Mehanizam za tipovanje, možete napisati type guard:

```typescript
interface User {
    name: string;
    age: number;
}

const isValidUser = (user: unknown): user is User => {
  return !!user && 
    typeof user === "object" &&
    "name" in user &&
    "age" in user;
}
```

Sintaksa za `user is User` pokazuje da tvoja funkcija vraća boolean vrednost (tačno ili netačno), koja, ako je tačna, znači da vrednost `user` zadovoljava interfejs `User`.

## Fail `tsconfig`

- **`tsconfig.json`**: Podešavanja kompajlera za TypeScript nalaze se u fajlu `tsconfig.json` na korenu vašeg projekta.

```json
{
  "compilerOptions": {
    "rootDir": "./src",
    "outDir": "./prod",
    "lib": ["ES2023"],
    "target": "ES2023",
    "module": "ES2022",
    "moduleResolution": "Node",
    "esModuleInterop": true,
    "skipLibCheck": true,
    "strict": true
  },
  "exclude": ["test/"]
}
```

Ovo su objašnjenja opcija kompajlera koje su korišćene u navedenom primeru:

- **`compilerOptions`**: Svojstvo `compilerOptions` koristi se za kontrolu načina na koji TypeScript kompajler radi.
- **`rootDir` i `outDir`**: `rootDir` i `outDir` govore TypeScript-u koja fascikla sadrži izvorne datoteke i u koju fasciklu treba smestiti prevedeni JavaScript kod.
- **`lib`**: Svojstvo `lib` određuje koje definicije tipova će kompajler koristiti i omogućava uključivanje podrške za određene verzije ECMAScript-a, DOM-a i druge biblioteke.
- **`module` i `moduleResolution`**: `module` i `moduleResolution` zajedno određuju kako projekat koristi module, bilo CommonJS ili ECMAScript module.
- **`esModuleInterop`**: `esModuleInterop` omogućava bolju kompatibilnost između CommonJS i ES modula automatskim kreiranjem namespace objekata za importe.
- **`skipLibCheck`**: Opcija `skipLibCheck` omogućava preskakanje provere `.d.ts` datoteka koje nisu referencirane putem importa u tvom kodu.
- **`strict`**: Opcija `strict` uključuje više provera, kao što su pravilno rukovanje vrednostima koje mogu biti `null` ili `undefined`, i upozorava kada TypeScript mora da koristi tip `any`.
- **`exclude`**: Svojstvo `exclude` na najvišem nivou govori kompajleru da ignoriše navedene TypeScript datoteke prilikom kompajliranja.
