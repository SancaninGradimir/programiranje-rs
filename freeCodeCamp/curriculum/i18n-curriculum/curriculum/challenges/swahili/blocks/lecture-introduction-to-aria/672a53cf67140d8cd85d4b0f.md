# --interactive--

Činiti statični sadržaj lako dostupnim može biti jednostavno, ali često menjajući sadržaj može predstavljati veći izazov. Ovde ulazi WAI-ARIA.

Istražimo šta je WAI-ARIA, njegova svrha, kako funkcioniše, i nekoliko primera.

WAI-ARIA predstavlja Web Accessibility Initiative - Accessible Rich Internet Applications. To su uputstva koja povećavaju pristupačnost dinamičnog sadržaja i elemenata korisničkog interfejsa (UI).

Zapamtite da WCAG i WAI-ARIA nisu isto. WCAG pruža opšte smernice za pristupačnost veb sajtova, dok WAI-ARIA pruža specifična pravila za omogućavanje da su dinamični i interaktivni sadržaji dostupni korisnicima asistivne tehnologije.

Glavna svrha WAI-ARIA je da poboljša pristupačnost dinamičkog sadržaja i delova korisničkog interfejsa koji nemaju odgovarajuće izvorne HTML elemente.

WAI-ARIA radi tako što uspostavlja skup svojstava elemenata koja možete dodati elementima HTML kako bi pružila više semantičke informacije. Ova svojstva elemenata su podeljena na uloge, stanja i atribute.

Uloga ARIA definiše svrhu elementa unutar sajta ili web aplikacije. Evo primera kako postaviti ulogu na `button` za element `div`.

```html
<div role="button">Click Me</div>
```

Kada ovo uradiš, pokazuješ asistencijskoj tehnologiji da je element dugme. Međutim, atributi ne pružaju nikakvu akciju. Pružanje `div` ovo `role` od `button` neće ga učiniti da izgleda ili radi kao dugme. Da bi izgledalo i radilo kao dugme, moraš da koristiš CSS i JavaScript kako bi postigao željeni rezultat.

Evo primera korišćenja HTML-a, CSS-a i JavaScript-a za kreiranje prilagođenog `button` elementa.

**NAPOMENA**: Ne brinite zbog razumevanja koda CSS i JavaScript. Naučićete te jezike u budućim modulima.

:::interactive_editor

```html
<link href="styles.css" rel="stylesheet">
<div id="custom-btn" role="button">Click Me</div>
<script src="index.js"></script>
```

```css
#custom-btn {
  display: inline-block;
  padding: 0.4em 1em;
  font-size: 1rem;
  font-family: sans-serif;
  color: buttontext;
  background-color: buttonface;
  border: 1px solid #ccc;
  border-radius: 4px;
  cursor: pointer;
  user-select: none;
  text-align: center;
}

#custom-btn:focus {
  outline: 2px solid Highlight;
  outline-offset: 2px;
}

#custom-btn:active {
  background-color: #ddd;
}
```

```js
const button = document.getElementById("custom-btn");

button.addEventListener("click", () => {
  alert("Button clicked!");
});

button.addEventListener("keydown", (e) => {
  if (e.key === "Enter" || e.key === " ") {
    e.preventDefault(); 
    button.click();
  }
});
```

:::

Uvek je bolje koristiti ugrađenu funkciju `button` ili `input` umesto `type="button"`.

Atributi ARIA pružaju dodatne informacije o elementima. Na primer, atribut ``aria-labelledby`` vam omogućava da povežete element sa specifičnom oznakom:

```html
<h2 id="header-id">About freeCodeCamp</h2>
<button id="button-id" aria-labelledby="header-id button-id">Learn More</button>
```

Ovo će učiniti funkcije razumljivim i dostupnim za korisnike asistivne tehnologije.

Da biste dobili najbolju korist od WAI-ARIA, pokušajte da koristite HTML nativno kada je to moguće jer obično pruža bolji direktan pristup.

Koristi WAI-ARIA samo kada HTML ne radi, i nemoj zaboraviti da testiraš sa asistivnim tehnologijama kao što su čitači ekrana, ili da pustiš osobe sa invaliditetom da testiraju svoj rad. Takođe, osiguraj da su tvoji WAI-ARIA stanja i atributi ažurirani zajedno sa sadržajem u realnom vremenu. Izbegavaj prekomerno korišćenje ARIA-a, jer to često može zbuniti.
