# --description--

U Python, izjava ``raise`` je moćno sredstvo koje vam omogućava ručno izazivanje grešaka u vašem kodu. Daje vam kontrolu nad time kada i kako greške nastaju, omogućavajući vam da kreirate specifične scenarije grešaka i implementirate određeno ponašanje programa.

Izjava za `raise` se koristi za bacanje eksplicitnih grešaka bilo gde u vašoj aplikaciji, omogućavajući vam da naznačite da je došlo do greške ili da nisu ispunjeni određeni zahtevi.

Izjava za `raise` od Python može se koristiti na nekoliko načina da izazove grešku. Osnovno, možete pokrenuti ugrađene izuzetke ili kreirati prilagođenu poruku o grešci. Evo jednostavnog primera:

```py
def check_age(age):
    if age < 0:
        raise ValueError('Age cannot be negative')
    return age

try:
    check_age(-5)
except ValueError as e:
    print(f'Error: {e}') # Error: Age cannot be negative
```

Možete videti ovde da `raise` je glavna reč koja uzrokuje grešku.

U ovom primeru, generišemo `ValueError` i specifičnu poruku kada se unosi nevažeća starost.

Izjava za `raise` se takođe može koristiti za ponovno podizanje trenutne greške, što je posebno važno pri rukovanju greškama:

```py
def process_data(data):
    try:
        result = int(data)
        return result * 2
    except ValueError:
        print('Logging: Invalid data received')
        raise  # Re-raises the same ValueError

try:
    process_data('abc')
except ValueError:
    print('Handled at higher level')
```

Ovde ključna reč `raise` (bez argumenata), ponovo uzrokuje trenutnu grešku koja se obrađuje.

Ovo vam omogućava da snimite ili izvršite čišćenje dok i dalje distribuira greške na stack poziva.

Možeš da kreiraš i podigneš prilagođene greške definisanjem sopstvenih klasa grešaka:

```py
class InsufficientFundsError(Exception):
    def __init__(self, balance, amount):
        self.balance = balance
        self.amount = amount
        super().__init__(f'Insufficient funds: ${balance} available, ${amount} requested')

def withdraw(balance, amount):
    if amount > balance:
        raise InsufficientFundsError(balance, amount)
    return balance - amount

try:
    new_balance = withdraw(100, 150)
except InsufficientFundsError as e:
    print(f'Transaction failed: {e}')
```

Ovde možeš videti da prilagođene klase grešaka nasleđuju `Exception` ili neku od njegovih podklasa.

Više o klasama i nasleđivanju naučićeš u narednim lekcijama. Za sada, zapamti da je ovo način za kreiranje sopstvenih grešaka sa posebnom logikom.

Izraz `raise` se takođe može koristiti zajedno sa glavnim terminom `from` za povezivanje grešaka, pokazujući vezu između različitih grešaka:

```py
def parse_config(filename):
    try:
        with open(filename, 'r') as file:
            data = file.read()
            return int(data)
    except FileNotFoundError:
        raise ValueError('Configuration file is missing') from None
    except ValueError as e:
        raise ValueError('Invalid configuration format') from e

config = parse_config('config.txt')
```

Ovde možete videti da `raise ... from None`, sprečava kontekst prethodne greške:

```bash
Traceback (most recent call last):
  File "main.py", line 12, in <module>
    config = parse_config('config.txt')
             ^^^^^^^^^^^^^^^^^^^^^^^^^^
  File "main.py", line 7, in parse_config
    raise ValueError('Configuration file is missing') from None
ValueError: Configuration file is missing
```

Na `raise ... from e`, povezuje novu grešku sa prethodnom, čuvajući niz grešaka.

```bash
Traceback (most recent call last):
  File "main.py", line 5, in parse_config
    return int(data)
           ^^^^^^^^^
ValueError: invalid literal for int() with base 10: ''

The above exception was the direct cause of the following exception:

Traceback (most recent call last):
  File "main.py", line 12, in <module>
    config = parse_config('config.txt')
             ^^^^^^^^^^^^^^^^^^^^^^^^^^
  File "main.py", line 9, in parse_config
    raise ValueError('Invalid configuration format') from e
ValueError: Invalid configuration format
```

Možete takođe uslovno baciti izuzetak koristeći deklaracije `assert`, koje su u osnovi kraći načini za `raise` i `AssertionError`:

```py
def calculate_square_root(number):
    assert number >= 0, 'Cannot calculate square root of negative number'
    return number ** 0.5

try:
    result = calculate_square_root(-4)
except AssertionError as e:
    print(f'Assertion failed: {e}')
```

Korišćenje ``raise`` je važno za kreiranje robusnih aplikacija gde morate sprovesti poslovna pravila, validirati ulaz i pružiti smislene poruke o greškama. Korišćenjem ``raise`` za strategiju, možete učiniti svoj kod pouzdanijim i lakšim za otklanjanje grešaka (debugging), istovremeno pružajući korisnicima jasnu povratnu informaciju o tome šta je pošlo naopako.
