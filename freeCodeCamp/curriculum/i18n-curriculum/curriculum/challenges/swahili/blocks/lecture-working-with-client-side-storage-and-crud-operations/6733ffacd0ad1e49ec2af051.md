# --description--

Kao što ste naučili u prethodnim lekcijama, kolačići su jednostavni podaci koje veb-sajtovi mogu čuvati na uređaju korisnika.

Međutim, možete čuvati složenije strukture podataka u kolačićima. Jedan uobičajen način je korišćenje JSON-a za čuvanje objekata ili nizova podataka. Evo primera:

```js
const userData = {
  name: "John Doe",
  age: 30,
  role: "admin"
};

document.cookie = "userInfo=" + JSON.stringify(userData) + "; path=/";
```

U ovom primeru kreiramo objekat sa korisničkim podacima, pretvaramo ga u JSON string, a zatim ga čuvamo u kolačiću. Kada želimo da preuzmemo te podatke, možemo parsirati JSON string nazad u objekat pomoću metode `JSON.parse()`.

Sada se možda pitate kakvu ulogu imaju HTTP zaglavlja u ovoj temi. Kolačiće server najčešće postavlja upravo pomoću HTTP zaglavlja.

Na primer, server može da pošalje sledeće zaglavlje:

```http
Set-Cookie: username=John Doe; expires=Thu, 31 Dec 2024 6:00:00 IST; path=/
```

Ovo zaglavlje govori pregledaču da postavi kolačić.

Kolačiće možemo postavljati i direktno u pregledaču pomoću JavaScript-a. Ovo je korisno za čuvanje podataka koje nije potrebno odmah slati serveru.

Imajte na umu da kolačići imaju ograničenje veličine od približno 4 KB, pa čuvanje previše podataka u njima može usporiti rad vaše veb aplikacije.

Čuvanje velike količine podataka u kolačićima može povećati mrežni saobraćaj, jer se kolačići šalju uz svaki HTTP zahtev.
