# --description--

`Versions` npm paketi u sekciji zavisnosti vašeg fajla package.json prate ono što se zove semantičko verzijsko označavanje (Semantic Versioning, SemVer), industrijski standard za izdavanje verzija softvera koji ima za cilj pojednostavljenje upravljanja zavisnostima. Biblioteke, okviri ili drugi alati objavljeni na npm-u treba da koriste SemVer kako bi jasno komunicirali vrstu promena koje projekti mogu očekivati ako ažuriraju.

Znati šta je SemVer može biti korisno kada razvijate aplikaciju koja koristi eksterne zavisnosti (što gotovo uvek radite). Jednog dana, vaše razumevanje ovih brojeva će vas spasiti od slučajnog unosenja promena koje pokreću pad vašeg projekta bez da razumete zašto stvari koje su juče radile iznenada danas ne rade. Tako funkcioniše semantičko verziranje prema zvaničnom sajtu:

```json
"package": "MAJOR.MINOR.PATCH"
```

Major verzija treba da se poveća kada napravite nekompatibilne promene u API. Minor verzija treba da se poveća kada dodate funkcionalnost koja je kompatibilna sa prethodnim verzijama. Patch verzija treba da se poveća kada napravite popravke grešaka koje su kompatibilne sa prethodnim verzijama. To znači da su Patch verzije ispravke grešaka, a Minor verzije dodaju nove funkcije, ali nijedna od njih ne pokvari ono što je prethodno radilo. Konačno, Major verzije dodaju promene koje neće biti kompatibilne sa prethodnim verzijama.

# --instructions--

U sekciji zavisnosti vašeg fajla ``package.json``, promenite verziju ``@freecodecamp/example`` tako da odgovara GLAVNA 1, SPORREDNA 2 i POPRAVKA 13

# --hints--

`"dependencies"` mora da sadrži `"@freecodecamp/example"`.

```js
fetch(code + '/_api/package.json')
  .then(response => response.json())
  .then(
    data => {
      assert.property(
        data.dependencies,
        '@freecodecamp/example',
        '"dependencies" does not include "@freecodecamp/example"'
      );
    },
    error => {
      throw new Error(error.message || error.responseText);
    }
  );
```

Verzija od `"@freecodecamp/example"` bi trebalo da bude `"1.2.13"`.

```js
fetch(code + '/_api/package.json')
  .then(response => response.json())
  .then(
    data => {
      assert.equal(
        data.dependencies['@freecodecamp/example'],
        '1.2.13',
        'Wrong version of "@freecodecamp/example". It should be 1.2.13'
      );
    },
    error => {
      throw new Error(error.message || error.responseText);
    }
  );
```
