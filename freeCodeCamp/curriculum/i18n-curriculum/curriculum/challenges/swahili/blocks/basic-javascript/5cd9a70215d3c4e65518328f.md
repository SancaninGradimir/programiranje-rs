# --description--

U <a href="/learn/javascript-algorithms-and-data-structures/basic-javascript/replace-loops-using-recursion" target="_blank" rel="noopener noreferrer nofollow">prethodnom izazovu</a> naucili ste kako da rekurzijom zamenite `for` petlju. Sada cemo pogledati slozeniju funkciju koja vraca niz celih brojeva od `1` do broja prosledjenog funkciji.

Kao sto je pomenuto u prethodnom izazovu, postojace <dfn>osnovni slucaj</dfn>. Osnovni slucaj govori rekurzivnoj funkciji kada treba da prestane da poziva samu sebe. To je jednostavan slucaj u kom je povratna vrednost vec poznata. Takodje postoji <dfn>rekurzivni poziv</dfn> koji poziva originalnu funkciju sa drugacijim argumentom. Ako je funkcija pravilno napisana, na kraju ce se stici do osnovnog slucaja.

Na primer, recimo da zelite da napisete rekurzivnu funkciju koja vraca niz brojeva od `1` do `n`. Ova funkcija treba da primi argument `n`, koji predstavlja krajnji broj. Zatim treba da poziva samu sebe sa manjim vrednostima od `n` dok ne dodje do `1`. Funkcija moze izgledati ovako:

```javascript
function countup(n) {
  if (n < 1) {
    return [];
  } else {
    const countArray = countup(n - 1);
    countArray.push(n);
    return countArray;
  }
}
console.log(countup(5));
```

Vrednost `[1, 2, 3, 4, 5]` ce biti prikazana u konzoli.

Na prvi pogled ovo moze biti zbunjujuce jer vrednost `n` *opada*, dok vrednosti u zavrsnom nizu *rastu*. To je zato sto se `push` izvrsava na kraju, nakon sto rekurzivni poziv vrati vrednost. Kada se `n` ubaci u niz, `countup(n - 1)` je vec evaluiran i vratio je `[1, 2, ..., n - 1]`.

# --instructions--

Deklarisana je funkcija `countdown` sa jednim parametrom (`n`). Funkcija treba da koristi rekurziju kako bi vratila niz celih brojeva od `n` do `1`, na osnovu parametra `n`. Ako se funkcija pozove sa brojem manjim od 1, treba da vrati prazan niz. Na primer, poziv sa `n = 5` treba da vrati niz `[5, 4, 3, 2, 1]`. Vasa funkcija mora koristiti rekurziju (pozivati samu sebe) i ne sme koristiti nijednu vrstu petlje.
