# --interactive--

Naučite se o nameni puta `preventDefault()` u događajima.

Svaki događaj koji se dešava u DOM ima određeno podrazumevano ponašanje. Događaj `click` u padajućem izborniku menja stanje tog elementa, po podrazumevanom. Pritiskanje tastera za razmak (space bar) na fokusiranom elementu aktivira taj element. Metoda `preventDefault()` na ovim elementima događaja sprečava da se to ponašanje desi.

Uzmimo primer. Objasnićmo element od `input` za korisnika da unese unutra:

```html
</label>
```

Zatim, moramo uhvatiti događaj `keydown` ili pratiti karakter unet na tastaturi. Zapamtite da ne želimo događaje `change` niti `input` ovde, jer nam trebaju podatke sa tastature.

```js
const input = document.querySelector("input");

input.addEventListener("keydown", (e) => {
    
})
```

`e.key` daje vrednost pritisnutog dugmeta, kao `a` za dugme `a` ili `Enter` za dugme `Enter`.

Za gore navedeni kod, kada kucate u `input`, slovo koje unosite biće prikazano u elementu `p`.

Ovo je dobro, ali ne želimo da se ti znakovi prikažu i u `input`. Ovde ulazi naša metoda za `preventDefault()`. Podrazumevano ponašanje `keydown` je prikazivanje znakova u polju za unos. Spreći ćemo to pozivanjem `e.preventDefault()`:

```css
  font-size: 1rem;
  border: 2px solid #ccc;
```

```js
const input = document.querySelector("input");
const output = document.getElementById("output");

input.addEventListener("keydown", (e) => {
  e.preventDefault();
  output.innerText = `You pressed the ${e.key} key`;
});
```

:::

Stoga, onemogućili ste podrazumevano ponašanje kako biste mogli da izvršite vaš specifični obradu događaja.

Još jedan uobičajen primer korišćenja metode `e.preventDefault` vezuje se za slanje forme. Podrazumevano, slanje forme šalje podatke na server i preusmerava stranicu. Korišćenjem `e.preventDefault()` sprečava se da se ovo desi.

```js
const form = document.querySelector("form");

form.addEventListener("submit", (e) => {
  e.preventDefault();
  // rest of code goes here
});
```

Onemogućavanje podrazumevanog ponašanja je korisno kada vam treba veća kontrola nad načinom na koji korisnik koristi stranicu, ali važno je uzeti u obzir stvari kao što je Pristupačnost – vaše prilagođeno ponašanje mora pružati iste funkcije kao i podrazumevano ponašanje.
