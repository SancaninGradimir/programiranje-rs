## JavaScript strana klijenta

- **JavaScript na strani klijenta**: Kod koji se izvršava na strani klijenta veb aplikacije, obično u web pretraživaču.
- **Ograničenja JavaScript na strani klijenta**: Iako je moćan, JavaScript na strani klijenta ima određena ograničenja:
  - Vrlo ograničen pristup fajlovima na klijentskoj lokaciji.
  - Nije namenjeno za obradu složene logike aplikacije.
  - Mogući su sigurnosni problemi (npr. otkrivanje osetljivih informacija kao što su lozinke baze podataka).
- **Razlog ograničenja**: Ova ograničenja postoje jer je JavaScript na strani klijenta prvobitno dizajnirana da se izvršava isključivo u web pretraživačima.

## Okruženje pregledača

- **JavaScript motori**: Veb pregledači obezbeđuju potrebno okruženje za izvršavanje JavaScript koda, uključujući JavaScript motor.
- **Primer dokumenta (DOM)**: Pretraživači pružaju pristup DOM, omogućavajući vam da pristupite funkcijama u vašem kodu HTML.
- **Историjski kontekst**: Ranije, JavaScript nije mogao biti pokren van pretraživača, ali sada može korišćenjem Node.js.

## Node.js šta je to?

- **Node.js**: Okruženje za izvršavanje JavaScript. U osnovi je definisano kao "okruženje za izvršavanje JavaScript otvorenog izvora i kompatibilno sa više platformi."
- **Otvoran izvor**: Kod Node.js je otvoren za javnost i održava ga velika zajednica programera.
- **Kompatibilnost sa više platformi**: Može raditi na svim operativnim sistemima, uključujući Windows, macOS i Linux.
- **Okruženje za izvršavanje JavaScript**: Omogućava vam pokretanje koda JavaScript izvan pregledača.

## Upotreba Node.js

- **Opšte aplikacije**: Node.js se koriste u različitim sektorima za:
  - Izgradnju web servera sa API koji obrađuju aplikacije za HTTP.
  - Razvoj web i mobilnih aplikacija.
  - Upravljanje složenim podacima i interakcija sa bazama podataka.
- **Primena u sektorima**: Ako koristite ili razvijate veb sajt ili web aplikaciju koja se povezuje sa bazom podataka ili obrađuje složene podatke, vrlo je verovatno da ćete koristiti Node.js u pozadini.
- **Glavne karakteristike**: Node.js je efikasan, skalabilan i ima veliku zajednicu programera i održavatelja širom sveta.

## Pretraživačko okruženje naspram Node runtime environment

- **Pretraživačko okruženje**: Dizajnirano je posebno za front-end razvoj mreže, pokreće JavaScript na strani klijenta.
- **Node okruženje za izvršavanje**: Dizajnirano je posebno za back-end razvoj mreže, pokreće JavaScript na strani servera.
- **Razlike u API**: Okruženje određuje dostupne API:
  - **Pretraživač**: Dostupnost API za DOM, ali sa ograničenjima pristupa sistemu fajlova klijentskog područja.
  - **Node**: Dostupnost skoro svim sistem resursima, uključujući sistem fajlova, ali ne i DOM.

## Stvari celog sveta

- **Globalna funkcija pretraživača (`window`)**: Omogućava pristup uslugama vezanim za pregledač:
  - Načini kontrole DOM.
  - Upravljanje kolačićima (cookies).
  - Upravljanje događajima pregledača.

- **Sveobuhvatni element za Node.js (`global`)**: Omogućava pristup specijalnim uslugama za Node.js:
  - Ugrađeni moduli za rad sa fajlovima korisničkog područja.
  - Mrežna sposobnost.
  - Interakcija sa operativnim sistemom.

## Kontrola verzija

- **Verzija pretraživača**: Ne kontroliš verziju okruženja pretraživača koju će vaši korisnici koristiti za posetu vašem sajtu.
- **Verzija Node.js**: Možete izabrati verziju Node.js koju želite da koristite na svom serveru.

## Važnost od Node.js

- **Full-stack razvoj**: Node.js je promenio web razvoj omogućavajući programerima da koriste JavaScript za izgradnju svih front-end i back-end delova full-stack aplikacija.
- **Efikasnost**: Proces razvoja je bolji jer programeri ne moraju da nauče novi programski jezik za razvoj back-end delova.
- **Uticaj**: Node.js je moćan, skalabilan i svestran alat koji je postao izuzetno važan za programere širom sveta.

## Prednost za Node.js

- **Jedik jezik za full-stack**: Možete implementirati sve delove prednjeg i zadnjeg dela web aplikacije koristeći JavaScript.
  - Smanjuje početnu težinu učenja.
  - Nema potrebe menjati između programskih jezika.
  - Povećava efikasnost i produktivnost.

- **Nezaustavni temelj, vođen događajima**: Odličan za razvoj aplikacija u stvarnom vremenu gde su responzivnost i efikasnost ključne za kreiranje dobrog korisničkog iskustva.

- **Jednoterađeni Event Loop**: Može efikasno da obrađuje veliki broj zahteva i I/O operacija istovremeno.
  - Pogodno je za aplikacije koje moraju da obrađuju mnogo zahteva istovremeno.

- **Velika zajednica**: Hiljade programera širom sveta nastavljaju da održavaju Node.js i dodaju nove funkcije.
  - Dostupni su mnogi resursi za učenje.
  - Lako je pronaći odgovore na pitanja.

- **npm (Node Package Manager)**: Moćan alat koji vam omogućava instaliranje i upravljanje paketima i modulima za vaše projekte.
  - Paketi su dostupni za mnogo namena: obradu aplikacija HTTP, rad sa bazama podataka, kreiranje datuma i vremena, testiranje koda i još mnogo toga.
  - Ponovno koristite kod koji je napisan, testiran i podeljen od strane drugih programera.
  - Ubrzava i poboljšava vaš radni tok (workflow).

- **Pristupačne cene**: Node.js je besplatno i sa slobodnog izvora.
  - Paketi, biblioteke i srodni alati su takođe obično besplatni.
  - To je važno kada se optimizuje budžet.

## Gubitak za Node.js

- **Jedno-nićni graničnik**: Radi sa jednom niti u trenutku, što znači da može obraditi samo jednu operaciju odjednom.
  - Iako arhitektura omogućava efikasnu obradu više zahteva istovremeno, zadaci koji koriste veliku snagu procesora mogu blokirati glavnu nit i uzrokovati probleme sa performansama.
  - Primjeri zadataka koji zahtevaju veliku snagu procesora: složene matematičke operacije, obrada slika i videa, šifrovanje.
  - Postoje načini da se ovo preuzme, ali obično povećavaju složenost programa.

- **Asinhrono programiranje (asynchronous)**: Node.js zavisi od asinhronog programiranja.
  - Pokreće se zadatak koji može trajati dugo, ali umesto da čeka da završi, glavni program nastavlja da radi dok taj zadatak teče paralelno.
  - Kada se zadatak završi, program obrađuje rezultat.
  - Često uključuje „callback pozive“ (callbacks), koji su funkcije koje definišu šta se dešava kada asinhrone operacije završe.
  - Može učiniti kod teškim za čitanje, razumevanje i pronalaženje grešaka.

- **Problemi sa kvalitetom paketa**: Kada birate pakete sa npm-a, neki možda nisu redovno održavani i mogu predstavljati rizik za vašu aplikaciju.
  - Programeri bi trebalo pažljivo da procene svaki paket.
  - Proverite da li paketi prate najbolje prakse za kvalitet i bezbednost.

## NVM (Node Version Manager)

- **NVM**: Alat za upravljanje višestrukim verzijama Node.js na jednom računaru.
- **Svrha**: Pomaže vam da lako prebacujete između različitih verzija Node.js za različite projekte.
- **Preporuke**: Preporučuje se instaliranje NVM-a pre Node.js radi buduće fleksibilnosti.

## Instalacija NVM-a na macOS/Linux

- **Alati za komandnu liniju za Xcode (macOS)**: Moraju biti ručno instalirani pre pokretanja skripte za instalaciju NVM na macOS-u.

```bash
xcode-select --install
```

- **Potvrdite instalaciju Xcode-a**:

```bash
xcode-select --version
```

- **Preuzmite i pokrenite skriptu za instalaciju NVM-a**: Pokrenite jednu od ovih komandi:

```bash
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.3/install.sh | bash
```

```bash
wget -qO- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.3/install.sh | bash
```

- **Ponovo pokrenite terminal**: Možda ćete morati da ponovo pokrenete terminal nakon završetka procesa kako biste učitali nove komande i konfiguracije.

- **Potvrdite instalaciju NVM**:

```bash
nvm -v
```

## Instaliranje Node.js korišćenjem NVM-a na macOS/Linux

- **Instalirajte najnoviju verziju LTS**: Instaliranje najnovije verzije Long-Term Support (LTS) za Node.js:

```bash
nvm install --lts
```

- **LTS (Long-Term Support)**: Ova verzija se obično preporučuje jer daje prioritet stabilnosti, pouzdanosti i bezbednosti umesto novih ili eksperimentalnih funkcija.
  - Temeljno je testirano i spremno za produkciju.
  - Garantuje dug period podrške sa fokusom na popravke grešaka i sigurnosne zakrpe.

- **Instalirajte specifičnu verziju**: Možete instalirati specifičnu verziju od Node.js navođenjem broja verzije:

```bash
nvm install <version>
```

Primer:

```bash
nvm install 20
```

- **Promeni na verziju**:

```bash
nvm use <version>
```

Primer:

```bash
nvm use 20
```

- **Pogledajte trenutnu verziju Node.js**:

```bash
node -v
```

## Kako instalirati NVM na Windows-u

- **Zvanična podrška za NVM**: Zvanično izdanje NVM ima podršku samo za Windows u specifičnim slučajevima.
  - Ako imate Windows Subsystem for Linux (WSL), možete pratiti vodič za macOS/Linux naveden iznad.

- **nvm-windows**: Alat za upravljanje verzijama Node.js za Windows.
  - Potpuno drugačiji projekat od zvaničnog NVM projekta.
  - Ista svrha: da pomogne u radu sa više verzija Node.js na jednom uređaju.

- **Pre instalacije**: Preporučuje se da se ukloni bilo koja prethodna instalacija Node-a pre instaliranja NVM za Windows.

- **Koraci instalacije**:
  - Idite na sekciju Releases zvanične baze podataka: `https://github.com/coreybutler/nvm-windows/releases`.
  - Preuzmite najnoviji instalater.
  - Pratite uputstva.

- **Ponovo pokrenite terminal**: Možda ćete morati da ponovite terminal ako komanda `nvm` nije odmah dostupna.

- **Potvrdi instalaciju**:

```powershell
nvm -v
```

## Osnovna naredba za NVM na Node.js

- **Napomena o komandama**: Sve komande rade sa NVM instaliranim na macOS/Linux-u, uključujući i WSL. Mogu raditi drugačije ili uopšte ne raditi za `nvm-windows`. Alternativne komande za `nvm-windows` su obezbeđene gde je to moguće.

### Politika izvršavanja PowerShell-a (Windows)

- **Zadajna politika izvršavanja**: Zadajna politika izvršavanja PowerShell-a je `Restricted`, koja sprečava pokretanje skripte kao sigurnosna mera.

- **Pokrenite PowerShell kao administrator**: Desni klik na PowerShell, a zatim kliknite na "Run as Administrator".

- **Pogledajte trenutnu politiku implementacije**:

```powershell
Get-ExecutionPolicy
```

- **Postavite politiku izvršavanja na RemoteSigned**: Omogućava pokretanje lokalnih skripti i `npm`:

```powershell
Set-ExecutionPolicy RemoteSigned
```

- **Nakon postavljanja politike**: Zatvori administratorsko PowerShell okno i otvori neadministratorsko PowerShell okno.

### Lista svih instaliranih verzija

- **Prikaži instalirane verzije**: Da biste listali sve instalirane verzije Node.js i proverili trenutno korišćenu verziju:

```bash
nvm ls
```

### Postavi podrazumevanu verziju za Node.js (NVM na macOS/Linux samo)

- **Postavljanje podrazumevanog**: Postavlja podrazumevanu verziju Node.js za nove terminal sesije:

```bash
nvm alias default <version>
```

Primer:

```bash
nvm alias default 22.20.0
```

- **Generiš specifična imena**: Unesite specifičnije ime za verziju:

```bash
nvm alias <new_name> <version>
```

Primer:

```bash
nvm alias new-api 18.20.8
```

- **Koristi specifično ime**:

```bash
nvm use <new_name>
```

Primer:

```bash
nvm use new-api
```

### Ukloni verziju od Node.js

- **Ukloni verziju**:

```bash
nvm uninstall <version>
```

Primer:

```bash
nvm uninstall 20.19.5
```

- **Deinstaliraj glavnu verziju (NVM na macOS/Linux)**: Možete ukloniti verziju pokretanjem komande `uninstall` i glavne verzije kao što su `22` ili `20`:

```bash
nvm uninstall 20
```

Ako imate više verzija te glavne izdanja instalirano, kao `20.14.0` i `20.12.2`, će se prvo ukloniti najnovija verzija. Pokretanje `nvm uninstall 20` će prvo ukloniti `20.14.0`.

### Pogledajte verziju od Node.js

- **Pogledajte trenutnu verziju**:

```bash
node -v
```

### Proverite verziju npm-a

- **npm**: Upravljač paketima za Node.js koji se koristi za instaliranje, objavljivanje i upravljanje softverskim paketima. Instalira se automatski kada instalirate Node.js.

- **Proveri npm verziju**:

```bash
npm -v
```

### Pokreni fajl JavaScript

- **Izvrši fajl za JavaScript**: Izvršava fajl JavaScript u trenutnom okruženju od Node.js:

```bash
node <file.js>
```

Primer:

```bash
node app.js
```

### Kreiraj projekat za Node.js

- **Inicijalizuj projekat**: Kreira novi projekat za Node.js kreiranjem fajla `package.json` koji prati njegove detalje i zavisnosti:

```bash
npm init
```

- **Podešavanje interakcije**: Komanda za `npm init` vas vodi kroz uputstvo korisnika linije komandi za inicijalizovanje fajla `package.json`.
  - Postavlja pitanja o imenu projekta, opisu, verziji itd.

- **Podrazumevana podešavanja**: Počinje sa podrazumevanim fajlom `package.json`:

```bash
npm init -y
```

au

```bash
npm init --yes
```

Instaliraj paket

- **Instalirajte poseban paket**:

```bash
npm install <package>
```

Primer:

```bash
npm install express
```

### Instaliraj sve zavisnosti

- **Instalirajte zavisnosti**: Instalirajte sve navedene zavisnosti iz fajla `package.json` projekta:

```bash
npm install
```
