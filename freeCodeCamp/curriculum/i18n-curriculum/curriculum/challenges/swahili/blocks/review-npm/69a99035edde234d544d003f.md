## npm šta je?

- **Značenje**: npm je onlajn registar softvera otvorenog koda, koji funkcioniše kao „tržište aplikacija“ za kod.
- **Zvanično značenje**: npm ne znači zvanično "Node Package Manager"; to je kružni akronim koji znači da "npm nije skraćenica za nijednu reč".
- **Tri povekana dela**:

- **Web sajt (npmjs.com)**: Za pretraživanje paketa u registru.
- **Registrar**: Velika javna baza podataka koja čuva sve pakete.
- **CLI (Command Line Interface)**: Alat instaliran zajedno sa Node.js za upravljanje zavisnostima projekta.

- **Alternativa**: Iako je npm najčešće korišćen, popularne alternative su **Yarn**, **PNPM**, i **Bun**.

## Fajl package.json

- **Svrha**: Konfiguracioni fajl koji čuva metapodatke, zavisnosti i skripte za pokretanje za projekat Node.js.
- **Ključni deo**:

- `name` na `version`: Rešava projekat.
  - `main`: Opisuje ulazni fajl, često `index.js`.
  - `license`: Definiše pravne uslove (podrazumevano je `ISC`).
  - `dependencies`: Paketi potrebni za pokretanje programa u produkciji.
  - `devDependencies`: Alati koji se koriste samo tokom razvoja, kao što su testna okruženja.

- **Kreiranje**:

- `npm init`: Pokreće proces integracije.
  - `npm init -y`: Instalira `package.json` sa podrazumevanim vrednostima odmah.

## Izdanje semantičke verzije (Semantic Versioning - SemVer)

- **Struktura**: `MAJOR.MINOR.PATCH`.
- **Pravilo**:

- **MAJOR**: Promene koje menjaju kompatibilnost sa postojećim kodom.
  - **MINOR**: Nove funkcionalnosti koje su kompatibilne sa prethodnom verzijom.
  - **PATCH**: Male ispravke grešaka i poboljšanja.

- **Znak za epidemiju**:

- `^` (Karetnik): Omogućava manje i korektivna ažuriranja (npr., `^1.2.3` omogućava `1.x.x`).
  - `~` (Tilda): Omogućava samo korektivna ažuriranja (npr., `~1.2.3` omogućava `1.2.x`).
  - `*` (Zvezdica): Kompatibilno sa svim verzijama; koristi se posebno za testiranje.

## Upravljanje zavisnostima

- **Instalacija**:

- `npm install <name>`: Dodaje zavisnost za produkciju.
- `npm install <name> -D`: Dodaje zavisnost za razvoj.
- `npm install <name>@<version>`: Instalira specifičnu verziju.

- **Moduli za Node na Locks**:

- `node_modules/`: Čuva stvarni kod instaliranih paketa.
  - `package-lock.json`: Zaključava stvarna verzija svih zavisnosti i „zavisnosti zavisnosti“ kako bi se osigurala stabilnost u različitim okruženjima.

- **Uklanjanje**: `npm uninstall <name>` uklanja paket i ažurira `package.json`.
- **Održavanje**:

- `npm outdated`: Instalira pakete sa dostupnim novim verzijama.
  - `npm update`: Ažurira pakete unutar dozvoljenog opsega SemVer-a.

## Dokumentacija npm-a

- **Funkcija**: Specifične komande definisane u delu `scripts` od `package.json` za automatsko pokretanje zadataka.
- **Izvršavanje**: Izvršava se putem `npm run <script_name>`.
- Postoje prečice za `start`, `test`, `stop` i `restart` (npr. `npm start`).
- **Sekvencijalno izvršavanje**:
- `&&`: Pokreće samo drugu komandu ako je prva uspešna.
- `;`: Izvršava komande jednu za drugom, bez obzira na uspeh.
- **Paralelno izvršavanje**: Koristite izvršilac `&` za istovremeno pokretanje procesa.
- **Argumenti**: Prosledite argumente tekstualno koristeći glagol `--` (npr. `npm start -- 8000`).

## Objavljivanje na npm registru

- **Pristup registru**: Baza podataka javnih i privatnih paketa.
- **Koraci za objavljivanje**:

1. Kreiraj besplatan nalog na `npmjs.com`.
2. Prijavite se putem terminala koristeći `npm login`.
3. Proverite da je `name` jedinstveno, a da su `version` važeći u `package.json`.
4. Kreiraj `README.md` i `.npmignore` (izbegavajući osetljive fajlove).
5. Pokrenite `npm publish` (ili `npm publish --access public` za pakete sa opsegom).

- **Wigo**: Koristi se za izbegavanje kolizija imena štampanjem ispod `@username/package-name`.

## Sistemi modula u Node.js

### CommonJS (CJS)

- **Podrazumevani sistem**: Način po dizajnu Node.js kako rukuje modulima; uvozi ih odjednom.
- **Sintaksa**:
- **Import**: `const module = require('./path')`.
- **Export**: `module.exports = { functionName }`.
- **Identifikacija**: Koristi se za fajlove `.cjs` ili fajlove `.js` kada nije definisano/navedeno `type` u `package.json`.

### ES Modules (ESM)

- **Normalizovana struktura**: Moderni standard za JavaScript; efikasno obavlja asinhroni unos.
- **Sintaksa**:

- **Izvoz**: `export { name }` ili `export default function`.
- **Uvoz**: `import { name } from './path.mjs'`.
- **Identifikacija**: Koristi se za fajlove `.mjs` ili fajlove `.js` kada `package.json` ima `"type": "module"`.
