# --interactive--

Izgled i ponašanje elemenata za unos veoma su važni pri izradi veb obrazaca. Obrazac koji odgovara na korisničke radnje poboljšava korisničko iskustvo i ne bi trebalo da izaziva zabunu.

CSS Pruža nekoliko pseudo-klasa unosa koje mogu učiniti vaše forme razumljivijim i prijatnijim za korisnika. Pogledajmo ove pseudo-klase u detalju.

Pseudo klasa `:focus` vam omogućava da ciljate na polje za unos kada korisnik izabere ili klikne, ističući aktivni deo unosa. Ovo pomaže korisnicima da lako identifikuju dok pišu:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css" />
<form>
  <input type="text" />
</form>
```

```css
input:focus {
  border: 2px solid crimson;
  outline: none;
}
```

:::

Kao što naziv govori, pseudo-klasa `:hover` aktivira se kada korisnik postavi pokazivač miša iznad elementa. Ona pruža vizuelnu povratnu informaciju pre interakcije i pokazuje da je element spreman za akciju.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css" />
<form>
  <input type="text" />
</form>
```

```css
input:hover {
  background-color: orange;
}
```

:::

U prethodnom primeru pozadinska boja polja za unos menja se u narandžastu kada korisnik pređe pokazivačem preko njega, čime se naglašava da je spremno za interakciju.

Pseudo-klasa `:checked` omogućava drugačije stilizovanje radio dugmeta ili polja za potvrdu kada ih korisnik izabere. Tako korisnik lako vidi koja je opcija odabrana.

Evo primera polja za potvrdu kojim korisnik prihvata uslove korišćenja sajta.

**NAPOMENA:** Deo CSS-a u ovom primeru koristi svojstva koja još nisu obrađena. Cilj je da stekneš predstavu o tome kako se pravi prilagođeno polje za potvrdu. Sve detalje ćeš naučiti u narednim lekcijama i radionicama.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css" />
<form>
  <label>
  Agree <input class="checkbox" type="checkbox" />
  </label>
</form>
```

```css
.checkbox {
  appearance: none;
  width: 18px;
  height: 18px;
  border: 2px solid #ccc;
  border-radius: 4px;
  display: inline-block;
  position: relative;
  cursor: pointer;
  transition: all 0.25s ease;
  vertical-align: middle; 
}

.checkbox:hover {
  border-color: #888;
}

.checkbox:checked {
  background-color: #4caf50;
  border-color: #4caf50;
}

.checkbox:checked::after {
  content: "";
  position: absolute;
  left: 4px;
  top: 0px;
  width: 5px;
  height: 10px;
  border: solid white;
  border-width: 0 2px 2px 0;
  transform: rotate(45deg);
}

.checkbox:focus {
  outline: 2px solid #90caf9;
  outline-offset: 2px;
}

```

:::

`:required` cilja polja za unos koja imaju atribut `required`. Time se korisniku jasno stavlja do znanja da mora popuniti to polje pre slanja obrasca.

Evo primera obrasca sa obaveznim poljima za unos:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css" />
<form action="#">
  <label for="name">Name:</label>
  <input type="text" id="name" name="name" required />

  <label for="email">Email:</label>
  <input type="email" id="email" name="email" required />

  <label for="phone">Phone Number:</label>
  <input type="tel" id="phone" name="phone" />

  <button type="submit">Submit</button>
</form>
```

```css
input:required {
  border: 2px solid orange;
}
```

:::

Tokom validacije obrasca možeš koristiti pseudo-klasu `:valid` za stilizovanje polja koja ispunjavaju uslove validacije, a `:invalid` za ona koja ih ne ispunjavaju. Uobičajeno je da se validna polja označe zelenom, a nevalidna crvenom bojom.

Ovo je primer korišćenja lažne klase za `:valid`:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css" />
<form>
  <label for="email">Email:</label>
  <input type="email" id="email" name="email" />
</form> 
```

```css
input:valid {
  border-color: green;
}
```

:::

Evo primera korišćenja klase `:invalid`:

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css" />
<form>  
  <label for="email">Email:</label>
  <input type="email" id="email" name="email" />
</form>
```

```css
input:invalid {
  border-color: crimson;
}
```

:::

Lažna klasa `:disabled` vam omogućava da izaberete i dekorujete onemogućene elemente unosa. Ovi elementi imaju svojstvo `disabled` koje sprečava korisnicima da sa njima interaguju. Onemogućeni unos ne može se kliknuti, fokusirati niti urediti.

Ovde je primer etikete i onemogućenog polja za unos.

U CSS, ciljamo na to onemogućeno polje i postavljamo mu pozadinsku boju od `lightgray` i menjamo pokazivač u `not-allowed`.

:::interactive_editor

```html
<link rel="stylesheet" href="styles.css" />
<form>
  <label for="name">Name:</label>
  <input class="text-input" type="text" id="name" name="name" disabled />
</form>
```

```css
.text-input:disabled {
  background-color: lightgray;
  cursor: not-allowed;
}
```

:::

Vrednost svojstva CSS za `cursor: not-allowed;` menja izgled pokazivača, ukazujući da akcija nije dozvoljena.

Kada se koristi za element, prikazuje pokazni prst sa oznakom kruga sa križnim linijom (obično krug sa uspravnom linijom) kako bi korisnicima ukazao da je taj element neinteraktivan ili da se na njega ne može kliknuti/sa njim ne može interagovati.

Evo još nekoliko primera pseudo-klasa za polja za unos i njihovih namena:

- `:autofill`: Koristi stilove za polja unosa koja pregledač automatski popunjava sa sačuvanim podacima.
- `:optional`: Dekorisanje elemenata unosa koji nisu obavezni i koje se može ostaviti prazno.
- `:in-range` na `:out-of-range`: Dekorisanje elemenata u zavisnosti od toga da li su njihove vrednosti unutar ili van definisanih granica.
