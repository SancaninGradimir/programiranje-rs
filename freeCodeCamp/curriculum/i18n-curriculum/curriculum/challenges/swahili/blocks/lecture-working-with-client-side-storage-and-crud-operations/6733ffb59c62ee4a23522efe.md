# --interactive--

`IndexedDB` je alat za čuvanje strukturiranih podataka u pregledaču. Ugrađen je u moderne veb pregledače i omogućava veb aplikacijama da efikasno čuvaju i preuzimaju JavaScript objekte.

Za razliku od drugih metoda skladištenja kao što je `localStorage`, koji ograničava čuvanje samo stringova, `IndexedDB` može da čuva JavaScript objekte, fajlove i skoro bilo koju vrstu podataka. Ovo ga čini pogodnim za veb aplikacije koje moraju da rade sa velikim i složenim strukturama podataka.

Sada, pogledajmo kako `IndexedDB` funkcioniše.

Prvi korak je otvaranje baze podataka. Evo primera:

:::interactive_editor

```js
let request = indexedDB.open("Sample DB", 1);

request.onerror = function(event) {
  console.log("Error opening database");
};

request.onsuccess = function(event) {
  let db = event.target.result;
  console.log("Database opened successfully");
};
```

:::

U ovom kodu otvaramo bazu podataka pod nazivom `"Sample DB"` i verzijom 1. Definišemo dve funkcije povratnog poziva: jednu za obradu grešaka i drugu kada se baza podataka uspešno otvori. `db` objekat koji dobijemo u uspešnom povratnom pozivu koristićemo za rad sa bazom podataka.

Ako proverite interfejs alata za programere u pregledaču, videćete svoju `Sample DB` bazu u odeljku `IndexedDB`. Kada imate otvorenu bazu podataka, možete početi da radite sa skladištima objekata.

Skladišta objekata u IndexedDB-u slična su tabelama u klasičnim bazama podataka. Ona čuvaju stvarne podatke koje želite da sačuvate. Evo kako se kreira skladište objekata:

```js
let request = indexedDB.open("Sample DB", 1);

request.onupgradeneeded = function(event) {
  let db = event.target.result;
  let objectStore = db.createObjectStore("customers", { keyPath: "id" });
};
```

Ovaj kod kreira skladište objekata pod nazivom `"customers"` i koristi `"id"` kao ključni put. Ključni put je poput primarnog ključa u običnoj bazi podataka - koristi se za jedinstvenu identifikaciju svakog zapisa.

Kako bismo dodali podatke u našu prodavnicu predmeta, uradili bismo nešto ovako. `db` Ovaj primer predstavlja strukturu baze podataka za `IndexedDB`.

```js
let transaction = db.transaction(["customers"], "readwrite");
let objectStore = transaction.objectStore("customers");
let request = objectStore.add({ id: 1, name: "John Doe", email: "john@example.com" });

request.onerror = function(event) {
  console.log("Error adding data");
};

request.onsuccess = function(event) {
  console.log("Data added successfully");
};
```

Ovaj kod dodaje novog korisnika u naše skladište objekata `"customers"`. Pokrećemo transakciju (što je način na koji grupišemo operacije baze podataka), dobijamo referencu na naše skladište objekata, a zatim dodajemo podatke.

Dohvatanje podataka se dešava na isti način. Pokrećemo transakciju, dobijamo našu prodavnicu stvari, pa koristimo metod kao `get` za dohvaćanje podataka:

```js
let transaction = db.transaction(["customers"]);
let objectStore = transaction.objectStore("customers");
let request = objectStore.get(1);

request.onerror = function(event) {
  console.log("Error retrieving data");
};

request.onsuccess = function(event) {
  console.log("Customer:", request.result);
};
```

Ovaj kod preuzima korisnika sa `id` vrednošću `1` iz našeg skladišta objekata `"customers"`.

Jedna od ključnih karakteristika za `IndexedDB` je što je asinhrono.

Ovo znači da kada interagujete sa `IndexedDB`, aktivnosti ne blokiraju glavne niti mrežne aplikacije. Ovo osigurava da vaša mrežna aplikacija ostaje adaptivna za uređaj čak i kada koristite velike količine podataka.

Iako `IndexedDB` nudi veliki kapacitet, ima strmu krivulju učenja u poređenju sa drugim jednostavnim opcijama skladištenja API koje mogu biti izazovne za početnike. Međutim, za aplikacije koje moraju da obrađuju velike količine strukturiranih podataka na strani klijenta, `IndexedDB` nudi neuporediv kapacitet.
