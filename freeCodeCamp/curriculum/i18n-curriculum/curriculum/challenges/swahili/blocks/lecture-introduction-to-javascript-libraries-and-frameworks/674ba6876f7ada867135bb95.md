# --description--

U početnim lekcijama, naučili ste kako da radite sa importima i eksportima u JavaScript. U ovoj lekciji, ćemo pogledati kako da uvezete i izvezete delove u React.

Export čini komponentu dostupnom za druge fajlove. Import omogućava fajlu da koristi komponentu koja je izveze negde drugde. U projektima React, komponentama se ponovo koriste i spajaju kako bi se izgradili korisnički interfejsi.

U ovom tutorijalu, naučićete kako da eksportujete delove React koristeći default exports i named exports, i kako da ih uvezete gde su potrebni.

U ovom primeru imamo komponentu `Cat` koja se nalazi u datoteci `Cat.jsx`. Koristimo ekstenziju `.jsx` zato što je ova datoteka prvenstveno namenjena JSX kodu.

Ova komponenta `Cat` vraća JSX oznake koje sadrže naslov i sliku mačke po imenu Mr. Whiskers.

```jsx
function Cat() {
  return (
    <div className="card">
      <h2>Mr. Whiskers</h2>
      <img
        src="https://cdn.freecodecamp.org/curriculum/cat-photo-app/running-cats.jpg"
        alt="Tuxedo cats running on dirt ground."
      />
    </div>
  );
}
```

Ako želimo da koristimo komponentu `Cat` u drugoj datoteci, prvo moramo da je izvezemo ovako:

```jsx
function Cat() {
  return (
    <div className="card">
      <h2>Mr. Whiskers</h2>
      <img
        src="https://cdn.freecodecamp.org/curriculum/cat-photo-app/running-cats.jpg"
        alt="Tuxedo cats running on dirt ground."
      />
    </div>
  );
}

export default Cat;
```

Koristimo ključnu reč `default` zato što će ovo biti podrazumevani izvoz iz modula. Komponentu možete izvesti i u istoj liniji u kojoj je definišete, ovako:

```jsx
export default function Cat() {
  return (
    <div className="card">
      <h2>Mr. Whiskers</h2>
      <img
        src="https://cdn.freecodecamp.org/curriculum/cat-photo-app/running-cats.jpg"
        alt="Tuxedo cats running on dirt ground."
      />
    </div>
  );
}
```

Možete odlučiti da podređene komponente uvozite u druge roditeljske komponente ili direktno u glavnu datoteku komponente. U ovom primeru uvešćemo komponentu `Cat` u glavnu datoteku komponente.

Svaki React projekat ima komponentu najvišeg nivoa, koja se obično zove `App.jsx`:

```jsx
export default function App() {
  return (
    // return component here
  );
}
```

Ova datoteka se obično nalazi u direktorijumu `src` vašeg projekta. Više o uobičajenoj strukturi projekata naučićete u sledećoj lekciji.

Da biste koristili komponentu `Cat` unutar glavne komponente `App`, potrebno je da je uvezete ovako:

```jsx
import Cat from "./Cat";

export default function App() {
  return (
    // return component here
  );
}
```

Sada, možete prebaciti deo `Cat` u sekciju `App` ovako:

```jsx
import Cat from "./Cat";

export default function App() {
  return (
    <Cat />
  );
}
```

Ova metoda funkcioniše dobro kada je fajl odgovoran da prenese samo jedan glavni deo, čineći sintaksu unosa jednostavnom i lako čitljivom. Zapamtite, fajl može imati samo jedan export od default, što ga čini boljim za fajl koji sadrži samo jedan glavni deo.

Sada, pošto znate kako da koristite default eksporte, hajde da istražimo *named exports*. Named exports omogućavaju fajlu da deli više delova ili više funkcija. Za razliku od default eksporta, oni moraju biti uvezeni koristeći originalno ime sa kojim su eksportovani (osim ako ne promenite ime koristeći reč `as`).

Ovo je važno kada se fajl koristi kao biblioteka delova. Na primer, ovde je fajl koji uključuje dva dela, Cat i Dog:

```jsx
// Animals.jsx
export function Cat() {
  return <h2>Mr. Whiskers</h2>;
}

export function Dog() {
  return <h2>Fido</h2>;
}
```

Koristimo reč export za svaki deo za pojedinicu. Da biste ih koristili u drugim fajlovima, ubacujemo ih koristeći okružnice:

```jsx
import { Cat, Dog } from "./Animals";

export default function App() {
  return (
    <div>
      <Cat />
      <Dog />
    </div>
  );
}
```

Sintaksa za `{ Cat, Dog }` govori JavaScript da uvez specifične eksporte sa imenima iz fajla `Animals.jsx`. Imena unutar vitičnih zagrada moraju se savršeno poklapati sa imenima koja su eksportovana. Ako morate da izbegnete kolizije imena, ili ako preferirate drugačije ime u trenutnom fajlu, možete promeniti naziv segmenta tokom uvoza koristeći reč as.

```jsx
import { Cat as Kitty } from "./Animals";

export default function App() {
  return <Kitty />;
}
```

Na kraju, datoteka može imati export jedan od default i mnogo imenovanih eksporti.

```jsx
// Animals.jsx
export default function Cat() {
  return <h2>Mr. Whiskers</h2>;
}

export function Dog() {
  return <h2>Fido</h2>;
}
```

Kada unosite mešoviti izvozi, export od default dolazi prvi (bez zagrada), praćen imenovanim izvozima (unutar zagrada):

```jsx
// App.jsx
import Cat, { Dog } from "./Animals";

export default function App() {
  return (
    <div>
      <Cat />
      <Dog />
    </div>
  );
}
```

U ovom primeru, `Cat` je export od default, pa ne radi sa vitičnim zagradama. `Dog` je export sa imenom, pa zahteva te zagrade.

Da biste razumeli kako default i named exports funkcionišu, možete organizovati delove React u više fajlova i ponovo ih koristiti gde god je potrebno. Kada gradite veći program, izbor pravog stila export pomoći će vam da održavate svoj kod čistim i jednostavnim za održavanje, što olakšava rad sa delovima napravljenima od strane drugih.
