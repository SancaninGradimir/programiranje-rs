# --description--

U JavaScript, `Set` je nešto ugrađeno za upravljanje kolekcijama podataka. Omogućava vam skladištenje jedinstvenih vrednosti bilo kog tipa, bilo da su primarni ili reference na objekte. `Set` osigurava da svaka vrednost unutar njega se pojavljuje samo jednom, što ga čini korisnim za poništavanje iz redova podataka ili rukovanje kolekcijama različitih vrednosti.

U vezi sa `WeakSet`, je poseban tip od `Set` sa malim komponentima koji mu omogućavaju skladištenje slabih referenci na objekte i oznake. Za razliku od `Set`, `WeakSet` ne zavisi od osnovnih stvari kao što su brojevi ili nizovi slova.

Umesto standardnog `Set`, `WeakSet` čuva samo objekte, a referenca na te objekte je "slaba", što znači da WeakSets ne uklanjaju sačuvane objekte sistemom za čišćenje memorije ako nema drugih referenci na njih. Jednostavnije rečeno, ako se nešto ne koristi nigde drugde u vašem kodu, ono je automatski uklonjeno kako bi se oslobodila memorija.

Ili da kreirate `Set`, koristite komponentu `Set` i postavite je u parametar:

```js
const myFirstSet = new Set();
```

Takođe možete inicijalizovati `Set` sa vrednošću:

```js
const treeSet = new Set(["Baobab", "Jackalberry", "Mopane Tree", "Breadfruit"]);
```

Kada pokrenete `Set` u konzoli, rezultat izgleda ovako:

```js
/*
Set(4) {"Baobab", "Jackalberry", "Mopane Tree", "Breadfruit"}
  [[Entries]]
  0: "Baobab"
    value: "Baobab"
  1: "Jackalberry"
    value: "Jackalberry"
  2: "Mopane Tree"
    value: "Mopane Tree"
  3: "Breadfruit"
    value: "Breadfruit"
  size: 4
  [[Prototype]]: Set
*/
```

Ako počnete `Set` sa vrednošću, možete koristiti metodu `add()` da dodate element u `Set`:

```js
const treeSet = new Set();

// Add items to the treeSet
treeSet.add("Baobab");
treeSet.add("Jackalberry");
treeSet.add("Mopane Tree");
treeSet.add("Breadfruit");
```

Rezultati i njihov izgled na konzoli ostaju takvi.

Nemojte zaboraviti da stare stvari će biti zaboravljene u `Set`:

```js
const treeSet = new Set();

// Add items to the treeSet
treeSet.add("Baobab");
treeSet.add("Jackalberry");
treeSet.add("Mopane Tree");
treeSet.add("Breadfruit");
treeSet.add("Baobab"); //duplicate item will be ignored

console.log(treeSet);
// Set(4) {"Baobab", "Jackalberry", "Mopane Tree", "Breadfruit"}
```

Drugi način koji možete koristiti za kontrolu `Set` je:

- `delete()`  
- `clear()`
- `has()`
- `entries()`
- `forEach()`
- `keys()`
- `values()`

Istražite kako ove metode funkcionišu pojedinačno. `delete()` uklanja navedeni element iz `Set`:

```js
const treeSet = new Set();

// Add items to the treeSet
treeSet.add("Baobab");
treeSet.add("Jackalberry");
treeSet.add("Mopane Tree");
treeSet.add("Breadfruit");

treeSet.delete("Breadfruit");

console.log(treeSet); // Set(3) {"Baobab", "Jackalberry", "Mopane Tree"}
```

`has()` proveri da li neka vrednost postoji unutar `Set`:

```js
const treeSet = new Set();

// Add items to the treeSet
treeSet.add("Baobab");
treeSet.add("Jackalberry");
treeSet.add("Mopane Tree");
treeSet.add("Breadfruit");

treeSet.delete("Breadfruit");

console.log(treeSet.has("Breadfruit")); // false
```

`entries()` obnovi stanje `Set` sa redom vrednosti u strukturu `[value, value]`:

```js
const treeSet = new Set();

// Add items to the treeSet
treeSet.add("Baobab");
treeSet.add("Jackalberry");
treeSet.add("Mopane Tree");
treeSet.add("Breadfruit");

treeSet.delete("Breadfruit");

console.log(treeSet.entries());
// SetIterator {"Baobab" => "Baobab", "Jackalberry" => "Jackalberry", "Mopane Tree" => "Mopane Tree"}
```

`keys()` i `values()` prikazuje vrednost u `Set`. `keys()` je samo alias za putanju `values()`:

```js
const treeSet = new Set();

// Add items to the treeSet
treeSet.add("Baobab");
treeSet.add("Jackalberry");
treeSet.add("Mopane Tree");
treeSet.add("Breadfruit");

treeSet.delete("Breadfruit");

console.log("Keys: ", treeSet.keys());
console.log("Values: ", treeSet.values());
// Keys: SetIterator {"Baobab", "Jackalberry", "Mopane Tree"}
// Values: SetIterator {"Baobab", "Jackalberry", "Mopane Tree"}
```

`forEach()` omogućava ponavljanje putem `Set`:

```js
const treeSet = new Set();

// Add items to the treeSet
treeSet.add("Baobab");
treeSet.add("Jackalberry");
treeSet.add("Mopane Tree");
treeSet.add("Breadfruit");

treeSet.delete("Breadfruit");

treeSet.forEach((tree) => console.log(tree));
/*
Baobab
Jackalberry
Mopane Tree
*/
```

`clear()` uklanja sve elemente reda:

```js
const treeSet = new Set();

// Add items to the treeSet
treeSet.add("Baobab");
treeSet.add("Jackalberry");
treeSet.add("Mopane Tree");
treeSet.add("Breadfruit");

treeSet.delete("Breadfruit");

treeSet.clear();

console.log(treeSet); // Set(0) {size: 0}
```

Takođe je važno napomenuti da postoji svojstvo `size` koje vraća broj elemenata u `Set`:

```js
const treeSet = new Set();

// Add items to the treeSet
treeSet.add("Baobab");
treeSet.add("Jackalberry");
treeSet.add("Mopane Tree");
treeSet.add("Breadfruit");

treeSet.delete("Breadfruit");

console.log(treeSet.size); // 3
```

Kao `Set`, takođe postoji komponenta za `WeakSet` koju možete koristiti da kreirate `WeakSet`:

```javascript
const treeWeakSet = new WeakSet();
```

`WeakSet` pia ina njia za `add()`, `delete()`, na `has()`:

```javascript
const treeWeakSet = new WeakSet();

const baobab = { name: "Baobab" };
const jackalberry = { name: "Jackalberry" };
const mopaneTree = { name: "Mopane Tree" };
const breadfruit = { name: "Breadfruit" };

treeWeakSet.add(baobab);
treeWeakSet.add(jackalberry);
treeWeakSet.add(mopaneTree);
treeWeakSet.add(breadfruit);

treeWeakSet.delete(jackalberry);
console.log(treeWeakSet.has(jackalberry)); // false

console.log(treeWeakSet);
```

U rezultatima, sadržaj `WeakSet` izgleda ovako:

```javascript
/*
WeakSet {{…}, {…}, {…}}
  [[Entries]]
    0: value: {name: "Mopane Tree"}
    1: value: {name: "Baobab"}
    2: value: {name: "Breadfruit"}
  [[Prototype]]: WeakSet
*/
```

Sadržaj je vidljiv ovde jer svojstva objekata i dalje imaju snažne reference, što te objekte drži u memoriji. Zapamtite da WeakSets nisu iterabilni, pa nema načina za iteriranje njihovih unosa ili pristupanje njima programski. Ono što vidite u rezultatima je samo pomoć za otklanjanje grešaka.

Ne zaboravite da se podržavaju samo simboli i stavke sa jasnim ključevima i vrednostima. Dodavanje osnovne stvari, kao što su brojevi ili niz slova, će izazvati grešku:

```js
treeWeakSet.add("Alan Smith");

console.log(treeWeakSet); // Invalid value used in weak set
//    at WeakSet.add (<anonymous>)
```

Glavna razlika između `Set` i `WeakSet` je u tome što `Set` čuva bilo koji tip vrednosti, dok `WeakSet` čuva samo objekte.

Evo drugih razlika koje su vidljive između `Set` i `WeakSet`:

| Atribut | Set | WeakSet |
| --- | --- | --- |
| Tipovi skladištenih vrednosti | Čuva bilo koji tip podataka | Čuva samo objekte |
| Reference | Snažna referenca | Slaba referenca |
| Iteracija | Pomaže za iteraciju preko `forEach` i petlji | Ne podržava iteraciju |
| Metode i svojstva | `add()`, `delete()`, `has()`, `keys()`, `values()`, `size`, i više | Samo `add()`, `delete()`, i `has()` |
| Upotreba | Kolekcija jedinstvenih vrednosti i uklanjanje iz niza | Efikasno praćenje memorije za reference na objekte |

Možete videti razlike u vrstama vrednosti koje ovi tipovi skupova mogu da skladište, njihovu pomoć u ponavljanju sačuvanih stavki i njihovu optimalnu upotrebu. Molimo vas da odvojite vreme za čitanje sadržaja ove tabele.
