# --description--

Mocking je proces zamene stvarnih podataka sa lažnim podacima koji imitiraju karakteristike stvarne komponente. Ovo omogućava testerima i programerima da testiraju određenu komponentu u aplikaciji bez oslanjanja na eksterne zavisnosti.

Uobiştan primer korišćenja mock podataka je kada rukujete pozivima ka API. Zamislite da imate poziv za fetch, kao što je ovo u vašoj aplikaciji:

```js
export const postService = {
  async getPosts() {
    const response = await fetch("https://api.example.com/posts");
    return response.json();
  },
};
```

Ako je potrebno testirati funkciju `postService`, možete koristiti maket (mock) za odgovor API umesto stalnih poziva API radi dobijanja podataka. Dobijanje podataka iz API zahteva vreme i zavisi od eksternih faktora kao što su dostupnost mreže i odgovori servera. Maketovanje (mocking) odgovora API eliminiše te eksterne faktore i omogućava programerima da rade sa predvidljivim i kontrolisanim setom podataka.

Ovde je primer simuliranog odgovora API:

```json
[
  {
    "id": 1,
    "title": "Understanding Async/Await in JavaScript",
    "content": "Async/Await makes asynchronous code look and behave like synchronous code...",
    "author": "Jane Doe",
    "created_at": "2025-04-10T14:32:00Z"
  },
  {
    "id": 2,
    "title": "10 Tips for Writing Clean Code",
    "content": "Clean code is not just about code formatting. It involves naming, architecture...",
    "author": "John Smith",
    "created_at": "2025-04-08T09:21:00Z"
  },
  {
    "id": 3,
    "title": "Exploring React 19 Features",
    "content": "React 19 comes with exciting features like new hooks and better performance...",
    "author": "Alex Lee",
    "created_at": "2025-04-07T18:47:00Z"
  }
]
```

Sada kada razumemo šta je *mocking*, uporedimo ga sa *stubbing*-om. Stubs su stvari koje vraćaju unapred programirane odgovore ili lažne podatke za ponašanje očekivano u aplikaciji. Na primer, možete koristiti stub za ponašanje konekcije sa bazom podataka u vašim testovima bez oslanjanja na stvarnu bazu podataka.

Ovde je primer stuba u Jestu:

```js
import { postService } from "./postService";
import { db } from "./db";

jest.mock("./db", () => ({
  db: {
    fetchPosts: jest.fn(),
  },
}));

test("should return stubbed posts", async () => {
  const fakePosts = [
    { id: 1, title: "Stubbed Post", content: "This is a stubbed post." },
  ];

  db.fetchPosts.mockResolvedValue(fakePosts);

  const posts = await postService.getPosts();

  expect(posts).toEqual(fakePosts);
});
```

`db.fetchPosts.mockResolvedValue(fakePosts);` je stub u ovom primeru.

Posledavni deo za diskusiju su *fakes*. Fakes su pojednostavljene verzije stvarnog dela bez složenosti ili nuspojava pravog objekta. Na primer, možete simulirati bazu podataka skladištima podataka u memoriji umesto interakcije sa stvarnom bazom podataka. Ovo će vam omogućiti da simulirate operacije baze podataka u memoriji, što će biti brže nego rad sa stvarnom bazom podataka. Drugi čest primer je korišćenje *fakes* kada radite sa sistemima fajlova. Možete kreirati lažni sistem fajlova umesto stvarnog tokom testiranja.

Kada nastavljate da gradite složeniju aplikaciju/program, razmislite o korišćenju mocks-a, stubs-a ili fakes-a gde je to prikladno u vašim testovima.
