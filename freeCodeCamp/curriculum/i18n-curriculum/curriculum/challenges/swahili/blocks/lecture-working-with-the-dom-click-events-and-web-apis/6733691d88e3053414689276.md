# --interactive--

U prethodnim lekcijama naučili ste kako da radite sa događajima koristeći metod `addEventListener()`. Ali postoji još jedan, nepreporučeni način rada sa događajima u JavaScript-u.

Inline event handler je poseban atribut HTML elementa koji se koristi za izvršavanje JavaScript koda kada se događaj dogodi.

Evo primera `button` elementa sa inline event handler-om za klik:

:::interactive_editor

```html
<button onclick="alert('Hello World!')">Show alert</button>
```

:::

Kada korisnik klikne dugme, poziva se funkcija `alert` i prikazuje se prozor sa porukom `Hello World!`.

Drugi način korišćenja inline event handler-a je pozivanje funkcije definisane unutar `script` oznake u HTML dokumentu.

Evo primera definisanja funkcije pod nazivom `changeBgColor` i njenog pozivanja iz inline event handler-a za klik:

:::interactive_editor

```html
<script>
  function changeBgColor() {
    document.body.style.backgroundColor = "lightblue";
  }
</script>

<button onclick="changeBgColor()">Change background color</button>
```

:::

Kada korisnik klikne na dugme, funkcija `changeBgColor` se poziva i pozadina stranice se menja u svetlo plavu boju. Iako je moguće koristiti rukovače događaja unutar linije kao što je ovo, to se ne smatra najboljom praksom.

Jedan razlog je što inline event handler može da poveže samo jedan event handler sa elementom. Ako želite da povežete više event handler-a sa jednim elementom, moraćete da koristite `addEventListener()`. Drugi razlog je što inline event handler meša HTML i JavaScript kod zajedno, što može otežati čitanje i održavanje koda. Bolje je držati HTML i JavaScript kod odvojeno koristeći `addEventListener()` za povezivanje događaja sa elementima.

Inline event handler-i se ne preporučuju za korišćenje u modernom JavaScript-u. Zbog toga je najbolja praksa da nastavite da koristite metod `addEventListener()` kada radite sa događajima u JavaScript-u.
