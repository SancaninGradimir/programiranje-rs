## TypeScript šta je to?

- **JavaScript**: JavaScript je jezik sa promenljivim tipovima tokom izvršavanja. To znači da parametri mogu primiti bilo koju vrednost prilikom pokretanja programa. Izazov jezika sa promenljivim tipovima je što nedostatak sigurnosti tipa može izazvati greške.

Na primer, čak i ako tvoja funkcija za JavaScript očekuje red podataka, i dalje ga možeš nazvati brojem:

```javascript
const getRandomValue = (array) => {
  return array[Math.floor(Math.random() * array.length)];
}

console.log(getRandomValue(10));
```

Rezultat ``console``, kao što je prikazano gore, biće ``undefined``.

- **TypeScript**: TypeScript dodaje podršku jeziku JavaScript uključujući nemenske tipove podataka (static typing), što pomaže u otkrivanju grešaka uzrokovanih neslaganjem tipova pre nego što pokrenete svoj kod.

Na primer, možete definisati tip parametra za `array` kao što sledi:

```typescript
const getRandomValue = (array: string[]) => {
  return array[Math.floor(Math.random() * array.length)];
}
```

Ova definicija tipa govori TypeScript da parametar `array` mora biti niz znakova. Zatim, kada navedete `getRandomValue` i date mu broj, dobijate grešku nazvanu greška kompilatora (compiler error).

- **Kompajler koda (Compiler)**: Prvo morate kompajlirati kod iz TypeScript u standardni JavaScript. Kada pokrenete kompajler, TypeScript će proveriti vaš kod i vratiti grešku za bilo kakve probleme gde tipovi ne odgovaraju.

## Tip podataka u TypeScript

- **Osnovni tipovi podataka u TypeScript**: Za osnovne tipove podataka `string`, `null`, `undefined`, `number`, `boolean`, i `bigint`, TypeScript generiše ključne reči koje odgovaraju ovim tipovima.

```typescript
let str: string = "Naomi";
let num: number = 42;
let bool: boolean = true;
let nope: null = null;
let nada: undefined = undefined;
```

- **Redovi podataka**: Moguće je definisati redicu specifičnog tipa sa dve različite sintakse.

```typescript
const arrOne: string[] = ["Naomi"];
const arrTwo: Array<string> = ["Camperchan"];
```

- **Vitu (Objects)**: Možete definisati stvarni strukturu objekta.

```typescript
const obj: { a: string, b: number } = { a: "Naomi", b: 42 };
```

Ako želiš nešto sa bilo kojim ključevima, ali gde sve vrednosti moraju biti sekvenca slova, postoje dva načina definisanja:

```typescript
const objOne: Record<string, string> = {};
const objTwo: { [key: string]: string } = {};
```

- **Još jedan važan tip u TypeScript**:
  - **`any`**: `any` pokazuje da vrednost može imati bilo koji tip. Govori kompajleru koda da se ne brine o tipu tog parametra.
  - **`unknown`**: `unknown` govori TypeScript da ti je stalo do tipa vrednosti, ali ne znaš tačno koji. `unknown` se obično preporučuje više od `any`.
  - **`void`**: Ovo je specijalni tip koji obično koristiš samo kada definišeš funkcije. Funkcije koje ne vraćaju vrednost koriste tip return od `void`.
  - **`never`**: Predstavlja tip koji nikada neće postojati.

- **Ključna reč za `type`**: Ova ključna reč je kao `const`, ali umesto deklarisanja parametra, možete deklarisati tip.

Važno je deklarisati specifične tipove kao što su kombinacioni tipovi ili tipovi koji sadrže samo određenu vrednost:

```typescript
type stringOrNumber = string | number;
type bot = "camperchan" | "camperbot" | "naomi";
```

- **`interface`**: Interfejsi su kao klase tipova. Mogu da implementiraju ili prošire druge interfejse; oni su vrste stvari po sebi i obično se preporučuju osim ako vam nije potrebna posebna funkcija koju pruža deklaracija `type`.

```typescript
interface wowie {
  zowie: boolean;
  method: () => void;
}
```

- **Definisati tip return**: Pia može da definiše tip _return_ funkcije.

Primer ispod objašnjava vrednost return kao niz znakova. Ako pokušate da vratite nešto drugo, TypeScript će vam dati grešku kompilatora.

```typescript
const getRandomValue = (array: string[]): string => {
  return array[Math.floor(Math.random() * array.length)];
}
```

## Generici

- **Definisanje generičkog tipa**: Možete definisati generički tip i navesti ga u vašoj funkciji. To se može smatrati kao specifičan parametar koji pružate funkciji kako biste kontrolisali ponašanje definicije tipa te funkcije.

Evo primera definisanja generičkog tipa za funkciju:

```typescript
const getRandomValue = <T>(array: T[]): T => {
  return array[Math.floor(Math.random() * array.length)];
}
const val = getRandomValue([1, 2, 4])
```

Sintaksia ya `<T>` mi govori TypeScript da definišete generički tip `T` za funkciju. `T` je uobičajeno ime za generičke tipove, ali možete koristiti bilo koje ime.

Zatim, govoriš TypeScript da je parametar za `array` niz vrednosti koje odgovaraju generičkom tipu, i da je vraćena vrednost element tog istog tipa.

- **Navođenje tipskog argumenta u pozivu funkcije**: Možete proslediti tipski argument za poziv funkcije korišćenjem uglastih zagrada između imena funkcije i njenih parametara.

Ovde je primer dodeljivanja tipa argumenta za poziv funkcije:

```typescript
const email = document.querySelector<HTMLInputElement>("#email");
console.log(email.value);
```

Ovo obaveštava TypeScript da će funkcija koju očekujete biti polje za unos.

## Sužavanje tipa (Type narrowing)

- **Stvarni pad**: U ovom primeru, dobijate grešku kompilatora koda kada pokušavate da dođete do svojstva `value` od `email` zato što `email` _može_ biti `null`.

```typescript
const email = document.querySelector<HTMLInputElement>("#email");
console.log(email.value);
```

Možete koristiti uslovnu izjavu da potvrdite `email` je _tačno_ pre dobijanja te karakteristike:

```typescript
const email = document.querySelector<HTMLInputElement>("#email");
if (email) {
  console.log(email.value);
}
```

Provera realnosti takođe može da funkcioniše u suprotnom smeru:

```typescript
const email = document.querySelector<HTMLInputElement>("#email");
if (!email) {
  throw new ReferenceError("Could not find email element!")
}
console.log(email.value);
```

Izbacivanje greške prekida izvršavanje logike ovog koda, što znači da kada dođe do poziva `console.log()`, TypeScript zna `email` ne može biti `null`.

- **Opcionalno lančanje (Optional chaining)**: Opcionalno lančanje ``?.`` je takođe vrsta sužavanja tipa, na osnovu pretpostavke da pristup svojstvu ne može doći ako vrednost ``email`` nije ``null``.

```typescript
const email = document.querySelector<HTMLInputElement>("#email");
console.log(email?.value);
```

- **Funkcija `typeof`**: Možete koristiti deklarativnu naredbu da proverite tip parametra koristeći funkciju `typeof`.

```typescript
const myVal = Math.random() > 0.5 ? 222 : "222";
if (typeof myVal === "number") {
  console.log(myVal / 10);
}
```

- **Ključna reč za `instanceof`**: Ako nešto dolazi iz klase objekata, možete koristiti ključnu reč `instanceof` da smanjite tip.

```typescript
const email = document.querySelector("#email");

if (email instanceof HTMLInputElement) {
    console.log(email.value);
}
```

- **Izlaska tipizacija (Casting)**: Kada `TypeScript` ne može direktno da utvrdi tip vrednosti, kao rezultat metode ``request.json()`` u primeru ispod, dobićete grešku kompajlera koda. Jedan način za rešavanje ovoga je izlaska tipizacija (casting), ali to smanjuje sposobnost `TypeScript` da detektuje potencijalne greške.

```typescript
interface User {
    name: string;
    age: number;
}

const printAge = (user: User) => 
  console.log(`${user.name} is ${user.age} years old!`)

const request = await fetch("url")
const myUser = await request.json() as User;
printAge(myUser);
```

- **Mlinzi wa aina (Type guard)**: Umesto prisile tipizacije, možete napisati type guard:

```typescript
interface User {
    name: string;
    age: number;
}

const isValidUser = (user: unknown): user is User => {
  return !!user && 
    typeof user === "object" &&
    "name" in user &&
    "age" in user;
}
```

Sintaksa `user is User` pokazuje da vaša funkcija vraća boolean vrednost, što znači da, ako je tačno, vrednost `user` zadovoljava interfejs `User`.

## Fajlovi za `tsconfig`

- **`tsconfig.json`**: Podešavanja agregatora za TypeScript se nalaze u fajlu `tsconfig.json` u glavnom direktorijumu vašeg projekta.

```json
{
  "compilerOptions": {
    "rootDir": "./src",
    "outDir": "./prod",
    "lib": ["ES2023"],
    "target": "ES2023",
    "module": "ES2022",
    "moduleResolution": "Node",
    "esModuleInterop": true,
    "skipLibCheck": true,
    "strict": true
  },
  "exclude": ["test/"]
}
```

Ovdje su objašnjenja opcija kolektora koje su korištene u gornjem primjeru:

- **`compilerOptions`**: Svojstvo `compilerOptions` je ono što kontroliše kako bundler TypeScript radi.
- **`rootDir` na `outDir`**: `rootDir` na `outDir` opisuju direktorijum koji sadrži vaše izvorni fajlove i direktorijum koji bi trebalo da sadrži prevedeni kod JavaScript.
- **`lib`**: Svojstvo `lib` određuje koji će tip definicije koristiti bundler, i omogućava vam da uključite podršku za specifične verzije ES, DOM, i druge.
- **`module` na `moduleResolution`**: `module` na `moduleResolution` zajedno rade da kontrolišu kako vaš paket koristi module – bilo da je reč o CommonJS ili ECMAScript-u.
- **`esModuleInterop`**: `esModuleInterop` omogućava odličnu kompatibilnost između CommonJS i ES modula tako što automatski kreira namespace objekte za uvoz (imports).
- **`skipLibCheck`**: Opcija `skipLibCheck` preskače proveru fajlova `.d.ts` koji nisu pomenuti u importima vašeg koda.
- **`strict`**: Flag `strict` omogućava nekoliko provera, kao što je osiguravanje pravilne obrade tipova koji mogu imati vrednost null i emitovanje upozorenja kada TypeScript vraća na `any`.
- **`exclude`**: Svojstvo na visokom nivou `exclude` govori bundleru da ignoriše ove fajlove TypeScript tokom bundleovanja.
