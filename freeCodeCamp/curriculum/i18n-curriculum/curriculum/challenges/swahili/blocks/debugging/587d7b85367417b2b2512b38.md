# --description--
Programi koji koriste grane, to jest oni koji rade različite stvari ako su ispunjeni određeni uslovi, zavise od deklaracija za `if`, `else if`, i `else` u JavaScript. Ti uslovi ponekad proveravaju da li je rezultat jednak određenoj vrednosti.

Ova logika se pominje (barem na engleskom) kao "if x equals y, then ..." što se brzo može prevesti u kod koristeći `=`, ili operator uslova. Ovo uzrokuje neočekivani tok kontrole u vašem programu.

Kao što je diskutovano u prethodnim izazovima, operater dodeljivanja (`=`) u JavaScript postavlja vrednost za ime varijable. A operateri `==` i `===` proveravaju jednakost (trostruki operator `===` proverava strogu jednakost, što znači da su jednaki i vrednost i tip).

Kod ispod postavlja `x` na 2, što se procenjuje kao `true`. Gotovo svaka vrednost sama po sebi u JavaScript se procenjuje kao `true`, osim onih poznatih kao 'falsy' vrednosti: `false`, `0`, `""` (prazan niz znakova), `NaN`, `undefined` i `null`.

```js
let x = 1;
let y = 2;
if (x = y) {

} else {

}
```

U ovom primeru, deo koda unutar deklaracije `if` će se izvršiti sa bilo kojom vrednošću od `y`, osim ako `y` nije falsy. Deo `else`, koji očekujemo da se izvrši ovde, neće zaista funkcionisati.
