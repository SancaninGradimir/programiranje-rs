# --interactive--

Hajde da naučimo nešto o kvantifikatorima u regularnim izrazima.

Zamislite da želite da pronađete identifikacioni broj od četiri cifre. Znate da možete koristiti klasu karaktera `\d`, pa biste je mogli napisati četiri puta. Da biste sprečili dodatne karaktere, koristićete sidra za početak i kraj stringa:

```js
const regex = /^\d\d\d\d$/;
```

Ovo radi – poreći će četiri slova broja. Ali umesto da pišete tu klasu često, možete koristiti prečicu.

Kvantifikatori se pišu u vitičastim zagradama sa jednim ili dva broja. Pogledajmo kvantifikator u našem obrascu:

```js
const regex = /^\d{4}$/;
```

Obratite pažnju da naš kvantifikator sadrži samo broj `4`. Ova sintaksa znači „pronađi prethodni karakter tačno četiri puta“. Pogledajmo kako radi:

:::interactive_editor

```js
const regex = /^\d{4}$/;
console.log(regex.test("123")); // false
console.log(regex.test("1234")); // true
console.log(regex.test("12345")); // false
console.log(regex.test("123456")); // false
console.log(regex.test("1234567")); // false
```

:::

Ovaj obrazac odgovara samo stringovima sa četiri cifre, zato što koristimo sidra i kvantifikator dozvoljava tačno četiri cifre. Ali možda identifikacioni broj treba da ima najmanje četiri cifre.

Da biste dozvolili četiri ili više cifara, dodajte zarez posle broja u kvantifikatoru:

```js
const regex = /^\d{4,}$/;
```

Sada naš obrazac dozvoljava četiri ili više cifara. Hajde da ga isprobamo:

:::interactive_editor

```js
const regex = /^\d{4,}$/;
console.log(regex.test("123")); // false
console.log(regex.test("1234")); // true
console.log(regex.test("12345")); // true
console.log(regex.test("123456")); // true
console.log(regex.test("1234567")); // true
```

:::

Identifikacioni broj sa sedam cifara je predugačak. Treba da ima najmanje četiri, a najviše šest cifara. To možete postići dodavanjem drugog broja u kvantifikator posle zareza:

```js
const regex = /^\d{4,6}$/;
```

Sada naš obrazac neće odgovarati identifikacionom broju sa sedam cifara, zato što prelazi maksimalnih šest cifara.

:::interactive_editor

```js
const regex = /^\d{4,6}$/;
console.log(regex.test("123")); // false
console.log(regex.test("1234")); // true
console.log(regex.test("12345")); // true
console.log(regex.test("123456")); // true
console.log(regex.test("1234567")); // false
```

:::

Zapamti da ne možeš koristiti ovu sintaksu samo za maksimalnu vrednost – uvek moraš postaviti minimalnu vrednost. Ali ako odvojiš minimalnu vrednost kao `1`, možeš postići isti rezultat.

Primili smo ažurirane zahteve od naših korisnika. Identiatori sada mogu opciono da počinju slovom. Već znamo klasu ponašanja za ovo, pa hajde da je dodamo u naš standardni opis:

```js
const regex = /^[a-zA-Z]\d{4,6}$/;
```

Ali sada prisiljavamo postojanje slova. Kako ga možemo učiniti opcionim?

Možete koristiti sintaksu regularnog izraza sa `0` kao minimalnu i `1` kao maksimalnu:

```js
const regex = /^[a-zA-Z]{0,1}\d{4,6}$/;
```

Međutim, zaista postoji poseban kratak prečica za jedan specifičan karakter – znak pitanja (`?`). Uzmićemo našu prečicu i promeniti je u znak pitanja:

```js
const regex = /^[a-zA-Z]?\d{4,6}$/;
```

Trebalo bi nam da potvrdimo rezultate:

:::interactive_editor

```js
const regex = /^[a-zA-Z]?\d{4,6}$/;
console.log(regex.test("123")); // false
console.log(regex.test("a1234")); // true
console.log(regex.test("12345")); // true
console.log(regex.test("az12345")); // false
console.log(regex.test("X123456")); // true
console.log(regex.test("1234567")); // false
```

:::

Naš format sada dozvoljava jedan opcijski karakter, praćen četiri do šest cifara.

Nažalost, otkrili smo da smo pogrešno razumeli zahteve. Moramo dozvoliti bilo koje slovo, bilo u kojoj količini, pre broja. Možemo koristiti regularni izraz sa minimalnom vrednošću `0` i bez postavljene maksimalne vrednosti:

```js
const regex = /^[a-zA-Z]{0,}\d{4,6}$/;
```

Ali naš obrazac ponovo postaje dug. Srećom, postoji kraći zapis za „pronađi prethodni karakter nula ili više puta“ — zvezdica (`*`). Zamenimo naš kvantifikator zvezdicom i isprobajmo obrazac:

:::interactive_editor

```js
const regex = /^[a-zA-Z]*\d{4,6}$/;
console.log(regex.test("123")); // false
console.log(regex.test("a1234")); // true
console.log(regex.test("12345")); // true
console.log(regex.test("az12345")); // true
console.log(regex.test("X123456")); // true
console.log(regex.test("1234567")); // false
```

:::

Trenutno uspešno upoređujemo bilo koji identifikator sa nulom ili više slova, praćenih četiri do šest brojeva. Međutim, čini se da ovo uzrokuje pad našeg sistema – zapravo bi trebalo da zahtevamo barem jedno slovo.

Ponovo, možemo koristiti kvantifikator sa minimalnim nivoom od 1 i bez postavljenog maksimalnog nivoa, ili možemo koristiti drugu posebnu sintaksu – znak plus (`+`):

:::interactive_editor

```js
const regex = /^[a-zA-Z]+\d{4,6}$/;
console.log(regex.test("123")); // false
console.log(regex.test("a1234")); // true
console.log(regex.test("12345")); // false
console.log(regex.test("az12345")); // true
console.log(regex.test("X123456")); // true
console.log(regex.test("1234567")); // false
```

:::

Sada neće proći identifikacioni brojevi koji ne počinju sa najmanje jednim slovom, bez obzira na broj cifara.

Korišćenjem kvantifikatora možete značajno poboljšati sažetost i čitljivost svojih regularnih izraza.
