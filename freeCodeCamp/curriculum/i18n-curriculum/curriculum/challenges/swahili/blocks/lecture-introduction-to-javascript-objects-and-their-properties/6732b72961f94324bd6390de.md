# --interactive--

U JavaScript, postoji nekoliko načina za proveru da li nešto ima određena svojstva. Razumevanje ovih metoda je važno za efikasno rad sa objektima, posebno kada obrađujete podatke iz eksternih izvora ili kada morate osigurati da određena svojstva postoje pre nego što ih koristite.

Pregledaće neke uobičajene metode: `hasOwnProperty()`, `Object.hasOwn()`, operater za `in`, i proveru protiv `undefined`.

Početi sa metodom `hasOwnProperty()`. Ova metoda vraća vrednost tačno ili netačno, ukazujući da li objekat poseduje navedeno svojstvo kao svoje. Evo primera:

:::interactive_editor

```js
const person = {
  name: "Alice",
  age: 30
};

console.log(person.hasOwnProperty("name")); // true
console.log(person.hasOwnProperty("job")); // false
```

:::

U ovom primeru imamo objekat nazvan `person` sa dva svojstva: `name` i `age`. Da bismo proverili da li je `name` svojstvo objekta `person`, koristimo metodu `hasOwnProperty()`. Pošto je `name` svojstvo, metoda će vratiti `true`. Ali kada koristimo `hasOwnProperty()` da proverimo da li je `job` svojstvo, vratiće `false` jer ne postoji u tom objektu.

`Object.hasOwn()` je moderan i preporučen način za proveru da li objekat ima svojstvo kao svoje sopstveno svojstvo (a ne nasleđeno). Možete ga zamisliti kao poboljšanu i sigurniju verziju metode `hasOwnProperty()`. Sintaksa je `Object.hasOwn(object, propertyName)` — prosleđujete objekat kao prvi argument, a naziv svojstva kao drugi.

Ovo je osnovni primer:

:::interactive_editor

```js
const person = {
  name: "Alice",
  age: 30
};

console.log(Object.hasOwn(person, "name")); // true
console.log(Object.hasOwn(person, "job")); // false
```

:::

U ovom primeru, `Object.hasOwn(person, "name")` vraća `true` zato što `name` postoji direktno u objektu `person`. `Object.hasOwn(person, "job")` vraća `false` zato što `job` nikada nije postavljen u tom objektu.

Veoma važna stvar koju treba razumeti jeste da `Object.hasOwn()` proverava samo da li svojstvo **postoji** — nije važno koja je vrednost tog svojstva. To znači da i dalje vraća `true` čak i ako je vrednost `0`, `false`, `null` ili `undefined`:

:::interactive_editor

```js
const user = {
  username: "coder123",
  score: 0,
  isActive: false,
  nickname: null
};

// Object.hasOwn() correctly reports these all exist
console.log(Object.hasOwn(user, "score"));    // true  (value is 0, but property exists)
console.log(Object.hasOwn(user, "isActive")); // true  (value is false, but property exists)
console.log(Object.hasOwn(user, "nickname")); // true  (value is null, but property exists)
console.log(Object.hasOwn(user, "email"));   // false (property was never added)

// Danger! Using if() directly gives wrong results for falsy values
if (user.score) {
  console.log("Has score"); // This will NOT print even though score exists!
}

// Safe! Object.hasOwn() gives correct result
if (Object.hasOwn(user, "score")) {
  console.log("Has score:", user.score); // Has score: 0
}
```

:::

Drugi način za proveru postojanja svojstva u objektu jeste korišćenje operatora `in`. Kao i `hasOwnProperty()`, operator `in` vraća `true` ako svojstvo postoji u objektu. Evo kako se koristi:

:::interactive_editor

```js
const person = {
  name: "Bob",
  age: 25
};
console.log("name" in person);  // true
```

:::

U ovom primeru, `"name" in person` vraća `true` zato što je `name` svojstvo objekta `person`.

Treći način je provera da li je svojstvo `undefined`. Ovaj način može biti koristan, ali ima određena ograničenja. Evo primera:

:::interactive_editor

```js
const car = {
  brand: "Toyota",
  model: "Corolla",
  year: 2020
};

console.log(car.brand !== undefined); // true
console.log(car.color !== undefined); // false
```

:::

U ovom kodu proveravamo da li `car.brand` i `car.color` nisu `undefined`. Ovo funkcioniše zato što pristup nepostojećem svojstvu objekta vraća `undefined`. Međutim, ovaj način može dati netačne rezultate ako svojstvo ima eksplicitnu vrednost `undefined`.

U praksi, izbor između ovih metoda često zavisi od specifičnih zahteva vašeg koda. Razumevanje razlika među njima pomoći će vam da napravite pravi izbor u različitim situacijama.
