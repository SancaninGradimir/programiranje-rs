# --description--

Pored GET-a, postoji još jedna uobičajena HTTP metoda, POST. POST je osnovna metoda koja se koristi za slanje podataka od klijenta kroz HTML forme. Prema REST konvenciji, POST se koristi za slanje podataka ili kreiranje novih entiteta u bazi podataka (novi korisnik, ili novi blog post). Nemate bazu podataka u ovom projektu, ali ipak ćete naučiti kako da rukujete POST zahtevima.

U ovim tipovima zahteva, podaci nisu vidljivi u URL-u, već su sakriveni u telu zahteva. Telo je deo HTTP zahteva, koji se takođe naziva opterećenjem. Iako podaci nisu vidljivi u URL-u, to ne znači da su privatni. Da biste videli zašto, pogledajte sirove sadržaje HTTP POST zahteva:

```http
POST /path/subpath HTTP/1.0
From: john@example.com
User-Agent: someBrowser/1.0
Content-Type: application/x-www-form-urlencoded
Content-Length: 20

name=John+Doe&age=25
```

Kao što vidite, telo je napisano kao query string. Ovo je osnovni format koji koriste HTML forme. Za Ajax, takođe možete koristiti JSON za rukovanje složenijim podacima. Postoji i drugi tip enkodiranja: multipart/form-data. On se koristi za slanje binarnih fajlova. U ovom zadatku, koristićete telo koje je URL encoded. Da biste parsirali podatke koji dolaze iz POST zahteva, morate koristiti `body-parser` paket. Ovaj paket vam omogućava da koristite niz middleware funkcija, koje mogu da parsiraju podatke u različitim formatima.

# --instructions--

`body-parser` je već instaliran i nalazi se u `package.json` fajlu vašeg projekta. `require`-ujte ga na vrhu `myApp.js` fajla i sačuvajte u promenljivu pod nazivom `bodyParser`. Middleware za rukovanje URL encoded podacima se dobija pozivom `bodyParser.urlencoded({extended: false})`. Prosledite funkciju koju vraća prethodni poziv u `app.use()`. Kao i obično, middleware mora biti postavljen pre svih ruta koje zavise od njega.

**Napomena:** `extended` je podešavanje koje definiše koji parser treba da koristi `body-parser`. Kada se koristi `extended=false`, koristi se standardni parser iz `querystring` biblioteke. Kada se koristi `extended=true`, koristi se `qs` biblioteka za parsiranje.

Kada koristite `extended=false`, vrednosti mogu biti samo stringovi ili nizovi podataka. Objekat koji se vraća pri korišćenju `querystring` ne nasleđuje direktno iz JavaScript `Object`, što znači da metode kao što su `hasOwnProperty`, `toString` neće biti dostupne. Proširena verzija dozvoljava fleksibilnije parsiranje podataka, ali nije kompatibilna sa JSON.
