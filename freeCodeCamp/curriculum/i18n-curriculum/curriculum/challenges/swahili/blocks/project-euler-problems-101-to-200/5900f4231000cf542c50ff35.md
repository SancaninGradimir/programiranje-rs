# --description--

RSA šifrovanje zavisi od sljedećih procedura:

Kreiraj dva različita bazna broja `p` i `q`.
Izračunaj `n=p*q` i `φ=(p-1)(q-1)`.
Pronađi kompletan broj `e`, `1 < e < φ`, na način da `gcd(e,φ) = 1`

Poruka u ovom sistemu je broj u opsegu `[0,n-1]`.
Šifrovirani tekst se zatim na neki način pretvara u poruku (broj u opsegu `[0,n-1]`).
Za šifrovanje teksta, za svaku poruku, `m`, c=m<sup>e</sup> mod n se računa.

Da bi se poruka dešifrovala, potrebno je izračunati `d` tako da važi `ed=1 mod φ`, a zatim za svaku šifrovanu poruku `c` izračunati m=c<sup>d</sup> mod n.

Postoje vrednosti `e` i `m` takve da m<sup>e</sup> mod n = m.
Nazivamo poruku `m` koja je m<sup>e</sup> mod n=m nezaštićena poruka.

Problem nastaje prilikom izbora `e` jer ne bi trebalo da ima previše vidljivih poruka.
Na primer, uzmite `p=19` i `q=37`.
Zatim `n=19*37=703` i `φ=18*36=648`.
Ako izaberemo `e=181`, onda, iako `gcd(181,648)=1` ukazuje da su sve moguće poruke m `(0≤m≤n-1)` vidljive prilikom računanja m<sup>e</sup> mod n.
Za bilo koji validan izbor od `e` postoji određena vidljiva poruka.
Važno je da broj vidljivih poruka bude na najnižem nivou.

Na osnovu `p` i `q` koji su dati, pronađi ukupan zbir svih vrednosti za `e`, `1 < e < φ(p,q)` i `gcd(e,φ)=1`, ili da broj skrivnih poruka za ovu vrednost od `e` bude na nivou najmanje.

# --solutions--

```js
function gcd(a, b) {
    if (b)
        return gcd(b, a % b);
    else
        return a;
}

function RSAEncryption(p, q) {
    let phi = (p - 1) * (q - 1);

    let best = Number.MAX_SAFE_INTEGER;
    let sum = 0;

    for (let e = 0; e < phi; ++e) {
        if (!(gcd(e, phi) == 1))
            continue;

        let msg = (gcd(p - 1, e - 1) + 1) * (gcd(q - 1, e - 1) + 1);

        if (best == msg) {
            sum += e;
        } else if (best > msg) {
            best = msg;
            sum = e;
        }
    }

    return sum;
}
```
