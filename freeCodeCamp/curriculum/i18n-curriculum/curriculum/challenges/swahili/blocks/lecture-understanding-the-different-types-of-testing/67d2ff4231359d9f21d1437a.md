# --description--

End-to-end testiranje, ili E2E skraćeno, meri stvarne scenarije upotrebe iz perspektive korisnika. Primeri E2E testnih scenarija uključuju testiranje procesa registracije i prijave ili testiranje plaćanja za sajt e-trgovine.

Da bismo bolje razumeli kako funkcioniše end-to-end testiranje, uzmimo stvarni primer iz koda freeCodeCamp-a. Ovaj primer koristi Playwright, popularni radni okvir za end-to-end testiranje koji je razvila kompanija Microsoft.

U kodu freeCodeCamp-a postoji direktorijum pod nazivom `e2e` koji sadrži testove za donacije, sertifikate, ispite i još mnogo toga. Unutar tog direktorijuma nalazi se datoteka `donate-page-donor.spec.ts` koja sadrži testove za stranicu donatora sa potvrđenim nalogom.

Prvi korak je navigacija na stranicu doprinosa kao test korisnik koji je već administrator:

```js
test.beforeEach(async ({ page }) => {
  execSync("node ./tools/scripts/seed/seed-demo-user --set-true isDonating");
  await page.goto("/donate");
});
```

Pošto postoji mnogo testova u ovoj datoteci, `beforeEach` hook će se izvršavati pre svakog testa.

Kada se korisnik nađe na stranici za donacije, videće poruku zahvalnosti zajedno sa predlozima kako da nastavi da podržava freeCodeCamp. Evo nekoliko testova za proveru određenog teksta koji postoji na stranici za donacije:

```js
test("should render the donate page correctly", async ({ page }) => {
  await expect(
    page.getByText("Thank you for your continued support")
  ).toBeVisible();

  await expect(
    page.getByText(
      "Your contributions are crucial in creating resources that empower millions of people to learn new skills and support their families."
    )
  ).toBeVisible();

  ...
});
```

Takođe postoje testovi koji proveravaju da li donator ima donatorsku vezu u meniju za navigaciju, kao i posebno ukrašen okvir oko njihove slike koji pokazuje da su donatori:

```js
test("The menu should have a supporters link", async ({ page }) => {
  const menuButton = page.getByTestId("header-menu-button");
  const menu = page.getByTestId("header-menu");

  await expect(menuButton).toBeVisible();
  await menuButton.click();

  await expect(menu).toBeVisible();

  await expect(page.getByRole("link", { name: "Supporters" })).toBeVisible();
});

test("The Avatar should have a special border for donors", async ({ page }) => {
  const container = page.locator(".avatar-container");
  await expect(container).toHaveClass("avatar-container gold-border");
});
```

Posedovanjem ovako detaljnih testova možete povećati pokrivenost testovima za svoju aplikaciju više nego samo testovima jedinica. End-to-end testovi takođe pomažu da se osigura da vaša aplikacija pravilno radi i da je pouzdana za korisnike. Drugi primeri alata za end-to-end testiranje uključuju Cypress, Selenium i Puppeteer.

Iako je end-to-end testiranje važno, ono je skupo jer zahteva mnogo vremena za pripremu, dizajn i održavanje. E2E testovi su najbolji za važne tokove korisnika, dok su testovi jedinica bolji za testiranje manjih delova aplikacije.
