## Čista funkcija protiv nečiste

- Čista funkcija je ona koja daje iste rezultate svaki put za isti ulaz i nema bočnih efekata. Njen rezultat zavisi samo od svog ulaza i ne menja nikakvo spoljašnje stanje.
- Nečiste funkcije imaju bočne efekte, koji su promene stanja programa koje se vide izvan funkcije.

## Performanse programa

- Funkcionalno programiranje je metod razvoja softvera koji naglašava korišćenje funkcija za rešavanje problema, fokusirajući se na ono što treba da se uradi umesto na tome kako to postići.
- Funkcionalno programiranje podstiče korišćenje metoda koje pomažu u izbegavanju nuspojava (side effects), kao što je upotreba nemenjivih struktura podataka i funkcija višeg reda.
- Ako se koristi ispravno, principi funkcionalnog programiranja dovode do čistijeg i lakše održivog koda.

## Kuringa

- Kuriranje je tehnika funkcionalnog programiranja koja pretvara funkciju sa višestrukim argumentima u niz funkcija, pri čemu svaka funkcija prima samo jedan argument.

Ovde je primer standardne funkcije naspram obuhvaćene funkcije:

:::interactive_editor

```js
// Regular function
function average(a, b, c) {
  return (a + b + c) / 3;
}

console.log(average(2, 3, 4)); // 3

// Curried function
function curriedAverage(a) {
  return function(b) {
    return function(c) {
      return (a + b + c) / 3;
    };
  };
}

console.log(curriedAverage(2)(3)(4)); // 3
```

:::

- Kuriranje može biti moćno, posebno kada koristite funkcije koje primaju mnogo argumenata.
- Kuriranje čini vaš kod lakšim za održavanje i ponovno korišćenje.
- Možete koristiti strelične funkcije da kreirate kurirane funkcije sa manje koda:

```js
const curriedAverage = a => b => c => (a + b + c) / 3;
```

- Iako [kuringa] može učiniti kod lakšim za izmenu i ponovnu upotrebu, takođe može učiniti da je kod težak za čitanje ako se preterano koristi.
