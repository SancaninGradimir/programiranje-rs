# --description--

Kada radiš sa CSS, možeš naići na određene izazove, kao što je održavanje stilova stabilnim i konzistentnim u različitim pretraživačima, izbegavanje ponavljanja, i rešavanje sve veće složenosti zbog broja fajlova CSS u tvom projektu. Radni okviri za CSS su velika pomoć u sprečavanju ovih problema od samog početka.

Sistem CSS uključuje prednapisan kod CSS koji možete koristiti za stilizovanje vaših komponenti HTML.

Korišćenjem radnog sistema CSS, možete ubrzati svoj radni tok, kreirati dosledan vizuelni stil na celoj veb stranici, učiniti da vaš dizajn izgleda stabilno u više pregledača i generalno održavati svoj kod CSS u boljem redosledu.

Možete zamisliti funkcionalni sistem CSS kao kutiju alata, gde možete brzo pronaći stil ili komponentu koja zadovoljava vaše potrebe i odmah je koristiti, bez potrebe da ga sami definišete koristeći svojstva CSS. Ovi sistemi često uključuju unapred napisane stilove i komponente koje možete vrlo lako koristiti, uključujući stilove za kreiranje podešavanja koja se prilagođavaju uređaju.

Neki od popularnih okvira za CSS su Tailwind CSS, Bootstrap, Materialize i Foundation.

Razgovaraćemo malo o dve vrste sisteme rada za CSS: sistem rada CSS koji zavisi od korišćenja malih parametara (utility-first), i sistem rada CSS koji zavisi od komponenti (component-based).

Tailwind CSS je utilitetski sistem CSS koji se oslanja na korišćenje malih parametara. To znači da ima male klase sa veoma specifičnim namenama, kao što su postavljanje ivice, unutrašnjeg razmaka ili pozadinske boje. Obično možete primeniti ove male klase direktno na elemente HTML.

Na primer:

```html
<button class="bg-blue-500 text-white font-bold py-2 px-4 rounded-full hover:bg-blue-700">
  Button
</button>
```

Ovaj dugme će imati plavu pozadinsku boju i beli podebljani tekst, vertikalni unutrašnji razmak od 2 rem, horizontalni unutrašnji razmak od 4 rem, i zaobljene ivice. Klasa `hover:bg-blue-700` će učiniti pozadinsku boju tamno plavom kada korisnik pređe mišem preko njega.

Kao što vidite, Tailwind klase CSS su veoma male, pa se mogu koristiti za pojedinačne komponente HTML kako je potrebno.

Međutim, Bootstrap je framework CSS koji se oslanja na komponente. Ima već gotove sekcije sa unapred definisanim stilovima koje možete lako dodati na vaš sajt. Ove komponente se obično nalaze u zvaničnoj dokumentaciji framework-a CSS, gde ih možete brzo kopirati i zalepiti u svoj projekat.

Na primer, ako želite kreirati grupu stavki u Bootstrapu, napisali biste nešto ovako u HTML:

```html
<div class="card" style="width: 25rem;">
  <ul class="list-group list-group-flush">
    <li class="list-group-item">HTML</li>
    <li class="list-group-item">CSS</li>
    <li class="list-group-item">JavaScript</li>
  </ul>
</div>
```

Ove prethodne klase će definisati standardne stilove za kartice, grupe stavki i pojedinačne elemente grupa stavki direktno.

Znamite da ovde, umesto dodavanja malih i opštih klasa, dodajemo ceo deo, uključujući strukturu HTML. Zato je Bootstrap komponentno zasnovan okvir.

```html
<div class="card" style="width: 25rem;">
  <ul class="list-group list-group-flush">
    <li class="list-group-item">HTML</li>
    <li class="list-group-item">CSS</li>
    <li class="list-group-item">JavaScript</li>
  </ul>
</div>
```

Iako radni sistemi CSS imaju mnogo prednosti, imaju i neke nedostatke. Važno je znati oba kako bi se donela ispravna odluka.

Pošto sistemi za rad CSS zavise od prethodno definisanih stilova, ako pokušate da dodate specifičan CSS preko njih, mogući su problemi sa sposobnošću selektora. Selektori koje koristi sistem za rad mogu imati veću sposobnost od selektora koje koristite u vašem specifičnom CSS, pa stoga ne može biti prikazan ispravan stil. Da biste ovo rešili, biće vam potrebno da koristite CSS sa većom sposobnošću, što može predstavljati dodatne izazove.

Elementi mogu smanjiti jedinstvenost u određenoj meri jer se njihovi uobičajeni stilovi koriste na mnogo sajtova. Međutim, ovo se može rešiti zamenom CSS sa vašim jedinstvenim stilovima.

Konačno, treba da razmotrite probleme sa performansama koje sistemi rada CSS mogu izazvati. Neki sistemi su veoma veliki, i može potrajati za učitavanje.

Kao programer, ćeš odlučiti da li je korišćenje funkcionog sistema CSS korisno za tvoj projekat, da li je upotreba potpuno specifičnog CSS bolja, ili možda kombinacija oba.
