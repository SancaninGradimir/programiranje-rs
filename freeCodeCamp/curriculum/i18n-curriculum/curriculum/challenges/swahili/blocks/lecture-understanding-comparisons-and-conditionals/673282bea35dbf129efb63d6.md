# --interactive--

`switch` statements na `if/else if/else` chains su sve strukture kontrole protoka u programu koje omogućavaju izvršavanje različitih delova koda na osnovu određenih uslova. Međutim, imaju različite karakteristike i upotrebe.

Izveštaj za `switch` procenjuje matematički izraz i upoređuje njegovu vrednost sa nizom članaka/sekcija `case`. Kada se pronađe jednačina, deo koda koji je povezan sa `case` će biti rešen. Evo osnovne strukture izveštaja za `switch`:

```js
switch (expression) {
  case value1:
    // code to be executed if expression === value1
    break;
  case value2:
    // code to be executed if expression === value2
    break;
  default:
    // code to be executed if expression doesn't match any case
}
```

Informacija o `break` na kraju svakog `case` je veoma važna. Govori programu da izađe iz sekcije switch čim se izvrši odgovarajući `case`. Bez toga, program će nastaviti sa izvršavanjem sledećih slučajeva, ponašanje poznato kao "fall-through".

Informacije za `switch` se obično koriste kada poredite jedan kriterijum sa više mogućih vrednosti. Često se koriste kada imate mnogo uslova za proveru naspram jednog kriterijuma. Evo primera korišćenja informacija `switch` za radne dane:

:::interactive_editor

```js
let dayOfWeek = 3; 

switch (dayOfWeek) {
    case 1:
        console.log("It's Monday! Time to start the week strong.");
        break;
    case 2:
        console.log("It's Tuesday! Keep the momentum going.");
        break;
    case 3:
        console.log("It's Wednesday! We're halfway there.");
        break;
    case 4:
        console.log("It's Thursday! Almost the weekend.");
        break;
    case 5:
        console.log("It's Friday! The weekend is near.");
        break;
    case 6:
        console.log("It's Saturday! Enjoy your weekend.");
        break;
    case 7:
        console.log("It's Sunday! Rest and recharge.");
        break;
    default:
        console.log("Invalid day! Please enter a number between 1 and 7.");
}
```

:::

Informacije za `switch` mogu se čitati lakše i biti kraće kada obrađuju mnogo mogućih vrednosti za jedan parametar.

Podaci za `if/else if` sa druge strane su fleksibilniji. Mogu da procene složene uslove i različite kriterijume u svakoj sekciji. To ih čini pogodnim za različite vrste scenarija. Evo primera kada možete koristiti podatke `if/else` umesto podataka `switch`:

:::interactive_editor

```js
let creditScore = 720; 
let annualIncome = 60000; 
let loanAmount = 200000; 

let eligibilityStatus;

if (creditScore >= 750 && annualIncome >= 80000) {
    eligibilityStatus = "Eligible for premium loan rates.";
} else if (creditScore >= 700 && annualIncome >= 50000) {
    eligibilityStatus = "Eligible for standard loan rates.";
} else if (creditScore >= 650 && annualIncome >= 40000) {
    eligibilityStatus = "Eligible for subprime loan rates.";
} else if (creditScore < 650) {
    eligibilityStatus = "Not eligible due to low credit score.";
} else {
    eligibilityStatus = "Not eligible due to insufficient income.";
}

console.log(eligibilityStatus);
```

:::

U ovom primeru, imamo godišnji prihod osobe i kreditni rejting, i proveravamo za koje vrste kredita bi mogao da kvalifikuje. Budući da radimo sa složenim logičkim procenama i mnogo kriterijuma, bolje je koristiti podatke iz `if/else` ovde umesto podataka iz `switch`.

Treba napomenuti da informacije za `switch` u JavaScript koriste strogo poređenje (`===`), što znači da ne vrši primorano tipiziranje. Ovo može biti prednost u pogledu predviđanja i izbegavanja manjih grešaka.

Ukratko, iako deklaracije `switch` i lanci `if/else if` sve omogućavaju multi-granu logiku u vašem kodu, imaju različite snage. Deklaracije `switch` su efikasne za obradu mnogo mogućih vrednosti za jedan kriterijum, dok lanci `if/else if` nude veću fleksibilnost za kompleksne uslove. Izbor između njih često zavisi od specifičnih potreba vašeg koda i preferencija za pojedinačni ili timski stil kodiranja.
