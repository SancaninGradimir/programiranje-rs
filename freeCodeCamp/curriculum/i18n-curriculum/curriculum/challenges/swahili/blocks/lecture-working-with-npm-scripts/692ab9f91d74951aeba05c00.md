# --description--

Sada ćeš naučiti o npm scripts.

Pre nego što uđemo duboko, hajde da pregledamo neke važne koncepte:

* **npm** je alat komandne linije koji možeš koristiti za instaliranje i upravljanje JavaScript paketima.
    
* **Paket (package)** je direktorijum koji organizuje povezani kod u logičku celinu.
    
* **script** ni mfululizo wa maagizo au amri zilizoandikwa kwa lugha ya programu ya scripting.
    

**npm scripts** su posebne komande koje možete definisati u fajlu `package.json` vašeg programa.

Pomaže u vođenju zadataka i ponavljajućih radnih procesa tokom faza razvoja, validacije i implementacije vašeg projekta.

Na primer, jedna upotreba npm skripti je da se `npm start` inicijalizuje celo vaše razvojno okruženje. Umesto da pišete sve komande jednu po jednu svaki put kada želite da pokrenete razvojno okruženje, možete samo izvršiti `npm start` i ceo proces će biti sproveden.

Takođe možete koristiti npm scripts za pokretanje provere koda širom vaše baze podataka i za izvršavanje testnih setova, kao i za druge namene.

Ovo su veoma moćni alati, pa hajde da istražimo njihove osnove.

## Kako kreirati datoteku package.json

Da bi mogao da kreiraš i pokrećeš prilagođene npm skripte, biće ti potrebna datoteka `package.json`.

Datoteka `package.json` je ključna za Node.js projekte. Ona sadrži metapodatke, zavisnosti i skripte projekta.

Kao podsjetnik, kreiranje fajla `package.json` zahteva da pokrenete komandu `npm init` u korijenski direktorijum tvog projekta. Zatim možete ručno uneti ime paketa, verziju, opis i druge detalje, ili ostaviti ta polja prazna kako biste koristili podrazumevane vrednosti.

Nakon završetka ovog procesa, imaćete fajl `package.json` u glavnom direktorijumu vašeg projekta.

Ako ste koristili sve zadane vrednosti, trebalo bi da vidite ove parove ključ-vrednost kada otvorite `package.json` u vašem editoru koda:

```json
{
  "name": "<project_name>",
  "version": "1.0.0",
  "main": "index.js",
  "scripts": {
    "test": "echo \"Error: no test specified\" && exit 1"
  },
  "author": "",
  "license": "ISC",
  "description": ""
}
```

Pogledajte ovu paru ključ-vrednost ovde:

```json
"scripts": {
  "test": "echo \"Error: no test specified\" && exit 1"
}
```

Ovo je mesto gde ćeš staviti svoje npm scripts, ispod ključa od `scripts`.

Naći ćete jedan skript tamo podrazumevano, test skript:

```json
"test": "echo \"Error: no test specified\" && exit 1"
```

Ovo je primer npm skripte definisane u `package.json`.

Ovo je osnovna sintaksa za definisanje npm skripte:

```json
"<script_name>": "<commands>"
```

Levo od dvotačke nalazi se naziv skripte. Uobičajeni nazivi su `run`, `start`, `test`, `build`, `dev` i `lint`. Ovi nazivi služe kao prečice za osnovne komande sa desne strane.

Na tački, dobijamo osnovne komande same.

Na primer:

```json
"scripts": {
  "start": "node app.js"
}
```

Da biste pokrenuli skriptu, ukucajte ovo u komandnu liniju:

```bash
npm run <script_name>
```

To je kako možete pokrenuti bilo koji specifični skriptovi definisani u `package.json`.

Ponekad možete koristiti ovu kraću verziju bez reči `run` za pokretanje skripti:

```bash
npm <script_name>
```

Ali ovo funkcioniše samo za posebne npm skripte pod nazivom `start`, `test`, `stop` i `restart`.

Ako se vratimo na naš primer, kao što imamo ovo u `package.json`:

```json
"scripts": {
  "start": "node app.js"
}
```

Da biste pokrenuli npm skriptu, morate napisati `npm run start` u komandnu liniju. Kada to učinite, stvarna komanda koja se izvršava je `node app.js`.

Pošto ova npm skripta ima kraću verziju, možete koristiti i `npm start`, a rezultati će biti isti.

U ovom slučaju, samo menjate jednostavnu komandu, `node app.js`, i drugu, `npm start`. Ali zamislite da imate niz složenih komandi koje treba izvršiti sekvencijalno ili paralelno, i da možete koristiti `npm start` za pokretanje svih. To bi uštedelo mnogo vremena, zar ne? Tamo se zaista vidi moć npm scripts.

Evo pregleda nekih od najčešće korišćenih npm skripti:

* `npm run`: Koristi se za pokretanje specifične npm skripte.
    
* `npm start`: Koristi se za pokretanje glavne aplikacije ili razvojnog servera.
    
* `npm lint`: Koristi se za izvršavanje provere stila koda kako bi se utvrdilo da li vaš kod ispunjava standarde stila i najbolje prakse.
    
* `npm test`: Koristi se za pokretanje kompleta testova projekta.
    
Postoji mnogo npm skripti za različite svrhe, ali ovo su neke od najčešćih koje ćete videti.

Takođe, uobičajena praksa je da se dvotačka koristi za grupisanje povezanih skripti pod istim prostorom imena. Na primer, `start:server` i `start:ui`.

## Kako pokrenuti sekvencijalne komande

Ako treba da pokrenete više naredbi uzastopno, imate dve opcije. Možete koristiti logički AND operator (`&&`) ili tačku-zarez (`;`). Pogledajmo njihove razlike.

Logički AND operator izvrši sledeću komandu samo ako je prva komanda uspešno pokrenuta.

Na primer:

```json
"scripts": {
  "build": "webpack --mode production",
  "start": "npm run build && node server.js"
}
```

Proverite da postoji logički AND operator u komandi `start`:

```json
"start": "npm run build && node server.js"
```

Hapa, `npm start` će pokušati da pokrene `npm run build` prvo, i ako se ovaj skript uspešno izvrši, `node server.js` će biti izvršen.

Ovo je vrlo uobičajen metod jer osigurava pouzdanost i sprečava gubljenje vremena i resursa za naknadne komande koje bi mogle da ne uspeju ili bacaju greške ukoliko prva komanda ne uspije.

Pored toga, možete koristiti operator tačke-zarez za izvršavanje sekvencijalnih komandi, bez obzira na to da li je prethodna komanda uspela ili ne.

U našem primeru, ove jednostavne promene biće imalo značajan uticaj na način kako se komande izvršavaju:

```json
"scripts": {
  "build": "webpack --mode production",
  "start": "npm run build; node server.js"
}
```

Obratite pažnju na komandu od `start` ponovo:

```json
"start": "npm run build; node server.js"
```

Sada, pošto koristimo polukuć umesto operatera `&&`, komanda `node server.js` će se izvršiti svaki put nakon `npm run build`, čak i ako `node server.js` nije uspešno.

```json
"start": "npm run build; node server.js"
```

## Kako pokrenuti paralelne komande

Ako je potrebno pokrenuti paralelne komande (istovremeno), možete koristiti operater `&`.

```json
"scripts": {
  "start:server": "node server/index.js",
  "start:client": "npm run build:watch",
  "dev": "npm run start:server & npm run start:client"
}
```

Međutim, treba da znate da ova opcija pokreće procese u pozadini.

Ovo znači da ako ga pokrenete u jednoj terminalskoj sesiji, možete videti sve njihove rezultate zajedno, što može biti teško za razumevanje.

Ostale opcije za izvršavanje paralelne komande su alati kao što su `concurrently` i `npm-run-all`.

## Kako proslediti argumente za npm scripts

Ako treba da proslediš argument direktno komandi koja se izvršava unutar npm skripte, možeš ih navesti tako što ćeš napisati dva znaka za prekid (`--`) praćena tim argumentima.

`--` Kažem npm-u da su sve naredne argumenti za osnovne komande koje će npm skript izvršiti.

Ovo je sintaksa rečenice:

```bash
npm run <script_name> -- <argument_1> <argument_2> <argument_3>
```

Na primer, recimo da si definisao ovaj skript za `npm start`:

```json
{
  "name": "project-example",
  "scripts": {
    "start": "node app.js"
  }
}
```

Kako možete specificirati port koji želite da pokrenete svoj server ako koristite `npm start`?

Recimo da želimo proslediti port jedan po jedan kao argument linije komandi. Napišeće se ovako:

```bash
npm start -- 8000
```

U vašem kodu, možete pronaći ove argumente u liniji `process.argv` koristeći indikatore:

```bash
process.argv[0] # The path to the node executable.
process.argv[1] # The script file being executed.
process.argv[2] # The first argument (8000).
```

Dakle, možete dobiti i koristiti broj porta gde god želite za `process.argv[2]`. Ovo je moć argumenata linije komandi pomešanih sa npm scripts-om.

Kao što možete da vidite, npm skriptovi su važni alati za pojednostavljenje procesa razvoja. Oni vam pomažu da pokrenete i pretvorite niz složenih i ponovno upotrebljivih komandi u jednostavne, konzistentne veze koje možete koristiti u svakodnevnom radu.
