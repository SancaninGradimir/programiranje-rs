# --description--

Komentari tokom programiranja se koriste da pruže više konteksta kodu ili da ostave napomene za tebe i druge.

Komentari su redovi ili delovi teksta koje engine JavaScript ne obrađuje kada se vaš kod izvršava. Oni služe samo ljudima koji čitaju taj kod, bilo da ste vi ili neko drugi.

JavaScript Pruža dva načina za dodavanje komentara u vaš kod: jednoredne komentare i višelinijske komentare.

Inline komentar se kreira korišćenjem dve kose crte na početku (`//`). Evo primera:

```js
// I am a single line comment in JavaScript
```

Ovaj tip komentara je dobar za kratke opise ili pojašnjenja.

Evo stvarnog primera iz projektnih fajlova kurikuluma freeCodeCamp-a:

```js
// This is to allow English to build without having to download the i18n files.
// It fails when trying to resolve the i18n-curriculum path if they don't exist.
const curriculumLocale = process.env.CURRICULUM_LOCALE ?? 'english';
const I18N_CURRICULUM_DIR = path.resolve(
  __dirname,
  curriculumLocale === 'english' ? '.' : 'i18n-curriculum/curriculum'
);
```

Ne brini da razumeš šta kod radi trenutno, jer je komplikovanije nego što si do sada naučio. Umesto toga, fokusiraj se na komentare koje je napisao programer. Ovi komentari pružaju važan kontekst zato što ovaj kod postoji.

Ovakvi komentari su važni za one koji rade u timu iz dva razloga:

1. Ostali programeri koji rade na projektu će razumeti svrhu ovog koda.

1. Pomaže da se spreči promena ili brisanje bez konsultacija sa timom, što može uzrokovati grešku ili probleme.

Drugi tip komentara su višelinijski komentari. Ovde je osnovna sintaksa:

```js
/*
 I am a multiline comment.
 This is helpful for longer explanations.
*/
```

Višelinijski komentari su važni kada je potrebno napisati dugačke opise, objašnjenja ili napomene u vašem kodu.

Hajde da ponovo pogledamo fajlove projekta kurikuluma freeCodeCamp-a da vidimo kako višelinijski komentari mogu se koristiti u stvarnom scenariju.

```js
/* Since there can be more than one way to complete a certification (using the
legacy curriculum or the new one, for instance), we need a certification
field to track which certification this belongs to. */
const dupeCertifications = [
  {
    certification: 'responsive-web-design',
    dupe: '2022/responsive-web-design'
  }
];
const hasDupe = dupeCertifications.find(
  cert => cert.dupe === meta.superBlock
);
```

Kao što je prethodno rečeno, ignoriši sav kod JavaScript jer koristi koncepte koji još nisu naučeni. Umesto toga, obrati pažnju na komentare koje je ostavio programer.

Razvojnik u timu, ili čak novi doprinosilac koji radi na projektu, može razumeti zašto je ovaj deo koda ovde i imati pun kontekst pre nego što počne da radi u ovom delu projekta.

Iako su komentari važni u programiranju, važno je izbegavati pisanje previše komentara. Nije potrebno dodavati komentare svakom redu koda, posebno ako je kod jednostavan i lako razumljiv.

Ovde je primer korišćenja komentara za opisivanje stvari koje su očigledne:

```js
// This code uses the const keyword to create a new variable called price.
// We are assigning the number 10 to the price variable.
const price = 10;
```

U ovom slučaju, nema potrebe za dodavanjem komentara jer kod govori sam za sebe. Cilj je poboljšati čitljivost, a ne da se preplavi nepotrebnim objašnjenjima.

Ako želiš da dodaš komentare u svoje lične projekte kada počneš da učiš da pišeš kod, to je u redu. Ali kada počneš da radiš na stvarnim projektima sa drugim razvojnicima, važno je da ne koristiš komentare za kod koji je lako razumljiv.

Takođe je važno da ne koristiš komentare kako bi pomogao da sakrije složen, previše komplikovan ili loše napisan kod. U takvim slučajevima, bolje je popraviti ili promeniti svoj kod kako bi drugi programeri mogli dobro razumeti šta se dešava.

Komentari su moćni alati za dokumentovanje vašeg koda i čine ga lakšim za razumevanje. Trebalo bi da koristite komentare kako biste pružili kontekst ili ostavili napomene za sebe i druge.
