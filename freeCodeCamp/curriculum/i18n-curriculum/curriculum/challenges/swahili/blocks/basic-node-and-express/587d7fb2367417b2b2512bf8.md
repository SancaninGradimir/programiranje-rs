# --description--

Postavite POST handler na ruti `/name`. To je ista ruta kao i prethodno. Pripremili smo formu na front-end HTML stranici. Ona će poslati iste podatke kao izazov 10 (query parametri). Ako je body parser ispravno postavljen, trebali biste da dobijete parametre u `req.body` objektu. Pogledajte primer kako izgleda biblioteka:

<blockquote>ruta: POST '/library'<br>urlencoded_body: userId=546&#x26;bookId=6754 <br>req.body: {userId: '546', bookId: '6754'}</blockquote>

Odgovorite sa istim JSON objektom kao i prethodno: `{name: 'firstname lastname'}`. Probajte da li vaša ruta radi koristeći HTML formu koju smo vam obezbedili na front-end stranici.

Savet: Postoje i druge HTTP metode pored GET i POST. I prema konvenciji, postoji odgovarajuća veza između HTTP metode i operacije koju ćete izvršiti na serveru. Mapiranje konvencije je:

POST (ponekad PUT) - Kreirajte novi resurs koristeći podatke poslate zahtevom,

GET - Čitajte postojeći resurs bez menjanja,

PUT ili PATCH (ponekad POST) - Ažurirajte resurs koristeći poslate podatke,

DELETE - Obrišite resurs.

Takođe postoji nekoliko drugih metoda koje se koriste za uspostavljanje konekcije sa serverom. Osim GET-a, sve druge gore navedene metode mogu da imaju opterećenje (tj. podatke u telu zahteva). Body parser middleware radi i sa ovim metodama.

Test 1 : Vaš API endpoint treba da odgovori sa tačnim imenom

```js
  const response = await fetch(code + '/name', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/x-www-form-urlencoded'
    },
    body: new URLSearchParams({ first: 'Mick', last: 'Jagger' }).toString()
  });
  if (!response.ok) {
    throw new Error(await response.text());
  }
  const data = await response.json();
  assert.equal(
    data.name,
    'Mick Jagger',
    'Test 1: "POST /name" route does not behave as expected'
  );
```
