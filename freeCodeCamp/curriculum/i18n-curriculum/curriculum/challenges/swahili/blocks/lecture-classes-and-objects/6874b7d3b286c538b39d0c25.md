# --description--

Posebne putanje u Python, koje su takođe poznate kao "magične putanje" ili "dunder putanje", predstavljaju posebne putanje za Python koje počinju i završavaju sa dve podvučnice (`__`). Sama reč "dunder" potiče od dve podvučnice (**d** za *double*, **under** za *underscores*).

Možda ste već koristili specijalnu metodu bez znanja. Svaki put kada napišete nešto kao `3 + 4`, Python poziva `3.__add__(4)` tiho u pozadini. To je specijalna metoda koja radi. Zbog toga, iako možete direktno pozvati specijalnu metodu, često to ne radite. Nešto kao `3 + 4` je jasnije i lakše za čitanje nego samostalno pozivanje `3.__add__(4)`.

Osim `__add__`, `__init__()` je drugi specijalni način koji ćete videti i koristiti više, jer je to adresa klase objekata. Postoje i drugi slični kao `__len__()` i `__str__()`.

Zamislite poseban proces između programera i prevoda jezika Python sam.

Zapamtite, nemojte da pozivate specifičnu putanju direktno. Umesto toga, Python ga poziva direktno kada se dogode određene akcije. Ove operacije uključuju:

- **Matematičke operacije kao sabiranje, oduzimanje, množenje, deljenje i druge**. Za sabiranje, `__add__()` se naziva, `__sub__()` za oduzimanje, `__mul__()` za množenje i `__truediv__()` za deljenje.
    
- **Operacije sa nizovima karaktera kao spajanje, ponavljanje, kreiranje šablona i konverzija u tekst**. `__add__()` se naziva za spajanje, `__mul__()` za ponavljanje, `__format__()` za kreiranje šablona, `__str__()` i `__repr__()` za konverziju u tekst, itd.
    
- **Operacije poređenja kao jednakost, manje od, veće od i druge**. `__eq__()` se naziva za proveru jednakosti, `__lt__()` za manje od, `__gt__()` za veće od, itd.
    
- **Operacije iteracije kao činanje nečega ponovljivim i pomeranje kroz elemente**. `__iter__()` se naziva vraćanjem izraza i `__next__()` uzimanjem sledećeg elementa.

Obično, tipovi podataka za Python kao sekvenca slova i brojeva već znaju kako da dodaju elemente, spajanje, poređenje jednakosti, korišćenje u petljama i slično.

Ali kada kreiraš sopstvenu klasu, Python neće automatski znati kako da rukuje tim stvarima.

Ovde su posebni putevi — omogućavaju vam da prilagodite ponašanje ugrađeno u Python.

Pretpostavimo da želite pronaći broj stranica u knjigama kreiranim od klase koja se nalazi ispod, ili uporediti i dobiti čitljivu sekvencu karaktera tih stavki. Evo šta se dešava bez specifične metode:

```python
class Book:
   def __init__(self, title, pages):
       self.title = title
       self.pages = pages

book1 = Book("Built Wealth Like a Boss", 420)
book2 = Book("Be Your Own Start", 420)

print(len(book1)) # TypeError: object of type 'Book' has no len()
print(str(book1)) # <__main__.Book object at 0x102ed2900>
print(book1 == book2) # False even though they have the same number of pages
```

Katika mfano huu:

- `len(book1)` nije uspeo jer Python ne zna kako da dobije dužinu tvog objekta knjige bez metode `__len__()`.
    
- `str(book1)` je ispisao nešto poput `<__main__.Book object at 0x102ed2900>` jer je to podrazumevana referenca kada nisi koristio `__str__()`.
    
- `book1 == book2` je vratilo `False` jer Python proverava samo da li su oba objekta isti u memoriji, a ne da li imaju isti sadržaj.
    

Evo kako možete definisati vaše specifične metode za `__len__()`, `__str__()`, i `__eq__()` ili raditi sa stvarima kreiranim iz klase `Book` lako:

```python
class Book:
   def __init__(self, title, pages):
       self.title = title
       self.pages = pages

   def __len__(self):
       return self.pages

   def __str__(self):
       return f"'{self.title}' has {self.pages} pages"

   def __eq__(self, other):
       return self.pages == other.pages
  
book1 = Book("Built Wealth Like a Boss", 420)
book2 = Book("Be Your Own Start", 420)

print(len(book1)) # 420
print(len(book2)) # 420
print(str(book1)) # 'Built Wealth Like a Boss' has 420 pages
print(str(book2)) # 'Be Your Own Start' has 420 pages
print(book1 == book2) # True
```

Drugi primer je scenarij kupovine gde radite sledeće:

- Dodaj predmete u vozilo
    
- Ukloni predmete iz vozila
    
- Dohvati broj predmeta u vozilu
    
- Proveri koji su predmeti prisutni u vozilu
    
- Proveri da li se neki predmet nalazi u vozilu
    
- Vrati ili prikaži predmet koji je u određenom modelu vozila

Iako možete imati putanju koja dodaje stvari u auto i uklanja određene stvari iz auta, možete kreirati posebnu putanju za sve ostale zadatke:

- `__len__()` dobiti dimenzije predmeta unutar vozila
    
- `__iter__()` ponoviti predmete unutar vozila kako biste ih videli
    
- `__contains__()` proveriti da li je nešto u automobilu
    
- `__getitem__()` vratiti ili prikazati predmet koji se nalazi na određenom dijagramu vozila

Ovde je primer klase koja `Cart` ima ove korisničke i specijalne metode:

```python
class Cart:
   def __init__(self):
       self.items = []

   def add(self, item):
       self.items.append(item)

   def remove(self, item):
       if item in self.items:
           self.items.remove(item)
       else:
           print(f'{item} is not in cart')

   def list_items(self):
       return self.items

   def __len__(self):
       return len(self.items)

   def __getitem__(self, index):
       return self.items[index]

   def __contains__(self, item):
       return item in self.items

   def __iter__(self):
       return iter(self.items)
```

Evo kako ih možete koristiti:

```python
cart = Cart()
cart.add('Laptop')
cart.add('Wireless mouse')
cart.add('Ergo keyboard')
cart.add('Monitor')

for item in cart:
   print(item, end=' ') # Laptop Wireless mouse Ergo keyboard Monitor

print(len(cart)) # 4
print(cart[3]) # Monitor

print('Monitor' in cart) # True
print('banana' in cart) # False

cart.remove('Ergo keyboard')

print(cart.list_items()) # ['Laptop', 'Wireless mouse', 'Monitor']

cart.remove('banana') # banana is not in cart
```

Ovo su neke metode koje ćete koristiti, specifične za Python u stvarnom životu.
