# --description--

Getteri i setteri su metode koje vam omogućavaju da kontrolišete kako se svojstva objekta klase pristplaćuju i menjaju. Getteri vam omogućavaju da dobijete vrednost, dok setteri postavljaju vrednost.

To se radi putem nečega što je poznato kao svojstvo (property). Ona povezuju getere i setere, omogućavajući pristup podacima.

Svojstva funkcionišu kao uobičajena svojstva, ali se ponašaju kao putanja ispod površine. Zamislite to kao podatke koje opisujete kao putanju, ali koji funkcionišu kao svojstvo. To znači da možete dobiti svojstva koristeći notaciju tačke umesto standardnih zagrada.

Glavna stvar koju svojstva rade je izvršavanje dodatne logike u pozadini kada dohvatate, postavljate ili brišete vrednosti koristeći ta svojstva. Ovo ih čini najboljim izborom kada želite da dohvatite ili pokrenete podatke unutar objekata.

Zašto onda koristiti svojstva umesto metoda? Ovo je više o čitljivosti i kontraktu. Čine vaš kod čistim i lakšim za čitanje.

Kada koristite funkciju, morate je uvek pozivati sa standardnim zagradama. Međutim, za svojstva, možete ga dobiti kao regularno svojstvo koristeći dot notaciju. To čini da vaš kod izgleda jednostavno čak i kada pokrećete dodatne zadatke u pozadini.

Na primer, možda želite da izračunate vrednost ili proverite da li je nova vrednost validna pre nego što je sačuvate. Umesto pozivanja metode za to, možete koristiti svojstvo kao svojstvo da to učinite.

Katika Python, dekorator je funkcija koja menja funkcionalnost drugih funkcija ili klasa, bez menjanja njihovog originalnog koda.

Iako je moguće kreirati prilagođene dekoretere, ovo je izvan opsega ovog tutorijala. Samo znajte da kreiranje klase dekoruje metodu i stavlja dekoreter `@property` na nju. Ovo menja metodu u atribut, tako da se može pristupiti kao atribut dok se poziva dekorisana metoda.

Ovo nas vodi ka getterima. Ovde je kako da kreirate jedan koristeći dekorator `@property`:

```py
class Circle:
    def __init__(self, radius):
        self._radius = radius

    @property
    def radius(self): # A getter to get the radius
        return self._radius
  
    @property
    def area(self):  # A getter to calculate area
        return 3.14 * (self._radius ** 2)

my_circle = Circle(3)

print(my_circle.radius) # 3
print(my_circle.area) # 28.26
```

Ovaj primer računa radijus i površinu kruga.

Pogledajte kako smo koristili `_radius` umesto radijusa unutar klase. Linija ispod je uobičajena konvencija za Python da bi pokazala da je svojstvo privatno. Drugim rečima, ukazuje da je namenjeno za interno korišćenje i ne bi trebalo pristupiti mu direktno izvan klase.

Da biste kreirali `setter` za radijus, na primer, morate definisati drugi način sa istim imenom i koristiti `@<property_name>.setter` preko njega:

Korišćenjem `self.radius` unutar `__init__`, osigurava se da *setter* funkcioniše prilikom kreiranja objekta, pa nevažeće vrednosti radijusa se uhvate odmah.

```py
class Circle:
    def __init__(self, radius):
        self.radius = radius # Calling the setter

    @property
    def radius(self):  # A getter to get the radius
        return self._radius

    @radius.setter
    def radius(self, value):  # A setter to set the radius
        if value <= 0:
            raise ValueError('Radius must be positive')
        self._radius = value

my_circle = Circle(3)
print('Initial radius:', my_circle.radius) # Initial radius: 3

my_circle.radius = 8
print('After modifying the radius:', my_circle.radius) # After modifying the radius: 8
```

U ovom primeru, setter za radijus ne postavlja samo radijus kruga, već pokreće validaciju koja osigurava da je radijus pozitivan broj.

Čim definišeš getter-e i setter-e, Python automatski poziva ispod haube svaki put kada koristiš standardnu sintaksu svojstva:

```py
my_circle.radius # This will call the getter
my_circle.radius = 4 # This will call the setter
```

Zapamtite da u `setter`-u ne smete koristiti isto ime svojstva prilikom postavljanja nove vrednosti. Ovo je zato što `self.radius = value` će pozvati `setter` unutar sopstvene metode `setter`-a, uzrokujući beskonačnu petlju i `RecursionError`. Stoga uvek koristite format sa prefiksom linije ispod `self._radius = value`.

Pošto možete da kontrolišete kako je svojstvo dostupno putem getera i kako se menja preko seta, takođe možete kontrolisati kako se briše koristeći deleter.

Deleter izvršava specifičnu logiku kada se koristi `del` izjava na atributu. Da biste kreirali deleter, koristite dekorator `@<property_name>.deleter`:

```py
class Circle:
    def __init__(self, radius):
        self.radius = radius

    # Getter
    @property
    def radius(self):
        return self._radius

    # Setter
    @radius.setter
    def radius(self, value):
        if value <= 0:
            raise ValueError("Radius must be positive")
        self._radius = value

    # Deleter
    @radius.deleter
    def radius(self):
        print("Deleting radius...")
        del self._radius
```

Evo kako se može koristiti deleter:

```py
# Create circle object with a radius
my_circle = Circle(33)
print("Initial radius:", my_circle.radius)  # 33

# Delete the radius
# This calls the deleter
del my_circle.radius # Deleting radius...
print("Radius deleted!") # Radius deleted!

# Try to access radius after deletion
try:
    print(my_circle.radius)
except AttributeError as e:
    print("Error:", e) # Error: 'Circle' object has no attribute '_radius'
```

Glavna stvar koju dobijate odavde je:

- Getteri vam omogućavaju da dobijete ili čak izračunate vrednost trenutno.
- Setteri vam omogućavaju da bezbedno promenite vrednost pokretanjem validacije pre postavljanja vrednosti.
- Svojstva su ono što povezuje getter-e i setter-e kako biste mogli da napišete logiku a pritom i dalje koristite notaciju tačke (dot notation).
- Deleteri vam omogućavaju da definišete šta se dešava kada je svojstvo obrisano.
