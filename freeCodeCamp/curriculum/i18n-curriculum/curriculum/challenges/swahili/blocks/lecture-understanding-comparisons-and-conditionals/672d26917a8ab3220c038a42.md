# --interactive--

U JavaScript, `null` i `undefined` postoje dva različita tipa podataka koji predstavljaju odsustvo vrednosti, ali funkcionišu drugačije u upoređivanju. Razumevanje kako ovi tipovi interaguju u različitim scenarijima upoređivanja je važno za pisanje stabilnog i bezgrešnog koda.

Počinjemo sa tipom `undefined`. Parametar nazvan `undefined`, kada je deklarisan, ali mu nije dodeljena vrednost. Ovo je podrazumevana vrednost za nedeklarisane parametre i funkcionalne parametre kojima nije dodeljen argument.

Tip za `null`, sa druge strane, je nulta vrednost koja predstavlja namerno odsustvo vrednosti. Često se koristi da pokaže da parametar nema vrednost po želji.

Kada uporedite `null` sa `undefined` koristeći operator jednakosti (`==`), JavaScript vrši konverziju tipa. To znači da pokušava da konvertuje varijable u isti tip pre nego što obavi poređenje. U ovom slučaju, `null` i `undefined` se smatraju jednakima:

:::interactive_editor

```js
console.log(null == undefined); // true
```

:::

Međutim, kada koristite operator stroge jednakosti (`===`), koji proverava vrednost i tip bez nametanja tipa, `null` i `undefined` se ne poklapaju:

:::interactive_editor

```js
console.log(null === undefined); // false
```

:::

Ova razlika je važna da se uzme u obzir kada pišete uslovne izjave ili izvršavate provere jednakosti u vašem kodu. Kada poredite `null` ili `undefined` sa drugom vrednošću koristeći operater jednakosti (`==`), ponašanje može biti nepredvidivo. Na primer:

:::interactive_editor

```js
console.log(null == 0);  // false
console.log(null == ''); // false
console.log(undefined == 0); // false
console.log(undefined == ''); // false
```

:::

Ovo poređenje vraća `false` jer su `null` i `undefined` jednaki za svako (i za sebe) samo prilikom korišćenja operatorsa jednakosti. Ponašanje `null` u drugim poređenjima je posebno teško:

:::interactive_editor

```js
console.log(null > 0);  // false
console.log(null == 0); // false
console.log(null >= 0); // true
```

:::

`undefined`, s druge strane, konstantno se menja u `NaN` u kontekstu brojeva, što uzrokuje da sve poređenja brojeva sa `undefined` vrate `false`:

:::interactive_editor

```js
console.log(undefined > 0);  // false
console.log(undefined < 0);  // false
console.log(undefined == 0); // false
```

:::

S obzi maže na ove male razlike, generalno se savetuje korišćenje operatorsa strogo jednakosti prilikom poređenja vrednosti, posebno kada gledate `null` i `undefined`. Ova metoda pomaže u izbegavanju prisile nepouzdljivog tipa i čini ponašanje vašeg koda predvidivim.

Ukratko, iako su i `null` i `undefined` korišćene za predstavljanje nepostojanja vrednosti, one obavljaju različite funkcije u poređenju. Razumevanje ovih razlika je ključno za pisanje čistog i bezgrešnog koda JavaScript.
