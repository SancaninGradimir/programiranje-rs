# --description--

U početnoj obuci, naučili ste o svojstvima elemenata kao atributima koje poseduje element. To znači da čuvaju podatke koji definišu stanje ili karakteristiku tog elementa.

Na primer, običan automobil ima marku i model. Marka i model mogu biti svojstva elemenata za klasu `Car`:

```python
class Car: 
    def __init__(self, brand, model): 
        self.brand = brand 
        self.model = model 

my_car = Car('Lamborghini', 'Gallardo') 
print(my_car.brand) # Lamborghini 
print(my_car.model) # Gallardo 
```

Ali ponekad ne znate koji vam atributi trebaju dok vaš program počne da se izvršava. Zamislite da pišete skriptu koja prima imena atributa od korisnika ili iz fajla za konfiguraciju. To nisu atributi koje možete fiksirati (hardkodovati) u kodu na samom početku.

Ovo je mesto gde ulazi u pitanja rukovanja svojstvima dinamički. Na ovaj način, možete pristupiti, menjati, pregledavati ili čak brisati svojstva koristeći njihova imena kao parametre, a ne kao fiksna imena u vašem kodu. Ovo vašem programu daje sposobnost prilagođavanja kako bi reagovao na različite podatke ili unos korisnika u realnom vremenu.

Python daje vam četiri ugrađena funkcije za rad sa svojstvima elemenata putem promena. One su `getattr()`, `setattr()`, `hasattr()`, i `delattr()`.

Omogućava vam da dobijete pristup, kreirate, pregledate i brišete atribute koristeći imena parametara. Hajde da istražimo svaki od njih kako funkcioniše.

`getattr()` omogućava čitanje svojstva iz elementa kada ne znate njegovo ime do vremena izvršavanja. Ako svojstvo ne postoji, vraća `AttributeError`, osim ako ne pružite podrazumevanu vrednost.

Da bi je koristio, prosleđuješ objekat, ime osobine i podrazumevanu vrednost po potrebi:

```python
getattr(object, attribute_name, default_value) 
```

Evo primera:

```python
class Person: 
    def __init__(self, name, age): 
        self.name = name 
        self.age = age 

person = Person('John Doe', 30) 
 
print(getattr(person, 'name')) # John Doe 
print(getattr(person, 'age')) # 30 
print(getattr(person, 'city', 'Milano')) # Milano
```

U gornjem primeru, `Milano` je zadana vrednost jer `city` nije u klasi `Person`.

Kao što smo ranije rekli, stvarna snaga `getattr()` se pojavljuje kada naziv atributa dolazi iz parametra, kao što je iz unosa korisnika ili nekog fajla.

U tom slučaju, ne možete koristiti standardnu sintaksu od `object.attribute_name` jer ime atributa nije stabilno.

```python
class Person: 
    def __init__(self, name, age): 
        self.name = name 
        self.age = age 

person = Person('John Doe', 30)

attr_name = input('Enter the attribute you want to see: ')
print(getattr(person, attr_name, 'Attribute not found'))
```

U ovom slučaju, ako korisnik unese `name`, videće `John Doe`, a ako unese `age`, videće `30`. Ako unese nešto što ne postoji u klasi kao što je `email`, videće `Attribute not found`.

Ovo je zapravo prednost obrade atributa na dinamičan način. Omogućava vašem kodu da reaguje na unos i podatke koje nikada ranije nije video.

Štaviše, možda želite da pregledate sve svojstva koja ima element, a ne samo ona koja već znate. Ugrađena funkcija za `dir()` vam omogućava da to uradite. Vraća listu svih imena svojstava na elementu. Evo kako ga koristiti:

```python
class Person: 
    def __init__(self, name, age): 
        self.name = name 
        self.age = age 

person = Person('John Doe', 30)

# Loop through all attributes of the person object with dir() function
for attr in dir(person):
    # Ignore dunder methods like __init__ or __str__ and regular methods
    if not attr.startswith('__') and not callable(getattr(person, attr)): 
        value = getattr(person, attr)
        print(f'{attr}: {value}')

# Output
# age: 30
# name: John Doe
```

U gornjem kola, `callable()` je ugrađena funkcija koja vraća `True` ako prosleđeni element može biti nazvan funkcijom ili metodom, i `False` inače. Kada se pregleda `not callable(getattr(person, attr))`, kola preskače metodu i ispisuje samo svojstva podataka kao što su `name` i `age`.

Funkcija za `setattr()` omogućava vam da kreirate novu osobinu ili ažurirate postojeću na način promena. Sintaksa je sledeća:

```python
setattr(object, attribute_name, value) 
```

Ovde je primer koji postavlja konfiguracione atribute na osnovu podataka iz konfiguracionog fajla ili varijable okruženja:

```python
class Configuration:
    pass

# Data loaded at runtime (like from a config or env file)
settings_data = {
    'server_url': 'https://api.example.com',
    'timeout_sec': 30,
    'max_retries': 5
}

config_obj = Configuration()

# Dynamically set attributes using dictionary keys and values
for attr_name, attr_value in settings_data.items():
    setattr(config_obj, attr_name, attr_value)

print(config_obj.server_url) # https://api.example.com
print(config_obj.timeout_sec) # 30
```

Takođe postoji `hasattr()`. Pre nego što nešto uradite sa nekom funkcijom ili je obrišete, dobra praksa je proveriti da li postoji. To je `hasattr()` što vam omogućava da to uradite. Proverava da li funkcija postoji i vraća `True` ili `False` u zavisnosti od rezultata.

Ovo je osnovna sintaksa:

```python
hasattr(object, attribute_name)  
```

Ovde je primer koji proverava promenu prisutnosti atributa u strukturi klase `Product`:

```python
class Product:
    def __init__(self, name, price):
        self.name = name
        self.price = price

product_a = Product('T-Shirt', 25)

required_attributes = ['name', 'price', 'inventory_id']

for attr in required_attributes:
    if not hasattr(product_a, attr):
        print(f"ERROR: Product is missing the required attribute: '{attr}'")
    else:
        # Access the attributes dynamically once their existence is confirmed
        print(f'{attr}: {getattr(product_a, attr)}')

# Output:
# name: T-Shirt
# price: 25
# ERROR: Product is missing the required attribute: 'inventory_id'
```

Greška je nastala zato što `inventory_id` ne postoji u klasi `Product` i njegovoj instanci.

Konačno, `delattr()` vam omogućava da obrišete svojstva putem promena:

```python
delattr(object, attribute_name) 
```

Na primer, zamislite da je karakteristika završila obradu, pa odlučite da očistite bilo koje osetljive ili privremene atribute koji mogu postojati pre snimanja konačne verzije. Nakon toga, možete koristiti `dir()` oko preostalih atributa:

```python
class UserSession:
    def __init__(self, user_id, token):
        self.user_id = user_id
        self.auth_token = token # sensitive
        self.temp_counter = 0 # temporary

session = UserSession(101, 'a1b2c3d4e5')

# List of attributes to remove dynamically before "saving" the session
attributes_to_clean = ['auth_token', 'temp_counter']

# Dynamically remove specified attributes
for attr in attributes_to_clean:
    if hasattr(session, attr):
        delattr(session, attr)
        print(f'Removed attribute: {attr}')

print('\nFinal attributes remaining:')

# Loop through the remaining attributes with dir()
for attr in dir(session):
    # Ignore dunder methods like __init__ or __str__ and regular methods
    if not attr.startswith('__') and not callable(getattr(session, attr)):
        print(f' - {attr}: {getattr(session, attr)}')

# Output:
# Removed attribute: auth_token
# Removed attribute: temp_counter

# Final attributes remaining:
#  - user_id: 101
```

Tako možeš da obrađuješ atribute putem promena!
