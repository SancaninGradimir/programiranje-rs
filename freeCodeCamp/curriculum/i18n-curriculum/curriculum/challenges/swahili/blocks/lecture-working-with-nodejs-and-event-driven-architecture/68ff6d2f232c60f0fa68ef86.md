# --description--

Dakle, pisali ste klijentski kod koristeći JavaScript, kod koji se izvršava na strani klijenta web aplikacije, obično u web pregledaču.

JavaScript klajentska strana može biti veoma moćna, ali ima određena ograničenja za kompleksnije zadatke.

Na primer:

- Ima veoma ograničen pristup lokalnim fajlovima računara.
- Nije dizajnirano da obrađuje složnu programsku logiku.
- Postoje i sigurnosni problemi. Na primer, ne želite da izložite osetljive informacije, kao što su lozinke baze podataka, u klijentskom kodu (client-side).

JavaScript na strani klijenta ima ove ograničenja jer je prvobitno dizajnirano da se izvršava samo u web pregledačima.

Web pretraživači pružaju okruženje potrebno za pokretanje koda JavaScript, uključujući JavaScript Engine. Takođe pruža pristup dokumentu Object Model (DOM), pa tako možete dobiti funkcije HTML u vašem kodu.

Prethodno, nismo mogli pokrenuti JavaScript van pretraživača, ali sada možemo koristeći Node.js.

Node.js je okruženje za izvršavanje JavaScript. Osnovno se definiše kao “okruženje za izvršavanje JavaScript otvorenog izvora i kompatibilno sa mnogim sistemima.”

Ispitajmo njegovo značenje:

- Node.js je open-source jer je njegov kod otvoren i javno dostupan, te ga održava velika zajednica programera.
- Kompatibilan je sa mnogim sistemima jer radi na bilo kom operativnom sistemu, uključujući Windows, macOS i Linux.
- I predstavlja okruženje za izvršavanje JavaScript jer omogućava pokretanje koda JavaScript izvan pretraživača.
    
Node.js se koristi u različitim sektorima. Programeri ga koriste za izgradnju web servera i API koji obrađuju aplikacije za HTTP, uključujući web i mobilne aplikacije.

Ukoliko koristite ili razvijate veb stranicu ili web aplikaciju koja se povezuje sa bazom podataka ili obrađuje osetljive podatke, postoji velika verovatnoća da ćete u nekom trenutku koristiti Node.js iza scene.

Efikasna je, skalabilna i ima veliku zajednicu programera i održavalaca širom sveta.

Sada kada znaš šta je Node.js, istražimo neke razlike između pregledača i Node okruženja za izvršavanje.

Pregledački okruženje je dizajnirano posebno za front-end razvoj, pa pokreće JavaScript na strani klijenta. Nasuprot tome, Node okruženje je dizajnirano posebno za back-end razvoj, pa pokreće JavaScript na strani servera. Ovo takođe određuje API koje pružaju.

Na primer, sa komandne linije možete dobiti API od DOM, ali postoje ograničenja pristupa sistemu fajlova lokalnog računara. S druge strane, iz okruženja izvršavanja Node-a možete dobiti skoro sve sistemske resurse, uključujući sistem fajlova, ali ne i DOM.

Oba okruženja JavaScript pružaju globalni objekat. U okruženju izvršavanja JavaScript pretraživača, on se zove `window`. Ovaj objekat `window` pruža pristup funkcijama vezanim za pretraživač, kao što su pristup metodama za upravljanje DOM, upravljanje kolačićima (cookies) i rukovanje događajima pretraživača. U okruženju Node.js, ovaj globalni objekat se zove `global`. Objekat `global` pruža pristup specifičnim funkcijama Node.js, kao što su ugrađeni moduli za rad sa lokalnim fajlovima, mrežama i interakciju sa operativnim sistemom.

Na kraju, druga važna razlika je što možete odabrati verziju Node.js koju želite koristiti na svom serveru. Međutim, nemate kontrolu nad verzijom okruženja pregledača koju će vaši korisnici koristiti za posetu vašoj veb stranici.

Node.js je postao izuzetno važan za programere širom sveta. To je moćan, skalabilan i široko korišćen alat koji je promenio web razvoj tako što omogućava programerima da koriste JavaScript za izgradnju prednjeg (front-end) i zadnjeg (back-end) dela kompletnih aplikacija. Ovo je učinilo proces razvoja mnogo lakšim jer programeri ne moraju da nauče novi programski jezik za back-end razvoj.
