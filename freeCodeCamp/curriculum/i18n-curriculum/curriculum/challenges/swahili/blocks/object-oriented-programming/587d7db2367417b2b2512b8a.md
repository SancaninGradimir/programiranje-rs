# --description--

U prethodnom izazovu, `bird` je imao javni atribut `name`. Smatra se da je javno jer ga je moguće pristupiti i menjati izvan definicije `bird`.

```js
bird.name = "Duffy";
```

Stoga, bilo koji deo vašeg koda može lako da promeni ime ``bird`` u bilo koju vrednost. Zamislite stvari kao što su lozinke i bankovni računi koji mogu lako da se izmene bilo kojim delom vašeg koda. To može izazvati mnogo problema.

Najlakši način da se ovaj javni atribut učini privatnim je kreiranje parametra unutar funkcije komponente. Ovo menja opseg tog parametra na unutrašnjost funkcije komponente umesto da bude dostupan globalno. Na ovaj način, parametar se može pristupiti i modifikovati samo iz metoda koje su takođe unutar funkcije komponente.

```js
function Bird() {
  let hatchedEgg = 10;

  this.getHatchedEggCount = function() { 
    return hatchedEgg;
  };
}
let ducky = new Bird();
ducky.getHatchedEggCount();
```

Ovo `getHatchedEggCount` je jedinstveni način, jer ima pristup privatnom parametru `hatchedEgg`. Ovo je moguće zato što `hatchedEgg` je deklarisano u istom kontekstu kao i `getHatchedEggCount`. U JavaScript, funkcija uvek ima pristup kontekstu u kojem je kreirana. Ovo se zove `closure`.
