# --interactive--

Rekurzija je moćan mehanizam koji vam omogućava ponovno pozivanje funkcije sve dok ne stignete do osnovnog slučaja (baze). Za razliku od standardne petlje, rekurzija vam omogućava da obrađujete nešto sa nepoznatom dubinom, kao što su duboko ugnježdeni elementi ili strukture, ili sistem fajlova. Međutim, možete je koristiti i za jednostavnije zadatke, kao što je brojanje nazad od određenog broja.

Kreiraćemo funkciju koja radi upravo tako. Nazivaćemo našu funkciju `recursiveCountdown`, i ona zahteva da primi broj. Napravićemo da odštampa ovaj broj u konzolu:

:::interactive_editor

```js
const recursiveCountdown = (number) => {
  console.log(number);
};

recursiveCountdown(5);
```

:::

Sada ako pozovemo i prosledimo broj 5, videćemo taj broj ispisano na našem terminalu. Ali ništa drugo se ne dešava – i broj 5 definitivno nije brojač koji pada.

Pre nego što počnemo da gradimo rekurzivni deo naše funkcije, moramo prvo postaviti osnovni slučaj. Ako ne postavite osnovni slučaj, vaš kod će nastaviti da radi sve dok ne pređe dodeljenu memoriju i izazove pad.

:::interactive_editor

```js
const recursiveCountdown = (number) => {
  if (number < 1) {
    return;
  }
  console.log(number);
};

recursiveCountdown(5);
```

:::

Za naše osnovne uslove, želimo da proverimo/prebrojimo ako je broj manji od 1. Kada dođemo do tog podrazumevanog stanja, možemo se vratiti kako bismo prekinuli izvršavanje funkcije.

Sada smo sigurno postavili osnovni slučaj, pa možemo implementirati rekurziju. Ključna stvar koja čini funkciju rekurzivnom je što ona poziva samu sebe tokom izvršavanja. U ovom slučaju, želimo da pozovemo funkciju nakon štampanja broja. Ali da bismo brojali nadole, naš novi broj mora biti jedan manji:

:::interactive_editor

```js
const recursiveCountdown = (number) => {
  if (number < 1) {
    return;
  }
  console.log(number);
  recursiveCountdown(number - 1);
};

recursiveCountdown(5); 
```

:::

Ovo će ispisati brojeve 5, 4, 3, 2 i 1 na konzoli.

Dobijamo naše pet brojeva! Ali šta ako bismo hteli da brojimo uzlazno umesto toga? Umesto pisanja potpuno nove funkcije, možemo promeniti strukturu našeg posta sa rekurzivnim pozivom:

:::interactive_editor

```js
const recursiveCountdown = (number) => {
    if (number < 1) {
        return;
    }
    recursiveCountdown(number - 1);
    console.log(number);
  };

recursiveCountdown(5);
```

:::

Ovo će ispisati brojeve 1, 2, 3, 4 i 5 na konzolu.

Ali, zašto to radi? Da biste ovo razumeli, morate da razumete *call stack*. Call stack je način na koji JavaScript prati i rešava pozive funkcija. Stak funkcioniše kao Last-In-First-Out (LIFO) red. Da biste bolje razumeli, dodajmo objavu u našu funkciju:

:::interactive_editor

```js
const recursiveCountdown = (number) => {
  console.log(`Function execution started for number: ${number}`);
  if (number < 1) {
    console.log(`Base case reached, begin resolving stack`);
    return;
  }
  console.log(`Calling recursiveCountdown with number: ${number - 1}`);
  recursiveCountdown(number - 1);
  console.log(`Function execution completed for number: ${number}`);
};

recursiveCountdown(5);
```

:::

Dodali smo četiri važne informacije ovde. Prvi događaj se izvršava kada počinje poziv funkcije. Treći događaj se izvršava pre nego što se pozove rekurzivna funkcija. I četvrti događaj se izvršava kada je izvršavanje funkcije završeno. Rezultat je:

```md
Function execution started for number: 5
Calling recursiveCountdown with number: 4
Function execution started for number: 4
Calling recursiveCountdown with number: 3
Function execution started for number: 3
Calling recursiveCountdown with number: 2
Function execution started for number: 2
Calling recursiveCountdown with number: 1
Function execution started for number: 1
Calling recursiveCountdown with number: 0
Function execution started for number: 0
Base case reached, begin resolving stack
Function execution completed for number: 1
Function execution completed for number: 2
Function execution completed for number: 3
Function execution completed for number: 4
Function execution completed for number: 5
```

Ali kako se ovo radi? Ovde ulazi steka poziva (call stack). Kada pomenemo `recursiveCountdown(5)`, taj poziv funkcije dodaje se na call stack.

Kada taj poziv funkcije stigne do mesta pozivanja `recursiveCountdown(4)`, mora da stane i sačeka te rezultate. Istovremeno, `recursiveCountdown(4)` dodaje se u call stack, iznad `recursiveCountdown(5)`.

Kada taj poziv funkcije stigne do sekcije za pozivanje `recursiveCountdown(3)`, mora da se zaustavi i sačeka te rezultate. Istovremeno, `recursiveCountdown(3)` se dodaje u *call stack*, iznad `recursiveCountdown(4)`.

Kada taj poziv funkcije stigne do mesta koje poziva `recursiveCountdown(2)`, mora da stane i sačeka te rezultate. Istovremeno, `recursiveCountdown(2)` se dodaje na pilu poziva (call stack), iznad `recursiveCountdown(3)`.

Kada taj poziv funkcije dođe do tačke pozivanja `recursiveCountdown(1)`, mora da stane i sačeka te rezultate. Istovremeno, `recursiveCountdown(1)` se dodaje u *call stack*, iznad `recursiveCountdown(2)`.

Na kraju, kada taj poziv funkcije stigne do sekcije pozivanja `recursiveCountdown(0)`, mora da se zaustavi i sačeka ti rezultati. Istovremeno, `recursiveCountdown(0)` se dodaje u call stack (stak slova poziva), iznad `recursiveCountdown(1)`.

Ali `recursiveCountdown(0)` ne poziva drugu funkciju – doseže naš bazni status, gde se vraća rano. Zato što je izvršenje te funkcije završeno, poziv funkcije može se smatrati "rešenim". Kada je poziv rešen, uklanja se sa stack-a.

Sada `recursiveCountdown(1)` više ne mora da čeka taj poziv – nalazi se na steku i može nastaviti izvršavanje.

`recursiveCountdown(1)` se rešava, uklanja sa steka, i omogućava `recursiveCountdown(2)` da nastavi izvršenje.

`recursiveCountdown(2)` se izvukuva, uklanja se sa steka, i omogućava `recursiveCountdown(3)` da nastavi izvršavanje.

`recursiveCountdown(3)` se razrešava, uklanja sa steka, i omogućavajući `recursiveCountdown(4)` da nastavi izvršavanje.

`recursiveCountdown(4)` se rešava, uklanja sa steka, i omogućavajući `recursiveCountdown(5)` da nastavi izvršavanje.

Na `recursiveCountdown(5)` se rešava uklanjanjem sa steka. Naš call stack je sada prazan, pa je rekurzija završena!

Ovo je osnovni pregled kako rekurzija radi u JavaScript. To je teška koncepcija, i trebalo bi da testirate kod i informacije iz objave dok ne osetite se udobno sa ponašanjem steka poziva (call stack).

Za malo zanimljivu lekciju, razgovarali smo kako nedostatak osnovnog slučaja može uzrokovati da vaš kod padne kada mu nestane memorija. To je zato što rekurzija nastavlja dodavati sve više funkcionalskih poziva na *call stack* dok ne dođe do preljeva steka (*stack overflow*). Kao ime popularne programerske zajednice.
