# --description--

Ažuriranje objekata u React state-u može biti izazovno ako ste navikli da direktno menjate vrednosti svojstava komponente.

React koristi stanje kao nepromenljivo, što znači da ga ne bi trebalo mijenjati direktno.

Pogledajmo šta se dešava kada pokušate direktno da promenite objekte u React state-u, a zatim ćemo analizirati ispravan način za to.

Pretpostavimo da imate nešto u stanju vaše komponente što predstavlja profil korisnika, i želite da korisnik ispravi svoj uzrast:

```jsx
import { useState } from "react";

function Profile() {
  const [user, setUser] = useState({
    name: "John Doe",
    age: 31,
    city: "LA",
  });

  // Change user age directly
  const handleAgeChange = (e) => {
    user.age = e.target.value;
    console.log(user);
  };

  return (
    <div>
      <h1>User Profile</h1>
      <p>Name: {user.name}</p>
      <p>Age: {user.age}</p>
      <p>City: {user.city}</p>

      <h2>Update User Age </h2>
      <input type="number" value={user.age} onChange={handleAgeChange} />
    </div>
  );
}

export default Profile;
```

Ovaj kod neće raditi jer direktno menja svojstvo `user.age`.

Iako će `console.log(user)` prikazati novu vrednost na konzoli, React neće ponovo prikazivati taj deo radi prikazivanja promena na korisničkom interfejsu jer niste koristili funkciju za postavljanje (setter), `setUser`.

Da biste direktno ažurirali objekat u state-u, potrebno je da koristite funkciju za postavljanje kako biste napravili novi objekat sa ažuriranim vrednostima. Na primer:

```js
const handleAgeChange = (e) => {
  setUser({
    age: e.target.value,
  });
};
```

To funkcioniše. Ali ako sada pogledate stranicu, starost korisnika je ažurirana, ali vrednosti imena i grada nestaju.

Ovo je zato što novi objekat koji ste prosledili funkciji za postavljanje je imao samo par ključ/vrednost za `age`.

Da biste to sprečili, prvo možete napraviti kopiju postojećeg objekta, a zatim izmeniti samo svojstvo koje želite da promenite, u ovom slučaju `age`.

Da biste to uradili, možete proslediti specifičnu funkciju nazvanu funkcija za ažuriranje vašoj funkciji podešavanja, `setUser`. Funkcija za ažuriranje prima čekajući state kao argument, ovde, ona se zove `prevUser`, i treba da vrati sledeći state:

```js
const handleAgeChange = (e) => {
  setUser((prevUser) => {
    const updatedUser = { ...prevUser, age: e.target.value };
    console.log('Previous State:', prevUser);
    console.log('Updated State:', updatedUser);
    return updatedUser;
  });
};
```

Kao što vidite, kreiramo novi objekat korisnika nazvan `updatedUser` koristeći spread sintaksu za kopiranje čekajućeg objekta korisnika, `...prevUser`. Zatim ažuriramo godine na osnovu unosa forme i vraćamo `updatedUser` u funkciji kao sledeći status.

Sada vaš projekat radi kako je očekivano, a ažuriranja unosa starosti ne menjaju korisničko ime niti naziv grada. Takođe možete videti početna i ažurirana stanja u konzoli.

Ovo je najbolji način da se nešto ispravi u stanju, posebno kada ažurira sve atribute/svojstva.

Da biste ispravili preostale osobine za `name` i `city`, možete napisati kao funkcije diferenciranja i povezati ih sa njihovim ulaznim delovima:

```js
const handleNameChange = (e) => {
  setUser((prevUser) => ({
    ...prevUser,
    name: e.target.value,
  }));
};

const handleCityChange = (e) => {
  setUser((prevUser) => ({
    ...prevUser,
    city: e.target.value,
  }));
};
```

Ili ih možete spojiti u jednu funkciju za podešavanje na ovaj način:

```js
const handleChange = (e) => {
  const { name, value } = e.target;
  setUser((prevUser) => ({
    ...prevUser,
    [name]: value,
  }));
};
```

Da bi ovo funkcionisalo, svaki deo unosa mora imati atribut `name`.

Evo kompletnog koda sada:

```jsx
import { useState } from "react";

function Profile() {
  const [user, setUser] = useState({ name: "John Doe", age: 31, city: "LA" });

  const handleChange = (e) => {
    const { name, value } = e.target;

    setUser((prevUser) => ({...prevUser, [name]: value}));
  };

  return (
    <div>
      <h1>User Profile</h1>
      <p>Name: {user.name}</p>
      <p>Age: {user.age}</p>
      <p>City: {user.city}</p>

      <h2>Update User Age </h2>
      <input type="number" name="age" value={user.age} onChange={handleChange} />

      <h2>Update User Name </h2>
      <input type="text" name="name" value={user.name} onChange={handleChange} />

      <h2>Update User City </h2>
      <input type="text" name="city" value={user.city} onChange={handleChange} />
    </div>
  );
}

export default Profile;
```
