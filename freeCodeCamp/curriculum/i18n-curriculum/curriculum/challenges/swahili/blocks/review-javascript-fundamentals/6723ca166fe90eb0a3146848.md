## Komponenta sekvence znakova i putanje za `toString()`

- **Definicija**: Literalni niz znakova koristi se za predstavljanje sekvence znakova. Literalni nizovi znakova kreiraju se korišćenjem komponentne funkcije za `String`, koja pokriva osnovnu vrednost unutar objekta.

:::interactive_editor

```js
const greetingObject = new String("Hello, world!");

console.log(typeof greetingObject); // "object"
```

:::

- **Metoda za `toString()`**: Ova metoda konvertuje vrednost u njen predstavnik kao niz znakova. To je metoda koju možete koristiti za brojeve, tačno ili netačno vrednosti (boolean), nizove podataka i objekte.

:::interactive_editor

```js
const num = 10;
console.log(num.toString()); // "10"

const arr = [1, 2, 3];
console.log(arr.toString()); // "1,2,3"
```

:::

Ovaj put prima opcioni brojčani sistem koji je broj od 2 do 36. Ova baza predstavlja brojčani sistem, kao što su baza 2 za binarni ili baza 8 za oktalni. Ako nije navedena baza, koristi se baza 10, odnosno decimalna.

:::interactive_editor

```js
const num = 10;
console.log(num.toString(2)); // "1010"(binary)
```

:::

## Komponenta broja

- **Definicija**: Komponenta `Number` se koristi za kreiranje objekta broja. Objekat broja ima nekoliko korisnih svojstava i metoda, kao što su metode `isNaN` i `toFixed`. Često ćete koristiti komponentu `Number` da biste konvertovali druge tipove podataka u tip broja.

:::interactive_editor

```js
const myNum = new Number("34");
console.log(typeof myNum); // "object"

const num = Number('100');
console.log(num); // 100

console.log(typeof num); // number
```

:::

## Najbolje prakse za imenovanje parametara i funkcija

- **camelCasing**: Po ugovoru, programeri za JavaScript koriste camel casing za imenovanje varijabli i funkcija. Camel casing je kada prva reč ima samo mala slova, a sledeće reči počinju velikim slovom. Primer `isLoading`.

- **Pribavljanje imena sa tačnom ili netačnom vrednošću**: Za kriterijume tačnih ili netačnih vrednosti, uobičajena je metoda korišćenja pomoćnih glagola kao što su "is", "has" ili "can".

```js
let isLoading = true;
let hasPermission = false;
let canEdit = true;
```

- **Imenovanje funkcija**: Za funkcije, ime mora jasno da pokaže šta ta funkcija radi. Za funkcije koje vraćaju vrednost tačno ili netačno (često nazivane predikati), možete koristiti prefikse "is", "has" ili "can". Za funkcije koje dohvaćaju podatke, uobičajeno je početi sa rečju "get". Za funkcije koje postavljaju podatke, uobičajeno je početi sa rečju "set". Za funkcije za rukovanje događajima (event handler), možete koristiti prefiks "handle" ili sufiks "Handler".

```js
function getUserData() { /* ... */ }

function isValidEmail(email) { /* ... */ }

function getProductDetails(productId) { /* ... */ }

function setUserPreferences(preferences) { /* ... */ }

function handleClick() { /* ... */ }
```

- **Ime davanje promenljivim unutar petlja**: Kada imenujete promenljive koje se koriste kao petlje, uobičajeno je koristiti jedan karakter poput `i`, `j`, ili `k`.

```js
for (let i = 0; i < array.length; i++) { /* ... */ }
```

## Rad sa redovima podataka sa nedostajućim vrednostima

- **Definicija**: Moguće je imati redove podataka sa praznim jazovima. Ovi jazovi se tumače kao delovi koji ne sadrže ništa unutra. Ovo se razlikuje od delova redova podataka sa vrednošću `undefined`. Ovakve vrste redova podataka poznate su kao redovi podataka sa jazovima.

:::interactive_editor

```js
const sparseArray = [1, , , 4];
console.log(sparseArray.length); // 4
```

## Linters na Formatters

- **Linters**: Linter je alat za analizu izvornog koda koji otkriva greške u programiranju, bugove, stilsko kršenja i sumnjive strukture. Primer popularnog lintera je ESLint.
- **Formatters**: Formateri su alati koji automatski formatiraju vaš kod kako bi pratio određeni stilski vodič. Primer popularnog formatera je Prettier.

## Upravljanje memorijom

- **Objašnjenje**: Upravljanje memorijom je proces kontrole memorije, alociranja kada je potrebno i oslobađanja kada više nije upotrebljivo. JavaScript koristi automatsko upravljanje memorijom. To znači da JavaScript (posebno motor JavaScript u vašem web pregledaču) upravlja alokacijom i oslobađanjem memorije umesto vas. Nosite potrebu da eksplicitno oslobađate memoriju u svom kodu. Ovaj automatski proces se često naziva "sakupljanje smeća".

## Instalacija

- **Definicija**: Binding je funkcija koja ima pristup parametrima u spoljnom opsegu (kontekstu jezika), čak i nakon što vanjska funkcija završi.

:::interactive_editor

```js
function outerFunction(x) {
  let y = 10;
  function innerFunction() {
    console.log(x + y);
  }
  return innerFunction;
}

let closure = outerFunction(5);
closure(); // 15
```

## Ključni pojam za `var` i Hoisting

- **Objašnjenje**: `var` je bio prvobitni način deklarisanja parametara pre 2015. Međutim, postojali su određeni problemi vezani za `var` u vezi sa opsegom, ponovnim deklarisanjem i ostalim stvarima. Zato moderne aplikacije za JavaScript koriste `let` i `const` umesto toga.
- **Ponovno deklarisanje parametara za `var`**: Ako pokušate ponovo da deklarišete parametar koristeći `let`, dobićete `SyntaxError`. Ali za `var`, dozvoljeno je ponovno deklarisanje parametara.

```js
// Uncaught SyntaxError: Identifier 'num' has already been declared 
let num = 19;
let num = 18;

var myNum = 5;
var myNum = 10; // This is allowed and doesn't throw an error

console.log(myNum) // 10
```

- **`var` u opsegu**: Varijable deklarisane za `var` unutar bloka (kao deklaracija `if` ili petlja `for`) i dalje mogu biti dostupne izvan tog bloka.

:::interactive_editor

```js
if (true) {
  var num = 5;
}
console.log(num); // 5
```

:::

- **Hoisting**: Ovo je uobičajeno ponašanje za JavaScript pomeranjem deklaracija varijabli na vrh svog opsega tokom faze kompilacije pre izvršavanja koda. Kada deklariš varijablu koristeći ključnu reč `var`, JavaScript pomera tu deklaraciju na vrh svog opsega.

:::interactive_editor

```js
console.log(num); // undefined
var num = 5;
console.log(num); // 5
```

:::

Kada deklarišete funkciju pomoću sintakse deklaracije funkcije, ime i telo funkcije se podižu. To znači da možete pozvati funkciju pre nego što je deklarišete u svom kodu.

:::interactive_editor

```js
sayHello(); // "Hello, World!"

function sayHello() {
  console.log("Hello, World!");
}
```

:::

Deklaracije promenljivih kreirane sa `let` ili `const` su podignute (hoisted), ali nisu inicijalizovane, i ne možete pristupiti im pre stvarnog deklarisanja u vašem kodu. Ovo ponašanje se često naziva "privremena mrtva zona".

:::interactive_editor

```js
console.log(num); // Throws a ReferenceError
let num = 10;
```

:::

## Rad sa Importsima, Exportsima i Modulima

- **Moduli**: Ovo je jedinica koda koja uključuje funkcije, klase objekata ili povezane varijable. Da biste kreirali modul, napišite svoj kod JavaScript u zasebnoj datoteci.
- **Exports**: Sve varijable, funkcije ili klase koje želite da budu dostupne drugim delovima vaše aplikacije moraju biti eksplicitno izbačene korišćenjem ključne reči `export`. Postoje dve vrste export: one sa imenom export i ona iz export od default.
- **Imports**: Da biste koristili stvari iz drugih delova vaše aplikacije, morate uvesti (import) korišćenjem ključne reči `import`. Vrste import su one sa imenom import, ona iz import od default, i one iz namespace-a import.

```js
// Within a file called math.js, we export the following functions:

// Named export
export function add(num1, num2) {
  return num1 + num2;
}

// Default export
export default function subtract(num1, num2) {
  return num1 - num2;
}

// Within another file, we can import the functions from math.js.

// Named import - This line imports the add function.
// The name of the function must exactly match the one exported from math.js.
import { add } from './math.js';

// Default import - This line imports the subtract function.
// The name of the function can be anything.
import subtractFunc from './math.js';

// Namespace import - This line imports everything from the file.
import * as Math from './math.js';

console.log(add(5, 3)); // 8
console.log(subtractFunc(5, 3)); // 2
console.log(Math.add(5, 3)); // 8
console.log(Math.subtract(5, 3)); // 2
```
