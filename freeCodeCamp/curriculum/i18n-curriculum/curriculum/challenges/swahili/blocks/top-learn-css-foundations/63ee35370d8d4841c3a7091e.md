# --description--

Odnosi nam omogućavaju da povežemo više čvorova na različite načine i da ih povežemo u grupama ili sekvencijama, jer pokazuju odnos između čvorova. Postoje četiri tipa odnosa uopšteno, ali za sada ćemo vam pokazati samo relacije porekla (lineage), predstavljene u CSS i na jednoj poziciji među izborima. Relacije porekla će odabrati elemente koji odgovaraju samo poslednjem izboru ako takođe imaju roditelja (roditelj, deda itd.) koji odgovara prethodnom izboru.

Dakle, nešto kao `.ancestor .child` će izabrati element sa klasom objekata `child` ako ima roditelja sa klasom objekata `ancestor`. Drugi način razmišljanja je da će dete biti izabrano samo ako je postavljeno unutar `ancestor`, bez obzira na koliko je duboko. Pogledajte primer ispod brzo i vidite da li možete reći koji elementi će biti izabrani prema pravilu CSS koje je dato:

```html
<!-- index.html -->

<div class="ancestor"> <!-- A -->
  <div class="contents"> <!-- B -->
    <div class="contents"> <!-- C -->
    </div>
  </div>
</div>

<div class="contents"></div> <!-- D -->
```

```css
/* styles.css */

.ancestor .contents {
  /* some declarations */
}
```

U ovom gornjem primeru, prvih dva elementa sa klasom objekata `contents` (`B` i `C`) će biti odabrana, ali poslednji element (`D`) neće biti odabran. Da li su vaše pretpostavke bile tačne?

Ne postoji ograničenje broja entiteta koje možete dodati u pravilo, tako da će `.one .two .three .four` biti potpuno važeći. Ovo će izabrati element sa klasom objekata `four` ako ima roditelja sa klasom objekata `three`, i ako taj roditelj ima svog roditelja sa klasom objekata `two`, i tako dalje. Generalno, trebalo bi izbegavati pokušaj odabira elemenata koji zahtevaju ovaj nivo ugnježdenosti, jer može biti složen i dugačak, te može izazvati probleme kada je reč o specifičnim pitanjima.
