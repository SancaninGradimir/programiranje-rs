# --description--

Videla si zaista generički tip u prošloj lekciji: `Array<string>`. Ali obično koristiš generičke tipove u funkcijama. Zapravo, mogu se smatrati kao poseban parametar koji daje funkciji da bi kontrolisao ponašanje definisanja tipa te funkcije.

Vraćamo se na naš početni primer funkcije za dobijanje slučajne vrednosti iz niza podataka:

```js
const getRandomValue = (array: string[]): string => {
  return array[Math.floor(Math.random() * array.length)];
}
```

Naša funkcija prima niz znakova i vraća niz znakova. Ali šta ako bismo hteli da prosledimo niz brojeva i vratimo brojeve?

Možete koristiti tip veze:

```js
const getRandomValue = (array: (string|number)[]): (string|number) => {
  return array[Math.floor(Math.random() * array.length)];
}
const val = getRandomValue([1, 2, 4])
```

Međutim, na ovaj način, iako smo prošli samo red brojeva, TypeScript pokazuje da razlika `val` može biti broj ILI niz slova.

Umesto toga, možete definisati generički tip za funkciju:

```js
const getRandomValue = <T>(array: T[]): T => {
  return array[Math.floor(Math.random() * array.length)];
}
const val = getRandomValue([1, 2, 4])
```

Postoji nekoliko važnih delova u našem novom primeru. Prvo, sintaksa `<T>` govori TypeScript da definiše tip generičkog `T` za funkciju. `T` je uobičajeno ime za generičke tipove, ali ga možete nazvati bilo kojim imenom koje želite (unutar ugovora o parametrima JavaScript).

Zatim, kažemo TypeScript da je nizski parametar niz vrednosti koje odgovaraju generičkom tipu, a vraćena vrednost je pojedinačni element tog istog tipa. Ovo omogućava TypeScript da pravilno utvrdi da je `val` broj, jer niz koji smo prosledili sadrži samo brojeve.

Ali šta je sa elementima koje ne kontrolišete? Istražimo put `document.querySelector()`:

```js
const email = document.querySelector("#email");
console.log(email.value);
```

Zato što izvršavamo pretraga po selektoru ID-a, TypeScript ne zna kakav je tip elementa koji tražimo. Zna da je `Element` nekog tipa, ali naš pokušaj dobijanja svojstva za `value` će vratiti grešku jer interfejs elementa višeg nivoa nema to svojstvo.

Srećom, možemo usvojiti vrstu predloga po operativnom/proceduralnom pozivu:

```js
const email = document.querySelector<HTMLInputElement>("#email");
console.log(email.value);
```

Pogledajte kako ponovo koristimo sintaksu uglastih zagrada (`<HTMLInputElement>`), ali ovog puta u pozivu funkcije. Ovo govori TypeScript da će element koji očekujemo dobiti biti element od `input`, i više ne dobijamo grešku atributa vrednosti jer elementi unosa imaju taj atribut. Naravno, `querySelector` može vratiti `null` ako element nije pronađen, a TypeScript će nam ovde dati grešku kolektora jer nećemo tretirati taj slučaj.

Ali ćeš naučiti o tome na sledećoj lekciji.
