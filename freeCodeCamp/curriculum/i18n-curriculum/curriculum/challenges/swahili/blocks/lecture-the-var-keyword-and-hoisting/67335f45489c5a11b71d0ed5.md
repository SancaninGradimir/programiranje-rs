# --interactive--

Hoisting je karakteristika u JavaScript koja često zbunjuje početnike, ali razumevanje može pomoći da se izbegnu male greške u vašem kodu. Jednostavnim rečima, hoisting je uobičajena karakteristika JavaScript koja pomera deklaracije promenljivih na vrh svog opsega tokom faze kompilacije pre nego što se kod izvrši.

Da biste razumeli hoisting, važno je znati da JavaScript se izvršava u dve faze: faza deklaracije i faza izvršenja. Tokom faze deklaracije, *engine* za JavaScript prolazi kroz vaš kod i priprema memorijsko mesto za varijable i funkcije. Ovde *hoisting* počinje da radi.

Početi sa *hoisting*-om varijabli, kada deklariš varijablu koristeći reč ``var``, `JavaScript` podiže deklaraciju te varijable na vrh svog opsega. Međutim, važno je razumeti da se podiže samo deklaracija, a ne inicijalizacija. To znači da možete koristiti varijablu u svom kodu pre nego što je deklarišete, ali će njena vrednost biti ``undefined`` dok ne dodelite stvarnu vrednost.

:::interactive_editor

```js
console.log(x); // undefined
var x = 5;
console.log(x); // 5
```

:::

U ovom kodu, iako koristimo `x` pre deklarisanja, ne dobijamo grešku; umesto toga, dobijamo `undefined`. Ovo je zato što JavaScript pomera deklaraciju `var x` unutar svog opsega, ali ne i inicijalizaciju `x = 5`. Kao da je kod prepisan ovako:

:::interactive_editor

```js
var x;
console.log(x); // undefined
x = 5;
console.log(x); // 5
```

:::

Funkcijski hoisting radi malo drugačije. Kada deklarišete funkciju koristeći sintaksu deklaracije `function`, ime funkcije i celo joj telo se podiže (hoists). To znači da možete pozvati funkciju pre nego što je deklarišete u vašem kodu. Evo primera funkcijskog hoistinga:

:::interactive_editor

```js
sayHello(); // "Hello, World!"

function sayHello(){
  console.log("Hello, World!");
}
```

:::

U ovom slučaju, možemo pozvati `sayHello()` pre njegove deklaracije jer je cela funkcija pomerena iznad svog opsega.

Važno je razumeti da hoisting radi drugačije od deklaracije `let` i `const` koje su uvedene u ES6.

```js
console.log(y); // Throws a ReferenceError
let y = 10;
```

Ove deklaracije se pomeraju, ali nisu inicijalizovane i ne možete ih pronaći pre stvarnog deklarisanja u vašem kodu. Ovo se često naziva vremenska mrtva zona (temporal dead zone).

Razumeti `hoisting` može pomoći u pisanju čistijeg, predvidljivijeg koda. Međutim, oslanjanje na `hoisting` može učiniti vaš kod teškim za čitanje i zahtevati više održavanja. Kao najbolja praksa, savetuje se da deklarišete svoje varijable u njihovom opsegu i funkcije pre nego što ih koristite, bez obzira na `hoisting`. Ovo čini ponašanje vašeg koda jasnijim i lakšim za druge, uključujući vas same kasnije, da razumeju.
