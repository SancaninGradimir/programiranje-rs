# --interactive--

Modifikatori, koji se često nazivaju i „flagovi“, menjaju ponašanje regularnog izraza. Prisetićemo se primera iz prethodne lekcije:

:::interactive_editor

```js
const regex = /freeCodeCamp/;
console.log(regex.test("freeCodeCamp")); // true
console.log(regex.test("freeCodeCamp is great")); // true
console.log(regex.test("I love freeCodeCamp")); // true
console.log(regex.test("freecodecamp")); // false
console.log(regex.test("FREECODECAMP")); // false
console.log(regex.test("free")); // false
console.log(regex.test("code")); // false
console.log(regex.test("camp")); // false
```

:::

Kao što se sećate, sekvenca malih slova i sekvenca velikih slova `freeCodeCamp` nije odgovarala obrascu. To je zato što standardni izraz podrazumevano uzima u obzir velika i mala slova.

Ali možemo li reći regularnom izrazu da ne razlikuje velika i mala slova? Da, postoji modifikator za to. Zastavica `i` čini da regex ne pravi razliku između velikih i malih slova. Kako je možemo koristiti? Zastavice se pišu posle završne kose crte u regularnom izrazu:

```js
const regex = /freeCodeCamp/i;
```

Pogledajte promene u regularnom izrazu na prvoj liniji. Sada možemo videti kako ovo menja stvari:

:::interactive_editor

```js
const regex = /freeCodeCamp/i;

console.log(regex.test("freeCodeCamp")); // true
console.log(regex.test("freeCodeCamp is great")); // true
console.log(regex.test("I love freeCodeCamp")); // true
console.log(regex.test("freecodecamp")); // true
console.log(regex.test("FREECODECAMP")); // true
console.log(regex.test("free")); // false
console.log(regex.test("code")); // false
console.log(regex.test("camp")); // false
```

:::

Pošto naš regularni izraz sada ne razlikuje velika i mala slova, i potpuno mala i potpuno velika verzija niza prolaze test. Ovo takođe funkcioniše i za nizove koji sadrže kombinaciju velikih i malih slova:

:::interactive_editor

```js
const regex = /freeCodeCamp/i;

console.log(regex.test("dO yOu LoVe fReEcOdEcAmP?")); // true
```

:::

Postoji još mnogo zastavica koje možete koristiti. Zastavica `g`, odnosno globalni modifikator, omogućava da se vaš regularni izraz poklopi sa obrascem više od jednog puta.

Istražite kako ovo utiče na naš kod. Videćete da smo ostali sa zastavicom `i` – obični korisnik može koristiti više zastavica (koliko god je potrebno) da bi postigao željeno ponašanje:

```js
const regex = /freeCodeCamp/gi;
```

Sačekajte malo... šta je ovo? Izgleda da globalni modifikator ne prolazi kroz neke od naših sekvenci kao što bi trebalo, već:

:::interactive_editor

```js
const regex = /freeCodeCamp/gi;

console.log(regex.test("freeCodeCamp")); // true
console.log(regex.test("freeCodeCamp is great")); // false
console.log(regex.test("I love freeCodeCamp")); // true
console.log(regex.test("freecodecamp")); // false
console.log(regex.test("FREECODECAMP")); // true
console.log(regex.test("free")); // false
console.log(regex.test("code")); // false
console.log(regex.test("camp")); // false
```

:::

Zašto? Zato što globalni modifikator čini regularni izraz "stateful". To znači da pamti gde je prethodno pronašao poklapanje. Dakle, kada pronađe prvi niz `freeCodeCamp`, pamti da je poklapanje počelo na indeksu `0`.

Zatim testiramo nad nizom `freeCodeCamp is great`, ali on ne počinje od indeksa `0`. Regularni izraz "zna" da je već pronašao poklapanje na indeksu `0`, pa iako je ovo drugi niz, počinje pretragu od kraja prethodnog poklapanja.

`freeCodeCamp` ima dužinu od karaktera `12`, pa je placeholder u `0` se završava na elementu `11`. Placeholder će se nastaviti na elementu `12`. I pošto `is great` ne odgovara `freeCodeCamp`, vraća `false`.

Zatim, pošto ne pronađe poklapanje, "resetuje" svoje stanje i sledeću pretragu ponovo započinje od indeksa `0`.

Ako promenimo naše rekorde tako da sekvenca na `0` bude odmah praćena drugom odgovarajućom sekvencom koja je kasnija od primera `11`:

:::interactive_editor

```js
const regex = /freeCodeCamp/gi;

console.log(regex.test("freeCodeCamp")); // true
console.log(regex.test("I loooooooove freeCodeCamp")); // true
```

:::

Standardni izraz kada je global, dobija novu funkciju nazvanu `lastIndex`. Ako uzmemo naš prethodni kod, hajde da istražimo kako ova funkcija radi:

:::interactive_editor

```js
const regex = /freeCodeCamp/gi;

console.log(regex.lastIndex); // 0
console.log(regex.test("freeCodeCamp")); // true
console.log(regex.lastIndex); // 12
console.log(regex.test("freeCodeCamp is great")); // false
console.log(regex.lastIndex); // 0
console.log(regex.test("I love freeCodeCamp")); // true
console.log(regex.lastIndex); // 19
console.log(regex.test("freecodecamp")); // false
console.log(regex.lastIndex); // 0
console.log(regex.test("FREECODECAMP")); // true
console.log(regex.lastIndex); // 12
console.log(regex.test("free")); // false
console.log(regex.lastIndex); // 0
console.log(regex.test("code")); // false
console.log(regex.lastIndex); // 0
console.log(regex.test("camp")); // false
```

:::

Kada vidite ovaj primer, možete videti kako se stanje standardne upotrebe menja sa svakim test pozivom koristeći `lastIndex` za praćenje njegove prethodne tačnosti.

Global zastavica je dobra kada vam treba da dobijete mnogo podudaranja iz jedne sekvence. Ali ako testirate mnogo fajlova sa jednim regularnim izrazom, bolje je ostaviti zastavicu `g`.

Pre nego što naučite o sledećoj karakteristici, morate naučiti o anhkor tagovima. Ishara caret-a (`^`) na početku regularnog izraza znači "slično početku sekvence":

```js
const start = /^freecodecamp/i;
```

Simbol dolara (`$`) na kraju obične rečenice, znači "slično kraju sekvence":

```js
const end = /freecodecamp$/i;
```

Odvojite vreme za poređenje rezultata na desnoj strani:

:::interactive_editor

```js
const start = /^freecodecamp/i;
const end = /freecodecamp$/i;
console.log(start.test("freecodecamp")); // true
console.log(end.test("freecodecamp")); // true
console.log(start.test("freecodecamp is great")); // true
console.log(end.test("freecodecamp is great")); // false
console.log(start.test("i love freecodecamp")); // false
console.log(end.test("i love freecodecamp")); // true
console.log(start.test("have met freecodecamp's founder")); // false
console.log(end.test("have met freecodecamp's founder")); // false
```

:::

Pogledajte kako početni tag se poklapa samo na početku sekvence, a završni tag samo na kraju sekvence? Ali šta je sa poklapanjem kroz više linija? Istražimo to:

:::interactive_editor

```js
const start = /^freecodecamp/i;
const end = /freecodecamp$/i;
const string = `I really love
freecodecamp
it's my favorite`;
console.log(start.test(string)); // false
console.log(end.test(string)); // false
```

:::

Iako je `freecodecamp` sam na svom redu, ne izvodi nikakvu proveru. To je zato što podrazumevano anker tagovi traže početak i kraj celog niza.

Međutim, možete učiniti da regex obrađuje više linija sa zastavicom `m`, ili multi-linijskim modifikatorom. Stavićmo to u naš regularni izraz da vidimo šta dobijemo:

:::interactive_editor

```js
const start = /^freecodecamp/im;
const end = /freecodecamp$/im;
const string = `I really love
freecodecamp
it's my favorite`;
console.log(start.test(string)); // true
console.log(end.test(string)); // true
```

:::

Sada se svi poklapaju! Pošto se `freecodecamp` nalazi sam u svom redu, sidro početka odgovara početku tog reda, a sidro kraja odgovara kraju tog reda.

Na kraju, imaš flag `d` ili modifikator indeksa. Zapamti da je flag `i` za nerazlikovanje velikih i malih slova, pa je modifikator indeksa zahtevao drugačiji flag.

Zastavica `d` dodaje dodatne informacije objektu poklapanja. Dodajmo je našem regularnom izrazu:

```js
const regex = /freecodecamp/di;
const string = "we love freecodecamp isn't freecodecamp great?";
console.log(string.match(regex));
```

Rezultati su:

```js
// [
//   'freecodecamp',
//   index: 8,
//   input: "we love freecodecamp isn't freecodecamp great?",
//   groups: undefined,
//   indices: [
//     0: [8, 20],
//     groups: undefined
//   ]
// ]
```

Naš objekat poklapanja dobija novo svojstvo `indices`! Ovo svojstvo je niz od dva broja: prvi predstavlja indeks u originalnom nizu gde poklapanje počinje, a drugi indeks odmah nakon završetka poklapanja. Ovaj niz takođe ima dodatno svojstvo `groups`, namenjeno imenovanim grupama za hvatanje.

Postoji još nekoliko zastavica koje bi trebalo da znate, ali se ređe koriste u svakodnevnom kodu.

Prva je Unicode modifikator, odnosno zastavica `u`. Ona proširuje mogućnosti regularnog izraza tako da može da radi sa posebnim Unicode znakovima.

Naučićete više o klasama znakova u sledećoj lekciji, ali zastava `u` daje vam pristup specijalnim klasama kao što je `Extended_Pictographic`, slično mnogim emotikonima:

:::interactive_editor

```js
const regex = /🍎/u;

const str = "I have an apple 🍎";
console.log(regex.test(str)); // true
```

:::

Takođe postoji zastavica za `v`, koja povećava mogućnost kompatibilnosti sa Unicode-om.

Drugo je sticky modifikator, ili zastava od `y`. Sticky modifikator radi kao globalni modifikator, ali sa malim razlikama.

Glavna razlika je u tome što standardni globalni izraz počinje od `lastIndex` i traži drugi podudarnost u svim preostalim delovima sekvence, ali standardni 'sticky' izraz vraća `null` i ponovo inicijalizuje `lastIndex` do `0` ako nema trenutnog podudaranja na prethodnom `lastIndex`.

Poslednji je single-line modifikator, ili zastava `s`. Zapamtite da multiline modifikator dozvoljava početne i krajnje oznake da odgovaraju početku i kraju linije, umesto celokupnoj sekvenci.

Single-line modifikator omogućava wildkard karakteristiku, predstavljenu tačkom/znakom za periodu (`.`) u regex-u, da odgovara prekidima linija – jer tretira sekvencu kao jednu liniju teksta.

Postoji mnogo modifikatora ovakve vrste, ali zastavice za `i` i `g` su one koje ćete najčešće koristiti, i najvažnije je da ih zapamtite.
