# --description--

Hajde da naučimo kako da pronađemo ili zamenimo sva pojavljivanja obrasca u jednom stringu.

Ranije ste naučili o metodama `replace()` i `match()`, kao i o globalnoj zastavici `g`. Sada možete spojiti to znanje kako biste obradili sva pojavljivanja obrasca u stringu. Podsetimo se prethodnog primera:

```js
const regex = /freeCodeCamp/;
const match = "freeCodeCamp".match(regex);
console.log(match);
```

Na kitu chetu cha matokeo cha kufananisha:

```js
// [
//   'freeCodeCamp',
//   index: 0,
//   input: 'freeCodeCamp',
//   groups: undefined
// ]
```

Ali šta ako string sadrži više pojavljivanja `freecodecamp`? Pogledajmo kako se `match()` ponaša u tom slučaju. Uporedićemo ga i sa metodom `replace()`:

```js
const regex = /freecodecamp/;
const str = "freecodecamp is the best we love freecodecamp";
const matched = str.match(regex);
const replaced = str.replace(regex, "freeCodeCamp");
console.log(matched);
console.log(replaced);
```

Na matokeo ni haya:

```js
// [
//   'freecodecamp',
//   index: 0,
//   input: 'freecodecamp is the best we love freecodecamp',
//   groups: undefined
// ]
// freeCodeCamp is the best we love freecodecamp
```

Opa! `match()` je vratio samo prvo poklapanje, a `replace()` je zamenio samo prvo poklapanje. To je zato što obe metode podrazumevano rade samo nad prvim pojavljivanjem obrasca.

Srećom, to možete izbeći korišćenjem globalne zastavice na regularnom izrazu. Dodajmo je:

```js
const regex = /freecodecamp/g;
const str = "freecodecamp is the best we love freecodecamp";
const matched = str.match(regex);
const replaced = str.replace(regex, "freeCodeCamp");
console.log(matched);
console.log(replaced);
```

Na thibitisha matokeo:

```js
// [ 'freecodecamp', 'freecodecamp' ]
// freeCodeCamp is the best we love freeCodeCamp
```

Uspelo je! Naš poziv za izmenu promenio je ceo niz malih slova od `freecodecamp`, a naš metod sa `match()` je upoređen oba.

Što je zanimljivo ovde jeste da kada koristite globalni regex sa `match()`, gubite dodatne informacije o grupama za hvatanje i indikatorima sekvenci karaktera koje bi bile dostupne u redu podudaranja.

Srećom, ECMAScript 2019 je doneo dve nove metode: `matchAll()` i `replaceAll()`. Kao i njihove starije verzije, prihvataju string ili regularni izraz, a `replaceAll()` prima i drugi argument sa tekstom za zamenu.

Međutim, za razliku od prethodnih metoda, `replaceAll()` i `matchAll()` će nam vratiti grešku ako im date standardni izraz bez međunarizacije. Implementiraćemo naš kod koristeći ove nove metode:

```js
const pattern = "freecodecamp";
const str = "freecodecamp is the best we love freecodecamp";
const matched = str.matchAll(pattern);
const replaced = str.replaceAll(pattern, "freeCodeCamp");
console.log(matched);
console.log(replaced);
```

Prema našim rezultatima:

```js
// {}
// freeCodeCamp is the best we love freeCodeCamp
```

Dobre vesti! `replaceAll()` radio je kako smo hteli – promenio je sve instance od `freecodecamp` napisane malim slovima i verziju koja je pravilno u stilu camelCase.

Ali šta je ta prazna stvar? `matchAll()` vraća specijalni tip objekta nazvan `Iterator`, što konzola freeCodeCamp nije spremna da obradi.

Kada pogledamo na našoj konzoli pretraživača, `Iterator` ima način za `next()`, koji možemo pozvati da bismo dobili sledeću vrednost:

```js
// RegExpStringIterator {  }
//  <prototype>: RegExp String Iterator {
//    next: ƒ next(),
//    Symbol(Symbol.toStringTag): "RegExp String Iterator"
//    <prototype>: Object { ... }
//  }
```

Kliknite na `matched.next()`, i zapišite rezultate:

```js
// {
//   "done": false,
//   "value": [
//     0: "freecodecamp"
//     groups: undefined
//     index: 0
//     input: "freecodecamp is the best we love freecodecamp"
//   ]
// }
```

Tamo je naš red mečeva! `next()` vraća nam nešto sa dve vrednosti: `done`, što je `false` kada su dostupni više elemenata u ciklusu, i `value` što je vrednost koju sada ponavljamo. Stoga, pozivajući ga još jednom:

```js
const regex = /freecodecamp/g;
const str = "freecodecamp is the best we love freecodecamp";
const matched = str.matchAll(regex);
const replaced = str.replaceAll(regex, "freeCodeCamp");
console.log(matched);
console.log(replaced);
console.log(matched.next());
console.log(matched.next());
```

```js
// {
//   "done": false,
//   "value": Array [ "freecodecamp"]
// }
//
// {
//   "done": false,
//   "value": Array [ "freecodecamp"]
// }
```

Sačekaj, zašto kaže `done` i dalje je `false`? Trebalo bi da ima samo dve slike u redu, zar ne? Hajde da ga pozovemo treći put da vidimo šta dobijamo:

```js
// {
//   "done": false,
//   "value": Array [ "freecodecamp"]
// }
//
// {
//   "done": false,
//   "value": Array [ "freecodecamp"]
// }
//
// {
//   "done": true,
//   "value": undefined
// }
```

Sada je `done` konačno `true`, ali zašto je vrednost `undefined`? Zato što je iterator koji vraća `matchAll()` lenj. On ne pronalazi sva poklapanja odjednom, već tek kada pozovete `next()`.

Kada pronađe podudarnost, nije `done`. Kada ne uspe da pronađe podudarnost i vrati `undefined`, jeste `done`. Ovo može delovati neprikladno, ali može mnogo pomoći kada vaš običan izraz zahteva veliku računarsku snagu.

Ako je primer mali, kao naš, možete jednostavno pretvoriti iterator u niz pomoću `Array.from()` i odmah dobiti sva poklapanja.

Hajde da izmenimo naš kod tako da koristi ovaj pristup — uklonićemo poziv `replaceAll`, pošto već znamo da radi:

```js
const regex = /freecodecamp/g;
const str = "freecodecamp is the best we love freecodecamp";
const matched = str.matchAll(regex);
console.log(Array.from(matched));
```

Na hatimaye tunapata safu yetu ya mechi:

```js
// [
//   'freecodecamp',
//   index: 0,
//   input: 'freecodecamp is the best we love freecodecamp',
//   groups: undefined
// ]
//
// [
//   'freecodecamp',
//   index: 33,
//   input: 'freecodecamp is the best we love freecodecamp',
//   groups: undefined
// ]
```

Ove moćne metode mogu pomoći pri obradi i izvlačenju podataka iz stringova bez narušavanja performansi ili čitljivosti koda.
