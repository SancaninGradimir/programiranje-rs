# --description--

Middleware je važan koncept u Express, i omogućava nam da obrađujemo zahteve i odgovore između klijenta i servera. Middleware funkcija ima pristup objektu zahteva, objektu odgovora i sledećoj funkciji u ciklusu zahtev-odgovor aplikacije.

Detaljno ćemo analizirati:

## Middleware šta je?

Middleware je u suštini komponenta koja se izvršava tokom ciklusa zahteva ka serveru. Obavlja različite funkcije kao što su snimanje zahteva, rukovanje greškama, analiza dolaznih zahteva ili serviranje statičkih fajlova.

Express Pruža jednostavan način definisanja middleware-a koji obrađuje dolazne zahteve pre nego što stignu do handler-a ruta podataka.

Kako middleware funkcioniše:

Kada se zahtev pošalje programu Express, middleware se izvršava u redosledu kojim je dodato. Protok je sledeći:

* Zahtev se šalje na server.
* Middleware funkcije obrađuju zahtev.
* Ako middleware pozove `next()`, prosleđuje kontrolu sledećem middleware-u ili rukovaocu putanje podataka.
* Ako middleware ne pozove `next()`, ciklus zahtev-odgovor prestaje.

```js
const express = require('express')
const app = express()

// Simple middleware that logs request info
app.use((req, res, next) => {
  console.log('Request URL:', req.url)
  console.log('Request Method:', req.method)
  next() // Passes control to the next middleware
})

app.get('/', (req, res) => {
  res.send('Hello World!')
})

app.listen(3000, () => {
  console.log('Server running on http://localhost:3000')
})
```

`app.use()`: Dodaje middleware uopšteno u vašu aplikaciju.

U ovom primeru middleware beleži informacije o zahtevu i poziva `next()` kako bi izvršavanje prešlo na sledeći middleware ili obrađivač rute.

Middleware je moćna funkcionalnost u Express-u koja ti omogućava da presretneš i izmeniš zahteve u različitim fazama njihove obrade. Middleware možeš koristiti za evidentiranje (logging), autentifikaciju ili čak izmenu objekata zahteva i odgovora pre nego što stignu do obrađivača ruta.
