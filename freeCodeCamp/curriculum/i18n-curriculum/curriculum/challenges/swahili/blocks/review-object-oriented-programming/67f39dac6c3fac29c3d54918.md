## Šta je objektno orijentisano programiranje?

- **Objektno orijentisano programiranje**: Paradigma programiranja u kojoj programeri koriste sve u svom kodu kao stvarni objekat iz sveta. Vrlo je poznato i kao OOP. Četiri glavna principa koji pomažu efikasnom organizovanju i upravljanju kodom su **enkapsulacija svojstava i metoda (encapsulation)**, **nasleđivanje**, **polimorfizam** i **apstrakcija**.
- **Klasa objekata**: Šema za kreiranje objekata. Svaki objekat kreiran iz klase ima atribute koji definišu podatke i metode koje definišu ponašanje tih objekata.

## Šta je instalacija funkcionalnosti i metoda?

- **Enkapsulacija svojstava i metoda (encapsulation)**: Spajanje svojstava i metoda objekta u jednu jedinicu. Omogućava sakrivanje unutrašnjeg stanja objekta iza jednostavnog seta javnih metoda i svojstava koja funkcionišu kao interfejsi. Iza tih interfejsa nalaze se privatna svojstva i metode koje kontrolišu kako podaci menjaju i ko ih može videti.
- **Primer enkapsulacije svojstava i metoda**: Ako želite da pratite saldo novčanika, dozvolićete udeposito i povlačenje novca, ali ne želite da iko god promeni saldo samog novčanika:

```py
class Wallet:
   def __init__(self, balance):
       self.__balance = balance # Private attribute

   def deposit(self, amount):
       if amount > 0:
           self.__balance += amount # Add to the balance safely

   def withdraw(self, amount):
       if 0 < amount <= self.__balance:
           self.__balance -= amount # Remove from the balance safely

account = Wallet(500)
print(account.__balance) # AttributeError: 'Wallet' object has no attribute '__balance'
```

- **Razlika između definisanja prefiksa u jednoj i dve linije za svojstvo**: Definisanje prefiksa u jednoj liniji za svojstvo i metodu znači da je namenjeno samo za unutrašnju upotrebu. Ovo je ugovor, i ne sprečava eksterni pristup svojstvu. Definisanje prefiksa u dve linije za svojstvo i metodu potpuno onemogućava eksterni pristup iz njihove klase.

## Getters i setters šta su?

- **Getteri i setteri**: Metode koje vam omogućavaju da kontrolišete kako se svojstva klase dobijaju i menjaju. Dobijate vrednost pomoću getter-a, a postavljate vrednost pomoću setter-a.
- **Svojstva (Properties)**: Obuhvataju getter-e i settere, omogućavajući pristup podacima. Koriste dodatnu logiku iza fasade kada dobijate, postavljate ili brišete vrednosti.
- **Zašto svojstva umesto metoda**: Svojstva se koriste umesto metoda radi bolje čitljivosti i čistijeg koda. Omogućavaju vam da dobijate vrednosti tačkastom zapisivanjem, kao standardna svojstva, bez običnih zagrada.
- **Kreiranje getter-a**: Da biste kreirali getter, koristite dekorator `@property`. Evo getter-a koja dobija krug kruga:

```py
class Circle:
    def __init__(self, radius):
        self._radius = radius

    @property
    def radius(self): # A getter to get the radius
        return self._radius
  
    @property
    def area(self):  # A getter to calculate area
        return 3.14 * (self._radius ** 2)

my_circle = Circle(3)

print(my_circle.radius) # 3
print(my_circle.area) # 28.26
```

- **Kreiranje setačera**: Da biste kreirali setter koji će biti umotan, morate definisati drugu metodu sa istim imenom i koristiti ``@<property_name>.setter`` na njemu:

```py
class Circle:
    def __init__(self, radius):
        self._radius = radius

    @property
    def radius(self):  # A getter to get the radius
        return self._radius

    @radius.setter
    def radius(self, value):  # A setter to set the radius
        if value <= 0:
            raise ValueError('Radius must be positive')
        self._radius = value

my_circle = Circle(3)
print('Initial radius:', my_circle.radius) # Initial radius: 3

my_circle.radius = 8
print('After modifying the radius:', my_circle.radius) # After modifying the radius: 8
```

- **Kako Python obrađuje getter-e i setter-e**: Kada definišete getter-e i setter-e, Python automatski poziva svaki put kada koristite standardnu sintaksu svojstava ovako:

```py
my_circle.radius # This will call the getter
my_circle.radius = 4 # This will call the setter
```

Kada postavljate vrednost, ne smete da postavite vrednost po samom imenu atributa jer će to izazvati `RecursionError`. Koristite drugačije interno ime, često sa prefiksom od jedne linije, za skladištenje vrednosti.

- **Deleter**: Nakon postavljanja i dobijanja vrednosti za *setter* i *getter*, možete kontrolisati kako se briše za `deleter`. Deleter implementira specifičnu logiku kada koristite deklaraciju `del` na svojstvu. Da biste kreirali deleter, koristite dekorator `@<property_name>.deleter`.

```py
  # Deleter
    @radius.deleter
    def radius(self):
        print("Deleting radius...")
        del self._radius
```

## Šta je nasleđivanje i kako se ponovo koristi kod?

- **Nasleđivanje**: Proces u kojem podklasa koristi atribute i metode roditeljske klase. Nasleđivanje podstiče ponovnu upotrebu koda, pruža jasnu hijerarhiju i menja ponašanje bez ponovnog pisanja svega. Da bi se implementiralo nasleđivanje, podklasa uzima ime roditeljske klase:

```py
class Parent:
    # Parent attributes and methods

class Child(Parent):
    # Child inherits, extends, and/or overrides where necessary
```

- **Direktno i višestrano nasleđe**: Kada dete nasleđuje osobine i putanje od jednog roditelja, kao što vidite gore, taj proces se naziva **direktno nasleđe**. Kada dete nasleđuje osobine i putanje od više od jednog roditelja, to je **višestrano nasleđe**. Sintaksa mu je ovakva:

```py
class Parent:
    # Attributes and methods for Parent

class Child:
    # Attributes and methods for Child

class GrandChild(Parent, Child):
    # GrandChild inherits from both Parent and Child
    # GrandChild can combine or override behavior from each
```

- **Konstruktor `super()`**: Konstruktor koji vam omogućava da pozovete metodu iz roditeljske klase, kada klasa ima drugačiju implementaciju te metode, ili dodaje tu metodu, bez prepisivanja koda.

## Šta je polimorfizam i kako omogućava ponovnu upotrebu koda?

- **Polimorfizam**: Princip OOP koji omogućava različitim klasama da koriste isto ime metode, ali svaka klasa ga implementira drugačije kada se pozove. Evo njegove sintakse:

```py
class A:
   def action(self): ...

class B:
   def action(self): ...

class C:
   def action(self): ...

Class().method()  # Works for A, B, or C
```

- **Polimorfizam zasnovan na nasleđivanje**: Roditelj definiše metodu, a svaka dete klasa je prepisuje za sopstvenu upotrebu.

## Šta je name mangling i kako funkcioniše?

- **Name mangling**: Proces gde Python interno menja ime svojstva koje ima dvolinijski prefiks dodavanjem jedne linije i imena klase kao prefiksa, menjajući `__attribute` u `_ClassName__attribute`.
- **Svrha name mangling-a**: Glavna svrha name mangling-a je sprečiti slučajno prepisanje svojstava i metoda prilikom korišćenja nasleđivanja. Ovde je kod koji to bolje objašnjava:

```py
class Parent:
    def __init__(self):
        self.__data = 'Parent data'

class Child(Parent):
    def __init__(self):
        super().__init__()
        self.__data = 'Child data'

c = Child()
print(c.__dict__) # {'_Parent__data': 'Parent data', '_Child__data': 'Child data'}
```

## Šta je apstrakcija i kako pomaže u organizovanju složenih sistema?

- **Apstrakcija**: Programski koncept u kojem su složene detalje implementacije objekta ili sistema skriveni, a prikazuju se samo ključne karakteristike. U Python i drugim programskim jezicima, apstrakcija pojednostavljuje složene sisteme povećavanjem ponovne upotrebljivosti.
- **Primer apstrakcije**: Dobar primer apstrakcije u svakodnevnom životu je automobil koji vam omogućava da koristite samo točkove, pedale i menjač bez znanja kako motor ili kočnice funkcionišu.
- **Kako Python implementira apstrakciju**: Python vrši apstrakciju putem modula `abc`. Modul pruža klasu `ABC` (baznu apstraktnu klasu) i dekorator `@abstractmethod`. Bazna apstraktna klasa (ABC) definiše metode i atribute koje potomci klase moraju da implementiraju. Ne može se koristiti za direktno kreiranje instanci.
- **Kako se apstraktna metoda definiše**: Apstraktna metoda je definisana sa `@abstractmethod` i mora biti preklopljena u klasama potomcima, čak i ako ima podrazumevanu implementaciju. Osnovna sintaksa apstrakcije je sledeća:

```py
from abc import ABC, abstractmethod

# Define an abstract base class
class AbstractClass(ABC):
    @abstractmethod
    def abstract_method(self):
        pass

# Concrete subclass that implements the abstract method
class ConcreteClassOne(AbstractClass):
    def abstract_method(self):
        print('Implementation in ConcreteClassOne')

# Another concrete subclass
class ConcreteClassTwo(AbstractClass):
    def abstract_method(self):
        print('Implementation in ConcreteClassTwo')
```
