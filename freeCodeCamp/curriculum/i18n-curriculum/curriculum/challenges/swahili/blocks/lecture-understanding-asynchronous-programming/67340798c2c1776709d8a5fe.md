# --description--

Atributi `async` i `defer` u HTML `script` elementima igraju važnu ulogu u načinu na koji se JavaScript datoteke preuzimaju i izvršavaju na veb stranicama. Razumevanje ovoga može poboljšati performanse tvog sajta i korisničko iskustvo. Kada uključiš `script` u HTML datoteku, to izgleda ovako:

```js
<script src="example.js"></script>
```

Kada pregledač naiđe na ovaj tag `script`, prestaje da čita HTML, preuzima skriptu, izvršava je, a zatim nastavlja sa čitanjem HTML. Ovo može usporiti učitavanje vaše veb stranice, posebno ako imate velike skripte.

Ovde su karakteristike `async` i `defer`. One pružaju načine za efikasnije preuzimanje skripte.

Počinjemo sa značajkom `async`:

```js
<script src="example.js" async></script>
```

Dodavanjem atributa `async` elementu `script`, pregledač će nastaviti da obrađuje HTML dok se skripta preuzima. Čim se skripta potpuno preuzme, pregledač će privremeno zaustaviti obradu HTML-a, izvršiti skriptu, a zatim nastaviti sa obradom HTML-a. Ovo može značajno ubrzati učitavanje stranice.

Važno je razumeti da se `async` skripte izvršavaju čim budu preuzete, što znači da se možda neće izvršiti redosledom koji želimo. Tu pomaže atribut `defer`. Evo kako izgleda atribut `defer`:

```js
<script src="example.js" defer></script>
```

Atribut `defer` je sličan atributu `async`. Međutim, `defer` skripte se ne izvršavaju odmah nakon preuzimanja. Umesto toga, čekaju da se obrada HTML-a završi. Pored toga, `defer` skripte se izvršavaju redosledom kojim se pojavljuju u HTML kodu.

Ukratko, koristi `async` za skripte kod kojih redosled izvršavanja nije važan, a `defer` kada želiš da se skripte izvrše tačno određenim redosledom. Oba atributa mogu značajno poboljšati vreme učitavanja stranice tako što omogućavaju pregledaču da nastavi obradu HTML-a dok se skripte preuzimaju u pozadini.
