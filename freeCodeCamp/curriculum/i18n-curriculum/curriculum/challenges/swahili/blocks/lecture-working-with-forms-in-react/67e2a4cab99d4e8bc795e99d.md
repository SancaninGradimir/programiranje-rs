# --description--

React 19 uključuje dve nove komponente, poznate kao server components i server actions.

Od te verzije, komponente servera su bile podrazumevane u okvirima kao što je Next.js, koji to direktno podržavaju.

Server actions s druge strane, je funkcija koja se izvršava na serveru, omogućavajući upravljanje formularima direktno na serveru bez potrebe za endpoint-om API.

Primer server akcije je sledeći:

```js
"use server";

async function submitForm(formData) {
  const name = formData.get("name");
  return { message: `Hello, ${name}!` };
}
```

Ova server akcija čita polje `name` iz forme i vraća string za to ime.

Da bi se pojednostavilo upravljanje stanjem za server akcije i uklonila potreba za JavaScript na strani klijenta sa jednostavnim formularom, tim React je implementirao hook `useActionState` u verziji 19.

Pažljivo pogledajmo ovaj hook i da vidimo kako radi.

Dokumentacija za React opisuje hook `useActionState` kao hook koji omogućava "validiranje stanja na osnovu rezultata akcije forme."

Ali ovo ne znači da možeš koristiti hook `useActionState` samo za forme. Takođe ga možeš koristiti za upravljanje klikovima na dugmad i drugim događajima, pod uslovom da imaš definisanu akciju.

Zapamti, jer je `useActionState` hook, ne možeš ga koristiti unutar server komponente.

Ovo je osnovna sintaksa za hook `useActionState`:

```js
const [state, action, isPending] = useActionState(actionFunction, initialState, permalink);
```

- `state` je trenutno stanje koje vraća akcija.

- `action` je funkcija koja pokreće akciju na serveru.

- `isPending` je booleanska vrednost koja pokazuje da li je akcija u toku ili ne.

- Parametar `actionFunction` je sam server action.

- `initialState` je parametar koji predstavlja početno stanje pre nego što akcija počne.

- `permalink` je opcionalna sekvenca slova koja prikazuje URL jedinstveni ID stranice koju forma menja.

Kada koristite hook za `useActionState`, osigurajte da imate definisanu akciju pre toga. Uzmićemo akciju iz prethodnog primera za ovo, sa malim izmenama:

```js
"use server";

export async function submitForm(_, formData) {
  const name = formData.get("name");

  const hour = new Date().getHours();
  let greeting;

  if (hour < 12) {
    greeting = "Good morning";
  } else if (hour < 18) {
    greeting = "Good afternoon";
  } else {
    greeting = "Good evening";
  }

  return { message: `${greeting}, ${name}` };
}
```

U vašem delu, morate uključiti hook za `useActionState` i pozvati viši nivo tela sekcije (pre deklaracije return) kao druge hookove. Takođe, morate uključiti akciju:

```jsx
"use client";

// Import the useActionState hook
import { useActionState } from "react";

// Import the submitForm action
import { submitForm } from "./actions/submitForm";

const Greeter = () => {

 // Initialize the hook
 const [state, submit, isPending] = useActionState(submitForm, {
   message: "",
 });

  return (
    <div className="flex flex-col items-center justify-center min-h-screen bg-gray-100 p-6">
      {/* Rest of component */}
    </div>
  );
};

export default Greeter;
```

Evo kompletnog koda sa malom dekoracijom/formatiranjem:

```jsx
"use client";

import { useActionState } from "react";
import { submitForm } from "./actions/submitForm";

const Greeter = () => {
  const [state, submit, isPending] = useActionState(submitForm, {
    message: "",
  });

  return (
    <div className="flex flex-col items-center justify-center min-h-screen bg-gray-100 p-6">
      <form
        action={submit}
        className="bg-white p-6 rounded-2xl shadow-md w-full max-w-md"
      >
        <h2 className="text-2xl text-center font-semibold text-gray-700 mb-4">
          Greet Someone
        </h2>

        <input
          type="text"
          name="name"
          placeholder="Enter your name"
          required
          className="w-full p-3 border border-gray-300 rounded-lg focus:outline-none focus:ring-2 focus:ring-green-400"
        />

        <button
          type="submit"
          disabled={isPending}
          className="w-full mt-4 p-3 bg-green-500 text-white font-semibold rounded-lg hover:bg-green-600 disabled:bg-gray-400 transition-all"
        >
          {isPending ? "Greeting..." : "Greet"}
        </button>

        {state.message && (
          <p className="mt-4 text-green-600 text-center font-medium">
            {state.message}
          </p>
        )}
      </form>
    </div>
  );
};

export default Greeter;
```

U pregledaču ćeš videti da se dugme obrasca menja iz `Greet` u `Greeting...` dok je akcija `isPending`, a pozdrav će prikazivati `Good morning, {name}`, `Good afternoon, {name}` ili `Good evening, {name}`, u zavisnosti od doba dana kada je obrazac poslat.

Zapamti da smo rekli da možeš takođe koristiti hook `useActionState` van forme?

U ovom primeru, uzimaćemo pet korisnika sa JSONPlaceholder-a klikom na dugme:

```js
"use server";

export async function getUsers() {
  const res = await fetch(
    "https://jsonplaceholder.typicode.com/users?_start=0&_limit=5/"
  );
  return await res.json();
}
```

Ovo je stilizovani UI:

```jsx
"use client";

import { useActionState } from "react";
import { getUsers } from "./actions/getUsers";

export default function FetchUsers() {
  const [users, fetchAction, isPending] = useActionState(getUsers, []);

  return (
    <div className="p-6 max-w-lg mx-auto">
      <button
        onClick={fetchAction}
        disabled={isPending}
        className="px-4 py-2 cursor-pointer bg-green-500 text-white rounded-lg hover:bg-green-600 disabled:bg-gray-400 font-bold"
      >
        {isPending ? "Fetching Users..." : "Fetch Users"}
      </button>

      <ul className="mt-4 space-y-2">
        {users.map((user) => (
          <li key={user.id} className="p-3 bg-gray-100 rounded-lg">
            <p className="font-semibold">{user.name}</p>
            <p className="text-sm text-gray-600">{user.email}</p>
          </li>
        ))}
      </ul>
    </div>
  );
}
```

U pretraživaču ćete videti da tekst dugmeta nije promenjen u `Fetching Users...` nakon klika.

Ovo se dešava zato što React koristi prijem podataka i prikazuje ga kao viši prioritet u odnosu na stanje `isPending`, što blokira `isPending` i uzrokuje grešku.

Da biste rešili ovaj problem, morate pokriti akciju unutar `startTransition`:

```jsx
"use client";

// import startTransition from React
import { useActionState, startTransition } from "react";
import { getUsers } from "./actions/getUsers";

export default function FetchUsers() {
  const [users, fetchAction, isPending] = useActionState(getUsers, []);

  return (
    <div className="p-6 max-w-lg mx-auto">
      <button
        {/* wrap fetchAction in startTransition */}
        onClick={() => startTransition(() => fetchAction())}
        disabled={isPending}
        className="px-4 py-2 bg-green-500 font-bold cursor-pointer text-white rounded-lg hover:bg-green-600 disabled:bg-gray-400"
      >
        {isPending ? 'Fetching Users...' : 'Fetch Users'}
      </button>

      <ul className="mt-4 space-y-2">
        {users.map((user) => (
          <li key={user.id} className="p-3 bg-gray-100 rounded-lg">
            <p className="font-semibold">{user.name}</p>
            <p className="text-sm text-gray-600">{user.email}</p>
          </li>
        ))}
      </ul>
    </div>
  );
}
```

Ako se pitate šta je `startTransition`, to je funkcija koja govori React da je ažuriranje stanja niog prioriteta i da može biti prekinuto. Ovo omogućava UI-u da ostane funkcionalan prilikom obrade asinhronih ažuriranja kao što su server akcije.

Tako koristiš hook za `useActionState`, unutar i van forme.

Zato što komponente servera ne podržavaju upravljanje stanjem, pa `useActionState` ne funkcioniše.

Hooks rade samo u specifičnom tipu delova/sekcija.
