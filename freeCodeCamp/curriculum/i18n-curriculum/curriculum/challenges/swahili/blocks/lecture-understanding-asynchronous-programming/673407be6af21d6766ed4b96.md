# --description--

U prethodnoj lekciji, pogledali smo šta je API od Fetch i kako ga koristiti. U ovoj lekciji, razgovaraćemo o metodama za HTTP za `GET`, `POST`, `PUT`, i `DELETE` za API od Fetch.

Počnimo sa najčešće korišćenim HTTP metodom, a to je `GET`. On se koristi za preuzimanje podataka sa servera. Kada koristiš `fetch()` bez navođenja metode, podrazumevano se koristi `GET`.

```js
fetch('https://api.example.com/data')
```

U ovom kodu, šaljemo zahtev od `GET` ka `https://api.example.com/data`. Sada, molimo vas da znate da ne možete koristiti ove podatke direktno; morate odgovor pretvoriti u format JSON. Tek nakon toga ga možete koristiti bilo gde u svom projektu. Evo primera kako to učiniti:

```js
fetch('https://api.example.com/data')
  .then(response => response.json())
  .then(data => console.log(data))
```

U ovom kodu odgovor dobijen iz Fetch API-ja je obećanje (promise), a koristimo obrađivač `.then` da odgovor pretvorimo u JSON format. U prethodnoj lekciji naučili ste da je `Promise` objekat koji predstavlja konačni završetak (ili neuspeh) asinhronog procesa i njegovu rezultatnu vrednost.

Vrednost obećanja nije poznata u trenutku kada se ono kreira. Postaje poznata tek kada se asinhroni proces završi. Kada povežemo dva `.then` obrađivača na `fetch` poziv, to se naziva ulančavanje obećanja (promise chaining), o čemu ćete učiti u sledećoj lekciji.

Do sada smo preuzimali resurse sa servera. Ali, da li ste znali da možemo i da šaljemo podatke serveru? Metoda `POST` koristi se za slanje podataka serveru radi kreiranja resursa. Evo primera `POST` zahteva koji se koristi za kreiranje resursa na serveru:

```js
fetch('https://api.example.com/users', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    name: 'John Doe',
    email: 'john@example.com'
  })
})
```

U ovom primeru, šaljemo zahtev za `POST` kreiranje novog korisnika. Specifikujemo metodu kao `POST`, postavljamo odgovarajuća zaglavlja i uključujemo `body` sa podacima koje želimo poslati. `body` zahteva da bude niz znakova, pa koristimo `JSON.stringify()` da konvertujemo naš objekat u niz znakova tipa JSON.

Metoda `PUT` koristi se za ažuriranje postojećih resursa na serveru. Evo primera:

```js
fetch('https://api.example.com/users/45', {
  method: 'PUT',
  headers: {
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    name: 'John Smith',
    email: 'john@example.com'
  })
})
```

U ovom primeru pažljivo pogledaj URL. Na njegovom kraju nalazi se `45`. To se obično koristi kao jedinstveni identifikator podataka koje želimo da ažuriramo. U kodu koristimo metodu `PUT`, a podatke koje želimo da ažuriramo prosleđujemo kroz `body`.

Metoda `DELETE` koristi se za brisanje resursa sa servera. Evo primera:

```js
fetch('https://api.example.com/users/45', {
  method: 'DELETE',
})
```

U ovom kodu koristimo metodu `DELETE` i identifikator na kraju URL-a kako bismo odredili koji resurs treba obrisati.
