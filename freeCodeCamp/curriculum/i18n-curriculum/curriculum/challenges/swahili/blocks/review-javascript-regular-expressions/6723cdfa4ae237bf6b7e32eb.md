# --interactive--

## Uobičajena upotreba i uobičajene metode

- **Definicija**: Regularni izrazi, ili Regex, koriste se za kreiranje „šablona“ (ili „obrazca“), koji se zatim može koristiti za proveru sekvenci karaktera, ekstrakciju teksta i više.

```js
const regex = /freeCodeCamp/;
```

- **Funkcija za `test()`**: Ova funkcija prima niz znakova, koji je sekvenca znakova koja se testira na poređenje sa regularnim izrazom. Ova funkcija će vratiti booleanski vrednost (tačno ili netačno) ako string odgovara regularnom izrazu.

:::interactive_editor

```js
const regex = /freeCodeCamp/;
const test = regex.test("e");
console.log(test); // false
```

:::

- **Funkcija `match()`**: Ova funkcija prima regularni izraz, ali može da primi i niz karaktera koji će biti formiran u regularni izraz. Funkcija `match` vraća niz rezultata podudarnosti za niz karaktera.

```js
const regex = /freeCodeCamp/;
const match = "freeCodeCamp".match(regex);
console.log(match); // ['freeCodeCamp', index: 0, input: 'freeCodeCamp', groups: undefined]
```

- **Funkcija `replace()`**: Ova funkcija prima dva argumenta: regularni izraz (ili niz znakova), i zamenski niz (ili funkciju koja se izvršava za svako poklapanje).

:::interactive_editor

```js
const regex = /Jessica/;
const str = "Jessica is rly kewl";
const replaced = str.replace(regex, "freeCodeCamp");
console.log(replaced); // "freeCodeCamp is rly kewl"
```

:::

- **Putanja za `replaceAll`**: Ova putanja se koristi za zamenu svih navedenih primera događaja sa novom sekvencijom karaktera. Ova putanja će vratiti grešku ako pružite regularni izraz bez globalnog zastavika.

:::interactive_editor

```js
const text = "I hate JavaScript! I hate programming!";
const newText = text.replaceAll("hate", "love");
console.log(newText);  // "I love JavaScript! I love programming!"
```

:::

- **Njia ya `matchAll`**: Ova putanja se koristi za dobijanje svih rezultata regularnog izraza definisanog nizom karaktera, uključujući i grupe za hvatanje (capturing groups), i vraćanjem ih kao atributa. Atribut je nešto što mu omogućava da prođe preko kolekcije stavki.

:::interactive_editor

```js
const str = "JavaScript, Python, JavaScript, Swift, JavaScript";
const regex = /JavaScript/g;

const iterator = str.matchAll(regex);

for (let match of iterator) {
  console.log(match[0]); // "JavaScript" for each match
}
```

:::

## Mehanizam za standardni govor

- **Definicija**: Modifikator, poznat i kao „flag“ (zastavica), menja ponašanje regularnog izraza.
- **Flag za `i`**: Ovaj flag čini da regex ignoriše razlike u slovima.

:::interactive_editor

```js
const regex = /freeCodeCamp/i;
console.log(regex.test("freecodecamp")); // true
console.log(regex.test("FREECODECAMP")); // true
```

:::

- **Bendera ya `g`**: Ova zastava, ili globalni limitator, dozvoljava vam normalnu upotrebu za više od jednog poređenja.

:::interactive_editor

```js
const regex = /freeCodeCamp/gi;
console.log(regex.test("freeCodeCamp")); // true
console.log(regex.test("freeCodeCamp is great")); // false
```

:::

- **Definicija anchor taga**: Anchor tag `^`, na početku običnog teksta, kaže "uporedi početak sekvence slova". Anchor tag `$`, na kraju običnog teksta, kaže "uporedi kraj sekvence slova".

:::interactive_editor

```js
const start = /^freeCodeCamp/i;
const end = /freeCodeCamp$/i;
console.log(start.test("freecodecamp")); // true
console.log(end.test("freecodecamp")); // true
```

:::

- **Flaga za `m`**: Anchor tagovi traže početak i kraj celog niza znakova. Ali možete koristiti regex da obradite više linija sa flagom `m`, ili multi-linijskim modifikatorom.

:::interactive_editor

```js
const start = /^freecodecamp/im;
const end = /freecodecamp$/im;
const str = `I love 
freecodecamp
it's my favorite
`;
console.log(start.test(str)); // true
console.log(end.test(str)); // true
```

:::

- **Bendera za `d`**: Ova zastava dodaje informacije koje dobijate u objekt rezultata poređenja.

```js
const regex = /freecodecamp/di;
const string = "we love freecodecamp isn't freecodecamp great?";
console.log(string.match(regex));
```

Rezultati su:

```js
// [
//   'freecodecamp',
//   index: 8,
//   input: "we love freecodecamp isn't freecodecamp great?",
//   groups: undefined,
//   indices: [
//     0: [8, 20],
//     groups: undefined
//   ]
// ]
```

- **Zastavica `u`**: Ova zastavica proširuje mogućnosti regularnih izraza omogućavajući podudaranje posebnih Unicode znakova. Zastavica `u` daje pristup posebnim klasama kao što je `Extended_Pictographic` za podudaranje većeg broja emodžija. Postoji i zastavica `v`, koja dodatno proširuje mogućnosti Unicode podudaranja.
- **Bendera za `y`**: Sticky zastavica radi kao globalna zastavica, ali sa manjim razlikama. Glavna razlika je u tome što će globalni regularni izraz početi od `lastIndex` i tražiti preostali deo sekvence za drugo podudaranje, dok će sticky regularni izraz resetovati null i postaviti `lastIndex` na 0 ako nema trenutnog podudaranja na prethodnom `lastIndex`.
- **Bendera za `s`**: Single-line zastavica omogućava wildcard karakter, predstavljen sa `.` u regexu, koji podudara linijske prekidi – što znači da se sekvenca tretira kao jedna linija teksta.

## Razredi slova

- **Wildcard `.`**: Klase znakova su posebna sintaksa koju možete koristiti za poređenje skupa ili podskupova znaka. Prvi klasa znakova koju treba da naučite je wildcard klasa. Wildcard se predstavlja tačkom, ili *dot*, i odgovara BILO KOJOM znaku IZVAN razmaka linija. Da biste dozvolili wildkard klasi da upoređuje razmake linija, zapamtite da morate koristiti flag `s`.

```js
const regex = /a./;
```

- **`\d`**: Ovo će uporediti sve cifre (`0-9`) u nizu slova.

```js
const regex = /\d/;
```

- **`\w`**: Ovo se koristi za poređenje bilo kog slova reči (`a-z0-9_`) u sekvenci znakova. Slovo reči je definisano kao bilo koji slova, od 'a' do 'z', ili broj od 0 do 9, ili znak poddonje crte (underscore).

```js
const regex = /\w/;
```

- **`\s`**: Klasa belog razmaka `\s`, predstavljena povratnikom praćenim `s`. Ova klasa karaktera će poklopiti bilo koji beli razmak, uključujući nove linije, razmake, tabulator i specifične unicode znakove za razmak.
- **Negiranje specifičnih klasa karaktera**: Da biste negirali jednu od ovih klasa karaktera, umesto korišćenja malog slova nakon povratnika, možete koristiti njegov ekvivalent velikim slovom. Sledeći primer ne odgovara cifri. Umesto toga, on odgovara bilo kom pojedinačnom karakteru koji NIJE cifra.

```js
const regex = /\D/;
```

- **Specijalne klase znakova**: Možete kreirati specijalne klase znakova tako što ćete znak koji želite da uparite staviti unutar uglatih zagrada.

```js
const regex = /[abcdf]/;
```

## Verifikacija prednjeg i zadnjeg pogleda

- **Objašnjenje**: Provera naprednog i nazad konteksta omogućava vam da uporedite specifične obrasce na osnovu prisustva ili odsustva okolnih obrazaca.
- **Pozitivna provera naprednog konteksta**: Ova provera će uporediti obrazac kada ga prati drugi obrazac. Da biste kreirali pozitivan prednji kontekst, morate početi sa obrascem koji želite da uporedite. Zatim, koristite obične zagrade za obuhvatanje obrasca koji želite da koristite kao vaš uslov. Nakon zatvarajućih zagrada, koristite `?=` da definišete taj obrazac kao pozitivan prednji kontekst.

```js
const regex = /free(?=code)/i;
```

- **Negativna provera unapred (Negative Lookahead)**: Ovo je vrsta uslova koja se koristi u regularnom izrazu za proveru da li određeni obrazac nije nastao ranije u nizu karaktera.

```js
const regex = /free(?!code)/i;
```

- **Pozitivna potvrda pozadinskog nalaza**: Ova potvrda će porediti model samo ako je vođena drugim specifičnim modelom, bez uključivanja modela koji su prethodili u poređenju.

```js
const regex = /(?<=free)code/i;
```

- **Negativna provera nazad pretraživanja**: Ova verifikacija osigurava da šablon nije poklopljen sa drugim specifičnim šablonom. Upoređuje samo da li pomenuti šablon nije poklopljen jednom sa dato nizom, bez uključivanja sekvence koja je prethodila tokom poređenja.

```js
const regex = /(?<!free)code/i;
```

## Motor za regex

- **Definicija**: Kvantifikator u regularnom izrazu određuje koliko puta bi se obrazac (ili deo obrazca) trebalo pojaviti. Pomaže u kontrolisanju broja pojavljivanja karaktera ili grupa tokom poklapanja. Sledeći primer koristi se za podudaranje karaktera koji je pojavio tačno četiri puta.

```js
const regex = /^\d{4}$/;
```

- **`*`** : Poklapa se sa 0 ili više događaja prethodnog elementa.
- **`+`**: Poklapa se sa 1 ili više događaja prethodnog elementa.
- **`?`**: Poklapa se sa događajem 0 ili 1 prethodnog elementa.
- **`{n}`**: Poklapa se sa tačno n događaja prethodnog elementa.
- **`{n,}`**: Poklapa se sa n ili više događaja prethodnog elementa.
- **`{n,m}`**: Poklapa se između događaja n i m prethodnog elementa.

## Grupe za hvatanje i povratne reference

- **Grupe za hvatanje**: Grupa za hvatanje vam omogućava da „uhvatite“ deo poklapanja sekvence kako biste ga mogli koristiti na bilo koji način koji vam je potreban. Grupe za hvatanje su definisane običnim zagradama koje sadrže uzorak za hvatanje, bez vodećih karaktera kao što je prefiks.

```js
const regex = /free(code)camp/i;
```

- **Nekaptureći grupe**: Nekaptureći grup je isto što i hvatačka grupa, ali ne čuva deo poklopljenog niza za kasniju upotrebu. Nekaptureći grupe su definisane sa `(?:...)`.

```js
const regex = /free(?:code)camp/i;
```

- **Nazadna referenca (Backreference)**: Nazadna referenca u regularnom izrazu odnosi se na način ponovnog korišćenja dela obrasca koje je prethodno pronađeno u istom izrazu. Omogućava vam da se vratite na uhvaćenu grupu (deo obrasca u zagradi) po njenom broju. Na primer, `$1` se odnosi na prvu uhvaćenu grupu.

:::interactive_editor

```js
const regex = /free(co+de)camp/i;
console.log("freecoooooooodecamp".replace(regex, "paid$1world"));
```

:::

- Možete koristiti backreference unutar samog regex-a da uparite isti tekst koji je uhvaćen prethodnom grupom, koristeći *backslash* i broj uhvaćene grupe. Primer:

:::interactive_editor

```js
const regex = /(hello) \1/i;
console.log(regex.test("hello hello")); // true
console.log(regex.test("hello world")); // false
```

:::
